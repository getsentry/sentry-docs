title: Setting Up a New SDK Repo
---

## Steps

#### 1. Repository scaffolding

- Create the repository in the `getsentry` org
- Configure branch protection rules:
  - Require CI checks to pass ([Required CI checks baseline](../standards/review-ci#required-ci-checks-baseline))
  - Require at least one approving review ([Required reviewers](../standards/review-ci#required-reviewers))
- Set up a `CODEOWNERS` file with team members

#### 2. Required files

- `AGENTS.md` ([AGENTS.md](../standards/repository-docs#agentsmd)) — Use the [AGENTS.md template](../templates/agents-md-template). **Skill: `sentry-skills:agents-md`** can generate initial content.
- `CONTRIBUTING.md` ([CONTRIBUTING.md](../standards/repository-docs#contributingmd)) — Use the [CONTRIBUTING.md template](../templates/contributing-md-template), customized for this SDK.
- `.github/PULL_REQUEST_TEMPLATE.md` ([PR template](../standards/repository-docs#pr-template)) — Use the [PR template](../templates/pr-template).
- `CHANGELOG.md` or equivalent
- `LICENSE` file

#### 3. CI pipeline

Set up CI with at minimum:
- Build verification
- Linting ([Linting and formatting in CI](../standards/code-quality#linting-and-formatting-in-ci))
- Type checking, if applicable for the language ([Type checking in CI](../standards/code-quality#type-checking-in-ci))
- Test suite
- Commit message format check ([Commit message format](../standards/code-submission#commit-message-format))
- Secret scanning ([Security practices](../standards/code-quality#security-practices))

#### 4. AI context files

([AI context file maintenance](../standards/repository-docs#ai-context-file-maintenance))
- `AGENTS.md` — required
- `CLAUDE.md`, `.cursorrules`, `.github/copilot-instructions.md` — recommended

#### 5. Release setup

- Craft configuration (`.craft.yml`)
- Version bump script (`scripts/bump-version.sh`)
- Release workflow (`.github/workflows/release.yml`)

#### 6. Rollback documentation

([Rollback procedures](../standards/coordination-maintenance#rollback-procedures))

Document:
- Who can publish and yank releases (at least two people)
- How to yank a release from the package registry
- Backup contact if the primary publisher is unavailable

#### 7. Verify with an AI tool

Clone the repository fresh. Point an AI coding tool at it with no additional context. Ask it to make a sample contribution. Verify that:
- `AGENTS.md` gives the tool enough context to understand the architecture
- The tool can figure out how to run tests
- The tool creates a properly formatted commit
- The tool opens a draft PR with a meaningful description

If the tool can't do this, the documentation needs work.
