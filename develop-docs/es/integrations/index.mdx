<div id="integration-backup-and-restore-scripts">
  ## Scripts de respaldo y restauración de integraciones
</div>

<div id="overview">
  ### Descripción general
</div>

Cuando trabajas en el desarrollo de integraciones de forma local, tu base de datos contiene configuraciones importantes que permiten que las integraciones funcionen correctamente. Si ejecutas `make reset-db` o necesitas eliminar tu entorno local, perderás toda esta configuración y tendrás que volver a configurar las integraciones desde cero.
Hay dos scripts que te ayudan a hacer copias de seguridad y a restaurar la configuración y los datos de configuración de tu integración local de Sentry.

Estos scripts te permiten:

- **Copia de seguridad**: Guardar el estado actual de tu integración en un archivo JSON
- **Restaurar**: Cargar nuevamente el estado de la integración en una base de datos limpia

<div id="what-data-is-backed-up">
  ### Qué datos se respaldan
</div>

Los scripts gestionan los siguientes modelos de Sentry en el orden de dependencias correcto:

- `IdentityProvider` - Configuraciones del proveedor de autenticación
- `Integration` - Instancias y configuraciones de integraciones  
- `Identity` - Asignaciones de identidades de usuario
- `OrganizationIntegration` - Configuraciones de integraciones específicas de la organización

<div id="prerequisites">
  ### Requisitos previos
</div>

- Entorno de desarrollo de Sentry configurado en local
- Entorno de Python con las dependencias de Sentry instaladas
- Acceso a tu base de datos local de Sentry

<div id="script-files">
  ### Archivos de script
</div>

Hay dos scripts (ubicados en `sentry` y `getsentry`):

- `save_integration_data` - Realiza una copia de seguridad de los datos de la integración
- `load_integration_data` - Restaura los datos de la integración

<div id="usage-instructions">
  ### Instrucciones de uso
</div>

<div id="step-1-save-your-integration-data">
  #### Paso 1: Guarda los datos de tu integración
</div>

Antes de ejecutar `make reset-db` o realizar cualquier cambio destructivo, haz una copia de seguridad del estado actual de tu integración:

```bash
# Navega al directorio de tu proyecto Sentry
cd /path/to/sentry # o /path/to/getsentry

# Ejecuta el script de guardado
bin/save_integration_data --output-file integration_backup.json
```


<div id="step-2-restore-your-integration-data">
  #### Paso 2: Restaura los datos de tu integración
</div>

Después de restablecer y preparar tu base de datos, restaura la configuración de la integración:

```bash
# Restauración básica (conserva los IDs de las organizaciones originales)
bin/load_integration_data --input-file integration_backup.json
```

**O, si necesitas cambiar el ID de la organización:**

```bash
# Restaurar y actualizar el ID de organización para objetos OrganizationIntegration
bin/load_integration_data --input-file integration_backup.json --org-id 123
```

Tras la restauración, se recuperarán todos los datos de la integración y podrás volver a usarla para el desarrollo local.
