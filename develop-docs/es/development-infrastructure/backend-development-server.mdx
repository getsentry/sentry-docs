(*consulta también: [Servidor de desarrollo de frontend](/es/frontend/development-server/)*)

Es posible ejecutar Sentry localmente sin tener que compilar el frontend. Para ello, debes omitir el servidor de desarrollo. En su lugar, configura:

```python
# ~/.sentry/sentry.conf.py
STATIC_URL = "https://sentry.io/_static/{version}"
STATIC_FRONTEND_APP_URL = "https://sentry.io/_static/dist/"
```

...y ejecuta:

```shell
sentry run web
```

Cuando visites `localhost:9001`, el navegador cargará JavaScript desde producción en lugar de tu carpeta estática local.

Esto, en la práctica, no hará nada salvo activar los web workers. A partir de ahora, tendrás que hacerlo manualmente:

* Iniciar los consumidores de Kafka pertinentes si los necesitas (p. ej., para la ingesta). Esto depende mucho del conjunto de datos en el que estés trabajando.
* Iniciar los workers de Celery usando `sentry run worker` para ejecutar la mayoría de los trabajos en segundo plano.

Usa `sentry run --help` para ver qué podrías estar pasando por alto al no ejecutar el devserver.

Para ejecutar el servidor de desarrollo con ngrok, consulta [Desarrollo con Ngrok](/es/development/ngrok/).
