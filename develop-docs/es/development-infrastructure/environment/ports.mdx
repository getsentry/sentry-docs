La siguiente es una lista no exhaustiva de puertos usados por servicios de Sentry o por cualquier dependencia de un servicio de Sentry en entornos de desarrollo. Tiene dos propósitos:

* Averiguar por qué se está usando un puerto en tu máquina de trabajo y qué proceso finalizar para liberarlo.
* Averiguar qué puertos probablemente sean seguros para asignar a nuevos servicios.

| Puerto  | Servicio                                  | Descripción
|:--------|:------------------------------------------|:-----------

| 9009    | Clickhouse                                | <Link to="/development/devservices/">Devservice</Link> `clickhouse`. Base de datos para Snuba.
| 8123    | Clickhouse                                |
| 9009    | Clickhouse                                |

| 1218    | [Snuba][]                                 | <Link to="/development/devservices/">Devservice</Link> `snuba`. Para buscar eventos.
| 1219    | [Snuba][] admin                           | No forma parte de la configuración de devservices, pero al ejecutar `snuba admin` escuchará en este puerto.
| 9092    | Kafka                                     | <Link to="/development/devservices/">Devservice</Link> `kafka`.
| 9093    | Kafka                                     |
| 6379    | Redis                                     | <Link to="/development/devservices/">Devservice</Link> `redis` (o quizás instalado con Homebrew en entornos más antiguos), responsable de cachés, ProjectConfigs de Relay y colas de Celery

| 8010    | uWSGI (region silo)                       | Se inicia/detiene con `sentry devserver --silo=region`. Sirve la app/API de Django. Webpack en 8000 hace reverse proxy a este servidor.

| 9000    | `sentry run web`                          | Puerto predeterminado heredado para `sentry run web`, cambiado a 9001 para evitar conflicto con Clickhouse.
| 9001    | `sentry run web`                          | Frontend básico sin Webpack ni Relay. <Link to="/development/environment/">Sentry Dev</Link> probablemente sea mejor. **En conflicto con el servidor de la guía de estilos de Sentry Dev.**

<div id="finding-out-whats-running-on-your-machine">
  ## Cómo saber qué se está ejecutando en tu máquina
</div>

* Usa `lsof -nP -i4 | grep LISTEN` para encontrar los puertos ocupados en macOS.
* El panel de Docker for Mac muestra los contenedores/servicios de desarrollo en ejecución, junto con los puertos asignados y la opción de iniciarlos o detenerlos.
