(*consulta también: [Servidor de desarrollo de backend](/es/backend/development-server/)*)

Puedes ejecutar únicamente un servidor de desarrollo de frontend que reenviará todas las solicitudes de API a los servidores de producción de Sentry. Para hacerlo, ejecuta el siguiente comando:

```shell
pnpm dev-ui
```

El servidor de desarrollo está disponible en [https://dev.getsentry.net:7999](https://dev.getsentry.net:7999). `dev.getsentry.net` es un alias de `localhost` y se prefiere porque admite subdominios. También puedes crear tu propio alias de dominio, como con ngrok, si lo deseas.

En cualquier caso, ahora puedes iniciar sesión en tu servidor de desarrollo con tus credenciales de Sentry.io. El flujo de inicio de sesión con SSO *NO* funcionará. En desarrollo, el formulario de inicio de sesión solo admite correo electrónico y contraseña.

Aunque el flujo de inicio de sesión con SSO no funcionará, las cookies de sesiones ya iniciadas funcionarán correctamente. Puedes usar nuestra extensión de navegador [Cookie Sync](https://github.com/getsentry/cookie-sync) para sincronizar las cookies de sesión de `*.sentry.io` a `*.dev.getsentry.net` y `*.sentry.dev` —esta última es el entorno de vista previa de GitHub alojado en Vercel.

* [Obtener Cookie Sync para Chrome](https://chrome.google.com/webstore/detail/sentry-cookie-sync/kchlmkcdfohlmobgojmipoppgpedhijh)
* [Obtener Cookie Sync para Firefox](https://addons.mozilla.org/en-US/firefox/addon/sentry-cookie-sync/)

### &quot;Tu conexión no es privada&quot;

Si ves un error similar al siguiente al acceder al servidor de desarrollo:

![your connection is not private example](../img/connectionNotPrivate.png)

Puedes conceder una excepción temporal en tu navegador o crear e instalar un certificado local y usar tu sistema operativo para marcarlo como &quot;de confianza&quot;. TLS es obligatorio para el servicio de desarrollo del frontend, ya que la API de producción se sirve a través de HTTPS.

Sigue los pasos [para habilitar HTTPS en tu servidor de desarrollo](/es/development/environment/#enabling-https).

Ahora puedes ejecutar el servidor de desarrollo con `pnpm dev-ui` y abrir [https://localhost:7999](https://localhost:7999). No deberías ver una advertencia indicando que tu conexión no es privada. También deberías ver un candado o un icono similar en la barra de direcciones de tu navegador.

![lock icon in address bar](../img/addressBarLockIcon.png)

**NOTA**: Para quienes usan Firefox con contraseña maestra, se te pedirá con este mensaje: &quot;Enter Password or Pin for &quot;NSS Certificate DB&quot;:&quot;

**NOTA**: Webpack muestra la IP desde donde se sirve la SPA (p. ej., `https://192.168.0.200:7999/`). No la uses o seguirás recibiendo el mensaje anterior. Los certificados locales generados son solo para `localhost`, `127.0.0.1` y `dev.getsentry.net`. Cualquiera de estos valores funcionará.
