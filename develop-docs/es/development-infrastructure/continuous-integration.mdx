  ## Despliegues

[GoCD](https://deploy.getsentry.net) se usa para los despliegues. Consulta la [guía interna del usuario](https://www.notion.so/sentry/GoCD-User-Guide-4f8456d2477c458095c4aa0e67fc38a6).

<div id="github-actions">
  ## GitHub Actions
</div>

GitHub Actions es nuestro sistema principal de CI y ejecuta nuestras pruebas en cada pull request y en las integraciones a master. Es obligatorio que las pruebas aprueben antes de poder integrar los cambios.

<div id="docker-images">
  ## Imágenes de Docker
</div>

Principalmente usamos [una acción compuesta de GitHub](https://github.com/getsentry/action-build-and-push-images) para crear y publicar imágenes. Esto garantiza que los flujos de trabajo para crear y publicar imágenes estén estandarizados y que las imágenes incluyan la metainformación adecuada. El estándar para la gestión de artefactos está disponible [aquí](https://www.notion.so/sentry/Standard-Spec-Artifact-Management-22a8b10e4b5d80ecb6dcca3eb4558f5b).

Usamos GHCR (GitHub Container Registry) para imágenes que se utilizan en CI, implementaciones autogestionadas y desarrollo local. GAR (Google Artifact Registry) se utiliza para imágenes que se usan en producción.

<div id="fast-revert">
  ## Reversión rápida
</div>

Para facilitar cambios rápidos que desbloqueen fallos en las ramas predeterminadas, muchos de nuestros repositorios cuentan con un flujo de trabajo de reversión rápida. Para usarlo, los PR pueden revertirse automáticamente en cuanto se aplique la etiqueta `Trigger: Revert`. Más detalles [aquí](https://www.notion.so/sentry/Standard-Spec-Fast-Revert-2388b10e4b5d8019bc98d863703d1b17?pvs=25)
