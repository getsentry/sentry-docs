A partir de la versión 24.8.0+, los usuarios podrán elegir entre dos tipos distintos de implementaciones de Sentry autohospedado.

**Solo errores**

La opción autohospedada “Solo errores” ofrece un conjunto reducido de funciones con énfasis en minimizar los recursos del sistema. Esta alternativa ligera incluye únicamente las siguientes funciones:

1. Issues
2. Alerts
3. Integrations
4. Dashboards
5. Releases
6. Discover

Para habilitar el autohospedado “Solo errores”, debes actualizar tu [.env file](https://github.com/getsentry/self-hosted/blob/master/.env) para incluir `COMPOSE_PROFILES=errors-only`.

**Funciones completas**

Esta es nuestra versión predeterminada de Sentry autohospedado. Incluye la mayoría de las funciones disponibles en nuestro producto SaaS. Esto abarca todo lo que ofrece la versión “Solo errores”, más lo siguiente:

1. [Traces](https://docs.sentry.io/product/explore/traces/)
2. [Profiles](https://docs.sentry.io/product/explore/profiling/)
3. [Replays](https://docs.sentry.io/product/explore/session-replay/)
4. [Insights](https://docs.sentry.io/product/insights/) (solicitudes, consultas, recursos, etc.)
5. [User Feedback](https://docs.sentry.io/product/user-feedback/)
6. [Insights](https://docs.sentry.io/product/insights/)
7. [Crons](https://docs.sentry.io/product/crons/)
8. [Metrics](https://docs.sentry.io/product/explore/metrics/)

Esta versión de Sentry se habilita de forma predeterminada al instalar. Asegúrate de que tu [.env file](https://github.com/getsentry/self-hosted/blob/master/.env) incluya `COMPOSE_PROFILES=feature-complete`.
