<div id="outbound-email">
  ## Correo saliente
</div>

Se recomienda usar un servidor SMTP externo debido a varios requisitos de remitente implementados por los principales proveedores de correo. Sin embargo, si estás probando o ejecutando una instancia autogestionada de Sentry en una red local, puedes usar el servidor SMTP integrado, basado en [exim4](https://hub.docker.com/r/tianon/exim4). La configuración predeterminada está preparada para usar este servidor. Solo necesitas definir una dirección válida para `mail.from` en `config.yml` y el [FQDN](https://en.wikipedia.org/wiki/Fully_qualified_domain_name) de tu instancia de Sentry para `SENTRY_MAIL_HOST` en `.env`.

Si quieres usar un servidor SMTP externo, puedes definir los parámetros correspondientes `mail.*` en el archivo `config.yml` e ignorar el servidor SMTP integrado. Si no tienes un servidor SMTP externo, pero necesitas ver los correos enviados, puedes establecer `mail.backend` en `console` en tu archivo `config.yml`. Consulta nuestra [documentación del servicio de correo](/es/backend/email/) para ver todos los detalles sobre qué significa y qué hace cada parámetro.

Si tus registros MX suelen resolverse a manejadores de correo internos de una red privada, consulta [este ticket](https://github.com/getsentry/self-hosted/issues/1202) para algunas indicaciones.

<Alert title="Warning" level="warning">
  Debido a la forma en que se superpone la configuración, si actualizas la configuración de <code>mail</code> a través de la interfaz web, también tendrás que comentar el valor predeterminado <code>mail.host: &#39;smtp&#39;</code> en tu <code>config.yml</code> para que se apliquen los ajustes que quieres.
</Alert>

Sentry no admite el envío de correo saliente con ningún otro protocolo que no sea SMTP (por ejemplo: la API de Sendgrid, la API de AWS SES, la API de Mailgun). Por lo general, estos servicios ofrecen un relay SMTP que puedes usar.

<div id="inbound-email">
  ## Correo entrante
</div>

Sentry ofrece un soporte muy limitado para correo entrante a través de [Mailgun](https://documentation.mailgun.com/en/latest/quickstart-receiving.html). Puedes encontrar toda la información sobre cómo configurarlo en nuestra [documentación del servicio de correo entrante](/es/backend/email/#inbound-email).
