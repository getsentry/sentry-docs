Esta sección contiene arquitecturas de referencia para Sentry autogestionado distintas de la configuración predeterminada de un solo nodo. No están pensadas para usarse tal cual, sino como referencia sobre cómo implementar Sentry autogestionado en tu infraestructura existente. Esta sección puede ayudarte a crear una estrategia de escalado si con el tiempo aumentan los volúmenes de tráfico.

Ten en cuenta que estas arquitecturas de referencia no contemplan dependencias externas de almacenamiento de datos como Kafka, Postgres, Redis, S3, etc. Si deseas hacerlo, consulta la sección [Configuraciones experimentales](/es/self-hosted/experimental/)
