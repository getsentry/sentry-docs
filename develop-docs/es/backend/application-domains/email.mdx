title: Correo electrónico
sidebar_order: 150

Sentry ofrece compatibilidad tanto para el correo saliente como para el entrante.

El correo entrante tiene un uso bastante limitado y actualmente solo admite procesar respuestas a las notificaciones de errores y notas.

<div id="outbound-email">
  ## Correo saliente
</div>

Necesitas configurar un proveedor SMTP para el correo saliente.

<dl>
<dt><code>mail.backend</code></dt>
<dd><markdown>
Declarado en `config.yml`.

El backend que se utilizará para la entrega de correo. Las opciones son `smtp`, `console` y `dummy`.

De forma predeterminada es `smtp`. Usa `dummy` si quieres desactivar la entrega de correo.

Usa `sentry.utils.email.PreviewBackend` para recibir correo localmente durante el desarrollo. De forma predeterminada se abrirán en la app nativa Mail si usas una MacBook. Selecciona iCloud como proveedor y usa la misma dirección de correo que tienes configurada en Sentry.

</markdown></dd>

<dt><code>mail.from</code></dt>
<dd><markdown>
Declarado en `config.yml`.

La dirección utilizada para el correo saliente en el encabezado `From`.

De forma predeterminada es `root@localhost`; también admite el formato `Full Name <fullname@example.com>`. Se recomienda encarecidamente cambiar este valor para garantizar una entrega de correo confiable.

</markdown></dd>

<dt><code>mail.host</code></dt>
<dd><markdown>
Declarado en `config.yml`.

El nombre de host al que conectarse para conexiones SMTP.

De forma predeterminada es `localhost`.

</markdown></dd>

<dt><code>mail.port</code></dt>
<dd><markdown>
Declarado en `config.yml`.

El puerto al que conectarse para conexiones SMTP.

De forma predeterminada es `25`.

</markdown></dd>

<dt><code>mail.username</code></dt>
<dd><markdown>
Declarado en `config.yml`.

El nombre de usuario que se utilizará al autenticarse con el servidor SMTP.

De forma predeterminada es `(empty)`.

</markdown></dd>

<dt><code>mail.password</code></dt>
<dd><markdown>
Declarado en `config.yml`.

La contraseña que se utilizará al autenticarse con el servidor SMTP.

De forma predeterminada es `(empty)`.

</markdown></dd>

<dt><code>mail.use-ssl</code></dt>
<dd><markdown>
Declarado en `config.yml`.

¿Debe Sentry usar SSL al conectarse al servidor SMTP? Es mutuamente excluyente con `mail.use-tls`.

De forma predeterminada es `false`.

</markdown></dd>

<dt><code>mail.use-tls</code></dt>
<dd><markdown>
Declarado en `config.yml`.

¿Debe Sentry usar STARTTLS al conectarse al servidor SMTP? Es mutuamente excluyente con `mail.use-ssl`.

De forma predeterminada es `false`.

</markdown></dd>

<dt><code>mail.list-namespace</code></dt>
<dd><markdown>
Declarado en `config.yml`.

El espacio de nombres de la lista de correo para los correos enviados por este servidor de Sentry. Debe ser un dominio del que seas titular (a menudo el mismo dominio que la parte de dominio del valor del parámetro de configuración `mail.from`) o `localhost`.

</markdown></dd>
</dl>

<div id="inbound-email">
  ## Correo entrante
</div>

Para la configuración, puedes elegir entre distintos backends.

<div id="mailgun">
  ### Mailgun
</div>

Empieza por elegir un dominio para gestionar el correo entrante. Es más sencillo si mantienes un dominio separado del resto. En nuestro ejemplo, vamos a elegir `inbound.sentry.example.com`. Deberás configurar los registros DNS de ese dominio según la documentación de Mailgun.

Crea una nueva ruta en Mailgun:

```
Prioridad:
  0
Expresión de filtrado:
  catch_all()
Acciones:
  forward("https://sentry.example.com/api/hooks/mailgun/inbound/")
Descripción:
  Manejador de correo entrante de Sentry
```

Configura Sentry con la configuración adecuada:

```yaml
# Tu clave API de Mailgun (utilizada para verificar webhooks entrantes)
mail.mailgun-api-key: ""

# Establece el nombre de host SMTP como tu dominio de entrada configurado
mail.reply-hostname: "inbound.sentry.example.com"

# Indica a Sentry que envíe los encabezados de correo apropiados para habilitar
# las respuestas entrantes
mail.enable-replies: true
```

Y, por último, actualiza Sentry con la configuración correspondiente:

```yaml
# Configura el nombre de host SMTP en tu dominio entrante configurado
mail.reply-hostname: "inbound.sentry.example.com"

# Indica a Sentry que envíe las cabeceras de correo adecuadas para habilitar
# las respuestas entrantes
mail.enable-replies: true
```

¡Listo! Ahora podrás responder a las notificaciones de actividad sobre errores desde tu cliente de correo electrónico.
