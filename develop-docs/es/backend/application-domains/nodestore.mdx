title: Almacenamiento de nodos
sidebar_order: 120
---

Sentry proporciona una abstracción llamada “nodestore”, que se utiliza para almacenar blobs de clave/valor.

El backend predeterminado simplemente los guarda como blobs comprimidos con gzip en la tabla “nodestore_node” de tu base de datos predeterminada.

<div id="django-backend">
  ## Backend de Django
</div>

El backend de Django almacena todos los datos en la tabla ‘nodestore&#95;node’, usando el patrón de blob JSON comprimido con gzip como texto.

El backend no ofrece opciones, así que simplemente debe configurarse como un diccionario vacío.

```python
SENTRY_NODESTORE = 'sentry.nodestore.django.DjangoNodeStorage'
SENTRY_NODESTORE_OPTIONS = {}
```


<div id="custom-backends">
  ## Backends personalizados
</div>

Si tienes una solución de almacenamiento de datos favorita, solo debe cumplir unas pocas reglas para funcionar con el almacenamiento de blobs de Sentry:

- establecer clave en valor
- obtener clave
- eliminar clave

Para obtener más información sobre cómo implementar tu propio backend, consulta `sentry.nodestore.base.NodeStorage`.
