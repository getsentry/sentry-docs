---
title: Métodos de redacción de PII
---

`remove`

: Eliminar el campo completo. Relay puede optar por establecerlo en `null` o
eliminarlo por completo.

```json
  {
    "rules": {
      "remove_ip": {
        "type": "ip",
        "redaction": {
          "method": "remove"
        }
      }
    },
    "applications": {
      "$string": ["remove_ip"]
    }
  }
```

`replace`

: Sustituye la clave por una cadena estática.

```json
  {
    "rules": {
      "replace_ip": {
        "type": "ip",
        "redaction": {
          "method": "replace",
          "text": "[censurado]"
        }
      }
    },
    "applications": {
      "$string": ["replace_ip"]
    }
  }
```


<div id="mask">
  ### `mask`
</div>

: Reemplaza cada carácter de la cadena coincidente por el carácter de “enmascaramiento” `*`. En comparación con `replace`, esto conserva la longitud de la cadena original.

```javascript
  {
    "rules": {
      "mask_ip": {
        "type": "ip",
        "redaction": {
          "method": "mask"
        }
      }
    },
    "applications": {
      "$string": ["mask_ip"]
    }
  }
```


<div id="hash">
  ### `hash`
</div>

: Reemplaza la cadena por una versión hasheada de ella misma. Las cadenas iguales producirán
el mismo hash, así que si, por ejemplo, decides hashear el ID de usuario en lugar de
reemplazarlo o eliminarlo, seguirás teniendo un recuento preciso de usuarios
afectados.

```javascript
  {
    "rules": {
      "hash_ip": {
        "type": "ip",
        "redaction": {
          "method": "hash"
        }
      }
    }
    "applications": {
      "$string": ["mask_ip"]
    }
  }
```
