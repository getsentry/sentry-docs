<div id="running-a-worker">
  ## Ejecutar un worker
</div>

Los workers se encargan de obtener tareas, ejecutarlas y enviar
el estado de finalizaci贸n de vuelta a un taskbroker. Antes de poder ejecutar un worker, necesitas
[iniciar un taskbroker](#running-taskbroker]). Hay algunas opciones para
ejecutar workers:

```bash
# Ejecutar un worker en primer plano
sentry run taskworker

# Ejecutar un worker como parte del devserver
sentry devserver --workers

# Llevar el worker al primer plano desde un modo de devservices como `ingest`
devservices foreground taskworker
```

<div id="running-a-task-scheduler">
  ## Ejecutar un planificador de tareas
</div>

El planificador de tareas es responsable de leer las programaciones de tareas desde
`settings.TASKWORKER_SCHEDULES` y lanzar tareas a medida que las programaciones vencen.

```bash
sentry run taskworker-scheduler
```

<div id="running-taskbroker">
  ## Ejecutar Taskbroker
</div>

Puedes iniciar Taskbroker como un servicio en segundo plano con `devservices`

```bash
devservices up --mode=taskbroker
```
