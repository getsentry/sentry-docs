title: Almacenamiento de series temporales
sidebar_order: 140
---

Sentry ofrece un servicio para almacenar datos de series temporales. Se utiliza principalmente para mostrar información agregada de eventos y proyectos, así como para calcular en tiempo real las tasas de eventos.

<div id="the-redissnuba-backend-recommended">
  ## El backend RedisSnuba (recomendado)
</div>

Este es el **único backend que funciona correctamente al 100%**:

```python
SENTRY_TSDB = 'sentry.tsdb.redissnuba.RedisSnubaTSDB'
```

Este backend se comunica con Snuba para las métricas relacionadas con la ingesta de eventos y con Redis para todo lo demás. Snuba necesita tener su propio consumidor de outcomes en ejecución, que actualmente no forma parte de devservices.

El TSDB de Redis encapsulado puede configurarse de la siguiente manera (consulta más abajo las opciones de Redis):

```python
SENTRY_TSDB_OPTIONS = {
    'redis': ... # diccionario de opciones de RedisTSDB aquí
}
```


<div id="the-dummy-backend">
  ## El backend ficticio
</div>

Como su nombre indica, *todos* los datos de la TSDB se descartarán al escribir y se sustituirán por ceros al leer:

```python
SENTRY_TSDB = 'sentry.tsdb.dummy.DummyTSDB'
```


<div id="the-redis-backend">
  ## El backend de Redis
</div>

El backend de Redis «básico» lee y escribe todos los datos en Redis. Varias columnas relacionadas con la ingesta de eventos (Organization Stats) mostrarán valores en cero, porque esos datos solo están disponibles en Snuba.

```python
SENTRY_TSDB = 'sentry.tsdb.redis.RedisTSDB'
```

De forma predeterminada, esto usará el clúster de Redis llamado `default`. Para usar un clúster diferente, proporciona la opción `cluster`, así:

```python
SENTRY_TSDB_OPTIONS = {
    'cluster': 'tsdb',
}
```
