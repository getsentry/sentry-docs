Esta sección incluye términos comunes y recursos para aprender más sobre el diseño de API. Si te estás iniciando en el diseño de API, este es un buen lugar para comenzar.

<div id="common-terms">
  ## Términos comunes
</div>

- **Recurso** es el objeto sobre el que realizas la acción con tu endpoint. Por ejemplo, en ProjectEndpoint el recurso es Project.

- **Identificador de recurso** puede ser un ID, como un ID de evento, o un slug, como el slug de una organización. Ten en cuenta que debe ser único. Por ejemplo, el identificador de recurso de proyecto de Sentry es \{organization}/\{project}, porque los proyectos solo son únicos dentro de su organización. Puedes encontrar más información al respecto en la sección slug vs. ID.

- **Método** es lo que haces con el recurso. Cada endpoint puede tener varios métodos. Por ejemplo, en ProjectEndpoint puedes tener un método GET que devuelve los detalles de un proyecto específico.

- **Colección** es básicamente un tipo de objeto. Puedes mapearlas a un tipo de objeto de Django como Organization o a un objeto de texto como un error.
