<div id="view-hierarchy">
  ## Jerarquía de vistas
</div>

El SDK nativo te permite adjuntar un archivo `view-hierarchy.json` que sigue la estructura descrita en [RFC#33](https://github.com/getsentry/rfcs/blob/main/text/0033-view-hierarchy.md). Esto está pensado principalmente para SDK descendentes (por ejemplo, [sentry-godot](https://github.com/getsentry/sentry-godot/pull/143)).

Para añadir un adjunto de jerarquía de vistas, debes configurar la ruta del archivo al inicializar el SDK. Supervisará el archivo y lo cargará junto con cualquier evento o fallo que se envíe a Sentry:

```c
sentry_options_add_view_hierarchy(options, "./view-hierarchy.json");
```

<Alert>
  Al usar `Crashpad` como backend de captura de fallos en el SDK nativo, el archivo debe llamarse exactamente `view-hierarchy.json` para que el sistema de ingesta lo procese correctamente.
</Alert>
