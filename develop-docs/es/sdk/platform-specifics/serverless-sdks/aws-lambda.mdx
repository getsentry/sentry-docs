Las funciones Lambda pueden escribirse en numerosos lenguajes de programación (JavaScript,
Python, Ruby, Java, Go, ...). Actualmente, Sentry admite la instrumentación
automática de funciones Lambda escritas en [JavaScript](https://docs.sentry.io/platforms/javascript/guides/aws-lambda/)
y [Python](https://docs.sentry.io/platforms/python/integrations/aws-lambda/).

Consulta la [documentación de Sentry](https://docs.sentry.io/product/integrations/cloud-monitoring/)
para saber cómo configurar la instrumentación sin servidor.

<div id="a-short-primer-into-aws-lambda-functions-and-layers">
  ## Una breve introducción a las funciones y capas de AWS Lambda.
</div>

Una función Lambda es un directorio comprimido que contiene todo lo que el entorno de ejecución de Lambda necesita para ejecutar el código que incluye. Debes tener el código de tu función, así como todas las dependencias, en el directorio de tu función Lambda. Las funciones Lambda se invocan mediante un evento (por ejemplo, visitar una URL) y se facturan por los milisegundos que la función tarda en completarse.

![Ciclo de vida de una función de AWS Lambda](aws-lambda-lifecycle.png)

Las capas de AWS Lambda pueden asociarse a las funciones. Las capas también son directorios comprimidos y contienen código adicional utilizado por tu función Lambda. Se usan para compartir código entre funciones o para añadir funcionalidades de terceros, como el SDK de Sentry, a tu función Lambda.

<div id="sentry-aws-lambda-integration">
  ## Integración de Sentry con AWS Lambda
</div>

Con las integraciones de AWS Lambda, los usuarios pueden añadir la funcionalidad de Sentry a su función Lambda sin cambiar el código de la propia función.

Sentry ofrece una capa de Lambda (Lambda Layer) que incluye el SDK de JavaScript (o el SDK de Python).

Consulta la documentación para saber cómo los clientes pueden instrumentar sus funciones Lambda: https://docs.sentry.io/product/integrations/cloud-monitoring/aws-lambda/

<div id="sentry-lambda-layer">
  ## Capa de Lambda de Sentry
</div>

Sentry está desplegando una capa de Lambda para funciones Lambda basadas en Node
(`SentryNodeServerlessSDK`) y una capa de Lambda para funciones Lambda basadas en Python
(`SentryPythonServerlessSDK`). Estas capas de Lambda se distribuyen
en varias regiones de AWS para estar más cerca de los usuarios.

Ambas capas incluyen la versión más reciente del SDK de JavaScript de Sentry (o del SDK de Python).

<div id="building-of-the-python-layer">
  ### Creación de la capa de Python
</div>

El contenido del directorio de la capa para Python se genera durante el flujo de trabajo de “CI” en GitHub Actions. Los pasos necesarios se pueden encontrar
[aquí](https://github.com/getsentry/sentry-python/blob/master/.github/workflows/ci.yml).

Esto preparará el directorio de la capa de Lambda, comprimirá todo y cargará el archivo ZIP resultante como artefacto en GitHub.

<div id="deployment-of-the-python-layer">
  ### Despliegue de la capa de Python
</div>

El archivo zip preparado en el paso anterior se implementará en AWS Lambda mediante
[Craft](https://github.com/getsentry/craft). La configuración de Craft para hacerlo
se puede encontrar [aquí](https://github.com/getsentry/sentry-python/blob/master/.craft.yml).

Para implementar una nueva versión de la capa de Lambda en AWS, debes publicar una nueva
versión del SDK de Python.

<div id="building-of-the-javascriptnode-layer">
  ### Compilación de la capa de JavaScript/Node
</div>

El contenido del directorio de la capa de JavaScript/Node se compila durante el flujo de trabajo "Build & Test" en GitHub Actions. Los pasos necesarios se pueden encontrar
[aquí](https://github.com/getsentry/sentry-javascript/blob/master/.github/workflows/build.yml).

Esto preparará el directorio de la capa de Lambda, comprimirá todo y subirá el archivo ZIP resultante como
un artefacto a GitHub.

<div id="deployment-of-the-javascriptnode-layer">
  ### Implementación de la capa de JavaScript/Node
</div>

El archivo zip preparado en el paso anterior se implementará en AWS Lambda mediante
[Craft](https://github.com/getsentry/craft). La configuración de Craft para hacerlo
se puede encontrar [aquí](https://github.com/getsentry/sentry-javascript/blob/master/.craft.yml).

Para implementar una nueva versión de la capa de Lambda en AWS, debes desencadenar una nueva
versión del SDK de JavaScript.

<div id="development-environment">
  ### Entorno de desarrollo
</div>

Al trabajar en la capa Sentry AWS Lambda, el flujo de trabajo consiste en tener una función Lambda de ejemplo en una cuenta de desarrollo de AWS y luego desplegar tu capa local y vincularla a esa función de ejemplo. Cómo hacerlo en Python se explica en la [guía de contribución](https://github.com/getsentry/sentry-python/blob/master/CONTRIBUTING.md#contributing-to-sentry-aws-lambda-layer) del SDK de Python.
