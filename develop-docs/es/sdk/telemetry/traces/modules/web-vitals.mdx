---
title: Módulo Web Vitals
---

El SDK debería auto-instrumentar:

- [Cargas de página](#pageloads-transaction-conventions) como transacciones
- [Interacciones](#interactions-span-conventions) que generen un [INP](https://web.dev/articles/inp) como spans independientes

<div id="pageloads-transaction-conventions">
  ## Convenciones de transacciones de carga de página
</div>

<div id="properties">
  ### Propiedades
</div>

| Propiedad | Descripción |
|:--|:--|
| `transaction`           | Una cadena de baja cardinalidad que identifica la ruta de carga de la página. Puede ser una plantilla de ruta si corresponde a la plataforma de frontend; p. ej., `"/settings/{project}/"` |
| `contexts.trace.op`     | Siempre `pageload` |
| `contexts.browser.name` | El navegador del cliente en el que se cargó la página. Los valores admitidos incluyen `Chrome`, `Firefox`, `Safari`, `Opera` y `Edge`. |

<div id="measurements">
  ### Mediciones
</div>

Las métricas de Web Vitals deben registrarse como mediciones en la transacción de carga de página.

```json
{
  "measurements": {
    "lcp": { "value": 2049, "unit": "milisegundo" },
    "fcp": { "value": 2049, "unit": "milisegundo" },
    "cls": { "value": 0.21, "unit": "" },
    "ttfb": { "value": 13, "unit": "milisegundo" },
  }
}
```

| Medición | Descripción |
|:--|:--|
| `measurements.lcp`      | Métrica [Largest Contentful Paint](https://web.dev/articles/lcp) detectada durante la carga de la página. |
| `measurements.fcp`      | Métrica [First Contentful Paint](https://web.dev/articles/fcp) detectada durante la carga de la página. |
| `measurements.cls`      | Métrica [Cumulative Layout Shift](https://web.dev/articles/cls) detectada durante la carga de la página. |
| `measurements.ttfb`     | Métrica [Time To First Byte](https://web.dev/articles/ttfb) detectada durante la carga de la página. |


<div id="interactions-span-conventions">
  ## Convenciones de intervalos de interacción
</div>

<div id="attributes">
  ### Atributos
</div>

| Atributo | Descripción |
|:--|:--|
| `transaction`      | Una cadena de baja cardinalidad que representa la ruta de la página donde se produjo el evento INP. Puede ser una plantilla de ruta si aplica a la plataforma de frontend; p. ej., `"/settings/{project}/"` |
| `description`      | Si corresponde, la cadena de selector que identifica el elemento en el que se produjo la interacción; p. ej., `"Sidebar > span.user > a"` |
| `op`               | `ui.interaction.click`, `ui.interaction.press`, `ui.interaction.hover` o `ui.interaction.drag` |

<div id="measurements">
  ### Mediciones
</div>

La métrica Interaction to Next Paint debe registrarse como una medición en el lapso de interacción.

```json
{
  "measurements": {
    "inp": { "value": 500, "unit": "milisegundos" },
  }
}
```

| Medición | Descripción |
|:--|:--|
| `measurements.inp` | La métrica [Interaction to Next Paint](https://web.dev/articles/inp) detectada para la interacción. |
