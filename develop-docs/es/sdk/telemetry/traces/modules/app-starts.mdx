---
title: Módulo de inicio de la app
---

El módulo de inicio de la app registra métricas de los spans en transacciones `ui.load` utilizando operaciones de span específicas.

<div id="transaction-conventions">
  # Convenciones de transacción
</div>

<div id="transaction-measurements">
  ## Mediciones de transacción
</div>

| Clave | Unidad | Descripción | Condiciones |
|:--|:--|:--|:--|
| `app_start_cold` | milisegundos | Tiempo para cargar el primer fotograma durante un inicio en frío de la app | obligatorio para inicios en frío de la app |
| `app_start_warm` | milisegundos | Tiempo para cargar el primer fotograma durante un inicio en caliente de la app | obligatorio para inicios en caliente de la app |

<div id="span-conventions">
  # Convenciones de span
</div>

<div id="span-operations">
  ## Operaciones de spans
</div>

| Operación de span | Descripción | Condiciones |
|:--|:--|:--|
| `app.start.cold` | Operación durante un inicio en frío | Debe haber al menos un span con la descripción "Cold Start" presente en una transacción |
| `app.start.warm` | Operación durante un inicio en caliente | Debe haber al menos un span con la descripción "Warm Start" presente en una transacción |
| `db.sql.query` | Consulta a la base de datos | - |
| `db.sql.transaction` | - | - |
| `db` | Operación en la base de datos | - |
| `file.read` | Lectura de un archivo desde el sistema de archivos | - |
| `file.write` | Escritura de un archivo en el sistema de archivos | - |
| `http.client` | Solicitud de red saliente | - |
| `ui.load` | Operación en una interfaz móvil | - |

<div id="android-specific-span-operations">
  ## Operaciones de span específicas de Android
</div>

| Operación de span | Descripción |
|:--|:--|
| `activity.load` | Llamada a `onCreate` de una actividad |
| `application.load` | Llamada a `onCreate` de la aplicación |
| `contentprovider.load` | Llamada a `onCreate` de un proveedor de contenido |
| `db.sql.room` | Consulta de base de datos con la biblioteca Room | - |
| `process.load` | Duración de tiempo empleada por el SO antes de la primera inicialización del proveedor de contenido |
