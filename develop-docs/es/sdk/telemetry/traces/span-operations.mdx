---
title: Operaciones de span
---

Las operaciones de span son un código breve que identifica el tipo de operación que el span está midiendo. Las operaciones de span son atributos de baja cardinalidad: deben ser lo más generales posible, sin dejar de ser legibles para las personas y útiles. Deben evitar incluir datos de alta cardinalidad, como ID y URL.

Se espera que las operaciones sigan las [convenciones semánticas de OpenTelemetry](https://github.com/open-telemetry/semantic-conventions/blob/main/docs/general/trace.md) tanto como sea posible.

Es importante mantener las categorías coherentes entre SDK e integraciones, ya que Sentry las utiliza en varias partes de la plataforma. Por ejemplo, tanto `db.init` como `db.query` se clasificarán como operaciones de base de datos (`db`). La configuración predeterminada del desglose de operaciones puede consultarse [aquí](https://github.com/getsentry/sentry/blob/9f36200d665209c9328f5ccc21135d3de9b65318/src/sentry/projectoptions/defaults.py#L80).

<Alert title="Nota">
  Las operaciones de span deben estar en minúsculas y usar snake_case
</Alert>

<div id="list-of-operations">
  # Lista de operaciones
</div>

Las siguientes tablas contienen ejemplos de operaciones usadas por los SDK y el producto Sentry. La columna de uso de la tabla incluye ejemplos de cómo utilizar esa categoría de operación, pero no son recomendaciones estrictas sobre cómo usarla. Mientras las categorías se mantengan coherentes, los desarrolladores de SDK pueden elegir libremente las acciones e identificadores que mejor se ajusten al caso de uso que están instrumentando.

Si no se proporciona una operación de span, Relay establece el valor `default`. El valor `default` también puede ser establecido por los SDK, por ejemplo, para evitar cambios incompatibles cuando ya no se requiere definir `op` en `TransactionContext`.
