---
title: Interfaz de plantilla
sidebar_order: 5
---

La interfaz de plantilla es útil para generar informes específicos del motor de plantillas cuando los rastreos de pila habituales no contienen datos de plantilla. Esto, por ejemplo, es necesario en el framework Django, donde las plantillas no se integran en el rastreo de pila de Python.

Una plantilla procesada. Generalmente se utiliza como un único marco en un rastreo de pila y solo debe usarse si el sistema de plantillas no proporciona rastreos de pila adecuados por sí mismo.

<div id="attributes">
  ## Atributos
</div>

Los atributos `filename`, `context_line` y `lineno` son obligatorios.

`lineno`

: El número de línea de la llamada.

`abs_path`

: La ruta absoluta a la plantilla en el sistema de archivos.

`filename`

: El nombre de archivo tal como se pasó al cargador de plantillas.

`context_line`

: Código fuente en `filename` en `lineno`.

`pre_context`

: Una lista de líneas de código fuente antes de `context_line` (en orden), normalmente
  `[lineno - 5:lineno]`.

`post_context`

: Una lista de líneas de código fuente después de `context_line` (en orden), normalmente
  `[lineno + 1:lineno + 5]`.

<div id="examples">
  ## Ejemplos
</div>

```json
{
  "template": {
    "abs_path": "/real/file/name.html",
    "filename": "file/name.html",
    "lineno": 3,
    "pre_context": [
      "línea1",
      "línea2"
    ],
    "context_line": "línea3",
    "post_context": [
      "línea4",
      "línea5"
    ],
  }
}
```
