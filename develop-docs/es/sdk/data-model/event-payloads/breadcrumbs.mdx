---
title: Interfaz de breadcrumbs
---

Sentry usa *breadcrumbs* para crear un rastro de eventos que ocurrieron antes de un problema. Estos eventos son muy similares a los registros tradicionales, pero pueden capturar datos estructurados más ricos.

Esta página ofrece información técnica sobre la estructura de un breadcrumb. Puedes leer una descripción general sobre el registro manual de breadcrumbs y su personalización en nuestra página de documentación de Sentry [Breadcrumbs](https://docs.sentry.io/platform-redirect/?next=/enriching-events/breadcrumbs/).

Un evento puede contener una propiedad `breadcrumbs` con una entrada, `values`, que es un arreglo de objetos breadcrumb. Las entradas se ordenan de la más antigua a la más reciente. En consecuencia, la última entrada de la lista debería ser la anterior al evento.
Alternativamente, la propiedad `breadcrumbs` puede ser en sí misma un arreglo de objetos breadcrumb.

El siguiente ejemplo ilustra la sección de breadcrumbs de la carga útil del evento y omite otros atributos para simplificar.

```json
{
  "breadcrumbs": {
    "values": [
      {
        "timestamp": "2016-04-20T20:55:53.845Z",
        "message": "Algo ocurrió",
        "category": "log",
        "data": {
          "foo": "bar",
          "blub": "blah"
        }
      },
      {
        "timestamp": "2016-04-20T20:55:53.847Z",
        "type": "navigation",
        "data": {
          "from": "/login",
          "to": "/dashboard"
        }
      }
    ]
  }
}
```

O, de forma alternativa:

```json
{
  "breadcrumbs": [
    {
      "timestamp": "2016-04-20T20:55:53.845Z",
      "message": "Algo ocurrió",
      "category": "log",
      "data": {
        "foo": "bar",
        "blub": "blah"
      }
    },
    {
      "timestamp": "2016-04-20T20:55:53.847Z",
      "type": "navigation",
      "data": {
        "from": "/login",
        "to": "/dashboard"
      }
    }
  ]
}
```

Un objeto breadcrumb tiene las siguientes propiedades:

`type` (opcional)

: El tipo de breadcrumb. De forma predeterminada, todos los breadcrumbs se registran como `default`, lo que hace que aparezcan como una entrada de Depuración, pero Sentry ofrece otros tipos que influyen en cómo se representan los breadcrumbs. Para más información, consulta la [descripción de los tipos de breadcrumb reconocidos](#breadcrumb-types).

`category` (opcional)

: Una cadena con puntos que indica qué es el crumb o de dónde proviene. Por lo general, es un nombre de módulo o una cadena descriptiva. Por ejemplo, `ui.click` podría usarse para indicar que ocurrió un clic en la UI o flask podría usarse para indicar que el evento se originó en el framework Flask.

*De forma interna representamos el color y el ícono de algunos crumbs según la categoría proporcionada. Para más información, consulta la [descripción de los tipos de breadcrumb reconocidos](#breadcrumb-types).*

`message` (opcional)

: Mensaje legible para humanos del breadcrumb.

Si se proporciona un mensaje, se representa como texto con todos los espacios en blanco preservados.

`data` (opcional)

: Datos arbitrarios asociados a este breadcrumb.

Contiene un diccionario cuyo contenido depende del tipo de breadcrumb. Los parámetros adicionales no admitidos por el tipo se muestran como una tabla de pares clave/valor.

`level` (opcional)

: Define el nivel de gravedad del breadcrumb. Los valores permitidos son, de mayor a menor: `fatal`, `error`, `warning`, `info` y `debug`. Los niveles se usan en la UI para enfatizar y desatenuar el crumb. El valor predeterminado es `info`.

`origin` (opcional)

: Cadena que representa el origen del breadcrumb. Suele usarse para identificar la fuente del breadcrumb. Por ejemplo, los SDK híbridos pueden identificar breadcrumbs nativos desde JS o Flutter.

`timestamp` (recomendado)

: Marca de tiempo que representa cuándo ocurrió el breadcrumb. El formato puede ser una cadena tal como se define en la [RFC 3339](https://tools.ietf.org/html/rfc3339) o un valor numérico (entero o flotante) que representa el número de segundos transcurridos desde la [época Unix](https://en.wikipedia.org/wiki/Unix_time).
Los breadcrumbs son más útiles cuando incluyen una marca de tiempo, ya que crean una cronología que conduce a un evento de `exception/error`.

<Alert>
  Los breadcrumbs no se ordenarán por marca de tiempo; mantienen el orden en el que se añadieron.
</Alert>


<div id="breadcrumb-types">
  ## Tipos de migas de pan
</div>

A continuación se describen los distintos tipos de migas de pan y cómo se ven sus propiedades `data`.

`default`

: Describe una miga de pan genérica. Suele ser un mensaje de registro o una miga de pan generada por el usuario. La parte `data` no está definida y, por lo tanto, se muestra íntegramente como una tabla de pares clave/valor.

```json
{
  "type": "default",
  "category": "started",
  "data": undefined,
  "level": "info",
  "message": "este es un mensaje",
  "timestamp": 1596814007.035
}
```

`debug`

: Suele ser un mensaje de registro. La parte `data` no está definida y, por ello, se muestra íntegramente como una tabla de pares clave/valor.

*Internamente mostramos migas de pan de tipo `default` que incluyen la categoría `console` como una miga de pan de tipo `debug`.*

```json
{
  "type": "debug",
  "category": "started",
  "data": null,
  "level": "info",
  "message": "este es un mensaje",
  "timestamp": 1596814007.035
}
```

`error`

: Un error ocurrido antes de la excepción.

```json
{
  "type": "error",
  "category": "error",
  "level": "error",
  "message": "este es un mensaje",
  "timestamp": 1596814007.035
}
```

`navigation`

: Un evento de navegación puede ser un cambio de URL en una aplicación web o una transición de interfaz en una aplicación móvil o de escritorio, etc.

*Internamente mostramos rutas de navegación (breadcrumbs) de tipo `default` que incluyen la categoría `navigation` como una ruta de tipo `navigation`.*

Su propiedad `data` tiene las siguientes subpropiedades:

`from` (Obligatorio)

Una cadena que representa el estado o la ubicación original de la aplicación.

`to` (Obligatorio)

Una cadena que representa el nuevo estado o la nueva ubicación de la aplicación.

```json
{
  "type": "navigation",
  "category": "navigation",
  "timestamp": "2016-04-20T20:55:53.845Z",
  "data": {
    "from": "/login",
    "to": "/dashboard"
  }
}
```

`http`

: Representa una solicitud HTTP enviada desde tu aplicación. Puede ser una solicitud AJAX de una aplicación web o una solicitud HTTP de servidor a servidor a un proveedor de servicios de API, etc.

Su propiedad `data` tiene las siguientes subpropiedades:

`url` (opcional)

La URL de la solicitud.

`method` (opcional)

El método de la solicitud HTTP.

`status_code` (opcional)

El código de estado HTTP de la respuesta.

`reason` (opcional)

Texto que describe el código de estado.

```json
{
  "type": "http",
  "category": "xhr",
  "data": {
    "url": "http://example.com/api/1.0/users",
    "method": "GET",
    "status_code": 200,
    "reason": "OK"
  },
  "timestamp": "2016-04-20T20:55:53.845Z"
}
```

`info`

: Información que ayuda a identificar la causa raíz del problema o para quién se produjo el error.

```json
{
  "type": "info",
  "category": "started",
  "level": "info",
  "message": "este es un mensaje",
  "timestamp": 1596813998.386
}
```

`query`

: Representa una consulta realizada en tu aplicación.

```json
{
  "type": "query",
  "category": "started",
  "level": "info",
  "message": "este es un mensaje",
  "timestamp": 1596813998.386
}
```

`transacción`

: Describe un evento de trazado.

*Internamente mostramos breadcrumbs de tipo `default` que contienen las categorías `sentry.transaction` y `sentry.event` como un breadcrumb de tipo `transaction`.*

```json
{
  "type": "default",
  "category": "sentry.transaction",
  "level": "info",
  "message": "este es un mensaje",
  "timestamp": 1596813997.646
}
```

`ui`

: Una interacción del usuario con la interfaz de tu app.

*Internamente, mostramos breadcrumbs de tipo `default` que incluyen la categoría `ui.*` como un breadcrumb de tipo `ui`.*

```json
{
  "type": "default",
  "category": "ui.click",
  "message": "div.css-bsbdc4-TextOverflow.e1kpcezi0",
  "level": "info",
  "timestamp": 1598613151.875368
}
```

`usuario`

: Una interacción de un usuario con la interfaz de tu aplicación.


```json
{
  "type": "user",
  "category": "click",
  "message": "div.css-bsbdc4-TextOverflow.e1kpcezi0",
  "level": "info",
  "timestamp": 1598613151.875368
}
```
