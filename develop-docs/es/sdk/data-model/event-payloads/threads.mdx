---
title: Interfaz de Threads
sidebar_order: 10
---

La interfaz de Threads especifica los hilos que estaban en ejecución en el momento en que ocurrió un evento. Estos hilos también pueden incluir trazas de pila.

Un [evento](/es/sdk/data-model/event-payloads/) puede contener uno o más hilos en un atributo llamado `threads`.

El atributo `threads` debe ser un objeto con el atributo `values`, que contenga uno o más valores que sean objetos con el formato descrito a continuación. Alternativamente, el atributo `threads` puede ser una lista plana de objetos con el formato siguiente.

Según la política de Sentry, el hilo que se bloqueó debido a una [exception](/es/sdk/data-model/event-payloads/exception/) no debe tener una traza de pila; en su lugar, se debe establecer el atributo `thread_id` en la excepción y Sentry conectará ambos.

<div id="attributes">
  ## Atributos
</div>

`id`

: **Obligatorio**. El ID del hilo. Normalmente, un número o una cadena numérica.
Debe ser único entre los hilos. Una excepción puede establecer el atributo `thread_id`
para hacer referencia cruzada a este hilo.

`crashed`

: *Opcional*. Indicador de si el hilo fue la causa del evento enviado, p. ej., debido a un bloqueo (crash).
De manera predeterminada es `false`.

`current`

: *Opcional*. Indicador de si el hilo estaba en primer plano. Un hilo está en primer plano cuando se está ejecutando.
De manera predeterminada es `false`.

`main`

: *Opcional*. Si corresponde, indicador de si el hilo era responsable de renderizar la interfaz de usuario.
En plataformas móviles, esto a menudo se denomina “main thread” o “ui thread”.

`name`

: *Opcional*. El nombre del hilo.

`state`

: *Opcional*. Estado del hilo en el momento del evento.

`held_locks`

: *Opcional*. Representa una colección de bloqueos (objetos monitor) mantenidos por un hilo. Un diccionario con las direcciones de memoria de los objetos de bloqueo y
sus respectivos motivos/detalles del bloqueo. El motivo del bloqueo corresponde a la [Lock Reason Interface](/es/sdk/data-model/event-payloads/lockreason/).

`stacktrace`

: *Opcional*. Un objeto de traza de pila correspondiente a la [Stack Trace Interface](/es/sdk/data-model/event-payloads/stacktrace/).

Si se trata de un evento de error, la traza de pila de la excepción principal debe declararse
en la [Exception Interface](/es/sdk/data-model/event-payloads/exception/).
Sentry moverá automáticamente la traza de pila del único hilo que se bloqueó si
hay una sola excepción.

<div id="examples">
  ## Ejemplos
</div>

El siguiente ejemplo ilustra la sección de hilos del [event payload](/es/sdk/data-model/event-payloads/) y omite
otros atributos para simplificar.

```json
{
  "threads": {
    "values": [
      {
        "id": "0",
        "name": "principal",
        "crashed": true,
        "main": true,
        "held_locks": {
          "0x0d3a2f0a": {},
          "0x07d7437b": {}
        },
        "state": "En espera",
        "stacktrace": {}
      }
    ]
  }
}
```

O bien:

```json
{
  "threads": [
    {
      "id": "0",
      "name": "main",
      "crashed": true,
      "main": true,
      "held_locks": {
        "0x0d3a2f0a": {},
        "0x07d7437b": {}
      },
      "state": "Runnable",
      "stacktrace": {}
    }
  ]
}
```
