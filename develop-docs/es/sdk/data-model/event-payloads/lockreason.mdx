---
title: Interfaz de motivo de bloqueo
---

Representa una instancia de un bloqueo retenido (objeto monitor) en un hilo. Por lo general, esta información es útil para determinar qué otro hilo está reteniendo el bloqueo en caso de que el hilo actual esté bloqueado. En lenguajes como `Java` o `Kotlin`, esto normalmente se representará mediante un tipo `java.lang.Object`.

<Alert>
  Los motivos de bloqueo siempre forman parte de un [hilo](/es/sdk/data-model/event-payloads/threads/). No pueden declararse como una propiedad de evento de nivel superior.
</Alert>

<div id="attributes">
  ## Atributos
</div>

`type`

: **Obligatorio**. Tipo de bloqueo en el hilo; opciones disponibles: blocked, waiting, sleeping y locked.

`address`

: *Opcional*. Dirección de memoria del objeto monitor.

`package_name`

: *Opcional*. Nombre del paquete del objeto monitor.

`class_name`

: *Opcional*. Nombre de la clase del objeto monitor.

`thread_id`

: *Opcional*. ID del hilo que mantiene el bloqueo.

<div id="examples">
  ## Ejemplos
</div>

```json
{
  "type": 8,
  "address": "0x07d7437b",
  "package_name": "android.database.sqlite",
  "class_name": "SQLiteConnection",
  "thread_id": 2
}
```
