<div id="what-are-preview-builds">
  ## ¿Qué son las compilaciones de vista previa?
</div>

Cuando abres un pull request en GitHub en el repositorio de sentry, Vercel puede compilar y desplegar automáticamente el código del frontend de tu rama en una URL de vista previa. Esto te permite ver el nuevo código tal como aparecería en producción.

Esto es muy útil para:

- Previsualizar nuevas funcionalidades del frontend que requieren interacciones complejas para entender cómo funcionan. Esto puede ser ideal si la funcionalidad específica que has creado tiene un cierto “feeling” que solo puede transmitirse al usar de verdad la interfaz en un navegador web. La compilación de vista previa permitirá a los revisores ver **rápidamente** cómo funciona tu funcionalidad.

- Validar que una corrección de errores realmente esté funcionando en producción y no sufra el síndrome de __funcionaba en mi máquina__.

- Compartir nuevas funcionalidades muy solicitadas por la comunidad que aún no se han integrado. Especialmente si la funcionalidad aún está en desarrollo, incluso puede ser posible **obtener comentarios de __usuarios reales__** antes de que se integre.

<div id="how-can-i-have-preview-builds-on-my-pull-request">
  ## ¿Cómo puedo obtener compilaciones de vista previa en mi Pull Request?
</div>

Cuando abres un Pull Request, la vista previa se comenzará a compilar y desplegar automáticamente mediante Vercel. Cuando termine (tras unos 4 minutos), encontrarás el enlace de la vista previa en el panel encima de los GitHub Checks que dice `This branch was successfully deployed`:

![Deberás hacer clic en "Show environments"](../img/githubPRShowEnvironmentsBar.png)

Haz clic en el conmutador `Show environments` y luego en `View deployment` en la fila `Preview - sentry`.

![Haz clic en "View deployment" en la fila `Preview - sentry`](../img/githubPRShowEnvironmentsExpanded.png)

<div id="why-dont-my-pull-requests-have-build-previews">
  ## ¿Por qué mis Pull Requests no tienen vistas previas de compilación?
</div>

Hay algunas consideraciones al usar las compilaciones de vista previa del frontend:

1. Los cambios del backend **no se desplegarán en las compilaciones de vista previa**. Puedes tener un PR que incluya cambios tanto en el frontend como en el backend, pero **solo se desplegarán los cambios del frontend**. Es importante recordar esto, porque si la vista previa del frontend depende de cambios del backend del API que aún no están en producción, tu vista previa reflejará de forma inexacta la funcionalidad que has creado. Las compilaciones de vista previa usan **la API de producción**.

2. Los componentes de frontend de `getsentry` (el repositorio) no están incluidos en las compilaciones de vista previa. No te alarmes si faltan funciones de SaaS (normalmente funciones de upsell).

3. Algunas funciones de Sentry **no** son compatibles, ya que aún se sirven desde el backend. Esto incluye (pero no se limita actualmente a):

    - La autenticación SSO **NO** funciona por el momento. Necesitarás iniciar sesión con una cuenta que forme parte de una organización que no requiera SSO.
    - El inicio de sesión con 2FA no funciona actualmente.
    - Las páginas de configuración de SSO no tienen vistas en React.
    - La instalación de integraciones no funcionará, ya que las URL de redirección serán incorrectas.
    - Faltan varias interfaces de autenticación (restablecimiento de contraseña, registro).

4. Solo los PR creados desde ramas dentro del propio repositorio pueden generar vistas previas; un PR creado desde un fork fallará con el mensaje “Authorization required to deploy.”
