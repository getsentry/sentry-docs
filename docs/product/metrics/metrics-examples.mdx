If you're not sure where to start, here are a few examples of how you can use custom metrics.

## Frontend Examples

<details open>
  <summary><b>Count button clicks on subpages</b></summary>
  
  * **Metric type**: `counter`

  * **Example JS code**: 

  ```js
  Sentry.metrics.increment("button_click", 1, {
    tags: { browser: "Firefox", subpage: "signup" }
  });
  ```
</details>

<details>
  <summary><b>Track number of user signups</b></summary>
    
  * **Metric type**: `counter`

  * **Example JS code**: 

  ```js
  Sentry.metrics.increment("signups", 1, {
    tags: { plan_type: “freemium”, region: "EU" }
  });
  ```
</details>

<details>
  <summary><b>Measure UX improvement to signup form</b></summary>
    
  * **Metric type**: `distribution`

  * **Example JS code**: 

  ```js
  Sentry.metrics.distribution("login.time_to_finish_form", 14.3, {
   tags: { subpage: “signup” },
   unit: “second”
  });
  ```
</details>

<details>
  <summary><b>Track the loading time for a component</b></summary>
    
  * **Metric type**: `distribution`

  * **Example JS code**: 

  ```js
  Sentry.metrics.distribution("component_load_time", 15.0, {
   tags: { type: "important" },
   unit: "millisecond"
  });
  ```
</details>

<details>
  <summary><b>Track API request payloads</b></summary>
    
  * **Metric type**: `distribution`

  * **Example JS code**: 

  ```js
  Sentry.metrics.distribution("api_request_payload", 136.3, {
    tags: { api_name: “maps_provider”, subpage: “signup” },
    unit: “kilobyte”
  });
  ```
</details>

<details>
  <summary><b>Count concurrent users in application</b></summary>
    
  * **Metric type**: `gauge`

  * **Example JS code**: 

  ```js
  Sentry.metrics.gauge("concurrent_users", 10, {
   tags: { browser: "Firefox" }
  });
  ```
</details>

<details>
  <summary><b>Track the number of items in a shopping cart</b></summary>
    
  * **Metric type**: `gauge`

  * **Example JS code**: 

  ```js
  Sentry.metrics.gauge("shopping_cart_item_count", itemCount, {
   tags: { platform: "web" }
  });
  ```
</details>

<details>
  <summary><b>Count the unique number of users on a page</b></summary>
    
  * **Metric type**: `set`

  * **Example JS code**: 

  ```js
  Sentry.metrics.set("user_view", "jane", {
   tags: {"page": "/home" }
  });
  ```
</details>

<details>
  <summary><b>Count unique number of user sessions</b></summary>
    
  * **Metric type**: `set`

  * **Example JS code**: 

  ```js
  Sentry.metrics.set("unique_active_sessions", sessionId, {
   tags: { platform: "mobile" }
  });
  ```
</details>


## Backend Examples

<details open>
  <summary><b>Count of API calls of a 3rd party API</b></summary>
    
  * **Metric type**: `counter`

  * **Example Python code**: 

  ```python
  sentry_sdk.metrics.incr(
   key = "api_calls",
   value = 1,
   tags = {"api": "third_party"}
  )
  ```
</details>

<details>
  <summary><b>Count of jobs executed per worker</b></summary>
    
  * **Metric type**: `counter`

  * **Example Python code**: 

  ```python
  sentry_sdk.metrics.incr(
   key = ”external_api_calls”,
   value = 1,
   tags = {”worker_id”: “485ea324-003d-4f30-a0f9”, “type”: ”standard”}
  )
  ```
</details>

<details>
  <summary><b>Processing time for a task</b></summary>
    
  * **Metric type**: `distribution`

  * **Example Python code**: 

  ```python
  sentry_sdk.metrics.distribution(
   key = ”processing_time”,
   value = 0.002,
   unit = ”second”,
   tags = {”task”: “example_task”}
  )
  ```
</details>

<details>
  <summary><b>Track ML model confidence during inference</b></summary>
    
  * **Metric type**: `distribution`

  * **Example Python code**: 

  ```python
  sentry_sdk.metrics.distribution(
   key = ”model_confidence”,
   value = 0.72,
   unit = ”ratio”,
   tags = {”model_id”: “a294c108”}
  )
  ```
</details>

<details>
  <summary><b>Track CPU usage over time</b></summary>
    
  * **Metric type**: `gauge`

  * **Example Python code**: 

  ```python
  sentry_sdk.metrics.gauge(
   key=”cpu_usage”,
   value=94,
   unit=”percent”
  )
  ```
</details>

<details>
  <summary><b>Track cache hit ratio</b></summary>
    
  * **Metric type**: `gauge`

  * **Example Python code**: 

  ```python
  sentry_sdk.metrics.gauge(
   key=”cache_hit_ratio”,
   value=85,
   unit=”percent”
  )
  ```
</details>

<details>
  <summary><b>Count unique server ids running a software version</b></summary>
    
  * **Metric type**: `set`

  * **Example Python code**: 

  ```python
  sentry_sdk.metrics.set(
   key=”server_ids_running”,
   value=“server_id”,
   tags={"software_version": software_version}
  )
  ```
</details>

<details>
  <summary><b>Track number of unique DB queries executed</b></summary>
    
  * **Metric type**: `set`

  * **Example Python code**: 

  ```python
  sentry_sdk.metrics.set(
   key=”db_queries”,
   value=query,
  tags={"query_type": "read"}
  )
  ```
</details>
