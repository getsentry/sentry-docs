Las integraciones internas están pensadas para integraciones personalizadas, únicas de tu organización. También pueden ser tan simples como un token a nivel de organización. Ya sea que uses solo la API o todas las funciones de la plataforma de integraciones combinadas, las integraciones internas se usan dentro de una única organización de Sentry.

Las integraciones internas no requieren un flujo de OAuth. Recibes un token de autenticación para toda la organización inmediatamente después de la creación:

![Diagrama que muestra la diferencia de proceso entre integraciones públicas e internas.](./img/authentication-flow.png)

Para ver un ejemplo de cómo crear una integración interna, consulta [nuestra integración de asignación de issues Round Robin](https://blog.sentry.io/2019/11/21/customize-your-sentry-workflow-a-sample-internal-integration) (o ve directamente [al código en GitHub](https://github.com/getsentry/sentry-round-robin)).

<div id="installation">
  ## Instalación
</div>

Crear una integración interna la instalará automáticamente en tu organización.

<div id="auth-tokens">
  ## Tokens de autenticación
</div>

Las integraciones internas generan automáticamente un [token de autenticación](/es/organization/integrations/integration-platform/#auth-tokens) al configurarse. Si necesitas varios tokens o obtener uno nuevo, ve a **Settings &gt; Developer Settings &gt; [Your Internal Integration]** para hacerlo.
Puedes tener hasta 20 tokens a la vez por cada integración interna.
Estos tokens no caducan automáticamente, pero puedes revocarlos manualmente cuando sea necesario.

![Imagen que muestra la opción de copiar tokens, crear nuevos tokens, revocar tokens y la fecha de creación en UTC.](./img/tokens.png)

<div id="image-requirements">
  ## Requisitos de imagen
</div>

Las integraciones internas no requieren subir logotipos ni iconos, pero puedes hacerlo si quieres. Los requisitos para estos son:

* El tamaño del logotipo debe estar entre 256 × 256 px y 1024 × 1024 px
* El formato del archivo debe ser PNG
* El fondo debe ser transparente (a menos que el logotipo ocupe todo el espacio)

Si la integración tiene un [componente de UI](/es/organization/integrations/integration-platform/ui-components), también puedes subir un icono que cumpla estos requisitos:

* El tamaño del icono debe estar entre 256 × 256 px y 1024 × 1024 px
* El archivo debe ser un PNG en formato RGBA
* Todos los píxeles deben ser negros o transparentes
