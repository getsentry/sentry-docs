Las integraciones de Sentry suscritas a los webhooks de errores pueden recibir información cuando se crea un error.

<Include name="feature-available-for-plan-business.mdx" />

<div id="sentry-hook-resource-header">
  ## Header Sentry-Hook-Resource
</div>

`'Sentry-Hook-Resource': 'error'`

<div id="attributes">
  ## Atributos
</div>

<div id="action">
  ### acción
</div>

- type: string
- description: la única opción disponible por ahora es `created`

<div id="dataerror">
  ### data['error']
</div>

- type: objeto
- description: el error que se generó

<div id="dataerrorurl">
  ### data['error']['url']
</div>

- tipo: cadena
- descripción: la URL de la API del error

<div id="dataerrorweb_url">
  ### data['error']['web_url']
</div>

- tipo: string
- descripción: la URL del error en la web

<div id="dataerrorissue_url">
  ### data['error']['issue_url']
</div>

- tipo: string
- descripción: la URL de la API del issue asociado

<div id="dataeventissue_id">
  ### data['event']['issue_id']
</div>

- type: string
- description: el ID de la incidencia

Si has configurado la identificación de usuarios, puedes encontrar los atributos del usuario en `data['error']['user']`.
