---

Las integraciones de Sentry que están disponibles como [acciones de reglas de alerta](/es/organization/integrations/integration-platform/#alerts) pueden recibir webhooks de alertas de incidencias.

La siguiente demostración interactiva muestra cómo configurar una integración interna que puede recibir alertas de incidencias.

<Arcade src="https://demo.arcade.software/v7uhzmdV6Q5PDzoVPAE6?embed&show_copy_link=true" />

<div id="sentry-hook-resource-header">
  ## Header Sentry-Hook-Resource
</div>

`'Sentry-Hook-Resource': 'event_alert'`

<div id="attributes">
  ## Atributos
</div>

<div id="action">
  ### action
</div>

* type: string
* description: siempre será `triggered`

### data[&#39;event&#39;]

* type: objeto
* description: el evento que desencadenó la regla de alerta

### data[&#39;event&#39;][&#39;url&#39;]

* type: string
* description: la URL de la API del evento

### data[&#39;event&#39;][&#39;web&#95;url&#39;]

* type: string
* description: la URL del evento

### data[&#39;event&#39;][&#39;issue&#95;url&#39;]

* type: string
* description: la URL de la API de la incidencia asociada

### data[&#39;event&#39;][&#39;issue&#95;id&#39;]

* type: string
* description: el ID de la incidencia

### data[&#39;triggered&#95;rule&#39;]

* type: string
* description: la etiqueta de la regla que se activó

<Alert>
  Los dos campos siguientes son únicamente para [componentes de la interfaz de usuario de acciones de reglas de alerta](/es/organization/integrations/integration-platform/ui-components/alert-rule-action).
</Alert>

### data[&#39;issue&#95;alert&#39;][&#39;title&#39;]

* type: string
* description: la etiqueta de la regla que se disparó

### data[&#39;issue&#95;alert&#39;][&#39;settings&#39;]

* type: object
* description: la configuración guardada para enrutar la alerta dentro del servicio externo
