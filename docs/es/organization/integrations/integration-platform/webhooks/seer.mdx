Las integraciones de Sentry suscritas a los webhooks de Seer pueden recibir notificaciones sobre el proceso de [Issue Fix](/es/product/ai-in-sentry/seer/issue-fix/) de Seer. Estos webhooks te permiten seguir el progreso del análisis automatizado de la causa raíz, la generación de soluciones y las correcciones de código.

<div id="sentry-hook-resource-header">
  ## Sentry-Hook-Resource Header
</div>

`'Sentry-Hook-Resource': 'seer'`

<div id="webhook-types">
  ## Tipos de webhook
</div>

Los webhooks de Seer admiten siete tipos de eventos que cubren todo el ciclo de vida de la resolución de incidencias con IA:

<div id="root-cause-analysis">
  ### Análisis de la causa raíz
</div>

* `seer.root_cause_started` - Se activa cuando comienza el análisis de la causa raíz
* `seer.root_cause_completed` - Se activa cuando el análisis de la causa raíz finaliza con resultados

<div id="solution-generation">
  ### Generación de soluciones
</div>

* `seer.solution_started` - Se activa cuando comienza la generación de la solución
* `seer.solution_completed` - Se activa cuando se ha generado la solución

<div id="code-generation">
  ### Generación de código
</div>

* `seer.coding_started` - Se activa cuando comienza la generación o corrección de código
* `seer.coding_completed` - Se activa cuando los cambios de código están listos
* `seer.pr_created` - Se activa cuando se crean las solicitudes de extracción (pull requests)

<div id="common-attributes">
  ## Atributos comunes
</div>

Todos los webhooks de Seer comparten estos atributos:

<div id="action">
  ### action
</div>

* type: string
* description: El evento específico de Seer que se produjo (p. ej., `root_cause_started`, `solution_completed`)

### data[&#39;run&#95;id&#39;]

* tipo: entero
* descripción: Identificador único de esta ejecución de análisis de Seer

### data[&#39;group&#95;id&#39;]

* tipo: entero
* descripción: ID de la incidencia de Sentry que se está analizando

<div id="event-specific-payloads">
  ## Cargas útiles específicas de eventos
</div>

<div id="root-cause-started">
  ### Análisis de causa raíz iniciado
</div>

Payload mínimo que indica que el análisis ha comenzado:

```json
{
  "action": "root_cause_started",
  "actor": {
    "id": "sentry",
    "name": "Sentry",
    "type": "application"
  },
  "data": {
    "run_id": 12345,
    "group_id": 1170820242
  },
  "installation": {
    "uuid": "a8e5d37a-696c-4c54-adb5-b3f28d64c7de"
  }
}
```

          "timeline_item_type": "human_action",

          },
          "is_most_important_event": false

<div id="use-cases">
  ## Casos de uso
</div>

Los webhooks de Seer te permiten:

1. **Seguir el progreso**: Supervisar en tiempo real el estado de Seer Issue Fix
2. **Crear notificaciones**: Enviar notificaciones personalizadas cuando las correcciones estén listas
3. **Trazabilidad**: Mantener un registro de todas las correcciones de Seer para tus incidencias
4. **Integraciones personalizadas**: Crear herramientas a medida que reaccionen a los resultados del análisis de Seer

<div id="best-practices">
  ## Mejores prácticas
</div>

* Guarda el `run_id` para correlacionar eventos de webhook relacionados
* Usa el `group_id` para vincular los webhooks al issue original de Sentry
