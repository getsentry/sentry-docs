Las integraciones de Sentry suscritas a los webhooks de comentarios pueden recibir información cuando se crea un comentario y cuando cambia de estado.

<div id="sentry-hook-resource-header">
  ## Header Sentry-Hook-Resource
</div>

`'Sentry-Hook-Resource': 'comment'`

<div id="attributes">
  ## Atributos
</div>

<div id="action">
  ### acción
</div>

- tipo: string
- descripción: puede ser `created`, `updated` o `deleted`

<div id="datacomment">
  ### data["comment"]
</div>

- type: string
- description: el contenido del comentario del issue

<div id="dataproject-slug">
  ### data["project slug"]
</div>

- type: cadena
- description: el slug del proyecto al que pertenece el issue del comentario

<div id="datacomment_id">
  ### data["comment_id"]
</div>

- tipo: int
- descripción: el ID del comentario

<div id="dataissue_id">
  ### data["issue_id"]
</div>

- type: int
- description: el ID del issue

<div id="datatimestamp">
  ### data["timestamp"]
</div>

- type: datetime
- description: fecha y hora en que se creó, actualizó o eliminó el comentario

<div id="payload">
  ## Payload
</div>

```json
{
  "action": "created",
  "data": {
    "comment": "agregando un comentario",
    "project_slug": "sentry",
    "comment_id": 1234,
    "issue_id": 100,
    "timestamp": "2022-03-02T21:51:44.118160Z"
  },
  "installation": { "uuid": "eac5a0ae-60ec-418f-9318-46dc5e7e52ec" },
  "actor": { "type": "user", "id": 1, "name": "colleen" }
}
```
