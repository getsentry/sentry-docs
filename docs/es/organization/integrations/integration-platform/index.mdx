La plataforma de integración de Sentry ofrece una manera para que servicios externos interactúen con Sentry mediante [webhooks](/es/organization/integrations/integration-platform/webhooks/), [componentes de interfaz](/es/organization/integrations/integration-platform/ui-components/) y la [API REST](/es/api). Las integraciones que usan esta plataforma son actores de primera clase dentro de Sentry.

<div id="example-app">
  ## Aplicación de ejemplo
</div>

Si eres nuevo en la plataforma de integraciones, puede ayudarte comenzar con un proyecto de ejemplo. Para facilitarte las cosas, hemos creado una [aplicación de ejemplo](https://github.com/getsentry/integration-platform-example) que puedes usar como punto de partida. Muestra las distintas funciones y los datos disponibles para tu aplicación:

![Interfaz de usuario de una aplicación kanban de demostración creada para mostrar funciones de la plataforma de integraciones.](./img/reference-implementation-frontend.png)

Es una aplicación kanban que usa muchas de las funciones disponibles para las integraciones de Sentry y proporciona código base tanto en Python como en TypeScript. También incluye una guía paso a paso para configurar tu primera integración, además de documentación y consejos de pruebas. Por lo general, la aplicación debería estar consumiendo datos en vivo de Sentry de forma segura en cuestión de minutos.

<div id="creating-an-integration">
  ## Creación de una integración
</div>

En [sentry.io](https://sentry.io), ve a **Settings &gt; Developer Settings**. Desde allí, puedes crear una integración [interna](/es/organization/integrations/integration-platform/internal-integration) o [pública](/es/organization/integrations/integration-platform/public-integration). Las integraciones internas solo pueden ser usadas por tu organización, mientras que las integraciones públicas pueden publicarse y están disponibles para todos los usuarios de Sentry.

![Página de configuración que permite al desarrollador elegir entre integraciones públicas o internas.](./img/dev-settings.png)

Hay varias opciones de configuración para tu integración, que se describen brevemente a continuación. Cada sección incluye enlaces a información más detallada.

<div id="webhooks">
  ## Webhooks
</div>

Los webhooks permiten que tu servicio reciba solicitudes sobre recursos específicos, según lo que elijas. Para obtener más información, consulta la [documentación completa sobre webhooks](/es/organization/integrations/integration-platform/webhooks/).

Para recibir eventos de webhooks, debes especificar la URL del webhook al crear una integración. Una vez que hayas especificado la URL, podrás activar “Alert Rule Action” y crear alertas de incidencias y de métricas que envíen notificaciones a tu integración.

<div id="ui-components">
  ## Componentes de la interfaz
</div>

La plataforma de integraciones de Sentry permite agregar componentes de interfaz enriquecidos a [sentry.io](https://sentry.io) mediante una sintaxis simple y declarativa. Para más información, consulta la [documentación completa de los componentes de la interfaz](/es/organization/integrations/integration-platform/ui-components/).

<div id="alerts">
  ## Alertas
</div>

Puedes habilitar cualquier integración como acción en las [alertas de incidencias](/es/organization/integrations/integration-platform/webhooks/issue-alerts) y las [alertas de métricas](/es/organization/integrations/integration-platform/webhooks/metric-alerts) activando el interruptor &quot;Alert Rule Action&quot;. La integración aparecerá como un servicio en la sección de acciones al crear o actualizar una regla de alerta. La demo interactiva a continuación muestra cómo configurar una integración que puede recibir alertas de Sentry.

<Arcade src="https://demo.arcade.software/v7uhzmdV6Q5PDzoVPAE6?embed&show_copy_link=true" />

Para que tu servicio reciba webhooks de reglas de alerta, debes tener `Send a notification via <your integration>` como acción en la regla. Una vez configurado, empezarás a recibir solicitudes de webhook cuando se activen alertas. Para más información sobre la solicitud y el contenido del payload, consulta la [documentación completa de Webhooks](/es/organization/integrations/integration-platform/webhooks/). Si quieres que los usuarios proporcionen más información específica de la app para gestionar estos webhooks (p. ej., asignados, equipos, canales de notificación), considera implementar el [componente de interfaz de usuario para la acción de regla de alerta](/es/organization/integrations/integration-platform/ui-components/alert-rule-action).

<div id="permissions">
  ## Permisos
</div>

Los permisos especifican el nivel de acceso que tu servicio requiere a los recursos de Sentry. Los alcances se aplican directamente a cualquier token de API emitido para la aplicación. En las integraciones públicas, Sentry pedirá a los usuarios que aprueben estos permisos durante la instalación. Para obtener más información, consulta la [documentación completa sobre permisos](/es/api/permissions/).

![Formulario que permite al desarrollador establecer qué permisos necesitará del usuario.](./img/permissions.png)

No puedes crear una integración ni modificar una existente para que tenga permisos superiores a la configuración de tu propio usuario. Los permisos están limitados al ámbito de la organización, por lo que al añadir alcances para incidencias y eventos, proyectos o equipos, se habilitará el acceso a incidencias y eventos, proyectos o equipos en toda la organización.

<div id="auth-tokens">
  ## Tokens de autenticación
</div>

Los tokens de autenticación se envían mediante un encabezado de autenticación y se usan para autenticarse como una cuenta de usuario ante la API. Las [integraciones públicas](/es/organization/integrations/integration-platform/public-integration) requieren un flujo de OAuth para obtener tokens. Las [integraciones internas](/es/organization/integrations/integration-platform/internal-integration) generan tokens automáticamente después de la instalación. Para obtener más información, consulta la [documentación completa de autenticación](/es/api/auth/).
