Los componentes de UI [issue linking](/es/organization/integrations/integration-platform/ui-components/issue-link) y [alert rule action](/es/organization/integrations/integration-platform/ui-components/alert-rule-action) pueden usar componentes `FormField` para mostrar atributos en sus respectivos modales. Estos componentes pueden renderizar formularios personalizados con campos especificados según los esquemas que se indican a continuación. Se comportan de forma similar a sus equivalentes en HTML.

<div id="text">
  ## Texto
</div>

<div id="schema">
  ### Esquema
</div>

```json {filename:schema.json}
{
  "type": "text",
  "label": <String>,
  "name": <String>,
  "default": <String>,
}
```

<div id="attributes">
  ### Atributos
</div>

* `label` - (Obligatorio) Texto de la etiqueta que se mostrará para el campo del formulario
* `name` - (Obligatorio) Valor que se usará en el atributo `name` del campo
* `default` - valor predeterminado con el que se rellenará previamente. Las opciones incluyen `issue.title` y `issue.description`. Solo se usa para los componentes de [enlace de incidencia](/es/organization/integrations/integration-platform/ui-components/issue-link).
  * `issue.title` - título de la incidencia de Sentry
  * `issue.description` - descripción de la incidencia de Sentry

<div id="select">
  ## Selección
</div>

<div id="schema">
  ### Esquema
</div>

```json {filename:schema.json}
{
  "type": "select",
  "label": <String>,
  "name": <String>,
  "uri": <URI>,
  "async": <Boolean>,
  "options": <Array<Array<String, String>>>,
  "depends_on": <Array<String>>,
  "skip_load_on_open": <Boolean>
}
```

<div id="attributes">
  ### Atributos
</div>

* `label` - (Obligatorio) Texto de la etiqueta que se mostrará para el campo del formulario
* `name` - (Obligatorio) Valor que se usará en el atributo `name` del campo
* `uri` - (Obligatorio si el desarrollador no proporciona `options`) URI desde la que obtener valores. Consulta nuestra documentación de [Pautas de URI](/es/organization/integrations/integration-platform/ui-components/#uri-guidelines) para obtener ayuda con el formato.
* `async` - (Opcional) Se usa solo si `uri` está presente. Si es true (predeterminado), consultará la URI a medida que el usuario escriba para mostrar sugerencias de autocompletado (consulta el formato de respuesta a continuación). Si es false, consultará la URI una vez al inicio para recuperar todas las opciones posibles. Esta solicitud *debe* tener éxito y la respuesta *debe* devolver datos en el formato que Sentry espera; de lo contrario, no se renderizará el componente completo.
* `options` - (Obligatorio si el desarrollador no proporciona `uri`) Lista estática de opciones con el formato `[["value1", "label1"], ["value2", "label2"]]`
* `depends_on` - (Opcional) Si el valor de un campo depende del valor de otro, se realizará una solicitud para cargar esas opciones cuando se establezca el campo del que depende.
* `skip_load_on_open` (Opcional) Si es true, este campo no cargará opciones cuando la página se cargue. En su lugar, esperará a que el usuario empiece a escribir antes de obtener opciones.

<div id="uri-request-format">
  ### Formato de solicitud URI
</div>

La solicitud enviada al `uri` para cargar opciones es una solicitud `GET`. Los parámetros se codifican en la cadena de consulta de la siguiente manera:

```json
{
  "installationId": <String>,
  "projectSlug": <String>,
  "query": <String>,
  "dependentData": <String>,
}
```

* `installationId` - El ID de la instalación asociada con la solicitud
* `projectSlug` - El slug del proyecto en Sentry
* `query` - El término de búsqueda que el usuario está escribiendo
* `dependentData` - Una cadena codificada en JSON con los datos dependientes si ese campo tiene `depends_on` configurado

<div id="uri-response-format">
  ### Formato de respuesta de URI
</div>

La respuesta de `uri`, cuando se especifique, debe ser JSON y ajustarse al siguiente formato:

```json
[
  {
    "label": <String>,
    "value": <String>,
    "default": <Boolean>,
  },
  ...
]
```
