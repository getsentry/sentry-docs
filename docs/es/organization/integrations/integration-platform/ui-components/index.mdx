La plataforma de integraciones de Sentry permite a los desarrolladores definir componentes que se renderizan en partes específicas de [sentry.io](https://sentry.io), mediante un esquema basado en JSON.

Puedes definir los siguientes componentes:

* [Enlace a incidencia](/es/organization/integrations/integration-platform/ui-components/issue-link) (usa componentes [FormField](/es/organization/integrations/integration-platform/ui-components/formfield))
* [Acciones de reglas de alerta](/es/organization/integrations/integration-platform/ui-components/alert-rule-action) (usa componentes [FormField](/es/organization/integrations/integration-platform/ui-components/formfield))
* [Enlace de stack trace](/es/organization/integrations/integration-platform/ui-components/stacktrace-link)

<div id="schema">
  ## Esquema
</div>

Los componentes de la IU se especifican en la sección de esquema de los detalles de la integración. El formato requerido de este esquema es:

```json {filename: schema.json}
{
  "elements": [
    // Esquemas de componentes que utilizará la integración
    // p. ej.: { "type": "alert-rule-action", ... }
  ]
}
```

<div id="uri-guidelines">
  ### Directrices sobre URI
</div>

Las claves `uri` en los elementos de tu esquema describen la ruta de la URI que recibirá una solicitud de ese componente de la interfaz. Sentry anexará la ruta del campo `uri` al host de la URL del webhook de tu integración.
Por ejemplo, si la URL del webhook es `https://example.com/webhook/` y el atributo `uri` de tu esquema es `/alerts/`, la integración recibirá solicitudes en `https://example.com/alerts/`.

<div id="error-handling">
  ## Manejo de errores
</div>

El renderizado del componente o funciona al 100% o no muestra nada. Para evitar que la integración se vea caótica por errores fuera de nuestro control, si falla cualquier parte del renderizado de un componente de terceros, no se mostrará nada.

Si hay un error en un componente de vinculación de issues, el enlace se deshabilitará.
