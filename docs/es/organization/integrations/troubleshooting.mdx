Sentry se integra con muchos servicios de terceros para facilitar tu experiencia de desarrollo. Lamentablemente, no siempre podemos actualizar automáticamente algunos de estos servicios cuando se modifica información crítica dentro de Sentry. Este documento está pensado para ayudarte a entender el comportamiento esperado que puede producirse al cambiar estas configuraciones. En algunos casos, es posible que tu equipo deba intervenir manualmente para que la integración vuelva a funcionar sin inconvenientes.

<div id="if-youve-changed-your-organization-slug">
  ## Si cambiaste el slug de tu organización
</div>

* Cualquier integración que cree enlaces directamente a incidencias de Sentry seguirá creando enlaces correctos de ahora en adelante, **pero los enlaces existentes ya no funcionarán como se espera**. Esto incluye, por ejemplo, notificaciones de Slack con enlaces a alertas de Sentry y tickets de Jira que apuntan a incidencias de Sentry.
* La mayoría de las [integraciones de despliegue](/es/organization/integrations/#deployment) usan [Sentry CLI](https://github.com/getsentry/sentry-cli) bajo el capó, que requiere proporcionar el slug de tu organización mediante la variable de entorno `SENTRY_ORG`. Para que estas integraciones sigan funcionando, **debes actualizar manualmente esta variable en el servicio correspondiente**.
* Si usas cualquiera de las siguientes integraciones, debes actualizar manualmente el slug de la organización para mantener la funcionalidad:

  * [Bitbucket Pipelines](/es/product/releases/setup/release-automation/bitbucket-pipelines/)
  * [GitHub Actions](/es/product/releases/setup/release-automation/github-actions/)

  * [Vercel](/es/organization/integrations/deployment/vercel/)
  * [Grafana](/es/organization/integrations/data-visualization/grafana/)
  * [Split](/es/organization/integrations/feature-flag/split/)
  * [Segment](/es/organization/integrations/data-visualization/segment/)

  * [OpenReplay](/es/organization/integrations/session-replay/openreplay/)
