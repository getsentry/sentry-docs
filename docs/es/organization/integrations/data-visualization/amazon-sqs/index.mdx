<Alert>
  La integración con Amazon SQS está en desuso y no se puede instalar en proyectos nuevos.
</Alert>

Amazon SQS es útil para un análisis más profundo de las excepciones, lo que hace que sea rápido y sencillo canalizarlas de vuelta a tus propios sistemas. O úsala para potenciar a otros equipos, como el de Business Intelligence.

Esta integración debe configurarse en cada proyecto en el que quieras usarla. La mantiene y da soporte la [comunidad de Sentry](https://open.sentry.io/).

<div id="install-and-configure">
  ## Instalar y configurar
</div>

<Alert>
  Para instalar esta integración se necesitan permisos de propietario, gestor o administrador de Sentry.
</Alert>

Ve a **Settings &gt; Integrations &gt; Amazon SQS**

![Instalar la integración de Amazon SQS](./img/amazon-sqs.png)

<div id="data-forwarding">
  ### Reenvío de datos
</div>

Configura el [reenvío de datos](/es/concepts/data-management/data-forwarding/) en **[Project] &gt; Settings &gt; Data Forwarding** y proporciona la información necesaria para la integración correspondiente.

La carga útil para Amazon es idéntica a la carga útil de eventos de nuestra API estándar y evolucionará con el tiempo. Para obtener más detalles sobre el formato de estos datos, consulta nuestra [documentación de la API](/es/api/events/retrieve-an-event-for-a-project/).
