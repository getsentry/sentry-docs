Atlas es un software de atención al cliente todo en uno, diseñado para mostrar la experiencia de tus clientes en una vista simple y cronológica. Vincula fácilmente sesiones de Atlas con incidencias de Sentry y enlaza incidencias de Sentry con sesiones de Atlas existentes.

Esta integración es mantenida y respaldada por la empresa que la creó. Para más detalles, consulta nuestra [documentación de la plataforma de integraciones](/es/organization/integrations/integration-platform/).

<div id="session-linking">
  ## Vinculación de sesiones
</div>

La vinculación de sesiones te permite asociar incidencias de Sentry con sesiones en Atlas para que tu equipo de soporte pueda verlas cuando esté ayudando a tus clientes.

La vinculación de sesiones se intenta automáticamente si Sentry ha identificado a un usuario. Sin embargo, si hay varias sesiones simultáneas, es necesario desambiguar. Por este motivo, la mejor práctica es identificar siempre las sesiones de Atlas desde Sentry.

Para ello, primero obtén el ID de la sesión desde Atlas llamando a lo siguiente desde el cliente:

```javascript
apiAxios.interceptors.request.use((config) => {
  config.headers["X-Atlas-Session-Id"] =
    window.Atlas?.recording?.getSessionId();
  return config;
});
```

Puedes enviar esto a Sentry desde tu SDK de cliente:

```javascript
Sentry.addEventProcessor((event) => {
  event.tags = {
    ...(event.tags || {}),
    ["atlasSupport.sessionId"]: window.Atlas?.recording?.getSessionId(),
  };
  return event;
});
```

o desde el SDK del servidor:

```python
sentry_sdk.set_tag("atlasSupport.sessionId", atlas_session_id)
```

Ahora, los incidentes de Sentry aparecerán en la línea de tiempo del cliente en Atlas:

![Incidentes de Sentry en la línea de tiempo del cliente dentro de Atlas](./img/atlas_timeline.png)

También dentro de la grabación de la sesión:

![Incidentes de Sentry en la grabación de la sesión dentro de Atlas](./img/atlas_session.png)
