Sentry admite el almacenamiento de datos tanto en EE. UU. como en la UE.

Puedes elegir dónde almacenar tus datos al configurar tu cuenta de Sentry, seleccionando la opción en el menú desplegable “Data Storage Location” de la sección “Create a New Organization”.

**Ten en cuenta que, una vez seleccionada, la ubicación de almacenamiento de datos no se puede cambiar. La única manera de modificarla es creando una organización nueva.**

Tu selección determina únicamente la ubicación de almacenamiento de datos. Sentry seguirá accediendo y procesando tus datos conforme a tu acuerdo con Sentry para el servicio de Sentry y a nuestra [Política de privacidad](https://sentry.io/privacy/).

La tabla siguiente muestra la ubicación donde tus datos se almacenarán en reposo, según tu selección.

| **Data Storage Location**     | **Physical Location** |
| ----------------------------- | --------------------- |
| United States of America (US) | Iowa, Estados Unidos  |
| European Union (EU)           | Fráncfort, Alemania   |

<div id="what-types-of-data-are-stored-where">
  ## Qué tipos de datos se almacenan y dónde
</div>

Aquí tienes una lista de los tipos de datos que se almacenarán en la ubicación de almacenamiento de datos que elijas (EE. UU. o UE).

<div id="data-stored-in-selected-location-us-or-eu">
  ### Datos almacenados en la ubicación seleccionada (EE. UU. o UE)
</div>

* Eventos de error, actividad y enlaces a incidencias
* Transacciones
* Perfiles
* Estado de salud de las versiones
* Versiones, símbolos de depuración y mapas fuente
* Metadatos de símbolos de depuración y metadatos de mapas fuente
* Reproducciones de sesiones
* Copias de seguridad de estos recursos

<div id="data-stored-in-us">
  ### Datos almacenados en EE. UU.
</div>

Aquí tienes una lista de los tipos de datos que pueden almacenarse en EE. UU.

* Cuentas de usuario, preferencias de notificaciones y autenticadores de 2FA
  (Las cuentas de usuario se comparten entre muchas organizaciones).
* Metadatos de integraciones de la organización
  (Las integraciones pueden compartirse entre organizaciones y muchas de nuestras integraciones solo admiten enviar sus webhooks a un único destino).
* Tokens de acceso para usuarios y organizaciones
* Ajustes, configuraciones y equipos de la organización
* Registros de auditoría de la organización
* Check-ins de cron
* Metadatos de proyectos
* Claves DSN
* Datos de uso detallados
* Aplicaciones de Sentry
* Metadatos de SSO, SAML y SCIM

Los metadatos que permiten a Sentry identificar una organización se replicarán fuera de la ubicación de almacenamiento de datos de la organización para facilitar el inicio de sesión y mantener la compatibilidad con versiones anteriores de las API. Siempre puedes confirmar la ubicación de tu organización consultando la página de configuración de tu organización.

<div id="data-stored-in-all-locations-us-and-eu">
  ### Datos almacenados en todas las ubicaciones (EE. UU. y UE)
</div>

A continuación se muestra una lista de los tipos de datos que se almacenarán en ambas ubicaciones de almacenamiento (EE. UU. y UE).

* Comprobaciones de disponibilidad

<Alert title="Supervisión de disponibilidad" level="warning">
  Para que la supervisión de disponibilidad funcione eficazmente, realizamos comprobaciones desde múltiples ubicaciones geográficas. Como resultado, los datos de estas comprobaciones pueden almacenarse fuera de la región de datos que seleccionaste, más allá de los compromisos de almacenamiento descritos en esta página.
</Alert>

<div id="using-data-storage-location-apis">
  ## Uso de las API de ubicación del almacenamiento de datos
</div>

Para asegurarte de que tus solicitudes a la API solo se procesen dentro de la ubicación de almacenamiento de datos que seleccionaste, usa el dominio específico de la región:

| **Ubicación de almacenamiento de datos** | **Dominio de la API** |
| ---------------------------------------- | --------------------- |
| Estados Unidos (US)                      | us.sentry.io          |
| Unión Europea (EU)                       | de.sentry.io          |

Para [los datos almacenados en EE. UU.](#data-stored-in-us), tu dominio de la API debe ser `sentry.io`.

<div id="switching-data-storage-locations-for-existing-organizations">
  ## Cambiar la ubicación de almacenamiento de datos para organizaciones existentes
</div>

<div id="saas">
  ### SaaS
</div>

Por ahora, no puedes cambiar la ubicación de almacenamiento de datos si ya tienes una organización de SaaS. La única manera de cambiarla es creando una organización nueva.

<div id="self-hosted">
  ### Autogestionado
</div>

Si tienes una cuenta de Sentry autogestionada, puedes [seguir estas instrucciones](/es/concepts/migration/) para migrar tus datos.

**Compatible con la migración:**

* Cuentas de usuario y ajustes
* Ajustes y configuración de la organización
* Proyectos y ajustes de proyecto
* Equipos y miembros
* Paneles
* Reglas de alerta, notificaciones e incidentes
* Búsquedas guardadas

**No compatible con la migración:**

* Eventos de error, actividad y enlaces a incidencias
* Transacciones
* Perfiles
* Reproducciones de sesión
* Check-ins de Cron
* Comprobaciones de disponibilidad
* Claves DSN
* Estado de la versión
* Versiones, símbolos de depuración y mapas de código fuente
* Aplicaciones de Sentry
* Tokens de acceso para usuarios y organizaciones
* Ajustes de notificaciones de usuario y autenticadores 2FA
* Codeowners del proyecto
* Integraciones
* Registros de auditoría de la organización

<div id="for-users-with-multiple-organizations">
  ## Para usuarios con varias organizaciones
</div>

Si tienes varias organizaciones, se tratarán por separado sin importar dónde se almacenen los datos. Esto significa que las suscripciones, el uso, los usuarios, los proyectos, etc., se gestionarán de forma independiente para cada organización.
