Todos los eventos tienen una huella. Los eventos con la misma huella se agrupan en un issue.

De forma predeterminada, Sentry ejecuta uno de nuestros algoritmos de agrupación integrados para generar una huella basada en la información disponible en el evento, como `stacktrace`, `exception` y `message`. Para ampliar el comportamiento de agrupación predeterminado o cambiarlo por completo, puedes usar una combinación de las siguientes opciones:

1. En tu SDK, usando la huella del SDK, como se documenta a continuación
2. En tu proyecto, usando [Fingerprint Rules](/es/concepts/data-management/event-grouping/fingerprint-rules/) o [Stack Trace Rules](/es/concepts/data-management/event-grouping/stack-trace-rules/)

En los SDK compatibles, puedes sobrescribir la agrupación predeterminada de Sentry pasando el atributo de huella como un array de strings. La longitud del array de una huella no está restringida. Esto funciona de manera similar a la [funcionalidad de reglas de huella](/es/concepts/data-management/event-grouping/fingerprint-rules/), que siempre está disponible y puede lograr resultados similares.

Puedes usar sustitución de variables para insertar valores dinámicos en la huella que normalmente se calcula en el servidor. Por ejemplo, puedes añadir el valor `{{ default }}` para incluir en la huella todo el hash de agrupación que se genera normalmente. Estos valores son los mismos que los de la creación de huellas en el servidor. Consulta [Variables](/es/concepts/data-management/event-grouping/fingerprint-rules/#variables) para más información.
