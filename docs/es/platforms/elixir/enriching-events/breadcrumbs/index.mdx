Sentry utiliza *migas* para crear un rastro de eventos que ocurrieron antes de un problema. Estos eventos son muy similares a los registros tradicionales, pero pueden capturar datos estructurados más ricos.

Esta página ofrece una descripción general del registro manual de migas y su personalización. Obtén más información sobre lo que se muestra en la página **Detalles del problema** y cómo filtrar las migas para resolver incidencias rápidamente en [Uso de migas](/es/product/error-monitoring/breadcrumbs).

<Alert title="Más información sobre el uso del SDK">
  Los desarrolladores que deseen modificar la interfaz de migas pueden consultar nuestra [documentación para desarrolladores sobre la interfaz de migas](https://develop.sentry.dev/sdk/data-model/event-payloads/breadcrumbs/).
</Alert>

<div id="manual-breadcrumbs">
  ## Migas de pan manuales
</div>

Puedes agregar migas de pan manualmente cuando ocurra algo interesante. Por ejemplo, podrías registrar una miga de pan manual si el usuario se autentica o se produce otro cambio de estado.

Registrar una miga de pan manualmente:

<PlatformContent includePath="enriching-events/breadcrumbs/breadcrumbs-example" />

Las claves de miga de pan disponibles son `type`, `category`, `message`, `level`, `timestamp` (que muchos SDK establecerán automáticamente por ti) y `data`, que es el lugar donde incluir cualquier información adicional que quieras que contenga la miga de pan. Usar claves distintas de estas seis no provocará un error, pero hará que los datos se descarten cuando Sentry procese el evento.
