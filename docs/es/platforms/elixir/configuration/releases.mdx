Un lanzamiento es una versión de tu código que se despliega en un <PlatformLink to="/configuration/environments/">entorno</PlatformLink>. Cuando le proporcionas a Sentry información sobre tus lanzamientos, puedes:

* Detectar problemas y regresiones introducidos en un nuevo lanzamiento
* Predecir qué commit causó un problema y quién probablemente es responsable
* Resolver problemas incluyendo el número del issue en el mensaje del commit
* Recibir notificaciones por correo electrónico cuando se despliegue tu código

<div id="setting-a-release">
  ## Configurar una versión
</div>

<PlatformContent includePath="set-release" notateUnsupported />

Cómo poner a disposición de tu código el nombre de la versión (o release) depende de ti. Por ejemplo, puedes usar una variable de entorno establecida durante el proceso de compilación o en el arranque inicial.

Al configurar el nombre de la versión, cada evento se etiqueta con ese nombre. Recomendamos informar a Sentry sobre una nueva versión antes de enviar eventos con ese nombre, ya que esto habilita algunas funciones adicionales. Obtén más información en nuestra documentación de [Releases](/es/product/releases/).

Si no informas a Sentry sobre una nueva versión, Sentry creará automáticamente una entidad de versión en el sistema la primera vez que vea un evento con ese ID de versión.

Después de configurar tu SDK, puedes instalar una integración de repositorio o proporcionar manualmente a Sentry tus propios metadatos de commits. Lee nuestra documentación sobre [configurar versiones](/es/product/releases/setup/) para obtener más información sobre integraciones, asociación de commits y cómo informar a Sentry al desplegar versiones.
