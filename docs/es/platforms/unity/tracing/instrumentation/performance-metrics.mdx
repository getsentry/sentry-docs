Los SDK de Sentry permiten enviar datos de métricas de rendimiento a Sentry. Son valores numéricos asociados a transacciones que se agregan y se muestran en Sentry.

<div id="custom-measurements">
  ## Mediciones personalizadas
</div>

Además de las métricas de rendimiento automáticas, el SDK permite establecer [mediciones de rendimiento personalizadas](/es/product/insights/overview/metrics/#custom-performance-measurements) en transacciones. Esto te permite definir mediciones importantes para tu aplicación y enviarlas a Sentry.

Para establecer una medición de rendimiento, debes proporcionar lo siguiente:

* name (`string`)
* value (cualquier tipo numérico: `float`, `integer`, etc.)
* unit (`string`, de forma predeterminada la cadena `none` si se omite)

Sentry admite agregar unidades personalizadas arbitrarias, pero recomendamos usar una de las [unidades compatibles que se indican a continuación](./#supported-measurement-units).

<div id="supported-measurement-units">
  ## Unidades de medida admitidas
</div>

Las unidades dan contexto a los valores de medida al aportarles significado; de lo contrario, serían números abstractos. Añadir unidades también permite que Sentry ofrezca controles (conversiones, filtros, etc.) basados en esas unidades. Para valores sin unidad, puedes proporcionar una cadena vacía o `none`.

<div id="duration-units">
  ### Unidades de duración
</div>

* `nanosecond`
* `microsecond`
* `millisecond`
* `second`
* `minute`
* `hour`
* `day`
* `week`

<div id="information-units">
  ### Unidades de información
</div>

* `bit`
* `byte`
* `kilobyte`
* `kibibyte`
* `megabyte`
* `mebibyte`
* `gigabyte`
* `gibibyte`
* `terabyte`
* `tebibyte`
* `petabyte`
* `pebibyte`
* `exabyte`
* `exbibyte`

<div id="fraction-units">
  ### Unidades fraccionarias
</div>

* `ratio`
* `percent`

Si quieres profundizar, puedes encontrar detalles sobre las unidades admitidas en nuestra [documentación sobre la ingesta de eventos](https://getsentry.github.io/relay/relay_metrics/enum.MetricUnit.html).
