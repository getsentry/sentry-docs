Con el [tracing](/es/product/insights/overview/), Sentry puede [conectar eventos provenientes de diferentes partes de tu juego](/es/platforms/unity/usage/automatic-error-capture/#connecting-errors-captured-on-different-layers), ya sean plugins nativos o un backend. Te ayuda a depurar problemas complejos que involucran scripts de C#, así como la capa nativa en todas las plataformas. También te permite supervisar el rendimiento de tu juego, midiendo métricas como el tiempo de inicialización, el tiempo dedicado a la carga de escenas y los ANR. Además de vincular distintos errores entre sí, Sentry captura trazas distribuidas compuestas por transacciones y spans, que miden servicios individuales y operaciones individuales dentro de esos servicios. Obtén más información sobre nuestro modelo en [Distributed Tracing](/es/product/sentry-basics/tracing/distributed-tracing/).

<Alert>
  Si estás adoptando tracing en un entorno de alto volumen, te recomendamos realizar pruebas antes del despliegue para asegurarte de que las características de rendimiento de tu servicio se mantengan dentro de lo esperado.
</Alert>

<div id="configure">
  ## Configurar
</div>

Primero, habilita el tracing y configura la tasa de muestreo para las transacciones. Configura la tasa de muestreo de tus transacciones de una de las siguientes maneras:

* Estableciendo una tasa de muestreo uniforme para todas las transacciones usando la opción <PlatformIdentifier name="traces-sample-rate" /> en la configuración de tu SDK con un número entre `0` y `1`. (Por ejemplo, para enviar el 20 % de las transacciones, establece <PlatformIdentifier name="traces-sample-rate" /> en `0.2`).
* Controlando la tasa de muestreo según la propia transacción y el contexto en el que se captura, proporcionando una función a la opción de configuración <PlatformIdentifier name="traces-sampler" />.

Las dos opciones son mutuamente excluyentes. Si estableces ambas, <PlatformIdentifier name="traces-sampler" /> tendrá prioridad.

Obtén más información sobre las <PlatformLink to="/configuration/options/#tracing-options">opciones</PlatformLink> de tracing, cómo usar la función <PlatformLink to="/configuration/sampling/#setting-a-sampling-function">TracesSampler</PlatformLink> o cómo <PlatformLink to="/configuration/sampling/#sampling-transaction-events">muestrear transacciones</PlatformLink>.

<div id="verify">
  ## Verificar
</div>

Prueba el rastreo iniciando y finalizando una transacción; *debes* hacerlo para que las transacciones se envíen a Sentry. Aprende cómo en nuestro contenido de <PlatformLink to="/tracing/instrumentation/custom-instrumentation/">Instrumentación personalizada</PlatformLink>.

Mientras haces pruebas, configura <PlatformIdentifier name="traces-sample-rate" /> en `1.0`, ya que así te aseguras de que cada transacción se envíe a Sentry. Cuando termines, quizá quieras establecer un valor más bajo para <PlatformIdentifier name="traces-sample-rate" />, o pasar a usar <PlatformIdentifier name="traces-sampler" /> para muestrear y filtrar tus transacciones de forma selectiva según datos contextuales.
