Si el conjunto de aplicaciones que quieres observar con Sentry abarca más que un único servicio o aplicación, el trazado distribuido puede aportar mucho valor.

<div id="what-is-distributed-tracing">
  ## ¿Qué es el rastreo distribuido?
</div>

En el contexto de rastrear eventos en un sistema distribuido, el rastreo distribuido es una potente herramienta de depuración. Imagina tu aplicación como una gran red de partes interconectadas. Por ejemplo, tu sistema podría estar repartido en distintos servidores o tu aplicación podría dividirse en servicios de backend y frontend, cada uno con su propia pila tecnológica.

Cuando surge un error o un problema de rendimiento, puede resultar difícil identificar la causa raíz debido a la complejidad del sistema. El rastreo distribuido te ayuda a seguir el recorrido de un evento a medida que atraviesa esta red intrincada, registrando cada paso. Al examinar estos rastreos, puedes reconstruir la secuencia de sucesos que conduce al evento de interés, identificar los componentes específicos implicados y entender sus interacciones. Esta visibilidad detallada te permite diagnosticar y resolver problemas con mayor eficacia, mejorando en última instancia la fiabilidad y el rendimiento de tu sistema distribuido.

  ## ¿Cómo usar el rastreo distribuido?
</div>

Recuerda que, para propagar la información de las trazas por todo tu sistema distribuido, debes usar Sentry en todos los servicios y aplicaciones implicados. Consulta la documentación del SDK correspondiente para saber cómo habilitar el rastreo distribuido en cada plataforma.
