Sentry permite capturar automáticamente una pantalla e incluirla como un <PlatformLink to="/enriching-events/attachments/">archivo adjunto</PlatformLink> cuando un usuario experimenta un error, una excepción o un bloqueo.

Esta función solo está disponible para SDK con interfaz de usuario, como los de aplicaciones móviles y de escritorio. También está limitada por si es posible o no realizar la captura. Por ejemplo, en algunos entornos, como iOS nativo, capturar una pantalla requiere el hilo de la interfaz (UI), que a menudo no está disponible en caso de bloqueo. Otro ejemplo en el que quizá no haya captura es cuando el evento ocurre antes de que la pantalla empiece a cargarse. Por lo tanto, esta función es, por naturaleza, una solución de mejor esfuerzo.

<div id="enabling-screenshots">
  ## Activar capturas de pantalla
</div>

Como las capturas de pantalla pueden contener <PlatformLink to="/data-management/sensitive-data/">PII</PlatformLink>, esta función es opcional. Puedes activar las capturas de pantalla como se muestra a continuación mediante **Tools &gt; Sentry**:

![Opciones de configuración de capturas](./img/editor-window.png)

O bien, si estás [configurando todo por programación](/es/platforms/unity/configuration/options/):

```csharp {tabTitle:ScriptableOptionsConfiguration}
options.AttachScreenshot = true;
```

<div id="screenshot-capture-mechanism">
  ## Mecanismo de captura de pantalla
</div>

Dado que el SDK de Unity está compuesto internamente por varios SDK, el mecanismo con el que se captura una captura de pantalla depende del origen del error.

* Los errores de C# dentro de tu juego se capturan mediante el [ScreenCapture](https://docs.unity3d.com/ScriptReference/ScreenCapture.html) integrado de Unity. Esto significa que las capturas de pantalla solo incluyen lo que es visible dentro de tu juego. Las superposiciones sobre tu juego no serán visibles.
* Los errores nativos se capturan con su SDK correspondiente. Si estás usando un complemento nativo para mostrar una superposición y ocurre un error, ese SDK intentará capturar una captura de pantalla que incluya la superposición.

<Alert>
  Las capturas de pantalla pueden contener PII de tu app. Por ejemplo, si tu juego tiene un formulario de registro y ocurre un error mientras se muestra el formulario.
</Alert>

<div id="viewing-screenshots">
  ## Ver capturas de pantalla
</div>

Si hay una disponible, verás una miniatura de la captura cuando hagas clic en un issue específico desde la página de [**Issues**](https://demo.sentry.io/issues/).

![Miniatura de captura de pantalla](./img/screenshot-thumbnail.png)

Una vez que hayas hecho clic en el ID del evento de un issue específico, podrás ver un resumen de todos los adjuntos, así como los eventos asociados, en la pestaña &quot;Attachments&quot;.

![Ejemplo de lista de capturas de pantalla](./img/screenshot-list-example.png)
