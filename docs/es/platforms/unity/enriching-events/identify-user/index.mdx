Los usuarios constan de algunas piezas de información fundamentales que conforman una identidad única en Sentry. Cada una es opcional, pero una **debe** estar presente para que el SDK de Sentry capture al usuario:

<DefinitionList>
  ### `id`

  Tu identificador interno del usuario.

  ### `username`

  El nombre de usuario. Suele usarse como una etiqueta más útil que el id interno.

  ### `email`

  Una alternativa o complemento al nombre de usuario. Sentry reconoce las direcciones de correo electrónico y puede mostrar elementos como Gravatars y habilitar funciones de mensajería.

  ### `ip_address`

  La dirección IP del usuario. Si el usuario no está autenticado, Sentry usa la dirección IP como identificador único del usuario.
  Los SDK del lado del servidor que instrumentan solicitudes entrantes intentarán extraer la dirección IP de los datos de la solicitud HTTP (el campo `request.env.REMOTE_ADDR` en JSON), si está disponible. Esto podría requerir <PlatformIdentifier name="send-default-pii" /> establecido en `true` en las opciones del SDK.

  Si el `ip_address` del usuario se establece en `"{{auto}}"`, Sentry inferirá la dirección IP a partir de la conexión entre tu aplicación y el servidor de Sentry.

  Si se omite el campo, el valor predeterminado es `null`. Sin embargo, por compatibilidad con versiones anteriores, ciertas plataformas (en particular JavaScript) tienen un valor predeterminado diferente para `"{{auto}}"`. Los SDK y otros clientes no deben depender de este comportamiento y deben establecer direcciones IP o `"{{auto}}"` explícitamente.

  Para dejar de almacenar direcciones IP de los usuarios en los datos de tus eventos, ve a la configuración de tu proyecto, haz clic en &quot;Security &amp; Privacy&quot; y habilita &quot;Prevent Storing of IP Addresses&quot;, o usa el filtrado de [server-side data](/es/security-legal-pii/scrubbing/) de Sentry para eliminar `$user.ip_address`. Agregar esa regla, en última instancia, anula cualquier otra lógica.
</DefinitionList>

Además, puedes proporcionar pares arbitrarios de clave/valor más allá de los nombres reservados, y el SDK de Sentry los almacenará junto con el usuario.
