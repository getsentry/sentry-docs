Sentry puede enriquecer tus eventos para facilitar su investigación almacenando archivos adicionales —como archivos de configuración o registros— como adjuntos.

<div id="creating-attachments">
  ## Creación de archivos adjuntos
</div>

La forma más sencilla de crear un archivo adjunto es usar una `path`. El SDK leerá el contenido del archivo cada vez que prepare un evento o transacción y luego agregará el archivo adjunto al mismo [envelope](https://develop.sentry.dev/sdk/data-model/envelopes/). Si el SDK no puede leer el archivo, registrará un mensaje de error y descartará el archivo adjunto.

<PlatformContent includePath="enriching-events/attachment-init-with-path" />

Como alternativa, usa `bytes` para inicializar un archivo adjunto. En ese caso, también debes especificar un nombre de archivo.

<PlatformContent includePath="enriching-events/attachment-init-with-bytes" />

Con [Offline Caching](/es/platforms/unity/configuration/options/#InitCacheFlushTimeout) habilitado, cada archivo adjunto se guarda en disco por cada evento o transacción que captures cuando el dispositivo esté sin conexión. Al usar archivos adjuntos grandes, este almacenamiento puede consumir espacio en disco si el dispositivo permanece sin conexión durante un período prolongado.

Además, puedes configurar estos parámetros:

`filename`

El nombre de archivo es el nombre que se mostrará en Sentry. Al usar bytes debes especificar un nombre de archivo, mientras que con una path no, ya que el SDK utilizará el último componente de la ruta.

`contentType`

El tipo de contenido almacenado en este archivo adjunto. Se puede usar cualquier [tipo MIME](https://www.iana.org/assignments/media-types/media-types.xhtml); el valor predeterminado es `application/octet-stream`.

`mimetype`

El tipo específico de contenido multimedia que determina cómo se representa el archivo adjunto en la interfaz de Sentry. Actualmente admitimos y podemos mostrar los siguientes tipos MIME:

* `text/plain`
* `text/css`
* `text/csv`
* `text/html`
* `text/javascript`
* `text/json` o `text/x-json` o `application/json` o `application/ld+json`
* `image/jpeg`
* `image/png`
* `image/gif`
* `image/webp`
* `image/avif`
* `video/webm`
* `video/mp4`

<div id="uploading-attachments">
  ## Carga de archivos adjuntos
</div>

Unity incluye un <PlatformLink to="/enriching-events/scopes/">scope</PlatformLink> global al que puedes agregar un archivo adjunto que se enviará con cada evento.

<PlatformContent includePath="enriching-events/attachment-upload" />

<Alert>
  Sentry permite como máximo 20 MB para una solicitud comprimida y hasta 100 MB de archivos adjuntos sin comprimir por evento, incluido el archivo de informe de bloqueo (si corresponde). Las cargas que superen este tamaño se rechazan con el error HTTP `413 Payload Too Large` y los datos se descartan de inmediato. Para agregar archivos más grandes o más archivos, considera opciones de almacenamiento secundario.
</Alert>

Los archivos adjuntos se conservan durante 30 días; si superas el almacenamiento total incluido en tu cuota, no se guardarán. Puedes eliminar archivos adjuntos o los eventos que los contienen en cualquier momento. Eliminar un archivo adjunto no afecta tu cuota: Sentry lo contabiliza en cuanto se almacena.

Obtén más información sobre cómo los archivos adjuntos afectan tu [cuota](/es/pricing/quotas/).

<div id="access-to-attachments">
  ### Acceso a los adjuntos
</div>

Para limitar el acceso a los adjuntos, ve a la **Configuración general** de tu organización y selecciona el menú desplegable *Acceso a los adjuntos* para establecer el acceso adecuado: cualquier miembro de tu organización; el propietario de la facturación de la organización; miembro, administrador, manager o propietario.

<Include name="common-imgs/attachments-access" />

De forma predeterminada, el acceso se otorga a todos los miembros cuando el almacenamiento está habilitado. Si un miembro no tiene acceso al proyecto, no podrá descargar un adjunto; el botón aparecerá atenuado en Sentry. El miembro solo podrá ver que hay un adjunto almacenado.

<div id="viewing-attachments">
  ## Ver archivos adjuntos
</div>

Los archivos adjuntos se muestran en la parte inferior de la página de **Detalles de la incidencia** del evento que se muestra.

<Include name="common-imgs/attachments-access-denied" />

También puedes encontrar los archivos adjuntos en la pestaña *Attachments* de la página de **Detalles de la incidencia**, donde puedes ver el *Type* del archivo adjunto, así como los eventos asociados. Haz clic en el ID del evento para abrir los **Detalles de la incidencia** de ese evento específico.

<div id="maximum-attachment-size">
  ## Tamaño máximo de archivo adjunto
</div>

El tamaño máximo de cada archivo adjunto se configura en `SentryOptions.MaxAttachmentSize`.

La unidad es bytes y el valor predeterminado es `20 MiB`. Consulta también el
[tamaño máximo de archivos adjuntos de Relay](/es/product/relay/options/)
para asegurarte de que tus archivos adjuntos no se descarten allí.
