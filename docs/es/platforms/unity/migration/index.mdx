<div id="migrating-to-300">
  ## Migración a 3.0.0
</div>

<div id="changes-to-the-initialization-behaviour-on-ios-and-android">
  ### Cambios en el comportamiento de inicialización en iOS y Android
</div>

La capa nativa en plataformas móviles (iOS y Android) ya no se autoinicializa antes de que se inicie el motor de juego de Unity. Antes, el SDK usaba las opciones en tiempo de compilación y las incorporaba en la capa nativa. Ahora, el SDK tomará las opciones pasadas al callback `Configure` y las usará para inicializar los SDK nativos. Esto permite a los usuarios modificar programáticamente las opciones del SDK nativo en tiempo de ejecución. El comportamiento de inicialización está controlado por las opciones `IosNativeInitializationType` y `AndroidNativeInitializationType`.

Para restaurar el comportamiento anterior, puedes establecer las opciones `IosNativeInitializationType` y `AndroidNativeInitializationType` en `BuildTime`.

<div id="changes-to-the-programmatic-configuration">
  ### Cambios en la configuración mediante código
</div>

Las configuraciones de Runtime y BuildTime se han unificado en un único script, OptionsConfiguration. Esto permite configurar el SDK en un solo lugar usando directivas del preprocesador, en lugar de tener que duplicar opciones en dos archivos distintos.

Aún puedes usar las clases de configuración anteriores, pero están obsoletas y se eliminarán en una versión futura. De ahora en adelante, deberías usar el nuevo script OptionsConfiguration y aprovechar las directivas del preprocesador al establecer opciones para diferentes plataformas.

<div id="migrating-to-0280">
  ## Migración a la 0.28.0
</div>

<div id="breaking-changes">
  ### Cambios importantes
</div>

Se han actualizado las clases base de los Scriptable Objects usados para la configuración mediante código:

* `Sentry.Unity.ScriptableOptionsConfiguration` ha pasado a ser `SentryRuntimeOptionsConfiguration`
* `Sentry.Unity.Editor.ScriptableOptionsConfiguration` ha pasado a ser `SentryBuildtimeOptionsConfiguration`

Si utilizas la configuración mediante código, tendrás que actualizar tu implementación con estas clases base.

<div id="migrating-to-0110">
  ## Migración a la 0.11.0
</div>

<div id="programmatic-configuration-changes">
  ### Cambios de configuración por código
</div>

Ya no necesitas llamar a `SentryUnity.Init`. En su lugar, puedes dejar que el SDK se inicialice automáticamente y modificar las opciones que toma de la ventana de configuración del editor de Sentry Unity proporcionando un objeto `ScriptableOptionsConfiguration`.

* Abre la ventana de configuración del editor de Sentry Unity.
* Ve a la pestaña `Options Config`.
* Haz clic en `Create options configuration`.
  * Se te pedirá que elijas una ubicación para tu versión del script de configuración de opciones scriptable.
  * También se creará una instancia de `ScriptableOptionsConfiguration` y se establecerá en la ventana de configuración.
* Agrega tu código al script recién creado.

Obtén más información en nuestra [documentación de opciones](/es/platforms/unity/configuration/options/).

<div id="migrating-from-070-or-earlier">
  ## Migración desde la 0.7.0 o anteriores
</div>

A partir de la versión 0.7.0, el SDK está [aplicando alias](https://blog.sentry.io/2022/02/24/alias-an-approach-to-net-assembly-conflict-resolution) a sus dependencias. Hacemos esto para evitar conflictos con otros paquetes. Esto también significa que, si dependías de nuestras dependencias para ciertos tipos como `System.Text.Json`, a partir de ahora tendrás que agregarlas a tu proyecto por tu cuenta.

<div id="migrating-from-030-or-earlier">
  ## Migrar desde la 0.3.0 o anteriores
</div>

El SDK de Sentry dejó de usar archivos JSON para almacenar opciones y ahora utiliza scriptable objects. Si migras desde la versión 0.3.0 o anterior, asegúrate de actualizar primero a la versión 0.15.0. Es la última versión que admite opciones almacenadas en formato JSON. Cuando abras la ventana del editor de configuración de Sentry en la versión 0.15.0, tu archivo JSON se convertirá automáticamente en un scriptable object. Esta compatibilidad se eliminará a partir de la versión 0.16.0 y posteriores.

<div id="migrating-from-sentryunitylite-to-sentryunity">
  ## Migración de `sentry.unity.lite` a `sentry.unity`
</div>

<Alert>
  El SDK Sentry Unity Lite está en desuso para dispositivos móviles, equipos de escritorio, WebGL y consolas.
  Los clientes que usan Unity 5.x aún pueden usar Sentry Unity Lite, ya que sigue siendo compatible con sentry.io.

  La versión actualizada del SDK de Sentry Unity requiere Unity 2019.4 o posterior con el perfil de scripting .NET Standard 2.0.
</Alert>

1. Elimina los archivos antiguos `Sentry.cs` y `SentrySdk.cs` de tu proyecto.
2. Elimina el código de inicialización antiguo `gameObject.AddComponent<SentrySdk>().Dsn = "___PUBLIC_DSN___";`.
3. Instala el nuevo [SDK de Sentry para Unity](/es/platforms/unity/).
4. Las llamadas a la API como `SentrySdk.CaptureMessage("Test event");` seguirán funcionando y no requieren ningún cambio.
5. Revisa la documentación para conocer más sobre las nuevas capacidades del SDK, como [enriquecimiento de eventos](/es/platforms/unity/enriching-events/), [tracing](/es/platforms/unity/tracing/) y [configuración](/es/platforms/unity/configuration/).
