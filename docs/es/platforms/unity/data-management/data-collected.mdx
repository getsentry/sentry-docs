---
title: Datos recopilados
description: "Consulta qué datos recopila el SDK de Sentry."
sidebar_order: 1
---

Sentry se toma muy en serio la privacidad de los datos y cuenta con configuraciones predeterminadas que priorizan la seguridad de la información, especialmente en lo referente a datos de identificación personal (PII). Cuando agregas el SDK de Sentry a tu aplicación, le permites recopilar datos y enviarlos a Sentry durante la ejecución de tu aplicación.

Los tipos de categorías y la cantidad de datos recopilados varían según las integraciones que hayas habilitado en el SDK de Sentry. A continuación, encontrarás una lista de las categorías de datos que recopila el SDK de Sentry para .NET:

<div id="http-headers">
  ## Encabezados HTTP
</div>

De forma predeterminada, el SDK de Sentry envía encabezados HTTP a Sentry, pero redacta cualquier encabezado que pueda contener datos confidenciales. (Consulta la [lista de encabezados](https://github.com/getsentry/sentry-dotnet/blob/main/src/Sentry/Internal/RedactedHeaders.cs#L7) que se redactan).

<div id="cookies">
  ## Cookies
</div>

De forma predeterminada, el SDK de Sentry no envía cookies.

Si quieres enviar cookies, establece la opción `SendDefaultPii = true` al inicializar el SDK de Sentry.

Puedes usar un [filtro de datos](/es/platforms/dotnet/data-management/sensitive-data/) para depurar información antes de enviarla a Sentry.

Cualquier [cookie confidencial](https://github.com/getsentry/relay/blob/249971e1702991c64cf0f7f89773fb019ffb4a95/relay-pii/src/convert.rs#L235-L263) que el SDK sí envíe será eliminada por Relay antes de que Sentry la almacene.

<div id="information-about-logged-in-user">
  ## Información sobre el usuario autenticado
</div>

De forma predeterminada, el SDK de Sentry no envía información sobre el usuario autenticado, como la dirección de correo electrónico, el ID de usuario o el nombre de usuario. Incluso si se habilita, el tipo de información del usuario autenticado que podrás enviar depende de las integraciones que actives en el SDK de Sentry. La mayoría de las integraciones no enviarán ningún dato del usuario. Algunas solo establecerán el ID de usuario, pero hay algunas que establecerán el ID de usuario, el nombre de usuario y la dirección de correo electrónico.

Para empezar a enviar información del usuario autenticado, establece la opción `SendDefaultPii = true` al inicializar el SDK de Sentry.

<div id="users-ip-address">
  ## Dirección IP de los usuarios
</div>

De forma predeterminada, el SDK de Sentry no envía la dirección IP del usuario.

Para habilitar el envío de la dirección IP del usuario, establece la opción `SendDefaultPii = true` al inicializar el SDK de Sentry.

Incluso si está habilitado, que se envíe o no la dirección IP del usuario dependerá de las integraciones que actives en el SDK de Sentry. La mayoría de las integraciones no establecerán la dirección IP del usuario.

Puedes [evitar que Sentry almacene direcciones IP](/es/platforms/native/enriching-events/identify-user/#ip_address) ya sea configurándolo en los ajustes de &quot;Security &amp; Privacy&quot; de tu proyecto de Sentry o usando [depuración de datos en el lado del servidor](/es/security-legal-pii/scrubbing/server-side-scrubbing/).

<div id="request-url">
  ## URL de la solicitud
</div>

De forma predeterminada, el [HttpRequestUrl](https://learn.microsoft.com/en-us/dotnet/api/System.UriComponents) de las solicitudes HTTP salientes y entrantes **siempre se envía a Sentry**. Esto incluye Scheme, Host, Port, LocalPath y Query.

Si `SendDefaultPii` se establece en `true`, entonces el [AbsoluteUri](https://learn.microsoft.com/en-us/dotnet/api/System.UriComponents) de las solicitudes HTTP salientes y entrantes se enviará a Sentry (incluidos Scheme, UserInfo, Host, Port, LocalPath, Query y Fragment).

Según tu aplicación, esto podría contener datos de información personal (PII).

<div id="request-query-string">
  ## Cadena de consulta de la solicitud
</div>

La cadena de consulta completa de las solicitudes HTTP entrantes y salientes **siempre se envía a Sentry**. Según tu aplicación, esto podría incluir datos personales (PII).

<div id="sql-queries">
  ## Consultas SQL
</div>

Aunque se envían consultas SQL a Sentry, ni la consulta completa (`SELECT * FROM users WHERE email = 'joe@shmoe.com'`) ni los valores de sus parámetros se envían nunca. En su lugar, se envía una versión con parámetros de la consulta (`SELECT * FROM users WHERE email = @email`).

<div id="device-unique-identifier">
  ## Identificador único del dispositivo
</div>

El SDK de Sentry no establece el `Device Unique Identifier` de forma predeterminada.

Si `SendDefaultPii` se establece en `true`, el SDK capturará automáticamente `SystemInfo.deviceUniqueIdentifier` y lo enviará con cualquier evento.

<div id="machine-name">
  ## Nombre de la máquina
</div>

El SDK de Sentry no envía el nombre de la máquina de forma predeterminada.

Si asignas un valor a `ServerName` en las opciones de Sentry, el SDK de Sentry lo enviará con cualquier evento.

De lo contrario, si `SendDefaultPii` está configurado en true, el SDK de Sentry capturará `Environment.MachineName` automáticamente y lo enviará con cualquier evento.

<div id="general-device-information">
  ## Información general del dispositivo
</div>

El SDK de Sentry recopila información sobre el dispositivo, como la versión y la compilación del sistema operativo, detalles de la CPU y la GPU, etc. Esta información se envía a Sentry de forma predeterminada.

<div id="data-captured-by-mobile-sdks">
  ## Datos recopilados por los SDK móviles
</div>

En plataformas móviles, el SDK de Sentry para .NET también puede recopilar datos de forma indirecta mediante los SDK de Android y iOS. Consulta la documentación de esos SDK para obtener más información sobre qué datos se recopilan.

* [Android](/es/platforms/android/data-management/data-collected/)
* [iOS](/es/platforms/apple/data-management/data-collected/)

<div id="data-captured-by-the-native-sdk">
  ## Datos recopilados por el SDK nativo
</div>

Al compilar para Windows, Linux u otras plataformas que utilizan Sentry Native, consulta los [datos recopilados por el SDK nativo](/es/platforms/native/data-management/data-collected/) para comprender qué datos se envían con esos eventos.
