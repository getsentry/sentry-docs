Los archivos de información de depuración permiten a Sentry extraer trazas de pila y proporcionar más
información sobre los informes de fallos en la mayoría de las plataformas compiladas. La información almacenada
en los archivos de depuración incluye nombres originales de funciones, rutas a archivos fuente y números de línea,
contexto del código fuente o la ubicación de variables en memoria. Sentry
puede usar parte de esta información y mostrarla en la página de detalles de la incidencia.

Cada plataforma principal utiliza distintos archivos de información de depuración. Actualmente admitimos
los siguientes formatos:

* [*archivos dSYM*](./file-formats/#mach-o-and-dsym) para iOS, iPadOS,
  tvOS, watchOS, macOS y visionOS
* [*símbolos ELF*](./file-formats/#executable-and-linkable-format-elf) para Linux y
  Android (NDK)
* [*archivos PDB*](./file-formats/#pe-and-pdb) para Windows y .NET
* [*símbolos Breakpad*](./file-formats/#breakpad-symbols) para todas
  las plataformas
* [*archivos WASM*](./file-formats/#wasm) para WebAssembly
* [*mapeos de ProGuard*](./file-formats/#proguard-mappings) para
  Java y Android

<Alert>
  Los source maps, aunque también son archivos de información de depuración, se gestionan
  de forma diferente en Sentry. Para más información, consulta [Source Maps en sentry-cli](/es/cli/releases/#sentry-cli-sourcemaps).
</Alert>

Sentry necesita acceso a los archivos de información de depuración de tu aplicación, así como a
las bibliotecas del sistema, para ofrecer informes de fallos completamente simbolizados. Puedes
[subir](./upload/) tus archivos a Sentry o colocarlos en un
[Symbol Server](./symbol-servers/) compatible para que Sentry los descargue cuando sea necesario.

<div id="managing-debug-information-files">
  ## Gestión de archivos de información de depuración
</div>

Después de cargarlos, los archivos de información de depuración se pueden ver y administrar
desde la sección *Debug Files* en la página de configuración del proyecto correspondiente. Esa página
muestra todos los archivos de depuración cargados y también permite configurar servidores de símbolos
para descargas automáticas.

Desde la página [Project Details](/es/product/projects/project-details/), ve a Configuración
y luego haz clic en *Debug Files* en la navegación de la página.

<Alert>
  Los archivos de ProGuard se muestran por separado, en la sección *ProGuard* de la página de configuración del proyecto.
</Alert>

<div id="retention-policy">
  ## Política de retención
</div>

Los archivos de depuración tienen un periodo de retención de *90 días*, mediante un mecanismo de caducidad por *tiempo de inactividad*.
Esto significa que los archivos de depuración cargados se conservan mientras se utilicen activamente para el procesamiento de eventos.
Cuando un archivo de depuración no se haya usado para procesar eventos entrantes durante al menos 90 días,
caducará automáticamente y podrá eliminarse.
