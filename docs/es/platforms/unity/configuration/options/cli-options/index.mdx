El SDK de Unity incluye [Sentry CLI](/es/product/cli/) como herramienta para cargar símbolos de depuración en Sentry. Estos símbolos se usan al procesar eventos enviados a Sentry para proporcionar trazas de pila y números de línea. Esta página describe las opciones disponibles para configurar Sentry CLI.

<div id="programmatic-configuration">
  ## Configuración mediante código
</div>

Al igual que la [configuración mediante código](/es/platforms/unity/configuration/options/programmatic-configuration/) del propio SDK, la carga de símbolos de depuración con Sentry CLI también admite configuración mediante código.

![Configuración de opciones](./img/sentry-cli-option-configuration.png)

El callback `Configure` se invoca durante el proceso de compilación y te permite modificar las opciones que usa Sentry CLI.

<div id="sentry-cli-options">
  ## Opciones de Sentry CLI
</div>

<SdkOption name="UploadSymbols" type="bool" categorySupported={["desktop"]}>
  Si el SDK debe subir automáticamente símbolos de depuración a Sentry. Esto requiere un token de autenticación válido, una organización y un proyecto.
</SdkOption>

<SdkOption name="UploadDevelopmentSymbols" type="bool" categorySupported={["desktop"]}>
  Si el SDK debe subir automáticamente símbolos de depuración a Sentry para compilaciones de desarrollo.
</SdkOption>

<SdkOption name="UploadSources" type="bool" categorySupported={["desktop"]}>
  Si el SDK debe subir automáticamente el código fuente a Sentry; esto permite que Sentry proporcione contexto de código fuente en los eventos.
</SdkOption>

<SdkOption name="UrlOverride" type="string" categorySupported={["desktop"]}>
  Esta opción te permite sobrescribir la URL predeterminada de Sentry en caso de que uses una instancia de Sentry autogestionada o un dominio personalizado.
</SdkOption>

<SdkOption name="Auth" type="string" categorySupported={["desktop"]}>
  El token de autenticación que se usará al subir símbolos a Sentry.
</SdkOption>

<SdkOption name="Organization" type="string" categorySupported={["desktop"]}>
  La organización que se usará al subir símbolos a Sentry.
</SdkOption>

<SdkOption name="Project" type="string" categorySupported={["desktop"]}>
  El proyecto que se usará al subir símbolos a Sentry.
</SdkOption>

<SdkOption name="IgnoreCliErrors" type="bool" categorySupported={["desktop"]}>
  Si se establece en true, Sentry CLI no terminará con un código de error si se encuentra con un error. Ten en cuenta que podrías tener fallos sin minificar/sin simbolizar en producción si falla la subida de símbolos de depuración. Al usar esta opción, deberías almacenar los sourcemaps generados y los archivos de depuración para volver a ejecutar el comando de subida de símbolos más adelante.
</SdkOption>
