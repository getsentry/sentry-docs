<Alert>
  La tasa de sesiones sin fallos requiere [compatibilidad nativa](/es/platforms/unity/native-support/). En las plataformas que aún no la tienen, el SDK marca las sesiones que no finalizaron correctamente como *anormales*, lo que hace que se registre un 100% de sesiones sin fallos. Si estás desarrollando en una plataforma que todavía no tiene compatibilidad nativa, deberías monitorear la tasa *anormal* de tu app.
</Alert>

Un lanzamiento es una versión de tu código que se despliega en un <PlatformLink to="/configuration/environments/">entorno</PlatformLink>. Cuando le das a Sentry información sobre tus lanzamientos, puedes:

* Identificar problemas y regresiones introducidos en un nuevo lanzamiento
* Predecir qué commit causó un problema y quién es probablemente responsable
* Resolver problemas incluyendo el número del issue en tu mensaje de commit
* Recibir notificaciones por correo electrónico cuando se despliegue tu código

<div id="setting-a-release">
  ## Configurar una versión
</div>

<PlatformContent includePath="set-release" notateUnsupported />

Cómo hagas que el nombre de la versión (o release) esté disponible para tu código depende de ti. Por ejemplo, podrías usar una variable de entorno definida durante el proceso de compilación o durante el arranque inicial.

Al configurar el nombre de la versión, cada evento se etiqueta con ese nombre. Te recomendamos informar a Sentry de una nueva versión antes de enviar eventos con ese nombre, ya que esto habilitará algunas funciones adicionales. Obtén más información en nuestra documentación de [Releases](/es/product/releases/).

Si no informas a Sentry de una nueva versión, Sentry creará automáticamente una entidad de versión en el sistema la primera vez que vea un evento con ese ID de versión.

Después de configurar tu SDK, puedes instalar una integración de repositorio o proporcionar manualmente a Sentry tus propios metadatos de commits. Lee nuestra documentación sobre [configuración de versiones](/es/product/releases/setup/) para obtener más información sobre integraciones, asociación de commits y cómo informar a Sentry al implementar versiones.

<div id="release-health">
  ## Estado de las versiones
</div>

Supervisa el [estado de las versiones](/es/product/releases/health/) observando la adopción por parte de los usuarios, el uso de la aplicación, el porcentaje de [crashes](/es/product/releases/health/#crashes) y los [datos de sesión](/es/product/releases/health/#sessions). El estado de las versiones te proporcionará información sobre el impacto de los crashes y los errores en la experiencia del usuario, y revelará tendencias con cada nuevo incidente mediante los gráficos y filtros de [Detalles de la versión](/es/product/releases/release-details/).

Para poder supervisar el estado de las versiones, el SDK envía datos de sesión.

<div id="sessions">
  ### Sesiones
</div>

Una sesión representa la interacción entre el usuario y la aplicación. Las sesiones incluyen una marca de tiempo, un estado (si la sesión fue correcta o si se bloqueó) y siempre están vinculadas a una versión. La mayoría de los SDK de Sentry pueden gestionar las sesiones automáticamente.

<PlatformContent includePath="configuration/auto-session-tracking" />

<Alert>
  El estado de salud de versiones para .NET está actualmente limitado a aplicaciones **cliente** de instancia única. Esto significa que puede usarse con modelos de aplicación como WPF, Xamarin, WinForms y similares.
  Se tiene previsto añadir compatibilidad con frameworks de servidor como ASP.NET.
</Alert>
