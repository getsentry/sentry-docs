<div id="migrating-from-iosentrysentry-5x-to-iosentrysentry-600">
  ## Migración de `io.sentry:sentry` `5.x` a `io.sentry:sentry` `6.0.0`
</div>

* El complemento de Kotlin se actualiza a `1.5`.

* La `languageVersion` de Kotlin se actualiza a `1.4`.

* Se elimina `Gson` como dependencia transitiva y se incorpora en el SDK.
  * Las clases del protocolo ahora implementan las interfaces `JsonSerializable` y `JsonDeserializer`.

* Spring Boot se ha actualizado a `2.5.13`.

* `SentryOptions#shutdownTimeout` pasa a llamarse `shutdownTimeoutMillis`.

* Se han eliminado los métodos `@Deprecated` y `@ApiStatus.ScheduledForRemoval`
  * `ITransaction#setRequest`
  * `ITransaction#getRequest`
  * `ITransaction#getContexts`
  * `SentryBaseEvent#getOriginThrowable`
  * `SentryOptions#getCacheDirSize`
  * `SentryOptions#setCacheDirSize`
  * `SentryOptions#isEnableSessionTracking`
  * `SentryOptions#setEnableSessionTracking`
  * Propiedad `sentry.enable-tracing`
  * `SentrySpringRequestListener`, se usa `SentrySpringFilter` en su lugar.
  * `SentryUserProviderEventProcessor`, usa `SentryUserProvider` en su lugar.

* `SentryOptions#enableScopeSync` ahora está habilitado de forma predeterminada.

* `ISpan` ahora tiene mayor precisión usando `System#nanoTime` en lugar de milisegundos.

* `TransactionNameProvider` ahora es una interfaz y `SpringMvcTransactionNameProvider` es la implementación predeterminada.

* `Hints` cambió su tipo de `Object` a `io.sentry.Hint`.

*Antiguo*:

```kotlin
Sentry.captureException(RuntimeException("excepción"), "miHintDeCadena")
```

*Nuevo*:

```kotlin
val hints = mutableMapOf<String, Any>("miHint" to "miHintDeCadena")
Sentry.captureException(RuntimeException("excepción"), hints)
```

* `SentryOptions#sendClientReports` ahora está habilitado de forma predeterminada. Para desactivarlo, consulta [sendClientReports](/es/platforms/java/configuration/#client-reports). Para obtener más información sobre la opción, consulta [sendClientReports](/es/platforms/java/configuration/options/#sendClientReports).

<div id="sentry-self-hosted-compatibility">
  ### Compatibilidad con Sentry autoalojado
</div>

* A partir de la versión `6.0.0` de `sentry`, se requiere [Sentry en la versión &gt;= v21.9.0](https://github.com/getsentry/self-hosted/releases) o debes desactivar manualmente el envío de informes del cliente mediante la opción `sendClientReports`. Esto solo aplica a Sentry autoalojado. Si usas [sentry.io](https://sentry.io), no necesitas hacer nada.

Hay más cambios y refactorizaciones, pero no afectan a las personas usuarias.
