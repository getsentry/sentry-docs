El SDK de Sentry para Java almacena el scope y el contexto en una variable local de hilo. Para asegurarte de que un método asíncrono tenga acceso al contexto correcto de Sentry, debes configurar un `SentryTaskDecorator` en el `ThreadPoolTaskExecutor`.

<div id="configuring-spring-mvc-async-task-executor">
  ## Configuración del ejecutor de tareas asíncronas de Spring MVC
</div>

Para propagar el scope de Sentry a métodos de Spring MVC que devuelven un `Callable` o un `StreamingResponseBody`, se debe configurar un `ThreadPoolTaskExecutor` decorado con `SentryTaskDecorator` en el `AsyncSupportConfigurer` mediante una implementación de `WebMvcConfigurer`:

```java {tabTitle:Java (Spring 5)}
import org.springframework.context.annotation.Configuration;
import org.springframework.core.task.AsyncTaskExecutor;
import org.springframework.scheduling.concurrent.ThreadPoolTaskExecutor;
import org.springframework.web.servlet.config.annotation.AsyncSupportConfigurer;
import org.springframework.web.servlet.config.annotation.WebMvcConfigurer;

import io.sentry.spring.SentryTaskDecorator;

@Configuration
class AsyncWebMvcConfiguration implements WebMvcConfigurer {

    @Override
    public void configureAsyncSupport(AsyncSupportConfigurer configurer) {
        configurer.setTaskExecutor(asyncExecutor());
    }

    private AsyncTaskExecutor asyncExecutor() {
        ThreadPoolTaskExecutor executor = new ThreadPoolTaskExecutor();
        executor.setTaskDecorator(new SentryTaskDecorator());
        executor.initialize();
        return executor;
    }
}
```

<div id="configuring-async-methods-task-executor">
  ## Configuración del ejecutor de tareas para métodos @Async
</div>

Para propagar el scope de Sentry a los métodos de Spring anotados con `@Async`, se debe configurar un `AsyncConfigurerSupport` personalizado que devuelva un `ThreadPoolTaskExecutor` decorado con `SentryTaskDecorator`:

```java {tabTitle:Java (Spring 5)}
import org.springframework.context.annotation.Configuration;
import org.springframework.scheduling.annotation.AsyncConfigurerSupport;
import org.springframework.scheduling.concurrent.ThreadPoolTaskExecutor;

import io.sentry.spring.SentryTaskDecorator;

import java.util.concurrent.Executor;

@Configuration
class AsyncMethodConfiguration extends AsyncConfigurerSupport {
    @Override
    public Executor getAsyncExecutor() {
        ThreadPoolTaskExecutor executor = new ThreadPoolTaskExecutor();
        executor.setTaskDecorator(new SentryTaskDecorator());
        executor.initialize();
        return executor;
    }
}
```
