Los usuarios se definen por varias piezas de información clave que conforman una identidad única en Sentry. Cada una es opcional, pero **debe** haber al menos una para que el SDK de Sentry capture al usuario:

<DefinitionList>
  ### `id`

  Tu identificador interno del usuario.

  ### `username`

  El nombre de usuario. Suele usarse como una etiqueta más útil que el id interno.

  ### `email`

  Una alternativa o complemento al nombre de usuario. Sentry reconoce las direcciones de correo electrónico y puede mostrar, por ejemplo, Gravatars y habilitar funciones de mensajería.

  ### `ip_address`

  La dirección IP del usuario. Si el usuario no está autenticado, Sentry usa la dirección IP como identificador único del usuario.
  El SDK intentará obtener la dirección IP de los datos de la solicitud HTTP en las solicitudes entrantes (campo `request.env.REMOTE_ADDR` en JSON), si está disponible. Para ello, debes tener <PlatformIdentifier name="send-default-pii" /> configurado en `true` en las opciones del SDK.

  Si el `ip_address` del usuario está establecido en `"{{auto}}"`, Sentry inferirá la dirección IP a partir de la conexión entre tu aplicación y el servidor de Sentry. Si se omite el campo, el valor predeterminado es `null`.

  Para no almacenar direcciones IP de usuarios en los datos de tus eventos, ve a la configuración de tu proyecto, haz clic en &quot;Security &amp; Privacy&quot; y habilita &quot;Prevent Storing of IP Addresses&quot;, o usa el [depurado de datos del lado del servidor](/es/security-legal-pii/scrubbing/) de Sentry para eliminar `$user.ip_address`. Agregar esta regla prevalece sobre cualquier otra lógica.
</DefinitionList>

Además, puedes proporcionar pares clave/valor arbitrarios más allá de los nombres reservados, y el SDK de Sentry los almacenará junto con el usuario.
