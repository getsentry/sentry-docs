---
title: Datos recopilados
description: "Consulta qué datos recopila el SDK de Sentry."
sidebar_order: 1
---

Sentry se toma muy en serio la privacidad de los datos y cuenta con configuraciones predeterminadas que priorizan la seguridad de la información, especialmente en lo referente a datos de identificación personal (PII). Al agregar el SDK de Sentry a tu aplicación, permites que recopile datos y los envíe a Sentry durante la ejecución de tu aplicación.

Los tipos de categorías y la cantidad de datos recopilados varían según las integraciones que hayas habilitado en el SDK de Sentry. Esta página enumera las categorías de datos que recopila el SDK de Sentry para Java.

Para muchas de las categorías listadas aquí, es necesario habilitar la <PlatformLink to="/configuration/options/#sendDefaultPii">opción sendDefaultPii</PlatformLink>.

<div id="http-headers">
  ## Cabeceras HTTP
</div>

<div id="outgoing-requests">
  ### Solicitudes salientes
</div>

De forma predeterminada, el SDK de Sentry no envía encabezados en las solicitudes HTTP salientes. Incluso cuando el envío de encabezados HTTP está habilitado, contamos con una [lista de bloqueo](https://github.com/getsentry/sentry-java/blob/main/sentry/src/main/java/io/sentry/util/HttpUtils.java#L21-L34) que filtra cualquier encabezado que contenga datos sensibles.

Para enviar todos los encabezados HTTP, configura <PlatformLink to="/configuration/options/#sendDefaultPii">`sendDefaultPii=true`</PlatformLink>.

<div id="incoming-requests">
  ### Solicitudes entrantes
</div>

De forma predeterminada, el SDK de Sentry envía a Sentry los encabezados de las solicitudes HTTP entrantes, pero filtra cualquier encabezado que contenga datos sensibles. (Consulta la [lista de encabezados](https://github.com/getsentry/sentry-java/blob/main/sentry/src/main/java/io/sentry/util/HttpUtils.java#L21-L34) que se filtran).

Para enviar todos los encabezados HTTP, configura <PlatformLink to="/configuration/options/#sendDefaultPii">`sendDefaultPii=true`</PlatformLink>.

<div id="cookies">
  ## Cookies
</div>

De forma predeterminada, el SDK de Sentry no envía cookies. Sentry intenta eliminar cualquier cookie que contenga información sensible, como las cookies de ID de sesión y de token CSRF.

Si quieres enviar cookies, establece <PlatformLink to="/configuration/options/#send-default-pii">`sendDefaultPii=true`</PlatformLink>.

<div id="information-about-logged-in-user">
  ## Información sobre el usuario conectado
</div>

De forma predeterminada, el SDK de Sentry no envía ninguna información sobre el usuario conectado, como la dirección de correo electrónico, el ID de usuario o el nombre de usuario. Incluso si se habilita, el tipo de información del usuario conectado que podrás enviar depende de las integraciones que actives en el SDK de Sentry. La mayoría de las integraciones no enviarán ninguna información de usuario. Algunas solo establecerán el ID de usuario, pero hay unas pocas que establecerán el ID de usuario, el nombre de usuario y la dirección de correo electrónico.

Para empezar a enviar información del usuario conectado, configura <PlatformLink to="/configuration/options/#send-default-pii">`sendDefaultPii=true`</PlatformLink>.

<div id="users-ip-addresses">
  ## Direcciones IP de los usuarios
</div>

De forma predeterminada, el SDK de Sentry no envía la dirección IP del usuario. Una vez habilitado, los servicios de backend de Sentry deducirán la dirección IP del usuario a partir de la solicitud entrante, a menos que ciertas integraciones que puedes activar anulen este comportamiento.

Para habilitar el envío de la dirección IP del usuario, configura <PlatformLink to="/configuration/options/#send-default-pii">`sendDefaultPii=true`</PlatformLink>.

<div id="request-url">
  ## URL de la solicitud
</div>

La URL completa de las solicitudes HTTP salientes y entrantes **siempre se envía a Sentry**. Dependiendo de tu aplicación, esto podría contener información de identificación personal (PII).

<div id="request-query-string">
  ## Cadena de consulta de la solicitud
</div>

La cadena de consulta completa de las solicitudes HTTP salientes y entrantes **siempre se envía a Sentry**. Según tu aplicación, esto podría incluir datos personales (PII).

<div id="request-body">
  ## Cuerpo de la solicitud
</div>

El cuerpo de las solicitudes HTTP entrantes se puede enviar a Sentry. Si se envía o no depende del tipo y del tamaño del cuerpo de la solicitud, como se describe a continuación:

- **Tipo de cuerpo de la solicitud:**
  - Se envían los cuerpos JSON y de formulario
- **Tamaño del cuerpo de la solicitud:** Existe una opción <PlatformLink to="/configuration/options/#maxRequestBodySize">maxRequestBodySize</PlatformLink> cuyo valor predeterminado es `NONE`. Esto significa que, por defecto, no se envía ningún cuerpo de la solicitud a Sentry.

<div id="source-context">
  ## Contexto del código fuente
</div>

Nuestros plugins de compilación para Gradle y Maven pueden subir tu código fuente a Sentry, que luego se puede usar para mostrar las líneas de código donde ocurrió un error en la página de detalles del issue.

Para enviar este contexto de código fuente a Sentry, debes habilitar la función como se describe en <PlatformLink to="/source-context/">la documentación de Source Context</PlatformLink>.

<div id="file-io">
  ## E/S de archivos
</div>

De forma predeterminada, el SDK de Sentry no envía el nombre ni la ruta de los archivos al instrumentar operaciones de E/S de archivos.

Si quieres enviar nombres y rutas de archivos, establece <PlatformLink to="/configuration/options/#send-default-pii">`sendDefaultPii=true`</PlatformLink>.

<div id="log-messages">
  ## Mensajes de registro
</div>

De forma predeterminada, el SDK de Sentry no envía los mensajes y parámetros de Logback sin codificar si se ha configurado un codificador. No obstante, sí enviará el mensaje codificado.

Si quieres enviar el mensaje y los parámetros sin codificar, configura <PlatformLink to="/configuration/options/#send-default-pii">`sendDefaultPii=true`</PlatformLink>.

<div id="sql-queries">
  ## Consultas SQL
</div>

Aunque las consultas SQL se envían a Sentry, ni la consulta SQL completa (`UPDATE app_user SET password='supersecret' WHERE id=1;`) ni los valores de sus parámetros se envían nunca. En su lugar, se envía una versión parametrizada de la consulta (`UPDATE app_user SET password=? WHERE id=?;`).
