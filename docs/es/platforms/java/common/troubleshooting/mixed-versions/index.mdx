No se admite usar varias dependencias del SDK de Java con versiones mezcladas, ya que es muy probable que provoque un fallo más adelante. Por este motivo, optamos por hacer que la aplicación falle durante la inicialización del SDK.

<div id="multiple-dependencies-with-mixed-versions">
  ## Múltiples dependencias con versiones mixtas
</div>

El siguiente fragmento muestra un conflicto de versiones mixtas causado por usar `sentry` en la versión `8.6.0` y `sentry-logback` en la versión `8.7.0`. Para solucionar el problema, establece la misma versión para todas las dependencias o <PlatformLink to="/configuration/bill-of-materials/">usa `sentry-bom`</PlatformLink>. Esto también puede ocurrir si estás usando una biblioteca interna que tenga definida una versión diferente.

```groovy {tabTitle:Gradle}{filename:build.gradle}
implementation('io.sentry:sentry:8.6.0')
implementation('io.sentry:sentry-logback:8.7.0')
```

```xml {tabTitle:Maven}{filename:pom.xml}
<dependencyManagement>
    <dependencies>
        <dependency>
            <groupId>io.sentry</groupId>
            <artifactId>sentry</artifactId>
            <version>8.6.0</version>
        </dependency>
        <dependency>
            <groupId>io.sentry</groupId>
            <artifactId>sentry-logback</artifactId>
            <version>8.7.0</version>
        </dependency>
    </dependencies>
</dependencyManagement>
```


<div id="build-plugin-and-explicit-dependency">
  ## Complemento de compilación y dependencia explícita
</div>

Al usar nuestro complemento de Gradle o Maven y definir manualmente dependencias adicionales del Sentry Java SDK, puedes acabar con versiones mezcladas. El siguiente fragmento muestra que el complemento está configurado para usar la versión `8.0.0`, pero hay una dependencia adicional establecida en la versión `8.1.0`. Para resolverlo, usa la misma versión o <PlatformLink to="/configuration/bill-of-materials/">usa `sentry-bom`</PlatformLink>.

```groovy {tabTitle:Gradle}{filename:build.gradle}
plugins { id "io.sentry.android.gradle" version "5.3.0" }

dependencies {
	implementation 'io.sentry:sentry-opentelemetry-agentless-spring:8.1.0'
}

sentry {
	autoInstallation {
		sentryVersion = "8.0.0"
	}
}
```


<div id="agent-version-and-build-time-version">
  ## Versión del agente y versión de compilación
</div>

Al usar `sentry-opentelemetry-agent`, podrías terminar utilizando una versión del agente que difiere de la de otras dependencias del SDK de Java de Sentry que estás usando. Esto tampoco es compatible. Usa la misma versión para `sentry-opentelemetry-agent` y para todas las demás dependencias del SDK de Java.
