[OpenTelemetry](https://opentelemetry.io/) potencia Sentry al ofrecer compatibilidad con más frameworks y bibliotecas que:

- Te permiten crear más spans, por ejemplo, en torno a las solicitudes.
- Proporcionan propagación de `Context` que también se encarga de propagar los `Scopes` de Sentry, garantizando que elementos como las etiquetas y otra información terminen en el evento correcto.
- Extraen información de trazas de las solicitudes entrantes y los mensajes consumidos.
- Inyectan información de trazas en las solicitudes salientes y los mensajes producidos.

Consulta el [repositorio de GitHub de OpenTelemetry](https://github.com/open-telemetry/opentelemetry-java-instrumentation/tree/main/instrumentation) para ver la lista completa de instrumentaciones disponibles.

En algunos casos, hay instrumentación disponible tanto de OpenTelemetry como de Sentry. En ese caso, configuramos automáticamente la instrumentación de Sentry para que no cree spans y, en su lugar, dejar que OpenTelemetry se encargue.

Puedes usar la API de OpenTelemetry o la API de Sentry para instrumentar manualmente tu aplicación. Ambas acabarán en Sentry. Si ya usas la API de Sentry, puedes seguir usándola al cambiarte a nuestra integración con OpenTelemetry. Si ya tienes instrumentación de OpenTelemetry, puedes enviar los spans resultantes a Sentry.
