<div id="control-data-truncation">
  ## Controlar el truncamiento de datos
</div>

Actualmente, cada etiqueta tiene un límite máximo de 200 caracteres. Las etiquetas que superen ese límite se truncarán, lo que puede hacer que se pierda información importante. Para conservar estos datos, puedes dividir la información en varias etiquetas.

Por ejemplo, una solicitud con etiqueta de más de 200 caracteres:

`https://empowerplant.io/api/0/projects/ep/setup_form/?user_id=314159265358979323846264338327&tracking_id=EasyAsABC123OrSimpleAsDoReMi&product_name=PlantToHumanTranslator&product_id=161803398874989484820458683436563811772030917980576`

La solicitud anterior, de más de 200 caracteres, se truncará a:

`https://empowerplant.io/api/0/projects/ep/setup_form/?user_id=314159265358979323846264338327&tracking_id=EasyAsABC123OrSimpleAsDoReMi&product_name=PlantToHumanTranslator&product_id=1618033988749894848`

<div id="expected-spans-are-missing">
  ## Faltan spans esperados
</div>

Nuestra auto-instrumentación, por ejemplo la que instrumenta solicitudes a bases de datos o llamadas HTTP, requiere que haya una transacción presente en el `Scope` para poder adjuntar un nuevo span secundario. Si faltan spans esperados, asegúrate de que `bindToScope` esté establecido en `true` al iniciar una transacción. Consulta también <PlatformLink to="/tracing/instrumentation/custom-instrumentation/#create-transaction-bound-to-the-current-scope">Crear una transacción vinculada al Scope actual</PlatformLink>.

<div id="jdbc-and-spring-boot-docker-compose">
  ## JDBC y `spring-boot-docker-compose`
</div>

Al usar `org.springframework.boot:spring-boot-docker-compose`, ten en cuenta que esta extensión sobrescribe la propiedad `spring.datasource.url` y, por tanto, elimina el prefijo `p6spy` necesario para usar `sentry-jdbc`. Por ello, si quieres probarlo en local, debes deshabilitar o quitar spring-boot-docker-compose.
