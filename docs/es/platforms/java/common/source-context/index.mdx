<div id="setting-up-code-mappings">
  ## Configurar asignaciones de código
</div>

Para usar commits sospechosos y el enlace de trazas de pila, debes configurar una asignación de código. Esta es una correspondencia entre el código fuente de tu repositorio y el código fuente de tus trazas de pila. Puedes encontrar información sobre cómo configurar asignaciones de código en nuestra documentación [Configurar asignaciones de código](/es/product/issues/suspect-commits/#2-set-up-code-mappings).

Sigue estos pasos para determinar correctamente **Stack Trace Root** y **Source Code Root** al configurar tu asignación de código:

Primero, navega a una traza de pila que quieras asignar. Busca un frame **In App**, indicado por una burbuja en el lado derecho del frame. El módulo de Java se mostrará como el primer texto en el lado izquierdo del encabezado del frame (marcado como **1** en la captura de pantalla a continuación). En este ejemplo, es `com.example.vu.android.MainActivity`. Al pasar el cursor sobre el módulo, se mostrará la ruta absoluta al archivo (marcada como **2** en la captura de pantalla a continuación). En este ejemplo, es `MainActivity.java`.

![Resaltando dónde encontrar el módulo y la ruta absoluta en la traza de pila](./img/java-code-mappings-stacktrace.png)

A continuación, calcula la ruta de archivo derivada. Toma el `module` del frame y reemplaza todos los caracteres `.` por `/`. Luego, reemplaza el último elemento (`MainActivity`, en este ejemplo) por la ruta absoluta encontrada anteriormente (`MainActivity.java`). En este ejemplo, la ruta de archivo derivada es `com/example/vu/android/MainActivity.java`.

Por último, compara la ruta derivada con la ruta que se encuentra en tu repositorio de código fuente. En este ejemplo, la carpeta `com/example/vu/android/` puede coincidir con la carpeta `src/com/example/vu/android/` en el código fuente. Con esa información, establece **Stack Trace Root** en `com/example/vu/android/` y **Source Code Root** en `src/com/example/vu/android/`. Esto le indica a Sentry que reemplace todas las rutas de archivo que comiencen con `com/example/vu/android/` por `src/com/example/vu/android/` al buscar el código fuente.

Dado que Sentry usará la primera asignación de código que coincida, se recomienda proporcionar siempre un valor no vacío para **Stack Trace Root** cuando sea posible. Un valor vacío en **Stack Trace Root** coincidirá con todas las rutas de archivo, lo que funcionará mal si se requieren múltiples asignaciones de código.
