Un lanzamiento es una versión de tu código que se despliega en un <PlatformLink to="/configuration/environments/">entorno</PlatformLink>. Cuando le das a Sentry información sobre tus lanzamientos, puedes:

* Identificar problemas y regresiones introducidos en un nuevo lanzamiento
* Predecir qué commit causó un problema y quién probablemente es responsable
* Resolver problemas incluyendo el número de la incidencia en el mensaje del commit
* Recibir notificaciones por correo electrónico cuando se despliegue tu código

Además, los lanzamientos se usan para aplicar [source maps](/es/platforms/javascript/sourcemaps/) a JavaScript minificado para ver el código fuente original y sin transformar.

<div id="setting-a-release">
  ## Configurar una versión
</div>

<PlatformContent includePath="set-release" notateUnsupported />

Cómo poner el nombre de la versión (o release) a disposición de tu código depende de ti. Por ejemplo, puedes usar una variable de entorno que se defina durante el proceso de compilación o en el arranque inicial.

Al configurar el nombre de la versión, cada evento se etiqueta con ese nombre. Recomendamos informar a Sentry sobre una nueva versión antes de enviar eventos con ese nombre, ya que así se habilitan algunas funciones adicionales. Obtén más información en nuestra documentación de [Releases](/es/product/releases/).

Si no informas a Sentry sobre una nueva versión, Sentry creará automáticamente una entidad de versión en el sistema la primera vez que vea un evento con ese ID de versión.

Después de configurar tu SDK, puedes instalar una integración de repositorio o proporcionar manualmente a Sentry tus propios metadatos de commits. Lee nuestra documentación sobre [configuración de releases](/es/product/releases/setup/) para obtener más información sobre integraciones, asociación de commits y cómo informar a Sentry al desplegar versiones.

<div id="release-health">
  ## Salud de las versiones
</div>

Supervisa la [salud de las versiones](/es/product/releases/health/) observando la adopción de los usuarios, el uso de la aplicación, el porcentaje de [bloqueos](/es/product/releases/health/#crashes) y los [datos de sesión](/es/product/releases/health/#sessions). La salud de las versiones ofrece información sobre el impacto de los bloqueos y los errores en la experiencia del usuario y revela tendencias con cada nueva incidencia mediante los gráficos y filtros de [Release Details](/es/product/releases/release-details/).

Para monitorear la salud de las versiones, el SDK envía datos de sesión.

<div id="sessions">
  ### Sesiones
</div>

Una sesión representa la interacción entre el usuario y la aplicación. Las sesiones incluyen una marca de tiempo, un estado (si la sesión fue correcta o si se bloqueó) y siempre están vinculadas a una versión. La mayoría de los SDK de Sentry pueden gestionar las sesiones automáticamente.
