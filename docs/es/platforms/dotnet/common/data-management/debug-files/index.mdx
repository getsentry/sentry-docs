Los archivos de información de depuración permiten que Sentry extraiga trazas de pila y ofrezca más
información sobre los informes de fallos en la mayoría de las plataformas compiladas. La información almacenada
en estos archivos incluye nombres de funciones originales, rutas a archivos fuente y números de línea,
contexto del código fuente y la ubicación de variables en memoria. Sentry
puede usar parte de esta información y mostrarla en la página de detalles de la incidencia.

Cada plataforma principal utiliza diferentes archivos de información de depuración. Actualmente admitimos
los siguientes formatos:

* [*archivos dSYM*](./file-formats/#mach-o-and-dsym) para iOS, iPadOS,
  tvOS, watchOS, macOS y visionOS
* [*símbolos ELF*](./file-formats/#executable-and-linkable-format-elf) para Linux y
  Android (NDK)
* [*archivos PDB*](./file-formats/#pe-and-pdb) para Windows y .NET
* [*símbolos Breakpad*](./file-formats/#breakpad-symbols) para todas las
  plataformas
* [*archivos WASM*](./file-formats/#wasm) para WebAssembly
* [*asignaciones de ProGuard*](./file-formats/#proguard-mappings) para
  Java y Android

<Alert>
  Los source maps, aunque también son archivos de información de depuración, se gestionan
  de forma diferente en Sentry. Para obtener más información, consulta [Source Maps en sentry-cli](/es/cli/releases/#sentry-cli-sourcemaps).
</Alert>

Sentry necesita acceso a los archivos de información de depuración de tu aplicación, así como a
las bibliotecas del sistema, para proporcionar informes de fallos completamente simbolicados. Puedes
[subir](./upload/) tus archivos a Sentry o colocarlos en un
[Symbol Server](./symbol-servers/) compatible para que Sentry los descargue cuando sea necesario.

<div id="managing-debug-information-files">
  ## Gestión de archivos de información de depuración
</div>

Una vez cargados, los archivos de información de depuración pueden verse y gestionarse
desde la sección *Debug Files* en la página de configuración del proyecto correspondiente. Esa página
muestra todos los archivos de depuración cargados y también permite configurar servidores de símbolos
para descargas automáticas.

Desde la página [Project Details](/es/product/projects/project-details/), haz clic en settings
y luego en *Debug Files* en la navegación de la página.

<Alert>
  Los archivos de ProGuard se muestran por separado, en la sección *ProGuard* de la página de configuración del proyecto.
</Alert>

<div id="retention-policy">
  ## Política de retención
</div>

Los archivos de depuración tienen un período de retención de *90 días* y utilizan un mecanismo de expiración por *tiempo de inactividad*.
Esto significa que los archivos de depuración cargados se conservan mientras se utilicen activamente para procesar eventos.
Una vez que un archivo de depuración no se haya utilizado para procesar eventos entrantes durante al menos 90 días,
expirará automáticamente y podrá eliminarse.
