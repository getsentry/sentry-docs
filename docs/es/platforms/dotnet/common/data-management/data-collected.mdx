---
title: Datos recopilados
description: "Consulta qué datos recopila el SDK de Sentry."
sidebar_order: 1
---

Sentry se toma muy en serio la privacidad de los datos y cuenta con configuraciones predeterminadas que priorizan su seguridad, especialmente cuando se trata de información de identificación personal (PII). Cuando agregas el SDK de Sentry a tu aplicación, permites que recopile datos y los envíe a Sentry durante la ejecución de tu aplicación.

Los tipos de categorías y la cantidad de datos recopilados varían según las integraciones que hayas habilitado en el SDK de Sentry. Aquí tienes una lista de las categorías de datos que recopila el SDK de Sentry para .NET:

<div id="http-headers">
  ## Encabezados HTTP
</div>

De forma predeterminada, el SDK de Sentry envía encabezados HTTP a Sentry, pero oculta cualquier encabezado que pueda contener datos sensibles. (Consulta la [lista de encabezados](https://github.com/getsentry/sentry-dotnet/blob/main/src/Sentry/Internal/RedactedHeaders.cs#L7) que se ocultan).

<div id="cookies">
  ## Cookies
</div>

De forma predeterminada, el SDK de Sentry no envía cookies.

Si quieres enviar cookies, establece la opción `SendDefaultPii = true` al inicializar el SDK de Sentry.

Puedes usar un [depurador de datos](/es/platforms/dotnet/data-management/sensitive-data/) para filtrar información antes de enviar datos a Sentry.

Cualquier [cookie confidencial](https://github.com/getsentry/relay/blob/249971e1702991c64cf0f7f89773fb019ffb4a95/relay-pii/src/convert.rs#L235-L263) que el SDK sí envíe será eliminada por Relay antes de que Sentry la almacene.

<div id="information-about-logged-in-user">
  ## Información sobre el usuario autenticado
</div>

De forma predeterminada, el SDK de Sentry no envía ninguna información del usuario autenticado, como la dirección de correo electrónico, el ID de usuario o el nombre de usuario. Incluso si se habilita, el tipo de información de usuario que podrás enviar depende de las integraciones que actives en el SDK de Sentry. La mayoría de las integraciones no enviarán información de usuario. Algunas solo establecerán el ID de usuario, pero hay algunas que también establecerán el ID de usuario, el nombre de usuario y la dirección de correo electrónico.

Para empezar a enviar información del usuario autenticado, configura la opción `SendDefaultPii = true` al inicializar el SDK de Sentry.

<div id="users-ip-address">
  ## Dirección IP de los usuarios
</div>

De forma predeterminada, el SDK de Sentry no envía la dirección IP del usuario.

Para habilitar el envío de la dirección IP del usuario, establece la opción `SendDefaultPii = true` al inicializar el SDK de Sentry.

Incluso si lo habilitas, que puedas enviar o no la dirección IP del usuario dependerá de las integraciones que actives en el SDK de Sentry. La mayoría de las integraciones no establecerán la dirección IP del usuario.

Puedes [evitar que Sentry almacene direcciones IP](/es/platforms/native/enriching-events/identify-user/#ip_address) configurándolo en los ajustes de “Security &amp; Privacy” de tu proyecto de Sentry o usando [depuración de datos del lado del servidor](/es/security-legal-pii/scrubbing/server-side-scrubbing/).

<div id="request-url">
  ## URL de la solicitud
</div>

De forma predeterminada, el [HttpRequestUrl](https://learn.microsoft.com/en-us/dotnet/api/System.UriComponents) de las solicitudes HTTP salientes y entrantes **siempre se envía a Sentry**. Esto incluye los componentes Scheme, Host, Port, LocalPath y Query.

Si `SendDefaultPii` se establece en `true`, entonces el [AbsoluteUri](https://learn.microsoft.com/en-us/dotnet/api/System.UriComponents) de las solicitudes HTTP salientes y entrantes se enviará a Sentry (incluyendo los componentes Scheme, UserInfo, Host, Port, LocalPath, Query y Fragment).

Según tu aplicación, esto podría contener datos personales (PII).

<div id="request-query-string">
  ## Cadena de consulta de la solicitud
</div>

La cadena de consulta completa de las solicitudes HTTP salientes y entrantes **siempre se envía a Sentry**. Dependiendo de tu aplicación, esto podría contener datos personales (PII).

<PlatformSection supported={["dotnet.aspnetcore", "dotnet.aspnet"]}>
  ## Cuerpo de la solicitud

  La opción <PlatformLink to="/configuration/options/#MaxRequestBodySize">MaxRequestBodySize</PlatformLink> se puede usar para configurar si el cuerpo de las solicitudes HTTP entrantes se enviará a Sentry.

  De forma predeterminada, está establecido en `RequestSize.None`, lo que significa que el cuerpo de la solicitud no se envía a Sentry.

  Si quieres que los cuerpos de las solicitudes se envíen a Sentry, debes establecerlo en `RequestSize.Small`, `RequestSize.Medium` o `RequestSize.Always`.
</PlatformSection>

<div id="sql-queries">
  ## Consultas SQL
</div>

Aunque las consultas SQL se envían a Sentry, ni la consulta completa (`SELECT * FROM users WHERE email = 'joe@shmoe.com'`) ni los valores de sus parámetros se envían nunca. En su lugar, se envía una versión con parámetros de la consulta (`SELECT * FROM users WHERE email = @email`).

<div id="machine-name">
  ## Nombre de la máquina
</div>

El SDK de Sentry no envía el nombre de la máquina de forma predeterminada.

Si asignas un valor a `ServerName` en las opciones de Sentry, el SDK de Sentry lo enviará con cualquier evento.

De lo contrario, si `SendDefaultPii` está establecido en true, el SDK de Sentry capturará automáticamente `Environment.MachineName` y lo enviará con cualquier evento.

<div id="general-device-information">
  ## Información general del dispositivo
</div>

El SDK de Sentry recopila información sobre el dispositivo, como la versión y la compilación de tu sistema operativo, detalles de CPU y GPU, etc. Esta información se envía a Sentry de forma predeterminada.

<PlatformSection supported={["dotnet.maui"]}>
  ## Datos capturados por los SDK móviles

  En plataformas móviles, el SDK de Sentry para .NET también puede capturar datos indirectamente a través de los SDK de Android y iOS. Consulta la documentación de esos SDK para obtener más información sobre qué datos se recopilan.

  * [Android](/es/platforms/android/data-management/data-collected/)
  * [iOS](/es/platforms/apple/data-management/data-collected/)
</PlatformSection>

<div id="native-crash-reporting">
  ## Informes de bloqueos nativos
</div>

Al compilar aplicaciones .NET con Ahead-of-Time (AOT), también se pueden capturar bloqueos nativos. Consulta los [datos capturados por el SDK nativo](/es/platforms/native/data-management/data-collected/) para comprender qué datos se envían con esos eventos.
