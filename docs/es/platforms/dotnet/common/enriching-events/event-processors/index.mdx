title: Procesadores de eventos
description: "Aprende cómo los procesadores de eventos pueden enriquecer eventos de forma global o en el alcance actual."
---

Los procesadores de eventos pueden enriquecer los eventos con información adicional, ya sea globalmente o dentro de un alcance específico. Aunque los procesadores de eventos son similares a <PlatformIdentifier name="before-send" /> y <PlatformIdentifier name="before-send-transaction" />, hay dos diferencias clave:

1. <PlatformIdentifier name="before-send" /> y <PlatformIdentifier name="before-send-transaction" /> están
  garantizados a ejecutarse después de todo el procesamiento, justo antes de que se envíe un evento. Los procesadores de eventos se ejecutan en un orden indeterminado, por lo que aún se pueden hacer cambios al evento después de que se ejecute el procesador de eventos.
2. <PlatformIdentifier name="before-send" /> y <PlatformIdentifier name="before-send-transaction" /> siempre se ejecutan de forma global. Los procesadores se pueden configurar para ejecutarse globalmente o solo dentro de un alcance específico.

<div id="duplicate-event-detection">
  ## Detección de eventos duplicados
</div>

De forma predeterminada, Sentry incluye un procesador global que detecta y descarta eventos duplicados.

Este procesador puede configurarse mediante `SentryOptions.DeduplicateMode` al inicializar el SDK de Sentry. Es un enum de indicadores con los siguientes valores:

* `DeduplicateMode.SameEvent`: Misma instancia de evento. Supone que no hay reutilización o agrupación de objetos.
* `DeduplicateMode.SameExceptionInstance`: Una excepción que se capturó dos veces.
* `DeduplicateMode.InnerException`: Una excepción ya capturada existe como excepción interna.
* `DeduplicateMode.AggregateException`: Una excepción ya capturada forma parte de la excepción agregada.
* `DeduplicateMode.All`: Todos los modos combinados.

El valor predeterminado de `SentryOptions.DeduplicateMode` es `DeduplicateMode.All ^ DeduplicateMode.InnerException` (por lo tanto, se descartarán todos los eventos duplicados detectados excepto los de excepciones internas). Sin embargo, si solo quisiéramos descartar eventos `SameEvent` y `SameExceptionInstance`, podríamos inicializar el SDK de la siguiente manera:

```csharp
SentrySdk.Init(options => {
  // ... configura otras opciones aquí
  options.DeduplicateMode = DeduplicateMode.SameEvent | DeduplicateMode.SameExceptionInstance;
})
```

Como alternativa, puedes desactivar por completo la detección de eventos duplicados llamando a `DisableDuplicateEventDetection`:

```csharp
SentrySdk.Init(options => {
  // ... configura otras opciones aquí
  options.DisableDuplicateEventDetection();
})
```
