El SDK de .NET ofrece una integración con [Hangfire](https://www.hangfire.io/) para supervisar tus trabajos al [crear check-ins automáticamente](/es/product/crons/job-monitoring/) para ellos. El SDK se basa en filtros de trabajo que se configuran cuando llamas a `UseSentry`. Por ejemplo:

```csharp
using Hangfire;
using Sentry.Hangfire;

GlobalConfiguration.Configuration.UseSentry();
```

Como alternativa, si estás usando el patrón de constructor (builder), tu código podría verse así:

```csharp
using Hangfire;
using Sentry.Hangfire;

var builder = WebApplication.CreateBuilder(args);
builder.Services.AddHangfire(configuration => configuration
    .UseSentry() // <- Añade Sentry para enviar check-ins automáticamente
);
```

Para indicarle al SDK qué tareas debe supervisar, debes asignar a la tarea un “monitor slug”. Puedes hacerlo estableciendo el atributo `SentryMonitorSlug` en la tarea. Por ejemplo, el siguiente fragmento muestra cómo configurar una tarea de tipo “fire-and-forget” que Sentry supervisará:

```csharp
using Hangfire;
using Sentry.Hangfire;

GlobalConfiguration.Configuration.UseSentry();
BackgroundJob.Enqueue<PricingUpdateWorker>(job => job.Execute());

public class PricingUpdateWorker
{
    [SentryMonitorSlug("update-pricing")]
    public void Execute()
    {
        // Realiza el trabajo para actualizar el precio
    }
}
```

El SDK registrará automáticamente un check-in con `CheckInStatus.InProgress` cuando se inicie el job y lo actualizará a `CheckInStatus.OK` si finaliza correctamente. En caso de producirse una excepción durante la ejecución, el check-in se actualizará a `CheckInStatus.ERROR`.
