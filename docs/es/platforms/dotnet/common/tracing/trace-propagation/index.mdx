Si el ecosistema de aplicaciones que quieres observar con Sentry abarca más que un único servicio o aplicación, el trazado distribuido puede aportar mucho valor.

<div id="what-is-distributed-tracing">
  ## ¿Qué es el tracing distribuido?
</div>

En el contexto de seguir eventos a través de un sistema distribuido, el tracing distribuido es una potente herramienta de depuración. Imagina tu aplicación como una amplia red de partes interconectadas. Por ejemplo, tu sistema puede estar repartido en distintos servidores, o tu aplicación puede dividirse en servicios de backend y frontend, cada uno con su propio stack tecnológico.

Cuando ocurre un error o un problema de rendimiento, puede ser difícil encontrar la causa raíz por la complejidad de un sistema así. El tracing distribuido te ayuda a seguir el recorrido de un evento mientras atraviesa esta red intrincada, registrando cada paso. Al examinar estos trazos (traces), puedes reconstruir la secuencia de sucesos que llevó al evento de interés, identificar los componentes específicos involucrados y entender cómo interactúan. Esta visibilidad detallada te permite diagnosticar y resolver problemas con mayor eficacia, mejorando en última instancia la confiabilidad y el rendimiento de tu sistema distribuido.

Recuerda que, para propagar la información de trazas en todo tu sistema distribuido, debes usar Sentry en todos los servicios y aplicaciones involucrados. Consulta la documentación del SDK correspondiente para saber cómo habilitar el rastreo distribuido en cada plataforma.
