Todos los SDK de Sentry incluyen integraciones que amplían la funcionalidad del SDK. La mayoría están habilitadas de forma predeterminada, aunque puedes deshabilitarlas si es necesario.

<div id="options">
  # Opciones
</div>

La mayoría de las integraciones pueden deshabilitarse mediante las opciones al inicializar el SDK. Por ejemplo, para deshabilitar la captura automática de excepciones no controladas:

```csharp
// Añádelo a tu bloque de inicialización del SDK de Sentry:
options.DisableAppDomainUnhandledExceptionCapture()
```

```fsharp
// Añádelo al bloque de inicialización de tu SDK de Sentry:
options.DisableAppDomainUnhandledExceptionCapture()
```


<div id="disableappdomainunhandledexceptioncapture">
  ## DisableAppDomainUnhandledExceptionCapture
</div>

Deshabilita la captura automática de excepciones no controladas.

<div id="disableruntimemarshalmanagedexceptioncapture">
  ## DisableRuntimeMarshalManagedExceptionCapture
</div>

Desactiva la captura automática de excepciones no controladas en iOS.

El controlador `AppDomainUnhandledException` [no captura de forma fiable las excepciones no controladas en iOS](https://github.com/getsentry/sentry-dotnet/pull/3909). En su lugar, se usa `Runtime.MarshalManagedException` para capturar excepciones no controladas en iOS.

<div id="disableunobservedtaskexceptioncapture">
  ## DisableUnobservedTaskExceptionCapture
</div>

Desactiva la captura automática de excepciones de tareas no observadas.

<div id="disablenetfxinstallationsintegration">
  ## DisableNetFxInstallationsIntegration
</div>

Desactiva que se agregue la lista de instalaciones de .NET Framework a los eventos.

<div id="disableappdomainprocessexitflush">
  ## DisableAppDomainProcessExitFlush
</div>

Deshabilita el vaciado de eventos en el controlador del evento `AppDomain.ProcessExit`.

<div id="sentry-native">
  # Sentry Native
</div>

En muchas plataformas, el SDK de .NET para Sentry incluye Sentry Native, que permite capturar informes de fallos de código nativo (no administrado). Si tienes problemas, puedes desactivar Sentry Native estableciendo la propiedad de compilación `SentryNative` en `false`:

```xml
<PropertyGroup>
  <SentryNative>false</SentryNative>
</PropertyGroup>
```

Nota: Sentry Native está desactivado de forma predeterminada para aplicaciones dirigidas a WASM.
