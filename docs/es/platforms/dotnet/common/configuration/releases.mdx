Un lanzamiento es una versión de tu código que se despliega en un <PlatformLink to="/configuration/environments/">entorno</PlatformLink>. Cuando proporcionas a Sentry información sobre tus lanzamientos, puedes:

* Identificar problemas y regresiones introducidos en un nuevo lanzamiento
* Predecir qué commit causó un problema y quién probablemente es responsable
* Resolver problemas incluyendo el número del issue en el mensaje del commit
* Recibir notificaciones por correo electrónico cuando se despliegue tu código

<div id="setting-a-release">
  ## Configurar una versión
</div>

<PlatformContent includePath="set-release" notateUnsupported />

Cómo poner el nombre de la versión (o release) a disposición de tu código depende de ti. Por ejemplo, puedes usar una variable de entorno que se configure durante el proceso de build o en el arranque inicial.

Al configurar el nombre de la versión, cada evento se etiqueta con ese nombre. Te recomendamos avisar a Sentry sobre una nueva versión antes de enviar eventos con ese nombre, ya que esto habilitará algunas funciones adicionales. Obtén más información en nuestra documentación de [Releases](/es/product/releases/).

Si no le informas a Sentry sobre una nueva versión, Sentry creará automáticamente una entidad de versión en el sistema la primera vez que vea un evento con ese ID de versión.

Después de configurar tu SDK, puedes instalar una integración de repositorio o proporcionar manualmente a Sentry tus propios metadatos de commits. Lee nuestra documentación sobre [configurar releases](/es/product/releases/setup/) para obtener más información sobre integraciones, asociación de commits y cómo informar a Sentry al desplegar versiones.

<div id="release-health">
  ## Salud de la versión
</div>

Supervisa la [salud de las versiones](/es/product/releases/health/) observando la adopción por parte de los usuarios, el uso de la aplicación, el porcentaje de [crashes](/es/product/releases/health/#crashes) y los [datos de sesión](/es/product/releases/health/#sessions). La salud de la versión ofrece información sobre el impacto de los crashes y los bugs en la experiencia del usuario, y revela tendencias con cada nuevo issue a través de los gráficos y filtros de [Detalles de la versión](/es/product/releases/release-details/).

Para monitorear la salud de la versión, el SDK envía datos de sesión.

<div id="sessions">
  ### Sesiones
</div>

Una sesión representa la interacción entre el usuario y la aplicación. Las sesiones incluyen una marca de tiempo, un estado (si la sesión fue correcta o si se bloqueó) y siempre están asociadas a una versión. La mayoría de los SDK de Sentry pueden gestionar las sesiones automáticamente.

<PlatformContent includePath="configuration/auto-session-tracking" />

<Alert>
  El soporte de salud de versiones para .NET está actualmente limitado a aplicaciones **cliente** de una sola instancia. Esto significa que puede usarse con modelos de aplicación como WPF, Xamarin, WinForms y similares.
  Está previsto el soporte para frameworks de servidor como ASP.NET.
</Alert>
