El comportamiento predeterminado de la mayoría de los SDK es enviar eventos por la red de forma asíncrona y en segundo plano. Esto significa que algunos eventos podrían perderse si la aplicación se cierra inesperadamente. Los SDK proporcionan mecanismos para manejar esta situación.
