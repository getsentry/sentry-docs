El SDK de Sentry se integra con tu entorno de ejecución y reporta automáticamente errores, excepciones no controladas y rechazos no manejados, así como otros tipos de errores según la plataforma.

<Alert>

Términos clave:

- Un _evento_ es una instancia de envío de datos a Sentry. Generalmente, estos datos son un error o una excepción.
- Un _issue_ es un conjunto de eventos similares.
- El envío de un evento se denomina _captura_.
  Cuando se captura un evento, se envía a Sentry.

</Alert>

La forma más común de captura es la de errores. Lo que puede capturarse como error varía según la plataforma. En general, si tienes algo que parece una excepción, puede capturarse. En algunos SDK también puedes omitir el argumento de <PlatformIdentifier name="capture-exception" /> y Sentry intentará capturar la excepción actual. También es útil para reportar manualmente errores o mensajes a Sentry.

Al capturar un evento, también puedes registrar las migas de pan (breadcrumbs) que llevaron a ese evento. Las migas de pan son diferentes de los eventos: no crean un evento en Sentry, sino que se almacenan en búfer hasta que se envíe el siguiente evento. Obtén más información sobre las migas de pan en nuestra <PlatformLink to="/enriching-events/breadcrumbs/">documentación de Breadcrumbs</PlatformLink>.

Otra operación común es capturar un mensaje sin contexto adicional. Un mensaje es información textual que se debe enviar a Sentry. Por lo general, nuestros SDK no capturan mensajes automáticamente, pero puedes capturarlos manualmente.

Los mensajes aparecen como incidencias en tu lista de incidencias, usando el propio mensaje como nombre de la incidencia.
