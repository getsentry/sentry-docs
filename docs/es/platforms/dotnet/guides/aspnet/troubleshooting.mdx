<div id="proxy-server">
  ### Servidor proxy
</div>

A menudo, tu servidor solo puede acceder a Internet a través de un servidor proxy.
Si es así, asegúrate de que tu servidor proxy esté incluido en `web.config` para que el `HttpClient`
que usa el SDK de Sentry pueda detectarlo.

```xml
<system.net>
    <defaultProxy>
      <proxy proxyaddress="http://proxy.address.here" bypassonlocal="true" />
    </defaultProxy>
</system.net>
```


<div id="tls-12-support-windows-server">
  ### Compatibilidad con TLS 1.2 y Windows Server
</div>

Si no puedes capturar eventos de ASP.NET en Sentry
en versiones antiguas de Windows Server, con una versión antigua de .NET Framework, debes habilitar TLS 1.2.

```
using System.Net;

ServicePointManager.SecurityProtocol = SecurityProtocolType.Tls12
```


<div id="sentrsdkinit-with-error">
  ### SentrSdk.Init con error
</div>

Si aparece el siguiente error durante la inicialización de Sentry, es posible que haya una configuración incorrecta en tu Web.config:

`System.IO.FileNotFoundException: Could not load file or assembly System.Net.Http, Version=4.0.0.0`

Puedes solucionarlo con una de estas tres opciones:

* Busca el `assemblyIdentity` de `System.Net.Http` y elimina el `bindingRedirect`.

```xml
<dependentAssembly>
  <assemblyIdentity name="System.Net.Http" publicKeyToken="b03f5f7f11d50a3a" culture="neutral" />
</dependentAssembly>
```

* Busca el `assemblyIdentity` de `System.Net.Http` y reduce `newVersion` a `4.0.0.0`.

```xml
<dependentAssembly>
  <assemblyIdentity name="System.Net.Http" publicKeyToken="b03f5f7f11d50a3a" culture="neutral" />
  <bindingRedirect oldVersion="0.0.0.0-4.2.0.0" newVersion="4.0.0.0" />
</dependentAssembly>
```

* Instala el paquete más reciente de `System.Net.Http` desde NuGet en tu proyecto.

Puedes encontrar más información sobre el problema [en este issue de GitHub](https://github.com/dotnet/runtime/issues/24382).


### Las etiquetas no se muestran si se configuran en Application&#95;Start

Esto sucede porque el `scope` de `requests` será distinto del que se usa durante la ejecución de Application&#95;Start.
Para aplicar las `Tags` de forma global, se recomienda configurarlas en `SentryOptions`, como se muestra en el siguiente ejemplo:

```csharp
SentrySdk.Init(options =>
{
  options.Dsn = "___PUBLIC_DSN___";
  options.SendDefaultPii = true;
  options.DefaultTags.Add("Etiqueta", "valor");
});
```
