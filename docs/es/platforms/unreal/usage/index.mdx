El SDK de Sentry se integra con tu entorno de ejecución y reporta automáticamente errores, excepciones no controladas y rechazos no gestionados, así como otros tipos de errores según la plataforma.

<Alert>

Términos clave:

- Un _evento_ es una instancia de envío de datos a Sentry. Generalmente, estos datos son un error o una excepción.
- Un _issue_ es un agrupamiento de eventos similares.
- Al reporte de un evento se le llama _captura_.
  Cuando se captura un evento, se envía a Sentry.

</Alert>

La forma más común de captura es la de errores. Lo que puede capturarse como error varía según la plataforma. En general, si tienes algo que parece una excepción, puede capturarse. Para algunos SDK, también puedes omitir el argumento de <PlatformIdentifier name="capture-exception" /> y Sentry intentará capturar la excepción actual. También es útil para reportar manualmente errores o mensajes a Sentry.

Mientras capturas un evento, también puedes registrar las breadcrumbs que condujeron a ese evento. Las breadcrumbs son diferentes de los eventos: no crearán un evento en Sentry, sino que se almacenarán en búfer hasta que se envíe el siguiente evento. Obtén más información sobre breadcrumbs en nuestra <PlatformLink to="/enriching-events/breadcrumbs/">documentación de Breadcrumbs</PlatformLink>.

<div id="capturing-messages">
  ## Captura de mensajes
</div>

Otra operación común es capturar un mensaje sin contexto adicional. Un mensaje es información textual que debe enviarse a Sentry. Normalmente, nuestros SDK no capturan mensajes automáticamente, pero puedes capturarlos manualmente.

Los mensajes aparecen como incidencias en tu flujo de trabajo de incidencias, con el propio mensaje como nombre de la incidencia.
