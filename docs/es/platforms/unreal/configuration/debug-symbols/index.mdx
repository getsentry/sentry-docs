Sentry requiere [archivos de información de depuración](/es/platforms/unreal/data-management/debug-files/) para simbolicar fallos. El SDK de Unreal Engine ofrece una funcionalidad de carga automática de esos archivos de símbolos que depende de [sentry-cli](/es/cli/). Esto se hace de forma automática, por lo que no es necesario ejecutar `sentry-cli` manualmente. La carga de símbolos ocurre durante la ejecución de `PostBuildSteps`, tal como se especifica en el archivo `Sentry.uplugin`. Los ejecutables de `sentry-cli` para Windows, macOS y Linux se incluyen como parte del paquete del SDK de Unreal Engine. También se pueden descargar manualmente desde el menú de configuración si el plugin se instaló desde el Marketplace de UE.

<Alert>
  En Android, la carga de símbolos de depuración la gestiona el [Sentry Android Gradle Plugin](/es/platforms/android/configuration/gradle). De forma predeterminada, actualiza automáticamente Gradle a la versión 7.5; no obstante, se puede volver a su versión anterior en caso de que surjan problemas de compatibilidad con otros plugins de terceros.
</Alert>

<div id="automatic-upload">
  ## Carga automática
</div>

La carga automática de símbolos de depuración está deshabilitada de forma predeterminada y requiere configuración. Para configurarla, ve a **Project Settings &gt; Plugins &gt; Sentry** y habilita “Upload debug symbols automatically”.

Como alternativa, puedes habilitar la carga de símbolos de depuración estableciendo la variable de entorno `SENTRY_UPLOAD_SYMBOLS_AUTOMATICALLY` en `True`. Esto puede ser especialmente útil en entornos de CI/CD donde la configuración manual no es práctica.

<Alert>
  Sentry cargará símbolos de depuración solo para las configuraciones de compilación y los tipos de destino que estén habilitados en la sección “Misc” de la configuración del plugin.
</Alert>

![Configuración de la carga de símbolos de depuración de Unreal Engine](./img/unreal-debug-symbols.png)

`sentry-cli` requiere que especifiques la organización y el proyecto con los que trabajar, así como un token para autenticarte en Sentry. El SDK de Unreal Engine ofrece dos formas de configurar estos ajustes: mediante un archivo de propiedades (predeterminado) o mediante variables de entorno.

<div id="properties-file">
  ### Archivo de propiedades
</div>

Para configurar `sentry-cli` mediante un archivo de propiedades, introduce el [token de autenticación](https://sentry.io/orgredirect/organizations/:orgslug/settings/auth-tokens/), el slug de la organización y el nombre del proyecto. Ten en cuenta que el SDK de Unreal Engine crea automáticamente un archivo `sentry.properties` en la raíz del directorio de tu proyecto para almacenar la configuración. Este archivo de configuración **nunca** debe hacerse público.

<div id="environment-variables">
  ### Variables de entorno
</div>

Puedes configurar `sentry-cli` estableciendo las siguientes variables de entorno, que se detectarán automáticamente durante la carga de los símbolos de depuración si no se encuentra el archivo de propiedades:

```bash
export SENTRY_ORG=___ORG_SLUG___
export SENTRY_PROJECT=___PROJECT_SLUG___
export SENTRY_AUTH_TOKEN=___ORG_AUTH_TOKEN___
```

<div id="manual-upload-using-sentry-cli">
  ## Carga manual con sentry-cli
</div>

Para cargar manualmente símbolos de depuración en Sentry, ejecuta `sentry-cli` desde la línea de comandos. Puedes usar los ejecutables incluidos en el paquete o seguir la [documentación de sentry-cli](/es/cli/installation/) para instalarlo de forma global. Para subir los símbolos de depuración, ejecuta el siguiente comando:

<OrgAuthTokenNote />

```bash
sentry-cli --auth-token ___ORG_AUTH_TOKEN___ debug-files upload --org ___ORG_SLUG___ --project ___PROJECT_SLUG___ PATH_TO_SYMBOLS
```

<div id="debug-symbols-for-macos-and-ios">
  ## Símbolos de depuración para macOS y iOS
</div>

Sentry requiere [*archivos dSYM*](/es/platforms/unreal/data-management/debug-files/file-formats/#mach-o-and-dsym) para simbolicar por completo tus trazas de pila en juegos de macOS/iOS. Para asegurarte de que los símbolos de depuración se generen y puedan ser procesados por el complemento de Sentry para Unreal durante la ejecución de `PostBuildSteps`, agrega los siguientes ajustes de compilación al archivo `.Target.cs` de tu proyecto:

```csharp
// Para Mac
MacPlatform.bUseDSYMFiles = true;

// Para iOS
IOSPlatform.bGeneratedSYM = true;
```

<Alert>
  Unreal Engine 5.3 y versiones posteriores son compatibles con el [flujo de trabajo modernizado de Xcode](https://dev.epicgames.com/documentation/en-us/unreal-engine/using-modern-xcode-in-unreal-engine-5.3-and-newer), que se alinea más estrechamente con las prácticas estándar de desarrollo de Apple. En este caso, la carga de símbolos puede gestionarse durante la [fase de compilación de Xcode](/es/platforms/apple/guides/macos/dsym/#xcode-build-phase) personalizada.
</Alert>
