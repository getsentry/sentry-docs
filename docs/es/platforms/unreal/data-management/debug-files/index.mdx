Los archivos de información de depuración permiten a Sentry extraer trazas de pila y ofrecer más
información sobre los informes de fallos en la mayoría de las plataformas compiladas. La información almacenada
en los archivos de depuración incluye nombres originales de funciones, rutas a archivos fuente y números de
línea, contexto del código fuente o la ubicación de variables en memoria. Sentry
puede usar parte de esta información y mostrarla en la página de detalles de la incidencia.

Cada plataforma principal utiliza distintos archivos de información de depuración. Actualmente admitimos
los siguientes formatos:

* [*Archivos dSYM*](./file-formats/#mach-o-and-dsym) para iOS, iPadOS,
  tvOS, watchOS, macOS y visionOS
* [*Símbolos ELF*](./file-formats/#executable-and-linkable-format-elf) para Linux y
  Android (NDK)
* [*Archivos PDB*](./file-formats/#pe-and-pdb) para Windows y .NET
* [*Símbolos Breakpad*](./file-formats/#breakpad-symbols) para todas las
  plataformas
* [*Archivos WASM*](./file-formats/#wasm) para WebAssembly
* [*Mapeos de ProGuard*](./file-formats/#proguard-mappings) para
  Java y Android

<Alert>
  Los source maps, aunque también son archivos de información de depuración, se gestionan
  de forma diferente en Sentry. Para obtener más información, consulta [Source maps en sentry-cli](/es/cli/releases/#sentry-cli-sourcemaps).
</Alert>

Sentry requiere acceso a los archivos de información de depuración de tu aplicación, así como
a las bibliotecas del sistema, para proporcionar informes de fallos totalmente simbolizados. Puedes
[subir](./upload/) tus archivos a Sentry o alojarlos en un
[Servidor de símbolos](./symbol-servers/) compatible para que Sentry los descargue cuando sea necesario.

<div id="managing-debug-information-files">
  ## Gestión de archivos de información de depuración
</div>

Después de subirlos, los archivos de información de depuración se pueden ver y gestionar
en la sección *Debug Files* de la página de configuración del proyecto correspondiente. Esa página
muestra todos los archivos de depuración subidos y también te permite configurar servidores de símbolos
para descargas automáticas.

Desde la página [Project Details](/es/product/projects/project-details/), ve a settings
y luego haz clic en *Debug Files* en la navegación de la página.

<Alert>
  Los archivos de ProGuard se muestran por separado, en la sección *ProGuard* de la página de configuración del proyecto.
</Alert>

<div id="retention-policy">
  ## Política de retención
</div>

Los archivos de depuración tienen un periodo de retención de *90 días*, con un mecanismo de expiración por *tiempo de inactividad*.
Esto significa que los archivos de depuración cargados se conservan mientras se usen activamente para procesar eventos.
Cuando un archivo de depuración no se haya utilizado para procesar eventos entrantes durante al menos 90 días,
expirará automáticamente y podrá eliminarse.
