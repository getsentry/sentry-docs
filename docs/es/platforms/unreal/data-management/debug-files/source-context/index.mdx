Si Sentry tiene acceso al código fuente de tu aplicación, puede mostrar fragmentos de código (*source context*) alrededor de la ubicación de los marcos de la pila, lo que ayuda a identificar rápidamente el código problemático.

Por ejemplo, esta es una traza de pila de una aplicación .NET. El contexto de código fuente contiene
el código C# original de la aplicación, con la ubicación del marco resaltada.

<div id="source-bundles">
  ### Paquetes de código fuente
</div>

Ciertos SDK, como el SDK de Python, pueden resolver automáticamente este contexto de código fuente,
porque tienen acceso a código fuente sin ofuscar en tiempo de ejecución.
En cambio, para obtener el contexto de código fuente en aplicaciones compiladas, es necesario cargar
el código fuente junto con los archivos de información de depuración.
La forma recomendada de hacerlo es usando `sentry-cli`.
Consulta [Creación de paquetes de código fuente](/es/cli/dif/#creating-source-bundles) para más información.

Después de cargarlos, puedes revisar y administrar los paquetes de código fuente igual que
cualquier otro archivo de información de depuración. Consulta [Administrar archivos de información de depuración](../#managing-debug-information-files).
Los paquetes de código fuente se etiquetarán con `sources` y tendrán el mismo nombre de archivo que el
archivo de información de depuración correspondiente del que se crearon.
