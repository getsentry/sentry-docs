<div id="structured-context">
  ## Contexto estructurado
</div>

La mejor forma de adjuntar datos personalizados es con un contexto estructurado. Un contexto siempre debe ser un objeto y sus valores pueden ser arbitrarios.

Luego, usa <PlatformIdentifier name="set-context" /> y asígnale al contexto un nombre único:

```cpp
USentrySubsystem* SentrySubsystem = GEngine->GetEngineSubsystem<USentrySubsystem>();

TMap<FString, FSentryVariant> ContextData;
AdditionalData.Add("Class", "Paladin");
AdditionalData.Add("Role", "Taunt");

SentrySubsystem->SetContext("Character", ContextData);
```

El mismo resultado se puede obtener llamando a la función correspondiente en Blueprint:

![Set context](./unreal_set_context.png)

Como alternativa, esta configuración se puede proporcionar al crash reporter [durante la inicialización](/es/platforms/unreal/configuration/setup-crashreporter/#configure-attributes).

<Alert>
  El SDK de Sentry adjunta automáticamente a todos los eventos ciertos valores de contexto, como `Engine version`, `Configuration` y `Target Type`, entre otros.
</Alert>

No hay restricciones para el nombre del contexto. En el objeto de contexto, se permiten todas las claves excepto `type`, que se usa internamente.

Obtén más información sobre las convenciones de contextos comunes en la [documentación para desarrolladores de la interfaz de contexts](https://develop.sentry.dev/sdk/data-model/event-payloads/contexts/).

<div id="size-limitations">
  ## Limitaciones de tamaño
</div>

Al enviar contexto, *ten en cuenta los límites de tamaño del payload*. Sentry no recomienda enviar todo el estado de la aplicación ni grandes blobs de datos en los contextos. Si superas el tamaño máximo del payload, Sentry responderá con el error HTTP `413 Payload Too Large` y rechazará el evento.

El SDK de Sentry hará todo lo posible por adaptarse a los datos que envíes y recortar payloads de contexto grandes. Algunos SDK pueden truncar partes del evento; para más detalles, consulta la [documentación para desarrolladores sobre el manejo de datos del SDK](https://develop.sentry.dev/sdk/expected-features/data-handling/).

<div id="additional-data">
  ## Datos adicionales
</div>

**Datos adicionales ha quedado obsoleto** en favor de los contextos estructurados.

Sentry solía admitir la adición de “datos adicionales” no estructurados mediante <PlatformIdentifier name="set-extra" />.
