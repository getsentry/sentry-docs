Una versión es una edición de tu código que se despliega en un <PlatformLink to="/configuration/environments/">entorno</PlatformLink>. Cuando proporcionas a Sentry información sobre tus versiones, puedes:

* Detectar problemas y regresiones introducidos en una versión nueva
* Predecir qué commit causó un problema y quién probablemente sea responsable
* Resolver problemas incluyendo el número del issue en tu mensaje de commit
* Recibir notificaciones por correo electrónico cuando se despliegue tu código

<div id="setting-a-release">
  ## Configuración de una versión
</div>

<PlatformContent includePath="set-release" notateUnsupported />

Cómo pongas a disposición de tu código el nombre de la versión (o release) depende de ti. Por ejemplo, puedes usar una variable de entorno definida durante el proceso de build o al inicio de la aplicación.

Definir el nombre de la versión etiqueta cada evento con ese nombre. Te recomendamos informar a Sentry sobre una nueva versión antes de enviar eventos con ese nombre, ya que esto habilitará algunas funciones adicionales. Obtén más información en nuestra documentación de [Releases](/es/product/releases/).

Si no informas a Sentry sobre una nueva versión, Sentry creará automáticamente una entidad de versión en el sistema la primera vez que detecte un evento con ese ID de versión.

Después de configurar tu SDK, puedes instalar una integración de repositorio o proporcionar manualmente a Sentry tus propios metadatos de commits. Consulta nuestra documentación sobre [configuración de versiones](/es/product/releases/setup/) para más información sobre integraciones, asociación de commits y cómo informar a Sentry al desplegar versiones.
