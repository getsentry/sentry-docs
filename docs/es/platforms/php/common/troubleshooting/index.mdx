<Expandable title="¿Cómo puedo registrar eventos internos del SDK?">

Un buen primer paso para depurar problemas del SDK es usar la opción `logger`. Para tu comodidad, el SDK incluye dos implementaciones: `DebugFileLogger` y `DebugStdOutLogger`.

```php
\Sentry\init([
    'logger' => new \Sentry\Logger\DebugFileLogger('/path/to/your/logfile.log'),
]);

// o

\Sentry\init([
    'logger' => new \Sentry\Logger\DebugStdOutLogger(),
]);
```

<Expandable title="Faltan variables en las trazas de pila">

Esto se debe a la configuración de PHP. PHP 7.4 introdujo un nuevo ajuste en el .ini, `zend.exception_ignore_args`, que hace que se ignoren los argumentos de la traza de pila. Revisa tu archivo .ini para asegurarte de que este ajuste esté en `Off` o `0`.

</Expandable>

<Expandable title="¿Por qué se truncó el valor de mi etiqueta?">

  Actualmente, cada etiqueta tiene un límite máximo de 200 caracteres. Las etiquetas que superen ese límite se truncarán, lo que podría hacer que se pierda información importante. Para conservar estos datos, puedes dividir la información en varias etiquetas.

  Por ejemplo, una solicitud etiquetada con más de 200 caracteres:

  `https://empowerplant.io/api/0/projects/ep/setup_form/?user_id=314159265358979323846264338327&tracking_id=EasyAsABC123OrSimpleAsDoReMi&product_name=PlantToHumanTranslator&product_id=161803398874989484820458683436563811772030917980576`

  La solicitud anterior de más de 200 caracteres se truncará a:

  `https://empowerplant.io/api/0/projects/ep/setup_form/?user_id=314159265358979323846264338327&tracking_id=EasyAsABC123OrSimpleAsDoReMi&product_name=PlantToHumanTranslator&product_id=1618033988749894848`

</Expandable>

<Expandable title="¿Por qué no veo datos de perfiles?">

Si no ves datos de perfiles en [sentry.io](https://sentry.io), prueba lo siguiente:

  - Asegúrate de que <PlatformLink to="/tracing/">Tracing esté habilitado</PlatformLink>.
  - Verifica que la instrumentación automática esté enviando datos de rendimiento a Sentry en la página **Performance** de [sentry.io](https://sentry.io).
  - Verifica que la <PlatformLink to="/tracing/instrumentation/custom-instrumentation">instrumentación personalizada</PlatformLink> esté enviando datos de rendimiento a Sentry en la página **Performance** de [sentry.io](https://sentry.io).
  - Asegúrate de que <PlatformLink to="/integrations/#environmentintegration">`Sentry\Integration\EnvironmentIntegration`</PlatformLink> esté habilitada.
  - Habilita el <PlatformLink to="/configuration/options/#logger">logger</PlatformLink> para ver qué hace el SDK bajo el capó.

</Expandable>

<Expandable title="¿Por qué los errores de trabajos recurrentes no aparecen en la página de detalles de mi monitor?">

  Puede que no hayas <PlatformLink to="/crons/#connecting-errors-to-cron-monitors">vinculado los errores a tu monitor</PlatformLink>.

</Expandable>

<Expandable title="¿Por qué no recibo alertas cuando mi monitor falla?">

  Puede que no hayas <PlatformLink to="/crons/#alerts">configurado alertas para tu monitor</PlatformLink>.

</Expandable>

<Expandable title="¿Cuál es la política de retención de datos de crons para los check-ins?">

  Nuestra política de retención de datos actual es de 90 días.

</Expandable>

<Expandable title="¿Admiten un horario de monitor con una expresión crontab de seis campos?">

  Actualmente, solo admitimos expresiones crontab de cinco campos.

</Expandable>

<Expandable title="Mis trabajos se agotan porque todos empiezan al mismo tiempo y a veces fallan, ¿cómo puedo solucionarlo?">

  Puedes ejecutar tus trabajos en segundo plano o usar la opción `checkInMargin` para ampliar la ventana.

</Expandable>
