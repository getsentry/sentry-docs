Todos los eventos tienen una huella digital. Los eventos con la misma huella digital se agrupan en un issue.

De forma predeterminada, Sentry ejecuta uno de nuestros algoritmos de agrupación integrados para generar una huella digital basada en la información disponible en el evento, como `stacktrace`, `exception` y `message`. Para ampliar el comportamiento de agrupación predeterminado o cambiarlo por completo, puedes usar una combinación de las siguientes opciones:

1. En tu SDK, usando la huella digital del SDK, como se documenta a continuación
2. En tu proyecto, usando [Fingerprint Rules](/es/concepts/data-management/event-grouping/fingerprint-rules/) o [Stack Trace Rules](/es/concepts/data-management/event-grouping/stack-trace-rules/)

En los SDK compatibles, puedes reemplazar la agrupación predeterminada de Sentry pasando el atributo de huella digital como un arreglo de cadenas. La longitud del arreglo de una huella digital no está restringida. Esto funciona de forma similar a la [funcionalidad de Fingerprint Rules](/es/concepts/data-management/event-grouping/fingerprint-rules/), que siempre está disponible y puede lograr resultados similares.

Puedes usar sustitución de variables para insertar valores dinámicos en la huella (fingerprint) que normalmente se calcula en el servidor. Por ejemplo, el valor `{{ default }}` se puede añadir para incluir en la huella todo el hash de agrupación que se genera normalmente. Estos valores son los mismos que en la creación de huellas del lado del servidor. Consulta [Variables](/es/concepts/data-management/event-grouping/fingerprint-rules/#variables) para obtener más información.
