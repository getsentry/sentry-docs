El SDK de Sentry se integra con tu entorno de ejecución y envía automáticamente informes de errores, excepciones no capturadas y rechazos no controlados, así como otros tipos de errores según la plataforma.

<Alert>

Términos clave:

- Un _evento_ es una instancia de envío de datos a Sentry. Generalmente, estos datos son un error o una excepción.
- Un _issue_ es un conjunto de eventos similares.
- Al envío de un evento se le llama _captura_.
  Cuando se captura un evento, se envía a Sentry.

</Alert>

La forma más común de capturar es capturar errores. Lo que puede capturarse como error varía según la plataforma. En general, si tienes algo que parece una excepción, se puede capturar. En algunos SDK, también puedes omitir el argumento de <PlatformIdentifier name="capture-exception" /> y Sentry intentará capturar la excepción actual. También es útil para enviar manualmente informes de errores o mensajes a Sentry.

Mientras capturas un evento, también puedes registrar las breadcrumbs que llevaron a ese evento. Las breadcrumbs son diferentes de los eventos: no crean un evento en Sentry, sino que se almacenan en búfer hasta que se envíe el siguiente evento. Obtén más información sobre las breadcrumbs en nuestra <PlatformLink to="/enriching-events/breadcrumbs/">documentación de Breadcrumbs</PlatformLink>.

  ## Captura de mensajes

Otra operación común es capturar un mensaje simple. Un mensaje es información textual que debe enviarse a Sentry. Normalmente, nuestros SDK no capturan mensajes automáticamente, pero puedes capturarlos manualmente.

Los mensajes aparecen como incidencias en tu flujo de incidencias, con el mensaje como nombre de la incidencia.
