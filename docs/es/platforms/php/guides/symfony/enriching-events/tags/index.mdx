**Las etiquetas** son pares de cadenas clave/valor que están indexados y se pueden buscar. Las etiquetas habilitan funciones en sentry.io como filtros y mapas de distribución de etiquetas. También te ayudan a acceder rápidamente a eventos relacionados y a ver la distribución de etiquetas para un conjunto de eventos. Entre los usos comunes de las etiquetas se incluyen el nombre de host, la versión de la plataforma y el idioma del usuario.

Indexaremos automáticamente todas las etiquetas de un evento, así como la frecuencia y la última vez que Sentry ha visto una etiqueta. También llevamos un registro del número de etiquetas distintas y podemos ayudarte a identificar puntos críticos en diversos problemas.

*Las claves de las etiquetas* tienen una longitud máxima de 32 caracteres y solo pueden contener letras (`a-zA-Z`), números (`0-9`), guiones bajos (`_`), puntos (`.`), dos puntos (`:`) y guiones (`-`).

*Los valores de las etiquetas* tienen una longitud máxima de 200 caracteres y no pueden contener el carácter de salto de línea (`\n`).

<Alert>
  Algunas etiquetas las establece automáticamente Sentry. Recomendamos encarecidamente no sobrescribir estas [etiquetas](/es/concepts/search/searchable-properties/#search-properties). En su lugar, nombra tus etiquetas según la nomenclatura de tu organización. Si sobrescribes una etiqueta establecida automáticamente, debes usar la [sintaxis de etiqueta explícita](/es/concepts/search/#explicit-tag-syntax) para buscarla.
</Alert>

Una vez que empieces a enviar datos con etiquetas, los verás al iniciar sesión en sentry.io. Allí podrás ver los filtros en la barra lateral de la página del proyecto, resumidos dentro de un evento y en la página Etiquetas para un evento agregado.

<div id="symfony-configuration">
  ## Configuración de Symfony
</div>

También puedes añadir una etiqueta mediante las [opciones de configuración de Symfony](/es/platforms/php/guides/symfony/configuration/symfony-options/) en `config/packages/sentry.yaml`.

{/* <!-- prettier-ignore --> */}

```yaml {filename:config/packages/sentry.yaml}
sentry:
    options:
        tags:
            app_tag1: "%env(APP_CUSTOM_TAG1)%"
            app_tag2: "value2"
```
