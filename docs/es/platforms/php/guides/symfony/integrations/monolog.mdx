---
title: Monolog
description: "Aprende a habilitar el SDK de Symfony de Sentry para capturar eventos de Monolog."
---

Al usar [Monolog](https://github.com/Seldaek/monolog) puedes configurar un controlador de [breadcrumbs](../../enriching-events/breadcrumbs/) para capturar mensajes de Monolog como breadcrumbs y otro controlador que capture mensajes como eventos en Sentry.
El controlador de breadcrumbs no envía nada directamente a Sentry; solo registra breadcrumbs que se adjuntarán a cualquier evento o excepción que se envíe a Sentry.

```yaml {filename:config/packages/sentry.yaml}
sentry:
    register_error_listener: false # Desactiva ErrorListener para evitar registros duplicados en Sentry
    register_error_handler: false # Desactiva las integraciones ErrorListener, ExceptionListener y FatalErrorListener del SDK base de PHP

services:
    # (Opcional) Configura el controlador de breadcrumbs como un servicio (necesario para el controlador de Monolog de breadcrumbs)
    Sentry\Monolog\BreadcrumbHandler:
        arguments:
            - '@Sentry\State\HubInterface'
            - !php/const Monolog\Logger::INFO # Configura el nivel de mensajes que se capturarán como breadcrumbs

monolog:
    handlers:
        # (Opcional) Registra el controlador de breadcrumbs como un controlador de Monolog
        sentry_breadcrumbs:
            type: service
            name: sentry_breadcrumbs
            id: Sentry\Monolog\BreadcrumbHandler
        # Registra el controlador como un controlador de Monolog para capturar mensajes como eventos
        sentry:
            type: sentry
            level: !php/const Monolog\Logger::ERROR # Configura el nivel de mensajes que se capturarán como eventos
            hub_id: Sentry\State\HubInterface
            fill_extra_context: true # Habilita el envío del contexto de Monolog a Sentry
            process_psr_3_messages: false # Desactiva la resolución de placeholders de PSR-3 en los mensajes informados
```
