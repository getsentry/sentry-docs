<div id="prerequisites">
  ## Requisitos previos
</div>

* Necesitas una [cuenta de Sentry](https://sentry.io/signup/) y un proyecto
* Tu aplicación de Symfony debe ejecutarse en PHP 7.2 o posterior

<div id="install">
  ## Instalación
</div>

Instala el paquete `sentry/sentry-symfony`:

```bash
composer require sentry/sentry-symfony
```

Si usas [Monolog](https://github.com/Seldaek/monolog) para informar eventos en lugar del enfoque típico de un listener de errores, necesitas esta [configuración adicional](integrations/monolog/) para registrar correctamente los errores.


<div id="configure">
  ## Configuración
</div>

Añade tu DSN a tu archivo `.env`:

```plain {filename:.env}
###> sentry/sentry-symfony ###
SENTRY_DSN="___PUBLIC_DSN___"
###< sentry/sentry-symfony ###
```

Al usar Symfony Flex, el bundle de Sentry está configurado para estar habilitado solo en el entorno `prod` de forma predeterminada. Puedes cambiar este comportamiento en tus archivos `config/packages/sentry.yaml` y `config/bundles.php`.

<Alert level="warning">
  Para recibir los argumentos de la traza en tus errores, asegúrate de establecer `zend.exception_ignore_args: Off` en tu php.ini
</Alert>


<div id="verify">
  ## Verificar
</div>

Para comprobar que tanto los errores del registrador como las excepciones se envían correctamente a sentry.io, puedes crear el siguiente controlador:

```php
<?php

namespace App\Controller;

use Psr\Log\LoggerInterface;
use Symfony\Bundle\FrameworkBundle\Controller\AbstractController;
use Symfony\Component\Routing\Attribute\Route;

class SentryTestController extends AbstractController
{
    public function __construct(private LoggerInterface $logger)
    {
    }

    #[Route('/_sentry-test', name: 'sentry_test')]
    public function testLog()
    {
        // el siguiente código probará si la integración de monolog registra en sentry
        $this->logger->error('Mi error personalizado registrado.', ['some' => 'Context Data']);

        // el siguiente código probará si una excepción no capturada se registra en sentry
        throw new \RuntimeException('Excepción de ejemplo.');
    }
}
```

Después de visitar la página `/_sentry-test`, puedes ver y resolver el error registrado iniciando sesión en [sentry.io](https://sentry.io) y abriendo tu proyecto. Al hacer clic en el título del error, se abrirá una página donde podrás ver información detallada y marcarlo como resuelto.
