---
title: Datos recopilados
description: "Consulta qué datos recopila el SDK de Sentry."
sidebar_order: 1
---

En Sentry nos tomamos muy en serio la privacidad de los datos y contamos con configuraciones predeterminadas que priorizan la seguridad de la información, especialmente en lo referente a datos de identificación personal (PII). Al agregar el SDK de Sentry a tu aplicación, autorizas que recopile datos y los envíe a Sentry durante la ejecución de tu aplicación.

Los tipos de categorías y la cantidad de datos recopilados varían según las integraciones que tengas habilitadas en el SDK de Sentry. A continuación, encontrarás una lista de las categorías de datos que recopila el SDK de Sentry para PHP:

<div id="http-headers">
  ## Encabezados HTTP
</div>

De forma predeterminada, el SDK de Sentry no envía encabezados HTTP.

Para empezar a enviar encabezados HTTP, establece la opción <PlatformLink to="/configuration/options/#send_default_pii">send_default_pii</PlatformLink> en `true`.

<div id="cookies">
  ## Cookies
</div>

De forma predeterminada, el SDK de Sentry no envía cookies.

Si quieres enviar cookies, configura la opción <PlatformLink to="/configuration/options/#send_default_pii">send_default_pii</PlatformLink> en `true`.

<div id="information-about-logged-in-user">
  ## Información sobre el usuario autenticado
</div>

De forma predeterminada, el SDK de Sentry no envía ninguna información sobre el usuario autenticado (como el ID de usuario o el nombre de usuario del suplantador).

Para empezar a enviar información del usuario autenticado, establece la opción <PlatformLink to="/configuration/options/#send_default_pii">send_default_pii</PlatformLink> en `true`.

<div id="users-ip-address">
  ## Dirección IP de los usuarios
</div>

De forma predeterminada, el SDK de Sentry no envía la dirección IP del usuario.

Para activar el envío de la dirección IP del usuario, establece la <PlatformLink to="/configuration/options/#send_default_pii">opción send_default_pii</PlatformLink> en `true`.

<div id="request-url">
  ## URL de la solicitud
</div>

La URL completa de las solicitudes HTTP salientes y entrantes **siempre se envía a Sentry**. Según tu aplicación, esto podría incluir datos personales (PII).

<div id="request-query-string">
  ## Cadena de consulta de la solicitud
</div>

La cadena de consulta completa de las solicitudes HTTP, tanto salientes como entrantes, **siempre se envía a Sentry**. Dependiendo de tu aplicación, esto podría incluir datos personales (PII).

<div id="request-body">
  ## Cuerpo de la solicitud
</div>

El cuerpo de las solicitudes HTTP entrantes se puede enviar a Sentry. Que se envíe o no depende del tipo y el tamaño del cuerpo, como se describe a continuación:

- **Tipo de cuerpo de la solicitud:**
  - Se envían los cuerpos JSON y de formularios
  - Los cuerpos sin procesar siempre se eliminan
  - Los archivos subidos en el cuerpo de la solicitud nunca se envían a Sentry
- **Tamaño del cuerpo de la solicitud:** Existe una <PlatformLink to="/configuration/options/#max_request_body_size">opción max_request_body_size</PlatformLink> que está configurada en `medium` de forma predeterminada. Esto significa que los cuerpos de solicitud más grandes no se envían a Sentry.

Si quieres evitar que se envíen cuerpos a Sentry por completo, establece `max_request_body_size` en `'never'`.

<div id="source-context">
  ## Contexto del código
</div>

Cuando se envía a Sentry una excepción no controlada, se adjunta una instantánea del código fuente que rodea la línea donde se originó el error.

Para dejar de enviar este contexto de código a Sentry, establece la opción <PlatformLink to="/configuration/options/#context_lines">context_lines</PlatformLink> en `0`.

<div id="local-variables-in-stack-trace">
  ## Variables locales en la traza de pila
</div>

Cuando se envían a Sentry errores y excepciones no gestionados, los nombres y valores de las variables locales establecidas en el momento en que ocurrieron los errores se envían al mismo tiempo.

Puedes dejar de enviar variables locales a Sentry estableciendo `zend.exception_ignore_args=1` en tu `php.ini`. En algunas distribuciones, este ajuste ya viene en `1` de forma predeterminada y, si quieres incluir variables locales, establece `zend.exception_ignore_args=0`.
