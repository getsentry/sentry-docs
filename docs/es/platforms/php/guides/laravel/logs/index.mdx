<div id="troubleshooting">
  ## Solución de problemas
</div>

Si tus logs no aparecen en Sentry, verifica o prueba lo siguiente:

Si estás probando en Tinker (`php artisan tinker`), asegúrate de llamar manualmente a `\Sentry\logger()->flush();` para enviar los logs a Sentry, ya que no se envían automáticamente.

Revisa tu `config/logging.php` y asegúrate de que el canal de registro predeterminado y el canal de stack permitan la configuración mediante variables de entorno:

```php {filename:config/logging.php}
    // ...
    'default' => env('LOG_CHANNEL', 'stack'),
    // ...
    'channels' => [
        'stack' => [
            'driver' => 'stack',
            'channels' => explode(',', (string) env('LOG_STACK', 'single')),
            'ignore_exceptions' => false,
        ],
    // ...
```

Asegúrate de que, si has definido `LOG_LEVEL` en tu archivo `.env`, esté establecido en un nivel que permita que los logs que esperas se envíen a Sentry.

Para probar tu configuración independientemente de la configuración de Laravel, puedes enviar un log de prueba a Sentry con el siguiente fragmento:

```php
\Sentry\logger()->info('Un mensaje de prueba del registro');
\Sentry\logger()->flush();
```

Si eso no funciona, verifica la configuración de tu DSN con `php artisan sentry:test`.
