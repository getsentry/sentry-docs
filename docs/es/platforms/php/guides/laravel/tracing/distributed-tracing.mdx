title: Trazabilidad distribuida
sidebar_order: 40
description: "Aprende a conectar transacciones de backend y frontend."
---

Si también usas Tracing para [JavaScript](/es/platforms/javascript/tracing/), puedes usar una función auxiliar para continuar la traza y propagar el contexto de traza iniciado en tu backend para habilitar la [trazabilidad distribuida](/es/platforms/javascript/tracing/trace-propagation/). (El contexto de traza contiene datos adicionales relacionados con la traza que se utilizan para el muestreo basado en trazas.)
Añade la siguiente línea a tu plantilla Blade que renderiza el `<head/>` de tu página:

```php {filename:app.blade.php}
<head>
...
{!! \Sentry\Laravel\Integration::sentryMeta() !!}
...
</head>
```

Esta función auxiliar generará dos etiquetas meta similares a `<meta name="sentry-trace" content="49879feb76c84743ba5034bd2d3f1ca3-7cb5535c930d4666-1"/>` y `<meta name="baggage" content="sentry-trace_id=49879feb76c84743ba5034bd2d3f1ca3,sentry-sample_rate=0.2,sentry-transaction=my-transaction,sentry-public_key=public,sentry-release=1.0.0,sentry-environment=prod">`. Nuestro SDK de JavaScript las detectará y luego continuará el trace y propagará el contexto de trace.
Como resultado, tu frontend y tu backend quedan conectados mediante el mismo trace.

De lo contrario, otros servicios de backend con Tracing se conectarán automáticamente.
