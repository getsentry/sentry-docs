Un lanzamiento es una versión de tu código que se despliega en un <PlatformLink to="/configuration/environments/">entorno</PlatformLink>. Cuando le das a Sentry información sobre tus lanzamientos, puedes:

* Identificar problemas y regresiones introducidos en un nuevo lanzamiento
* Predecir qué commit causó un problema y quién probablemente es responsable
* Resolver problemas al incluir el número del issue en el mensaje del commit
* Recibir notificaciones por correo electrónico cuando se despliega tu código

Además, los lanzamientos se usan para aplicar [source maps](/es/platforms/javascript/sourcemaps/) a JavaScript minificado y ver el código fuente original, sin transformar.

  ## Configurar una versión
</div>

<PlatformContent includePath="set-release" notateUnsupported />

Cómo haces disponible el nombre de la versión (o release) para tu código depende de ti. Por ejemplo, puedes usar una variable de entorno que se establezca durante el proceso de compilación o durante el arranque inicial.

Definir el nombre de la versión etiqueta cada evento con ese nombre. Recomendamos informar a Sentry sobre una nueva versión antes de enviar eventos con ese nombre, ya que así se desbloquean algunas funciones adicionales. Obtén más información en nuestra documentación de [Releases](/es/product/releases/).

Si no informas a Sentry sobre una nueva versión, Sentry creará automáticamente una entidad de versión en el sistema la primera vez que vea un evento con ese ID de versión.

Después de configurar tu SDK, puedes instalar una integración de repositorio o proporcionar manualmente a Sentry tus propios metadatos de commits. Consulta nuestra documentación sobre [configuración de versiones](/es/product/releases/setup/) para más información sobre integraciones, asociación de commits y cómo informar a Sentry al desplegar versiones.
