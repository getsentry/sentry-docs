Este documento abarca problemas comunes que puedes encontrar al usar el SDK de Sentry Kotlin Multiplatform y ofrece pasos para solucionarlos.

Si necesitas ayuda adicional, [pregunta en GitHub](https://github.com/getsentry/sentry-kotlin-multiplatform/issues/new/choose). Los clientes con un plan de pago también pueden ponerse en contacto con [soporte](https://help.sentry.io/).

## &quot;Falta la declaración de API&quot; tras la revisión de la App Store

A partir del 1 de mayo de 2024, Apple exige que todas las apps enviadas a la App Store incluyan un listado de las API relacionadas con la privacidad que utilizan, junto con los motivos por los que las usan. Si recibiste un correo de Apple con el mensaje &quot;ITMS-91053: Missing API declaration&quot;, tu app no cumple con los requisitos. Para solucionarlo, sigue nuestra guía [Apple Privacy Manifest](/es/platforms/kotlin/guides/kotlin-multiplatform/data-management/apple-privacy-manifest).

<div id="tests-not-working">
  ## Las pruebas no funcionan
</div>

Si configuraste el SDK de Sentry Kotlin Multiplatform y las pruebas siguen fallando con el siguiente error: `ld: framework 'Sentry' not found`, sigue estos pasos como solución temporal:

<Alert>
  Este ejemplo muestra cómo solucionar el problema para el target del simulador de iOS, pero el mismo enfoque se puede usar para otros targets.
</Alert>

**1. Descarga el xcframework de Sentry**

* Elige la versión de Sentry Cocoa según la versión indicada en la [tabla de compatibilidad de versiones](/es/platforms/kotlin/guides/kotlin-multiplatform/#cocoa-sdk-version-compatibility-table).
* Descarga `Sentry.xcframework.zip` desde [GitHub](https://github.com/getsentry/sentry-cocoa/releases) y descomprímelo.

**2. Crea un directorio de frameworks e inserta `Sentry.framework`**

Crea un directorio `/Frameworks` en el directorio donde se encuentra `test.kexe` y coloca allí `Sentry.framework`.

* `Sentry.framework` se encuentra dentro de `ios-arm64_x86_64-simulator`.
* `test.kexe` normalmente estará en `build/bin/iosSimulatorArm64/debugTest`.

**3. Agrega opciones del linker**

Modifica `shared/build.gradle.kts` para incluir opciones del linker, como se muestra a continuación:

```kotlin {filename:shared/build.gradle.kts}
  listOf(
      iosX64(),
      iosArm64(),
      iosSimulatorArm64(),
  ).forEach {
      it.binaries.framework {
          baseName = "shared"
      }
      it.compilations.all {
          if (compilationName == "test" && target.platformType == KotlinPlatformType.native) {
              compilerOptions.configure {
                  freeCompilerArgs.add("-linker-options")
                  freeCompilerArgs.add("-F/your/path/Carthage/Build/Sentry.xcframework/ios-arm64_x86_64-simulator/")
              }
          }
      }
  }
```

**4. Ejecuta las pruebas**
