<div id="install">
  ## Instalación
</div>

Sentry recopila datos mediante un SDK dentro del entorno de ejecución de tu aplicación.

Para instalar el SDK de Kotlin Multiplatform, agrega lo siguiente a tu archivo `build.gradle.kts` en el módulo compartido:

```kotlin {filename:shared/build.gradle.kts}
plugins {
    kotlin("multiplatform")
    kotlin("native.cocoapods")
    // .. tus otros plugins
}

kotlin {
  android()
  iosX64()
  iosArm64()
  iosSimulatorArm64()

  sourceSets {
    val commonMain by getting {
      dependencies {
        implementation("io.sentry:sentry-kotlin-multiplatform:{{@inject packages.version('sentry.kotlin.kmp', '0.0.1-alpha.2') }}")
      }
    }

    // Target de Android
    val androidMain by getting {
      dependsOn(commonMain)
    }

    // Targets de Apple:
    val iosMain by getting {
      dependsOn(commonMain)
    }
  }

  cocoapods {
    // resto de la configuración
    summary = "Descripción del Módulo Compartido"
    homepage = "Enlace a la página principal del Módulo Compartido"
    ios.deploymentTarget = "14.1"
    podfile = project.file("../iosApp/Podfile")
    // Asegúrate de usar la versión apropiada según nuestra Tabla de Compatibilidad de Versiones del SDK de Cocoa.
    // TODO: enlace a la tabla en el repositorio de GitHub de sentry-kotlin-multiplatform
    pod("Sentry") {
      // Consulta la tabla de compatibilidad de versiones para obtener la versión correcta
      version = "8.36.0"
      linkOnly = true
      extraOpts += listOf("-compiler-option", "-fmodules")
    }
  }
}
```

<div id="swift-package-manager-spm">
  ### Swift Package Manager (SPM)
</div>

Swift Package Manager (SPM) es una potente herramienta para gestionar dependencias en Swift que permite a los desarrolladores disfrutar de una integración más nativa con Xcode.
Si ya usas SPM o lo prefieres frente a otros gestores de paquetes en plataformas de Apple, esta guía te mostrará cómo instalar el SDK de Kotlin Multiplatform con Swift Package Manager.

#### Instalar

Para instalar el Sentry Kotlin Multiplatform SDK, debes usar CocoaPods y Swift Package Manager simultáneamente, lo que puede parecer poco convencional al principio, especialmente si sueles usar SPM para todas tus dependencias. Sin embargo, puedes lograr una integración fluida usando el [plugin de Kotlin para CocoaPods](https://kotlinlang.org/docs/native-cocoapods-dsl-reference.html#cocoapods-block) específicamente para compilar y crear el framework compartido. Luego puedes seguir gestionando el resto de las dependencias con SPM como de costumbre.

Sigue los pasos de nuestro método de instalación principal [CocoaPods](/es/platforms/kotlin/guides/kotlin-multiplatform/manual-setup/#install) para instalar el Sentry Kotlin Multiplatform SDK.

Una vez que hayas incorporado el framework compartido en tu aplicación mediante CocoaPods, puedes continuar usando SPM para otras dependencias sin interrumpir tu flujo de trabajo.
