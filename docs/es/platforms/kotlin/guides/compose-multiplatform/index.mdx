<Alert level="warning" title="Compatibilidad experimental">
  Si bien el SDK de Kotlin Multiplatform de Sentry funciona en proyectos de Compose Multiplatform (CMP), no tiene compatibilidad nativa ni ha sido completamente probado para CMP. Algunas funciones pueden comportarse de manera inesperada o proporcionar datos incompletos. Úsalo con precaución en entornos de producción y reporta cualquier problema que encuentres.
</Alert>

<div id="overview">
  ## Descripción general
</div>

**Compose Multiplatform** es el framework de UI declarativa de JetBrains que te permite crear interfaces de usuario nativas para escritorio, web y dispositivos móviles usando una única base de código en Kotlin. Amplía Jetpack Compose más allá de Android para crear aplicaciones multiplataforma.

Debes usar el [Sentry Kotlin Multiplatform SDK](/es/platforms/kotlin/guides/kotlin-multiplatform/), que ofrece seguimiento de errores en múltiples targets de Kotlin. Aunque las funciones del SDK en proyectos de Compose Multiplatform usan las mismas API que en configuraciones KMP puras, es importante tener en cuenta algunas diferencias:

* **Compatibilidad experimental**: El SDK de KMP funciona en proyectos de CMP y actualmente permite capturar bloqueos y errores, además de otras funciones básicas disponibles a través del Sentry KMP SDK.
* **No nativo de CMP**: No hay optimizaciones ni pruebas específicas para CMP.
* **Casos límite sin probar**: Algunos comportamientos específicos de cada plataforma pueden estar incompletos.

<div id="debug-symbols">
  ## Símbolos de depuración
</div>

Los stack traces de errores en tu app pueden verse ilegibles u ofuscados, lo que dificulta diagnosticar problemas. Para que los stack traces sean claros y legibles en Sentry, debes subir los símbolos de depuración de tu app.

<div id="android">
  ### Android
</div>

En aplicaciones Android, debes cargar manualmente los archivos de mapeo de ProGuard con `sentry-cli`. Consulta la [guía para cargar el mapeo de ProGuard](/es/cli/dif/#proguard-mapping-upload) para obtener instrucciones detalladas.

<div id="ios">
  ### iOS
</div>

Para las aplicaciones iOS, sigue la [guía para cargar símbolos de depuración](/es/platforms/apple/guides/ios/dsym) y configurar la subida de símbolos de depuración.
