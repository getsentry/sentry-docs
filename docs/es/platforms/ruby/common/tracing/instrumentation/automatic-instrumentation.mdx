Muchas integraciones para frameworks populares capturan transacciones automáticamente. Si tienes configurado alguno de los siguientes frameworks para el reporte de errores de Sentry y usas la gema correspondiente de Sentry, empezarás a ver trazas de inmediato:

- Ruby on Rails (`sentry-rails`)
- Sidekiq (`sentry-sidekiq`)
- Delayed Job (`sentry-delayed_job`)
- Resque (`sentry-resque`)

Se instrumentan spans para las siguientes operaciones dentro de una transacción:

- Consultas de base de datos en Rails que usan `ActiveRecord`
  - incluye sistemas de bases de datos comunes como Postgres y MySQL
- Solicitudes HTTP salientes realizadas con `Net::HTTP`
- Operaciones de Redis

Los spans solo se crean dentro de una transacción existente. Si no estás usando ninguno de los frameworks compatibles, deberás <PlatformLink to="/tracing/instrumentation/custom-instrumentation/">crear transacciones manualmente</PlatformLink>.
