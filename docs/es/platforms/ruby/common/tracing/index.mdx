Con [tracing](/es/product/insights/overview/), Sentry supervisa el rendimiento de tu software, midiendo métricas como el throughput y la latencia, y mostrando el impacto de los errores en múltiples sistemas. Sentry captura trazas distribuidas compuestas por transacciones y spans, que miden servicios individuales y operaciones individuales dentro de esos servicios. Obtén más información sobre nuestro modelo en [trazado distribuido](/es/product/sentry-basics/tracing/distributed-tracing/).

<Alert>
  Si vas a adoptar tracing en un entorno de alto throughput, te recomendamos probar antes del despliegue para asegurarte de que el rendimiento de tu servicio cumpla con lo esperado.
</Alert>

<div id="configure">
  ## Configurar
</div>

Primero, habilita el tracing y configura la tasa de muestreo para las transacciones. Establece la tasa de muestreo para tus transacciones de una de estas maneras:

* Estableciendo una tasa de muestreo uniforme para todas las transacciones usando la opción <PlatformIdentifier name="traces-sample-rate" /> en la configuración del SDK a un número entre `0` y `1`. (Por ejemplo, para enviar el 20% de las transacciones, establece <PlatformIdentifier name="traces-sample-rate" /> en `0.2`).
* Controlando la tasa de muestreo según la propia transacción y el contexto en el que se captura, proporcionando una función a la opción de configuración <PlatformIdentifier name="traces-sampler" />.

Las dos opciones están pensadas para ser mutuamente excluyentes. Si configuras ambas, <PlatformIdentifier name="traces-sampler" /> tendrá prioridad.

Obtén más información sobre las <PlatformLink to="/configuration/options/#tracing-options">opciones</PlatformLink> de tracing, cómo usar la función <PlatformLink to="/configuration/sampling/#setting-a-sampling-function">traces&#95;sampler</PlatformLink> o cómo <PlatformLink to="/configuration/sampling/#sampling-transaction-events">muestrear transacciones</PlatformLink>.

<div id="verify">
  ## Verificar
</div>

Prueba el rastreo iniciando y finalizando una transacción; *debes* hacerlo para que las transacciones puedan enviarse a Sentry. Aprende cómo en nuestra sección de <PlatformLink to="/tracing/instrumentation/custom-instrumentation/">Instrumentación personalizada</PlatformLink>.

Mientras realizas pruebas, establece <PlatformIdentifier name="traces-sample-rate" /> en `1.0`, ya que eso garantiza que cada transacción se enviará a Sentry. Una vez que termines de probar, quizá quieras establecer un valor más bajo para <PlatformIdentifier name="traces-sample-rate" /> o pasar a usar <PlatformIdentifier name="traces-sampler" /> para muestrear y filtrar tus transacciones de forma selectiva según datos contextuales.
