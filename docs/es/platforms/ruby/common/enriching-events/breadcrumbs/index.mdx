Sentry utiliza *breadcrumbs* para crear un rastro de eventos que ocurrieron antes de un problema. Estos eventos son muy similares a los registros tradicionales, pero pueden capturar datos estructurados más completos.

Esta página ofrece una descripción general del registro manual de breadcrumbs y su personalización. Obtén más información sobre lo que se muestra en la página **Detalles del problema** y cómo puedes filtrar los breadcrumbs para resolver incidencias más rápido en [Uso de breadcrumbs](/es/product/error-monitoring/breadcrumbs).

<Alert title="Aprende sobre el uso del SDK">
  Los desarrolladores que quieran modificar la interfaz de breadcrumbs pueden obtener más información en nuestra [documentación para desarrolladores sobre la interfaz de Breadcrumbs](https://develop.sentry.dev/sdk/data-model/event-payloads/breadcrumbs/).
</Alert>

<div id="manual-breadcrumbs">
  ## Breadcrumbs manuales
</div>

Puedes agregar breadcrumbs manualmente cuando ocurra algo interesante. Por ejemplo, podrías registrar manualmente un breadcrumb si el usuario se autentica o si se produce otro cambio de estado.

Registrar un breadcrumb manualmente:

<PlatformContent includePath="enriching-events/breadcrumbs/breadcrumbs-example" />

Las claves disponibles para los breadcrumbs son `type`, `category`, `message`, `level`, `timestamp` (que, de forma predeterminada, usa la hora del reloj del sistema) y `data`, que es donde puedes incluir cualquier información adicional que quieras que el breadcrumb contenga. Usar claves distintas de estas seis no provocará un error, pero hará que los datos se descarten cuando Sentry procese el evento.

  ## Personalizar breadcrumbs
</div>

Este hook recibe un breadcrumb ya armado y <PlatformLink to="/configuration/filtering/#using-hints">un objeto `hint`</PlatformLink> que contiene metadatos adicionales. La función puede modificar el breadcrumb o decidir descartarlo por completo devolviendo `nil`:
