Los usuarios se definen por algunas piezas de información clave que conforman una identidad única en Sentry. Cada una es opcional, pero **debe** haber al menos una para que el SDK de Sentry capture al usuario:

<DefinitionList>
  ### `id`

  Tu identificador interno del usuario.

  ### `username`

  El nombre de usuario. Suele usarse como una etiqueta más útil que el id interno.

  ### `email`

  Una alternativa o complemento al nombre de usuario. Sentry reconoce las direcciones de correo electrónico y puede mostrar elementos como Gravatar y habilitar funciones de mensajería.

  ### `ip_address`

  La dirección IP del usuario. Si el usuario no está autenticado, Sentry usa la dirección IP como identificador único del usuario.
  El SDK intentará extraer la dirección IP de los datos de la solicitud HTTP en solicitudes entrantes (campo `request.env.REMOTE_ADDR` en JSON), si está disponible. Para ello, <PlatformIdentifier name="send-default-pii" /> debe estar establecido en `true` en las opciones del SDK.

  Si el `ip_address` del usuario está configurado como `"{{auto}}"`, Sentry inferirá la dirección IP a partir de la conexión entre tu aplicación y el servidor de Sentry. Si se omite el campo, el valor predeterminado es `nil`.

  Para dejar de almacenar direcciones IP de usuarios en los datos de tus eventos, ve a la configuración de tu proyecto, haz clic en &quot;Security &amp; Privacy&quot; y habilita &quot;Prevent Storing of IP Addresses&quot;, o usa la función de depuración [server-side data](/es/security-legal-pii/scrubbing/) de Sentry para eliminar `$user.ip_address`. Agregar una regla de este tipo finalmente prevalece sobre cualquier otra lógica.
</DefinitionList>

Además, puedes proporcionar pares clave/valor arbitrarios más allá de los nombres reservados, y el SDK de Sentry los almacenará junto con el usuario.
