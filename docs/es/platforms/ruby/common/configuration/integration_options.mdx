<div id="sentry-rails">
  ## sentry-rails
</div>

`rails.report_rescued_exceptions`

Rails captura las excepciones en los middleware `ActionDispatch::ShowExceptions` o `ActionDispatch::DebugExceptions`, según el entorno. De forma predeterminada, Sentry reportará las excepciones incluso cuando estas sean rescatadas por esos middleware.

Puedes desactivar esto estableciéndolo en `false`:

```ruby
config.rails.report_rescued_exceptions
```

`rails.skippable_job_adapters`

Si no quieres informar eventos de un adaptador de ActiveJob en particular, o si ya cuentas con una mejor configuración para ello, puedes usar esta opción para omitir el envío de eventos de ActiveJob en ese adaptador.

```ruby
config.rails.skippable_job_adapters = ["ActiveJob::QueueAdapters::MyAdapter"]
```

`rails.tracing_subscribers`

Los suscriptores de tracing son responsables de capturar spans de trazado a partir de la instrumentación de Rails. Los suscriptores predeterminados son:

* `Sentry::Rails::Tracing::ActionViewSubscriber`
* `Sentry::Rails::Tracing::ActiveRecordSubscriber`
* `Sentry::Rails::Tracing::ActiveStorageSubscriber`

```ruby
# puedes añadir un nuevo suscriptor
config.rails.tracing_subscribers << MySubscriber
# o reemplazar el conjunto por completo
config.rails.tracing_subscribers = [MySubscriber]
```


## sentry-delayed&#95;job

`delayed_job.report_after_job_retries`

`sentry-delayed_job` captura excepciones cuando ocurren durante la ejecución de un job. Puedes hacer que espere hasta que se agoten los reintentos. El valor predeterminado es `false`.

```
config.delayed_job.report_after_job_retries = true
```


<div id="sentry-sidekiq">
  ## sentry-sidekiq

`sidekiq.report_after_job_retries`

`sentry-sidekiq` captura excepciones cuando ocurren durante la ejecución de un trabajo. Puedes hacer que espere hasta que se agoten los reintentos. El valor predeterminado es `false`.

```
config.sidekiq.report_after_job_retries = true
```
