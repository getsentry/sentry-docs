El nombre de la transacción actual se usa para agrupar transacciones en nuestro
producto [Insights](/es/product/insights/), además de anotar los eventos de error
con su punto de fallo.

El nombre de la transacción puede hacer referencia a la ruta actual de la aplicación web o a la
tarea que se está ejecutando. Por ejemplo:

* `GET /api/{version}/users/`
* `UserListView`
* `myapp.tasks.renew_all_subscriptions`

Idealmente, el nombre de la transacción no contiene valores variables como IDs
de usuario, y mantiene una baja cardinalidad, a la vez que identifica de forma única una parte del
código que te interesa.

Muchas de nuestras integraciones con frameworks ya establecen un nombre de transacción, aunque puedes definir uno tú mismo.
