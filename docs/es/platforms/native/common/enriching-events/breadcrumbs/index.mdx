Sentry usa *migas de pan* para crear un rastro de eventos que ocurrieron antes de un problema. Estos eventos son muy similares a los logs tradicionales, pero pueden registrar datos estructurados más ricos.

Esta página ofrece una descripción general del registro manual de migas de pan y su personalización. Obtén más información sobre lo que se muestra en la página **Detalles del problema** y cómo filtrar las migas de pan para resolver problemas rápidamente en [Uso de migas de pan](/es/product/error-monitoring/breadcrumbs).

<Alert title="Conoce el uso del SDK">
  Los desarrolladores que quieran modificar la interfaz de migas de pan pueden obtener más información en nuestra [documentación para desarrolladores sobre la interfaz de Breadcrumbs](https://develop.sentry.dev/sdk/data-model/event-payloads/breadcrumbs/).
</Alert>

<div id="manual-breadcrumbs">
  ## Migas de pan manuales
</div>

Puedes añadir migas de pan de forma manual cuando ocurra algo relevante. Por ejemplo, podrías registrar manualmente una miga de pan si el usuario se autentica o se produce otro cambio de estado.

Registrar manualmente una miga de pan:

<PlatformContent includePath="enriching-events/breadcrumbs/breadcrumbs-example" />

Las claves disponibles para las migas de pan son `type`, `category`, `message`, `level`, `timestamp` y `data`, que es donde puedes incluir cualquier información adicional que quieras que contenga la miga de pan. Usar claves distintas de estas seis no provocará un error, pero hará que los datos se descarten cuando Sentry procese el evento.
