Los usuarios se componen de algunas piezas de información clave que definen una identidad única en Sentry. Cada una es opcional, pero una **debe** estar presente para que el SDK de Sentry capture al usuario:

<DefinitionList>
  ### `id`

  Tu identificador interno del usuario.

  ### `username`

  El nombre de usuario. Suele usarse como una mejor etiqueta que el id interno.

  ### `email`

  Una alternativa o un complemento al nombre de usuario. Sentry reconoce direcciones de correo electrónico y puede mostrar elementos como Gravatars y habilitar funciones de mensajería.

  ### `ip_address`

  La dirección IP del usuario. Si el usuario no está autenticado, Sentry usa la dirección IP como identificador único.
  Los SDK del lado del servidor que instrumentan solicitudes entrantes intentarán obtener la dirección IP de los datos de la solicitud HTTP (campo `request.env.REMOTE_ADDR` en JSON), si está disponible. Eso podría requerir que <PlatformIdentifier name="send-default-pii" /> esté configurado en `true` en las opciones del SDK.

  Si la `ip_address` del usuario se establece en `"{{auto}}"`, Sentry inferirá la dirección IP a partir de la conexión entre tu aplicación y el servidor de Sentry.

  Si se omite el campo, el valor predeterminado es `null`. Sin embargo, por motivos de compatibilidad retroactiva, ciertas plataformas (en particular JavaScript) tienen un valor predeterminado distinto para `"{{auto}}"`. Los SDK y otros clientes no deben depender de este comportamiento y deben establecer direcciones IP o `"{{auto}}"` explícitamente.

  Para no almacenar las direcciones IP de los usuarios en los datos de tus eventos, ve a la configuración de tu proyecto, haz clic en &quot;Security &amp; Privacy&quot; y habilita &quot;Prevent Storing of IP Addresses&quot;, o usa el depurado de [datos del lado del servidor](/es/security-legal-pii/scrubbing/) de Sentry para eliminar `$user.ip_address`. Agregar esta regla prevalecerá sobre cualquier otra lógica.
</DefinitionList>

Además, puedes proporcionar pares clave/valor arbitrarios más allá de los nombres reservados, y el SDK de Sentry los almacenará junto con el usuario.
