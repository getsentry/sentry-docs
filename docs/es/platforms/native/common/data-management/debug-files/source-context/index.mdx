Si Sentry tiene acceso al código fuente de tu aplicación, puede mostrar fragmentos de código (*source context*) alrededor de la ubicación de los marcos de la pila, lo que ayuda a identificar rápidamente el código problemático.

Por ejemplo, aquí tienes un rastreo de pila de una aplicación .NET. El contexto del código fuente contiene
el código fuente original en C# de la aplicación, con la ubicación del marco resaltada.

![Captura de pantalla de un rastreo de pila que muestra el contexto del código fuente.](./img/source-context-screenshot.png)

<div id="source-bundles">
  ### Paquetes de código fuente
</div>

Algunos SDK, como el SDK de Python, pueden resolver este contexto de origen automáticamente,
porque tienen acceso a código fuente no ofuscado en tiempo de ejecución.
En cambio, para obtener el contexto de origen en aplicaciones compiladas, es necesario subir el código fuente
junto con los archivos de información de depuración.
La forma recomendada de hacerlo es usando `sentry-cli`.
Consulta [Crear paquetes de código fuente](/es/cli/dif/#creating-source-bundles) para más información.

Después de subirlos, puedes revisar y gestionar los paquetes de código fuente igual que
cualquier otro archivo de información de depuración. Consulta [Gestionar archivos de información de depuración](../#managing-debug-information-files).
Los paquetes de código fuente se etiquetarán con `sources` y tendrán el mismo nombre de archivo que el
archivo de información de depuración correspondiente del que se crearon.
