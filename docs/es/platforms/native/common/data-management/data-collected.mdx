---
title: Datos recopilados
description: "Consulta qué datos recopila el SDK de Sentry."
sidebar_order: 1
---

Sentry se toma muy en serio la privacidad de los datos y cuenta con configuraciones predeterminadas que priorizan la seguridad de la información, especialmente en lo referente a datos de identificación personal (PII). Al agregar el SDK de Sentry a tu aplicación, permites que recopile datos y los envíe a Sentry durante el tiempo de ejecución de tu aplicación.

Los tipos de categorías y la cantidad de datos recopilados varían según las integraciones que tengas habilitadas en el SDK de Sentry. A continuación, encontrarás una lista de categorías de datos que recopila el SDK Sentry Native:

<div id="device-information">
  ## Información del dispositivo
</div>

El SDK de Sentry recopila información sobre el dispositivo, como el nombre, la versión y la compilación de tu sistema operativo o de tu distribución de Linux. Esta información se envía a Sentry de forma predeterminada.

<div id="thread-stack-information">
  ## Información de la pila de hilos
</div>

Cuando se produce un bloqueo, la pila de cada hilo se recopila y se envía a Sentry como parte de la instantánea Minidump para los backends `crashpad` y `breakpad`. Esta información se envía a Sentry de forma predeterminada, pero se descarta después de procesar el evento en el backend.

Estos archivos no se almacenan de forma predeterminada, pero puedes [habilitar el almacenamiento de Minidumps](/es/platforms/native/guides/minidumps/enriching-events/attachments/#store-minidumps-as-attachments) en la organización de Sentry o en la configuración del proyecto.

El backend `inproc` realiza el recorrido de la pila únicamente en el cliente y, por lo tanto, solo envía los rastros de pila resultantes.
