Un lanzamiento es una versión de tu código que se despliega en un <PlatformLink to="/configuration/environments/">entorno</PlatformLink>. Cuando proporcionas a Sentry información sobre tus lanzamientos, puedes:

* Identificar problemas y regresiones introducidos en un nuevo lanzamiento
* Predecir qué commit causó un problema y quién probablemente sea responsable
* Resolver problemas incluyendo el número del issue en tu mensaje de commit
* Recibir notificaciones por correo electrónico cuando tu código se despliegue

<div id="setting-a-release">
  ## Configurar una release
</div>

<PlatformContent includePath="set-release" notateUnsupported />

Cómo hagas que el nombre de la release (o versión) esté disponible para tu código depende de ti. Por ejemplo, puedes usar una variable de entorno que se defina durante el proceso de compilación o en el arranque inicial.

Configurar el nombre de la release etiqueta cada evento con ese nombre. Recomendamos informar a Sentry sobre una nueva release antes de enviar eventos con ese nombre, ya que esto habilitará algunas funciones adicionales. Obtén más información en nuestra documentación sobre [Releases](/es/product/releases/).

Si no informas a Sentry sobre una nueva release, Sentry creará automáticamente una entidad de release en el sistema la primera vez que vea un evento con ese ID de release.

Tras configurar tu SDK, puedes instalar una integración de repositorio o proporcionar manualmente a Sentry tus propios metadatos de commits. Lee nuestra documentación sobre [configurar releases](/es/product/releases/setup/) para más información sobre integraciones, asociación de commits y cómo informar a Sentry al desplegar releases.

<div id="release-health">
  ## Estado de la versión
</div>

Supervisa la [salud de las versiones](/es/product/releases/health/) observando la adopción por parte de los usuarios, el uso de la aplicación, el porcentaje de [crashes](/es/product/releases/health/#crashes) y los [datos de sesión](/es/product/releases/health/#sessions). El estado de la versión ofrece información sobre el impacto de los crashes y errores en la experiencia del usuario, y revela tendencias con cada nuevo incidente a través de los gráficos y filtros de [Detalles de la versión](/es/product/releases/release-details/).

Para supervisar el estado de la versión, el SDK envía datos de sesión.

<div id="sessions">
  ### Sesiones
</div>

Una sesión representa la interacción entre el usuario y la aplicación. Las sesiones incluyen una marca de tiempo, un estado (si la sesión fue correcta o si falló) y siempre están vinculadas a una versión. La mayoría de los SDK de Sentry pueden gestionar las sesiones automáticamente.
