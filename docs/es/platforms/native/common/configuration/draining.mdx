El comportamiento predeterminado de la mayoría de los SDK es enviar eventos por la red de forma asíncrona en segundo plano. Esto significa que algunos eventos pueden perderse si la aplicación se cierra inesperadamente. Los SDK ofrecen mecanismos para manejar esta situación.
