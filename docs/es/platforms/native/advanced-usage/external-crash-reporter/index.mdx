---
title: Informador externo de fallos
description: "Cómo configurar un informador externo de fallos con el SDK nativo."
sidebar_order: 2100
---

El SDK nativo se puede configurar para funcionar con un informador externo de fallos, que
puede usarse para mostrar al usuario información sobre el fallo, recopilar sus comentarios o
realizar otras acciones cuando se produce un fallo.

El informador externo de fallos es un ejecutable definido por el usuario, distinto del
informador de fallos del sistema, que el SDK nativo inicia cuando se produce un fallo. Recibe
la ruta del informe de fallos como único argumento de línea de comandos y
es responsable de enviar dicho informe a Sentry.

[Sentry Desktop Crash Reporter](https://github.com/getsentry/sentry-desktop-crash-reporter)
es una implementación de referencia lista para usar de un informador de fallos. Es una aplicación
de escritorio multiplataforma creada con [.NET](https://dot.net) y [Uno Platform](https://platform.uno/).

![Sentry Desktop Crash Reporter](./img/sentry-desktop-crash-reporter.png)

Para configurar el SDK nativo para usar un informador externo de fallos, establece la
opción `external_crash_reporter_path` en la ruta del ejecutable del informador externo de fallos
al inicializar el SDK:

```c
sentry_options_t *options = sentry_options_new();
sentry_options_set_external_crash_reporter_path(options, "/path/to/external_crash_reporter");
/* ... */
sentry_init(options);
```
