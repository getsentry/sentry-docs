El SDK de Sentry se integra en tu entorno de ejecución y envía automáticamente errores, excepciones no controladas y rechazos no gestionados, así como otros tipos de errores según la plataforma.

<Alert>

Términos clave:

- Un _evento_ es una instancia de envío de datos a Sentry. Generalmente, estos datos son un error o una excepción.
- Un _issue_ es un agrupamiento de eventos similares.
- El envío de un evento se denomina _captura_.
  Cuando se captura un evento, se envía a Sentry.

</Alert>

La forma más común de captura es la de errores.

Mientras capturas un evento, también puedes registrar las breadcrumbs que llevaron a ese evento. Las breadcrumbs son diferentes de los eventos: no crearán un evento en Sentry, sino que se almacenarán en búfer hasta que se envíe el siguiente evento. Obtén más información sobre las breadcrumbs en nuestra <PlatformLink to="/enriching-events/breadcrumbs/">documentación de Breadcrumbs</PlatformLink>.

  ## Captura de mensajes
</div>

Otra operación común es capturar un mensaje sin más. Un mensaje es información textual que debe enviarse a Sentry. El SDK no captura mensajes automáticamente, pero puedes capturarlos manualmente.

Los mensajes aparecen como incidencias en tu lista de incidencias, usando el propio mensaje como nombre de la incidencia.

<PlatformContent includePath="capture-message" />

El SDK puede inicializarse con la opción `attach_stacktrace` establecida en `true` (valor predeterminado: `false`) para adjuntar una traza de pila a todos los eventos creados a partir de llamadas a `capture_message`.
