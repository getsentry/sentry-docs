<div id="compile-and-create-debug-information-files">
  ## Compilar y crear archivos de información de depuración
</div>

<div id="linux">
  ### Linux
</div>

Activa la información de depuración completa para tu compilación de release:

```toml {filename:Cargo.toml}
[profile.release]
debug = true
```

Compila el binario de la versión, extrae la información de depuración y quítasela al binario:

```bash
cargo build --release
objcopy --only-keep-debug target/release/app{,.d}
objcopy --strip-debug --strip-unneeded target/release/app
objcopy --add-gnu-debuglink target/release/app{.d,}
```

<div id="macos">
  ### macOS
</div>

Usa un perfil como el siguiente para tus compilaciones de lanzamiento:

```toml {filename:Cargo.toml}
[profile.release]
debug = true
split-debuginfo = "packed"
strip = true
```

Compila el binario de la versión; la información de depuración estará disponible en archivos separados y se eliminará del binario automáticamente:

```bash
cargo build --release
```

<div id="upload-debug-information-and-source-files">
  ## Sube información de depuración y archivos fuente
</div>

Usa Sentry CLI para subir información de depuración y archivos fuente:

```bash
sentry-cli debug-files upload --include-sources .
```

<div id="troubleshooting">
  ## Solución de problemas
</div>

Si no deseas quitar los símbolos del binario, deberías desactivar la integración `debug-images` del SDK; de lo contrario, te toparás con el problema documentado [aquí](https://github.com/getsentry/sentry-rust/issues/470#issuecomment-1472136215), que podría provocar que aparezcan fotogramas duplicados en tus issues de Sentry.
