Cuando incorporas Sentry a tu aplicación, obtienes mucha información valiosa sobre errores y rendimiento. Y tener mucha información está bien, siempre que sea la adecuada y en una cantidad razonable.

Los SDK de Sentry ofrecen varias opciones de configuración para ayudarte a filtrar eventos.

También ofrecemos [filtros de entrada (Inbound Filters)](/es/concepts/data-management/filtering/) para filtrar eventos en sentry.io. Aun así, recomendamos filtrar en el cliente, porque evita el costo de enviar eventos que en realidad no necesitas. Obtén más información sobre los [campos disponibles en un evento](https://develop.sentry.dev/sdk/data-model/event-payloads/).

<div id="filtering-error-events">
  ## Filtrado de eventos de error
</div>

Configura tu SDK para filtrar eventos de error usando la devolución de llamada <PlatformIdentifier name="before-send" /> y configurando, habilitando o deshabilitando integraciones.

### Uso de <PlatformIdentifier name="before-send" />

Todos los SDK de Sentry admiten el callback <PlatformIdentifier name="before-send" />. Como se ejecuta justo antes de enviar el evento al servidor, es tu última oportunidad para decidir no enviar datos o editarlos. <PlatformIdentifier name="before-send" /> recibe el objeto del evento como parámetro, que puedes usar para modificar los datos del evento o descartarlo por completo devolviendo `None`, según tu lógica personalizada y los datos disponibles en el evento.
