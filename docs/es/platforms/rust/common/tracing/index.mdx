Con [tracing](/es/product/insights/), Sentry supervisa el rendimiento de tu software, midiendo métricas como el throughput y la latencia, y mostrando el impacto de los errores en múltiples sistemas. Sentry captura trazas distribuidas compuestas por transacciones y spans, que miden servicios individuales y las operaciones dentro de esos servicios. Obtén más información sobre nuestro modelo en [trazabilidad distribuida](/es/product/sentry-basics/tracing/distributed-tracing/).

<Alert>
  Si estás adoptando tracing en un entorno de alto throughput, te recomendamos realizar pruebas antes del despliegue para asegurarte de que las características de rendimiento de tu servicio se mantengan dentro de lo esperado.
</Alert>

<div id="configure">
  ## Configurar
</div>

Primero, habilita el rastreo y configura la tasa de muestreo para las transacciones. Establece la tasa de muestreo para tus transacciones de cualquiera de estas formas:

* Estableciendo una tasa de muestreo uniforme para todas las transacciones usando la opción <PlatformIdentifier name="traces-sample-rate" /> en la configuración de tu SDK con un número entre `0` y `1`. (Por ejemplo, para enviar el 20% de las transacciones, establece <PlatformIdentifier name="traces-sample-rate" /> en `0.2`.)
* Controlando la tasa de muestreo según la propia transacción y el contexto en el que se captura, proporcionando una función a la opción de configuración <PlatformIdentifier name="traces-sampler" />.

Estas dos opciones son mutuamente excluyentes. Si configuras ambas, <PlatformIdentifier name="traces-sampler" /> tendrá prioridad.

Obtén más información sobre las <PlatformLink to="/configuration/options/#tracing-options">opciones</PlatformLink> de rastreo o sobre cómo <PlatformLink to="/configuration/sampling/#sampling-transaction-events">muestrear transacciones</PlatformLink>.

<div id="verify">
  ## Verificar
</div>

Verifica que el tracing esté funcionando correctamente usando nuestra <PlatformLink to="/tracing/instrumentation/automatic-instrumentation/">instrumentación automática</PlatformLink> o iniciando y finalizando una transacción con <PlatformLink to="/tracing/instrumentation/custom-instrumentation/">instrumentación personalizada</PlatformLink>.

Mientras realizas pruebas, establece <PlatformIdentifier name="traces-sample-rate" /> en `1.0`, ya que eso garantiza que cada transacción se envíe a Sentry. Una vez que termines las pruebas, quizá quieras configurar un valor más bajo de <PlatformIdentifier name="traces-sample-rate" />, o cambiar a <PlatformIdentifier name="traces-sampler" /> para muestrear y filtrar selectivamente tus transacciones según datos contextuales.
