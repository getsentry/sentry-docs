Esta integración rastrea los bloqueos de la app. Esta función está disponible en iOS, tvOS y macOS.

Intentar usar una app que no responde es extremadamente frustrante para los usuarios. Hay muchas razones por las que una app puede dejar de responder, como código de ejecución prolongada, errores de bucles infinitos, etc. Con el seguimiento de bloqueos de la app puedes detectarlos y corregirlos.

Cada vez que se detecta un bloqueo de la app, Sentry crea un evento de error. El seguimiento de bloqueos de la app está habilitado de forma predeterminada. Para deshabilitarlo, establece la opción `enableAppHangTracking` en `false`:

```javascript
import * as Sentry from "@sentry/react-native";

Sentry.init({
  dsn: "DSN",
  enableAppHangTracking: false,
});
```

La integración de detección de bloqueos de la app tiene un tiempo de espera predeterminado de dos (2) segundos. Esto representa el tiempo mínimo durante el cual una app puede estar sin responder antes de considerarse bloqueada. Puedes cambiar el tiempo de espera configurando la opción `appHangTimeoutInterval`:

```javascript
import * as Sentry from "@sentry/react-native";

Sentry.init({
  dsn: "DSN",
  appHangTimeoutInterval: 1,
});
```

Para obtener más información sobre cómo se detectan los bloqueos de la app, consulta la [documentación](/es/platforms/apple/configuration/app-hangs/) de `sentry-cocoa`.
