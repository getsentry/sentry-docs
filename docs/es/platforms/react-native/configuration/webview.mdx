---

Para supervisar errores y el rendimiento en webviews de aplicaciones React Native, debes configurar los SDK de Sentry tanto en la aplicación móvil como en los proyectos web que se ejecutan dentro de esos webviews.

<div id="application-monitoring">
  ## Monitorización de la aplicación
</div>

El SDK de Sentry para React Native no interactúa con las vistas web de la aplicación móvil. Para monitorizar los errores y el rendimiento en la parte de React Native de la aplicación, sigue las [instrucciones de configuración del SDK de Sentry para React Native](/es/platforms/react-native/).

<div id="webview-monitoring">
  ## Monitoreo de WebView
</div>

Para monitorear errores y rendimiento en vistas web, elige el SDK de Sentry para JavaScript adecuado según el framework utilizado en el proyecto web.

* [Sentry React Native SDK](/es/platforms/react-native) (si el proyecto web usa React Native Web o Expo)
* [Sentry React SDK](/es/platforms/javascript/guides/react/) (si el proyecto web usa React)
* [Otros SDK de Sentry para JavaScript](/es/platforms/javascript)

<div id="notes">
  ## Notas
</div>

* El SDK de JavaScript debe inicializarse cuando se crea la vista web y se cerrará cuando se cierre la vista web. Para monitorear todas las vistas web de la aplicación, inicializa el SDK de JavaScript cada vez que se cree una vista web.

* Si se accede al proyecto web de forma remota y no es exclusivo de la aplicación móvil (por ejemplo, si el proyecto también funciona como una aplicación web independiente), recomendamos usar proyectos de Sentry separados para la aplicación móvil y para el proyecto web.
