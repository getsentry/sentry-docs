Un lanzamiento es una versión de tu código que se despliega en un <PlatformLink to="/configuration/environments/">entorno</PlatformLink>. Cuando proporcionas a Sentry información sobre tus lanzamientos, puedes:

* Identificar problemas y regresiones introducidos en un nuevo lanzamiento
* Predecir qué commit causó un problema y quién probablemente es responsable
* Resolver problemas incluyendo el número del issue en el mensaje del commit
* Recibir notificaciones por correo electrónico cuando se despliegue tu código

Además, los lanzamientos se usan para aplicar [source maps](/es/platforms/javascript/sourcemaps/) a JavaScript minificado para ver el código fuente original sin transformaciones.

<div id="setting-a-release">
  ## Configurar una versión (release)
</div>

<PlatformContent includePath="set-release" notateUnsupported />

Cómo hacer que el nombre de la versión (o release) esté disponible para tu código depende de ti. Por ejemplo, puedes usar una variable de entorno definida durante el proceso de compilación o en el arranque inicial.

Al configurar el nombre de la versión, cada evento se etiqueta con ese nombre. Te recomendamos informar a Sentry sobre una nueva versión antes de enviar eventos con ese nombre, ya que así se habilitan algunas funciones adicionales. Obtén más información en nuestra documentación de [Releases](/es/product/releases/).

Si no informas a Sentry sobre una nueva versión, Sentry creará automáticamente una entidad de versión en el sistema la primera vez que detecte un evento con ese ID de versión.

Después de configurar tu SDK, puedes instalar una integración de repositorio o proporcionar manualmente a Sentry tus propios metadatos de commits. Consulta nuestra documentación sobre [configuración de versiones](/es/product/releases/setup/) para obtener más información sobre integraciones, asociación de commits y cómo informar a Sentry al realizar un deploy de versiones.

<div id="release-health">
  ## Estado de la versión
</div>

Supervisa el [estado de las versiones](/es/product/releases/health/) observando la adopción por parte de los usuarios, el uso de la aplicación, el porcentaje de [fallos](/es/product/releases/health/#crashes) y los [datos de sesión](/es/product/releases/health/#sessions). El estado de la versión ofrece información sobre el impacto de los fallos y errores en la experiencia del usuario y revela tendencias con cada nueva incidencia a través de los gráficos y filtros de [Detalles de la versión](/es/product/releases/release-details/).

Para supervisar el estado de la versión, el SDK envía datos de sesión.

<div id="sessions">
  ### Sesiones
</div>

Una sesión representa la interacción entre el usuario y la aplicación. Las sesiones incluyen una marca de tiempo, un estado (si la sesión fue correcta o si se bloqueó) y siempre están vinculadas a una versión. La mayoría de los SDK de Sentry pueden gestionar las sesiones automáticamente.
