Cuando incorporas Sentry a tu aplicación, obtienes mucha información valiosa sobre errores y rendimiento. Y contar con mucha información es bueno, siempre que sea la adecuada y en un volumen razonable.

Los SDK de Sentry ofrecen varias opciones de configuración para ayudarte a filtrar eventos.

También ofrecemos [filtros de entrada](/es/concepts/data-management/filtering/) para filtrar eventos en sentry.io. Sin embargo, recomendamos filtrar en el cliente, porque evita la sobrecarga de enviar eventos que en realidad no necesitas. Obtén más información sobre los [campos disponibles en un evento](https://develop.sentry.dev/sdk/data-model/event-payloads/).

<div id="filtering-error-events">
  ## Filtrado de eventos de error
</div>

Configura tu SDK para filtrar eventos de error mediante el callback <PlatformIdentifier name="before-send" /> y la configuración, activación o desactivación de integraciones.

### Uso de <PlatformIdentifier name="before-send" />

Todos los SDK de Sentry admiten el método de callback <PlatformIdentifier name="before-send" />. Como se invoca justo antes de que el evento se envíe al servidor, esta es tu última oportunidad para decidir no enviar los datos o editarlos. <PlatformIdentifier name="before-send" /> recibe el objeto del evento como parámetro, que puedes usar para modificar los datos del evento o descartarlo por completo devolviendo `null`, según tu lógica personalizada y los datos disponibles en el evento.

Ten en cuenta también que los breadcrumbs se pueden filtrar, como se explica en [nuestra documentación de Breadcrumbs](/es/product/error-monitoring/breadcrumbs/).

<div id="event-hints">
  #### Indicadores de eventos
</div>

La devolución de llamada <PlatformIdentifier name="before-send" /> recibe tanto el `event` como un segundo argumento, `hint`, que contiene uno o más indicadores.

Normalmente, un `hint` incluye la excepción original para poder extraer datos adicionales o afectar la agrupación. En este ejemplo, la huella digital se fuerza a un valor común si se ha capturado una excepción de cierto tipo:

Cuando el SDK crea un evento o un breadcrumb para su envío, este suele generarse a partir de algún tipo de objeto de origen. Por ejemplo, un evento de error suele crearse a partir de un registro de log o de una instancia de excepción. Para una mejor personalización, los SDK envían estos objetos a ciertas devoluciones de llamada (<PlatformIdentifier name="before-send" />, <PlatformIdentifier name="before-breadcrumb" />) o al sistema de procesadores de eventos del SDK.

<div id="using-hints">
  ### Uso de sugerencias
</div>

Las sugerencias están disponibles en dos lugares:

1. <PlatformIdentifier name="before-send" /> / <PlatformIdentifier name="before-breadcrumb" />
2. `eventProcessors`

Las `hints` de eventos y breadcrumbs son objetos que contienen diversa información usada para armar un evento o un breadcrumb. Por lo general, las `hints` incluyen la excepción original para poder extraer datos adicionales o influir en la agrupación.

En los eventos, las sugerencias contienen propiedades como `event_id`, `originalException`, `syntheticException` (usada internamente para generar un stack trace más limpio) y cualquier `data` arbitrario que adjuntes.

En los breadcrumbs, el uso de las `hints` depende de la implementación. Para solicitudes XHR, la sugerencia contiene el propio objeto xhr; para interacciones de usuario, la sugerencia contiene el elemento del DOM, el nombre del evento, y así sucesivamente.

<div id="hints-for-events">
  #### Sugerencias para eventos
</div>

`originalException`

La excepción original que hizo que el SDK de Sentry creara el evento. Esto es útil para cambiar cómo el SDK de Sentry agrupa los eventos o para extraer información adicional.

`syntheticException`

Cuando se lanza una cadena o un objeto que no es un error, Sentry crea una excepción sintética para que puedas obtener un seguimiento de pila básico. Esta excepción se almacena aquí para extraer más datos.

### Uso de <PlatformIdentifier name="ignore-errors" />

Puedes usar la opción <PlatformIdentifier name="ignore-errors" /> para excluir errores que coincidan con un patrón específico. Esta opción recibe una lista de cadenas y expresiones regulares para comparar con el mensaje de error. Cuando uses cadenas, también se filtrarán las coincidencias parciales; si necesitas una coincidencia exacta, utiliza patrones de regex en su lugar.

<div id="filtering-transaction-events">
  ## Filtrado de eventos de transacción
</div>

Para evitar que ciertas transacciones se envíen a Sentry, usa la opción de configuración <PlatformIdentifier name="traces-sampler" /> o <PlatformIdentifier name="before-send-transaction" />, que te permite definir una función para evaluar la transacción actual y descartarla si no te interesa.

### Uso de <PlatformIdentifier name="traces-sampler" />

**Nota:** Las opciones de configuración <PlatformIdentifier name="traces-sampler" /> y <PlatformIdentifier name="traces-sample-rate" /> son excluyentes entre sí. Si defines un <PlatformIdentifier name="traces-sampler" /> para filtrar determinadas transacciones, también debes manejar el caso de las transacciones no filtradas devolviendo la tasa a la que quieres que se muestren.

También te permite muestrear distintas transacciones a diferentes tasas.

Si la transacción que se está procesando tiene una transacción principal (de un servicio ascendente que llama a este servicio), la decisión de muestreo del principal (ascendente) siempre se incluirá en los datos del contexto de muestreo, para que tu <PlatformIdentifier name="traces-sampler" /> pueda elegir si y cuándo heredar esa decisión. En la mayoría de los casos, heredar es la opción correcta para evitar romper trazas distribuidas. Una traza rota no incluirá todos tus servicios. ### Uso de <PlatformIdentifier name="ignore-transactions" />

Puedes usar la opción <PlatformIdentifier name="ignore-transactions" /> para excluir transacciones que coincidan con un patrón específico. Esta opción recibe una lista de cadenas y expresiones regulares para comparar con el nombre de la transacción. Cuando uses cadenas, las coincidencias parciales también se excluirán; si necesitas filtrar por coincidencia exacta, usa patrones de expresiones regulares en su lugar.
