El SDK de Sentry para React Native es compatible con el motor `Hermes` de forma nativa.

<div id="source-maps">
  ## Mapas de origen
</div>

Consulta nuestra documentación de React Native sobre [mapas de origen](/es/platforms/react-native/sourcemaps/) para aprender a subir mapas de origen para tu app.

<div id="troubleshooting">
  ## Solución de problemas
</div>

Esta sección describe los problemas más comunes al usar Hermes y cómo solucionarlos.

<div id="multiple-ways-of-enabling-source-maps-in-xcode">
  ### Varias formas de habilitar los Source Maps en Xcode
</div>

Si usas `EXTRA_PACKAGER_ARGS`, solo necesitas el source map del empaquetador. Este es el comportamiento predeterminado si utilizaste [`@sentry/wizard`](/es/platforms/react-native/#install). Si usas [`SOURCEMAP_FILE`](https://reactnative.dev/docs/debugging-release-builds#enabling-source-maps), es necesario el source map combinado de Hermes.

<div id="check-rn-tooling-steps">
  ### Revisa los pasos del tooling de RN
</div>

Nuestra documentación de React Native sobre [Source Maps](/es/platforms/react-native/sourcemaps/) se basa en [`react-native-xcode.sh`](https://github.com/facebook/react-native/blob/0.71-stable/scripts/react-native-xcode.sh) para iOS y en [`BundleHermesCTask.kt`](https://github.com/facebook/react-native/blob/0.71-stable/packages/react-native-gradle-plugin/src/main/kotlin/com/facebook/react/tasks/BundleHermesCTask.kt) para Android. Si tienes problemas con los artefactos generados manualmente, revisa los pasos de tu tooling de React Native usando los enlaces anteriores.

<Alert>
  React Native [`0.70`](https://github.com/facebook/react-native/blob/e3a5fbe72f966b27b967192317d7072db52d1c8c/scripts/react-native-xcode.sh#L141) y CodePush Update minifican el bundle del empaquetador de forma predeterminada.
</Alert>

<Alert>
  [`BundleHermesCTask.kt`](https://github.com/facebook/react-native/blob/0.71-stable/packages/react-native-gradle-plugin/src/main/kotlin/com/facebook/react/tasks/BundleHermesCTask.kt) solo está disponible a partir de la versión `0.71`. Para versiones anteriores, usa [`react.gradle`](https://github.com/facebook/react-native/blob/0.70-stable/react.gradle).
</Alert>

Para más detalles, consulta [este issue](https://github.com/facebook/react-native/issues/34043) en el repositorio de React Native.

<div id="dont-combine-hermes-and-ram-bundles">
  ### No combines Hermes y los RAM Bundles
</div>

Si usas Hermes, no deberías tener habilitada la función de [RAM Bundles](/es/platforms/react-native/manual-setup/ram-bundles/). En Hermes, al cargar el bytecode, `mmap` garantiza que no se cargue el archivo completo. Usar Hermes con RAM Bundles puede causar problemas, porque esos mecanismos no son compatibles entre sí.

<div id="minimum-required-versions">
  ### Versiones mínimas necesarias
</div>

Sentry añadió compatibilidad para compilaciones de `react-native` que usan el motor `hermes`, lo que requirió cambios en el SDK de Sentry, `sentry-cli` y en el propio Sentry.

Los clientes de Sentry que usan el producto SaaS ([sentry.io](https://sentry.io/)) deben actualizar el SDK y `sentry-cli`.
La versión mínima necesaria para el SDK es `@sentry/react-native` [versión del SDK `1.3.3`](https://github.com/getsentry/sentry-react-native/releases/tag/1.3.3),
y `@sentry/cli` [versión `1.51.1`](https://github.com/getsentry/sentry-cli/releases/tag/1.51.1).

Para los usuarios de Sentry autohospedado, la versión mínima necesaria es [f07352b](https://hub.docker.com/r/getsentry/sentry/tags?page=1\&name=f07352b).

Una vez que tengas la versión mínima del SDK, Sentry ofrece la integración estándar tal como se describe en la [documentación de Sentry para React Native](/es/platforms/react-native/).

<div id="upload-using-sentry-fastlane-plugin">
  ### Carga con el plugin Sentry Fastlane
</div>

Para subir los sourcemaps generados con Fastlane, usa la acción `sentry_upload_sourcemap`. Consulta cómo instalar el plugin Sentry Fastlane [aquí](https://github.com/getsentry/sentry-fastlane-plugin).

<OrgAuthTokenNote />

```ruby {tabTitle:Android}
sentry_upload_sourcemap(
  auth_token: '___ORG_AUTH_TOKEN___',
  org_slug: '___ORG_SLUG___',
  project_slug: '___PROJECT_SLUG___',
  version: '...',
  app_identifier: '...', # identificador del paquete de tu app
  build: '...', # número de compilación opcional de tu app
  dist: '...', # distribución opcional de la versión, normalmente el número de compilación
  sourcemap: ['index.android.bundle', 'index.android.bundle.map'],
  rewrite: true
)
```

```ruby {tabTitle:iOS}
sentry_upload_sourcemap(
  auth_token: '___ORG_AUTH_TOKEN___',
  org_slug: '___ORG_SLUG___',
  project_slug: '___PROJECT_SLUG___',
  version: '...',
  app_identifier: '...', # identificador del paquete de tu app
  build: '...', # número de compilación opcional de tu app
  dist: '...', # distribución opcional de la versión, normalmente el número de compilación
  sourcemap: ['main.jsbundle', 'main.jsbundle.map'],
  rewrite: true
)
```
