Las siguientes integraciones del sistema forman parte de la biblioteca estándar o del propio intérprete y están habilitadas de forma predeterminada. Para saber qué hacen y cómo desactivarlas si generan problemas, sigue leyendo.

<div id="whats-enabled-by-default">
  ## Qué viene habilitado por defecto
</div>

<div id="inboundfilters">
  ### InboundFilters
</div>

_Nombre de importación: `Sentry.inboundFiltersIntegration`_

Esta integración te permite ignorar errores específicos según el tipo,
el mensaje o las URL de una excepción determinada.

De forma predeterminada, ignorará los errores que comienzan con `Script error` o `JavaScript error: Script error`.

Para configurar esta integración, usa directamente las opciones del SDK `ignoreErrors`, `ignoreTransactions`, `denyUrls`
y `allowUrls`. Ten en cuenta que `denyUrls` y `allowUrls`
solo funcionan para excepciones capturadas, no para eventos de mensajes sin procesar.

<div id="functiontostring">
  ### FunctionToString
</div>

_Nombre de importación: `Sentry.functionToStringIntegration`_

Esta integración permite que el SDK proporcione las funciones y los nombres de métodos originales, incluso cuando dichas funciones o métodos estén envueltos por nuestros controladores de errores o de breadcrumbs.

<div id="breadcrumbs">
  ### Breadcrumbs
</div>

*Nombre de importación: `Sentry.breadcrumbsIntegration`*

Esta integración envuelve las API nativas para capturar breadcrumbs. De forma predeterminada, el SDK de Sentry envuelve todas las API.

Opciones disponibles:

```javascript
{
  // Registrar llamadas a `console.log`, `console.debug`, etc.
  console: boolean;

  // Registrar todos los eventos de clic y pulsación de teclas
  // - Cuando se proporcione un objeto con la clave `serializeAttribute`,
  //   la integración de Breadcrumbs buscará los atributos indicados en los elementos del DOM
  //   al generar el rastro de migas.
  //   Los elementos coincidentes irán seguidos de sus atributos personalizados
  //   en lugar de sus nombres `id` o `class`.
  dom: boolean | { serializeAttribute: string | string[] };

  // Registrar solicitudes HTTP realizadas con la API Fetch
  fetch: boolean;

  // Registrar llamadas a `history.pushState` y similares
  history: boolean;

  // Registrar cada vez que enviemos un evento al servidor
  sentry: boolean;

  // Registrar solicitudes HTTP realizadas con la API XHR
  xhr: boolean;
}
```


<div id="nativelinkederrors">
  ### NativeLinkedErrors
</div>

*Nombre de importación: `Sentry.nativeLinkedErrorsIntegration`*

Esta integración te permite configurar errores vinculados que se leen de manera recursiva hasta un límite especificado. Luego se realiza una búsqueda por una clave específica en el objeto Error capturado. De forma predeterminada, la integración registra hasta cinco errores y la clave utilizada para la recursión es `"cause"`.

Opciones disponibles:

```javascript
{
  key: string; // valor predeterminado: "cause"
  limit: number; // valor predeterminado: 5
}
```

Aquí tienes un ejemplo de código de cómo podría implementarse:

```javascript
<Button
  title="Obtener reseñas"
  onPress={async () => {
    const movie = event.target.dataset.title;
    try {
      await fetchMovieReviews(movie);
    } catch (e) {
      const fetchError = new Error(`No se pudieron obtener reseñas para: ${movie}`);
      // @ts-expect-error la propiedad cause no está expuesta en la clase de error predeterminada.
      fetchError.cause = e;
      Sentry.captureException(fetchError);
    }
  }}
/>
```


<div id="httpcontext">
  ### HttpContext
</div>

_Import name: `Sentry.httpContextIntegration`_

Esta integración adjunta al evento información de la solicitud HTTP, como la URL, el agente de usuario, el referente y otros encabezados.
Nos permite catalogar y etiquetar correctamente los eventos con información específica del sistema operativo, el navegador y la versión.

<div id="dedupe">
  ### Dedupe
</div>

*Nombre de importación: `Sentry.dedupeIntegration`*

Esta integración está habilitada de forma predeterminada, pero solo elimina duplicados de ciertos eventos. Puede ser útil si estás recibiendo muchos errores duplicados. Ten en cuenta que Sentry solo compara trazas y huellas digitales.

```javascript
Sentry.init({
  dsn: "___PUBLIC_DSN___",
  integrations: [Sentry.dedupeIntegration()],
});
```


<div id="modifying-system-integrations">
  ## Modificar las integraciones del sistema
</div>

Para desactivar las integraciones del sistema, establece `defaultIntegrations: false` al llamar a `init()`.

Para anular su configuración, proporciona una nueva instancia con tu configuración en la opción `integrations`. Por ejemplo, para desactivar que el navegador capture las llamadas a la consola:

```javascript
Sentry.init({
  dsn: "___PUBLIC_DSN___",

  integrations: [
    Sentry.breadcrumbsIntegration({
      console: false,
    }),
  ],
});
```


<div id="removing-an-integration">
  ### Eliminar una integración
</div>

Este ejemplo elimina la integración que agrega breadcrumbs (rastros de navegación) al evento, la cual está habilitada de forma predeterminada:

```javascript
Sentry.init({
  // ...
  integrations: function (integrations) {
    // las integraciones incluirán todas las integraciones predeterminadas
    return integrations.filter(function (integration) {
      return integration.name !== "Breadcrumbs";
    });
  },
});
```
