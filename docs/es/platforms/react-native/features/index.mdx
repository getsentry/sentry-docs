---

El SDK de React Native de Sentry permite el reporte automático de errores y excepciones, e identifica problemas de rendimiento en tu aplicación. A continuación encontrarás una lista de funcionalidades disponibles en este SDK.

- <Link to="/product/issues/">Seguimiento de errores</Link> de bloqueos nativos automático (usando tanto `sentry-cocoa` como `sentry-android`).
- Detección automática de <Link to="/platforms/android/configuration/app-not-respond">Application Not Responding (ANR)</Link> en Android y <Link to="/platforms/apple/guides/ios/configuration/app-hangs/">App Hangs</Link> en iOS.
- Almacenamiento de eventos sin conexión:
  - Android: Caché sin conexión cuando el dispositivo está offline; enviamos un informe una vez que se reinicia la aplicación.
  - iOS: Caché sin conexión cuando el dispositivo no puede conectarse; enviamos un informe cuando recibimos otro evento.
- Eventos <PlatformLink to="/enriching-events/context/">enriquecidos</PlatformLink> con datos del dispositivo.
- <PlatformLink to="/data-management/debug-files/source-context/">Contexto de código fuente</PlatformLink> muestra fragmentos de tu código alrededor de la ubicación de los frames de la pila.
- <Link to="https://reactnative.dev/blog/2019/07/03/version-60#native-modules-are-now-autolinked">Autolinking</Link>.
- <PlatformLink to="/enriching-events/breadcrumbs/">Breadcrumbs</PlatformLink> creadas para solicitudes `http` salientes con XHR y Fetch; eventos de la UI y del sistema; y registros de consola.
- <Link to="/product/releases/health/">Salud de versiones</Link> hace seguimiento de usuarios y sesiones sin bloqueos.
- <Link to="/product/insights/overview/">Tracing</Link> crea transacciones automáticamente para:
  - tiempo de <PlatformLink to="/tracing/instrumentation/automatic-instrumentation/#app-start-instrumentation">inicio de la app</PlatformLink>.
  - <PlatformLink to="/tracing/instrumentation/automatic-instrumentation/#enable-routing-instrumentation">instrumentación de enrutamiento</PlatformLink> (React Navigation v4 y superior y React Native Navigation).
  - solicitudes de <PlatformLink to="/tracing/instrumentation/automatic-instrumentation/#fetchxml-request-instrumentation">XHR y Fetch</PlatformLink>.
  - eventos de <PlatformLink to="/tracing/instrumentation/user-interaction-instrumentation/">interacción del usuario</PlatformLink> (toques, gestos).
  - <PlatformLink to="/tracing/instrumentation/automatic-instrumentation/#slow-and-frozen-frames">seguimiento de frames lentos y congelados</PlatformLink>.
  - <PlatformLink to="/tracing/instrumentation/automatic-instrumentation/#stall-tracking">seguimiento de bloqueos</PlatformLink> del bucle de JavaScript.
  - <PlatformLink to="/tracing/instrumentation/automatic-instrumentation/#react-profiler">React Profiler</PlatformLink> hace seguimiento de componentes de React.
- Bajo el capó, el SDK se basa en nuestro <Link to="/platforms/javascript/">SDK de JavaScript</Link>, lo que hace que todas las funciones disponibles para JavaScript también lo estén en este SDK.
- Simbolicación en el dispositivo para JavaScript (en Debug).
- <PlatformLink to="/manual-setup/ram-bundles/">Compatibilidad con RAM bundles</PlatformLink>.
- <PlatformLink to="/manual-setup/hermes/">Compatibilidad con Hermes</PlatformLink>.
- <PlatformLink to="/manual-setup/expo/">Compatibilidad con Expo</PlatformLink> lista para usar.
- <PlatformLink to="/enriching-events/attachments/">Adjuntos</PlatformLink> enriquecen tus eventos al almacenar archivos adicionales, como archivos de configuración o de registro.
- <PlatformLink to="/user-feedback/">Comentarios de usuario</PlatformLink> permite recopilar información del usuario cuando ocurre un evento.
- <PlatformLink to="/enriching-events/viewhierarchy/">Jerarquía de vistas</PlatformLink> muestra la estructura de los componentes nativos en el momento en que se produjo un error.
