---
title: Lanzamiento y distribución opcionales
description: "Obtén información sobre cómo configurar atributos opcionales de source maps con SDK de React Native más antiguos."
sidebar_order: 4
---

Para atribuir correctamente los eventos enviados a Sentry a una versión, **es necesario establecer tanto `release` como `dist`.** Puedes definir estos valores en la llamada a `init`, como se explica en nuestra documentación de [Lanzamientos y estado](/es/platforms/react-native/configuration/releases/).

```javascript
Sentry.init({
  dsn: "___DSN___",
  release: "my-project-name@2.3.12",
  dist: "52",
});
```

El script automático de subida de source maps no detecta valores personalizados para las releases. Esto significa que, si usas un valor personalizado para tu release, distinto de la versión incluida con el Xcode o Android Studio integrados, el script automático de subida de source maps no funcionará. En su lugar, puedes establecer las variables de entorno `SENTRY_RELEASE` y `SENTRY_DIST`, que el script automático de subida de source maps usará. Esto garantizará que se usen los valores personalizados de `release` y `dist` indicados.

```bash
export SENTRY_RELEASE="my-project-name@2.3.12"
export SENTRY_DIST="52"
```
