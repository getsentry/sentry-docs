Si Sentry tiene acceso al código fuente de tu aplicación, puede mostrar fragmentos de código (*contexto de origen*) alrededor de la ubicación de los fotogramas de la pila, lo que ayuda a identificar rápidamente el código problemático.

Por ejemplo, aquí tienes un rastreo de pila de una aplicación .NET. El contexto de origen contiene
el código fuente original en C# de la aplicación, con la ubicación del fotograma resaltada.

![Captura de pantalla de un rastreo de pila que muestra el contexto de origen.](./img/source-context-screenshot.png)

<div id="source-bundles">
  ### Paquetes de origen
</div>

Ciertos SDK, como el SDK de Python, pueden resolver este contexto de origen automáticamente,
porque tienen acceso a código fuente sin ofuscar en tiempo de ejecución.
En cambio, para obtener contexto de origen en aplicaciones compiladas, el código fuente
debe subirse junto con los archivos de información de depuración.
La forma recomendada de hacerlo es usando `sentry-cli`.
Consulta [Creación de paquetes de origen](/es/cli/dif/#creating-source-bundles) para más información.

Para obtener contexto de origen para el código JavaScript de React Native, sigue las instrucciones de la guía de [mapas de origen](/es/platforms/react-native/sourcemaps/).
Para el código nativo de Android e iOS, sigue las instrucciones de la guía de [símbolos de depuración](/es/platforms/react-native/upload-debug/).

Una vez subidos, puedes revisar y gestionar los paquetes de origen igual que
cualquier otro archivo de información de depuración. Consulta [Gestión de archivos de información de depuración](../#managing-debug-information-files).
Los paquetes de origen se etiquetarán con `sources` y tendrán el mismo nombre de archivo que el
archivo de información de depuración correspondiente del que se crearon.
