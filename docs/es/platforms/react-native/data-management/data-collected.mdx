---
title: Datos recopilados
description: "Consulta qué datos recopila el SDK de Sentry."
sidebar_order: 1
---

Sentry se toma muy en serio la privacidad de los datos y cuenta con configuraciones predeterminadas que priorizan la seguridad de la información, especialmente la relativa a datos de identificación personal (PII). Al agregar el SDK de Sentry a tu aplicación, permites que recopile datos y los envíe a Sentry durante la ejecución de tu aplicación.

Los tipos de categorías y la cantidad de datos recopilados varían según las integraciones que tengas habilitadas en el SDK de Sentry. En esta página se enumeran las categorías de datos que recopila el SDK de Sentry para React Native.

Muchas de las categorías enumeradas aquí requieren que establezcas <PlatformLink to="/configuration/options/#sendDefaultPii">`sendDefaultPii: true`</PlatformLink> en tu configuración de `Sentry.init({})`.

<div id="http-headers">
  ## Encabezados HTTP
</div>

De forma predeterminada, los eventos de error de solicitudes Fetch y XHR fallidas de `Sentry.httpClientIntegration` no incluyen encabezados.

Para empezar a enviar encabezados, establece <PlatformLink to="/configuration/options/#sendDefaultPii">`sendDefaultPii: true`</PlatformLink> en la configuración de `Sentry.init({})`.

<div id="user-agent">
  ## User-Agent
</div>

De forma predeterminada, el SDK de Sentry adjunta el User-Agent de la solicitud HTTP al evento. Para desactivar este comportamiento, elimina `Sentry.httpContextIntegration` del array de integraciones.

<div id="cookies">
  ## Cookies
</div>

De forma predeterminada, el SDK de Sentry no envía cookies. Sentry intenta eliminar cualquier cookie que contenga información confidencial, como las cookies de ID de sesión y del token CSRF.

Para comenzar a enviar cookies, establece <PlatformLink to="/configuration/options/#send-default-pii">`sendDefaultPii: true`</PlatformLink> en tu configuración de `Sentry.init({})`.

<div id="information-about-logged-in-user">
  ## Información sobre el usuario autenticado
</div>

De forma predeterminada, el SDK de Sentry no envía información sobre el usuario autenticado, como la dirección de correo electrónico, el ID de usuario o el nombre de usuario. Incluso si lo habilitas, el tipo de información del usuario autenticado que podrás enviar depende de las integraciones que actives en el SDK de Sentry. La mayoría de las integraciones no enviarán ningún dato de usuario. Algunas solo establecerán el ID de usuario, pero hay otras que establecerán el ID de usuario, el nombre de usuario y la dirección de correo electrónico.

Para empezar a enviar información del usuario autenticado, configura <PlatformLink to="/configuration/options/#send-default-pii">`sendDefaultPii: true`</PlatformLink> en tu `Sentry.init({})`.

<div id="users-ip-addresses">
  ## Direcciones IP de los usuarios
</div>

De forma predeterminada, los servicios de backend de Sentry infieren la dirección IP del usuario a partir de la solicitud entrante.

Para desactivar el envío de la dirección IP del usuario, reemplaza el valor predeterminado por una cadena personalizada, por ejemplo, llamando a `Sentry.setUser({ip_address: '0.0.0.0'})`.

<div id="request-url">
  ## URL de la solicitud
</div>

La URL completa de la solicitud y el encabezado Referer de las solicitudes HTTP salientes **siempre se envían a Sentry**. Según tu aplicación, esto podría incluir datos personales (PII).

<div id="request-query-string">
  ## Cadena de consulta de la solicitud
</div>

La cadena de consulta completa de las solicitudes HTTP salientes **siempre se envía a Sentry**. Según tu aplicación, esto podría incluir datos personales (PII).

<div id="device-information">
  ## Información del dispositivo
</div>

De forma predeterminada, el SDK de Sentry no envía el nombre del dispositivo en Android.

Si quieres enviar el nombre del dispositivo, establece <PlatformLink to="/configuration/options/#send-default-pii">`sendDefaultPii: true`</PlatformLink> en la configuración de `Sentry.init({})`.

<div id="console-logs">
  ## Registros de consola
</div>

De forma predeterminada, el SDK de Sentry envía a Sentry los registros de la consola de JS, que pueden contener datos personales (PII).

Para desactivar el envío de registros de consola, establece `console: false` en la configuración de `Sentry.breadcrumbsIntegration`; consulta <PlatformLink to="/integrations/default/#breadcrumbs">la documentación de Breadcrumbs</PlatformLink>.

<div id="screenshots">
  ## Capturas de pantalla
</div>

La <PlatformLink to="/enriching-events/screenshots">función de capturas de pantalla</PlatformLink> está desactivada de forma predeterminada, pero al habilitarla, las capturas pueden contener datos personales (PII).

<div id="view-hierarchy">
  ## Jerarquía de vistas
</div>

La <PlatformLink to="/enriching-events/viewhierarchy">función de jerarquía de vistas</PlatformLink> está desactivada de forma predeterminada, pero cuando se habilita, la jerarquía de vistas puede contener datos personales (PII) al usar la propiedad `accessibilityIdentifier` con información personal.

<div id="session-replay">
  ## Reproducción de sesión
</div>

De forma predeterminada, nuestro SDK de reproducción de sesión oculta todo el contenido de texto, imágenes, vistas web y la entrada del usuario. Esto ayuda a garantizar que no se expongan datos sensibles. Puedes encontrar <PlatformLink to="/session-replay/privacy/">más detalles en la documentación de Reproducción de sesión</PlatformLink>.

<div id="source-context">
  ## Contexto de origen
</div>

De forma predeterminada, las herramientas de compilación del SDK de Sentry para React Native subirán únicamente los bundles de JS de las aplicaciones y sus mapas de código fuente (source maps) a Sentry. Para desactivar la subida de fuentes, establece `SENTRY_DISABLE_AUTO_UPLOAD=true` en tus variables de entorno; consulta <PlatformLink to="/sourcemaps/#disable-automatic-upload/">la documentación de Source Maps</PlatformLink>.

Para habilitar la subida del código fuente nativo, activa la función como se describe en <PlatformLink to="/upload-debug/">la documentación de Debug Symbols</PlatformLink>.
