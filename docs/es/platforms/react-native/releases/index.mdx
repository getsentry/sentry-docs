El SDK de React Native utiliza los siguientes canales para distribuir actualizaciones del SDK:

<div id="pre-release">
  ## `Pre-release`
</div>

Las pre-releases (alfa, beta, RC,…) se usan para cambios más grandes y potencialmente más relevantes, como nuevas funciones o versiones mayores.

<div id="latest">
  ## `Latest`
</div>

Publicamos continuamente versiones principales/secundarias/correcciones rápidas desde la rama `main`. Estas versiones pasan por todos nuestros controles de calidad internos y son muy seguras de usar. Están pensadas para ser la opción predeterminada para la mayoría de los equipos.

<div id="stable">
  ## `Stable`
</div>

Las versiones pasan de `Latest` a `Stable` cuando han sido utilizadas en producción durante un tiempo y a cierta escala. Antes de considerar una versión como `Stable`, se tienen en cuenta el tiempo que lleva disponible, su nivel de adopción y otras métricas de calidad y estabilidad. Este tipo de versiones llevarán la etiqueta `Stable` en la <Link to="https://github.com/getsentry/sentry-react-native/releases/">página de lanzamientos</Link>.

<div id="faq">
  ## Preguntas frecuentes
</div>

<div id="where-can-i-find-the-latest-stable-release-for-any-given-sdk">
  ### ¿Dónde puedo encontrar la última versión estable de cualquier SDK?
</div>

En la <Link to="https://github.com/getsentry/sentry-react-native/releases/">página de versiones</Link> del repositorio de GitHub del SDK. Las versiones estables tendrán la etiqueta `Stable` en el nombre de la versión. También puedes usar el cuadro de búsqueda en la parte superior derecha para encontrar las versiones más recientes.

<div id="does-this-mean-the-release-channels-are-not-stable">
  ### ¿Esto significa que los canales de lanzamiento no son estables?
</div>

No. La estabilidad y el rendimiento son las principales prioridades en cada lanzamiento. El estado `Stable` indica que la versión ha sido probada y ha demostrado funcionar en entornos reales y a gran escala.

<div id="how-often-do-we-mark-sdks-as-stable">
  ### ¿Con qué frecuencia marcamos los SDK como estables?
</div>

El ciclo de versiones estables depende de varios factores y puede variar, pero el objetivo es publicar versiones estables aproximadamente cada mes.

<div id="should-i-only-use-stable-releases">
  ### ¿Debería usar solo versiones estables?
</div>

Esta decisión es tuya. Las versiones estables se han probado en uso real y a gran escala. Están diseñadas para ofrecer una experiencia fiable y coherente. Ten en cuenta que el canal de versión `Stable` recibirá nuestras funciones más recientes más tarde que otros canales.

<div id="how-does-this-relate-to-the-stability-of-a-feature">
  ### ¿Cómo se relaciona esto con la estabilidad de una función?
</div>

Un estado de lanzamiento `Stable` no garantiza la estabilidad de una función del SDK ni de una función del producto. Una función nueva que aún sea experimental podría formar parte de una versión estable del SDK. Cualquier función experimental estará claramente señalizada y se ofrecerá de forma opcional (opt‑in).
