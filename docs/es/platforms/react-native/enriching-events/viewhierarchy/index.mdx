---
title: "Jerarquía de vistas"
description: "Obtén más información sobre cómo depurar la jerarquía de vistas cuando ocurre un error. Sentry empareja la representación de la jerarquía de vistas con el evento original, brindándote información adicional sobre los problemas."
---

Sentry permite generar una representación JSON de la jerarquía de vistas de un error y la incluye como un <PlatformLink to="/enriching-events/attachments/">adjunto</PlatformLink>.

Esta función solo se aplica a SDK con interfaz de usuario, como los de aplicaciones móviles y de escritorio. En algunos entornos, como iOS nativo, renderizar la jerarquía de vistas requiere el hilo de la interfaz y, en caso de un bloqueo, es posible que no esté disponible. Otro ejemplo en el que la jerarquía de vistas podría no estar disponible es cuando el evento ocurre antes de que la pantalla comience a cargarse. Por lo tanto, esta función es inherentemente una solución de mejor esfuerzo.

<Alert>
  La desofuscación de jerarquías de vistas está totalmente admitida para SDK nativos y React Native, pero actualmente no se admite para Flutter.
</Alert>

<div id="viewing-view-hierarchy-attachments">
  ## Visualización de adjuntos de la jerarquía de vistas
</div>

Las jerarquías de vistas aparecen en la pestaña &quot;Attachments&quot;, donde puedes ver todos los adjuntos y los eventos asociados. Haz clic en el ID del evento para abrir la página [Issue Details](/es/product/issues/issue-details) de ese evento específico.

<Include name="common-imgs/viewhierarchy-list-example" />

En la página **Issue Details**, puedes interactuar con el adjunto de la jerarquía de vistas en una sección llamada &quot;View Hierarchy&quot;. Esta sección representa el estado de tu aplicación en el momento del evento de error. Hay tres modos de visualización: vista en árbol, wireframe y vista detallada del nodo seleccionado. Puedes seleccionar nodos tanto en el árbol como en el wireframe para ver las propiedades recopiladas por el SDK. El SDK informa las siguientes claves para cada nodo de la vista: `alpha`, `visible`, `x`, `y`, `width`, `height`, `type` e `identifier`, si corresponde, aunque puede haber valores adicionales específicos del SDK. Esta función puede usarse como herramienta exploratoria para depurar problemas de diseño, visualizar contenido renderizado innecesariamente u obtener una mejor comprensión de la relación entre vistas.

<div id="view-hierarchy-mapping-to-react-native-components">
  ## Correspondencia de la jerarquía de vistas con los componentes de React Native
</div>

La jerarquía de vistas, tal como la captura el SDK, representa la interfaz de usuario nativa generada a partir de la estructura de componentes de React Native. En general, cada elemento de React se corresponde con un componente nativo, pero hay excepciones como el [aplanamiento de vistas](https://reactnative.dev/architecture/view-flattening), tal como se describe en la documentación de React Native.

Los componentes nativos que envuelven la aplicación (creados por React Native) son otra parte de la jerarquía de vistas.

<div id="ios">
  ### iOS
</div>

El siguiente fragmento de código muestra componentes de React Native y de UIKit nativo tal como se incluirían en la jerarquía de vistas.

{/*<!-- prettier-ignore -->*/}

```javascript
                                 // UIWindow
                                 // RCTRootContentView
                                 // ...
function MyComponent() {
  return (
    <View testID='id'>           // RTCView - id
      <Text>¡Hola mundo!</Text>  // RCTTextView
    </View>
  );
}
```

<div id="android">
  ### Android
</div>

El siguiente fragmento de código muestra componentes nativos de React Native y Android tal como aparecerían en la jerarquía de vistas.

{/*<!-- prettier-ignore -->*/}

```javascript
                                 // com.android.internal.policy.DecorView
                                 // android.widget.LinearLayout
                                 // ...
                                 // com.facebook.react.ReactRootView
function MyComponent() {
  return (
    <View>                       // com.facebook.react.views.view.ReactViewGroup
      <Text>¡Hola, mundo!</Text>  // com.facebook.react.views.text.ReactTextView
    </View>
  );
}
                                 // android.view.View - navigationBarBackground
                                 // android.view.View - statusBarBackground
```
