Sentry permite tomar automáticamente una captura de pantalla e incluirla como un <PlatformLink to="/enriching-events/attachments/">archivo adjunto</PlatformLink> cuando un usuario experimenta un error, una excepción o un bloqueo.

Esta función solo está disponible para SDK con interfaz de usuario, como los de aplicaciones móviles y de escritorio. También depende de si es posible o no tomar una captura de pantalla. Por ejemplo, en algunos entornos, como iOS nativo, tomar una captura de pantalla requiere el hilo de la interfaz, que a menudo no está disponible en caso de bloqueo. Otro ejemplo en el que puede no haber una captura de pantalla es cuando el evento ocurre antes de que la pantalla empiece a cargarse. Por lo tanto, esta función, por su naturaleza, es una solución de “mejor esfuerzo”.

<div id="enabling-screenshots">
  ## Habilitar capturas de pantalla
</div>

Como las capturas de pantalla pueden contener <PlatformLink to="/data-management/sensitive-data/">PII</PlatformLink>, esta función es opcional (opt‑in). Puedes habilitarlas como se muestra a continuación:

<PlatformContent includePath="enriching-events/attach-screenshots" />

Cuando `attachScreenshot` está habilitado y se captura un evento de error, las capturas de pantalla se transfieren desde los SDK nativos a React Native a través del bridge. Esto puede afectar negativamente el rendimiento en dispositivos de gama baja; sin embargo, la nueva arquitectura de React Native mejorará el rendimiento de la transferencia, ya que los datos no se serializan.

<div id="viewing-screenshots">
  ## Ver capturas de pantalla
</div>

Si hay una, verás una miniatura de la captura cuando hagas clic en un issue específico desde la página de [**Issues**](https://demo.sentry.io/issues/).

![Miniatura de captura de pantalla](./img/screenshot-thumbnail.png)

Una vez que hayas hecho clic en el ID del evento de un issue específico, podrás ver un resumen de todos los adjuntos, así como los eventos asociados, en la pestaña "Attachments".
