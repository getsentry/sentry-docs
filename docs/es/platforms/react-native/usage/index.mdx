El SDK de Sentry se integra en tu entorno de ejecución y reporta automáticamente errores, excepciones no controladas y rechazos no gestionados, además de otros tipos de errores según la plataforma.

<Alert>

Términos clave:

- Un _evento_ es un envío de datos a Sentry. Por lo general, estos datos son un error o una excepción.
- Un _issue_ es un grupo de eventos similares.
- Al envío de un evento se le llama _captura_.
  Cuando se captura un evento, se envía a Sentry.

</Alert>

La forma más común de captura es la de errores. Lo que puede capturarse como error varía según la plataforma. En general, si tienes algo que parece una excepción, puede capturarse. En algunos SDK, también puedes omitir el argumento de <PlatformIdentifier name="capture-exception" /> y Sentry intentará capturar la excepción actual. También es útil para reportar manualmente errores o mensajes a Sentry.

Mientras capturas un evento, también puedes registrar los breadcrumbs que llevan a ese evento. Los breadcrumbs son distintos de los eventos: no crean un evento en Sentry, sino que se almacenan en búfer hasta que se envíe el siguiente evento. Más información sobre los breadcrumbs en nuestra <PlatformLink to="/enriching-events/breadcrumbs/">documentación de Breadcrumbs</PlatformLink>.
