<div id="group-transactions">
  ## Agrupar transacciones
</div>

Cuando Sentry captura transacciones, se les asigna un nombre de transacción. Este nombre generalmente lo genera automáticamente el SDK de Sentry según las integraciones del framework que estés usando. Si no puedes aprovechar la generación automática de transacciones (o quieres personalizar cómo se generan los nombres de las transacciones), puedes usar un procesador global de eventos que se registra al inicializar el SDK con tu configuración.

<div id="control-data-truncation">
  ## Controlar el truncamiento de datos
</div>

Actualmente, cada etiqueta tiene un límite máximo de 200 caracteres. Las etiquetas que superen ese límite se truncarán, lo que podría hacer que se pierda información importante. Para conservar estos datos, puedes dividir la información en varias etiquetas.

Por ejemplo, una solicitud con etiqueta de más de 200 caracteres:

`https://empowerplant.io/api/0/projects/ep/setup_form/?user_id=314159265358979323846264338327&tracking_id=EasyAsABC123OrSimpleAsDoReMi&product_name=PlantToHumanTranslator&product_id=161803398874989484820458683436563811772030917980576`

La solicitud anterior, con más de 200 caracteres, se truncará a:

`https://empowerplant.io/api/0/projects/ep/setup_form/?user_id=314159265358979323846264338327&tracking_id=EasyAsABC123OrSimpleAsDoReMi&product_name=PlantToHumanTranslator&product_id=1618033988749894848`
