Con [tracing](/es/product/insights/overview/), Sentry supervisa el rendimiento de tu software, midiendo métricas como throughput y latencia, y mostrando el impacto de los errores en múltiples sistemas. Sentry captura trazas distribuidas compuestas por transacciones y spans, que miden servicios individuales y operaciones individuales dentro de esos servicios. Obtén más información sobre nuestro modelo en [Distributed Tracing](/es/product/sentry-basics/tracing/distributed-tracing/).

<div id="configure">
  ## Configurar
</div>

Primero, habilita el rastreo (tracing) y configura la tasa de muestreo para las transacciones. Establece la tasa de muestreo para tus transacciones de una de estas dos formas:

* Definiendo una tasa de muestreo uniforme para todas las transacciones mediante la opción <PlatformIdentifier name="traces-sample-rate" /> en la configuración de tu SDK, con un valor entre `0` y `1`. (Por ejemplo, para enviar el 20% de las transacciones, establece <PlatformIdentifier name="traces-sample-rate" /> en `0.2`).
* Controlando la tasa de muestreo según la propia transacción y el contexto en el que se captura, proporcionando una función a la opción de configuración <PlatformIdentifier name="traces-sampler" />.

Estas dos opciones están pensadas para usarse de forma mutuamente exclusiva. Si configuras ambas, <PlatformIdentifier name="traces-sampler" /> tendrá prioridad.

Obtén más información sobre las <PlatformLink to="/configuration/options/#tracing-options">opciones</PlatformLink> de rastreo, cómo usar la función <PlatformLink to="/configuration/sampling/#setting-a-sampling-function">tracesSampler</PlatformLink> o cómo <PlatformLink to="/configuration/sampling/#sampling-transaction-events">muestrear transacciones</PlatformLink>.

<div id="verify">
  ## Verificar
</div>

Comprueba que el tracing funcione correctamente usando nuestra <PlatformLink to="/tracing/instrumentation/automatic-instrumentation/">instrumentación automática</PlatformLink> o iniciando y finalizando una transacción con <PlatformLink to="/tracing/instrumentation/custom-instrumentation/">instrumentación personalizada</PlatformLink>.

Mientras haces pruebas, establece <PlatformIdentifier name="traces-sample-rate" /> en `1.0`, ya que así te aseguras de que cada transacción se envíe a Sentry. Cuando termines de probar, quizá prefieras configurar un valor más bajo para <PlatformIdentifier name="traces-sample-rate" />, o pasar a usar <PlatformIdentifier name="traces-sampler" /> para muestrear y filtrar tus transacciones de forma selectiva según datos contextuales.
