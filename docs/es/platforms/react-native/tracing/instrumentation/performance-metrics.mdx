Los SDK de Sentry permiten enviar datos de métricas de rendimiento a Sentry. Son valores numéricos asociados a las transacciones que se agregan y se muestran en Sentry.

El SDK de React Native recopila automáticamente las siguientes métricas de rendimiento:

* <PlatformLink to="/tracing/instrumentation/automatic-instrumentation/#app-start-instrumentation"> Tiempo de inicio de la app en frío y en caliente </PlatformLink>
* <PlatformLink to="/tracing/instrumentation/automatic-instrumentation/#slow-and-frozen-frames"> Renderizado de fotogramas lentos y congelados </PlatformLink>

<div id="custom-measurements">
  ## Mediciones personalizadas
</div>

Además de las métricas de rendimiento automáticas, el SDK permite establecer [mediciones de rendimiento personalizadas](/es/product/insights/overview/metrics/#custom-performance-measurements) en transacciones. Esto te permite definir mediciones importantes para tu aplicación y enviarlas a Sentry.

Para establecer una medición de rendimiento, debes proporcionar lo siguiente:

* nombre (`string`)
* valor (cualquier tipo numérico: `float`, `integer`, etc.)
* unidad (`string`, de forma predeterminada la cadena `none` si se omite)

Sentry admite agregar unidades personalizadas arbitrarias, pero recomendamos usar una de las [unidades compatibles que se indican a continuación](./#supported-measurement-units).

```javascript
// Registrar la cantidad de memoria utilizada
Sentry.setMeasurement("memoryUsed", 123, "byte");

// Registrar el tiempo cuando el componente Footer se renderiza en la página
Sentry.setMeasurement("ui.footerComponent.render", 1.3, "second");

// Registrar la cantidad de veces que se leyó localStorage
Sentry.setMeasurement("localStorageRead", 4);
```

<div id="supported-measurement-units">
  ## Unidades de medición compatibles
</div>

Las unidades le dan significado a valores de medición que, de otro modo, serían números abstractos. Agregar unidades también permite que Sentry ofrezca controles —conversiones, filtros, etc.— basados en esas unidades. Para valores sin unidad, puedes proporcionar una cadena vacía o `none`.

<div id="duration-units">
  ### Unidades de duración
</div>

* `nanosecond`
* `microsecond`
* `millisecond`
* `second`
* `minute`
* `hour`
* `day`
* `week`

<div id="information-units">
  ### Unidades de información
</div>

* `bit`
* `byte`
* `kilobyte`
* `kibibyte`
* `megabyte`
* `mebibyte`
* `gigabyte`
* `gibibyte`
* `terabyte`
* `tebibyte`
* `petabyte`
* `pebibyte`
* `exabyte`
* `exbibyte`

<div id="fraction-units">
  ### Unidades fraccionarias
</div>

* `ratio`
* `percent`

Si quieres profundizar, puedes encontrar detalles sobre las unidades admitidas en nuestra [documentación de ingesta de eventos](https://getsentry.github.io/relay/relay_metrics/enum.MetricUnit.html).
