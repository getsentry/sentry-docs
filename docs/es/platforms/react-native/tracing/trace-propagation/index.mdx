Si el panorama de aplicaciones que quieres observar con Sentry incluye más que un único servicio o aplicación, el trazado distribuido puede aportar mucho valor.

<div id="what-is-distributed-tracing">
  ## ¿Qué es el rastreo distribuido?
</div>

En el contexto de seguir eventos a través de un sistema distribuido, el rastreo distribuido es una potente herramienta de depuración. Imagina tu aplicación como una amplia red de partes interconectadas. Por ejemplo, tu sistema podría estar repartido entre distintos servidores o tu aplicación podría dividirse en servicios de backend y frontend, cada uno con su propia pila tecnológica.

Cuando surge un error o un problema de rendimiento, puede resultar difícil identificar la causa raíz debido a la complejidad de un sistema así. El rastreo distribuido te ayuda a seguir el recorrido de un evento a medida que atraviesa esta compleja red, registrando cada paso que da. Al analizar estos rastros, puedes reconstruir la secuencia de eventos que condujeron al suceso de interés, identificar los componentes específicos implicados y entender cómo interactúan. Esta visibilidad detallada te permite diagnosticar y resolver problemas de forma más eficaz, mejorando en última instancia la confiabilidad y el rendimiento de tu sistema distribuido.

  ## ¿Cómo usar el rastreo distribuido?
</div>

<PlatformContent includePath="distributed-tracing/how-to-use/" />

Recuerda que, para propagar la información de las trazas en todo tu sistema distribuido, debes usar Sentry en todos los servicios y aplicaciones implicados. Consulta la documentación del SDK correspondiente para saber cómo habilitar el rastreo distribuido en cada plataforma.
