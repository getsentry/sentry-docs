title: Símbolos de depuración

description: "Aprende a usar el complemento de Gradle de Sentry o sentry-cli para proporcionar símbolos de depuración a Sentry."
---

Los símbolos de depuración te ayudan a obtener trazas de pila legibles, que Sentry muestra en la página **Detalles del issue**, para ayudarte a gestionar un issue. Ofrecemos varios métodos para proporcionar símbolos de depuración a Sentry.

<div id="when-to-upload">
  ## Cuándo subir
</div>

Con la configuración predeterminada, las trazas de pila completas están disponibles en tus errores de Java/Kotlin desde el primer momento, a menos que uses [enableProguardInReleaseBuilds](https://reactnative.dev/docs/signed-apk-android#enabling-proguard-to-reduce-the-size-of-the-apk-optional). En ese caso, debes subir los archivos de asignación (mapping) de ProGuard/R8 generados por el [Android Gradle Plugin](https://developer.android.com/studio/build/shrink-code#shrink-code), para que Sentry pueda mostrarte trazas de pila correctas.

<div id="uploading-with-sentry-gradle-plugin">
  ## Carga con el Sentry Gradle Plugin
</div>

Sube los [archivos de mapeo de ProGuard/R8 para Android y los símbolos nativos](/es/platforms/android/enhance-errors/proguard/) usando el método recomendado: nuestra integración con Gradle.

Los errores que se originan en la capa nativa de las apps de Android requieren cargar ciertos archivos de información de depuración. Por ejemplo, para una app de Android con soporte de [NDK](https://developer.android.com/ndk/), es necesario subir los archivos de depuración DWARF.

Para cargar símbolos nativos en compilaciones de React Native para Android, usa la [opción `uploadNativeSymbols`](/es/platforms/react-native/manual-setup/manual-setup/#enable-sentry-agp) del Sentry Gradle Plugin.

<div id="uploading-with-xcode">
  ## Carga con Xcode
</div>

Para subir símbolos de depuración nativos de iOS, configura una fase de compilación de tipo Run Script para cargar los símbolos en Sentry con Sentry CLI. Puedes encontrar la documentación correspondiente en la página de [Configuración manual](/es/platforms/react-native/manual-setup/manual-setup/#upload-debug-symbols-to-sentry).

<div id="uploading-with-sentry-cli">
  ## Carga con sentry-cli
</div>

Carga manualmente los [archivos de asignación (mapping) de Android ProGuard/R8 y los símbolos nativos](/es/cli/) usando `sentry-cli` [upload-proguard](/es/cli/dif/#proguard-mapping-upload) y [debug-files](/es/cli/dif/#uploading-files).

<div id="uploading-source-context-with-sentry-gradle-plugin">
  ## Cargar Source Context con el complemento Gradle de Sentry
</div>

Para habilitar [Source Context](/es/platforms/android/data-management/debug-files/source-context/), usa la opción [`includeSourceContext`](/es/platforms/react-native/manual-setup/manual-setup/#enable-sentry-agp) del complemento Gradle de Sentry.

<div id="uploading-source-context-with-xcode">
  ## Cargar el contexto de código fuente con Xcode
</div>

Para habilitar [Source Context](/es/platforms/apple/guides/ios/data-management/debug-files/source-context/), configura una [fase de compilación de Run Script para cargar símbolos de depuración en Sentry](/es/platforms/react-native/manual-setup/manual-setup/#upload-debug-symbols-to-sentry) y establece `SENTRY_INCLUDE_NATIVE_SOURCES=true` antes de compilar.
