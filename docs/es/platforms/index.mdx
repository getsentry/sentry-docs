Sentry ofrece observabilidad a nivel de código que facilita diagnosticar problemas y aprender continuamente sobre la salud del código de tu aplicación. Los datos se capturan mediante un SDK dentro del entorno de ejecución de tu aplicación. El equipo de Sentry crea y mantiene SDK para la mayoría de los lenguajes y frameworks populares, pero también existe un amplio ecosistema respaldado por la comunidad.

Si tu caso de uso es muy específico o no está cubierto por Sentry, encontrarás documentación sobre cómo reportar eventos por tu cuenta usando nuestra [API de SDK](https://develop.sentry.dev/sdk/overview/).

<Alert>
  Se considera una mala práctica usar los SDK de Sentry en plugins, paquetes y bibliotecas de terceros (como SDK de pagos o widgets embebibles) destinados a ser consumidos por otras apps. Si una app usa un SDK de Sentry además de un paquete de terceros con su propio SDK de Sentry instalado, pueden ocurrir los siguientes problemas:

  * Conflictos de versiones.
  * Los [manejadores de señales](https://en.wikipedia.org/wiki/C_signal_handling) pueden sobrescribirse entre sí, lo que puede provocar que uno de los SDK de Sentry funcione de forma incorrecta.
  * Los SDK y bibliotecas de terceros pueden filtrar datos PII de los usuarios a la organización de Sentry del tercero.

  También <Link to="https://sentry.zendesk.com/hc/en-us/articles/23961784367387">desaconsejamos usar Sentry junto con otras herramientas de APM</Link> porque puede provocar varios efectos secundarios que afectan negativamente el rendimiento y la estabilidad de la aplicación.
</Alert>
