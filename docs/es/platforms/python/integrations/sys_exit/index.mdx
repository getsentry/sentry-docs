La `SysExitIntegration` registra las llamadas a `sys.exit` al capturar la excepción `SystemExit` que provoca `sys.exit`.

<div id="install">
  ## Instalación
</div>

```bash {tabTitle:pip}
pip install "sentry-sdk"
```

```bash {tabTitle:uv}
uv add "sentry-sdk"
```


<div id="configure">
  ## Configurar
</div>

`SysExitIntegration` está deshabilitada de forma predeterminada y el SDK solo captura automáticamente las excepciones `SystemExit` si esta integración se habilita manualmente.

Una llamada a `sys.exit` se considera “correcta” cuando la función recibe un valor de `0` o `None`. Pasar cualquier otro valor da lugar a una salida “incorrecta”.

Para habilitar `SysExitIntegration` y configurarla para que solo capture las llamadas a `sys.exit` incorrectas, usa lo siguiente:

<SignInNote />

```python
import sentry_sdk
from sentry_sdk.integrations.sys_exit import SysExitIntegration

sentry_sdk.init(
    dsn="___PUBLIC_DSN___",
    # Agrega datos como encabezados de solicitud e IP de usuarios, si aplica;
    # consulta https://docs.sentry.io/platforms/python/data-management/data-collected/ para más información
    send_default_pii=True,
    integrations=[SysExitIntegration()],
)
```

Si quieres capturar llamadas exitosas y fallidas a `sys.exit`, usa lo siguiente en su lugar:

```python
import sentry_sdk
from sentry_sdk.integrations.sys_exit import SysExitIntegration

sentry_sdk.init(
    dsn="___PUBLIC_DSN___",
    integrations=[SysExitIntegration(capture_successful_exits=True)],
)
```


<div id="verify">
  ## Verificar
</div>

Al ejecutar el siguiente fragmento, debería informarse a Sentry una excepción `SystemExit`.

```python
import sys

import sentry_sdk
from sentry_sdk.integrations.sys_exit import SysExitIntegration

sentry_sdk.init(
    dsn="___PUBLIC_DSN___",
    integrations=[SysExitIntegration()],
)

sys.exit(1)
```

Si usas `capture_successful_exits=True`, al llamar a `sys.exit(0)` también se debería informar una excepción `SystemExit` a Sentry.

<Alert title="SystemExit generado manualmente">
  Ten en cuenta que `SysExitIntegration` solo captura las excepciones `SystemExit` que se generan al llamar a `sys.exit`. Si tu código genera `SystemExit` sin llamar a `sys.exit`, la excepción no se reportará a Sentry.
</Alert>
