El SDK de Sentry se integra en tu entorno de ejecución y reporta automáticamente errores, excepciones no controladas y rechazos no gestionados, además de otros tipos de errores según la plataforma.

<Alert>

Términos clave:

- Un _evento_ es un envío de datos a Sentry. Por lo general, se trata de un error o una excepción.
- Un _issue_ es un grupo de eventos similares.
- Al envío de un evento se le llama _captura_.
  Cuando se captura un evento, se envía a Sentry.

</Alert>

Las excepciones de Python se capturan en Sentry. Puedes omitir el argumento de <PlatformIdentifier name="capture-exception" /> y Sentry intentará capturar la excepción actual. También sirve para reportar manualmente excepciones o mensajes a Sentry.

Al capturar un evento, también puedes registrar las breadcrumbs que llevaron a ese evento. Las breadcrumbs son diferentes de los eventos: no crearán un evento en Sentry, sino que se almacenarán en búfer hasta que se envíe el siguiente. Obtén más información sobre las breadcrumbs en nuestra <PlatformLink to="/enriching-events/breadcrumbs/">documentación de Breadcrumbs</PlatformLink>.

  ## Captura de mensajes

Otra operación habitual es capturar un mensaje simple. Un mensaje es información textual que debería enviarse a Sentry. Por lo general, nuestros SDK no capturan mensajes automáticamente, pero puedes capturarlos manualmente.

Los mensajes aparecen como problemas en tu lista de incidencias, con el propio mensaje como nombre del problema.
