Todos los eventos tienen una huella digital. Los eventos con la misma huella se agrupan en un issue.

De forma predeterminada, Sentry ejecuta uno de nuestros algoritmos de agrupación integrados para generar una huella digital basada en la información disponible en el evento, como `stacktrace`, `exception` y `message`. Para ampliar el comportamiento de agrupación predeterminado o cambiarlo por completo, puedes usar una combinación de las siguientes opciones:

1. En tu SDK, usando la huella digital del SDK, como se detalla a continuación
2. En tu proyecto, usando [Fingerprint Rules](/es/concepts/data-management/event-grouping/fingerprint-rules/) o [Stack Trace Rules](/es/concepts/data-management/event-grouping/stack-trace-rules/)

Puedes anular la agrupación predeterminada de Sentry pasando el atributo de huella digital como un arreglo de cadenas. La longitud del arreglo de una huella no está restringida. Esto funciona de manera similar a la [funcionalidad de fingerprint rules](/es/concepts/data-management/event-grouping/fingerprint-rules/), que puede lograr resultados similares.
