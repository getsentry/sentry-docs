<Alert level="warning" title="Aviso de obsolescencia">
  Un nuevo SDK de Python ha reemplazado esta versión obsoleta. Sentry conserva esta documentación para clientes que usan el cliente antiguo. Recomendamos usar el [SDK de Python actualizado](/es/platforms/python/) para proyectos nuevos.
</Alert>

Para integrar Sentry con Python, puedes usar la biblioteca Raven for Python (raven-python). Es el cliente oficial de Python independiente para Sentry. Puede utilizarse con cualquier intérprete moderno de Python, ya sea CPython 2.x o 3.x, PyPy o Jython. Es un proyecto de código abierto y está disponible bajo una licencia BSD muy permisiva.

<div id="installation">
  ## Instalación
</div>

Si aún no lo has hecho, empieza por descargar Raven. La forma más fácil es con *pip*:

```bash
pip install raven --upgrade
```

<div id="configuring-the-client">
  ## Configuración del cliente
</div>

La configuración se define durante la inicialización del cliente. El cliente es una clase que puede instanciarse con una configuración específica y, a partir de entonces, todos los reportes pueden realizarse desde esa instancia. Por lo general, se crea una instancia de forma global y luego se importa según sea necesario. Para empezar, lo único que necesitas es tu DSN:

```python
from raven import Client
client = Client('___DSN___')
```

<div id="capture-an-error">
  ## Capturar un error
</div>

El uso más básico de raven es registrar un error específico que ocurre:

```python
from raven import Client

cliente = Client('___DSN___')

try:
    1 / 0
except ZeroDivisionError:
    client.captureException()
```

<div id="adding-context">
  ## Añadir contexto
</div>

Gran parte de la utilidad de Sentry proviene de los datos de contexto adicionales que acompañan a los eventos. El cliente de Python lo facilita proporcionando métodos para establecer datos de contexto locales al hilo, que luego se envían automáticamente con todos los eventos. Por ejemplo, puedes usar [`user_context()`](/es/platforms/python/legacy-sdk/api/#raven.Client.user_context "raven.Client.user_context") para definir la información del usuario actual:

```python
def manejar_solicitud(request):
    cliente.contexto_de_usuario({
        'email': request.user.email
    })
```
