<div id="what-is-distributed-tracing">
  ## ¿Qué es el trazado distribuido?
</div>

En el contexto de observar eventos en un sistema distribuido, el trazado distribuido es una potente herramienta de depuración. Imagina tu aplicación como una gran red de partes interconectadas. Por ejemplo, tu sistema puede estar desplegado en distintos servidores o tu aplicación puede dividirse en servicios de backend y frontend, cada uno con su propio stack tecnológico.

Cuando ocurre un error o un problema de rendimiento, puede ser difícil identificar la causa de origen debido a la complejidad de un sistema así. El trazado distribuido te ayuda a seguir el recorrido de un evento mientras atraviesa esta red compleja, registrando cada paso que da. Al examinar estos trazos, puedes reconstruir la secuencia de eventos que conduce al evento de interés, identificar los componentes específicos involucrados y comprender sus interacciones. Esta visibilidad detallada te permite diagnosticar y resolver problemas de forma más eficaz, mejorando en última instancia la fiabilidad y el rendimiento de tu sistema distribuido.

Si ejecutas aplicaciones de JavaScript en tu sistema distribuido, asegúrate de que esos dos encabezados estén incluidos en tu lista de permitidos de CORS y de que no los bloqueen ni los eliminen tus servidores proxy, puertas de enlace o cortafuegos.

<div id="how-to-use-distributed-tracing">
  ## ¿Cómo usar el rastreo distribuido?
</div>

<PlatformContent includePath="distributed-tracing/how-to-use/" />

Recuerda que, para propagar la información de trazas en todo tu sistema distribuido, debes usar Sentry en todos los servicios y aplicaciones implicados. Consulta la documentación del SDK correspondiente para saber cómo habilitar el rastreo distribuido en cada plataforma.
