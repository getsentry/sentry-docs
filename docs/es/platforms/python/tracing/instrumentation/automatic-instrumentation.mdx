Muchas integraciones para frameworks populares capturan transacciones automáticamente. Si ya tienes configurado cualquiera de los siguientes frameworks para los reportes de errores de Sentry, empezarás a ver trazas de inmediato:

- Todos los frameworks web basados en WSGI (Django, Flask, Pyramid, Falcon, Bottle)
- Celery
- Aplicaciones web de AIOHTTP
- Redis Queue (RQ)

Se instrumentan spans para las siguientes operaciones dentro de una transacción:

- Consultas a bases de datos que usan SQLAlchemy o el ORM de Django
- Solicitudes HTTP realizadas con HTTPX, Requests o la biblioteca estándar
- Subprocesos creados
- Operaciones de Redis

Los spans solo se crean dentro de una transacción existente. Si no estás usando ninguno de los frameworks compatibles, deberás <PlatformLink to="/tracing/instrumentation/custom-instrumentation/">crear transacciones manualmente</PlatformLink>.
