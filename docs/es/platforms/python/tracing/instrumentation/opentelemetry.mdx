<div id="opentelemetry-and-sentry">
  ## OpenTelemetry y Sentry
</div>

Con el SDK de OpenTelemetry de Sentry, un `Span` de OpenTelemetry se convierte en una `Transaction` o un `Span` de Sentry. El primer `Span` que pasa por el `SpanProcessor` de Sentry se considera una `Transaction`, y cualquier `Span` hijo se asocia a esa primera `Transaction` al comprobar el contexto del `Span` padre. Esto vale tanto para el `Span` raíz de OpenTelemetry como para cualquier `Span` de nivel superior del sistema. Por ejemplo, una solicitud enviada del frontend al backend creará un `Span` raíz de OpenTelemetry con su correspondiente `Transaction` de Sentry. La solicitud en el backend creará una nueva `Transaction` de Sentry para el `Span` de OpenTelemetry. La `Transaction` y el `Span` de Sentry están vinculados como un rastro para facilitar la navegación y el seguimiento de errores.
