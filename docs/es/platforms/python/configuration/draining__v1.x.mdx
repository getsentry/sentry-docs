---
title: Apagado y vaciado
sidebar_order: 80
description: "Obtén más información sobre el comportamiento predeterminado de nuestro SDK si la aplicación se cierra inesperadamente."
---

De forma predeterminada, el SDK envía eventos por la red en un hilo en segundo plano. Esto significa que algunos eventos podrían perderse si la aplicación se cierra inesperadamente. El SDK ofrece mecanismos para manejarlo.

El SDK de Python se vacía automáticamente al apagarse, a menos que se elimine la `AtexitIntegration` o la clave de configuración `shutdown_timeout`
se establezca en 0. Para vaciar manualmente, el cliente proporciona un método `close`:

```python
from sentry_sdk import Hub

client = Hub.current.client
if client is not None:
    client.close(timeout=2.0)
```

Después de llamar a `close`, ya no se podrá usar el cliente. Es importante
invocar `close` únicamente justo antes de cerrar la aplicación.

Como alternativa, el método `flush` vacía la cola de eventos mientras mantiene
el cliente habilitado para seguir utilizándolo.
