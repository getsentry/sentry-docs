<div id="structured-context">
  ## Contexto estructurado
</div>

La mejor manera de adjuntar datos personalizados es con un contexto estructurado. Un contexto siempre debe ser un objeto y sus valores pueden ser arbitrarios.

Luego, usa <PlatformIdentifier name="set-context" /> y asígnale al contexto un nombre único:

<PlatformContent includePath="enriching-events/set-context" />

No hay restricciones para el nombre del contexto. En el objeto de contexto, todas las claves están permitidas excepto `type`, que se utiliza internamente.

Obtén más información sobre las convenciones para contextos comunes en la [documentación para desarrolladores de la interfaz de contexts](https://develop.sentry.dev/sdk/data-model/event-payloads/contexts/).

<div id="size-limitations">
  ## Limitaciones de tamaño
</div>

Al enviar contexto, _ten en cuenta los límites de tamaño del payload_. Sentry no recomienda enviar todo el estado de la aplicación ni blobs de datos grandes en los contextos. Si superas el tamaño máximo del payload, Sentry responderá con el error HTTP `413 Payload Too Large` y rechazará el evento.

El SDK de Sentry hará todo lo posible por adaptarse a los datos que envíes y recortar payloads de contexto grandes. El SDK puede truncar partes del evento; para más detalles, consulta la [documentación para desarrolladores sobre el manejo de datos del SDK](https://develop.sentry.dev/sdk/expected-features/data-handling/).

<div id="additional-data">
  ## Datos adicionales
</div>

**Los datos adicionales están obsoletos** en favor de contextos estructurados.

Sentry antes permitía agregar “datos adicionales” no estructurados mediante <PlatformIdentifier name="set-extra" />.
