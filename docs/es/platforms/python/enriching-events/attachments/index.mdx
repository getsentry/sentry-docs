Sentry puede enriquecer tus eventos para facilitar su investigación almacenando archivos adicionales —como archivos de configuración o de registro— como adjuntos.

<div id="uploading-attachments">
  ## Cargar archivos adjuntos
</div>

<PlatformContent includePath="enriching-events/add-attachment" />

<Alert>
  Sentry permite como máximo 20 MB por solicitud comprimida y hasta 100 MB de archivos adjuntos sin comprimir por evento, incluido el archivo del informe de fallos (si corresponde). Las cargas que superen estos límites se rechazan con el error HTTP `413 Payload Too Large` y los datos se descartan de inmediato. Para agregar archivos más grandes o más archivos, considere opciones de almacenamiento secundario.
</Alert>

Los archivos adjuntos se conservan durante 30 días; si supera el almacenamiento total incluido en su cuota, los archivos adjuntos no se almacenarán. Puede eliminar archivos adjuntos o los eventos que los contienen en cualquier momento. Eliminar un archivo adjunto no afecta su cuota: Sentry cuenta un archivo adjunto para su cuota en cuanto se almacena.

Obtenga más información sobre cómo los archivos adjuntos afectan su [cuota](/es/pricing/quotas/).

<div id="add-or-modify-attachments-before-sending">
  ## Agregar o modificar archivos adjuntos antes de enviar
</div>

Es posible agregar, eliminar o modificar archivos adjuntos antes de que se envíe un evento mediante
el hook <PlatformLink to="/configuration/options/#before-send"><PlatformIdentifier name="before-send" /></PlatformLink>
o un <PlatformLink to="/enriching-events/event-processors/">procesador de eventos</PlatformLink>.

```python
from sentry_sdk.scope import add_global_event_processor
from sentry_sdk.types import Event, Hint
from sentry_sdk.attachments import Attachment

@add_global_event_processor
def event_processor(event: Event, hint: Hint):
    hint["attachments"] = [Attachment(bytes=b"Hello World!", filename="attachment.txt")]
    return event
```

<div id="access-to-attachments">
  ### Acceso a adjuntos
</div>

Para limitar el acceso a los adjuntos, ve a la **Configuración general** de tu organización y selecciona el menú desplegable *Acceso a adjuntos* para configurar el acceso adecuado: cualquier miembro de tu organización, el propietario de facturación de la organización, miembro, administrador, manager o propietario.

<Include name="common-imgs/attachments-access" />

De forma predeterminada, el acceso se concede a todos los miembros cuando el almacenamiento está habilitado. Si un miembro no tiene acceso al proyecto, no podrá descargar un adjunto; el Button aparecerá deshabilitado (en gris) en Sentry. El miembro solo podrá ver que hay un adjunto almacenado.

<div id="viewing-attachments">
  ## Visualización de archivos adjuntos
</div>

Los archivos adjuntos se muestran en la parte inferior de la página de **Detalles del issue** del evento que se está visualizando.

<Include name="common-imgs/attachments-access-denied" />

Además, los archivos adjuntos también aparecen en la pestaña *Attachments* de la página de **Detalles del issue**, donde puedes ver el *Type* del adjunto, así como los eventos asociados. Haz clic en el ID del evento para abrir los **Detalles del issue** de ese evento específico.
