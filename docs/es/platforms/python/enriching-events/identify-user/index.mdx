Los usuarios se definen por algunas piezas clave de información que conforman una identidad única en Sentry. Cada una es opcional, pero **debe** haber al menos una presente para que el SDK de Sentry pueda capturar al usuario:

<DefinitionList>
  ### `id`

  Tu identificador interno del usuario.

  ### `username`

  El nombre de usuario. Suele usarse como una mejor etiqueta que el id interno.

  ### `email`

  Una alternativa o complemento al nombre de usuario. Sentry reconoce las direcciones de correo electrónico y puede mostrar elementos como Gravatar y habilitar capacidades de mensajería.

  ### `ip_address`

  La dirección IP del usuario. Si el usuario no está autenticado, Sentry usa la dirección IP como identificador único.
  El SDK intentará obtener la dirección IP de los datos de la solicitud HTTP en las solicitudes entrantes (campo `request.env.REMOTE_ADDR` en JSON), si está disponible. Esto requiere <PlatformIdentifier name="send-default-pii" /> configurado en `true` en las opciones del SDK.

  Si la `ip_address` del usuario está establecida en `"{{auto}}"`, Sentry inferirá la dirección IP a partir de la conexión entre tu aplicación y el servidor de Sentry. Si se omite el campo, el valor predeterminado es `None`.

  Para dejar de almacenar direcciones IP de usuarios en los datos de tus eventos, ve a la configuración del proyecto, haz clic en &quot;Security &amp; Privacy&quot; y habilita &quot;Prevent Storing of IP Addresses&quot;, o usa el [borrado de datos del lado del servidor](/es/security-legal-pii/scrubbing/) de Sentry para eliminar `$user.ip_address`. Agregar esta regla finalmente anula cualquier otra lógica.
</DefinitionList>

Además, puedes proporcionar pares clave/valor arbitrarios más allá de los nombres reservados, y el SDK de Sentry los almacenará junto con el usuario.
