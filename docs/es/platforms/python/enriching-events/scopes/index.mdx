Los ámbitos almacenan datos adicionales que el SDK añade a tu evento cuando lo envía a Sentry. Aunque los SDK suelen gestionar los ámbitos automáticamente, entender cómo funcionan y cómo puedes administrarlos manualmente puede ser útil.

<div id="what-is-a-scope">
  ## ¿Qué es un Scope?
</div>

Un scope gestiona los datos de un evento. Por ejemplo, el SDK almacena [contexts](../context/) y [breadcrumbs](../breadcrumbs/) en el scope.

Hay tres tipos de scopes. Exactamente un scope de cada tipo estará activo en un momento específico.

- *Global scope*: Un único scope compartido globalmente que almacena datos relevantes para toda la aplicación (como la versión).
- *Isolation scope*: Scope local al hilo creado para cada ciclo de vida de solicitud-respuesta para almacenar datos relevantes de la solicitud.
- *Current scope*: Scope local al hilo creado para cada span para almacenar datos relevantes del span.

El SDK y las integraciones incorporadas del SDK administran automáticamente los scopes. Por ejemplo, las integraciones de frameworks web crean un isolation scope para cada solicitud atendida. Cuando llamas a una función de API de nivel superior, como <PlatformIdentifier name="set-tag" />, el SDK determina el scope correcto en el que operar.

Al enviar un evento a Sentry, los datos finales aplicados al evento son el resultado de combinar los tres scopes, aplicando los datos de cada uno en turno. Primero se aplica el global scope, seguido por el isolation scope y luego el current scope.

<div id="changing-the-scope">
  ## Cambiar el alcance
</div>

Por lo general, recomendamos usar la API de nivel superior para gestionar los alcances, ya que la gestión automática de alcances del SDK cubre la mayoría de los casos de uso.

Sin embargo, si tu caso de uso requiere acceso directo al objeto de alcance, puedes usar el administrador de contexto <PlatformIdentifier name="new-scope" />. <PlatformIdentifier name="new-scope" /> bifurca el alcance actual, te permite modificar el nuevo alcance mientras el administrador de contexto está activo y luego restaura el alcance original. Usar <PlatformIdentifier name="new-scope" /> te permite enviar datos solo para un evento específico, como [modificar el contexto](../context/). Es aproximadamente equivalente al administrador de contexto <PlatformIdentifier name="push-scope" /> en versiones anteriores (1.x) del SDK.

<Alert>

Evita llamar a la API de nivel superior dentro del administrador de contexto <PlatformIdentifier name="new-scope" />. La API de nivel superior podría interactuar con un alcance distinto del que produce <PlatformIdentifier name="new-scope" />, lo que podría causar resultados no deseados. Mientras estés dentro del administrador de contexto <PlatformIdentifier name="new-scope" />, llama a los métodos directamente sobre el alcance que produce <PlatformIdentifier name="new-scope" />.

</Alert>

Usar <PlatformIdentifier name="new-scope" /> te permite adjuntar información adicional, como agregar etiquetas personalizadas.
