El SDK de Sentry se integra con tu entorno de ejecución y reporta automáticamente errores, excepciones no controladas y rechazos no gestionados, además de otros tipos de errores según la plataforma.

<Alert>

Términos clave:

- Un _evento_ es una instancia de envío de datos a Sentry. Generalmente, estos datos corresponden a un error o una excepción.
- Un _issue_ es un agrupamiento de eventos similares.
- El envío de un evento se denomina _captura_.
  Cuando se captura un evento, se envía a Sentry.

</Alert>

Al capturar un evento, también puedes registrar las migas de pan (breadcrumbs) que conducen a ese evento. Las breadcrumbs son diferentes de los eventos: no generan un evento en Sentry, sino que se almacenan en búfer hasta que se envíe el siguiente evento. Obtén más información sobre las breadcrumbs en nuestra <PlatformLink to="/enriching-events/breadcrumbs/">documentación de Breadcrumbs</PlatformLink>.

<div id="capturing-messages">
  ## Captura de mensajes
</div>

Otra operación común es capturar un mensaje sin contexto adicional. Un mensaje es información textual que debe enviarse a Sentry. Por lo general, nuestros SDK no capturan mensajes automáticamente, pero puedes capturarlos manualmente.

Los mensajes aparecen como problemas en tu flujo de issues, con el propio mensaje como nombre del problema.
