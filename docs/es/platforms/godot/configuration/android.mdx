Se requieren pasos adicionales para exportar tu juego a Android con la integración de Sentry.

<div id="setting-up-export-for-android">
  ## Configurar la exportación para Android
</div>

Si nunca has exportado para Android, sigue la [guía de exportación de Godot para Android](https://docs.godotengine.org/en/stable/tutorials/export/exporting_for_android.html) para configurar las rutas del Android SDK y del Java SDK en los Ajustes del editor de Godot.

<div id="step-1-install-gradle-build-template">
  ### Paso 1: Instalar la plantilla de compilación de Gradle
</div>

Exportar a Android con el SDK de Sentry requiere usar el sistema de compilación Gradle de Godot. De forma predeterminada, Godot no usa Gradle, por lo que es necesario habilitarlo.

Si aún no lo has hecho, primero instala la plantilla de compilación para Android. En el editor de Godot, ve a `Project -> Install Android Build Template...` y haz clic en “Install” en el cuadro de confirmación. Esto crea un proyecto de Android basado en Gradle en `res://android/build/` dentro del directorio de tu proyecto. El SDK de Sentry se integrará automáticamente con este proyecto de Gradle.

<div id="step-2-configure-android-export">
  ### Paso 2: Configurar la exportación para Android
</div>

1. **Crear un ajuste preestablecido de exportación**: Si no tienes uno, crea uno: ve a `Project -> Export...` y añade un nuevo ajuste preestablecido de exportación para Android.
2. **Habilitar la compilación con Gradle**: En el ajuste preestablecido de exportación, ve a `Options -> Gradle Build` y habilita la opción `Use Gradle Build`.
3. **Configurar la plantilla de código fuente** (opcional): Si estás usando <PlatformLink to="/configuration/stack-traces/">plantillas de exportación con símbolos de depuración</PlatformLink>, configura el campo **Android Source Template** para que apunte al archivo `android_source.zip` generado durante el proceso de compilación de la plantilla.
4. **Configurar otros ajustes**: Configura el nombre del paquete, los iconos, los permisos y otros ajustes específicos de Android según sea necesario.

![Android Export Settings](imgs/android_export.png)

<div id="step-3-export-your-project">
  ### Paso 3: Exporta tu proyecto
</div>

Una vez configurado, exporta tu proyecto con normalidad. Durante la exportación, Godot automáticamente:

- Generará nuevas plantillas usando el sistema de compilación Gradle
- Incluirá las dependencias del SDK de Sentry en la configuración de Gradle
- Compilará tu APK/AAB con la integración de Sentry

<div id="known-limitations">
  ## Limitaciones conocidas
</div>

Aunque el soporte para Android es amplio, existen algunas limitaciones actuales:

- Las capturas de pantalla automáticas en caso de fallos aún no están disponibles en Android. ([#238](https://github.com/getsentry/sentry-godot/issues/238))
- Los archivos adjuntos faltan en caso de fallos nativos en Android. ([#243](https://github.com/getsentry/sentry-godot/issues/243))

Estas limitaciones se abordarán en versiones futuras del SDK.
