<Alert>
  La tasa de sesiones sin fallos requiere [soporte nativo](/es/platforms/unity/native-support/). En las plataformas que aún no tienen soporte nativo, el SDK marca las sesiones que no terminaron correctamente como *anormales*, lo que provoca que se registren sesiones 100% sin fallos. Si estás desarrollando en una plataforma que aún no tiene soporte nativo, deberías monitorear la tasa *anormal* de tu app.
</Alert>

Un lanzamiento es una versión de tu código que se despliega en un <PlatformLink to="/configuration/environments/">entorno</PlatformLink>. Cuando le das a Sentry información sobre tus lanzamientos, puedes:

* Identificar problemas y regresiones introducidos en un nuevo lanzamiento
* Predecir qué commit causó un problema y quién probablemente es responsable
* Resolver problemas incluyendo el número del issue en tu mensaje de commit
* Recibir notificaciones por correo electrónico cuando se despliegue tu código

<div id="setting-a-release">
  ## Configurar una versión
</div>

<PlatformContent includePath="set-release" notateUnsupported />

Cómo poner el nombre de la versión (o release) a disposición de tu código depende de ti. Por ejemplo, puedes usar una variable de entorno que se defina durante el proceso de compilación o en el arranque inicial.

Definir el nombre de la versión etiqueta cada evento con ese nombre. Recomendamos informar a Sentry sobre una nueva versión antes de enviar eventos con ese nombre, ya que esto habilitará algunas funciones adicionales. Obtén más información en nuestra documentación de [Releases](/es/product/releases/).

Si no informas a Sentry sobre una nueva versión, Sentry creará automáticamente una entidad de versión en el sistema la primera vez que vea un evento con ese ID de versión.

Después de configurar tu SDK, puedes instalar una integración de repositorio o proporcionar manualmente a Sentry tus propios metadatos de commits. Consulta nuestra documentación sobre [configuración de versiones](/es/product/releases/setup/) para obtener más información sobre integraciones, asociación de commits y cómo informar a Sentry al implementar versiones.

<div id="release-health">
  ## Salud de la versión
</div>

Supervisa la [salud de las versiones](/es/product/releases/health/) observando la adopción por parte de los usuarios, el uso de la aplicación, el porcentaje de [cierres inesperados](/es/product/releases/health/#crashes) y los [datos de sesión](/es/product/releases/health/#sessions). La salud de la versión ofrece información sobre el impacto de los cierres inesperados y los errores en la experiencia del usuario, y revela tendencias con cada nueva incidencia a través de los gráficos y filtros de [Release Details](/es/product/releases/release-details/).

Para poder supervisar la salud de la versión, el SDK envía datos de sesión.

<div id="sessions">
  ### Sesiones
</div>

Una sesión representa la interacción entre el usuario y la aplicación. Las sesiones incluyen una marca de tiempo, un estado (si la sesión fue correcta o si se bloqueó) y siempre están vinculadas a una versión. La mayoría de los SDK de Sentry pueden gestionar las sesiones automáticamente.

<PlatformContent includePath="configuration/auto-session-tracking" />

<Alert>
  El monitoreo de salud de versiones para .NET actualmente está limitado a aplicaciones **cliente** de una sola instancia. Esto significa que puede usarse con modelos de aplicación como WPF, Xamarin, WinForms y similares.
  El soporte para frameworks de servidor como ASP.NET está planificado.
</Alert>
