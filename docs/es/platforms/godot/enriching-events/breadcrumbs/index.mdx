Sentry utiliza *migas de pan* para crear un rastro de eventos ocurridos antes de un problema. Estos eventos son muy similares a los registros tradicionales, pero pueden capturar datos estructurados más ricos.

Esta página ofrece una vista general del registro manual de migas de pan y su personalización. Obtén más información sobre lo que se muestra en la página **Detalles del problema** y cómo filtrar las migas de pan para resolver problemas rápidamente en [Uso de migas de pan](/es/product/error-monitoring/breadcrumbs).

<Alert title="Conoce el uso del SDK">
  Los desarrolladores que quieran modificar la interfaz de migas de pan pueden consultar nuestra [documentación para desarrolladores sobre la interfaz de migas de pan](https://develop.sentry.dev/sdk/data-model/event-payloads/breadcrumbs/).
</Alert>

<div id="manual-breadcrumbs">
  ## Breadcrumbs manuales
</div>

Puedes añadir breadcrumbs manualmente cuando suceda algo relevante. Por ejemplo, podrías registrar un breadcrumb si el usuario se autentica o se produce otro cambio de estado.

Registrar un breadcrumb manualmente:

<PlatformContent includePath="enriching-events/breadcrumbs/breadcrumbs-example" />

Las claves de breadcrumb disponibles son `type`, `category`, `message`, `level`, `timestamp` (que muchos SDK establecerán automáticamente por ti) y `data`, que es donde poner cualquier información adicional que quieras que el breadcrumb incluya. Usar claves distintas de estas seis no provocará un error, pero hará que los datos se descarten cuando Sentry procese el evento.
