Sentry puede enriquecer tus eventos para facilitar su investigación almacenando archivos adicionales, como archivos de configuración o de registro, como adjuntos.

<div id="creating-attachments">
  ## Creación de adjuntos
</div>

La forma más sencilla de crear un adjunto es usar una `path`. El SDK leerá el contenido del archivo cada vez que prepare un evento o transacción y luego añadirá el adjunto al mismo [envelope](https://develop.sentry.dev/sdk/data-model/envelopes/). Si el SDK no puede leer el archivo, registrará un mensaje de error y descartará el adjunto.

<PlatformContent includePath="enriching-events/attachment-init-with-path" />

Como alternativa, usa `bytes` para inicializar un adjunto. Al hacerlo, también debes especificar un nombre de archivo.

<PlatformContent includePath="enriching-events/attachment-init-with-bytes" />

<Alert title="Nota">
  Si tu SDK admite caché sin conexión, algo habitual en dispositivos móviles, cada adjunto se guarda en disco por cada evento o transacción que captures cuando el dispositivo esté sin conexión. Al usar adjuntos grandes, este almacenamiento puede consumir el espacio en disco si el dispositivo permanece sin conexión durante un período prolongado.
</Alert>

Además, puedes establecer estos parámetros:

`filename`

El nombre de archivo es el nombre que se mostrará en Sentry. Al usar bytes debes especificar un nombre de archivo, mientras que con una ruta no, ya que el SDK usará el último componente de la ruta.

`content_type`

El tipo de contenido multimedia específico que determina cómo se muestra el adjunto en la interfaz de Sentry. Se puede usar cualquier [tipo MIME](https://www.iana.org/assignments/media-types/media-types.xhtml); el valor predeterminado es `application/octet-stream`.

Actualmente admitimos y podemos mostrar los siguientes tipos MIME:

* `text/plain`
* `text/css`
* `text/csv`
* `text/html`
* `text/javascript`
* `text/json` o `text/x-json` o `application/json` o `application/ld+json`
* `image/jpeg`
* `image/png`
* `image/gif`

<div id="uploading-attachments">
  ## Carga de adjuntos
</div>

Puedes añadir adjuntos que se enviarán con cada evento usando <PlatformIdentifier name="SentrySDK.add-attachment()" />.

<PlatformContent includePath="enriching-events/attachment-upload" />

<Alert>
  Sentry permite como máximo 20 MB por solicitud comprimida y hasta 100 MB de adjuntos sin comprimir por evento, incluido el archivo de informe de fallo (si corresponde). Las cargas que superen este tamaño se rechazarán con el error HTTP `413 Payload Too Large` y los datos se descartarán de inmediato. Para añadir archivos más grandes o más archivos, considera opciones de almacenamiento secundario.
</Alert>

Los adjuntos se conservan durante 30 días. Los adjuntos enviados recientemente no se almacenarán si has superado tu cuota total de almacenamiento. Puedes eliminar adjuntos o los eventos que los contienen en cualquier momento. Eliminar un adjunto no afecta a tu cuota: Sentry lo cuenta en tu cuota en cuanto se almacena.

Obtén más información sobre cómo los adjuntos afectan a tu [cuota](/es/pricing/quotas/).

<div id="access-to-attachments">
  ### Acceso a archivos adjuntos
</div>

Para limitar el acceso a los archivos adjuntos, ve a la **Configuración general** de tu organización y selecciona el menú desplegable *Acceso a archivos adjuntos* para definir los permisos adecuados: cualquier miembro de tu organización, el propietario de facturación, miembro, administrador, manager o propietario.

<Include name="common-imgs/attachments-access" />

De forma predeterminada, cuando el almacenamiento está habilitado, el acceso se concede a todos los miembros. Si un miembro no tiene acceso al proyecto, no podrá descargar un archivo adjunto; el botón aparecerá en gris en Sentry. El miembro solo podrá ver que hay un archivo adjunto almacenado.

<div id="viewing-attachments">
  ## Visualización de archivos adjuntos
</div>

Los archivos adjuntos se muestran en la parte inferior de la página **Issue Details** del evento que se está viendo.

<Include name="common-imgs/attachments-access-denied" />

También puedes encontrarlos en la pestaña *Attachments* de la página **Issue Details**, donde puedes ver el *Type* del archivo adjunto, así como los eventos asociados. Haz clic en el ID del evento para abrir los **Issue Details** de ese evento específico.
