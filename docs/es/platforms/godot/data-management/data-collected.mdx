---
title: Datos recopilados
description: "Consulta qué datos recopila el SDK de Sentry."
sidebar_order: 1
---

Sentry se toma muy en serio la privacidad de los datos y cuenta con ajustes predeterminados que priorizan la seguridad de la información, especialmente en lo referente a los datos de identificación personal (PII). Al agregar el SDK de Sentry a tu aplicación, permites que recopile datos y los envíe a Sentry durante el tiempo de ejecución de tu aplicación.

Los tipos de categorías y la cantidad de datos recopilados varían según las integraciones que tengas habilitadas en el SDK de Sentry. A continuación encontrarás una lista de las categorías de datos que recopila el SDK nativo de Sentry:

<div id="user-information">
  ## Información del usuario
</div>

De forma predeterminada, el SDK de Sentry no envía información sobre los usuarios, como la dirección de correo electrónico, el ID de usuario o el nombre de usuario. Sin embargo, si la <PlatformLink to="/configuration/options/#send_default_pii">opción `send_default_pii`</PlatformLink> está habilitada, los servicios de backend de Sentry inferirán la dirección IP del usuario a partir de la solicitud entrante.

También puedes establecer la información del usuario desde el código:

```GDScript
var user := SentryUser.new()
user.id = "id_personalizado"
user.email = "bob@ejemplo.com"
user.username = "bob"
user.ip_address = "127.0.0.1"
SentrySDK.set_user(user)
```


<div id="device-information">
  ## Información del dispositivo
</div>

El SDK de Sentry recopila información sobre el dispositivo, como el nombre, la versión y la compilación de tu sistema operativo o distribución de Linux. Esta información se envía a Sentry de forma predeterminada.

<div id="screenshots">
  ## Capturas de pantalla
</div>

La <PlatformLink to="/enriching-events/screenshots">funcionalidad de capturas de pantalla</PlatformLink> está desactivada de forma predeterminada. Si decides habilitarla en las opciones, cualquier captura realizada puede contener datos sensibles visibles en la aplicación en el momento del error.

<div id="godot-log-file">
  ## Archivo de registro de Godot
</div>

De forma predeterminada, el SDK de Sentry adjunta automáticamente el archivo de registro de Godot a los eventos que envía. Dado que este archivo contiene todos los mensajes que se imprimen en tu código y por el propio motor, podría incluir inadvertidamente información sensible. Para mejorar la protección de la privacidad, puedes desactivar la opción <PlatformLink to="/configuration/options/#attach_log">`attach_log`</PlatformLink> en la configuración del proyecto de Godot.

<div id="thread-stack-information">
  ## Información de la pila de hilos
</div>

En el momento del fallo, la pila de cada hilo se recopila y se envía a Sentry como parte de la instantánea de Minidump para el backend `crashpad`. Esta información se envía a Sentry de forma predeterminada, pero se descarta después de procesar el evento en el backend.

Estos archivos no se almacenan por defecto, pero puedes <PlatformLink to="/data-management/store-minidumps-as-attachments/">habilitar el almacenamiento de Minidumps</PlatformLink> en la configuración de la organización o del proyecto en Sentry.
