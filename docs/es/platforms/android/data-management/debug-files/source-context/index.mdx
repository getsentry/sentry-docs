Si Sentry tiene acceso al código fuente de tu aplicación, puede mostrar fragmentos de código (*contexto de código fuente*) alrededor de la ubicación de los frames de la pila, lo que ayuda a identificar rápidamente el código problemático.

Por ejemplo, aquí tienes un rastro de pila de una aplicación .NET. El contexto de código fuente contiene
el código C# original de la aplicación, con la ubicación del frame resaltada.

![Captura de pantalla de un rastro de pila que muestra el contexto de código fuente.](./img/source-context-screenshot.png)

<div id="source-bundles">
  ### Paquetes de código fuente
</div>

Algunos SDK, como el SDK de Python, pueden resolver este contexto de origen automáticamente,
porque tienen acceso a código fuente sin ofuscar en tiempo de ejecución.
En cambio, para obtener contexto de origen en aplicaciones compiladas, es necesario cargar el código fuente
junto con los archivos de información de depuración.
La forma recomendada de hacerlo es usando `sentry-cli`.
Consulta [Creación de paquetes de código fuente](/es/cli/dif/#creating-source-bundles) para más información.

<Alert>
  Para Android, sigue las instrucciones de la guía [Contexto de código fuente en Android](/es/platforms/android/enhance-errors/source-context).
</Alert>

Después de cargarlos, puedes revisar y administrar los paquetes de código fuente igual que
cualquier otro archivo de información de depuración. Consulta [Administrar archivos de información de depuración](../#managing-debug-information-files).
Los paquetes de código fuente se etiquetarán con `sources` y tendrán el mismo nombre de archivo que el
archivo de información de depuración correspondiente del que se crearon.
