El SDK de Android enriquece automáticamente el contexto con información adicional sobre el dispositivo, el sistema y la versión de lanzamiento de tu aplicación.

El contexto predeterminado de Android ayuda a correlacionar errores con factores como tipos específicos de dispositivos y versiones, para que puedas encontrar la causa raíz del problema más rápido.

SDK utilizado

- Nombre
- Versión

Sistema operativo

- Nombre
- Versión
- Versión del kernel
- Si la aplicación tenía acceso root

Información del paquete

- Hora de inicio de la aplicación
- Nombre del bundle
- ID del bundle
- Versión
- Compilación
- Si la aplicación se instaló por sideloading
- Permisos de la aplicación (cuáles se otorgaron y cuáles no)

Tamaño de la memoria

- Memoria total disponible en el dispositivo
- Memoria libre en ese momento
- Si la memoria estaba baja

Almacenamiento

- Tamaño total de almacenamiento
- Tamaño de almacenamiento libre
- Tamaño total de almacenamiento externo
- Tamaño de almacenamiento externo libre

Información del dispositivo

- Fabricante
- Marca
- Modelo
- ID del modelo
- Si el teléfono estaba en línea durante el error/fallo
- Arquitecturas compatibles
- Tiempo de arranque
- Tipo de conexión

Nivel de batería

- Si el dispositivo se está cargando en este momento
- Temperatura de la batería en grados Celsius

Información de la pantalla

- Orientación de la pantalla
- Densidad de la pantalla
- DPI de la pantalla
- Altura en píxeles
- Ancho en píxeles

Indicador de emulador

- Si el dispositivo es un emulador

Configuración regional

- Zona horaria (p. ej., Europe/Vienna)
- Configuración regional (p. ej., en_US)
