Los usuarios se definen por algunos datos clave que conforman una identidad única en Sentry. Cada uno es opcional, pero uno **debe** estar presente para que el SDK de Sentry capture al usuario:

<DefinitionList>
  ### `id`

  <Alert>
    Si no proporcionas un `id`, el SDK utilizará `installationId`, que se genera aleatoriamente una sola vez durante la instalación de la app.
  </Alert>

  Tu identificador interno del usuario.

  ### `username`

  El nombre de usuario. Suele usarse como mejor etiqueta que el id interno.

  ### `email`

  Una alternativa, o un complemento, al nombre de usuario. Sentry reconoce las direcciones de correo electrónico y puede mostrar, por ejemplo, Gravatars y habilitar funciones de mensajería.

  ### `ip_address`

  La dirección IP del usuario. Si el usuario no está autenticado, Sentry utiliza la dirección IP como identificador único del usuario.
  Los SDK del lado del servidor que instrumentan solicitudes entrantes intentarán extraer la dirección IP de los datos de la solicitud HTTP (campo `request.env.REMOTE_ADDR` en JSON), si está disponible. Esto podría requerir <PlatformIdentifier name="send-default-pii" /> establecido en `true` en las opciones del SDK.

  Si el `ip_address` del usuario está configurado como `"{{auto}}"`, Sentry inferirá la dirección IP a partir de la conexión entre tu app y el servidor de Sentry.

  Si se omite el campo, el valor predeterminado es `null`. Sin embargo, por motivos de compatibilidad retroactiva, ciertas plataformas (en particular JavaScript) tienen un valor predeterminado distinto para `"{{auto}}"`. Los SDK y otros clientes no deben depender de este comportamiento y deben establecer direcciones IP o `"{{auto}}"` explícitamente.

  Para dejar de almacenar las direcciones IP de los usuarios en los datos de tus eventos, ve a la configuración de tu proyecto, haz clic en &quot;Security &amp; Privacy&quot; y habilita &quot;Prevent Storing of IP Addresses&quot;, o usa el [scrubbing de datos del lado del servidor](/es/security-legal-pii/scrubbing/) de Sentry para eliminar `$user.ip_address`. Agregar esa regla anula cualquier otra lógica.
</DefinitionList>

Además, puedes proporcionar pares arbitrarios clave/valor más allá de los nombres reservados, y el SDK de Sentry los almacenará junto con el usuario.
