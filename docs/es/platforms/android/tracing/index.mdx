Con [tracing](/es/product/insights/overview/), Sentry hace un seguimiento del rendimiento de tu software, midiendo métricas como el rendimiento (throughput) y la latencia, y mostrando el impacto de los errores en múltiples sistemas. Sentry captura trazas distribuidas compuestas por transacciones y spans, que miden servicios individuales y operaciones individuales dentro de esos servicios. Obtén más información sobre nuestro modelo en [trazado distribuido](/es/product/sentry-basics/tracing/distributed-tracing/).

<Alert>
  Si vas a adoptar tracing en un entorno de alto tráfico, te recomendamos hacer pruebas antes del despliegue para asegurarte de que las características de rendimiento de tu servicio cumplan con las expectativas.
</Alert>

<div id="configure">
  ## Configurar
</div>

Primero, habilita el trazado y configura la tasa de muestreo para las transacciones. Establece la tasa de muestreo de tus transacciones de alguna de estas formas:

* Configurando una tasa de muestreo uniforme para todas las transacciones usando la opción <PlatformIdentifier name="traces-sample-rate" /> en la configuración de tu SDK con un número entre `0` y `1`. (Por ejemplo, para enviar el 20 % de las transacciones, establece <PlatformIdentifier name="traces-sample-rate" /> en `0.2`).
* Controlando la tasa de muestreo según la propia transacción y el contexto en el que se captura, proporcionando una función a la opción de configuración <PlatformIdentifier name="traces-sampler" />.

Las dos opciones están pensadas para ser mutuamente excluyentes. Si configuras ambas, <PlatformIdentifier name="traces-sampler" /> tendrá prioridad.

Obtén más información sobre las <PlatformLink to="/configuration/options/#tracing-options">opciones</PlatformLink> de trazado, cómo usar la función <PlatformLink to="/configuration/sampling/#setting-a-sampling-function">tracesSampler</PlatformLink> o cómo <PlatformLink to="/configuration/sampling/#sampling-transaction-events">muestrear transacciones</PlatformLink>.

<div id="verify">
  ## Verificar
</div>

Comprueba que el tracing funcione correctamente usando nuestra <PlatformLink to="/tracing/instrumentation/automatic-instrumentation/">instrumentación automática</PlatformLink> o iniciando y finalizando una transacción con <PlatformLink to="/tracing/instrumentation/custom-instrumentation/">instrumentación personalizada</PlatformLink>.

Pon a prueba el tracing iniciando y finalizando una transacción; es algo que debes hacer para que las transacciones se envíen a Sentry. Aprende cómo en nuestro contenido de <PlatformLink to="/tracing/instrumentation/custom-instrumentation/">Instrumentación personalizada</PlatformLink>.

Mientras pruebas, establece <PlatformIdentifier name="traces-sample-rate" /> en `1.0`, ya que así te aseguras de que cada transacción se envíe a Sentry. Cuando termines, quizá quieras bajar el valor de <PlatformIdentifier name="traces-sample-rate" /> o pasar a usar <PlatformIdentifier name="traces-sampler" /> para muestrear y filtrar tus transacciones de forma selectiva según datos contextuales.
