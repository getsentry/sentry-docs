Si el panorama de aplicaciones que quieres observar con Sentry abarca más que un único servicio o aplicación, el trazado distribuido puede aportar mucho valor.

<div id="what-is-distributed-tracing">
  ## ¿Qué es el tracing distribuido?
</div>

En el contexto de rastrear eventos en un sistema distribuido, el tracing distribuido es una potente herramienta de depuración. Imagina tu aplicación como una gran red de partes interconectadas. Por ejemplo, tu sistema podría estar repartido en distintos servidores, o tu aplicación podría dividirse en servicios de backend y frontend, cada uno con su propia pila tecnológica.

Cuando ocurre un error o un problema de rendimiento, puede ser difícil identificar la causa raíz debido a la complejidad de un sistema así. El tracing distribuido te ayuda a seguir el recorrido de un evento a medida que atraviesa esta intrincada red, registrando cada paso que da. Al examinar estos trazos, puedes reconstruir la secuencia de eventos que llevó al evento de interés, identificar los componentes específicos involucrados y comprender sus interacciones. Esta visibilidad detallada te permite diagnosticar y resolver problemas con mayor eficacia, mejorando en última instancia la fiabilidad y el rendimiento de tu sistema distribuido.

  ## ¿Cómo usar el rastreo distribuido?
</div>

<PlatformContent includePath="distributed-tracing/how-to-use/" />

Recuerda que, para propagar la información de trazas en todo tu sistema distribuido, debes usar Sentry en todos los servicios y aplicaciones implicados. Consulta la documentación del SDK correspondiente para saber cómo habilitar el rastreo distribuido en cada plataforma.

<div id="tracepropagationtargets">
  ### tracePropagationTargets
</div>

El SDK de Android adjuntará los encabezados `sentry-trace` y `baggage` a todas las solicitudes salientes de forma predeterminada. Para limitar esto solo a URL específicas, agrega esas URL a la lista de permitidos `tracePropagationTargets` (puedes usar tanto cadenas como expresiones regulares). El SDK entonces solo adjuntará los encabezados `sentry-trace` y `baggage` a las solicitudes salientes que coincidan con la lista de permitidos.

<Alert>
  La opción `tracePropagationTargets` puede contener URL completas (p. ej., &quot;https://api.myproject.org/request/path&quot;), subcadenas (p. ej., &quot;api.myproject.org&quot;) o expresiones regulares (p. ej., &quot;https://[\w]*.myproject.org/.*&quot;).
</Alert>

Dado que el valor de `tracePropagationTargets` es &quot;.*&quot; de forma predeterminada, los encabezados de trazado se adjuntan a todas las solicitudes, a menos que se especifique lo contrario.

```java {tabTitle:Java}
SentryAndroid.init(context, options -> {
    final ArrayList<String> tracePropagationTargets = new ArrayList<>();
    tracePropagationTargets.add("https://api.miproyecto.org/");

    options.setDsn("___DSN_PUBLICO___");
    options.setTracePropagationTargets(tracePropagationTargets);
});
```

```kotlin {tabTitle:Kotlin}
SentryAndroid.init(context, { options ->
    options.setDsn("___PUBLIC_DSN___")
    options.setTracePropagationTargets(listOf(
        "https://api.myproject.org/"
    ))
})
```

<div id="w3c-traceparent-header-support">
  ## Compatibilidad con el encabezado W3C Traceparent
</div>

A partir de la versión 8.22.0, el SDK de Java también admite el encabezado HTTP W3C `traceparent` para solicitudes salientes. Este encabezado sigue la [especificación W3C Trace Context](https://develop.sentry.dev/sdk/telemetry/traces/distributed-tracing/#w3c-trace-context-header) y ofrece interoperabilidad con otros sistemas de trazabilidad que admiten este estándar.

Para habilitar la propagación del encabezado W3C `traceparent`, establece la opción [`propagateTraceparent`](/es/platforms/android/configuration/options/#propagateTraceparent) en `true`. Cuando está habilitado, el encabezado `traceparent` se propagará junto con los encabezados `sentry-trace` y `baggage` en las solicitudes HTTP salientes. Al igual que los otros encabezados, se adjunta a todas las solicitudes de forma predeterminada, o solo a aquellas que coincidan con tu configuración de `tracePropagationTargets` si has definido una.
