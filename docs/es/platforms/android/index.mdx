<div id="install">
  ## Instalar
</div>

Sentry captura datos mediante un SDK dentro del entorno de ejecución de tu aplicación. Estos SDK son específicos de cada plataforma y permiten que Sentry entienda en profundidad cómo funciona tu aplicación.

Recomendamos instalar el SDK con nuestro [Sentry Wizard](https://github.com/getsentry/sentry-wizard) ejecutando uno de los siguientes comandos en el directorio de tu proyecto:

```bash {tabTitle:brew}
brew install getsentry/tools/sentry-wizard && sentry-wizard -i android
```

```bash {tabTitle:npx}
npx @sentry/wizard@latest -i android
```

```bash {tabTitle:macOS (Intel/x64)}
downloadUrl="https://github.com/getsentry/sentry-wizard/releases/download/v5.1.0/sentry-wizard-darwin-x64"
curl -L $downloadUrl -o sentry-wizard
chmod +x sentry-wizard
./sentry-wizard -i android
```

```bash {tabTitle:macOS (Apple Silicon/arm64)}
downloadUrl="https://github.com/getsentry/sentry-wizard/releases/download/v5.1.0/sentry-wizard-darwin-arm64"
curl -L $downloadUrl -o sentry-wizard
chmod +x sentry-wizard
./sentry-wizard -i android
```

```bash {tabTitle:Linux (x64)}
downloadUrl="https://github.com/getsentry/sentry-wizard/releases/download/v5.1.0/sentry-wizard-linux-x64"
curl -L $downloadUrl -o sentry-wizard
chmod +x sentry-wizard
./sentry-wizard -i android
```

```bash {tabTitle:Linux (arm64)}
downloadUrl="https://github.com/getsentry/sentry-wizard/releases/download/v5.1.0/sentry-wizard-linux-arm64"
curl -L $downloadUrl -o sentry-wizard
chmod +x sentry-wizard
./sentry-wizard -i android
```

```powershell {tabTitle:Windows}
$downloadUrl = "https://github.com/getsentry/sentry-wizard/releases/download/v5.1.0/sentry-wizard-win-x64.exe"
Invoke-WebRequest $downloadUrl -OutFile sentry-wizard.exe
./sentry-wizard.exe -i android
```

Esto aplicará un parche a tu proyecto y configurará el SDK. Solo necesitas aplicar el parche una vez; luego puedes agregar los archivos parcheados a tu sistema de control de versiones.
Si lo prefieres, también puedes <PlatformLink to="/manual-setup/">configurar el SDK manualmente</PlatformLink> o seguir las instrucciones a continuación para adaptar la [configuración](#configure).

<Expandable title="El Sentry Wizard realizará las siguientes tareas">
  El asistente te pedirá que inicies sesión en Sentry. Luego hará automáticamente lo siguiente por ti:

  * Actualizar el archivo `build.gradle` de tu app con el plugin de Gradle de Sentry y configurarlo
  * Actualizar tu `AndroidManifest.xml` con la configuración predeterminada de Sentry
  * Crear `sentry.properties` con un token de autenticación para subir los mapeos de ProGuard (este archivo se agrega automáticamente a `.gitignore`)
  * Agregar un error de ejemplo a la actividad principal (Main Activity) de tu app para verificar tu configuración de Sentry
</Expandable>

<div id="configure">
  ## Configurar
</div>

La configuración se realiza mediante el archivo de la aplicación `AndroidManifest.xml`. Aquí tienes un ejemplo de configuración para empezar:

```xml {filename:AndroidManifest.xml}
<application>
  <!-- Obligatorio: establece el identificador de tu proyecto en sentry.io (DSN) -->
  <meta-data android:name="io.sentry.dsn" android:value="___PUBLIC_DSN___" />
  <!-- Agrega datos como encabezados de la solicitud, dirección IP del usuario y nombre del dispositivo; consulta https://docs.sentry.io/platforms/android/data-management/data-collected/ para más información -->
  <meta-data android:name="io.sentry.send-default-pii" android:value="true" />
  <!-- ___PRODUCT_OPTION_START___ performance -->
  <!-- Habilita la API de rendimiento estableciendo una tasa de muestreo; ajusta en el entorno de producción -->
  <meta-data android:name="io.sentry.traces.sample-rate" android:value="1.0" />
  <!-- ___PRODUCT_OPTION_END___ performance -->
  <!-- ___PRODUCT_OPTION_START___ profiling -->
  <!-- Habilita el perfilado; ajusta en el entorno de producción. Esto se evalúa solo una vez por sesión -->
  <meta-data android:name="io.sentry.traces.profiling.session-sample-rate" android:value="1.0" />
  <!-- Establece el ciclo de vida del perfilado; puede ser `manual` (controlado mediante `Sentry.startProfiler()` y `Sentry.stopProfiler()`) o `trace` (inicia y detiene automáticamente un perfil cuando una traza muestreada comienza y finaliza) -->
  <meta-data android:name="io.sentry.traces.profiling.lifecycle" android:value="trace" />
  <!-- Habilita el perfilado al iniciar la app -->
  <meta-data android:name="io.sentry.traces.profiling.start-on-app-start" android:value="true" />
  <!-- ___PRODUCT_OPTION_END___ profiling -->
  <!-- ___PRODUCT_OPTION_START___ logs -->
  <!-- Habilita el envío de logs a Sentry -->
  <meta-data android:name="io.sentry.logs.enabled" android:value="true" />
  <!-- ___PRODUCT_OPTION_END___ logs -->
  <!-- ___PRODUCT_OPTION_START___ session-replay -->
  <!-- Registra reproducciones de sesión para el 100% de los errores y el 10% de las sesiones -->
  <meta-data android:name="io.sentry.session-replay.on-error-sample-rate" android:value="1.0" />
  <meta-data android:name="io.sentry.session-replay.session-sample-rate" android:value="0.1" />
  <!-- ___PRODUCT_OPTION_END___ session-replay -->
</application>
```

<div id="verify">
  ## Verificar
</div>

Comprueba que tu aplicación esté enviando eventos a Sentry añadiendo el siguiente fragmento, que incluye un error intencional. Deberías ver el error reportado en Sentry en unos minutos.

```java
import androidx.appcompat.app.AppCompatActivity;
import android.os.Bundle;
import java.lang.Exception;
import io.sentry.Sentry;

public class MyActivity extends AppCompatActivity {
  protected void onCreate(Bundle savedInstanceState) {
    super.onCreate(savedInstanceState);
    try {
      throw new Exception("Esta es una prueba.");
    } catch (Exception e) {
      Sentry.captureException(e);
    }
  }
}
```

```kotlin
import androidx.appcompat.app.AppCompatActivity
import android.os.Bundle
import io.sentry.Sentry

class MyActivity : AppCompatActivity() {
  override fun onCreate(savedInstanceState: Bundle?) {
    super.onCreate(savedInstanceState)
    try {
      throw Exception("Esta es una prueba.")
    } catch (e: Exception) {
      Sentry.captureException(e)
    }
  }
}
```
