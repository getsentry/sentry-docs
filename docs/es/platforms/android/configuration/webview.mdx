---

Para supervisar errores y el rendimiento en las vistas web de aplicaciones de Android, debes configurar los SDK de Sentry tanto en la aplicación móvil como en los proyectos web que se ejecutan dentro de esas vistas web.

<div id="application-monitoring">
  ## Supervisión de la aplicación
</div>

El SDK de Sentry para Android no interactúa con las vistas web de la aplicación móvil. Para supervisar los errores y el rendimiento en la parte de Android de la aplicación, sigue las [instrucciones de configuración del SDK de Sentry para Android](/es/platforms/android/).

<div id="webview-monitoring">
  ## Supervisión de WebView
</div>

Para supervisar errores y el rendimiento en vistas web, elige el SDK de Sentry para JavaScript adecuado según el framework utilizado en el proyecto web.

* [Sentry React SDK](/es/platforms/javascript/guides/react/) (si el proyecto web usa React)
* [Sentry Angular SDK](/es/platforms/javascript/guides/angular/) (si el proyecto web usa Angular)
* [Otros SDK de Sentry para JavaScript](/es/platforms/javascript)

<div id="notes">
  ## Notas
</div>

* El SDK de JavaScript debe inicializarse cuando se cree la vista web y se cerrará cuando esta se cierre. Para monitorear todas las vistas web de la aplicación, inicializa el SDK de JavaScript cada vez que se cree una vista web.

* Si el proyecto web se accede de forma remota y no es exclusivo de la aplicación móvil (por ejemplo, si también funciona como una aplicación web independiente), recomendamos usar proyectos de Sentry separados para la aplicación móvil y para el proyecto web.
