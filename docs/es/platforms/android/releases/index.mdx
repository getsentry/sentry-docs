El SDK de Android utiliza los siguientes canales para publicar actualizaciones del SDK:

<div id="pre-release">
  ## `Pre-release`
</div>

Las pre-releases (alfa, beta, RC,…) se usan para cambios más grandes y potencialmente más importantes, como nuevas funciones o versiones mayores.

<div id="latest">
  ## `Latest`
</div>

Publicamos continuamente versiones mayor/menor/hotfix desde la rama `main`. Estas versiones pasan por todos nuestros filtros internos de calidad y son muy seguras de usar. Están pensadas para ser la opción predeterminada para la mayoría de los equipos.

<div id="stable">
  ## `Stable`
</div>

Las versiones pasan de `Latest` a `Stable` cuando se han usado en producción durante cierto tiempo y a determinada escala. Antes de considerar una versión como `Stable`, se evalúan factores como el tiempo que lleva disponible, el nivel de adopción y otras métricas de calidad y estabilidad. Este tipo de versiones tendrá la etiqueta `Stable` en la <Link to="https://github.com/getsentry/sentry-java/releases/">página de lanzamientos</Link>.

<div id="faq">
  ## Preguntas frecuentes
</div>

<div id="where-can-i-find-the-latest-stable-release-for-any-given-sdk">
  ### ¿Dónde puedo encontrar la última versión estable de un SDK en particular?
</div>

En la <Link to="https://github.com/getsentry/sentry-java/releases/">página de lanzamientos</Link> del repositorio de GitHub del SDK. Las versiones estables tendrán la etiqueta `Stable` en el nombre de la versión. También puedes usar el cuadro de búsqueda en la esquina superior derecha para encontrar las versiones más recientes.

<div id="does-this-mean-the-release-channels-are-not-stable">
  ### ¿Eso significa que los canales de lanzamiento no son estables?
</div>

No. La estabilidad y el rendimiento son las principales prioridades en cada versión. El estado de lanzamiento `Stable` indica que la versión se ha probado y ha demostrado funcionar en entornos reales y a gran escala.

<div id="how-often-do-we-mark-sdks-as-stable">
  ### ¿Con qué frecuencia marcamos los SDK como estables?
</div>

El ciclo de versiones estables depende de varios factores y puede variar, pero el objetivo es publicar versiones estables aproximadamente cada mes.

<div id="should-i-only-use-stable-releases">
  ### ¿Debería usar solo versiones estables?
</div>

Esta decisión es tuya. Las versiones estables se han probado en escenarios reales y a gran escala. Están diseñadas para ofrecer una experiencia fiable y coherente. Ten en cuenta que el canal de versiones `Stable` recibirá nuestras funciones más recientes más tarde que otros canales.

<div id="how-does-this-relate-to-the-stability-of-a-feature">
  ### ¿Cómo se relaciona esto con la estabilidad de una función?
</div>

Un estado de versión `Stable` no garantiza la estabilidad de una función del SDK ni de una función del producto. Una función nueva que aún sea experimental podría formar parte de una versión estable del SDK. Cualquier función experimental estará claramente indicada y se ofrecerá como opción voluntaria (opt‑in).
