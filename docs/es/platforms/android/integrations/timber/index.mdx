La biblioteca `sentry-android-timber` ofrece compatibilidad con [Timber](https://github.com/JakeWharton/timber) para Sentry mediante [Timber Tree](https://github.com/JakeWharton/timber/blob/10f0adce3921ad2929ddf2f3b7fecda2cf3148a5/timber/src/main/java/timber/log/Timber.kt#L20), que envía eventos y breadcrumbs a Sentry. Una vez configurada esta integración, puedes usar la API estática de Timber.

El código fuente está disponible [en GitHub](https://github.com/getsentry/sentry-java/tree/master/sentry-android-timber/src/main/java/io/sentry/android/timber).

<div id="auto-installation-with-the-sentry-android-gradle-plugin">
  ## Instalación automática con el complemento de Sentry para Gradle en Android
</div>

<div id="install">
  ### Instalación
</div>

A partir de la versión `3.1.0`, el complemento Sentry Android Gradle añadirá automáticamente la dependencia `sentry-android-timber`. El complemento solo añadirá la dependencia `sentry-android-timber` si se detecta una dependencia de `timber` en el classpath.

Añade el complemento Sentry Android Gradle en `build.gradle`:

```groovy
plugins {
  id "io.sentry.android.gradle" version "{{@inject packages.version('sentry.java.android.gradle-plugin', '3.0.0') }}"
}
```

```kotlin
plugins {
  id("io.sentry.android.gradle") version "{{@inject packages.version('sentry.java.android.gradle-plugin', '3.0.0') }}"
}
```

Luego, inicializa el [SDK de Android](/es/platforms/android/#configure).

El SDK de Android agrega automáticamente `SentryTimberIntegration` si la dependencia `sentry-android-timber` se encuentra en el classpath. La integración se agrega con `minEventLevel` establecido en `ERROR`, y `minBreadcrumbLevel` y `minLogsLevel` establecidos en `INFO`.

No obstante, puedes sobrescribir el comportamiento predeterminado agregando tu propia instancia de `SentryTimberIntegration`. Para ello, consulta la sección de [instalación manual](/es/platforms/android/integrations/timber/#configure) a continuación.

<div id="disable">
  ### Desactivar
</div>

Si quieres desactivar la integración de Timber (por ejemplo, para no exceder tu cuota de eventos), pero seguir usando otras funciones de instalación automática, elimina la dependencia `sentry-android-timber` de la configuración de la app en `app/build.gradle`:

```groovy
configurations.configureEach {
  exclude group: "io.sentry", module: "sentry-android-timber"
}
```

```kotlin
configurations.configureEach {
  exclude(group = "io.sentry", module = "sentry-android-timber")
}
```

<div id="manual-installation">
  ## Instalación manual
</div>

<div id="install">
  ### Instalación
</div>

Para agregar la integración de Timber, [inicializa manualmente](/es/platforms/android/configuration/manual-init/#manual-initialization) el SDK de Android y luego agrega la dependencia `sentry-android-timber`. Con Gradle:

```groovy
implementation 'io.sentry:sentry-android:{{@inject packages.version('sentry.java.android', '4.2.0') }}'
implementation 'io.sentry:sentry-android-timber:{{@inject packages.version('sentry.java.android.timber', '4.2.0') }}'
```

<div id="configure">
  ### Configurar
</div>

La configuración debe realizarse lo antes posible en el ciclo de vida de tu aplicación.

```kotlin
import io.sentry.android.core.SentryAndroid
import io.sentry.android.timber.SentryTimberIntegration
import timber.log.Timber
// import BuildConfig

SentryAndroid.init(this) { options ->
  if (!BuildConfig.DEBUG) {

    // valores por defecto:
    // minEventLevel = ERROR
    // minBreadcrumbLevel = INFO
    // minLogsLevel = SentryLogLevel.INFO,
    options.addIntegration(
      SentryTimberIntegration(
        minEventLevel = SentryLevel.ERROR,
        minBreadcrumbLevel = SentryLevel.INFO,
        minLogsLevel = SentryLogLevel.INFO
      )
    )
  } else {
    Timber.plant(Timber.DebugTree())
  }
}
```

```java
import io.sentry.android.core.SentryAndroid;
import io.sentry.android.timber.SentryTimberIntegration;
import timber.log.Timber
// import BuildConfig

SentryAndroid.init(this, options -> {
   if (!BuildConfig.DEBUG) {

     // default values:
     // valores por defecto:
     // minEventLevel = ERROR
     // minBreadcrumbLevel = INFO
     // minLogsLevel = SentryLogLevel.INFO,
         new SentryTimberIntegration(
             SentryLevel.ERROR,
             SentryLevel.INFO,
             SentryLogLevel.INFO
         )
     );
   } else {
     Timber.plant(new Timber.DebugTree());
   }
});
```

<div id="verify">
  ## Verificar
</div>

Este fragmento provoca un error intencional para que puedas comprobar que todo funciona en cuanto lo configures:

```kotlin
import androidx.appcompat.app.AppCompatActivity
import android.os.Bundle
import java.lang.Exception
import timber.log.Timber

class MyActivity : AppCompatActivity() {
  override fun onCreate(savedInstanceState: Bundle?) {
    super.onCreate(savedInstanceState)
    try {
      throw Exception("Esta es una prueba.")
    } catch (e: Exception) {
      Timber.e(e);
    }
  }
}
```

```java
import androidx.appcompat.app.AppCompatActivity;
import android.os.Bundle;
import java.lang.Exception;
import timber.log.Timber;

public class MyActivity extends AppCompatActivity {
  protected void onCreate(Bundle savedInstanceState) {
    super.onCreate(savedInstanceState);
    try {
      throw new Exception("Esta es una prueba.");
    } catch (Exception e) {
      Timber.e(e);
    }
  }
}
```

<div id="support-with-sentry-logs">
  ## Compatibilidad con Sentry Logs
</div>

<Alert>
  Los Sentry Logs para Timber son compatibles a partir de la versión `8.17.0` del Sentry Android SDK.
</Alert>

Los registros de Timber en o por encima de `minLogsLevel` capturados por Sentry se envían automáticamente como <PlatformLink to="/logs/">Sentry Logs</PlatformLink>, si está habilitado.
