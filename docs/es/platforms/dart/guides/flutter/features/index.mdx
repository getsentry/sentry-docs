El SDK de Flutter de Sentry permite reportar automáticamente errores y excepciones, e identifica problemas de rendimiento en tu aplicación. A continuación, una lista de las funcionalidades disponibles en este SDK.

**Características:**

* Bajo el capó, el SDK se basa en el <Link to="/platforms/dart">SDK de Dart de Sentry</Link>:
  * Necesitas como mínimo Dart `3.5.0` y Flutter `3.24.0`.
  * Este SDK incluye todas las <Link to="/platforms/dart/features">funcionalidades del SDK de Dart de Sentry</Link>.
* Seguimiento automático de errores de bloqueos nativos (en <Link to="/platforms/android">Android</Link> y <Link to="/platforms/apple/guides/ios">iOS</Link>), incluyendo:
  * Código en Java, Kotlin, C y C++ para Android.
  * ObjC, Swift y C para iOS.
* Detección automática de <Link to="/platforms/android/configuration/app-not-respond">ANR (Application Not Responding)</Link> en Android y de <Link to="/platforms/apple/guides/ios/configuration/app-hangs/">bloqueos de la app</Link> en iOS.
* Almacenamiento de eventos sin conexión.
* Eventos <PlatformLink to="/enriching-events/context">enriquecidos</PlatformLink> con datos de dispositivo.
* Migas de pan capturadas automáticamente:
  * mediante el <PlatformLink to="/enriching-events/breadcrumbs/#automatic-breadcrumbs">SDK de Flutter</PlatformLink>.
  * a través de los SDKs nativos <Link to="/platforms/android/enriching-events/breadcrumbs/#automatic-breadcrumbs">Automatic Breadcrumbs for Android</Link> y <Link to="/platforms/apple/enriching-events/breadcrumbs/#automatic-breadcrumbs">Automatic Breadcrumbs for iOS</Link>.
  * así como `http` con el <Link to="/platforms/dart/enriching-events/breadcrumbs/#automatic-breadcrumbs">SDK de Dart</Link>.
* Integraciones para sqflite, enrutamiento y más. Para consultar la lista completa, visita <PlatformLink to="/integrations">integraciones</PlatformLink>.
* <Link to="/product/releases/health">Salud de versiones</Link> hace un seguimiento de usuarios y sesiones sin fallos.
* <PlatformLink to="/enriching-events/attachments">Adjuntos</PlatformLink> que pueden enriquecer tu evento al almacenar archivos adicionales, como archivos de configuración o de registro.
* <PlatformLink to="/tracing">Trazas</PlatformLink> que pueden registrar:
  * <PlatformLink to="/integrations/app-start-instrumentation">Tiempo de inicio de la app</PlatformLink>.
  * <PlatformLink to="/integrations/routing-instrumentation/#time-to-initial-display">Tiempo hasta la visualización inicial y hasta la visualización completa</PlatformLink>.
  * <PlatformLink to="/integrations/slow-and-frozen-frames-instrumentation">Fotogramas lentos y congelados</PlatformLink>.
  * <PlatformLink to="/integrations/user-interaction-instrumentation">Interacción del usuario</PlatformLink>, que incluye clics, pulsaciones prolongadas, toques, etc.
  * Para ver la lista completa, consulta las <PlatformLink to="/tracing/instrumentation/automatic-instrumentation">instrumentaciones automáticas</PlatformLink>.
* <PlatformLink to="/user-feedback">Opiniones de los usuarios</PlatformLink>, que permite recopilar comentarios de los usuarios cuando ocurre un evento inesperado.
* <PlatformLink to="/enriching-events/screenshots">Captura de pantalla</PlatformLink> y <PlatformLink to="/enriching-events/viewhierarchy">Jerarquía de vistas</PlatformLink> adjuntas por errores.
* <PlatformLink to="/profiling">Profiling</PlatformLink> recopila información detallada sobre tu código a nivel de funciones.
  * Actualmente, el perfilado es compatible con **iOS** y **macOS**.
  * Recopila perfiles en varias capas de lenguaje, incluidos los lenguajes nativos (como Swift y Objective-C) y también Dart.
* <PlatformLink to="/data-management/debug-files/source-context">Contexto del código fuente</PlatformLink> muestra fragmentos de código alrededor de la ubicación de los marcos de pila.
* <PlatformLink to="/upload-debug">Sentry Dart Plugin</PlatformLink> facilita y automatiza la carga de símbolos de depuración.

**Limitaciones web:**

Sentry también es compatible con Flutter Web, con las siguientes limitaciones:

* Los títulos de los issues no son legibles para personas, ya que muestran el nombre minificado.
* Los rastreos de pila no se simbolican al compilar con WebAssembly (WASM).
* Los stack traces no se simbolizan cuando se cargan como una extensión del navegador.
* El almacenamiento en caché sin conexión no es compatible para las excepciones.

**Limitaciones de escritorio:**

Sentry también es compatible con Flutter en Linux y Windows, con las siguientes limitaciones:

* Los bloqueos nativos no son compatibles en Windows.
* [Release Health](/es/product/releases/health/) no es compatible.

Al ejecutar en macOS, puedes esperar las mismas funciones que en iOS.
