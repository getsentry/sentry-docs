<div id="features">
  ## Funciones
</div>

Además de capturar errores, puedes supervisar las interacciones entre varios servicios o aplicaciones si [habilitas el tracing](/es/concepts/key-terms/tracing/). También puedes recopilar y analizar perfiles de rendimiento de usuarios reales con [profiling](/es/product/explore/profiling/).

Selecciona qué funciones de Sentry quieres instalar, además de Error Monitoring, para obtener a continuación las instrucciones correspondientes de instalación y configuración.

Sentry recopila datos mediante un SDK en el tiempo de ejecución de tu aplicación. Estos son específicos de cada plataforma y permiten que Sentry entienda en profundidad cómo funciona tu aplicación.

Recomendamos instalar el SDK con nuestro [Sentry Wizard](https://github.com/getsentry/sentry-wizard) ejecutando uno de los siguientes comandos en el directorio de tu proyecto:

```bash {tabTitle:brew}
brew install getsentry/tools/sentry-wizard && sentry-wizard -i flutter
```

```powershell {tabTitle:Windows}
$downloadUrl = "https://github.com/getsentry/sentry-wizard/releases/download/v5.1.0/sentry-wizard-win-x64.exe"
Invoke-WebRequest $downloadUrl -OutFile sentry-wizard.exe
./sentry-wizard.exe -i flutter
```

```bash {tabTitle:macOS (Intel/x64)}
downloadUrl="https://github.com/getsentry/sentry-wizard/releases/download/v5.1.0/sentry-wizard-darwin-x64"
curl -L $downloadUrl -o sentry-wizard
chmod +x sentry-wizard
./sentry-wizard -i flutter
```

```bash {tabTitle:macOS (Apple Silicon/arm64)}
downloadUrl="https://github.com/getsentry/sentry-wizard/releases/download/v5.1.0/sentry-wizard-darwin-arm64"
curl -L $downloadUrl -o sentry-wizard
chmod +x sentry-wizard
./sentry-wizard -i flutter
```

```bash {tabTitle:Linux (x64)}
downloadUrl="https://github.com/getsentry/sentry-wizard/releases/download/v5.1.0/sentry-wizard-linux-x64"
curl -L $downloadUrl -o sentry-wizard
chmod +x sentry-wizard
./sentry-wizard -i flutter
```

```bash {tabTitle:Linux (arm64)}
downloadUrl="https://github.com/getsentry/sentry-wizard/releases/download/v5.1.0/sentry-wizard-linux-arm64"
curl -L $downloadUrl -o sentry-wizard
chmod +x sentry-wizard
./sentry-wizard -i flutter
```

```bash {tabTitle:npx}
npx @sentry/wizard@latest -i flutter
```

Esto aplicará un parche a tu proyecto y configurará el SDK. Solo necesitas aplicar el parche una vez; luego puedes agregar los archivos parcheados a tu sistema de control de versiones.
Si lo prefieres, también puedes [configurar el SDK manualmente](/es/platforms/dart/guides/flutter/manual-setup/) o seguir las instrucciones a continuación para ajustar la [configuración](#configure).

<div id="configure">
  ## Configuración
</div>

La configuración debe realizarse lo antes posible en el ciclo de vida de tu aplicación.

```dart
import 'package:flutter/widgets.dart';
import 'package:sentry_flutter/sentry_flutter.dart';

Future<void> main() async {
  await SentryFlutter.init(
    (options) {
      options.dsn = '___PUBLIC_DSN___';
      // Agrega encabezados de la solicitud y la IP de los usuarios. Para más información, visita:
      // https://docs.sentry.io/platforms/dart/guides/flutter/data-management/data-collected/
      options.sendDefaultPii = true;
      // ___PRODUCT_OPTION_START___ performance
      // Establece tracesSampleRate en 1.0 para capturar el 100% de las transacciones en el tracing.
      // Recomendamos ajustar este valor en producción.
      options.tracesSampleRate = 1.0;
      // ___PRODUCT_OPTION_END___ performance
      // ___PRODUCT_OPTION_START___ profiling
      // La tasa de muestreo del profiling es relativa a tracesSampleRate
      // Configurarlo en 1.0 perfilará el 100% de las transacciones muestreadas:
      // Nota: La versión alfa de profiling está disponible para iOS y macOS desde la versión 7.12.0 del SDK
      options.profilesSampleRate = 1.0;
      // ___PRODUCT_OPTION_END___ profiling
      // ___PRODUCT_OPTION_START___ logs

      // Habilita el envío de logs a Sentry
      options.enableLogs = true;
      // ___PRODUCT_OPTION_END___ logs
      // ___PRODUCT_OPTION_START___ session-replay
      // Registra repeticiones de sesión para el 100% de los errores y el 10% de las sesiones
      options.replay.onErrorSampleRate = 1.0;
      options.replay.sessionSampleRate = 0.1;
      // ___PRODUCT_OPTION_END___ session-replay
    },
    appRunner: () => runApp(
      SentryWidget(
        child: MyApp(),
      ),
    ),
  );

  // También puedes configurar SENTRY_DSN, SENTRY_RELEASE, SENTRY_DIST y
  // SENTRY_ENVIRONMENT mediante una variable de entorno de Dart (--dart-define)
}
```

<div id="verify">
  ## Verificar
</div>

Comprueba que tu aplicación esté enviando eventos a Sentry añadiendo el siguiente fragmento, que incluye un error intencional. Deberías ver el error informado en Sentry en unos minutos.

```dart
import 'package:sentry/sentry.dart';

try {
  throw StateError('Excepción de prueba de Sentry');
} catch (exception, stackTrace) {
  await Sentry.captureException(
    exception,
    stackTrace: stackTrace,
  );
}
```
