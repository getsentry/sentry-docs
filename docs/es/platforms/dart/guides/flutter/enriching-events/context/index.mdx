<div id="structured-context">
  ## Contexto estructurado
</div>

La mejor forma de adjuntar datos personalizados es con un contexto estructurado. Un contexto siempre debe ser un objeto y sus valores pueden ser arbitrarios.

Luego, usa <PlatformIdentifier name="set-context" /> y asigna al contexto un nombre único:

<PlatformContent includePath="enriching-events/set-context" />

No hay restricciones para el nombre del contexto. En el objeto de contexto, todas las claves están permitidas excepto `type`, que se usa internamente.

Obtén más información sobre las convenciones de los contextos comunes en la [documentación para desarrolladores de la interfaz de contexts](https://develop.sentry.dev/sdk/data-model/event-payloads/contexts/).

<div id="size-limitations">
  ## Limitaciones de tamaño
</div>

Al enviar contexto, _ten en cuenta los límites de tamaño del payload_. Sentry no recomienda enviar todo el estado de la aplicación ni grandes blobs de datos en los contextos. Si superas el tamaño máximo del payload, Sentry responderá con el error HTTP `413 Payload Too Large` y rechazará el evento.

El SDK de Sentry hará todo lo posible por adaptar los datos que envíes y recortar los payloads de contexto grandes. Algunos SDK pueden truncar partes del evento; para más detalles, consulta la [documentación para desarrolladores sobre el manejo de datos del SDK](https://develop.sentry.dev/sdk/expected-features/data-handling/).

<div id="additional-data">
  ## Datos adicionales
</div>

**La función Datos adicionales está obsoleta**, sustituida por contextos estructurados.

Sentry antes permitía agregar “datos adicionales” no estructurados mediante <PlatformIdentifier name="set-extra" />.
