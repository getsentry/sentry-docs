Sentry puede enriquecer tus eventos para una investigación más detallada almacenando archivos adicionales —como archivos de configuración o registros— como adjuntos.

<div id="creating-attachments">
  ## Creación de adjuntos
</div>

La manera más sencilla de crear un adjunto es usar una `path`. El SDK leerá el contenido del archivo cada vez que prepare un evento o una transacción y luego agregará el adjunto al mismo [envelope](https://develop.sentry.dev/sdk/data-model/envelopes/). Si el SDK no puede leer el archivo, registrará un mensaje de error y descartará el adjunto.

<PlatformContent includePath="enriching-events/attachment-init-with-path" />

Como alternativa, usa `bytes` para inicializar un adjunto. Al hacerlo, también debes especificar un nombre de archivo.

<PlatformContent includePath="enriching-events/attachment-init-with-bytes" />

Si tu SDK admite caché sin conexión, algo habitual en móviles, cada adjunto se almacena en disco por cada evento o transacción que captures cuando el dispositivo esté sin conexión. Al usar adjuntos grandes, este almacenamiento puede consumir el espacio en disco si el dispositivo permanece sin conexión durante un período prolongado. Puedes especificar el [tamaño máximo de adjunto](#maximum-attachment-size) para descartar adjuntos grandes y evitar este problema.

Además, puedes configurar estos parámetros:

`filename`

El nombre de archivo es el nombre que se mostrará en Sentry. Cuando uses bytes debes especificar un nombre de archivo, mientras que con una ruta no es necesario, ya que el SDK usará el último componente de la ruta.

`contentType`

El tipo de contenido almacenado en este adjunto. Puede usarse cualquier [tipo MIME](https://www.iana.org/assignments/media-types/media-types.xhtml); el valor predeterminado es `application/octet-stream`.

`mimetype`

El tipo específico de contenido multimedia que determina cómo se muestra el adjunto en la interfaz de Sentry. Actualmente admitimos y podemos renderizar los siguientes tipos MIME:

* `text/plain`
* `text/css`
* `text/csv`
* `text/html`
* `text/javascript`
* `text/json` o `text/x-json` o `application/json` o `application/ld+json`
* `image/jpeg`
* `image/png`
* `image/gif`

<div id="uploading-attachments">
  ## Carga de archivos adjuntos
</div>

Los archivos adjuntos residen en el <PlatformLink to="/enriching-events/scopes/">Scope</PlatformLink>. Puedes agregar un archivo adjunto en el Scope global para enviarlo con cada evento o agregarlo en el <PlatformLink to="/enriching-events/scopes/#local-scopes">Scope local</PlatformLink> para enviarlo solo con un evento específico.

<PlatformContent includePath="enriching-events/attachment-upload" />

<Alert>
  Sentry permite como máximo 20 MB por solicitud comprimida y hasta 100 MB de adjuntos sin comprimir por evento, incluido el archivo del informe de bloqueo (si corresponde). Las cargas que superen este tamaño se rechazan con el error HTTP `413 Payload Too Large` y los datos se descartan de inmediato. Para agregar archivos más grandes o en mayor cantidad, considera opciones de almacenamiento secundario.
</Alert>

Los archivos adjuntos se conservan durante 30 días; si superas el almacenamiento total incluido en tu cuota, no se guardarán. Puedes eliminar adjuntos o los eventos que los contienen en cualquier momento. Eliminar un adjunto no afecta tu cuota: Sentry lo cuenta en cuanto se almacena.

Obtén más información sobre cómo los adjuntos afectan tu [cuota](/es/pricing/quotas/).

<div id="access-to-attachments">
  ### Acceso a archivos adjuntos
</div>

Para limitar el acceso a los archivos adjuntos, ve a la **Configuración general** de tu organización y selecciona el menú desplegable *Acceso a archivos adjuntos* para establecer el nivel de acceso adecuado: cualquier miembro de tu organización, el propietario de la facturación de la organización, member, admin, manager u owner.

<Include name="common-imgs/attachments-access" />

De forma predeterminada, el acceso se concede a todos los miembros cuando el almacenamiento está habilitado. Si un miembro no tiene acceso al proyecto, no podrá descargar un archivo adjunto; el Button aparecerá en gris en Sentry. El miembro solo podrá ver que hay un archivo adjunto almacenado.

<div id="viewing-attachments">
  ## Visualización de archivos adjuntos
</div>

Los archivos adjuntos se muestran en la parte inferior de la página **Detalles del issue** para el evento que se muestra.

<Include name="common-imgs/attachments-access-denied" />

También puedes encontrarlos en la pestaña *Attachments* de la página **Detalles del issue**, donde puedes ver el *Type* del archivo adjunto, así como los eventos asociados. Haz clic en el ID del evento para abrir los **Detalles del issue** de ese evento específico.

<div id="maximum-attachment-size">
  ## Tamaño máximo de archivo adjunto
</div>

El tamaño máximo para cada archivo adjunto se define en `SentryOptions.maxAttachmentSize`.

La unidad es bytes y el valor predeterminado es `20 MiB`. Revisa también el
[tamaño máximo de archivos adjuntos de Relay](/es/product/relay/options/)
para asegurarte de que no se descarten allí.
