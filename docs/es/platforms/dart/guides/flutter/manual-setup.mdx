Sentry recopila datos mediante un SDK dentro del entorno de ejecución de tu aplicación. Estos son específicos de la plataforma y permiten que Sentry tenga un conocimiento profundo de cómo funciona tu aplicación.

```yml {filename:pubspec.yaml}
dependencies:
  sentry_flutter: ^{{@inject packages.version('sentry.dart.flutter') }}
```

<div id="configure">
  ## Configurar
</div>

La configuración debe realizarse lo antes posible en el ciclo de vida de tu aplicación.

```dart
import 'package:flutter/widgets.dart';
import 'package:sentry_flutter/sentry_flutter.dart';

Future<void> main() async {
  await SentryFlutter.init(
    (options) {
      options.dsn = '___PUBLIC_DSN___';
      // Añade encabezados de solicitud e IP para usuarios. Para más información, visita:
      // https://docs.sentry.io/platforms/dart/guides/flutter/data-management/data-collected/
      options.sendDefaultPii = true;
      // ___PRODUCT_OPTION_START___ performance
      // Establece tracesSampleRate en 1.0 para capturar el 100% de las transacciones para el rastreo.
      // Recomendamos ajustar este valor en producción.
      options.tracesSampleRate = 1.0;
      // ___PRODUCT_OPTION_END___ performance
      // ___PRODUCT_OPTION_START___ profiling
      // La tasa de muestreo para el perfilado es relativa a tracesSampleRate
      // Establecer en 1.0 perfilará el 100% de las transacciones muestreadas:
      // Nota: El alfa de perfilado está disponible para iOS y macOS desde la versión 7.12.0 del SDK
      options.profilesSampleRate = 1.0;
      // ___PRODUCT_OPTION_END___ profiling
      // ___PRODUCT_OPTION_START___ logs
      // Habilita el envío de logs a Sentry
      options.enableLogs = true;
      // ___PRODUCT_OPTION_END___ logs
    },
    appRunner: () => runApp(
      SentryWidget(
        child: MyApp(),
      ),
    ),
  );

  // también puedes configurar SENTRY_DSN, SENTRY_RELEASE, SENTRY_DIST y
  // SENTRY_ENVIRONMENT mediante variable de entorno de Dart (--dart-define)
}
```

<div id="verify">
  ## Verificar
</div>

Comprueba que tu app esté enviando eventos a Sentry añadiendo el siguiente fragmento, que incluye un error intencional. Deberías ver el error informado en Sentry en unos minutos.

```dart
import 'package:sentry/sentry.dart';

try {
  throw StateError('Excepción de prueba de Sentry');
} catch (exception, stackTrace) {
  await Sentry.captureException(
    exception,
    stackTrace: stackTrace,
  );
}
```
