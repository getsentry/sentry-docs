En esta página aprenderás cómo especificar en qué endpoints se agrega la información de trazas.

De forma predeterminada, la información de trazas (encabezados `sentry-trace` y `baggage`) se agregará a todas las solicitudes HTTP salientes. Si quieres limitar las URL en las que se agrega la información de trazas, puedes hacerlo usando la opción `tracePropagationTargets`:

<PlatformContent includePath="distributed-tracing/limiting-traces/" />

En el ejemplo anterior, la información de trazas se agregará a todas las solicitudes a URL que contengan `myproject.org` y a todos los subdominios de `otherservice.org`, como `https://api.otherservice.org/something/` y `https://payment.otherservice.org/something/`.

Consulta la documentación de nuestras <PlatformLink to="/configuration/options/#trace-propagation-targets">opciones de configuración</PlatformLink> para obtener más información sobre la opción `trace_propagation_targets`.
