Si el ecosistema de tu aplicación que quieres observar con Sentry abarca más que un solo servicio o aplicación, el rastreo distribuido puede aportar mucho valor.

<div id="what-is-distributed-tracing">
  ## ¿Qué es el tracing distribuido?
</div>

En el contexto de rastrear eventos en un sistema distribuido, el tracing distribuido es una potente herramienta de depuración. Imagina tu aplicación como una amplia red de componentes interconectados. Por ejemplo, tu sistema puede estar repartido entre distintos servidores o tu aplicación puede dividirse en servicios de backend y frontend, cada uno con su propia pila tecnológica.

Cuando se produce un error o un problema de rendimiento, puede resultar difícil identificar la causa raíz debido a la complejidad del sistema. El tracing distribuido te ayuda a seguir el recorrido de un evento mientras atraviesa esta compleja red, registrando cada paso. Al analizar estos trazos, puedes reconstruir la secuencia de eventos que condujo al evento de interés, identificar los componentes específicos implicados y comprender sus interacciones. Esta visibilidad detallada te permite diagnosticar y resolver problemas con mayor eficacia, mejorando en última instancia la fiabilidad y el rendimiento de tu sistema distribuido.

  ## ¿Cómo usar el rastreo distribuido?
</div>

Recuerda que, para propagar la información de rastreo en todo tu sistema distribuido, debes usar Sentry en todos los servicios y aplicaciones implicados. Consulta la documentación del SDK correspondiente para saber cómo habilitar el rastreo distribuido en cada plataforma.
