---
title: Inicialización nativa
description: "Aprende a inicializar manualmente los SDK nativos."
sidebar_order: 12
---

De forma predeterminada, el SDK de Flutter inicializa el SDK nativo desde el método `init` que se invoca en la capa de Flutter. Como resultado, el SDK tiene actualmente la limitación de no capturar fallos nativos que ocurran antes de que se llame al método `init` en la capa de Flutter. Puedes inicializar tú mismo los SDK nativos para superar esta limitación o si quieres definir opciones personalizadas más allá de lo que ofrece actualmente el SDK de Flutter.

Para hacerlo, establece [autoInitializeNativeSdk](/es/platforms/dart/guides/flutter/configuration/options/#autoInitializeNativeSdk) en `false` en las opciones de init:

```dart {2}
await SentryFlutter.init((options) {
  options.autoInitializeNativeSdk = false;
}, appRunner: () => runApp(SentryWidget(child: const MyApp())));
```

Esto evitará que el SDK de Flutter inicialice automáticamente los SDK nativos.

A continuación, inicializa los SDK nativos según se indica en las guías de abajo.

* [Android](/es/platforms/android/configuration/manual-init/#manual-initialization)
* [iOS](/es/platforms/apple/guides/ios/manual-setup/)
* [Navegador](/es/platforms/javascript/#configure)

<Alert>
  Para Web, deberás instalar el SDK de JavaScript inyectando manualmente el script del cargador en la etiqueta `<head>` de tu HTML.

  Los SDK de Android e iOS ya vienen incluidos con el SDK de Flutter.
</Alert>
