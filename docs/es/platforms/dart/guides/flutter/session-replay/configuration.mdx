<div id="general-integration-configuration">
  ## Configuración general de la integración
</div>

Las siguientes opciones se pueden configurar en el campo `options.replay` de tu SDK de Sentry para Flutter, en `SentryFlutter.init((options) { ... })`:

| Clave                   | Tipo     | Predeterminado | Descripción                                                                                                                                                                                                                                                       |
| ----------------------- | -------- | --------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| sessionSampleRate | `double` | `0`     | La tasa de muestreo para las repeticiones que empiezan a grabar de inmediato y abarcan toda la sesión del usuario. `1.0` recopila todas las repeticiones y `0` no recopila ninguna.                                                                                                      |
| onErrorSampleRate | `double` | `0`     | La tasa de muestreo para las repeticiones que se registran cuando ocurre un error. Este tipo de repetición guardará hasta un minuto de eventos previos al error y continuará grabando hasta que la sesión termine. `1.0` captura todas las sesiones con error y `0` no captura ninguna. |
| quality | [SentryReplayQuality](https://pub.dev/documentation/sentry_flutter/latest/sentry_flutter/SentryReplayQuality.html) | `SentryReplayQuality.medium` | Define la calidad de imagen de la repetición de sesión. Cuanto mayor sea la calidad, más precisa será la repetición, pero también implicará transferir más datos y una mayor carga de CPU. |

<div id="configure-network-details">
  ## Configurar los detalles de red
</div>

Para capturar información básica sobre las solicitudes de red, debes usar nuestras integraciones de Sentry para el cliente HTTP que utilices.
Así, el cliente HTTP instrumentado con Sentry podrá capturar los detalles de red durante la reproducción.

* [Dio](/es/platforms/dart/integrations/dio/)
* [HTTP](/es/platforms/dart/integrations/http-integration/)

<div id="configure-screen-names-and-navigation-details">
  ## Configurar nombres de pantalla y detalles de navegación
</div>

Para capturar los nombres de pantalla y las rutas de navegación (breadcrumbs), debes agregar nuestro `SentryNavigatorObserver` a los `navigatorObservers` de tu aplicación y definir nombres de ruta cuando navegues a una nueva pantalla.

Obtén más información sobre cómo configurar `SentryNavigatorObserver` en nuestra guía de [Routing Instrumentation](/es/platforms/dart/guides/flutter/integrations/routing-instrumentation/#configure).

<div id="configure-user-interactions">
  ## Configurar interacciones de usuario
</div>

Para capturar migas de interacción del usuario, como toques en botones, debes envolver tu widget raíz con `SentryWidget` y definir claves únicas para tus botones.

Obtén más información sobre cómo configurar las interacciones de usuario en nuestra guía [User Interaction Instrumentation](/es/platforms/dart/guides/flutter/integrations/user-interaction-instrumentation/).
