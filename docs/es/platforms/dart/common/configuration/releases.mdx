Un lanzamiento es una versión de tu código que se despliega en un <PlatformLink to="/configuration/environments/">entorno</PlatformLink>. Cuando proporcionas a Sentry información sobre tus lanzamientos, puedes:

* Identificar problemas y regresiones introducidos en un nuevo lanzamiento
* Predecir qué commit causó un problema y quién es probablemente responsable
* Resolver problemas incluyendo el número del issue en tu mensaje de commit
* Recibir notificaciones por correo electrónico cuando tu código se despliegue

<div id="setting-a-release">
  ## Configurar una versión
</div>

<PlatformContent includePath="set-release" notateUnsupported />

Cómo pongas a disposición de tu código el nombre de la versión (o release) depende de ti. Por ejemplo, puedes usar una variable de entorno definida durante el proceso de compilación o en el arranque inicial.

Definir el nombre de la versión etiqueta cada evento con ese nombre. Recomendamos informar a Sentry de una nueva versión antes de enviar eventos con ese nombre, ya que esto desbloqueará algunas funciones adicionales. Obtén más información en nuestra documentación de [Releases](/es/product/releases/).

Si no informas a Sentry de una nueva versión, Sentry creará automáticamente una entidad de versión en el sistema la primera vez que vea un evento con ese ID de versión.

Después de configurar tu SDK, puedes instalar una integración de repositorio o proporcionar manualmente a Sentry tus propios metadatos de commits. Lee nuestra documentación sobre [configuración de versiones](/es/product/releases/setup/) para más información sobre integraciones, asociación de commits y cómo informar a Sentry al implementar versiones.
