---
title: Datos recopilados
description: "Consulta qué datos recopila el SDK de Sentry."
sidebar_order: 1
---

Sentry se toma muy en serio la privacidad de los datos y cuenta con configuraciones predeterminadas que priorizan su seguridad, especialmente cuando se trata de información de identificación personal (PII). Al agregar el SDK de Sentry a tu aplicación, permites que recopile datos y los envíe a Sentry durante la ejecución de tu aplicación.

Los tipos de categorías y la cantidad de datos recopilados varían según las integraciones que tengas habilitadas en el SDK de Sentry. En esta página se enumeran las categorías de datos que recopila el SDK de Sentry para Dart.

Muchas de las categorías enumeradas aquí requieren que habilites la <PlatformLink to="/configuration/options/#sendDefaultPii">opción sendDefaultPii</PlatformLink>.

<div id="http-headers">
  ## Encabezados HTTP
</div>

De forma predeterminada, el SDK de Sentry no envía encabezados HTTP. Incluso cuando el envío de encabezados HTTP está habilitado, contamos con una [lista de bloqueo](https://github.com/getsentry/sentry-dart/blob/ea1d45d64f52551e2e033c50b0ff8512e3d8a4e3/dart/lib/src/utils/http_sanitizer.dart#L9C1-L21C1) que filtra cualquier encabezado que contenga datos sensibles.

Para empezar a enviar encabezados HTTP, configura <PlatformLink to="/configuration/options/#sendDefaultPii">`sendDefaultPii = true`</PlatformLink>.

<div id="information-about-logged-in-user">
  ## Información sobre el usuario autenticado
</div>

De forma predeterminada, el SDK de Sentry no envía información sobre el usuario autenticado, como el correo electrónico, el ID de usuario o el nombre de usuario. Incluso si lo habilitas, el tipo de datos de usuario que podrás enviar depende de las integraciones que actives en el SDK de Sentry. La mayoría de las integraciones no enviarán ningún dato de usuario. Algunas solo establecerán el ID de usuario, pero hay algunas que también establecerán el ID de usuario, el nombre de usuario y el correo electrónico.

Para empezar a enviar información del usuario autenticado, configura <PlatformLink to="/configuration/options/#send-default-pii">`sendDefaultPii = true`</PlatformLink>.

<div id="users-ip-addresses">
  ## Direcciones IP de los usuarios
</div>

De forma predeterminada, el SDK de Sentry no envía la dirección IP del usuario. Una vez habilitado, los servicios de backend de Sentry inferirán la dirección IP del usuario a partir de la solicitud entrante, a menos que ciertas integraciones que puedes habilitar invaliden este comportamiento.

Para habilitar el envío de la dirección IP del usuario, configura <PlatformLink to="/configuration/options/#send-default-pii">`sendDefaultPii = true`</PlatformLink>.

<div id="request-url">
  ## URL de la solicitud
</div>

La URL completa de las solicitudes HTTP salientes y entrantes **siempre se envía a Sentry**. Dependiendo de tu aplicación, esto podría incluir datos personales (PII).

<div id="request-query-string">
  ## Cadena de consulta de la solicitud
</div>

La cadena de consulta completa de las solicitudes HTTP, tanto salientes como entrantes, **siempre se envía a Sentry**. Dependiendo de tu aplicación, esto podría contener datos personales (PII).

<div id="request-body">
  ## Cuerpo de la solicitud
</div>

El cuerpo de las solicitudes HTTP entrantes se puede enviar a Sentry. Si se envía o no depende del tipo y del tamaño del cuerpo, como se describe a continuación:

- **Tipo de cuerpo de la solicitud:**
  - Se envían los cuerpos JSON y de formulario
- **Tamaño del cuerpo de la solicitud:** Hay una opción <PlatformLink to="/configuration/options/#maxRequestBodySize">maxRequestBodySize</PlatformLink> configurada en `NONE` de forma predeterminada. Esto significa que, por defecto, no se envía ningún cuerpo de solicitud a Sentry.

<div id="file-io">
  ## E/S de archivos
</div>

De forma predeterminada, el SDK de Sentry no envía el nombre ni la ruta de los archivos cuando <PlatformLink to="/integrations/file">instrumenta la E/S de archivos</PlatformLink>.

Si quieres enviar nombres y rutas de archivos, establece <PlatformLink to="/configuration/options/#send-default-pii">`sendDefaultPii = true`</PlatformLink>.

<div id="runtime-information">
  ## Información en tiempo de ejecución
</div>

De forma predeterminada, el SDK recopila información básica en tiempo de ejecución, como la versión de Dart y la plataforma.

Cuando <PlatformLink to="/configuration/options/#send-default-pii">`sendDefaultPii = true`</PlatformLink>, se recopilan detalles adicionales del entorno de ejecución:

- Ruta del ejecutable, p. ej., `dart`
- Ubicaciones del ejecutable resueltas, p. ej., `/system/bin/app_process64`
- Ruta del script, p. ej., `file:///main.dart`

<div id="sql-queries">
  ## Consultas SQL
</div>

Aunque las consultas SQL se envían a Sentry, ni la consulta SQL completa (`UPDATE app_user SET password='supersecret' WHERE id=1;`) ni los valores de sus parámetros se enviarán nunca. En su lugar, se envía una versión parametrizada de la consulta (`UPDATE app_user SET password=? WHERE id=?;`).
