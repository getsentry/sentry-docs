Con el [rastreo](/es/product/insights/overview/), Sentry supervisa el rendimiento de tu software, midiendo métricas como el rendimiento (throughput) y la latencia, y mostrando el impacto de los errores en múltiples sistemas. Sentry captura trazas distribuidas compuestas por transacciones y spans, que miden servicios individuales y las operaciones individuales dentro de esos servicios. Obtén más información sobre nuestro modelo en [Rastreo distribuido](/es/product/sentry-basics/tracing/distributed-tracing/).

<Alert>
  Si vas a adoptar el rastreo en un entorno de alto rendimiento, te recomendamos probar antes del despliegue para asegurarte de que las características de rendimiento de tu servicio se mantengan dentro de lo esperado.
</Alert>

<div id="configure">
  ## Configurar
</div>

Primero, habilita el tracing y configura la tasa de muestreo para las transacciones. Establece la tasa de muestreo de tus transacciones de una de estas maneras:

* Configurando una tasa de muestreo uniforme para todas las transacciones usando la opción <PlatformIdentifier name="traces-sample-rate" /> en la configuración de tu SDK con un número entre `0` y `1`. (Por ejemplo, para enviar el 20 % de las transacciones, establece <PlatformIdentifier name="traces-sample-rate" /> en `0.2`).
* Controlando la tasa de muestreo en función de la propia transacción y del contexto en el que se captura, proporcionando una función a la opción de configuración <PlatformIdentifier name="traces-sampler" />.

Las dos opciones están pensadas para ser mutuamente excluyentes. Si estableces ambas, <PlatformIdentifier name="traces-sampler" /> tendrá prioridad.

Obtén más información sobre las <PlatformLink to="/configuration/options/#tracing-options">opciones</PlatformLink> de tracing, cómo usar la función <PlatformLink to="/configuration/sampling/#setting-a-sampling-function">TracesSampler</PlatformLink> o cómo <PlatformLink to="/configuration/sampling/#sampling-transaction-events">muestrear transacciones</PlatformLink>.

<div id="verify">
  ## Verificar
</div>

Prueba el tracing iniciando y finalizando una transacción; *debes* hacerlo para que las transacciones se envíen a Sentry. Aprende cómo en nuestro contenido de <PlatformLink to="/tracing/instrumentation/custom-instrumentation/">Instrumentación personalizada</PlatformLink>.

Mientras realizas las pruebas, establece <PlatformIdentifier name="traces-sample-rate" /> en `1.0`, ya que eso garantiza que cada transacción se envíe a Sentry. Cuando termines, quizá quieras usar un valor más bajo para <PlatformIdentifier name="traces-sample-rate" />, o cambiar a <PlatformIdentifier name="traces-sampler" /> para muestrear y filtrar tus transacciones selectivamente según datos contextuales.
