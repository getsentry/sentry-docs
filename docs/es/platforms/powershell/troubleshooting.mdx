<div id="the-event-is-dropped-by-sentry-because-its-too-large">
  ## Sentry descarta el evento porque es demasiado grande
</div>

> Sentry: (Error) Sentry rechazó el “envelope” 5bfe4129cb2446c08768b16479865035. Código de estado: RequestEntityTooLarge. Detalle del error: no se pudo leer el cuerpo de la solicitud. Causas del error: una carga alcanzó el límite de tamaño.

Puedes hacer que el [SDK de Sentry guarde estos “envelopes” en una ruta](https://github.com/getsentry/sentry-dotnet/blob/e62013ca63436d2a786fb229570e96f42c7c4044/src/Sentry/Http/HttpTransportBase.cs#L341-L363) para diagnosticar por qué son demasiado grandes.
Establece la variable de entorno `SENTRY_KEEP_LARGE_ENVELOPE_PATH` en el directorio donde quieres que se guarden.
Asegúrate de que el proceso en el que se ejecuta el SDK tenga permisos de escritura allí.

<div id="unhandled-exceptions-are-not-captured">
  ## Las excepciones no controladas no se capturan
</div>

Actualmente, el SDK captura las excepciones que se controlan explícitamente mediante uno de los siguientes métodos:

- canalizando el `ErrorRecord` dentro de una instrucción `catch` o en un `trap` usando `$_ | Out-Sentry`
- usando `Invoke-WithSentry { ... }`
