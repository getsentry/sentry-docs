Sentry permite desminificar JavaScript mediante [mapas de código fuente](https://blog.sentry.io/2015/10/29/debuggable-javascript-with-source-maps/). Esto te permite ver el contexto del código obtenido de las trazas de pila en su forma original sin transformar, lo cual es especialmente útil para depurar código minificado (p. ej., UglifyJS) o código transpilado desde un lenguaje de mayor nivel (p. ej., TypeScript, ES6).

La mayor parte del proceso es la misma tanto si usas este SDK como el [nuevo SDK unificado de JavaScript para navegador](/es/platforms/javascript/), por lo que la documentación principal para trabajar con mapas de código fuente se encuentra [aquí](/es/platforms/javascript/sourcemaps/). La única diferencia es cómo especificas la versión (release) en la configuración de tu SDK.

<div id="specify-the-release-in-ravenjs">
  ## Especifica la versión en Raven.js
</div>

Si subes tú mismo los artefactos de los mapas de origen (source maps), debes especificar la versión en la configuración de tu cliente Raven.js. Sentry usará el nombre de la versión para asociar los datos de eventos procesados con los archivos que has subido mediante la [API de releases](/es/api/releases/), [sentry-cli](/es/cli/) o [sentry-webpack-plugin](https://github.com/getsentry/sentry-webpack-plugin). Este paso es opcional si alojas los source maps en el servidor remoto, pero sigue siendo recomendable.

```javascript
Raven.config("tu-dsn", {
  release: "mi-proyecto@2.3.12",
}).install();
```
