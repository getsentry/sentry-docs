<div id="prerequisites">
  ## Requisitos previos
</div>

Necesitas:

* Una [cuenta](https://sentry.io/signup/) de Sentry y un [proyecto](/es/product/projects/)
* Tu aplicación en funcionamiento
* Angular versión `17.0.0` o superior

<Expandable level="warning" title="¿Usas Angular 16 o una versión anterior?">
  Si usas Angular 16 o una versión anterior, instala y <PlatformLink to="/manual-setup">configura el SDK manualmente</PlatformLink>. El SDK de Sentry para Angular sigue funcionando con Angular 14 y posteriores, pero los asistentes de instalación requieren al menos Angular 17.
</Expandable>

<div id="step-1-install">
  ## Paso 1: Instalación
</div>

Para instalar Sentry, ejecuta el siguiente comando en tu proyecto:

```bash
npx @sentry/wizard@latest -i angular
```

El asistente te guía a través del proceso de configuración y te solicita habilitar funciones adicionales (opcionales) de Sentry para tu aplicación, más allá del monitoreo de errores.

<PlatformContent includePath="getting-started-features-expandable" />

Esta guía asume que habilitas todas las funciones y permites que el asistente añada un componente de ejemplo a tu aplicación. Puedes agregar o quitar funciones en cualquier momento, pero configurarlas ahora te ahorrará el trabajo de hacerlo manualmente más tarde.

<Expandable title="¿Qué cambia el asistente de instalación dentro de tu aplicación?">
  * Agrega una llamada a `Sentry.init()` en tu archivo `main.ts`
  * Registra los servicios de Sentry `ErrorHandler` y `TraceService` en tu archivo `app.config.ts`
  * Crea `.sentryclirc` con un token de autenticación para cargar source maps (este archivo se agrega automáticamente a `.gitignore`)
  * Te guía para habilitar la carga de source maps al crear una compilación de producción, tanto localmente como en CI
  * Agrega un componente de ejemplo a tu aplicación para ayudar a verificar tu configuración de Sentry
</Expandable>

<div id="step-3-verify">
  ## Paso 3: Verificar
</div>

Si aún no has probado tu configuración de Sentry, hagámoslo ahora. Puedes confirmar que el SDK de Sentry está enviando datos a tu proyecto de Sentry usando el componente de ejemplo creado por el asistente de instalación:

1. Abre en tu navegador la página en la que agregaste el componente de ejemplo.
2. Haz clic en el botón &quot;Throw error&quot;. Esto provoca un error.

Sentry captura este error por ti. Además, al hacer clic en el botón se inicia un trace para medir el tiempo que tarda en generarse el error.

<Alert level="success" title="Sugerencia">
  No olvides explorar el código del componente de ejemplo en tu proyecto para entender qué sucede después de hacer clic en el botón.
</Alert>
