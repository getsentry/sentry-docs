<div id="step-1-install">
  ## Paso 1: Instalar
</div>

Para instalar Sentry con el asistente de instalación, ejecuta el siguiente comando en tu proyecto:

```bash
npx @sentry/wizard@latest -i nuxt
```

El asistente luego te guía por el proceso de configuración y te pide habilitar funciones adicionales (opcionales) de Sentry para tu aplicación, más allá del monitoreo de errores.

<PlatformContent includePath="getting-started-features-expandable" />

Esta guía asume que habilitas todas las funciones y permites que el asistente cree una página o un componente de ejemplo. Puedes agregar o quitar funciones en cualquier momento, pero configurarlas ahora te ahorrará el esfuerzo de hacerlo manualmente más adelante.

<Expandable title="¿Qué cambia el asistente de instalación dentro de tu aplicación?">
  * Crea `sentry.(client|server).config.ts` para inicializar el SDK
  * Crea o actualiza tu `nuxt.config.ts` para agregar opciones de compilación que permitan subir mapas de origen y habilitar la autoinstrumentación mediante plugins de Vite
  * Crea `.env.sentry-build-plugin` con un token de autenticación para subir mapas de origen (este archivo se agrega automáticamente a `.gitignore`)
  * Agrega una página y una ruta de ejemplo a tu aplicación para ayudarte a verificar la configuración de Sentry
    * Si no pudo crear una página, el asistente agrega un componente de ejemplo en su lugar
</Expandable>

<div id="step-3-verify-your-setup">
  ## Paso 3: Verifica tu configuración
</div>

Si aún no has probado tu configuración de Sentry, hagámoslo ahora. Puedes confirmar que Sentry funciona correctamente y envía datos a tu proyecto de Sentry usando la página de ejemplo creada por el asistente de instalación.

<Expandable level="warning" title="El asistente creó un componente de ejemplo en lugar de una página.">
  El asistente crea un componente `SentryErrorButton` cuando no puede agregar una página de ejemplo a tu proyecto. Esto suele ocurrir si no tienes un archivo `app.vue` o si ese archivo no contiene el componente `NuxtPage`. Para verificar tu configuración, sigue estos pasos antes de continuar con la siguiente sección, “Ver datos capturados en Sentry”:

  1. Agrega el componente `SentryErrorButton` a una página y ábrela en tu navegador. En la mayoría de las aplicaciones Nuxt, esto será en localhost.
  2. Haz clic en el botón “Throw Sample Error”, que genera un error e inicia un trazo de rendimiento en el lado del cliente.

  Para probar también el lado del servidor, consulta la sección “Verify” en la [guía de configuración manual](/es/platforms/javascript/guides/nuxt/manual-setup/).
</Expandable>

Para probar Sentry, puedes ejecutar tu aplicación Nuxt en modo producción o desarrollo.
**Recomendamos probar en modo producción**, ya que es lo más parecido al entorno de tu aplicación desplegada.

<Alert>
  De forma predeterminada, el SDK añadirá la configuración del servidor como `sentry.server.config.mjs` a la compilación.
  Para encontrar la ruta exacta a este archivo, habilita el modo `debug` en tu configuración de Sentry dentro de `nuxt.config.ts`.
  Sentry imprimirá la ruta exacta durante el proceso de compilación.
</Alert>

<div id="run-in-production-mode-recommended">
  ### Ejecutar en modo producción (recomendado)
</div>

Después de compilar con `nuxi build`, ejecuta tu proyecto y asegúrate de cargar Sentry en el servidor agregándolo explícitamente con `--import` (más información sobre esta opción en [Métodos de instalación](/es/platforms/javascript/guides/nuxt/install)).

```
# Inicia tu aplicación después de compilar tu proyecto con `nuxi build`
node --import ./.output/server/sentry.server.config.mjs .output/server/index.mjs
```

<div id="run-in-development-mode">
  ### Ejecutar en modo desarrollo
</div>

El archivo de configuración del servidor se genera en el directorio `.nuxt` la primera vez que ejecutas `nuxt dev`. Si eliminas tu directorio `.nuxt`, tendrás que ejecutar `nuxt dev` una vez sin la variable `NODE_OPTIONS` para volver a generarlo.

Si solo quieres usar Sentry en el cliente o solo necesitas un monitoreo básico de errores en el servidor, puedes omitir la opción `--import` al ejecutar tu aplicación.

```
# Ejecuta el servidor de desarrollo con el flag --import después de ejecutar `nuxt dev` una vez (sin el flag)
NODE_OPTIONS='--import ./.nuxt/dev/sentry.server.config.mjs' nuxt dev
```

Después de compilar y ejecutar tu proyecto:

1. Abre la página de ejemplo `/sentry-example-page` en tu navegador. En la mayoría de las aplicaciones Nuxt, estará en localhost.
2. Haz clic en el Button &quot;Throw sample error&quot;. Esto provoca dos errores:
   * un error en el frontend
   * un error en la ruta de la API

Sentry captura ambos errores por ti. Además, el clic en el botón inicia un rastreo de rendimiento para medir el tiempo que tarda en completarse la solicitud a la API.

<div id="view-captured-data-in-sentry">
  ### Ver los datos capturados en Sentry
</div>

Ahora, ve a tu proyecto en [Sentry.io](https://sentry.io/) para ver los datos recopilados (puede tardar unos momentos en que aparezcan).
