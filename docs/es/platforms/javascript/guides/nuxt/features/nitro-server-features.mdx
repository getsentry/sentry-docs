---
title: Funcionalidades del servidor Nitro
description: "Conoce qué funcionalidades del servidor Nitro instrumenta automáticamente el SDK de Sentry para Nuxt."
---

El SDK de Sentry para Nuxt instrumenta automáticamente varias API del servidor Nitro para ofrecer supervisión del rendimiento y seguimiento de errores en tu código del lado del servidor.

<div id="server-middleware">
  ## Middleware del servidor
</div>

<Alert>Disponible a partir de la versión `@sentry/nuxt` 10.17.0.</Alert>

El SDK de Sentry para Nuxt instrumenta automáticamente los [manejadores de middleware del servidor](https://nuxt.com/docs/4.x/guide/directory-structure/server#server-middleware), además de los manejadores de eventos del servidor. Esto incluye las funciones del manejador y sus callbacks `onBeforeResponse` y `onRequest`.

<div id="cache">
  ## Caché
</div>

<Alert>Disponible a partir de la versión `@sentry/nuxt` 10.21.0.</Alert>

El SDK de Sentry para Nuxt instrumenta automáticamente las [API de caché](https://nitro.build/guide/cache) para las siguientes funciones de Nitro:

- `defineCachedEventHandler`
- `cachedEventHandler`
- `cachedFunction`

Esto añade spans a tus transacciones que representan operaciones de caché y sus duraciones, incluidos los aciertos de caché.

<div id="storage">
  ## Almacenamiento
</div>

<Alert>Disponible a partir de la versión `@sentry/nuxt` 10.21.0.</Alert>

El SDK de Sentry para Nuxt instrumenta automáticamente las [API de almacenamiento](https://nuxt.com/docs/4.x/guide/directory-structure/server#server-storage) (también conocido como almacenamiento KV) y las operaciones más comunes.

Ten en cuenta que solo se instrumentarán los montajes de almacenamiento configurados en `nitro.storage` de tu `nuxt.config.ts`. Los spans de almacenamiento aparecerán de forma similar a los de [cache](#cache), ya que ambos utilizan la misma API de almacenamiento de Nitro.

<div id="database">
  ## Base de datos
</div>

<Alert>Disponible a partir de la versión `@sentry/nuxt` 10.21.0.</Alert>

El SDK de Sentry para Nuxt instrumenta automáticamente la [API de base de datos](https://nitro.build/guide/database) de Nitro. Esto añade spans a tus transacciones que representan operaciones de consultas a la base de datos y sus duraciones, lo que te permite identificar consultas lentas, hacer seguimiento de su frecuencia y optimizar tus interacciones con la base de datos.

La instrumentación de base de datos solo se aplica a los mounts de base de datos configurados en `nitro.database` en tu `nuxt.config.ts`, y únicamente cuando `nitro.experimental.database` está establecido en `true`.
