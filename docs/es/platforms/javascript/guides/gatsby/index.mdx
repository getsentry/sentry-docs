<div id="features">
  ## Funciones
</div>

Además de capturar errores, puedes supervisar las interacciones entre varios servicios o aplicaciones [habilitando el tracing](/es/concepts/key-terms/tracing/). También puedes llegar más rápido a la causa raíz de un error o de un problema de rendimiento viendo una reproducción tipo video de una sesión de usuario con [session replay](/es/product/explore/session-replay/web/getting-started/).

Selecciona qué funciones de Sentry te gustaría instalar además de Error Monitoring para obtener a continuación las instrucciones correspondientes de instalación y configuración.

Para usar Sentry con tu aplicación de Gatsby, necesitas usar `@sentry/gatsby` (el SDK de Gatsby de Sentry):

```bash {tabTitle:npm}
npm install @sentry/gatsby --save
```

```bash {tabTitle:yarn}
yarn add @sentry/gatsby
```

```bash {tabTitle:pnpm}
pnpm add @sentry/gatsby
```

<Alert>
  `@sentry/gatsby` es un envoltorio del paquete `@sentry/react`, con funcionalidad adicional específica de Gatsby. Todos los métodos disponibles en el paquete `@sentry/react` también pueden importarse desde `@sentry/gatsby`.
</Alert>

Registra el complemento `@sentry/gatsby` en tu archivo de configuración de Gatsby (normalmente `gatsby-config.js`).

```javascript {filename:gatsby-config.js}
module.exports = {
  plugins: [
    {
      resolve: "@sentry/gatsby",
    },
  ],
};
```

<div id="configure">
  ## Configurar
</div>

Luego, configura `Sentry.init`. Para ello, crea un archivo nuevo llamado `sentry.config.js` en la raíz de tu proyecto y agrega el siguiente código:

```javascript {filename:sentry.config.(js|ts)}
import * as Sentry from "@sentry/gatsby";

Sentry.init({
  dsn: "___PUBLIC_DSN___",

  // Agrega encabezados de solicitud y la IP de los usuarios; para obtener más información, visita:
  // https://docs.sentry.io/platforms/javascript/guides/gatsby/configuration/options/#sendDefaultPii
  sendDefaultPii: true,

  integrations: [
    // ___PRODUCT_OPTION_START___ rendimiento
    Sentry.browserTracingIntegration(),
    // ___PRODUCT_OPTION_END___ rendimiento
    // ___PRODUCT_OPTION_START___ repetición de sesión
    Sentry.replayIntegration(),
    // ___PRODUCT_OPTION_END___ repetición de sesión
    // ___PRODUCT_OPTION_START___ comentarios de usuarios
    Sentry.feedbackIntegration({
      // La configuración adicional del SDK va aquí; por ejemplo:
      colorScheme: "system",
    }),
    // ___PRODUCT_OPTION_END___ comentarios de usuarios
  ],
  // ___PRODUCT_OPTION_START___ registros

  // Habilita el envío de registros a Sentry
  enableLogs: true,
  // ___PRODUCT_OPTION_END___ registros

  // ___PRODUCT_OPTION_START___ rendimiento
  // Establece tracesSampleRate en 1.0 para capturar el 100 %
  // de las transacciones de trazas.
  // Recomendamos ajustar este valor en producción
  // Más información en
  // https://docs.sentry.io/platforms/javascript/configuration/options/#traces-sample-rate
  tracesSampleRate: 1.0,
  // ___PRODUCT_OPTION_END___ rendimiento
  // ___PRODUCT_OPTION_START___ repetición de sesión

  // Captura Replay para el 10 % de todas las sesiones,
  // y para el 100 % de las sesiones con un error
  // Más información en
  // https://docs.sentry.io/platforms/javascript/session-replay/configuration/#general-integration-configuration
  replaysSessionSampleRate: 0.1,
  replaysOnErrorSampleRate: 1.0,
  // ___PRODUCT_OPTION_END___ repetición de sesión
});
```
