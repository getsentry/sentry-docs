Para usar raven-node con CoffeeScript u otra biblioteca que sobrescriba Error.prepareStackTrace, podrías encontrarte con la excepción “Traceback does not support Error.prepareStackTrace being defined already.”

Para evitar que raven-node (y la biblioteca subyacente de trazas de pila en bruto) requiera Traceback, puedes pasar tu propia stackFunction en las opciones. Por ejemplo:

```coffeescript
client = new raven.Client('___PUBLIC_DSN___', {
    stackFunction: {{ Tu función de stack }}
});
```

Por ejemplo:

```coffeescript
client = new raven.Client('___PUBLIC_DSN___', {
    stackFunction: Error.prepareStackTrace
});
```
