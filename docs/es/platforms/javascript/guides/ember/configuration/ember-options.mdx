Todas las opciones del SDK de Sentry pueden pasarse a `init`:

```javascript
import * as Sentry from "@sentry/ember";

Sentry.init({
  // Opciones de Sentry
  dsn: "___PUBLIC_DSN___",
});
```

El complemento `@sentry/ember` incluye opciones para gestionar la instrumentación específica de Ember; estas opciones se configuran directamente en la configuración del propio complemento.

```javascript
ENV["@sentry/ember"] = {
  // Opciones específicas de Ember
};
```

La siguiente documentación corresponde a la configuración específica de Ember; para las opciones de Sentry, [consulta las opciones básicas](/es/platforms/javascript/guides/ember/configuration/options)

<div id="tracing-considerations">
  ### Consideraciones sobre el tracing
</div>

La integración de tracing de Sentry ya está configurada mediante el complemento de Ember, con instrumentación personalizada de Ember para el enrutamiento, los componentes y el runloop. Carga el tracing de Sentry como un chunk adicional para instrumentar tu aplicación. Si quieres desactivar esta instrumentación automática y dejar de recibir las transacciones asociadas, establece `disablePerformance` en tu configuración. Consulta el ejemplo a continuación:

```javascript
ENV["@sentry/ember"] = {
  disablePerformance: true,
};
```

<div id="routes">
  ### Rutas
</div>

Si quieres capturar los tiempos de los hooks `beforeModel`, `model`, `afterModel`, así como de `setupController` en una de tus rutas, `@sentry/ember` exporta la función `instrumentRoutePerformance`, que puedes usar sustituyendo la exportación predeterminada por una ruta envuelta.

```javascript
import Route from "@ember/routing/route";
import { instrumentRoutePerformance } from "@sentry/ember";

class MyRoute extends Route {
  model() {
    //…
  }
}

export default instrumentRoutePerformance(MyRoute);
```

<div id="classic-components">
  ### Componentes clásicos
</div>

Los tiempos de renderizado de los componentes clásicos también están habilitados de forma predeterminada, con una opción para capturar tiempos de renderizado solo cuando superan cierta duración. Para cambiar este mínimo, puedes modificar `minimumComponentRenderDuration` en tu configuración.

```javascript
ENV["@sentry/ember"] = {
  minimumComponentRenderDuration: 0, // Al establecer esto en cero, se capturarán todos los componentes clásicos.
};
```

Para desactivar la instrumentación de componentes, puedes establecer `disableInstrumentComponents` en tu configuración.

```javascript
ENV["@sentry/ember"] = {
  disableInstrumentComponents: true,
};
```

<div id="runloop">
  ### Runloop
</div>

La duración de cada cola en el runloop de tu aplicación se registra de forma predeterminada, siempre que supere el umbral definido en tu configuración por `minimumRunloopQueueDuration`

```javascript
ENV["@sentry/ember"] = {
  minimumRunloopQueueDuration: 0, // Establecerlo en cero capturará todas las duraciones de las colas del runloop
};
```

Si quieres desactivar la instrumentación del runloop, puedes configurar `disableRunloopPerformance` en tu archivo de configuración.

```javascript
ENV["@sentry/ember"] = {
  disableRunloopPerformance: true,
};
```
