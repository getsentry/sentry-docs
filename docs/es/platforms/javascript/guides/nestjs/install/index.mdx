<div id="how-to-decide-which-installation-method-to-use">
  ## Cómo decidir qué método de instalación usar
</div>

La mayoría de las aplicaciones de Node hoy en día están escritas en, o se compilan a, CommonJS (CJS), que usa `require()` para cargar módulos.
Nuestro método de instalación recomendado cuando usas CommonJS es requerir el archivo `instrument.js` al principio de tu aplicación. Sin embargo, si tu aplicación se ejecuta en modo [ECMAScript Modules](https://nodejs.org/api/esm.html#introduction) (ESM), esto no funcionará; en ese caso, puedes seguir la [documentación de ESM](./esm).

Ten en cuenta que, aunque tu aplicación esté escrita en ESM (usando `import`), aún puede _ejecutarse_ en CJS. En ese caso, debes seguir las [instrucciones de CommonJS](./commonjs).

<div id="my-application-uses-the-default-nestjs-setup">
  ### Mi aplicación usa la configuración predeterminada de Nest.js
</div>

Nest.js transpila a CJS por defecto, así que deberías seguir las [instrucciones de instalación de CommonJS](./commonjs). Sigue leyendo si tienes una aplicación o una configuración de compilación más personalizada.

<div id="my-application-uses-require">
  ### Mi aplicación usa `require`
</div>

Si estás usando `require()` en tu aplicación, debes seguir las [instrucciones de CommonJS](./commonjs).

<div id="my-application-uses-import">
  ### Mi aplicación usa `import`
</div>

Si usas `import` en tu aplicación, el método de instalación depende de cómo se _ejecute_. Si compilas tu aplicación (por ejemplo, a una carpeta `/dist` o similar) antes de ejecutarla, revisa cómo queda el código compilado. ¿El código compilado usa `require`? Entonces sigue las [instrucciones de CommonJS](./commonjs). Si el código compilado usa `import`, sigue las [instrucciones de ESM](./esm).

Si no compilas tu código, sigue las [instrucciones de ESM](./esm).
