<div id="step-1-install">
  ## Paso 1: Instalación
</div>

Para instalar Sentry con el asistente de instalación, ejecuta el siguiente comando en tu proyecto:

```bash
npx @sentry/wizard@latest -i nextjs
```

El asistente te guía por el proceso de configuración y te pide habilitar funciones adicionales (opcionales) de Sentry para tu aplicación, más allá del monitoreo de errores.

<PlatformContent includePath="getting-started-features-expandable" />

Esta guía asume que habilitas todas las funciones y permites que el asistente cree una página y una ruta de ejemplo. Puedes agregar o quitar funciones en cualquier momento, pero configurarlas ahora te ahorrará tener que hacerlo manualmente más adelante.

<Expandable title="¿Qué cambia el asistente de instalación dentro de tu aplicación?">
  * Crea archivos de configuración con las llamadas predeterminadas a `Sentry.init()` para todos los entornos de ejecución (Node.js, Browser y Edge)
  * Agrega un hook de instrumentación de Next.js a tu proyecto (`instrumentation.ts`)
  * Crea o actualiza tu configuración de Next.js con los ajustes predeterminados de Sentry
  * Crea componentes de manejo de errores (`global-error.(jsx|tsx)` y `_error.jsx` para el Pages Router) si aún no existen
  * Crea `.sentryclirc` con un token de autenticación para subir mapas de origen (este archivo se agrega automáticamente a `.gitignore`)
  * Agrega una página y una ruta de ejemplo a tu aplicación para ayudarte a verificar tu configuración de Sentry
</Expandable>

<div id="step-2-configure">
  ## Paso 2: Configuración
</div>

Si prefieres configurar Sentry manualmente, estos son los archivos de configuración que crearía el asistente:

<div id="client-side-configuration">
  ### Configuración del cliente
</div>

```javascript {tabTitle:Client} {filename:instrumentation-client.(js|ts)}
import * as Sentry from "@sentry/nextjs";

Sentry.init({
  dsn: "___PUBLIC_DSN___",

  // Agrega los encabezados de la solicitud y la IP de los usuarios; para más información, visita:
  // https://docs.sentry.io/platforms/javascript/guides/nextjs/configuration/options/#sendDefaultPii
  sendDefaultPii: true,

  integrations: [
    // ___PRODUCT_OPTION_START___ session-replay
    // La repetición solo puede habilitarse en el lado del cliente
    Sentry.replayIntegration(),
    // ___PRODUCT_OPTION_END___ session-replay
    // ___PRODUCT_OPTION_START___ user-feedback
    Sentry.feedbackIntegration({
      // La configuración adicional del SDK va aquí, por ejemplo:
      colorScheme: "system",
    }),
    // ___PRODUCT_OPTION_END___ user-feedback
  ],
  // ___PRODUCT_OPTION_START___ logs

  // Habilita el envío de logs a Sentry
  enableLogs: true,
  // ___PRODUCT_OPTION_END___ logs

  // ___PRODUCT_OPTION_START___ performance
  // Establece tracesSampleRate en 1.0 para capturar el 100%
  // de las transacciones de trazado.
  // Recomendamos ajustar este valor en producción
  // Más información en
  // https://docs.sentry.io/platforms/javascript/configuration/options/#traces-sample-rate
  tracesSampleRate: 1.0,
  // ___PRODUCT_OPTION_END___ performance
  // ___PRODUCT_OPTION_START___ session-replay
  // Captura Replay para el 10% de todas las sesiones
  // y para el 100% de las sesiones con un error
  // Más información en
  // https://docs.sentry.io/platforms/javascript/session-replay/configuration/#general-integration-configuration
  replaysSessionSampleRate: 0.1,
  replaysOnErrorSampleRate: 1.0,
  // ___PRODUCT_OPTION_END___ session-replay
});

// ___PRODUCT_OPTION_START___ performance
export const onRouterTransitionStart = Sentry.captureRouterTransitionStart;
// ___PRODUCT_OPTION_END___ performance
```

<div id="server-side-configuration">
  ### Configuración del servidor
</div>

```javascript {tabTitle:Server} {filename:sentry.server.config.(js|ts)}
import * as Sentry from "@sentry/nextjs";

Sentry.init({
  dsn: "___PUBLIC_DSN___",

  // Agrega encabezados de la solicitud y la IP de los usuarios; para más información, visita:
  // https://docs.sentry.io/platforms/javascript/guides/nextjs/configuration/options/#sendDefaultPii
  sendDefaultPii: true,
  // ___PRODUCT_OPTION_START___ logs

  // Habilita el envío de logs a Sentry
  enableLogs: true,
  // ___PRODUCT_OPTION_END___ logs

  // ___PRODUCT_OPTION_START___ performance
  // Establece tracesSampleRate en 1.0 para capturar el 100 %
  // de las transacciones para el rastreo.
  // Recomendamos ajustar este valor en producción
  // Más información en
  // https://docs.sentry.io/platforms/javascript/configuration/options/#traces-sample-rate
  tracesSampleRate: 1.0,
  // ___PRODUCT_OPTION_END___ performance
});
```

Para ver instrucciones detalladas de configuración manual, consulta nuestra [guía de configuración manual](/es/platforms/javascript/guides/nextjs/manual-setup/).

<div id="step-3-verify-your-setup">
  ## Paso 3: Verifica tu configuración
</div>

<Include name="nextjs-turbopack-warning-expandable.mdx" />

Si aún no has probado tu configuración de Sentry, hagámoslo ahora. Puedes confirmar que Sentry funciona correctamente y que envía datos a tu proyecto de Sentry usando la página de ejemplo y la ruta creadas por el asistente de instalación:

1. Abre la página de ejemplo `/sentry-example-page` en tu navegador. En la mayoría de las aplicaciones de Next.js, estará en localhost.
2. Haz clic en el botón &quot;Throw error&quot;. Esto provoca dos errores:

* un error en el frontend
* un error en la ruta de la API

Sentry captura ambos errores por ti. Además, el clic en el botón inicia un rastreo de rendimiento para medir cuánto tarda en completarse la solicitud a la API.

<Alert level="success" title="Consejo">
  No olvides explorar el código de los archivos de ejemplo en tu proyecto para entender qué sucede después de hacer clic en el botón.
</Alert>
