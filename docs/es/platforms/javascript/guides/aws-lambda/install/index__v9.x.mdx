<div id="which-installation-method-should-i-use">
  ## ¿Qué método de instalación debería usar?
</div>

Elegir el método de instalación adecuado para usar Sentry en tus funciones de AWS Lambda depende de un par de factores.

Las funciones Lambda pueden escribirse y ejecutarse en CommonJS, usando la sintaxis `require`, o en EcmaScript Modules (ESM), usando la sintaxis `import`.
Para que Sentry funcione correctamente, debes seguir la guía correspondiente al formato en el que tus funciones se **ejecutan**.

<div id="my-lambda-function-is-written-in-typescript">
  ### Mi función Lambda está escrita en TypeScript
</div>

Si usas TypeScript, es probable que tu función Lambda se transpile a CommonJS antes de ejecutarse. En ese caso, sigue las [instrucciones para CommonJS](#my-lambda-function-uses-require).
Ten en cuenta que TypeScript también puede configurarse para generar ESM; en ese caso, sigue las [instrucciones para ESM](#my-lambda-function-uses-import).

<div id="my-lambda-function-uses-require">
  ### Mi función Lambda usa `require`
</div>

Si usas `require()` en tu función, sigue las instrucciones de CommonJS. Elige entre [usar nuestra capa de Lambda (recomendado)](./install/layer__v9.x) o [instalar el paquete NPM de Sentry para AWS](./install/npm__v9.x).

<div id="my-lambda-function-uses-import">
  ### Mi función Lambda usa `import`
</div>

Si usas la sintaxis `import` en tu función y _no_ estás transpilando el código a CommonJS, sigue las [instrucciones de ESM](./install/esm-npm__v9.x).

<div id="can-i-use-the-lambda-layer-for-esm-functions">
  ### ¿Puedo usar la capa de Lambda para funciones ESM?
</div>

Por ahora, la capa de Lambda solo está disponible para funciones CommonJS. Si usas ESM, tendrás que configurar Sentry manualmente.
Estamos trabajando en una capa de Lambda para ESM, pero primero debemos resolver varias limitaciones relacionadas con OpenTelemetry.

<div id="should-i-use-the-lambda-layer-or-the-npm-package">
  ### ¿Debería usar la capa de Lambda o el paquete de NPM?
</div>

Para empezar, por ahora solo puedes usar la capa en funciones CommonJS. Si usas ESM, tendrás que configurar Sentry manualmente con el paquete de NPM.

En general, recomendamos usar la capa de Lambda porque no requiere desplegar ninguna dependencia de Sentry junto con tu función.
Usando la capa, puedes lograr el mismo nivel de personalización que con el paquete de NPM.
Aun así, hay dos razones por las que podrías preferir usar el paquete de NPM:

1. Quieres minimizar el tamaño de la función Lambda y aplicar tree-shaking a las partes del código del SDK que no necesitas. Una razón relacionada podría ser que estás transpilando tu código y quieres transpilar también tus dependencias.
2. Ya usas paquetes de NPM y despliegas `node_modules` con tu función, y no quieres añadir una (u otra) capa de Lambda a tus funciones.
