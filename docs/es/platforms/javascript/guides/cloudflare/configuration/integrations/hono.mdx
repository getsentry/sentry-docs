_Nombre de importación: `Sentry.honoIntegration`_

Esta integración viene habilitada de forma predeterminada. Si quieres modificar las integraciones predeterminadas, lee [esto](./../#modifying-default-integrations).

`honoIntegration` captura automáticamente los errores desde la función `onError` de Hono y los envía a Sentry. De forma predeterminada, la integración no captura errores con códigos de estado HTTP 3xx o 4xx.

<div id="options">
  ## Opciones
</div>

Puedes configurar `honoIntegration` pasando un objeto de opciones a la función.

<div id="shouldhandleerror">
  ### `shouldHandleError`
</div>

Esta opción te permite proporcionar una función que determine si debe capturarse un error, dándote control total sobre qué errores se envían a Sentry.

La función recibe el error como argumento y debe devolver `true` si el error debe informarse, y `false` en caso contrario.

Por ejemplo, para informar todos los errores excepto los 404, añade esto al array de integraciones al inicializar Sentry:

```javascript
integrations: [
  honoIntegration({
    shouldHandleError(error) {
      // return true // Informaría todos los errores

      if (error instanceof HTTPException && error.status === 404) {
        // No informar 404
        return false;
      }
      // Informar todos los demás errores
      return true;
    },
  }),
]
```
