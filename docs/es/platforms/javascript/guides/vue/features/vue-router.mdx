---

Si usas Vue Router, emplea nuestra instrumentación de enrutamiento para obtener mejores nombres de transacciones parametrizados.

Nuestra instrumentación de enrutamiento es compatible con `vue-router` v2, v3 y v4.

Aquí tienes un ejemplo completo de cómo usarla:

```javascript {filename:main.js}
import Vue from "vue";
import App from "./App";
import * as Sentry from "@sentry/vue";
import Router from "vue-router";
import HelloWorld from "@/components/HelloWorld";
import Foo from "@/components/Foo";
import Bar from "@/components/Bar";

Vue.use(Router);

const router = new Router({
  // configuración de tu enrutador
});

Sentry.init({
  Vue,
  dsn: "___PUBLIC_DSN___",
  tracesSampleRate: 1.0,
  integrations: [
    Sentry.browserTracingIntegration({ router }),
  ],
});

new Vue({
  el: "#app",
  router,
  components: { App },
  template: "<App/>",
});
```


<div id="configuration">
  ## Configuración
</div>

Puedes pasar un objeto de configuración opcional como segundo argumento a la integración de rastreo del navegador:

```javascript
Sentry.browserTracingIntegration({
  router,
  routeLabel: "path",
});
```

Las opciones disponibles son:

| Clave      | Tipo   | Predeterminado | Descripción                                                                                                                                                                                                                                                                     |
| ---------- | ------ | -------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| routeLabel | string | `name`         | La etiqueta que se usará para las transacciones de ruta. Puede ser `name` o `path`. Si es `name`, la transacción usará `route.name` (si está definido); de lo contrario, usará la ruta. Si lo estableces en `path`, puedes evitar este comportamiento y usar siempre el `path`. |
