Los símbolos de depuración te ayudan a obtener trazas de pila legibles, que Sentry muestra en la página de **Detalles de la incidencia**, para ayudarte a priorizarla.

<Alert>
  Actualmente no admitimos la subida automática de mapas de origen que no sean de JavaScript ni scripts de compilación para Android. Debes subir los archivos manualmente [con nuestra CLI](/es/cli/dif/#uploading-files).
</Alert>

<div id="javascript-source-maps">
  ## Mapas de origen de JavaScript
</div>

Los mapas de origen de JavaScript se suben automáticamente durante la compilación al ejecutar `cordova build`.

<div id="uploading-using-xcode">
  ## Carga con Xcode
</div>

Para ejecutar la carga de la información de depuración al compilar tu app, agrega lo siguiente como una fase de “Run Script” en Xcode. Asegúrate también de establecer `DEBUG_INFORMATION_FORMAT` en la configuración de tu proyecto en `DWARF and dSYM file`.

```bash
export SENTRY_PROPERTIES=sentry.properties
function getProperty {
    PROP_KEY=$1
    PROP_VALUE=`cat $SENTRY_PROPERTIES | grep "$PROP_KEY" | cut -d'=' -f2`
    echo $PROP_VALUE
}
if [ ! -f $SENTRY_PROPERTIES ]; then
    echo "advertencia: SENTRY: ¡no se encontró el archivo sentry.properties! Se omitirá la carga de símbolos."
    exit 0
fi
echo "# Leyendo propiedad desde $SENTRY_PROPERTIES"
SENTRY_CLI=$(getProperty "cli.executable")
SENTRY_COMMAND="$SENTRY_CLI debug-files upload \"$DWARF_DSYM_FOLDER_PATH\""
$SENTRY_COMMAND
```
