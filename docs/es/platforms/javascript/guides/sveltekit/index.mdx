<div id="step-1-install">
  ## Paso 1: Instalación
</div>

Para instalar Sentry con el asistente de instalación, ejecuta el siguiente comando en tu proyecto:

```bash
npx @sentry/wizard@latest -i sveltekit
```

El asistente te guía por el proceso de configuración y te pide habilitar funciones adicionales (opcionales) de Sentry para tu aplicación, más allá del monitoreo de errores.

<PlatformContent includePath="getting-started-features-expandable" />

Esta guía asume que habilitas todas las funciones y permites que el asistente cree una página de ejemplo. Puedes agregar o quitar funciones en cualquier momento, pero configurarlas ahora te ahorrará el trabajo de hacerlo manualmente más adelante.

<Expandable title="¿Qué cambia el asistente de instalación dentro de tu aplicación?">
  * Crea o actualiza `hooks.(client|server).js` para inicializar el SDK e instrumentar los [hooks de SvelteKit](https://kit.svelte.dev/docs/hooks)
  * Crea o actualiza `vite.config.js` para añadir la subida de mapas de origen y la autoinstrumentación mediante plugins de Vite
  * Si usas SvelteKit `2.31.0` o superior:
    * Crea `instrumentation.server.js` para inicializar el SDK del lado del servidor
    * Habilita la instrumentación y el trazado del lado del servidor en `svelte.config.js`
  * Crea `.env.sentry-build-plugin` con un token de autenticación para subir mapas de origen (este archivo se agrega automáticamente a `.gitignore`)
  * Añade una página de ejemplo a tu aplicación para ayudarte a verificar tu configuración de Sentry
</Expandable>

<div id="step-3-verify-your-setup">
  ## Paso 3: Verifica tu configuración
</div>

Si aún no has probado tu configuración de Sentry, hagámoslo ahora. Puedes confirmar que Sentry funciona correctamente y envía datos a tu proyecto de Sentry usando la página y la ruta de ejemplo creadas por el asistente de instalación:

1. Abre la página de ejemplo `/sentry-example-page` en tu navegador
2. Haz clic en el botón &quot;Throw Sample Error&quot;. Esto provoca dos errores:
   * un error en el frontend
   * un error dentro de la ruta de la API

Sentry captura ambos errores por ti. Además, al hacer clic en el botón, se inicia un trace para medir el tiempo que tarda en completarse la solicitud a la API.

<Alert level="success" title="Consejo">
  No olvides explorar el código de los archivos de ejemplo en tu proyecto para entender qué ocurre después de hacer clic en el botón.
</Alert>
