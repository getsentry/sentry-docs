Sentry te ayuda a capturar tus componentes de React y obtener información adicional sobre tu aplicación. Puedes configurarlo para usar nombres de componentes de React en lugar de selectores.

Así, en lugar de ver selectores como este:

```html
button.en302zp1.app-191aavw.e16hd6vm2[role="button"][data-test-id="common-options"]
```

También puedes ver exactamente qué componente de React se utilizó, por ejemplo:

```html
CommonOptions
```

<div id="component-name-capturing">
  ## Captura de nombres de componentes
</div>

Puedes capturar los nombres de los componentes de React en tu aplicación mediante un [plugin de Babel](https://www.npmjs.com/package/@sentry/babel-plugin-component-annotate), lo que puede habilitar flujos de trabajo potentes y reducir la ambigüedad. Por ejemplo, puedes buscar reproducciones donde se hizo clic en un componente específico de React. También puedes ver el nombre del componente en el que se hizo clic en breadcrumbs y spans en la página de **Performance** de Sentry. Esto reduce significativamente la ambigüedad a la hora de determinar con qué elemento se interactuó, en comparación con intentar deducirlo mirando solo el selector CSS del elemento (lo cual se vuelve aún más ambiguo tras el proceso de minificación).

Estamos trabajando para lanzar más funciones que aprovechen la captura de nombres de componentes en el futuro y recomendamos encarecidamente que configures tu proyecto para usarla.

<div id="how-to-install">
  ### Cómo instalar
</div>

<Alert>
  Ten en cuenta que tu SDK de navegador de Sentry debe estar en la versión `7.91.0` o superior antes de poder usar estas funciones.
  Solo se pueden rastrear componentes de React en archivos `.jsx` o `.tsx`.
</Alert>

Hay dos formas de configurar la captura de nombres de componentes de React en tu aplicación:

1. Instalando los [plugins del empaquetador de Sentry](https://www.npmjs.com/package/@sentry/bundler-plugin-core)
2. Instalando directamente [@sentry/babel-plugin-component-annotate](https://www.npmjs.com/package/@sentry/babel-plugin-component-annotate)

Recomendamos la primera opción porque los plugins del empaquetador de Sentry incluyen funciones adicionales útiles para mejorar tus flujos de trabajo en Sentry, como: carga automática de sourcemaps,
creación de releases, detección automática del nombre de la release e inyección de la release.

**Configurar usando plugins del empaquetador**:

Sigue las instrucciones en la página de npm del empaquetador que usa tu proyecto:

* [Rollup](https://www.npmjs.com/package/@sentry/rollup-plugin)
* [Vite](https://www.npmjs.com/package/@sentry/vite-plugin)
* [Webpack](https://www.npmjs.com/package/@sentry/webpack-plugin)

Ten en cuenta que, aunque existe un plugin de Sentry para **esbuild**, actualmente no es compatible la captura de nombres de componentes de React.

Una vez que hayas seguido las instrucciones para instalar el plugin del empaquetador y lo hayas agregado a la configuración de tu empaquetador, habilita la captura de nombres de componentes estableciendo su flag en `true`:

```javascript
// Ejemplo específico para Vite, consulta la documentación para otros empaquetadores
sentryVitePlugin({
  // ... otras opciones arriba
    reactComponentAnnotation: {
      enabled: true,
      // puedes ignorar componentes de la anotación con esta opción
      ignoredComponents: ["MyComponentThatShouldNotBeAnnotated"]
    }
}),
```

<Alert level="warning">
  Es posible que te encuentres con errores como `Error: Passing props on "Fragment"!`

  Para evitar estos errores, puedes excluir componentes específicos de la anotación usando la opción `ignoredComponents`, como se muestra en el ejemplo anterior.
</Alert>

<Alert>
  **Si usas Rollup como tu empaquetador**, asegúrate de colocar `sentryRollupPlugin` antes de cualquier otro plugin que transforme tu JSX.
</Alert>

Ahora podrás ver los nombres de tus componentes en breadcrumbs, spans y buscar repeticiones por nombre de componente.
Consulta [esta página](/es/product/reference/search/searchable-properties/session-replay/#clickcomponent_name) para obtener más información sobre cómo buscar reproducciones por nombre de componente.

<div id="how-it-works">
  ### Cómo funciona
</div>

El plugin de Babel analiza el código fuente JSX de tu aplicación en tiempo de compilación y añade atributos `data` adicionales. Estos atributos luego aparecen en los nodos del DOM del HTML generado de tu aplicación, indicando el componente de React y el archivo del que proviene cada nodo.

Por ejemplo, si tuvieras un componente llamado `MyAwesomeComponent` en el archivo `myAwesomeComponent.jsx`:

```javascript
function MyAwesomeComponent() {
  return <div>¡Este es un componente genial e increíble!</div>
}
```

Después de que tu empaquetador aplicara el plugin y compilara tu proyecto, el nodo del DOM resultante se vería así:

```html
<div data-sentry-component="MyAwesomeComponent" data-sentry-source-file="myAwesomeComponent.jsx">¡Este es un componente genial e increíble!</div>
```

El SDK de Sentry para navegadores recogerá el valor de estos atributos `data` y los recopilará cuando los usuarios interactúen con tus componentes.
