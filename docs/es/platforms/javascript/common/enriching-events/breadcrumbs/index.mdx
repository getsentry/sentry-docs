Sentry utiliza *migas de pan* para crear un rastro de eventos que ocurrieron antes de un problema. Estos eventos son muy similares a los registros tradicionales, pero pueden capturar datos estructurados más ricos.

Esta página ofrece una descripción general del registro manual de migas de pan y su personalización. Obtén más información sobre lo que se muestra en la página **Detalles del problema** y cómo puedes filtrar las migas de pan para resolver problemas rápidamente en [Uso de migas de pan](/es/product/error-monitoring/breadcrumbs).

<Alert title="Aprende sobre el uso del SDK">
  Los desarrolladores que deseen modificar la interfaz de migas de pan pueden obtener más información en nuestra [documentación para desarrolladores sobre la interfaz Breadcrumbs](https://develop.sentry.dev/sdk/data-model/event-payloads/breadcrumbs/).
</Alert>

<div id="manual-breadcrumbs">
  ## Migas de pan manuales
</div>

Puedes agregar migas de pan manualmente cuando ocurra algo interesante. Por ejemplo, podrías registrar una miga de pan si el usuario se autentica o se produce otro cambio de estado.

Primero, necesitas importar el SDK, como de costumbre:

<PlatformContent includePath="enriching-events/import" />

Registra manualmente una miga de pan:

<PlatformContent includePath="enriching-events/breadcrumbs/breadcrumbs-example" />

Las claves de migas de pan disponibles son `type`, `category`, `message`, `level`, `timestamp` (que muchos SDK establecerán automáticamente por ti) y `data`, donde puedes incluir cualquier información adicional que quieras que contenga la miga de pan. Usar claves distintas de estas seis no generará un error, pero hará que los datos se omitan cuando Sentry procese el evento.

Puedes elegir entre los siguientes niveles de log de las migas de pan: `"fatal"`, `"error"`, `"warning"`, `"log"`, `"info"` y `"debug"`.
