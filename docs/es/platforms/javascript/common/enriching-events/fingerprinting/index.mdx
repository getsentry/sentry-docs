---
title: "Huella de eventos"
description: "Aprende a sobrescribir la huella predeterminada en tu SDK."
---

Todos los eventos tienen una huella. Los eventos con la misma huella se agrupan en un issue.

De forma predeterminada, Sentry ejecuta uno de nuestros algoritmos de agrupación integrados para generar una huella según la información disponible en el evento, como `stacktrace`, `exception` y `message`. Para ampliar el comportamiento de agrupación predeterminado o cambiarlo por completo, puedes usar una combinación de las siguientes opciones:

1. En tu SDK, usando SDK Fingerprinting, como se documenta a continuación
2. En tu proyecto, usando [Fingerprint Rules](/es/concepts/data-management/event-grouping/fingerprint-rules/) o [Stack Trace Rules](/es/concepts/data-management/event-grouping/stack-trace-rules/)

En los SDK compatibles, puedes sobrescribir la agrupación predeterminada de Sentry pasando el atributo de huella como un array de cadenas. La longitud del array de una huella no está restringida. Esto funciona de forma similar a la [funcionalidad de Fingerprint Rules](/es/concepts/data-management/event-grouping/fingerprint-rules/), que siempre está disponible y puede lograr resultados similares.
