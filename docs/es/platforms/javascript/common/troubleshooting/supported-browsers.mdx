Los SDK más recientes de JavaScript de Sentry requieren compatibilidad con ES2020. Las versiones mínimas de navegador compatibles son:

- Chrome 80
- Edge 80
- Safari 14, iOS Safari 14.4
- Firefox 74
- Opera 67
- Samsung Internet 13.0

Además, los SDK de JavaScript de Sentry requieren que la API `fetch` esté disponible.
Si necesitas admitir entornos tipo navegador que no tengan `fetch` disponible, deberías incluir un polyfill para la API `fetch`.

<div id="supporting-earlier-javascript-versions">
  ## Compatibilidad con versiones anteriores de JavaScript
</div>

El SDK de JavaScript de Sentry utiliza sintaxis y funcionalidades de ES2020 para ofrecer la mejor experiencia posible y el menor tamaño de paquete.
Si quieres admitir versiones anteriores, debes transpilar tu código con un transpilador como Babel, SWC o TypeScript y agregar polyfills.

<div id="8x-browser-support">
  ## Compatibilidad del navegador en 8.x
</div>

Los SDK de JavaScript de Sentry requieren compatibilidad con ES2018 y soporte para [`globalThis`](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/globalThis). Las versiones mínimas de navegador compatibles son:

- Chrome 71
- Edge 79
- Safari 12.1, iOS Safari 12.2
- Firefox 65
- Opera 58
- Samsung Internet 10

<div id="7x-browser-support">
  ## Compatibilidad del navegador en 7.x
</div>

La versión `7.x` de los SDK de JavaScript de Sentry usa sintaxis ES6, junto con otras características del lenguaje ES6+, como el operador de propagación de objetos. Si estás transpiliando tu código para apuntar a navegadores más antiguos que no admiten dicha sintaxis, deberás incluir el SDK de Sentry en ese proceso.
