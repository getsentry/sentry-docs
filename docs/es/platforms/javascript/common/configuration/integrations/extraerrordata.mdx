_Nombre de importación: `Sentry.extraErrorDataIntegration`_

Esta integración extrae todos los atributos no nativos del objeto de error y los añade al evento como datos adicionales. Si el objeto de error tiene un método .toJSON(), la integración ExtraErrorData lo ejecutará para extraer información adicional.

<div id="options">
  ## Opciones
</div>

<div id="depth">
  ### `depth`
</div>

_Tipo: `number`_

Límite de profundidad al que debe llegar el serializador de objetos. El valor predeterminado es 3. Cualquier elemento que supere el límite establecido se
reemplazará con la notación estándar del REPL de Node.js: [Object], [Array], [Function] o
un valor primitivo.

<div id="captureerrorcause">
  ### `captureErrorCause`
</div>

_Tipo: `boolean`_

Indica si el serializador debe capturar la `cause` del error. El valor predeterminado es true. Para obtener más información, consulta [Error: cause en MDN Web Docs](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Error/cause).
