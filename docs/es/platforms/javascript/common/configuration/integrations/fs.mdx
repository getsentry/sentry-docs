<Alert>
  Esta integración solo funciona en los entornos de ejecución de Node.js y Bun.
</Alert>

*Nombre de importación: `Sentry.fsIntegration`*

La `fsIntegration` crea spans para operaciones de la API `fs`, como leer y escribir archivos. La integración usa el paquete [`@opentelemetry/instrumentation-fs`](https://www.npmjs.com/package/@opentelemetry/instrumentation-fs).

<Alert level="warning" title="Posible impacto en el rendimiento">
  La `fsIntegration` puede añadir una sobrecarga significativa a tu aplicación.
  Especialmente en escenarios con mucho I/O de archivos, por ejemplo cuando
  ejecutas un servidor de desarrollo de un framework, incluir esta integración puede
  ralentizar considerablemente tu aplicación.
</Alert>

```JavaScript
Sentry.init({
  integrations: [Sentry.fsIntegration()],
});
```


<div id="options">
  ## Opciones
</div>

<div id="recordfilepaths">
  ### `recordFilePaths`
</div>

_Tipo: `boolean | undefined`_

Establecer esta opción en `true` incluirá cualquier argumento de ruta de archivo de tus llamadas a la API de `fs` como atributos del span.
De forma predeterminada es `false`.

<div id="recorderrormessagesasspanattributes">
  ### `recordErrorMessagesAsSpanAttributes`
</div>

_Tipo: `boolean | undefined`_

Si configuras esta opción en `true`, se incluirán los mensajes de error de las llamadas fallidas a la API de `fs` como atributo del span.
De forma predeterminada es `false`.
