*Import name: `Sentry.supabaseIntegration`*

La `supabaseIntegration` añade instrumentación para el cliente de Supabase y captura spans tanto de autenticación como de operaciones de base de datos.

<div id="installation">
  ## Instalación
</div>

Necesitas tener instalados tanto el SDK de Sentry como la biblioteca de Supabase. Para ver las instrucciones de instalación de Supabase, consulta la [documentación de Supabase JavaScript](https://supabase.com/docs/reference/javascript/introduction).

<div id="configuration">
  ## Configuración
</div>

Este es el método preferido para la mayoría de los casos de uso y sigue el patrón estándar de integración de Sentry.

```javascript
import * as Sentry from '@sentry/browser';
import { createClient } from '@supabase/supabase-js';

const supabaseClient = createClient('TU_URL_SUPABASE', 'TU_CLAVE_SUPABASE');

Sentry.init({
  dsn: 'TU_DSN',
  integrations: [
    Sentry.browserTracingIntegration(),
    Sentry.supabaseIntegration({ supabaseClient })
  ],
  tracesSampleRate: 1.0,
});
```

<div id="generated-spans">
  ## Fragmentos generados
</div>

La integración ofrece monitoreo completo tanto de la autenticación como de las operaciones de base de datos:

<div id="authentication-spans">
  ### Spans de autenticación
</div>

La integración instrumenta automáticamente las siguientes operaciones de autenticación:

* `signInWithPassword`
* `signOut`
* `signInAnonymously`
* `signInWithOAuth`
* `signInWithIdToken`
* `signInWithOtp`
* `signInWithSSO`
* `signUp`
* `verifyOtp`
* `reauthenticate`

También se instrumentan las operaciones de administración:

* `createUser`
* `deleteUser`
* `listUsers`
* `getUserById`
* `updateUserById`
* `inviteUserByEmail`

<div id="database-operation-spans">
  ### Intervalos de operaciones de base de datos
</div>

Estos intervalos se utilizan para alimentar la función [Query Insights](/es/product/insights/backend/queries/) de Sentry, que proporciona métricas de rendimiento y análisis de tus operaciones de base de datos. Con Query Insights, puedes identificar consultas lentas, hacer seguimiento de la frecuencia de consultas y optimizar tus interacciones con la base de datos.

* `db.table`: La tabla consultada
* `db.schema`: El esquema de la base de datos
* `db.url`: La URL de la instancia de Supabase
* `db.sdk`: Información del cliente
* `db.system`: Configurado como &#39;postgresql&#39;
* `db.query`: Los parámetros de la consulta
* `db.body`: El cuerpo de la solicitud (para mutaciones)

<div id="queue-operation-spans-in-progress">
  ### Spans de operaciones de colas (en progreso)
</div>

Próximamente, el SDK de Sentry también admitirá la generación de spans para las interacciones con las colas de Supabase. Para más información, sigue este [issue de GitHub](https://github.com/getsentry/sentry-javascript/issues/14611).

<div id="error-tracking">
  ## Seguimiento de errores
</div>

La integración, automáticamente:

* Captura errores de operaciones fallidas
* Añade breadcrumbs para operaciones de base de datos
* Incluye contexto detallado sobre la operación que falló
