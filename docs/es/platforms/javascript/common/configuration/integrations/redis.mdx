<Alert>
  Esta integración solo funciona en los runtimes de Node.js y Bun.
</Alert>

*Nombre de importación: `Sentry.redisIntegration`*

Esta integración se habilita de forma predeterminada cuando el monitoreo de rendimiento está activado. Si quieres modificar tus integraciones predeterminadas, lee [esto](./../#modifying-default-integrations).

La `redisIntegration` añade instrumentación para las bibliotecas `ioredis` y `redis` para capturar spans usando [`@opentelemetry/instrumentation-ioredis`](https://www.npmjs.com/package/@opentelemetry/instrumentation-ioredis) y [`@opentelemetry/instrumentation-redis-4`](https://www.npmjs.com/package/@opentelemetry/instrumentation-redis-4).

```JavaScript
Sentry.init({
  integrations: [Sentry.redisIntegration()],
});
```


<div id="supported-versions">
  ## Versiones compatibles
</div>

- `ioredis`: `>=2.0.0 <6`
- `redis`: `>=4.0.0`

<div id="options">
  ## Opciones
</div>

<div id="cacheprefixes">
  ### `cachePrefixes`
</div>

_Tipo: `(string)[]`_

Define los prefijos de caché para las claves de caché que deben capturarse como un span de caché. Configurar, por ejemplo, `['user:']` capturará las claves de caché que comiencen con `user:`.
