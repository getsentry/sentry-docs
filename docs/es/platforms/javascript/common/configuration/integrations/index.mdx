El SDK de Sentry utiliza integraciones para conectarse con la funcionalidad de bibliotecas populares, instrumentar tu aplicación automáticamente y ofrecerte los mejores datos desde el primer momento.

Las integraciones añaden automáticamente instrumentación de errores, instrumentación de rendimiento y/o información de contexto adicional a tu aplicación. Algunas se habilitan de forma predeterminada, pero puedes desactivarlas o modificar su configuración.

<div id="modifying-default-integrations">
  ## Modificar las integraciones predeterminadas
</div>

Para desactivar las integraciones del sistema, establece `defaultIntegrations: false` al llamar a `init()`.

Para sobrescribir su configuración, proporciona una nueva instancia con tu configuración en la opción `integrations`. Por ejemplo, para desactivar que el navegador capture las llamadas a la consola:

```javascript
Sentry.init({
  dsn: "___PUBLIC_DSN___",

  integrations: [
    Sentry.linkedErrorsIntegration({
      limit: 7,
    }),
  ],
});
```

</div>

Puedes agregar integraciones adicionales en tu llamada a `init`:

Esto es útil si solo quieres habilitar una integración en un entorno específico o si prefieres cargarla más adelante.
Para los demás casos, recomendamos usar la opción `integrations`.

<div id="lazy-loading-integrations">
  ## Carga diferida de integraciones
</div>

La carga diferida te permite agregar integraciones conectables sin aumentar el tamaño inicial del bundle. Puedes hacerlo de dos maneras:

<div id="1-dynamic-import-recommended">
  ### 1. Importación dinámica (recomendado)
</div>

Puedes agregar la integración con una importación dinámica usando `import()`. Este método carga la integración desde el paquete de npm. Para evitar problemas con `import()`, usa un empaquetador que admita importaciones dinámicas. Si estás usando una herramienta como Vite en tu proyecto, es probable que el proceso de empaquetado ya esté configurado.

<PlatformContent includePath="configuration/integrations/lazy-loading-replay" />

<div id="2-load-from-cdn-with-lazyloadintegration">
  ### 2. Cargar desde el CDN con `lazyLoadIntegration()`
</div>

También puedes cargar de forma diferida integraciones conectables mediante `Sentry.lazyLoadIntegration()`. Esto intentará cargar la integración desde el CDN de Sentry.
Ten en cuenta que esta función generará un rechazo si no logra cargar la integración desde el CDN de Sentry, lo cual puede suceder si un usuario tiene un bloqueador de anuncios o si hay un problema de red. Asegúrate siempre de manejar los rechazos de esta función en tu aplicación.

```javascript
async function loadHttpClient() {
  const httpClientIntegration = await Sentry.lazyLoadIntegration(
    "httpClientIntegration"
  );
  Sentry.addIntegration(httpClientIntegration());
}
```

La carga diferida está disponible para las siguientes integraciones:

- `replayIntegration`
- `replayCanvasIntegration`
- `feedbackIntegration`
- `feedbackModalIntegration`
- `feedbackScreenshotIntegration`
- `captureConsoleIntegration`
- `contextLinesIntegration`
- `linkedErrorsIntegration`
- `dedupeIntegration`
- `extraErrorDataIntegration`
- `httpClientIntegration`
- `reportingObserverIntegration`
- `rewriteFramesIntegration`
- `browserProfilingIntegration`

</PlatformCategorySection>

<div id="removing-a-default-integration">
  ## Eliminar una integración predeterminada
</div>

Si solo quieres eliminar una o algunas de las integraciones predeterminadas, en lugar de deshabilitarlas todas con `defaultIntegrations: false`, puedes usar la siguiente sintaxis para excluir las que no quieras.

Este ejemplo elimina la integración que agrega breadcrumbs al evento, la cual está habilitada de forma predeterminada:

```javascript
Sentry.init({
  // ...
  integrations: function (integrations) {
    // integrations contendrá todas las integraciones por defecto
    return integrations.filter(function (integration) {
      return integration.name !== "Breadcrumbs";
    });
  },
});
```
