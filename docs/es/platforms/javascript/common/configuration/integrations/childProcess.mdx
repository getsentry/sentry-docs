<Alert>
  Esta integración solo funciona en Node.js `>=20` y requiere la versión `8.39.0` o posterior del SDK.
</Alert>

*Nombre de importación: `Sentry.childProcessIntegration`*

Esta integración está habilitada de forma predeterminada. Si quieres modificar las integraciones predeterminadas, consulta <PlatformLink to="/configuration/integrations/#modifying-default-integrations">Cómo modificar las integraciones predeterminadas</PlatformLink>.

La `childProcessIntegration` captura breadcrumbs e eventos de errores de `child_process` y `worker_threads`, así como códigos de salida distintos de cero de `child_process`.

```JavaScript
Sentry.init({
  integrations: [Sentry.childProcessIntegration()],
});
```


<div id="options">
  ## Opciones
</div>

<div id="includechildprocessargs">
  ### `includeChildProcessArgs`
</div>

_Tipo: `boolean`_

Si se establece en `true`, la integración incluirá los argumentos utilizados para iniciar procesos secundarios.

<div id="captureworkererrors">
  ### `captureWorkerErrors`
</div>

_Tipo: `boolean`_

De forma predeterminada, esta opción es `true`. Si se establece en `false`, la integración no capturará errores de hilos worker.
