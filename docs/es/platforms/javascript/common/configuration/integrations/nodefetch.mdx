<Alert>
  Esta integración solo funciona en los entornos de ejecución (runtimes) de Node.js y Bun.
</Alert>

*Nombre de importación: `Sentry.nativeNodeFetchIntegration`*

Esta integración está habilitada de forma predeterminada a partir de la v8.0.0. Si quieres modificar tus integraciones predeterminadas, lee [esto](./../#modifying-default-integrations).

La `nativeNodeFetchIntegration` hace dos cosas:

1. Captura spans de solicitudes fetch.
2. Captura breadcrumbs de solicitudes fetch.

```JavaScript
Sentry.init({
  integrations: [Sentry.nativeNodeFetchIntegration()],
});
```


<div id="options">
  ## Opciones
</div>

<div id="breadcrumbs">
  ### `breadcrumbs`
</div>

_Tipo: `boolean`_

Si se establece en false, no se capturarán las migas de navegación.

<div id="ignoreoutgoingrequests">
  ### `ignoreOutgoingRequests`
</div>

_Tipo: `(url: string) => boolean`_

Permite definir un método para filtrar las solicitudes salientes según la URL. Si el método devuelve `true`, la solicitud se ignorará.

<div id="spans">
  ### `spans`
</div>

_Tipo: `boolean`_

Si se establece en `false`, no se capturarán span.

<div id="requesthook">
  ### `requestHook`
</div>

_Tipo: `(span: Span, request: Request) => void`_

Una función de devolución de llamada que te permite agregar atributos personalizados o modificar el span para solicitudes `fetch` salientes. La función se invoca con el span y el objeto nativo `Request` de fetch.

<div id="responsehook">
  ### `responseHook`
</div>

_Tipo: `(span: Span, response: Response) => void`_

Una función de callback que permite añadir atributos personalizados o modificar el span en función de la respuesta de las solicitudes fetch salientes. La función se invoca con el span y el objeto nativo `Response` de fetch.
