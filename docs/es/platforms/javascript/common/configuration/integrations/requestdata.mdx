<Alert>

Esta integración solo funciona en entornos de servidor (Node.js, Bun, Deno).

</Alert>

_Nombre de la importación: `Sentry.requestDataIntegration`_

Esta integración está habilitada de forma predeterminada. Si quieres modificar tus integraciones predeterminadas, lee [esto](./../#modifying-default-integrations).

Esta integración añade datos de solicitudes entrantes a los eventos de transacción y de error que ocurren durante el manejo de la solicitud.

<PlatformContent includePath="configuration/requestdata" />

<div id="options">
  ## Opciones
</div>

<div id="include">
  ### `include`
</div>

Al pasar la opción `include`, puedes definir qué datos de una solicitud quieres capturar:

```javascript
requestDataIntegration({
  // Controla qué tipos de datos se añaden al evento
  include: {
    cookies: boolean  // default: true,
    data: boolean  // default: true,
    headers: boolean  // default: true,
    ip: boolean  // default: false,
    query_string: boolean  // default: true,
    url: boolean  // default: true,
    user: boolean | {
      id: boolean  // default: true,
      username: boolean  // default: true,
      email: boolean  // default: true,
    },
  },
  // Controla cómo se reportará la transacción. Las opciones son 'path' (`/some/route`),
  // 'methodPath' (`GET /some/route`) y 'handler' (el nombre de la función manejadora de la ruta,
  // si está disponible)
  transactionNamingScheme: string  // valor predeterminado: 'methodPath',
})
```
