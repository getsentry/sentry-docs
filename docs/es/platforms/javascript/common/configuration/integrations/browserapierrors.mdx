<Alert>
  Esta integración solo funciona en un entorno de navegador.
</Alert>

*Nombre de importación: `Sentry.browserApiErrorsIntegration`*

Esta integración está habilitada de forma predeterminada. Si quieres modificar tus integraciones predeterminadas, lee [esto](./../#modifying-default-integrations).

Esta integración envuelve las API nativas de tiempo y eventos (`setTimeout`, `setInterval`, `requestAnimationFrame`, `addEventListener/removeEventListener`) en bloques `try/catch` para controlar excepciones asíncronas.

```JavaScript
Sentry.init({
  integrations: [
    Sentry.browserApiErrorsIntegration({
      setTimeout: true,
      setInterval: true,
      requestAnimationFrame: true,
      XMLHttpRequest: true,
      eventTarget: true,
      unregisterOriginalCallbacks: true,
    }),
  ],
});
```

<div id="eventtarget">
  ### `eventTarget`
</div>

_Tipo: `boolean | string[]`_

Instrumenta el método nativo del navegador `addEventListener` para un conjunto de destinos de eventos predeterminados. Para reemplazar los destinos predeterminados, proporciona un array de cadenas con los nombres de los destinos.

Lista de destinos de eventos predeterminados:

- `EventTarget`
- `Window`
- `Node`
- `ApplicationCache`
- `AudioTrackList`
- `BroadcastChannel`
- `ChannelMergerNode`
- `CryptoOperation`
- `EventSource`
- `FileReader`
- `HTMLUnknownElement`
- `IDBDatabase`
- `IDBRequest`
- `IDBTransaction`
- `KeyOperation`
- `MediaController`
- `MessagePort`
- `ModalWindow`
- `Notification`
- `SVGElementInstance`
- `Screen`
- `SharedWorker`
- `TextTrack`
- `TextTrackCue`
- `TextTrackList`
- `WebSocket`
- `WebSocketWorker`
- `Worker`
- `XMLHttpRequest`
- `XMLHttpRequestEventTarget`
- `XMLHttpRequestUpload`

<div id="unregisteroriginalcallbacks">
  ### `unregisterOriginalCallbacks`
</div>

_Tipo: `boolean`_

Anula el registro de los callbacks originales de `EventTarget.addEventListener`.
Si tienes problemas con esta integración (o el SDK) que provoquen invocaciones dobles de un callback de `addEventListener`, establece esta opción en `true`.
Esto suele indicar que el SDK se inicializó demasiado tarde en el ciclo de vida de la página.
Si es así, considera inicializar el SDK lo antes posible para evitar este problema.
