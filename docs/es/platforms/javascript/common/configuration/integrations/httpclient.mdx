<Alert>

Esta integración solo funciona en el entorno del navegador.

</Alert>

_Nombre de importación: `Sentry.httpClientIntegration`_

Esta integración captura errores en solicitudes fallidas de Fetch y XHR y adjunta información de la solicitud y la respuesta.

De forma predeterminada, los eventos de error no incluyen datos de encabezados ni cookies. Puedes cambiar este comportamiento estableciendo `sendDefaultPii: true` en la configuración raíz de `Sentry.init({})`.

<PlatformContent includePath="configuration/http-client" />

<Alert level="warning" title="Nota">

Debido a las limitaciones de las API de Fetch y XHR, la recopilación de cookies y encabezados de las solicitudes y respuestas se realiza según el mejor esfuerzo posible. Esto significa que ciertos encabezados pueden faltar en el evento creado por la integración.

</Alert>

<div id="options">
  ## Opciones
</div>

<div id="failedrequeststatuscodes">
  ### `failedRequestStatusCodes`
</div>

_Tipo: `(number|[number, number])[]`_

Este arreglo puede contener tuplas de `[begin, end]` (ambos inclusive),
códigos de estado individuales o una combinación de ambos.
Valor predeterminado: `[[500, 599]]`

<div id="failedrequesttargets">
  ### `failedRequestTargets`
</div>

_Tipo: `(string|RegExp)[]`_

Un arreglo de destinos de solicitud que deben considerarse; por ejemplo, `['http://example.com/api/test']` hará que cualquier solicitud a esta URL se interprete como un fallo. Este arreglo puede contener expresiones regulares, cadenas o una combinación de ambas.
Valor predeterminado: `[/.*/]`

<div id="senddefaultpii">
  ### `sendDefaultPii`
</div>

_Tipo: `boolean`_

<Alert type="warning">
  Esta opción debe especificarse en las opciones de nivel raíz de Sentry.init, ¡no en las
  opciones de la integración!
</Alert>

Esta opción es necesaria para capturar encabezados y cookies.
