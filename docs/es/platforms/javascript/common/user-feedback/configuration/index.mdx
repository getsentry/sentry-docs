  ## Widget de Opiniones del Usuario

  El Widget de Comentarios de Usuario ofrece muchas opciones de personalización y, si las opciones disponibles no son suficientes, puedes [usar tu propia interfaz](#bring-your-own-widget).

  Las siguientes opciones de configuración se aplican a la versión 8.0.0 y posteriores del SDK de JavaScript. El Widget de Feedback en la versión 7 del SDK fue un lanzamiento beta y ahora está descontinuado.

  ![Imagen que muestra las opciones de personalización disponibles para el Widget de Comentarios de Usuario](./img/user-feedback-widget-customization.png)

  ### General

  Las siguientes opciones pueden configurarse para la integración en `feedbackIntegration({})`:

  | Key            | Type                                                                                   | Default           | Description                                                                                                                                                                                                                                |
  | -------------- | -------------------------------------------------------------------------------------- | ----------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | ---------- | ------------------------------------------------------------------------ |
  | `autoInject`   | `boolean`                                                                              | `true`            | Inserta el widget de comentarios en la aplicación cuando se agrega la integración. Establece `autoInject: false` si quieres llamar directamente a `feedback.attachTo()` o `feedback.createWidget()`, o si solo quieres mostrar el widget en determinadas vistas. |
  | `showBranding` | `boolean`                                                                              | `true`            | Muestra el logotipo de Sentry dentro del formulario.                                                                                                                                                                                       |
  | `colorScheme`  | <code>&quot;system&quot;                                                               | &quot;light&quot; | &quot;dark&quot;</code>                                                                                                                                                                                                                    | `"system"` | Muestra las opciones de tema de color. `"system"` usará el esquema de color de tu sistema operativo. |
  | `id`           | `string`                                                                               | `sentry-feedback` | El atributo `id` del `<div>` que contiene el widget de comentarios. Consulta [Personalización de CSS](#css-customization) para obtener más información.                                                                                     |
  | `tags`         | `Record<string, number \| string \| boolean \| bigint \| symbol \| null \| undefined>` | `{}`              | <PlatformLink to="/enriching-events/tags/">Etiquetas</PlatformLink> para establecer en el evento de comentarios.                                                                                                                           |

  ### Inyección automática vs. inyección manual

  Ya sea que quieras usar la inyección automática o controlar las cosas manualmente, primero debes pasar la integración a través del array `integrations` en `Sentry.init()`.

  Si tienes `autoInject: true`, se insertará un botón en la página que activará la aparición del formulario para que el usuario pueda ingresar sus comentarios. Si en cambio quieres controlar cuándo ocurre esta inyección, llama al método `feedback.createWidget()` para obtener una referencia a un objeto `Actor`, y luego llama a `appendToDom()` para insertarlo en la página.

  <PlatformContent includePath="user-feedback/manual-injection" />

  Lee más sobre cómo [usar tu propio botón](#bring-your-own-button) a continuación.

  ### Usuario y formulario

  Se pueden configurar las siguientes opciones de la integración en `feedbackIntegration({})`:

  | Clave              | Tipo                     | Predeterminado                        | Descripción                                                                                                                                  |
  | ------------------ | ------------------------ | ------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------- |
  | `showName`         | `boolean`                | `true`                                | Muestra el campo de nombre en el formulario de feedback.                                                                                     |
  | `showEmail`        | `boolean`                | `true`                                | Muestra el campo de correo electrónico en el formulario de feedback.                                                                         |
  | `enableScreenshot` | `boolean`                | `true`                                | Permite al usuario enviar una captura de pantalla adjunta con su feedback. Para instalaciones self-hosted, también se requiere la versión 24.4.2. |
  | `isNameRequired`   | `boolean`                | `false`                               | Hace obligatorio el campo de nombre en el formulario de feedback.                                                                            |
  | `isEmailRequired`  | `boolean`                | `false`                               | Hace obligatorio el campo de correo electrónico en el formulario de feedback.                                                                |
  | `useSentryUser`    | `Record<string, string>` | `{ email: 'email', name: 'username'}` | Establece los campos `email` y `name` según los campos de usuario correspondientes del SDK de Sentry definidos con `Sentry.setUser`.        |

  #### Contexto del usuario

  Si has establecido un contexto de usuario llamando a <PlatformLink to="/enriching-events/identify-user/">`Sentry.setUser()`</PlatformLink>, esos valores se usarán como predeterminados para los campos `name` y `email`. Si esos campos están ocultos al usuario, los valores predeterminados se enviarán igualmente junto con el mensaje de feedback.

  A continuación se muestra un ejemplo de configuración con campos de usuario personalizados.

  ```javascript
  Sentry.setUser({
    fullName: "Jane Doe",
    email: "foo@example.com",
  });

  Sentry.init({
    dsn: "___PUBLIC_DSN___",
    integrations: [
      Sentry.feedbackIntegration({
        useSentryUser: {
          name: "fullName",
          email: "email",
        },
      }),
    ],
  });
  ```

  ### Personalización de texto

  La mayoría del texto que ves en el widget de Feedback predeterminado puede personalizarse.

  Las siguientes opciones pueden configurarse para la integración en `feedbackIntegration({})`:

  | Clave                         | Predeterminado                           | Descripción                                                                                                               |
  | ----------------------------- | ---------------------------------------- | ------------------------------------------------------------------------------------------------------------------------- | ---------------- | ---------------------------------------------------------------------------------------------------------------------- |
  | `triggerLabel`                | `"Report a Bug"`                         | La etiqueta del botón insertado que abre el formulario de comentarios al hacer clic.                                      |
  | `triggerAriaLabel`            | `triggerLabel`                           |                                                                                                                           | `"Report a Bug"` | La etiqueta aria del botón insertado que abre el formulario de comentarios al hacer clic. Disponible desde el SDK v8.20.0. |
  | `formTitle`                   | `"Report a Bug"`                         | El título en la parte superior del formulario de comentarios.                                                             |
  | `submitButtonLabel`           | `"Send Bug Report"`                      | La etiqueta del botón de envío usado en el formulario de comentarios.                                                     |
  | `cancelButtonLabel`           | `"Cancel"`                               | La etiqueta de los botones de cancelar usados en el formulario de comentarios.                                            |
  | `confirmButtonLabel`          | `"Confirm"`                              | La etiqueta de los botones de confirmación usados en el formulario de comentarios.                                        |
  | `addScreenshotButtonLabel`    | `"Add a screenshot"`                     | La etiqueta del botón para añadir una captura de pantalla al formulario.                                                  |
  | `removeScreenshotButtonLabel` | `"Remove screenshot"`                    | La etiqueta del botón para eliminar la captura de pantalla del formulario.                                                |
  | `nameLabel`                   | `"Name"`                                 | La etiqueta del campo de nombre.                                                                                           |
  | `namePlaceholder`             | `"Your Name"`                            | El marcador de posición del campo de nombre.                                                                               |
  | `emailLabel`                  | `"Email"`                                | La etiqueta del campo de correo electrónico.                                                                               |
  | `emailPlaceholder`            | `"your.email@example.org"`               | El marcador de posición del campo de correo electrónico.                                                                   |
  | `isRequiredLabel`             | `"(required)"`                           | La etiqueta que se muestra junto a un campo obligatorio.                                                                   |
  | `messageLabel`                | `"Description"`                          | La etiqueta del campo de descripción de los comentarios.                                                                   |
  | `messagePlaceholder`          | `"What's the bug? What did you expect?"` | El marcador de posición del campo de descripción de los comentarios.                                                       |
  | `successMessageText`          | `"Thank you for your report!"`           | El mensaje que se muestra tras enviar los comentarios correctamente.                                                       |
  | `highlightToolText`           | `"Highlight"`                            | La etiqueta del botón que resalta partes de la captura de pantalla. Disponible desde el SDK v10.10.0.                     |
  | `hideToolText`                | `"Hide"`                                 | La etiqueta del botón que oculta partes de la captura de pantalla. Disponible desde el SDK v10.10.0.                      |
  | `removeHighlightText`         | `"Remove"`                               | La etiqueta del botón que elimina una sección resaltada/oculta de la captura de pantalla. Disponible desde el SDK v10.10.0. |

  Ejemplo de personalización:

  ```javascript
  Sentry.init({
    dsn: "___PUBLIC_DSN___",
    integrations: [
      Sentry.feedbackIntegration({
        buttonLabel: "Comentarios",
        submitButtonLabel: "Enviar comentarios",
        formTitle: "Enviar comentarios",
      })
    ],
  });
  ```

  ### Personalización de CSS

  Puedes personalizar la ubicación de los componentes de feedback en tu página, así como las fuentes y los colores del tema para el modo claro y oscuro.

  En todos los casos, puedes establecer variables CSS para sobrescribir cualquier valor. Por defecto, el contenedor `<div>` tiene el atributo `id="sentry-feedback"`, por lo que puedes usar el selector `#sentry-feedback` para definir variables CSS que sobrescriban los valores predeterminados.

  El siguiente ejemplo muestra cómo personalizar el color de fondo para los temas claro y oscuro mediante la sobrescritura de variables CSS:

  ```css
  #sentry-feedback {
    --trigger-background: #cccccc;
  }
  @media (prefers-color-scheme: dark) {
    #sentry-feedback {
      --trigger-background: #222222;
    }
  }
  ```

  También puedes hacer lo mismo configurando los valores del tema en JavaScript:

  ```javascript
  Sentry.init({
    dsn: "___PUBLIC_DSN___",
    integrations: [
      Sentry.feedbackIntegration({
        themeLight: {
          background: "#cccccc",
        },
        themeDark: {
          background: "#222222",
        },
      }),
    ],
  });
  ```

  Los siguientes valores solo están disponibles como variables CSS y se aplican tanto al tema oscuro como al claro.

  | Variable CSS    | Predeterminado                                     | Descripción                                                                                                                                                                   |
  | --------------- | -------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
  | `--font-family` | `"system-ui, 'Helvetica Neue', Arial, sans-serif"` | La familia de fuentes predeterminada a utilizar.                                                                                                                              |
  | `--font-size`   | `14px`                                             | El tamaño de la fuente.                                                                                                                                                       |
  | `--z-index`     | `100000`                                           | El z-index del widget.                                                                                                                                                        |
  | `--inset`       | `auto 0 0 auto`                                    | Por defecto, el widget tiene posición fija y se ubica en la esquina inferior derecha.                                                                                         |
  | `--page-margin` | `16px`                                             | El margen desde el borde de la pantalla en el que debe posicionarse el widget. También acepta valores abreviados como `10px 20px 30px 10px` para márgenes superior/derecho/inferior/izquierdo. |

  Los colores se pueden personalizar definiendo variables CSS que reemplacen los valores predeterminados, o pasando `themeLight` y/o `themeDark` a `feedbackIntegration({})`.

  | Variable CSS          | Clave de Configuración | Modo Claro Predeterminado                 | Modo Oscuro Predeterminado                | Descripción                                                                                                                      |
  | --------------------- | ---------------------- | ----------------------------------------- | ----------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------- |
  | `--foreground`        | `foreground`           | `#2b2233`                                 | `#ebe6ef`                                 | Color de primer plano (texto).                                                                                                   |
  | `--background`        | `background`           | `#ffffff`                                 | `#29232f`                                 | Color de fondo del widget (botón y formulario inyectados).                                                                       |
  | `--accent-foreground` | `accentForeground`     | `#ffffff`                                 | `#ffffff`                                 | Color de primer plano para el botón de envío.                                                                                    |
  | `--accent-background` | `accentBackground`     | `rgba(88, 74, 192, 1)`                    | `rgba(88, 74, 192, 1)`                    | Color de fondo para el botón de envío.                                                                                           |
  | `--success-color`     | `successColor`         | `#268d75`                                 | `#2da98c`                                 | Color utilizado para componentes relacionados con éxito (como el color del texto cuando se envía correctamente el feedback). |
  | `--error-color`       | `errorColor`           | `#df3338`                                 | `#f55459`                                 | Color utilizado para componentes relacionados con errores (como el color del texto cuando hay un error al enviar el feedback). |
  | `--box-shadow`        | `boxShadow`            | `0px 4px 24px 0px rgba(43, 34, 51, 0.12)` | `0px 4px 24px 0px rgba(43, 34, 51, 0.12)` | Estilo de sombra utilizado para el widget (botón y formulario inyectados).                                                       |
  | `--outline`           | `outline`              | `1px auto var(--accent-background)`       | `1px auto var(--accent-background)`       | Contorno para los campos del formulario cuando están enfocados.                                                                  |

  #### Prioridad del tema

  Las variables CSS tienen prioridad sobre los valores de configuración en `feedbackIntegration()`. En este ejemplo, la configuración hace parecer que el texto podría ser `white` o `black`, pero como hay una variable CSS definida, el texto siempre será `red`.

  ```html
  <script>
    Sentry.init({
      dsn: "___PUBLIC_DSN___",
      integrations: [
        Sentry.feedbackIntegration({
          themeLight: {
            foreground: "#000",
          },
          themeDark: {
            foreground: "#fff",
          },
        }),
      ],
    });
  </script>

  <style>
    #sentry-feedback {
      --foreground: "rojo"; /* reemplaza tanto `#fff` como `#000` anteriores */
    }
  </style>
  ```

  #### Selector de ID personalizado

  Es posible configurar el valor de `id` con algo distinto al predeterminado `sentry-feedback` y usarlo como selector al sobrescribir variables CSS.

  ```html
  <script>
    Sentry.init({
      dsn: "___PUBLIC_DSN___",
      integrations: [
        Sentry.feedbackIntegration({
          id: "feedback-theme", // El valor por defecto es 'sentry-feedback'
        }),
      ],
    });
  </script>

  <style>
    /* Seleccionar el id personalizado */
    #feedback-theme {
      --foreground: "red";
    }
  </style>
  ```

  ### Callbacks de eventos

  Como a veces es útil saber cuándo un usuario empezó a interactuar con el formulario de comentarios, hemos hecho posible que agregues registros personalizados o inicies y detengas temporizadores en segundo plano en la página que te indiquen cuándo terminó el usuario.

  Las siguientes opciones pueden configurarse para la integración en `feedbackIntegration({})`:

  ```javascript
  Sentry.init({
    dsn: "___PUBLIC_DSN___",
    integrations: [
      Sentry.feedbackIntegration({
        onFormOpen: () => {},
        onFormClose: () => {},
        onSubmitSuccess: (data: FeedbackFormData, eventID: string) => {},
        onSubmitError: (error: Error) => {},
      }),
    ],
  });
  ```

  Puedes enlazar directamente a un elemento de feedback guardado implementando el callback `onSubmitSuccess`. La URL que debes construir es `https://${orgSlug}.sentry.io/issues/feedback/?projectSlug=${projectSlug}&eventId=${eventId}`.

  *Nota:* En la v9 y anteriores del SDK, la firma de `onSubmitSuccess` era `(data: FeedbackFormData) => {}`.

  ### Trae tu propio Button

  Puedes usar tu propio botón en lugar del botón inyectado por defecto para activar la visualización del formulario, llamando a `feedback.attachTo()` para que el SDK adjunte un listener de clic a tu botón. También puedes proporcionar las mismas opciones de personalización que acepta el constructor (por ejemplo, para etiquetas de texto y colores).

  ```javascript
  Sentry.init({
    dsn: "___PUBLIC_DSN___",
    integrations: [
      Sentry.feedbackIntegration({
        // Desactivar la inyección del widget predeterminado
        autoInject: false,
      })
    ],
  });

  // Obtener la instancia devuelta por `feedbackIntegration()`
  const feedback = Sentry.getFeedback();

  feedback?.attachTo(document.querySelector("#your-button"), {
    formTitle: "¡Reportar un error!",
  });
  ```

  ```typescript {tabTitle: NextJs}
  Sentry.init({
    dsn: "___PUBLIC_DSN___",
    integrations: [
      Sentry.feedbackIntegration({
        // Desactivar la inyección del widget predeterminado
        autoInject: false,
      })
    ],
  });

  function AttachToFeedbackButton() {
    const [feedback, setFeedback] = useState();
    // Leer `getFeedback` solo en el cliente, para evitar errores de hidratación durante el renderizado del servidor
    useEffect(() => {
      setFeedback(Sentry.getFeedback());
    }, []);

    const buttonRef = useRef();
    useEffect(() => {
      if (feedback) {
        const unsubscribe = feedback.attachTo(buttonRef.current)
        return unsubscribe;
      }
      return () => {};
    }, [feedback]);

    return (
      <button type="button" ref={buttonRef}>
        Enviar comentarios
      </button>
    );
  }
  ```

  Alternativamente, puedes llamar a `feedback.createForm()` y tener control total sobre cuándo se carga el formulario, se agrega al DOM y finalmente se muestra al usuario.

  ```javascript
  Sentry.init({
    dsn: "___PUBLIC_DSN___",
    integrations: [
      Sentry.feedbackIntegration({
        // Desactiva la inserción del widget predeterminado
        autoInject: false,
      })
    ],
  });

  // Obtén la instancia que devuelve `feedbackIntegration()`
  const feedback = Sentry.getFeedback();

  const form = await feedback?.createForm();
  form.appendToDom();
  form.open();
  ```

  ```typescript {tabTitle: NextJS}
  function CreateFeedbackFromButton() {
    const [feedback, setFeedback] = useState();
    // Lee `getFeedback` solo en el cliente para evitar errores de hidratación durante el renderizado del servidor
    useEffect(() => {
      setFeedback(Sentry.getFeedback());
    }, []);

    // No renderices el botón de comentarios personalizado si la integración de Feedback no está instalada
    if (!feedback) {
      return null;
    }

    return (
      <button
        type="button"
        onClick={async () => {
          const form = await feedback.createForm();
          form.appendToDom();
          form.open();
        }}
      >
        Enviar comentarios
      </button>
    );
  }
  ```

  ### Trae tu propio widget

  También puedes usar tus propios componentes de UI para recopilar feedback y pasar el objeto de datos de feedback a la función `captureFeedback()`. La función `captureFeedback` acepta dos parámetros:

  * Un objeto de JavaScript con una propiedad obligatoria `message` y las propiedades opcionales `name` y `email`.
  * Un objeto opcional de «hints».

  ```javascript
  Sentry.captureFeedback(
    {
      name: "Jane Doe", // opcional
      email: "email@example.org", // opcional
      message: "Este es un ejemplo de comentario", // obligatorio
    },
    {
      includeReplay: true, // opcional
      attachments: [], // opcional
    }
  );
  ```

  Aquí tienes un ejemplo sencillo:

  ```html
  <form id="my-feedback-form">
    <input name="name" placeholder="Tu nombre" />
    <input name="email" placeholder="Tu correo electrónico" />
    <textarea name="message" placeholder="¿Cuál es el problema?" />
    <input type="file" name="attachment" />
    <button type="submit">Enviar</button>
  </form>
  ```

  ```javascript
  import { captureFeedback } from "@sentry/react";

  document
    .getElementById("my-feedback-form")
    .addEventListener("submit", (event) => {
      event.preventDefault();

      const formData = new FormData(event.currentTarget);
      const attachment = async () => {
        const attatchmentField = formData.get("attachment") as File;
        const data = new Uint8Array(
          await attatchmentField.arrayBuffer()
        );
        const attachmentData = {
          data,
          filename: 'upload', // O pasar attatchmentField.name,
          // No es necesario establecer `contentType` porque está codificado en el array `data`
        };
        return attachmentData;
      };

      Sentry.captureFeedback({
        name: String(formData.get('name')),
        email: String(formData.get('email')),
        message: String(formData.get('message')),
      }, {
        attachments: [await attachment()],
      });
    });
  ```

<div id="loading-strategies">
  ### Estrategias de carga
</div>

Dado que `feedbackIntegration` es una integración orientada al usuario, ofrecemos dos estrategias de carga con implicaciones en el tamaño del paquete.

<Alert>
  Para la mayoría de los usuarios, recomendamos usar `feedbackIntegration` en la llamada a `Sentry.init`.
  Esto configurará los comentarios de usuarios con valores predeterminados adecuados, en consonancia con el
  entorno.
</Alert>

Todos los ejemplos de código de esta página usan `feedbackIntegration` como predeterminado porque está disponible independientemente de si elegiste el método de instalación mediante CDN o NPM. Sin embargo, la implementación de `feedbackIntegration` difiere entre ambos métodos de instalación. Para usuarios de NPM, `feedbackIntegration` es un alias de `feedbackSyncIntegration`. Para usuarios de CDN, `feedbackIntegration` es un alias de `feedbackAsyncIntegration`.

<div id="feedbacksyncintegration">
  #### `feedbackSyncIntegration`
</div>

Si instalaste el SDK con NPM, esta estrategia de carga se usa de forma predeterminada. La estrategia no está disponible si instalaste el SDK mediante el CDN.

Esta integración incluye todo el código necesario para renderizar el Button "Send Feedback", así como el formulario que se activa al hacer clic en el Button. Elegir esta estrategia de carga implica aceptar el mayor tamaño de paquete inicial en tu aplicación. La ventaja es que el widget de feedback se abrirá con seguridad cuando el usuario interactúe con él; en cualquier caso, se requiere conexión de red para enviar el mensaje de feedback.

<div id="feedbackasyncintegration">
  #### `feedbackAsyncIntegration`
</div>

Si instalaste el SDK con el CDN, esta estrategia de carga se usa de forma predeterminada. Si usaste NPM, puedes seguir eligiendo esta estrategia de carga agregando esta integración a tu llamada a `Sentry.init`.

Esta integración incluye la cantidad mínima de código necesaria para mostrar el Button "Send Feedback" en pantalla cuando se carga la página. Cuando el usuario hace clic en ese botón, la integración cargará de forma asíncrona integraciones internas desde `https://browser.sentry-cdn.com` que muestran el formulario y permiten que el usuario escriba su mensaje. La ventaja es que el paquete es más pequeño. La desventaja es que la carga asíncrona puede fallar si, por ejemplo, el usuario tiene un bloqueador de anuncios.

<div id="when-installed-via-cdn">
  #### Cuando se instala mediante CDN
</div>

Para quienes usan CDN, `feedbackIntegration` es un alias de `feedbackAsyncIntegration`.

```javascript {tabTitle: Default Strategy}
import * as Sentry from "@sentry/browser";

Sentry.init({
  dsn: "___PUBLIC_DSN___",
  integrations: [
    // Usa la estrategia predeterminada, un alias de `feedbackAsyncIntegration`
    Sentry.feedbackIntegration({
      // La configuración adicional del SDK va aquí, por ejemplo:
      colorScheme: "system",
    }),
  ],
});
```

```javascript {tabTitle: Async Strategy}
import * as Sentry from "@sentry/browser";

Sentry.init({
  dsn: "___PUBLIC_DSN___",
  integrations: [
    // Usa la estrategia async para que todo se incluya en la carga de la página
    Sentry.feedbackAsyncIntegration({
      // La configuración adicional del SDK va aquí, por ejemplo:
      colorScheme: "system",
    }),
  ],
});
```

```javascript {tabTitle: Default Strategy}
import * as Sentry from "@sentry/browser";

```

```javascript {tabTitle: Sync Strategy}
import * as Sentry from "@sentry/browser";

Sentry.init({
  dsn: "___PUBLIC_DSN___",

    // Usa la estrategia sincrónica para que todo se incluya en el bundle durante la carga de la página

```

```javascript {tabTitle: Async Strategy}
import * as Sentry from "@sentry/browser";

Sentry.init({

});
```


<div id="crash-report-modal">
  ## Modal de informe de fallos
</div>

Puedes personalizar el modal de informe de fallos según las necesidades de tu organización, por ejemplo, con fines de localización. Todas las opciones se pueden pasar mediante la llamada a `Sentry.showReportDialog`.

| Param            | Default                                                                                              |
| ---------------- | ---------------------------------------------------------------------------------------------------- |
| `eventId`        | Establece manualmente el ID del evento.                                                              |
| `dsn`            | Establece manualmente el DSN al que se informará.                                                    |
| `user`           | Establece manualmente los datos del usuario _[un objeto con las claves que se indican a continuación]_. |
| `user.email`     | Dirección de correo electrónico del usuario.                                                         |
| `user.name`      | Nombre del usuario.                                                                                  |
| `lang`           | _[automático]_ – (**Sobrescribe el código de idioma de Sentry.**)                                    |
| `title`          | Parece que tenemos problemas.                                                                        |
| `subtitle`       | Nuestro equipo ya ha sido notificado.                                                                |
| `subtitle2`      | Si quieres ayudar, cuéntanos a continuación qué pasó. – (**No visible en resoluciones de pantalla pequeñas.**) |
| `labelName`      | Nombre                                                                                               |
| `labelEmail`     | Correo electrónico                                                                                   |
| `labelComments`  | ¿Qué pasó?                                                                                           |
| `labelClose`     | Cerrar                                                                                               |
| `labelSubmit`    | Enviar                                                                                               |
| `errorGeneric`   | Se produjo un error desconocido al enviar tu informe. Inténtalo de nuevo.                            |
| `errorFormEntry` | Algunos campos no son válidos. Corrige los errores e inténtalo de nuevo.                             |
| `successMessage` | Tu opinión se ha enviado. ¡Gracias!                                                                   |
| `onLoad`         | n/a - (**Un callback opcional que se invocará cuando se abra el widget.**)                           |
| `onClose`        | n/a - (**Un callback opcional que se invocará cuando se cierre el widget.**)                         |
