La forma más sencilla de proporcionar a Sentry archivos de información de depuración es
subirlos con `sentry-cli`. Según tu flujo de trabajo, quizá quieras
subirlos como parte de tu pipeline de compilación o al desplegar y publicar tu
aplicación:

<Include name="common-imgs/debug-files-workflow" />

<PlatformContent includePath="debug-files">
  Los archivos se pueden subir con el comando `debug-files upload`. Este comando analizará
  recursivamente una carpeta dada en busca de archivos y los subirá a Sentry:
</PlatformContent>

Para ver todas las opciones disponibles y obtener más información, consulta [*Uploading Debug
Information*](/es/cli/dif/#uploading-files).

Los archivos de depuración deben subirse antes de desplegar o publicar tu
aplicación para que se puedan procesar los informes de fallos. Para pruebas manuales, usa
`sentry-cli debug-files upload --wait` antes de enviar el primer fallo o error
nativo.

Si subes un archivo que Sentry indicó previamente como faltante en un informe de
fallo, Sentry puede tardar hasta una hora en empezar a usar ese archivo para nuevos
informes de fallos. Los eventos e incidencias existentes no se vuelven a procesar.

En Sentry, tus archivos subidos se asocian con proyectos. Puedes ver y
gestionar las subidas en **Project Settings &gt; Debug Files**. Esta pantalla muestra las
propiedades más importantes de los archivos de depuración:

1. El identificador de depuración. Puede haber varias entradas que compartan el mismo
   identificador si la información de depuración está dividida entre varios archivos.
2. El nombre del archivo de depuración. Sentry usa el nombre en el sistema de archivos al
   subir el archivo con `sentry-cli`.
3. La arquitectura y el tipo de archivo. Esto se puede usar para distinguir archivos
   en ejecutables y acompañantes de depuración si sus identificadores de depuración y nombres
   coinciden.
4. La información de depuración disponible en estos archivos. Esto puede incluir `unwind`,
   `debug`, `symtab` y `sources`.
5. Metadatos como el tamaño y la hora de subida.

Si varios proyectos requieren el mismo archivo, debe volver a subirse. Sentry
no accede a archivos de depuración almacenados en otros proyectos para simbolizar informes de
fallos.
