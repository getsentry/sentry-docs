Si Sentry tiene acceso al código fuente de tu aplicación, puede mostrar fragmentos de código (*contexto de origen*) alrededor de la ubicación de los frames de la pila, lo que ayuda a identificar rápidamente el código problemático.

Por ejemplo, aquí tienes una traza de pila de una aplicación .NET. El contexto de origen contiene
el código fuente original en C# de la aplicación, con la ubicación del frame resaltada.

![Captura de pantalla de una traza de pila que muestra el contexto de origen.](./img/source-context-screenshot.png)

<div id="source-bundles">
  ### Paquetes de código fuente
</div>

Algunos SDK, como el de Python, pueden resolver este contexto de código fuente automáticamente,
porque tienen acceso al código no ofuscado en tiempo de ejecución.
En cambio, para obtener el contexto de código fuente en aplicaciones compiladas, el código
debe cargarse junto con los archivos de información de depuración.
La forma recomendada de hacerlo es usar `sentry-cli`.
Consulta [Creación de paquetes de código fuente](/es/cli/dif/#creating-source-bundles) para más información.

Después de cargarlos, puedes revisar y administrar los paquetes de código fuente igual que
cualquier otro archivo de información de depuración. Consulta [Administración de archivos de información de depuración](../#managing-debug-information-files).
Los paquetes de código fuente se etiquetarán con `sources` y tendrán el mismo nombre de archivo que el
archivo de información de depuración correspondiente a partir del cual se crearon.
