<div id="fixing-angular-service-worker-caching-issues">
  ## Solucionar problemas de caché del service worker de Angular
</div>

Si usas el [service worker de Angular](https://angular.dev/ecosystem/service-workers), podrías encontrar problemas de caché al subir mapas de origen con Sentry CLI.
Esto ocurre porque Sentry CLI inyecta debugIds en los chunks de JavaScript generados después del proceso de compilación.
Esto hace que los hashes de archivo en `ngsw.json` para esos chunks dejen de ser válidos.
Para corregir los hashes inválidos, vuelve a generar `ngsw.json` después de que se hayan inyectado los debugIds usando el siguiente comando:

```bash
node_modules/.bin/ngsw-config <dist> <config>
```

- `dist` Ruta a los archivos de los chunks generados. El `ngsw.json` con los hashes antiguos debería estar allí.
- `config` Ubicación de `ngsw-config.json` (debería estar en la raíz del proyecto)
