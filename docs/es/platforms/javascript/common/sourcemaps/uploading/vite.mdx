<Alert>
  Esta guía asume que usas una **versión del SDK de Sentry `7.47.0` o posterior**.
  Si estás en una versión anterior y quieres cargar source maps, te recomendamos
  actualizar tu SDK a la versión más reciente.
</Alert>

Puedes usar el plugin Sentry Vite para crear versiones automáticamente y cargar source maps a Sentry al empaquetar tu app.

<div id="automatic-setup">
  ## Configuración automática
</div>

La manera más sencilla de configurar la subida de source maps con Vite es usando el Sentry Wizard:

<Include name="sourcemaps-wizard-instructions.mdx" />

Si prefieres configurar manualmente la subida de source maps con Vite, sigue los pasos a continuación.

<div id="manual-setup">
  ## Configuración manual
</div>

Instala el plugin de Sentry para Vite:

```bash {tabTitle:npm}
npm install @sentry/vite-plugin --save-dev
```

```bash {tabTitle:yarn}
yarn add @sentry/vite-plugin --dev
```

```bash {tabTitle:pnpm}
pnpm add @sentry/vite-plugin --save-dev
```


<div id="configuration">
  ### Configuración
</div>

Para subir los source maps debes configurar un [Organization Token](https://sentry.io/orgredirect/organizations/:orgslug/settings/auth-tokens/).

Como alternativa, también puedes usar un [Personal Token](https://sentry.io/orgredirect/organizations/:orgslug/settings/account/api/auth-tokens/) con los permisos “Project: Read &amp; Write” y “Release: Admin”.

Los tokens de autenticación pueden pasarse al plugin de forma explícita con la opción `authToken`, mediante la variable de entorno `SENTRY_AUTH_TOKEN` o con un archivo `.env.sentry-build-plugin` (no olvides añadirlo a tu archivo `.gitignore`, ya que es información sensible) en el directorio de trabajo al compilar tu proyecto.
Recomendamos añadir el token de autenticación a tu entorno de CI/CD como variable de entorno.

Obtén más información sobre cómo configurar el plugin en nuestra [documentación del plugin Sentry Vite](https://www.npmjs.com/package/@sentry/vite-plugin).

<OrgAuthTokenNote />

```bash {filename:.env.sentry-build-plugin}
SENTRY_AUTH_TOKEN=___ORG_AUTH_TOKEN___
```

Ejemplo:

```javascript {filename:vite.config.js}
import { defineConfig } from "vite";
import { sentryVitePlugin } from "@sentry/vite-plugin";

export default defineConfig({
  build: {
    sourcemap: true, // La generación de source maps debe estar activada
  },
  plugins: [
    // Coloca el plugin de Vite de Sentry después de todos los otros plugins
    sentryVitePlugin({
      org: "___ORG_SLUG___",
      project: "___PROJECT_SLUG___",
      authToken: process.env.SENTRY_AUTH_TOKEN,
    }),
  ],
});
```

<Alert>
  Generar mapas de origen (sourcemaps) puede exponerlos públicamente y provocar la filtración
  de tu código fuente. Puedes evitarlo configurando tu servidor para denegar el
  acceso a los archivos `.js.map`, o usando la opción [`sourcemaps.filesToDeleteAfterUpload` del Sentry Vite Plugin](https://www.npmjs.com/package/@sentry/vite-plugin#sourcemapsfilestodeleteafterupload)
  para eliminar los mapas de origen después de subirlos a Sentry.
</Alert>

<Alert>
  El plugin de Sentry para Vite no sube mapas de origen en modo watch/desarrollo.
  Te recomendamos ejecutar una compilación de producción para probar tu configuración.
</Alert>
