  ## Configuración manual
</div>

<div id="1-generate-source-maps">
  ### 1. Generar mapas de origen
</div>

Primero, configura el compilador de TypeScript para generar mapas de origen:

```json {filename:tsconfig.json}
{
  "compilerOptions": {
    "sourceMap": true,
    "inlineSources": true,

    // Establece `sourceRoot` en "/" para eliminar el prefijo de la ruta de compilación de
    // las referencias al código fuente generadas. Esto mejorará la agrupación de incidencias en Sentry.
    "sourceRoot": "/"
  }
}
```

<Alert>
  Generar mapas de fuentes puede exponerlos públicamente y provocar la filtración de tu código. Puedes evitarlo configurando el servidor para denegar el acceso a los archivos `.js.map` o eliminando los mapas de fuentes antes de desplegar la aplicación.
</Alert>

<div id="dealing-with-tslib">
  ## Cómo manejar TSLib
</div>

Durante la compilación, TypeScript inyecta algunas de sus dependencias en tiempo de ejecución en los archivos de salida que produce cuando es necesario. Puede incluir polyfills para generadores de funciones o APIs que quizá no estén disponibles en todos los entornos. Sin embargo, la ausencia de fuentes originales hace imposible mapear los frames del código compilado a las fuentes originales.

Como solución, tendrás que indicarle al compilador de TypeScript que use `tslib`, su propio paquete de terceros (que internamente forma parte del compilador), en lugar de inyectar dependencias en tiempo de ejecución. Solo necesitarás cambiar lo que está dentro del archivo de configuración de TypeScript, no tu código fuente. Así es como se hace:

1. Asegúrate de que `tslib` figure como dependencia en tu archivo `package.json`.
2. Una vez hecho esto, agrega dos entradas en la sección `compilerOptions` de tu `tsconfig.json`:

- `"noEmitHelpers": true` y
- `"importHelpers": true`.

¡Eso es todo! Ahora deberías poder mapear correctamente los source maps para todos los frames de tu stack trace, incluidos los fragmentos internos del código del compilador de TypeScript.
