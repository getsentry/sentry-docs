<PlatformContent includePath="performance/configure-sample-rate" />

* Puedes establecer una tasa de muestreo uniforme para todas las transacciones configurando la opción <PlatformIdentifier name="traces-sample-rate" /> en la configuración de tu SDK con un número entre `0` y `1`. (Por ejemplo, para enviar el 20% de las transacciones, establece <PlatformIdentifier name="traces-sample-rate" /> en `0.2`).
* Para un control más granular del muestreo, puedes definir la tasa de muestreo según la propia transacción y el contexto en el que se captura, proporcionando una función a la opción de configuración <PlatformIdentifier name="traces-sampler" />.

Ambas opciones son excluyentes. Si se configuran las dos, <PlatformIdentifier name="traces-sampler" /> tendrá prioridad.

Puedes encontrar explicaciones y ejemplos más detallados sobre la configuración de muestreo en [Configurar el muestreo](./configure-sampling).

<div id="distributed-tracing">
  ## Rastreo distribuido
</div>

Sentry captura trazas distribuidas compuestas por transacciones y spans, que miden, respectivamente, los servicios individuales y las operaciones dentro de esos servicios. Obtén más información sobre nuestro modelo en [Rastreo distribuido](/es/product/sentry-basics/tracing/distributed-tracing/).

<div id="verify">
  ## Verificar
</div>

<PlatformSection supported={["javascript"]} notSupported={["javascript.cordova"]}>
  Verifica que el tracing funcione correctamente usando nuestra <PlatformLink to="/tracing/instrumentation/automatic-instrumentation/">instrumentación automática</PlatformLink> o iniciando y finalizando una transacción con <PlatformLink to="/tracing/instrumentation/custom-instrumentation/">instrumentación personalizada</PlatformLink>.
</PlatformSection>

Mientras haces pruebas, establece <PlatformIdentifier name="traces-sample-rate" /> en `1.0`, ya que así garantizas que cada transacción se envíe a Sentry. Cuando termines, quizá quieras usar un valor más bajo para <PlatformIdentifier name="traces-sample-rate" />, o pasar a <PlatformIdentifier name="traces-sampler" /> para muestrear y filtrar tus transacciones selectivamente según datos contextuales.

Si dejas la tasa de muestreo en `1.0`, se enviará una transacción cada vez que una persona usuaria cargue una página o navegue dentro de tu aplicación. Según el volumen de tráfico de tu aplicación, esto puede implicar muchas transacciones. Si tienes una aplicación backend de alta carga, quizá te convenga usar un valor más bajo para <PlatformIdentifier name="traces-sample-rate" />, o pasar a <PlatformIdentifier name="traces-sampler" /> para muestrear y filtrar tus transacciones selectivamente según datos contextuales.

<div id="disabling-tracing">
  ## Desactivar el tracing
</div>

Si quieres desactivar el tracing, *no deberías* configurar `tracesSampleRate`
en absoluto. Establecerlo en `0` no desactiva el tracing; simplemente nunca enviará
trazas a Sentry.
En su lugar, ni `tracesSampleRate` ni `tracesSampler` deberían definirse en la configuración de tu SDK para desactivar completamente el tracing.
