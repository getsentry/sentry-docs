El objetivo principal de la versión 8 es mejorar nuestras API de monitoreo de rendimiento, la API de integraciones y la compatibilidad con ESM. Esta versión incluye cambios incompatibles porque eliminamos API obsoletas, reestructuramos el contenido de los paquetes de npm e incorporamos nuevas dependencias de OpenTelemetry.

<div id="migration-codemod">
  ## Codemod de migración
</div>

Antes de actualizar a la versión `8.x` del SDK, recomendamos pasar a la última versión `7.x`. Para corregir la mayoría de las deprecaciones en `7.x`, puedes usar el codemod [`@sentry/migr8`](https://www.npmjs.com/package/@sentry/migr8) para actualizar automáticamente cómo usas el SDK. `@sentry/migr8` requiere Node 18 o superior.

```bash
npx @sentry/migr8@latest
```

Nuestra herramienta de migración te permitirá seleccionar qué actualizaciones ejecutar y actualizará tu código automáticamente. En algunos casos, no podemos cambiar el código por ti de forma automática. Esos casos se marcarán con un comentario `TODO(sentry)`. Asegúrate de revisar todos los cambios en el código después de ejecutar `@sentry/migr8`. Para más detalles sobre las funciones en desuso, consulta nuestra documentación: [Deprecations in 7.x](./v7-deprecations). Aunque exista `@sentry/migr8`, recomendamos leer la guía de migración, ya que `@sentry/migr8` no cubre todos los cambios necesarios para migrar.

<div id="new-tracing-apis">
  ### Nuevas API de tracing
</div>

La API de instrumentación personalizada para tracing se ha renovado en `8.x`. Se han incorporado nuevos métodos y se han eliminado `startTransaction` y `span.startChild`. Consulta la [nueva documentación de las API de tracing](./v8-new-performance-api/) para obtener más información.
