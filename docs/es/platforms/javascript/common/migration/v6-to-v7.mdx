La versión v7 del SDK de JavaScript requiere una versión autoalojada de Sentry `20.6.0` o superior.

El objetivo principal de la versión 7 es reducir el tamaño del paquete. Esta versión introduce cambios incompatibles porque eliminamos APIs obsoletas, actualizamos nuestras herramientas de compilación y reestructuramos el contenido de los paquetes de npm. A continuación, detallamos todos los cambios incompatibles que debes considerar al actualizar.

**Resumen** Si solo usas funciones básicas de Sentry, o simplemente copiaste y pegaste los ejemplos de configuración de nuestra documentación, esto es lo que cambió para ti:

- Si instalaste paquetes adicionales de Sentry, como `@sentry/tracing` junto con tu SDK de Sentry (por ejemplo, `@sentry/react` o `@sentry/node`), asegúrate de actualizar todos a la versión 7.
- Nuestros paquetes de CDN ahora son ES6: deberás reconfigurar tus etiquetas de script si quieres
  seguir dando soporte a ES5 e IE11 en la nueva versión del SDK.
- Los archivos CommonJS distribuidos serán ES6. Usa un transpiler si necesitas dar soporte a versiones antiguas de Node.
- Actualizamos la versión de TypeScript con la que generamos nuestros tipos a `3.8.3`. Verifica si tus proyectos TypeScript que usan la versión `3.7` o inferior aún compilan. De lo contrario, actualiza tu versión de TypeScript.
- `whitelistUrls` y `blacklistUrls` pasaron a llamarse `allowUrls` y `denyUrls` en las opciones de `Sentry.init()`.
- La integración `UserAgent` ahora se llama `HttpContext`.
- Si usas Tracing y lo tienes habilitado, es posible que debas ajustar la configuración de CORS en tu servidor.

Consulta nuestra [guía de migración detallada](https://github.com/getsentry/sentry-javascript/blob/develop/docs/migration/v6-to-v7.md) para obtener más información sobre cómo migrar de `6.x` a `7.x`.
