El SDK de Cocoa usa [swizzling](https://nshipster.com/method-swizzling/) para ofrecer algunas funcionalidades listas para usar, sin código repetitivo. Las siguientes funcionalidades usan swizzling:

**macOS**

* <PlatformLink to="/enriching-events/breadcrumbs/">Breadcrumbs para eventos táctiles</PlatformLink>
* <PlatformLink to="/tracing/instrumentation/automatic-instrumentation/#network-tracking">Instrumentación automática de solicitudes HTTP</PlatformLink>
* <PlatformLink to="/tracing/instrumentation/automatic-instrumentation/#file-io-tracing">Instrumentación automática de operaciones de E/S de archivos</PlatformLink>
* <PlatformLink to="/tracing/instrumentation/automatic-instrumentation/#core-data-tracing">Instrumentación automática de operaciones de Core Data</PlatformLink>
* <PlatformLink to="/tracing/trace-propagation/">Encabezado sentry-trace añadido automáticamente a las solicitudes HTTP para trazabilidad distribuida</PlatformLink>
* <PlatformLink to="/configuration/http-client-errors/">Errores del cliente HTTP</PlatformLink>

**iOS, tvOS y Catalyst**

* <PlatformLink to="/enriching-events/breadcrumbs/">Breadcrumbs para eventos táctiles y navegación con UIViewControllers</PlatformLink>
* <PlatformLink to="/tracing/instrumentation/automatic-instrumentation/#uiviewcontroller-tracing">Instrumentación automática de UIViewControllers</PlatformLink>
* <PlatformLink to="/tracing/instrumentation/automatic-instrumentation/#network-tracking">Instrumentación automática de solicitudes HTTP</PlatformLink>
* <PlatformLink to="/tracing/instrumentation/automatic-instrumentation/#file-io-tracing">Instrumentación automática de operaciones de E/S de archivos</PlatformLink>
* <PlatformLink to="/tracing/instrumentation/automatic-instrumentation/#core-data-tracing">Instrumentación automática de operaciones de Core Data</PlatformLink>
* <PlatformLink to="/tracing/trace-propagation/">Encabezado sentry-trace añadido automáticamente a las solicitudes HTTP para trazabilidad distribuida</PlatformLink>
* <PlatformLink to="/tracing/instrumentation/automatic-instrumentation/#user-interaction-tracing">Transacciones de interacción de usuario para clics en la interfaz</PlatformLink>
* <PlatformLink to="/configuration/http-client-errors/">Errores del cliente HTTP</PlatformLink>

Desde Cocoa 7.5.0, puedes inhabilitar el swizzling mediante opciones. Al deshabilitar el swizzling, el SDK desactiva las funcionalidades anteriores:

```swift {tabTitle:Swift}
import Sentry

SentrySDK.start { options in
    options.dsn = "___PUBLIC_DSN___"
    options.enableSwizzling = false
}
```

```objc {tabTitle:Objective-C}
@import Sentry;

[SentrySDK startWithConfigureOptions:^(SentryOptions *options) {
    options.dsn = @"___PUBLIC_DSN___";
    options.enableSwizzling = NO;
}];
```


<div id="deactivate-swizzling-for-specific-classes">
  ## Desactivar el swizzling para clases específicas
</div>

Para desactivar el swizzling en clases específicas, puedes usar la opción `swizzleClassNameExcludes`, disponible a partir de la versión `8.23.0` del Sentry Cocoa SDK. El SDK comprueba si el nombre de una clase que planea aplicar swizzling contiene algún nombre configurado con esta opción. Por ejemplo, si agregas `MyUIViewController` a esta lista, el Sentry Cocoa SDK excluye las siguientes clases del swizzling:

* `YourApp.MyUIViewController`
* `YourApp.MyUIViewControllerA`
* `MyApp.MyUIViewController`

```swift {tabTitle:Swift}
import Sentry

SentrySDK.start { options in
    options.swizzleClassNameExcludes = [
      "MyUIViewController",
    ]
}
```

```objc {tabTitle:Objective-C}
@import Sentry;

[SentrySDK startWithConfigureOptions:^(SentryOptions *options) {
    options.swizzleClassNameExcludes = [NSSet setWithObjects:
      @"MyUIViewController",
      nil
    ];
}];
```
