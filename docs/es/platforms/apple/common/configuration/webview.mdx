title: WebView

---

Para supervisar errores y el rendimiento en vistas web de aplicaciones para iOS, iPadOS y macOS, debes configurar los SDK de Sentry tanto en la aplicación móvil como en los proyectos web que se ejecutan dentro de esas vistas web.

<div id="application-monitoring">
  ## Supervisión de la aplicación
</div>

El SDK de Sentry para Cocoa no interactúa con las vistas web de la aplicación móvil. Para supervisar errores y el rendimiento en la parte nativa de la aplicación, sigue las [instrucciones de configuración del SDK de Sentry para Cocoa](/es/platforms/apple/).

<div id="webview-monitoring">
  ## Monitorización de WebView
</div>

Para monitorizar errores y rendimiento en vistas web, elige el SDK de Sentry para JavaScript adecuado según el framework utilizado en el proyecto web.

* [Sentry React SDK](/es/platforms/javascript/guides/react/) (si el proyecto web usa React)
* [Sentry Angular SDK](/es/platforms/javascript/guides/angular/) (si el proyecto web usa Angular)
* [Otros SDK de Sentry para JavaScript](/es/platforms/javascript)

<div id="notes">
  ## Notas
</div>

* El SDK de JavaScript debe inicializarse cuando se cree la vista web y se cerrará cuando esta se cierre. Para monitorear todas las vistas web de la aplicación, inicializa el SDK de JavaScript cada vez que se cree una vista web.

* Si se accede al proyecto web de forma remota y no es exclusivo de la aplicación móvil (por ejemplo, si el proyecto también funciona como una aplicación web independiente), recomendamos usar proyectos de Sentry separados para la aplicación móvil y para el proyecto web.
