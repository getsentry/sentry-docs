title: Datos recopilados

Sentry se toma muy en serio la privacidad de los datos y cuenta con configuraciones predeterminadas que priorizan la seguridad de la información, especialmente cuando se trata de datos de identificación personal (PII). Al agregar el SDK de Sentry a tu aplicación, permites que recopile datos y los envíe a Sentry durante la ejecución de tu aplicación.

Los tipos de categorías y la cantidad de datos recopilados varían según las integraciones que hayas habilitado en el SDK de Sentry. Esta página enumera las categorías de datos que recopila el SDK de Apple.

<Alert>

</Alert>

<div id="http-headers">
  ## Encabezados HTTP
</div>

Los <PlatformLink to="/configuration/http-client-errors">errores del cliente HTTP</PlatformLink>, habilitados de forma predeterminada, envían a Sentry los encabezados HTTP de la solicitud y la respuesta que fallaron. El SDK utiliza una [lista de bloqueo](https://github.com/getsentry/sentry-cocoa/blob/main/Sources/Swift/Tools/HTTPHeaderSanitizer.swift) para filtrar cualquier encabezado que contenga datos sensibles.

<div id="users-ip-addresses">
  ## Direcciones IP de los usuarios
</div>

De forma predeterminada, el SDK de Sentry no envía la dirección IP del usuario. Una vez habilitado, los servicios de backend de Sentry inferirán la dirección IP del usuario a partir de la solicitud entrante, a menos que ciertas integraciones que puedes habilitar sustituyan este comportamiento.

Para habilitar el envío de la dirección IP del usuario, establece <PlatformLink to="/configuration/options/#sendDefaultPii">`sendDefaultPii=true`</PlatformLink>.

<div id="request-url">
  ## URL de la solicitud
</div>

Cuando el SDK de Apple envía URL a Sentry, siempre envía una URL depurada, lo que significa que elimina la cadena de consulta y el fragmento de la URL. Aunque el SDK de Apple depura la URL eliminando la cadena de consulta y el fragmento, según tu aplicación, estos podrían contener datos personales (PII).

Los breadcrumbs de red y los errores del cliente HTTP, ambos habilitados de forma predeterminada, envían una URL depurada para las solicitudes HTTP salientes. Puedes deshabilitar los breadcrumbs de red configurando la opción `enableNetworkBreadcrumbs` en `false` y puedes deshabilitar los errores del cliente HTTP configurando la opción `enableCaptureFailedRequests` en `false`.

Cuando habilitas <PlatformLink to="/tracing">tracing</PlatformLink>, que está deshabilitado de forma predeterminada, el <PlatformLink to="/tracing/instrumentation/automatic-instrumentation/#network-tracking">rastreo de red</PlatformLink> envía una URL depurada para las solicitudes HTTP salientes.

<div id="source-context">
  ## Contexto del código fuente
</div>

Puedes cargar tu código fuente en Sentry, que luego puede usarse para mostrar las líneas de código donde ocurrió un error en la página de detalles del incidente, mediante <PlatformLink to="/sourcecontext/#1-manually-upload-with-the-sentry-cli">sentry-cli</PlatformLink> o el <PlatformLink to="/sourcecontext/#2-fastlane-plugin">plugin Sentry Fastlane</PlatformLink>.

Para habilitar el envío de este contexto de código fuente a Sentry, debes activar la función como se describe en <PlatformLink to="/sourcecontext/">la documentación de Source Context</PlatformLink>.

<div id="file-io">
  ## E/S de archivos
</div>

Cuando habilitas el rastreo, que está deshabilitado de forma predeterminada, el SDK de Apple <PlatformLink to="/tracing/instrumentation/automatic-instrumentation/#file-io-tracing">instrumenta las operaciones de E/S de archivos</PlatformLink> y envía los nombres y rutas de archivo a Sentry.

<div id="core-data-queries">
  ## Consultas de Core Data
</div>

Cuando habilitas <PlatformLink to="/tracing">tracing</PlatformLink>, que está deshabilitado de forma predeterminada, el SDK de Apple <PlatformLink to="/tracing/instrumentation/automatic-instrumentation/#core-data-tracing">instrumenta las consultas de Core Data</PlatformLink> y las envía a Sentry. Ni la consulta SQL completa (`SELECT 'User' WHERE name == 'username'`), ni los valores de sus parámetros se enviarán nunca. En su lugar, se envía una versión parametrizada de la consulta (`SELECT 'User' WHERE name == %@`).

<PlatformSection supported={["apple.ios", "apple.visionos"]}>

<div id="screenshots">
  ## Capturas de pantalla
</div>

La <PlatformLink to="/enriching-events/screenshots">función de capturas de pantalla</PlatformLink> está deshabilitada de forma predeterminada, pero, cuando se habilita, las capturas pueden contener datos personales (PII).

<div id="view-hierarchy">
  ## Jerarquía de vistas
</div>

La <PlatformLink to="/enriching-events/viewhierarchy">función de jerarquía de vistas</PlatformLink> está deshabilitada de forma predeterminada, pero, cuando se habilita, la jerarquía de vistas puede contener datos personales (PII) si se usa la propiedad `accessibilityIdentifier` con información personal.

</PlatformSection>

<PlatformSection supported={["apple.ios"]}>

<div id="session-replay">
  ## Session Replay
</div>

De forma predeterminada, nuestro SDK de Session Replay enmascara todo el contenido de texto, imágenes, vistas web y la entrada del usuario. Esto ayuda a garantizar que no se expongan datos sensibles. Puedes encontrar <PlatformLink to="/session-replay/#privacy">más detalles en la documentación de Session Replay</PlatformLink>.

</PlatformSection>
