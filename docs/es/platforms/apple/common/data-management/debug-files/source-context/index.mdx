Si Sentry tiene acceso al código fuente de tu aplicación, puede mostrar fragmentos de código (*contexto de origen*) alrededor de la ubicación de los marcos de la pila, lo que ayuda a identificar rápidamente el código problemático.

Por ejemplo, aquí tienes una traza de pila de una aplicación Swift. El contexto de origen contiene
el código fuente original de la aplicación en Swift, con la ubicación del marco resaltada.

![Captura de pantalla de una traza de pila que muestra el contexto de origen.](./img/source-context-screenshot.png)

<div id="source-bundles">
  ### Paquetes de código fuente
</div>

Algunos SDK, como el SDK de Python, pueden resolver este contexto de código fuente automáticamente,
porque tienen acceso a código fuente sin ofuscar en tiempo de ejecución.
En cambio, para obtener el contexto de código fuente de aplicaciones compiladas, el código fuente
debe subirse junto con los archivos de información de depuración.
La forma recomendada de hacerlo es usando `sentry-cli`.
Consulta [Creación de paquetes de código fuente](/es/cli/dif/#creating-source-bundles) para más información.

Para obtener contexto de código fuente para tu proyecto de Apple, sigue las instrucciones de la guía [Carga de símbolos de depuración](/es/platforms/apple/sourcecontext).

Una vez subidos, puedes revisar y administrar los paquetes de código fuente igual que
cualquier otro archivo de información de depuración. Consulta [Gestión de archivos de información de depuración](../#managing-debug-information-files).
Los paquetes de código fuente se etiquetarán con `sources` y tendrán el mismo nombre de archivo que el
archivo de información de depuración correspondiente del que fueron creados.
