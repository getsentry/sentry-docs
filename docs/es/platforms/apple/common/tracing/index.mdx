Con [tracing](/es/product/insights/overview/), Sentry supervisa el rendimiento de tu software, midiendo métricas como el rendimiento (throughput) y la latencia, y mostrando el impacto de los errores en múltiples sistemas. Sentry captura trazas distribuidas compuestas por transacciones y spans, que miden servicios individuales y las operaciones dentro de esos servicios. Obtén más información sobre nuestro modelo en [Distributed Tracing](/es/product/sentry-basics/tracing/distributed-tracing/).

La instrumentación automática para supervisar el rendimiento de tu aplicación se encuentra actualmente en versión preliminar. También puedes usar instrumentación personalizada para capturar transacciones.

<div id="configure">
  ## Configurar
</div>

Primero, habilita el tracing y configura la tasa de muestreo para las transacciones. Establece la tasa de muestreo para tus transacciones de una de estas maneras:

* Configurando una tasa de muestreo uniforme para todas las transacciones usando la opción <PlatformIdentifier name="traces-sample-rate" /> en la configuración de tu SDK con un número entre `0` y `1`. (Por ejemplo, para enviar el 20% de las transacciones, establece <PlatformIdentifier name="traces-sample-rate" /> en `0.2`).
* Controlando la tasa de muestreo según la transacción y el contexto en el que se captura, proporcionando una función a la opción de configuración <PlatformIdentifier name="traces-sampler" />.

Estas dos opciones son mutuamente excluyentes. Si configuras ambas, <PlatformIdentifier name="traces-sampler" /> tendrá prioridad.

Obtén más información sobre las <PlatformLink to="/configuration/options/#tracing-options">opciones</PlatformLink> de tracing, cómo usar la función <PlatformLink to="/configuration/sampling/#setting-a-sampling-function">tracesSampler</PlatformLink> o cómo <PlatformLink to="/configuration/sampling/#sampling-transaction-events">muestrear transacciones</PlatformLink>.

<div id="verify">
  ## Verificar
</div>

Verifica que el tracing funcione correctamente usando nuestra <PlatformLink to="/tracing/instrumentation/automatic-instrumentation/">instrumentación automática</PlatformLink> o iniciando y finalizando una transacción con <PlatformLink to="/tracing/instrumentation/custom-instrumentation/">instrumentación personalizada</PlatformLink>.

Prueba el tracing iniciando y finalizando una transacción; *debes* hacerlo para que las transacciones se envíen a Sentry. Aprende cómo en el contenido de <PlatformLink to="/tracing/instrumentation/custom-instrumentation/">instrumentación personalizada</PlatformLink>.

Mientras haces pruebas, establece <PlatformIdentifier name="traces-sample-rate" /> en `1.0`, ya que eso garantiza que cada transacción se envíe a Sentry. Una vez que termines, quizá quieras usar un valor más bajo para <PlatformIdentifier name="traces-sample-rate" />, o pasar a <PlatformIdentifier name="traces-sampler" /> para muestrear y filtrar selectivamente tus transacciones según datos contextuales.
