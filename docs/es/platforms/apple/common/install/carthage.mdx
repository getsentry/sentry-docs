Para integrar Sentry en tu proyecto de Xcode con Carthage, especifícalo en tu *Cartfile*:

```ruby
github "getsentry/sentry-cocoa" "{{@inject packages.version('sentry.cocoa') }}"
```

Ejecuta `carthage update` para descargar el framework y arrastra el *Sentry.xcframework* generado a tu proyecto de Xcode. También ofrecemos una versión ya compilada para cada versión, que puedes descargar en las [releases de GitHub](https://github.com/getsentry/sentry-cocoa/releases).

La configuración de Carthage de Sentry genera 3 frameworks diferentes:

* Sentry
* SentryPrivate
* SentrySwiftUI

*Sentry* es obligatorio para todos los proyectos. *SentrySwiftUI* es necesario para medir el rendimiento de tus vistas de SwiftUI. *SentryPrivate* solo se necesita si modificaste el proceso de compilación de Carthage para generar frameworks estáticos.

El SDK admite la integración de Sentry con visionOS mediante Carthage a partir de `sentry-cocoa 8.20.0`.


<div id="macs-with-apple-silicon-and-xcframeworks">
  ## Mac con Apple Silicon y XCFrameworks
</div>

Si usas Sentry con Carthage en Mac con Apple Silicon o quieres usar XCFrameworks, utiliza Carthage >= 0.37.0, que introdujo [compatibilidad con XCFramework](https://github.com/Carthage/Carthage/releases/tag/0.37.0). Asegúrate de ejecutar `carthage build --use-xcframeworks --no-use-binaries` después de `carthage update --use-xcframeworks`. Carthage 0.37.0 necesita reconstruir el XCFramework a partir del código fuente porque no puede manejar dependencias de GitHub que descargan binarios. Más información en sus [notas de la versión](https://github.com/Carthage/Carthage/releases/tag/0.37.0).
