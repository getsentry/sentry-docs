---
title: Swift Package Manager (SPM)
description: "Integra Sentry en tu proyecto de Xcode usando Swift Package Manager (SPM)."
---

Para integrar Sentry en tu proyecto de Xcode usando Swift Package Manager (SPM), abre tu app en Xcode y ve a **File &gt; Add Packages**. Luego añade el SDK ingresando la URL del repositorio de Git en el campo de búsqueda de la esquina superior derecha:

```
https://github.com/getsentry/sentry-cocoa.git
```

Puedes definir la regla de dependencias seleccionando la versión del SDK (o la rama) y luego haciendo clic en el Button &quot;Add Package&quot;.
A continuación, se te pedirá que elijas una de las opciones `Sentry`, `Sentry-Dynamic` o `SentrySwiftUI`.

* `Sentry` es el framework estático y es la opción recomendada si prefieres que la app arranque más rápido.
* `Sentry-Dynamic` es el framework dinámico.
* `SentrySwiftUI` se usa para medir el rendimiento de las vistas de SwiftUI; consulta más información en la [documentación](/es/platforms/apple/guides/ios/tracing/instrumentation/swiftui-instrumentation/).

<Alert>
  Xcode permite elegir muchas opciones, pero solo debes elegir una de las
  opciones anteriores.
</Alert>

Como alternativa, si tu proyecto usa un archivo `Package.swift` para gestionar las dependencias, puedes especificar el target con:

```swift {tabTitle:Swift}
.package(url: "https://github.com/getsentry/sentry-cocoa", from: "{{@inject packages.version('sentry.cocoa') }}"),
```

<Alert>
  Las etiquetas de versión o las ramas deben incluir el archivo Package.swift; de lo contrario, Xcode no podrá instalar el paquete. Esto significa que las versiones anteriores a `4.4.3` *no* se pueden instalar con SPM.
</Alert>
