El SDK de Sentry se integra con tu entorno de ejecución y reporta automáticamente errores, excepciones no controladas y rechazos no controlados, así como otros tipos de errores según la plataforma.

<Alert>

Términos clave:

- Un _evento_ es una instancia de envío de datos a Sentry. Por lo general, estos datos corresponden a un error o una excepción.
- Un _issue_ es un conjunto de eventos similares.
- Al envío de un evento se le llama _captura_.
  Cuando se captura un evento, se envía a Sentry.

</Alert>

<div id="capturing-messages">
  ## Captura de mensajes
</div>

La operación más sencilla es capturar un mensaje básico: información textual que se debe enviar a Sentry. Por lo general, nuestros SDK no capturan mensajes automáticamente, pero puedes capturarlos manualmente.

Los mensajes aparecen como problemas en tu flujo de problemas, con el contenido del mensaje como nombre del problema.

<div id="capturing-errors">
  ## Captura de errores
</div>

El uso más común de los SDK de Apple es capturar errores. En general, si tienes algo que parece una excepción, puede capturarse, aunque esto varía según la plataforma. En algunos SDK, también puedes omitir el argumento de <PlatformIdentifier name="capture-exception" /> y Sentry intentará capturar la excepción actual. Esto también es útil para el reporte manual de errores o mensajes a Sentry.

Al capturar un evento, también puedes registrar las migas que conducen a ese evento. Las migas (breadcrumbs) son diferentes de los eventos: no crearán un evento en Sentry, sino que se almacenarán en búfer hasta que se envíe el siguiente. Obtén más información en nuestra <PlatformLink to="/enriching-events/breadcrumbs/">documentación de Breadcrumbs</PlatformLink>.
