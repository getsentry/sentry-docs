En esta página aprenderás a adjuntar archivos, como logs o archivos de configuración, a tus eventos. Podrás ver estos adjuntos en [sentry.io](https://sentry.io/) junto con otros detalles del evento.

<div id="creating-attachments">
  ## Creación de archivos adjuntos
</div>

La forma más sencilla de crear un archivo adjunto es usar un `path`. El SDK leerá el contenido del archivo cada vez que prepare un evento o transacción y luego agregará el archivo adjunto al mismo [envelope](https://develop.sentry.dev/sdk/data-model/envelopes/). Si el SDK no puede leer el archivo, registrará un mensaje de error y descartará el archivo adjunto.

<PlatformContent includePath="enriching-events/attachment-init-with-path" />

Como alternativa, usa `bytes` para inicializar un archivo adjunto. Al hacerlo, también debes especificar un nombre de archivo.

<PlatformContent includePath="enriching-events/attachment-init-with-bytes" />

Si tu SDK admite caché sin conexión, algo típico en dispositivos móviles, cada archivo adjunto se guarda en disco por cada evento o transacción que captures cuando el dispositivo esté sin conexión. Al usar archivos adjuntos grandes, este almacenamiento puede consumir espacio en disco si el dispositivo permanece sin conexión durante un período prolongado. Puedes especificar el [tamaño máximo del archivo adjunto](#maximum-attachment-size) para descartar archivos adjuntos grandes y evitar este problema.

Además, puedes configurar estos parámetros:

`filename`

El nombre de archivo es el nombre que se mostrará en Sentry. Al usar bytes debes especificar un nombre de archivo, mientras que con un path no es necesario, ya que el SDK usará el último componente del path.

`contentType`

El tipo de contenido almacenado en este archivo adjunto. Se puede usar cualquier [tipo MIME](https://www.iana.org/assignments/media-types/media-types.xhtml); el valor predeterminado es `application/octet-stream`.

`mimetype`

El tipo específico de contenido multimedia que determina cómo se representa el archivo adjunto en la interfaz de Sentry. Actualmente admitimos y podemos representar los siguientes tipos MIME:

* `text/plain`
* `text/css`
* `text/csv`
* `text/html`
* `text/javascript`
* `text/json` o `text/x-json` o `application/json` o `application/ld+json`
* `image/jpeg`
* `image/png`
* `image/gif`
* `image/webp`
* `image/avif`
* `video/webm`
* `video/mp4`

<div id="uploading-attachments">
  ## Carga de archivos adjuntos
</div>

Los adjuntos residen en el <PlatformLink to="/enriching-events/scopes/">Scope</PlatformLink>. Puedes agregar un adjunto en el scope global para enviarlo con cada evento o agregarlo en el <PlatformLink to="/enriching-events/scopes/#local-scopes">scope local</PlatformLink> para enviarlo solo con un evento específico.

<Alert>
  Actualmente no se admite agregar adjuntos a fallos.
</Alert>

<PlatformContent includePath="enriching-events/attachment-upload" />

<Alert>
  Sentry permite como máximo 20 MB para una solicitud comprimida y 100 MB de adjuntos sin comprimir por evento, incluido el archivo de informe de fallos (si corresponde). Las cargas que superen este tamaño se rechazan con el error HTTP `413 Payload Too Large` y los datos se descartan de inmediato. Para agregar archivos más grandes o en mayor cantidad, considera opciones de almacenamiento secundario.
</Alert>

Los adjuntos se conservan durante 30 días; si excedes el almacenamiento total incluido en tu cuota, no se almacenarán. Puedes eliminar adjuntos o los eventos que los contienen en cualquier momento. Eliminar un adjunto no afecta tu cuota: Sentry lo cuenta en cuanto se almacena.

Obtén más información sobre cómo los adjuntos afectan tu [cuota](/es/pricing/quotas/).

<div id="access-to-attachments">
  ### Acceso a adjuntos
</div>

Para limitar el acceso a los adjuntos, ve a la **Configuración general** de tu organización y selecciona el menú desplegable *Acceso a adjuntos* para establecer el acceso adecuado: cualquier miembro de tu organización, el propietario de la facturación de la organización, miembro, administrador, administrador de equipo o propietario.

<Include name="common-imgs/attachments-access" />

De forma predeterminada, el acceso se concede a todos los miembros cuando el almacenamiento está habilitado. Si un miembro no tiene acceso al proyecto, no podrá descargar un adjunto; el botón aparecerá en gris en Sentry. El miembro solo podrá ver que hay un adjunto almacenado.

<div id="viewing-attachments">
  ## Visualización de adjuntos
</div>

Los adjuntos se muestran en la parte inferior de la página de **Detalles de la incidencia** para el evento que se está mostrando.

<Include name="common-imgs/attachments-access-denied" />

Además, los adjuntos también aparecen en la pestaña *Adjuntos* de la página de **Detalles de la incidencia**, donde puedes ver el *Tipo* de adjunto, así como los eventos asociados. Haz clic en el ID del evento para abrir los **Detalles de la incidencia** de ese evento específico.

<div id="maximum-attachment-size">
  ## Tamaño máximo de archivos adjuntos
</div>

El tamaño máximo para cada archivo adjunto se establece en `SentryOptions.maxAttachmentSize`.

La unidad es bytes y el valor predeterminado es `20 MiB`. Revisa también el
[tamaño máximo de archivo adjunto de Relay](/es/product/relay/options/)
para asegurarte de que tus archivos adjuntos no se descarten allí.
