Todos los eventos tienen una huella. Los eventos con la misma huella se agrupan en un issue.

De forma predeterminada, Sentry ejecuta uno de nuestros algoritmos de agrupación integrados para generar una huella basada en la información disponible en el evento, como `stacktrace`, `exception` y `message`. Para ampliar el comportamiento de agrupación predeterminado o cambiarlo por completo, puedes usar una combinación de las siguientes opciones:

1. En tu SDK, usando la huella del SDK, como se documenta a continuación
2. En tu proyecto, usando [Reglas de huella](/es/concepts/data-management/event-grouping/fingerprint-rules/) o [Reglas de stack trace](/es/concepts/data-management/event-grouping/stack-trace-rules/)

Puedes reemplazar la agrupación predeterminada de Sentry pasando el atributo de huella como un arreglo de cadenas. La longitud del arreglo de una huella no está restringida. Esto funciona de manera similar a la [funcionalidad de reglas de huella](/es/concepts/data-management/event-grouping/fingerprint-rules/), que puede lograr resultados similares.

Puedes usar sustitución de variables para incorporar valores dinámicos en la huella, que por lo general se calcula en el servidor. Por ejemplo, el valor `{{ default }}` se puede agregar para incluir en la huella todo el hash de agrupación que normalmente se genera. Estos valores son los mismos que los de la huella configurada en el servidor. Consulta [Variables](/es/concepts/data-management/event-grouping/fingerprint-rules/#variables) para obtener más información.
