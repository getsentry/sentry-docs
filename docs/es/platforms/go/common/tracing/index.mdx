Con el [rastreo](/es/product/insights/overview/), Sentry supervisa el rendimiento de tu software, mide métricas como el rendimiento (throughput) y la latencia, y muestra el impacto de los errores en múltiples sistemas. Sentry captura trazas distribuidas compuestas por transacciones y spans, que miden servicios individuales y operaciones específicas dentro de esos servicios. Obtén más información sobre nuestro modelo en [Rastreo distribuido](/es/product/sentry-basics/tracing/distributed-tracing/).

<Alert>
  Si vas a adoptar el rastreo en un entorno de alto rendimiento (throughput), te recomendamos probar antes del despliegue para asegurarte de que las características de rendimiento de tu servicio cumplan con las expectativas.
</Alert>

<div id="requirements">
  ## Requisitos
</div>

* El rastreo (tracing) está disponible a partir de la versión ≥ 0.9.0 del SDK de Go de Sentry.

<div id="configure">
  ## Configurar
</div>

Primero, habilita el tracing y configura la tasa de muestreo para las transacciones. Establece la tasa de muestreo de tus transacciones de cualquiera de estas maneras:

* Configurando una tasa de muestreo uniforme para todas las transacciones usando la opción <PlatformIdentifier name="traces-sample-rate" /> en la configuración de tu SDK con un número entre `0` y `1`. (Por ejemplo, para enviar el 20% de las transacciones, establece <PlatformIdentifier name="traces-sample-rate" /> en `0.2`).
* Controlando la tasa de muestreo según la transacción y el contexto en el que se captura, proporcionando una función a la opción de configuración <PlatformIdentifier name="traces-sampler" />.

Estas dos opciones son mutuamente excluyentes. Si configuras ambas, <PlatformIdentifier name="traces-sampler" /> tendrá prioridad.

<PlatformContent includePath="performance/configure-sample-rate" />

Obtén más información sobre las <PlatformLink to="/configuration/options/#tracing-options">opciones</PlatformLink> de tracing, cómo usar la función <PlatformLink to="/configuration/sampling/#setting-a-sampling-function">TracesSampler</PlatformLink> o cómo <PlatformLink to="/configuration/sampling/#sampling-transaction-events">muestrear transacciones</PlatformLink>.

<div id="verify">
  ## Verificar
</div>

Prueba el tracing iniciando y finalizando una transacción; debes hacerlo para que las transacciones se envíen a Sentry. Aprende cómo en nuestro contenido de <PlatformLink to="/tracing/instrumentation/custom-instrumentation/">Instrumentación personalizada</PlatformLink>.

Mientras pruebas, configura <PlatformIdentifier name="traces-sample-rate" /> en `1.0`, ya que así te aseguras de que cada transacción se enviará a Sentry. Cuando termines, quizá quieras establecer un valor más bajo para <PlatformIdentifier name="traces-sample-rate" />, o cambiar a <PlatformIdentifier name="traces-sampler" /> para muestrear y filtrar tus transacciones de forma selectiva según datos contextuales.
