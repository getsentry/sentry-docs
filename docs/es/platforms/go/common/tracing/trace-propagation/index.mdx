Si el panorama general de aplicaciones que deseas observar con Sentry abarca más que un solo servicio o aplicación, el trazado distribuido puede aportar mucho valor.

<div id="what-is-distributed-tracing">
  ## ¿Qué es el trazado distribuido?
</div>

En el contexto de seguir eventos a través de un sistema distribuido, el trazado distribuido es una potente herramienta de depuración. Imagina tu aplicación como una amplia red de partes interconectadas. Por ejemplo, tu sistema puede estar distribuido en distintos servidores o tu aplicación puede dividirse en servicios de backend y frontend, cada uno con su propia pila tecnológica.

Cuando ocurre un error o un problema de rendimiento, puede resultar difícil identificar la causa raíz debido a la complejidad de un sistema así. El trazado distribuido te ayuda a seguir el recorrido de un evento mientras atraviesa esta intrincada red, registrando cada paso que da. Al examinar estos trazos, puedes reconstruir la secuencia de eventos que condujo al evento de interés, identificar los componentes específicos involucrados y comprender sus interacciones. Esta visibilidad detallada te permite diagnosticar y resolver problemas de forma más eficaz, mejorando en última instancia la fiabilidad y el rendimiento de tu sistema distribuido.

Si ejecutas aplicaciones JavaScript en tu sistema distribuido, asegúrate de que esos dos encabezados estén en tu lista de permitidos de CORS y que no sean bloqueados ni eliminados por tus servidores proxy, puertas de enlace o firewalls.

Recuerda que, para propagar la información de trazas en todo tu sistema distribuido, debes usar Sentry en todos los servicios y aplicaciones implicados. Consulta la documentación del SDK correspondiente para aprender cómo habilitar el rastreo distribuido en cada plataforma.
