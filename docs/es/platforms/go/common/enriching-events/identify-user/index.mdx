Los usuarios se componen de algunas piezas de información clave que conforman una identidad única en Sentry. Cada una es opcional, pero **al menos una** debe estar presente para que el SDK de Sentry capture al usuario:

<DefinitionList>
  ### `id`

  Tu identificador interno del usuario.

  ### `username`

  El nombre de usuario. Suele usarse como una etiqueta más útil que el id interno.

  ### `name`

  El nombre y/o apellido del usuario.

  ### `email`

  Una alternativa o un complemento al nombre de usuario. Sentry reconoce direcciones de correo electrónico y puede mostrar, por ejemplo, Gravatars y habilitar funciones de mensajería.

  ### `ip_address`

  La dirección IP del usuario. Si el usuario no está autenticado, Sentry usa la dirección IP como identificador único del usuario.
  El SDK intentará obtener la dirección IP de los datos de la solicitud HTTP en las solicitudes entrantes (campo `request.env.REMOTE_ADDR` en JSON), si está disponible. Esto requiere que <PlatformIdentifier name="send-default-pii" /> esté establecido en `true` en las opciones del SDK.

  Si el `ip_address` del usuario está configurado en `"{{auto}}"`, Sentry inferirá la dirección IP a partir de la conexión entre tu app y el servidor de Sentry. Si se omite el campo, el valor predeterminado es `null`.

  Para no almacenar las direcciones IP de los usuarios en los datos de tus eventos, ve a la configuración de tu proyecto, haz clic en “Security &amp; Privacy” y habilita “Prevent Storing of IP Addresses”, o usa el depurado de [server-side data](/es/security-legal-pii/scrubbing/) de Sentry para eliminar `$user.ip_address`. Agregar esta regla, en última instancia, anula cualquier otra lógica.
</DefinitionList>

Además, puedes proporcionar pares clave/valor arbitrarios más allá de los nombres reservados, y el SDK de Sentry los almacenará junto con el usuario.
