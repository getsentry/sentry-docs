---
title: Datos recopilados
description: "Consulta qué datos recopila el SDK de Sentry."
sidebar_order: 1
---

Sentry se toma muy en serio la privacidad de los datos y cuenta con configuraciones predeterminadas que priorizan la seguridad de la información, especialmente cuando se trata de datos de identificación personal (PII). Al agregar el SDK de Sentry a tu aplicación, permites que recopile datos y los envíe a Sentry durante el tiempo de ejecución de tu aplicación.

Los tipos de categorías y la cantidad de datos recopilados varían según las integraciones que hayas habilitado en el SDK de Sentry. Esta página enumera las categorías de datos que recopila el SDK de Sentry para Go.

Para muchas de las categorías enumeradas aquí, es necesario habilitar la <PlatformLink to="/configuration/options">opción SendDefaultPII</PlatformLink>.

<div id="http-headers">
  ## Encabezados HTTP
</div>

De forma predeterminada, el SDK de Sentry no envía ningún encabezado HTTP. Incluso cuando el envío de encabezados HTTP está habilitado, contamos con una [lista de bloqueo](https://github.com/getsentry/sentry-go/blob/master/interfaces.go#L157-163) que filtra cualquier encabezado que contenga datos sensibles.

Para comenzar a enviar encabezados HTTP, configura la <PlatformLink to="/configuration/options">opción SendDefaultPII</PlatformLink> en true.

<div id="cookies">
  ## Cookies
</div>

De forma predeterminada, el SDK de Sentry no envía cookies. Sentry intenta eliminar cualquier cookie que contenga información sensible, como las cookies de ID de sesión y del token CSRF.

Si quieres enviar cookies, establece la opción <PlatformLink to="/configuration/options">SendDefaultPII option</PlatformLink> en true.

<div id="users-ip-addresses">
  ## Direcciones IP de los usuarios
</div>

De forma predeterminada, el SDK de Sentry no envía la dirección IP del usuario. Una vez habilitado, los servicios de backend de Sentry inferirán la dirección IP del usuario a partir de la solicitud entrante, a menos que ciertas integraciones que puedes habilitar anulen este comportamiento.

Para habilitar el envío de la dirección IP del usuario, establece la opción <PlatformLink to="/configuration/options">SendDefaultPII</PlatformLink> en true.

<div id="request-url">
  ## URL de la solicitud
</div>

La URL completa de las solicitudes HTTP entrantes **siempre se envía a Sentry**. Según tu aplicación, esto podría incluir datos de identificación personal (PII).

<div id="request-query-string">
  ## Cadena de consulta de la solicitud
</div>

La cadena de consulta completa de las solicitudes HTTP entrantes **siempre se envía a Sentry**. Según tu aplicación, esto podría incluir datos personales (PII).
