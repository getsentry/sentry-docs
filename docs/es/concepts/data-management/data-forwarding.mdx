Sentry permite reenviar eventos de error procesados a determinados proveedores de terceros, como [Segment](https://segment.com) y [Amazon SQS](https://aws.amazon.com/sqs/).

Esto suele ser útil cuando quieres analizar las excepciones con mayor profundidad o habilitar a otros equipos, como el de inteligencia de negocio.

Configura el reenvío de datos yendo a **[Project] » Settings » Data Forwarding** y luego proporcionando la información requerida para la integración correspondiente.

Solo se reenviarán eventos de error. No se admite el reenvío de eventos de transacción.

<div id="amazon-sqs">
  ## Amazon SQS
</div>

La integración con Amazon SQS facilita y agiliza el envío de excepciones de vuelta a tus propios sistemas.

La carga útil para Amazon es idéntica a nuestra carga útil estándar de eventos de la API y evolucionará con el tiempo. Para más detalles sobre el formato de estos datos, consulta nuestra [documentación de la API](/es/api/events/retrieve-an-event-for-a-project/).

<div id="segment">
  ## Segment
</div>

La integración de Segment generará eventos de *Error Captured* dentro de tu canalización de datos. Estos eventos **solo** se registrarán para eventos de `error`, y únicamente cuando haya un ID presente en el contexto del usuario. La forma general del evento será aproximadamente la siguiente:

```json
{
  "userId": "1",
  "event": "Error capturado",
  "properties": {
    "environment": "production",
    "eventId": "002c8bbde8324dae9f12a0b96f5b1e51",
    "exceptionType": "ValueError",
    "release": "a2def1",
    "transaction": "/api/0/users/{user}/",
    "userAgent": "Mozilla/5.0 (X11; Linux x86_64; rv:54.0) Gecko/20100101 Firefox/54.0",
    "page": {
      "url": "https://sentry.io/api/0/users/{user}/",
      "method": "GET",
      "search": "",
      "referer": "https://sentry.io/"
    }
  },
  "timestamp": "2017-05-20T15:29:06Z"
}
```
