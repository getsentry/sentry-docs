  ## Visión general
</div>

<div id="what-is-a-sample-rate">
  ### ¿Qué es una tasa de muestreo?
</div>

Agregar Sentry a tu app te ofrece mucha información valiosa sobre errores y rendimiento que de otro modo no tendrías. Y mucha información es buena, siempre que sea la correcta y en una cantidad razonable. Puedes usar tasas de muestreo para capturar solo un porcentaje específico de eventos, como errores y trazas.

<div id="why-not-capture-all-events">
  ### ¿Por qué no capturar todos los eventos?
</div>

Recomendamos muestrear tus transacciones por dos razones:

1. Capturar un único trace tiene una sobrecarga mínima, pero capturar traces en cada carga de página o solicitud a la API puede añadir una carga indeseable a tu sistema.

2. Habilitar el muestreo te permite gestionar mejor la cantidad de eventos enviados a Sentry, para ajustar el volumen a las necesidades y al presupuesto de tu organización.

Elige una tasa de muestreo que equilibre la precisión de los datos con el rendimiento y el almacenamiento. Procura recopilar suficientes datos para obtener información útil sin sobrecargar los recursos. Si no estás seguro, empieza con una tasa baja y auméntala gradualmente a medida que entiendas mejor tus patrones de tráfico.

<div id="sampling-rate-options">
  ## Opciones de frecuencia de muestreo
</div>

<Alert>
  Algunas de las opciones a continuación no están disponibles en todos los SDK. Consulta nuestra documentación específica de cada plataforma para obtener más información.
</Alert>

<div id="error-events">
  ### Eventos de error
</div>

* **SampleRate** - Configura la tasa de muestreo para los eventos de error, en un rango de 0.0 a 1.0. El valor predeterminado es 1.0, lo que significa que se enviará el 100% de los eventos de error. Si se establece en 0.1, solo se enviará el 10% de los eventos de error. Los eventos se seleccionan al azar.

<div id="tracing">
  ### Rastreo
</div>

* **tracesSampleRate** - Un número entre `0` y `1` que controla la probabilidad de que una transacción concreta se envíe a Sentry. (`0` representa el 0% y `1` el 100%.) Se aplica por igual a todas las transacciones creadas en la app. Debe definirse este o `tracesSampler` para habilitar el rastreo.

* **tracesSampler** - Una función que determina la probabilidad de que una transacción concreta se envíe a Sentry. Recibe automáticamente información sobre la transacción y el contexto en el que se crea, y debe devolver un número entre 0 (0% de probabilidad) y 1 (100% de probabilidad). También puede usarse para filtrar transacciones devolviendo 0 para las no deseadas. Debe definirse este o `tracesSampleRate` para habilitar el rastreo.

Obtén más información sobre el [rastreo](/es/product/tracing/) en Sentry.

<div id="session-replay">
  ### Reproducción de sesiones
</div>

* **replaysSessionSampleRate** - La tasa de muestreo para las reproducciones que comienzan a grabar de inmediato y abarcan toda la sesión del usuario. 1.0 recopila todas las reproducciones y 0 no recopila ninguna.

* **replaysOnErrorSampleRate** - La tasa de muestreo para las reproducciones que se registran cuando ocurre un error. Este tipo de reproducción guardará hasta un minuto de eventos anteriores al error y seguirá grabando hasta que finalice la sesión. 1.0 captura todas las sesiones con error y 0 no captura ninguna.

Obtén más información sobre la [reproducción de sesiones](/es/product/explore/session-replay/) en Sentry.
