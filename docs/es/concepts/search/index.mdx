La búsqueda está disponible en varias funciones de [sentry.io](https://sentry.io), como **Issues**, **Discover** y **Dashboards**.

<div id="query-syntax">
  ## Sintaxis de consulta
</div>

<Alert>
  Solo necesitarás usar la sintaxis de consulta si estás utilizando la [API](/es/api/) de Sentry. Recibirás sugerencias precargadas cuando empieces a escribir tus términos en la barra de búsqueda en cualquier parte de [Sentry.io](https://sentry.sentry.io/).
</Alert>

Las consultas de búsqueda se construyen con el patrón `key:value`, con una búsqueda sin formato opcional al final. Cada par `key:value` es un `token` y la búsqueda sin formato opcional también es un único `token`. Los `tokens` formados por pares `key:value` se tratan como propiedades de incidencias o eventos. La búsqueda sin formato opcional se considera un único `token` y busca en los títulos y mensajes de los eventos.

Por ejemplo:

```
is:resolved user.username:"Jane Doe" server:web-8 example error
```

En el ejemplo anterior, hay tres claves (`is:`, `user.username:`, `server:`), pero cuatro tokens:

* `is:resolved`
* `user.username:"Jane Doe"`
* `server:web-8`
* `example error`

Los tokens `is:resolved` y `user.username:"Jane Doe"` son tokens de búsqueda estándar porque ambos usan palabras clave reservadas. Consulta [Propiedades de issues](/es/concepts/search/searchable-properties/#issue-properties) y [Propiedades de eventos](/es/concepts/search/searchable-properties/#event-properties) para conocer el uso adecuado de palabras clave. El token `server:web-8` señala una etiqueta personalizada enviada por el SDK de Sentry. Consulta [Etiquetas personalizadas](/es/concepts/search/searchable-properties/#custom-tags) para obtener más información sobre cómo configurar etiquetas.

El token `example error` utiliza la búsqueda sin procesar opcional y se pasa como parte de la consulta de búsqueda de issues (que usa una coincidencia CONTAINS similar a SQL). Al usar la búsqueda sin procesar opcional, puedes proporcionar una sola cadena y la consulta usa esa cadena completa.

Los términos de búsqueda deberían autocompletarse y, cuando no lo hagan, significa que tu término no es compatible con el conjunto de datos. Si completas el término y presionas la tecla Intro/Enter, se mostrará un mensaje de error.

<div id="comparison-operators">
  ### Operadores de comparación
</div>

La búsqueda de Sentry admite el uso de operadores de comparación:

* mayor que (**`>`**)
* menor que (**`<`**)
* mayor o igual que (**`>=`**)
* menor o igual que (**`<=`**)

Por lo general, cuando buscas con propiedades numéricas o de duración, deberías usar operadores de comparación en lugar de solo dos puntos (`:`) para buscar coincidencias exactas, ya que es poco probable que exista una coincidencia exacta.

Aquí tienes algunos ejemplos de búsquedas válidas con operadores de comparación:

* `event.timestamp:>2023-09-28T00:00:00-07:00`
* `count_dead_clicks:<=10`
* `transaction.duration:>5s`

<div id="using-or-and-and">
  ### Uso de `OR` y `AND`
</div>

<Alert>
  Las condiciones de búsqueda `OR` y `AND` solo están disponibles para [Discover](/es/product/explore/discover-queries/), [Insights Overview](/es/product/insights/overview/) y [Metric Alerts](/es/product/alerts-notifications/metric-alerts/).
</Alert>

Usa `OR` y `AND` entre tokens y paréntesis `()` para agrupar condiciones. `AND` también puede usarse entre campos agregados y no agregados. Sin embargo, `OR` no.

* Los campos no agregados filtran datos según etiquetas o atributos específicos. Por ejemplo, `user.username:jane` es un campo no agregado.

* Los agregados filtran datos en escalas numéricas. Por ejemplo, `count()` es una función agregada y `count():>100` es un filtro agregado.

Algunos ejemplos de uso de la condición `OR`:

```
# una consulta `OR` válida
browser:Chrome OR browser:Opera

# una consulta `OR` no válida
user.username:janedoe OR count():>100
```

Además, las consultas priorizan `AND` sobre `OR`. Por ejemplo, &quot;x `AND` y `OR` z&quot; equivale a &quot;(x `AND` y) `OR` z&quot;. Se pueden usar paréntesis para cambiar la agrupación. Por ejemplo, &quot;x `AND` (y `OR` z)&quot;.

<div id="multiple-values-on-the-same-key">
  ### Varios valores en la misma clave
</div>

Puedes buscar varios valores para la misma clave colocando los valores en una lista. Por ejemplo, &quot;x:[value1, value2]&quot; encontrará los mismos resultados que &quot;x:value1 `OR` x:value2&quot;. Al hacerlo, la búsqueda devuelve incidencias/eventos que coinciden con cualquiera de los términos.

Un ejemplo de búsqueda en la misma clave con una lista de valores:

```
release:[12.0, 13.0]
```

Actualmente no puedes usar este tipo de búsqueda con la palabra clave `is`, ni puedes usar comodines con este tipo de búsqueda.

<div id="explicit-tag-syntax">
  ### Sintaxis de etiqueta explícita
</div>

Te recomendamos no usar nunca palabras clave reservadas (como `project_id`) como etiquetas. Pero si lo haces, debes usar la siguiente sintaxis para buscarla:

```
tags[project_id]:tag_value
```

<div id="advanced">
  ### Avanzado
</div>

Sentry también ofrece las siguientes opciones de búsqueda avanzadas:

<div id="exclusion">
  #### Exclusión
</div>

De forma predeterminada, los términos de búsqueda usan el operador `AND`; es decir, devuelven la intersección de los issues/eventos que coinciden con todos los términos de búsqueda.

Para cambiar esto, puedes usar el operador de negación `!` para *excluir* un parámetro de búsqueda.

```
is:unresolved !user.email:example@customer.com
```

En el ejemplo anterior, la consulta de búsqueda devuelve todas las incidencias que están sin resolver *y* que no han afectado al usuario con la dirección de correo electrónico `example@customer.com`.

<div id="wildcards">
  #### Comodines
</div>

Buscar admite el comodín `*` como marcador de posición para caracteres y cadenas específicas.

```
browser:"Safari 11*"
```

En el ejemplo anterior, la consulta de búsqueda hará coincidir valores de `browser` como &quot;Safari 11.0.2&quot;, &quot;Safari 11.0.3&quot;, etc.

También puedes combinar operadores de la siguiente manera:

```
!message:"*Timeout"
```

En el ejemplo anterior, la consulta de búsqueda devuelve resultados que no tienen valores de mensaje como `ConnectionTimeout`, `ReadTimeout`, etc.

<div id="page-filters">
  ## Filtros de página
</div>

Los filtros de página te permiten acotar los resultados mostrados en una página seleccionando proyectos, entornos y rangos de fechas específicos. Una vez que configures tus filtros, se mantendrán mientras navegas por las páginas de Sentry.

![Selector de proyecto](./img/project-selector.png)

Si quieres ver los datos de todos tus proyectos, haz clic en &quot;Clear&quot; en el selector de proyectos para quitar cualquier filtro de proyecto.
