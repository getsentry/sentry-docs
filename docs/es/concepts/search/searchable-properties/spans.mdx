Los spans son el elemento de datos subyacente capturado en los SDK de Sentry que, en conjunto, conforman un trace.

Puedes buscar por propiedades de span en las siguientes páginas de [Sentry](https://sentry.io):

* [Traces](/es/product/explore/traces/) - al buscar un trace que contenga spans con propiedades coincidentes

Cuando busques propiedades de span en la página **Traces**, la búsqueda devolverá cualquier trace que contenga *uno o más spans* que coincidan con tu filtro de propiedades de span.

<div id="searchable-properties">
  ## Propiedades que se pueden buscar
</div>

A continuación encontrarás una lista de claves y tokens que se pueden usar en la búsqueda de spans:

<DefinitionList>
  ### `action`

  El tipo de acción del span; por ejemplo, `SELECT` para un span de SQL o `POST` para un span de HTTP.

  * **Tipo:** cadena

  ### `browser.name`

  El nombre del navegador que generó el span.

  * **Tipo:** cadena

  ### `acierto de caché`

  Si una lectura de caché ha dado acierto o fallo para la clave de caché consultada. Los valores son `'true'` o `'false'`.

  * **Tipo:** cadena

  ### `description`

  Descripción parametrizada y depurada del span.

  * **Tipo:** cadena

  ### `device.class`

  La clase del dispositivo es un campo sintético que se calcula usando información del dispositivo presente en el contexto, como el modelo (en dispositivos iOS) y especificaciones del dispositivo como `processor_frequency` (en dispositivos Android).

  * **Tipo:** cadena

  ### `dominio`

  Alcance general de la acción del span; por ejemplo, las tablas involucradas en un span de `db` o el nombre de host en un span de `http`.

  * **Tipo:** cadena

  ### `duration`

  El tiempo total que dura el span.

  * **Tipo:** duración

  ### `environment`

  Se refiere a la convención de nombres de tu despliegue de código. Por ejemplo, `development`, `testing`, `staging`, etc. [Más información](/es/product/sentry-basics/environments/). En algunas páginas de [Sentry](https://sentry.io/), puedes filtrar por entorno usando un menú desplegable.

  * **Tipo:** cadena

  ### `file_extension`

  La extensión de archivo de un tramo de recurso.

  * **Tipo:** cadena

  ### `group`

  Hash único de la descripción del span.

  * **Tipo:** cadena

  ### `http.decoded_response_content_length`

  El [tamaño del cuerpo descodificado](https://developer.mozilla.org/en-US/docs/Web/API/PerformanceResourceTiming/decodedBodySize) del recurso.

  * **Tipo:** cadena

  ### `http.response_content_length`

  El [tamaño del cuerpo codificado](https://developer.mozilla.org/en-US/docs/Web/API/PerformanceResourceTiming/encodedBodySize) del recurso.

  * **Tipo:** cadena

  ### `http.response_transfer_size`

  El [tamaño total de la transferencia](https://developer.mozilla.org/en-US/docs/Web/API/PerformanceResourceTiming/transferSize) del recurso.

  * **Tipo:** cadena

  ### `messaging.destination.name`

  El nombre de la cola o del tema en el que se escribe o publica el mensaje. El consumidor también lee de esta cola.

  * **Tipo:** cadena

  ### `messaging.message.id`

  El identificador único de un mensaje de la cola o del sistema de mensajería.

  * **Tipo:** cadena

  ### `module`

  El módulo de Insights con el que está asociado el span, por ejemplo, `cache`, `db`, `http`, etc.

  * **Tipo:** cadena

  ### `op`

  La operación del span, por ejemplo, `http.client`, `middleware`.

  * **Tipo:** cadena

  ### `os.name`

  Nombre del sistema operativo.

  * **Tipo:** cadena

  ### `platform`

  Nombre de la plataforma. De manera predeterminada es `other` y solo aplica a plataformas de esta lista:
  `as3`, `c`, `cfml`, `cocoa`, `csharp`, `elixir`, `go`, `groovy`, `haskell`, `java`, `javascript`, `native`, `node`, `objc`, `other`, `perl`, `php`, `python`, `ruby`

  * **Tipo:** cadena

  ### `release`

  Una release es una versión de tu código desplegada en un entorno. Puedes crear un token que coincida exactamente con una release, o elegir la más reciente usando `release:latest`. [Más información](/es/product/releases/usage/sorting-filtering/#latest-release).

  * **Tipo:** cadena

  ### `resource.render_blocking_status`

  El [estado de bloqueo de renderizado](https://developer.mozilla.org/en-US/docs/Web/API/PerformanceResourceTiming/renderBlockingStatus) del recurso.

  * **Tipo:** cadena

  ### `sdk.name`

  Nombre del SDK de Sentry que envió el evento.

  * **Tipo:** cadena

  ### `sdk.version`

  Versión del SDK de Sentry que envió el evento.

  * **Tipo:** cadena

  ### `self_time`

  La duración del span, excluyendo la duración de sus spans secundarios.

  * **Tipo:** cadena

  ### `status`

  Estado de la operación que representa el span.

  ### `status_code`

  El código de estado de la respuesta HTTP.

  * **Tipo:** cadena

  ### `system`

  Sistema de bases de datos (p. ej., `postgresql` o `mysql`).

  * **Tipo:** cadena

  ### `trace.status`

  El estado de éxito o fracaso del rastro del span.

  * **Tipo:** cadena

  ### `transaction`

  Nombre de la transacción que la contiene.

  * **Tipo:** cadena

  ### `transaction.method`

  Método HTTP de la transacción que lo contiene.

  * **Tipo:** cadena

  ### `transaction.op`

  Operación de la transacción que la contiene.

  * **Tipo:** cadena

  ### `user.email`

  Una alternativa o un complemento al nombre de usuario. Sentry reconoce las direcciones de correo electrónico y, por ello, puede mostrar elementos como Gravatars y habilitar funciones de mensajería.

  * **Tipo:** cadena

  ### `user.id`

  Identificador interno de la aplicación específico para el usuario.

  * **Tipo:** cadena

  ### `user.username`

  Nombre de usuario, que suele ser una etiqueta mejor que `user.id`.

  * **Tipo:** cadena
</DefinitionList>
