[Issues](/es/product/issues/) son un conjunto de uno o más eventos de error. Las propiedades de los issues que se pueden buscar incluyen estado, asignación, recuentos agregados y antigüedad. Si configuraste el [seguimiento de evaluación para feature flags](/es/product/issues/issue-details/feature-flags/#evaluation-tracking), puedes buscar issues que tengan eventos de error en los que el valor evaluado del feature flag sea `true` o `false`. Puedes buscar por propiedades de los issues en la página **Issues** y en **Dashboards**, en el generador de widgets, según el conjunto de datos que selecciones.

<div id="searchable-properties">
  ## Propiedades con capacidad de búsqueda
</div>

A continuación, se muestra una lista de claves y tokens que se pueden usar en la búsqueda de issues.

<Alert>
  Solo necesitarás usar la sintaxis de consulta para los tipos de propiedades con capacidad de búsqueda **datetime** y **relative time** si estás usando la [API](/es/api/) de Sentry.
</Alert>

<DefinitionList>
  ### `edad`

  Devuelve incidencias creadas desde el momento indicado por el valor. La sintaxis es similar al comando find de Unix. Sufijos admitidos: `m - minutos`, `h - horas`, `d - días`, `w - semanas`. Por ejemplo, `age:-24h` devuelve incidencias nuevas en las últimas 24 horas, mientras que `age:+12h` devuelve las que tienen más de 12 horas. Introducir `age:+12h age:-24h` devolvería incidencias creadas entre 12 y 24 horas atrás.

  * **Tipo:** tiempo relativo

  ### `app.en_foreground`

  Indica si la app está en primer plano o en segundo plano. Los valores posibles son `1/0` o `true/false`

  * **Tipo:** booleano

  ### `asignado`

  Devuelve las incidencias asignadas al/los usuario(s) o equipo(s) definidos. Los valores pueden ser un ID de usuario (tu dirección de correo), `me` para ti mismo, `none` para sin asignar, `my_teams` o `#team-name` para equipos a los que perteneces.

  * **Tipo:** usuario de equipo u organización

  ### `asignado_o_sugerido`

  Devuelve incidencias asignadas a los usuarios o equipos definidos, o bien sugeridas para asignarse a esos usuarios o equipos y que actualmente no tienen asignación. Los posibles asignatarios se determinan comparando las [reglas de propiedad](/es/product/issues/ownership-rules/) y los [commits sospechosos](/es/product/issues/suspect-commits/). Los valores pueden ser un ID de usuario (tu dirección de correo), `me` para ti mismo, `none` para sin asignado/sugerencia, `my_teams` o `#team-name` para equipos a los que perteneces.

  * **Tipo:** usuario de equipo u organización

  ### `bookmarks`

  Devuelve los issues guardados como favoritos por el usuario definido. Los valores pueden ser tu ID de usuario (tu dirección de correo electrónico) o `me` para ti.

  * **Tipo:** usuario de equipo u organización

  ### `device.arch`

  Arquitectura de la CPU

  * **Tipo:** cadena

  ### `device.brand`

  Marca del dispositivo

  * **Tipo:** cadena

  ### `device.family`

  Familia del dispositivo. Por lo general, la parte común del nombre del modelo entre generaciones. Por ejemplo, iPhone, Samsung Galaxy.

  * **Tipo:** cadena

  ### `device.locale`

  En desuso

  * **Tipo:** cadena

  ### `device.model_id`

  Revisión interna de hardware para identificar el dispositivo con precisión.

  * **Tipo:** n/d

  ### `device.orientation`

  Describe la orientación del dispositivo y puede ser `portrait` o `landscape`.

  * **Tipo:** cadena

  ### `device.screen_density`

  Densidad de píxeles de la pantalla del dispositivo.

  * **Tipo:** cadena

  ### `device.screen_dpi`

  Cantidad de puntos por pulgada de la pantalla del dispositivo.

  * **Tipo:** cadena

  ### `device.screen_height_pixels`

  Altura de la pantalla del dispositivo en píxeles.

  * **Tipo:** cadena

  ### `device.screen_width_pixels`

  Ancho de la pantalla del dispositivo en píxeles.

  * **Tipo:** cadena

  ### `device.uuid`

  Obsoleto

  * **Tipo:** UUID

  ### `dist`

  Distingue variantes de compilación o despliegue de la misma versión de una aplicación. Por ejemplo, “dist” puede ser el número de compilación de una build de Xcode o el código de versión de una build de Android.

  * **Tipo:** cadena

  ### `error.controlado`

  Indica si el usuario ha manejado la excepción — por ejemplo, usando try...catch. Un error se considera manejado si todos los rastros de pila lo manejan. Los valores pueden ser `1/0` o `true/false`

  * **Tipo:** booleano

  ### `error.main_thread`

  Indica si el error se produjo en el hilo principal. Los valores pueden ser `1/0` o `true/false`

  * **Tipo:** booleano

  ### `error.mechanism`

  Un objeto que describe el mecanismo que generó esta excepción.

  * **Tipo:** matriz

  ### `error.type`

  Tipo de excepción. Por ejemplo, `ValueError`.

  * **Tipo:** matriz

  ### `error.no manejado`

  La inversión de `error.handled`.

  * **Tipo:** booleano

  ### `error.value`

  Valor original de un campo que provoca o presenta el error.

  * **Tipo:** matriz

  ### `event.timestamp`

  Devuelve issues con fecha y hora coincidentes.

  * **Tipo:** fecha y hora

  ### `event.type`

  Tipo de evento (transacción, error, predeterminado, CSP, etc.). El tipo de transacción no está disponible en **Issues**.

  * **Tipo:** cadena

  ### `firstRelease`

  Devuelve los problemas detectados por primera vez en la versión indicada. Puede coincidir exactamente con la versión de un lanzamiento, o usar `first-release:latest` para elegir el lanzamiento más reciente.

  * **Tipo:** fecha y hora

  ### `primeraVezVisto`

  Devuelve incidencias cuyo primer avistamiento coincide. La sintaxis es la misma que en `age`.

  * **Tipo:** fecha y hora

  ### `flags`

  Para [evaluaciones de flags de funciones](/es/product/issues/issue-details/feature-flags/#evaluation-tracking) establecidas en `true` o `false`, el nombre del flag de función. Por ejemplo, la sintaxis para buscar un flag con la clave `my_flag` y el valor `true` es `flags["my_flag"]:true`.

  * **Tipo:** booleano

  ### `geo.city`

  Nombre completo de la ciudad

  * **Tipo:** cadena

  ### `geo.country_code`

  Código de país de la ISO 3166-1

  * **Tipo:** cadena

  ### `geo.region`

  Nombre oficial del país

  * **Tipo:** cadena

  ### `has`

  Devuelve resultados que contienen la etiqueta o el campo definidos, pero no buscan por el valor de esa etiqueta o campo. Por ejemplo, al ingresar `has:user` se encontrarían eventos con la etiqueta `user`.

  * **Tipo:** error

  ### `http.method`

  Método HTTP de la [solicitud](https://develop.sentry.dev/sdk/data-model/event-payloads/request/) que creó el evento.

  * **Tipo:** cadena

  ### `http.referer`

  Identifica la página web desde la que se solicitó el recurso.

  * **Tipo:** cadena

  ### `http.status_code`

  Código de estado HTTP que indica si una respuesta fue correcta. Por ejemplo, `200` o `404`.

  * **Tipo:** cadena

  ### `http.url`

  URL completa de la solicitud que provocó el error, pero sin ningún parámetro

  * **Tipo:** cadena

  ### `id`

  El ID del evento o de la repetición. En **Issues**, usa solo el valor del ID sin la clave `id`.

  * **Tipo:** UUID

  ### `is`

  Las propiedades de un problema. Los valores pueden ser: `unresolved`, `resolved`, `archived`, `assigned`, `unassigned`, `for_review`, `linked` o `unlinked`. El valor `for_review` filtra los problemas no resueltos y solo muestra los recién no resueltos o los nuevos que aún no han sido marcados como “reviewed”. Los valores `linked` y `unlinked` devuelven problemas según si están vinculados a un rastreador externo o no.

  * **Tipo:** estado

  ### `issue`

  El código corto de la incidencia, por ejemplo `SENTRY-ABC`.

  * **Tipo:** cadena

  ### `issue.category`

  La categoría de la incidencia. Por ejemplo: `error`, `rendimiento`, `replay` y `cron`.

  * **Tipo:** cadena

  ### `issue.type`

  El tipo específico de incidencia. Por ejemplo, `issue.type:performance_n_plus_one_db_queries` devuelve las incidencias de rendimiento por consultas a la base de datos n+1.

  * **Tipo:** cadena

  ### `últimaVezVisto`

  Fecha y hora de la última vez que se vio el evento. Por ejemplo, `lastSeen:+30d` devuelve incidencias vistas por última vez hace 30 días o más; `lastSeen:-2d` devuelve incidencias vistas por última vez en los últimos dos días. Es similar a `age`.

  * **Tipo:** fecha y hora

  ### `nivel`

  Gravedad del evento (por ejemplo: fatal, error, advertencia). En transacciones, establézcalo siempre en info.

  * **Tipo:** cadena de caracteres

  ### `location`

  Ubicación donde se produjo el error.

  * **Tipo:** cadena

  ### `message`

  Devuelve errores cuyo mensaje coincida o transacciones cuyo nombre de transacción coincida. También hace coincidir cualquier mensaje que contenga el valor proporcionado. Buscar `message:undefined` hará coincidir un evento con el mensaje `undefined is not an object`. Las búsquedas de texto sin formato (búsquedas sin la clave `message`) también se comprueban contra este campo. En el caso de errores, el mensaje puede ser una concatenación de elementos, por lo que las búsquedas podrían arrojar resultados inesperados.

  * **Tipo:** cadena

  ### `os.build`

  La revisión interna de la compilación del sistema operativo.

  * **Tipo:** cadena

  ### `os.kernel_version`

  La cadena de versión del kernel, independiente. Suele ser toda la salida de la llamada al sistema `uname`.

  * **Tipo:** cadena

  ### `os.distribution_name`

  El nombre de la distribución de Linux. Corresponde a `ID` en [`/etc/os-release/`](https://www.freedesktop.org/software/systemd/man/latest/os-release.html).

  * **Tipo:** cadena

  ### `os.distribution_version`

  La versión de la distribución de Linux. Corresponde a `VERSION_ID` en [`/etc/os-release`](https://www.freedesktop.org/software/systemd/man/latest/os-release.html).

  * **Tipo:** cadena

  ### `platform.name`

  Nombre de la plataforma

  * **Tipo:** cadena

  ### `project`

  El nombre del proyecto. En algunas páginas de [sentry.io](https://sentry.io), también puedes filtrar por proyecto mediante un menú desplegable.

  * **Tipo:** cadena

  ### `project.id`

  El ID del proyecto.

  * **Tipo:** número

  ### `lanzamiento`

  Un lanzamiento es una versión de tu código desplegada en un entorno. Puedes crear un token que coincida exactamente con la versión de un lanzamiento, o usar `release:latest` para seleccionar el lanzamiento más reciente. [Más información](/es/product/releases/usage/sorting-filtering/#latest-release).

  * **Tipo:** cadena

  ### `release.build`

  El número que identifica una versión de tu app. Por ejemplo, `CFBundleVersion` en iOS o `versionCode` en Android. [Más información](/es/product/releases/usage/sorting-filtering/).

  * **Tipo:** número

  ### `release.package`

  El identificador único del proyecto o la aplicación. Por ejemplo, `CFBundleIdentifier` en iOS o `packageName` en Android. [Más información](/es/product/releases/usage/sorting-filtering/).

  * **Tipo:** cadena

  ### `release.stage`

  El uso que registra tu versión en comparación con otras versiones. Los valores pueden ser `adopted`, `low` o `replaced`. [Más información](/es/product/releases/health/#adoption-stages).

  * **Tipo:** cadena

  ### `release.version`

  Una versión abreviada del nombre; el nombre sin el paquete o una versión corta del hash. [Más información](/es/product/releases/usage/sorting-filtering/).

  * **Tipo:** cadena

  ### `sdk.name`

  Nombre del SDK de Sentry que envió el evento.

  * **Tipo:** cadena

  ### `sdk.version`

  Versión del SDK de Sentry que envió el evento.

  * **Tipo:** cadena

  ### `stack.abs_path`

  La ruta absoluta del archivo de origen. En eventos, es un arreglo; en incidencias, es un único valor.

  * **Tipo:** matriz, valor único

  ### `stack.filename`

  La ruta del archivo de origen relativa al directorio raíz del proyecto. En eventos, es un arreglo. En incidencias, es un único valor.

  * **Tipo:** arreglo, valor único

  ### `stack.function`

  Nombre de la función que se está invocando. En eventos, es una matriz. En incidencias, es un único valor.

  * **Tipo:** arreglo, valor único

  ### `stack.module`

  Ruta del módulo específica de la plataforma. Por ejemplo, `sentry.interfaces.Stacktrace`. En eventos, esto es un array. En incidencias, es un único valor.

  * **Tipo:** matriz, valor único

  ### `stack.package`

  El «paquete» que contenía el frame. Según la plataforma, puede ser algo distinto. En C#, puede ser el nombre del ensamblado. En código nativo, puede ser la ruta de la biblioteca dinámica u otra cosa. En eventos, esto es una matriz. En issues, es un único valor.

  * **Tipo:** matriz, valor único

  ### `vecesVistas`

  Devuelve resultados con un recuento coincidente. (Igual que `count()` en eventos.)

  * **Tipo:** número

  ### `timestamp`

  Marca de tiempo de finalización de la transacción. Devuelve los eventos cuya fecha y hora coinciden.

  * **Tipo:** fecha y hora

  ### `title`

  Título del error o de la transacción.

  * **Tipo:** cadena

  ### `trace`

  Una traza es el registro de toda la operación que quieres medir o monitorizar — como la carga de una página— y puede localizarse mediante el UUID generado por el SDK de Sentry.

  * **Tipo:** UUID

  ### `transaction`

  Para [transacciones](/es/product/insights/overview/transaction-summary/#what-is-a-transaction), el nombre de la transacción. Para errores, el nombre de la transacción asociada.

  * **Tipo:** cadena

  ### `unreal.crash_type`

  El [tipo de contexto de fallo de Unreal](https://docs.unrealengine.com/4.27/en-US/API/Runtime/Core/GenericPlatform/ECrashContextType/)

  * **Tipo:** cadena

  ### `user.email`

  Una alternativa, o un complemento, al nombre de usuario. Sentry reconoce las direcciones de correo electrónico y, por lo tanto, puede mostrar elementos como Gravatars y habilitar funciones de mensajería.

  * **Tipo:** cadena

  ### `user.id`

  Identificador interno de la aplicación específico para el usuario.

  * **Tipo:** cadena

  ### `user.ip`

  Dirección IP del usuario. Sentry utiliza la dirección IP como identificador único para usuarios no autenticados.

  * **Tipo:** cadena

  ### `user.username`

  Nombre de usuario, que suele ser una etiqueta mejor que `user.id`.

  * **Tipo:** cadena
</DefinitionList>
