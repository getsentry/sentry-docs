La búsqueda de Sentry ofrece palabras clave reservadas, como `is`, `user`, `server` y `browser`, que puedes usar para buscar en las propiedades de incidencias, eventos y reproducciones (así como en el caso especial de versiones). También puedes crear [etiquetas personalizadas](#custom-tags) en las que buscar. Esta página ofrece orientación sobre cómo usar estas propiedades y enlaza a las respectivas listas de propiedades buscables.

<div id="search-tips">
  ## Consejos de búsqueda
</div>

Los términos de búsqueda deberían autocompletarse; cuando no lo hacen, significa que tu conjunto de datos no es válido. Si introduces un término de búsqueda no válido y pulsas Return o Enter, se mostrará un mensaje de error. Por ejemplo, en **Discover**, no puedes buscar usando una función de agrupación (propiedades con paréntesis en la clave, por ejemplo `epm()`) si aún no tienes una función de agrupación incluida en las columnas de la tabla &quot;Results&quot;.

Cuando buscas usando una propiedad que es una función, si la clave incluye un parámetro, igualmente necesitas añadir un filtro o valor para tener un token de búsqueda completo. Por ejemplo, la función `count_unique(field)` toma el campo que elijas como parámetro y luego añades un número como valor para filtrar. Así, una búsqueda típica podría verse así: `count_unique(user):>=20`.

Otras cosas a tener en cuenta:

* Las propiedades de búsqueda de tipo duración o numérico se usan generalmente con un [operador de comparación](/es/concepts/search/#comparison-operators) en lugar de solo dos puntos (`:`) para encontrar coincidencias exactas, ya que es poco probable que exista una coincidencia exacta.
* Las propiedades con la notación &quot;No requiere un parámetro&quot; siguen necesitando un filtro o valor, pero no llevan un parámetro dentro de los paréntesis. Por ejemplo, `epm():>12`.
* Los campos/parámetros predeterminados en las funciones de búsqueda se pueden actualizar. Por ejemplo, los parámetros predeterminados `column`, `operator` y `value` se pueden actualizar en `count_if(column,operator,value)`.

<div id="custom-tags">
  ## Etiquetas personalizadas
</div>

Además, puedes usar cualquier etiqueta que hayas especificado como token. Las etiquetas son pares clave/valor que se asignan a un evento, y puedes usarlas más adelante como desglose o acceso rápido para encontrar eventos relacionados.

La mayoría de los SDK admiten configurar etiquetas al <PlatformLink to="/enriching-events/tags/">configurar el alcance</PlatformLink>.

Algunos usos comunes de las etiquetas incluyen:

* El nombre de host del servidor
* La versión de tu plataforma (por ejemplo, iOS 5.0)
* El idioma del usuario

<div id="device-classification">
  ## Clasificación de dispositivos
</div>

`device.class` ofrece una forma sencilla para que los desarrolladores comprendan el nivel de rendimiento del dispositivo cliente de un evento en una única propiedad fácil de buscar. Esto es especialmente útil para proyectos que abarcan una amplia gama de dispositivos móviles, caso en el que los desarrolladores normalmente tendrían que revisar una gran variedad de especificaciones y modelos en iOS y Android.

Los valores posibles para `device.class` son `high`, `medium` y `low`, que indican el nivel de rendimiento estimado del dispositivo. Es una propiedad calculada en función de las siguientes especificaciones:

| `device.class` para iOS | `model`                             |
| ----------------------- | ----------------------------------- |
| high                    | Serie iPhone 12 o superior          |
| medium                  | Serie iPhone 7 a serie iPhone 11    |
| low                     | Serie iPhone 6 o inferior           |

| `device.class` para Android | `processor_count` | `processor_frequency` | `memory_size` |
| --------------------------- | ----------------- | --------------------- | ------------- |
| high                        | &gt;= 8              | &gt;= 2500 MHz           | &gt;= 6 GiB      |
| medium                      | &gt;= 8              | &gt;= 2000 MHz           | &gt;= 4 GiB      |
| low                         | &lt; 8               | &lt; 2000 MHz            | &lt; 4 GiB       |

Estas clasificaciones se basan en un análisis de los dispositivos móviles disponibles en el mercado actual y están sujetas a cambios a medida que el mercado evoluciona.
