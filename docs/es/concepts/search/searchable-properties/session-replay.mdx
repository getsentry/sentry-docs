[Session Replay](/es/product/explore/session-replay/) ofrece una reproducción similar a un video de las interacciones de los usuarios en un sitio o aplicación web. Se capturan todas las interacciones de los usuarios, incluidas las visitas a páginas, los movimientos del mouse, los clics y los desplazamientos, lo que ayuda a los desarrolladores a conectar los puntos entre un problema conocido y cómo lo experimentó el usuario en la interfaz.

Puedes buscar por propiedades de reproducción de sesiones en la página **Replay**.

<div id="searchable-properties">
  ## Propiedades buscables
</div>

A continuación encontrarás una lista de claves y tokens que puedes usar en la búsqueda de reproducción de sesiones.

<DefinitionList>
  ### `actividad`

  La actividad de la repetición se calcula según la cantidad de errores, la cantidad de eventos de la interfaz de usuario y la duración de la repetición. Se representa como un número del `1` al `10`.

  * **Tipo:** número

  ### `browser.name`

  Nombre del navegador web del usuario. Por ejemplo, `Chrome`, `Firefox` o `Safari`.

  * **Tipo:** cadena

  ### `browser.version`

  La cadena de la versión del navegador.

  * **Tipo:** cadena

  ### `click.alt`

  El `alt` de un elemento en el que se hizo clic. Por ejemplo, `"a good dog"` coincidiría con el elemento `<img src="/lassie.jpeg" alt="a good dog" />`.

  * **Tipo:** cadena

  ### `click.class`

  La `class` del elemento en el que se hizo clic. No hace falta el `.` inicial. Por ejemplo, `btn-primary` coincidiría con el elemento `<a class="btn btn-primary">Save</a>`.

  * **Tipo:** cadena

  ### `click.component_name`

  El nombre del componente de frontend en el que se hizo clic. Por ejemplo, `MyAwesomeComponent` coincidiría con el componente de React `<MyAwesomeComponent>`.

  Nota: Esta propiedad requiere que tu proyecto tenga [configurada la captura de nombres de componentes de React](/es/platforms/javascript/guides/react/features/component-names).

  * **Tipo:** cadena

  ### `click.id`

  El `id` de un elemento en el que se hizo clic. No es necesario incluir el `#` inicial. Por ejemplo, `reset-password` coincidiría con el elemento `<a id="reset-password">Reset</a>`.

  * **Tipo:** cadena

  ### `click.label`

  El `aria-label` de un elemento en el que se hizo clic. Por ejemplo, `Expand` coincidiría con el elemento `<button aria-label="Expand"><img src="/icons/expand.png"/></button>`.

  * **Tipo:** cadena

  ### `click.role`

  El `role` de un elemento en el que se hizo clic. Por ejemplo, `button` coincidiría tanto con `<button>Guardar</button>` como con `<a role="button">Enviar</a>`.

  * **Tipo:** cadena

  ### `click.selector`

  Un elemento identificado mediante un subconjunto de la sintaxis de selectores CSS. Por ejemplo, `#section-1`, `span.active`, `span[role=button]` o `.active[role=button]` coincidirían con el elemento `<span id="section-1" class="active" role="button"/>`. Ten en cuenta que no se admiten combinadores de CSS, pseudoselectores ni selectores de atributos distintos de `=`.

  * **Tipo:** cadena

  ### `click.tag`

  El nombre de etiqueta de un elemento en el que se hizo clic. Por ejemplo, `input` coincidiría con `<input name="username" />`.

  * **Tipo:** cadena

  ### `click.testid`

  El `data-testid` o `data-test-id` del elemento en el que se hizo clic. Por ejemplo, `user-name` coincidiría con el elemento `<a data-testid="user-name">User Name</a>`.

  * **Tipo:** cadena

  ### `click.textContent`

  El `textContent` inmediato del elemento en el que se hizo clic. Por ejemplo, `Save` coincidiría con `<button>Save</button>`, pero no con `<button><h1>Save</h1></button>`.

  * **Tipo:** cadena

  ### `click.title`

  El `title` de un elemento en el que se hizo clic. Por ejemplo, `Save this comment` coincidiría con el elemento `<a title="Save this comment" class="btn btn-primary">Save</a>`.

  * **Tipo:** cadena

  ### `count_dead_clicks`

  El número de [clics muertos](/product/explore/session-replay/replay-page-and-filters/#:~:text=Dead%20Clicks%3A%20User%20clicks%20on%20a%20and%20button%20tags%20that%20do%20not%20result%20in%20any%20page%20activity%20after%207%20seconds%20\(i.e.%20no%20HTML%20was%20added%2C%20removed%2C%20or%20updated%3B%20no%20visual%20changes%20were%20observed%20in%20the%20page\).) dentro de una reproducción.

  * **Tipo:** número

  ### `count_errors`

  La cantidad de errores en una repetición.

  * **Tipo:** número

  ### `count_rage_clicks`

  La cantidad de [clics de rabia](/product/explore/session-replay/replay-page-and-filters/#:~:text=Rage%20Clicks%3A%20Five%20or%20more%20clicks%20on%20a%20dead%20element%20\(it%20exhibits%20no%20page%20activity%20after%207%20seconds.\)%20Rage%20clicks%20are%20a%20subset%20of%20dead%20clicks) en una reproducción.

  * **Tipo:** número

  ### `count_segments`

  El número de segmentos dentro de una repetición. Más segmentos representan mayor actividad a lo largo del tiempo.

  * **Tipo:** número

  ### `count_traces`

  El número de trazas en una reproducción.

  * **Tipo:** número

  ### `count_urls`

  El número de URLs que el usuario visitó durante una grabación de sesión.

  * **Tipo:** número

  ### `dead.selector`

  Similar a la propiedad de búsqueda `click.selector`, pero solo consulta [clics muertos](/product/explore/session-replay/replay-page-and-filters/#:~:text=Dead%20Clicks%3A%20User%20clicks%20on%20a%20and%20button%20tags%20that%20do%20not%20result%20in%20any%20page%20activity%20after%207%20seconds%20\(i.e.%20no%20HTML%20was%20added%2C%20removed%2C%20or%20updated%3B%20no%20visual%20changes%20were%20observed%20in%20the%20page\).). Un elemento identificado mediante un subconjunto de la sintaxis de selectores CSS. Por ejemplo, `#section-1` o `span.active` o `span[role=button]` o `.active[role=button]` coincidirían con el elemento `<span id="section-1" class="active" role="button"/>`. Ten en cuenta que no se admiten combinadores CSS, pseudoselectores ni selectores de atributos distintos de `=`.

  * **Tipo:** cadena

  ### `device.brand`

  Marca del dispositivo.

  * **Tipo:** cadena

  ### `device.family`

  Familia del dispositivo. Por lo general, la parte común del nombre de un modelo a lo largo de generaciones. Por ejemplo, iPhone, Samsung Galaxy.

  * **Tipo:** cadena

  ### `device.model_id`

  Revisión interna de hardware para identificar el dispositivo con exactitud.

  * **Tipo:** n/d

  ### `device.name`

  Detalles del dispositivo.

  * **Tipo:** cadena

  ### `dist`

  Distingue las variantes de compilación o implementación de la misma versión de una aplicación. Por ejemplo, el dist puede ser el número de compilación de Xcode o el código de versión de Android.

  * **Tipo:** cadena

  ### `duration`

  Duración de una repetición en segundos.

  * **Tipo:** número

  ### `error_ids`

  ID de eventos de error que ocurrieron dentro de una reproducción.

  * **Tipo:** matriz

  ### `id`

  El ID del evento o de la repetición. En **Issues**, usa solo el valor del ID sin la clave `id`.

  * **Tipo:** UUID

  ### `is_archived`

  Si la reproducción ha sido archivada.

  * **Tipo:** booleano

  ### `nivel`

  Gravedad del evento (p. ej., fatal, error, advertencia). En transacciones, establécelo siempre en info.

  * **Tipo:** cadena

  ### `os.name`

  El nombre del sistema operativo. Por ejemplo, `Windows`, `Mac OS X` o `Linux`.

  * **Tipo:** cadena

  ### `os.version`

  La versión del sistema operativo.

  * **Tipo:** cadena

  ### `platform`

  Nombre de la plataforma. Esto es solo una propiedad de métricas para [plataformas válidas](https://github.com/getsentry/relay/blob/9d0d2dc54233d46043a369e1dacf7db900953c14/relay-general/src/protocol/constants.rs#L1-L21); su valor predeterminado es `other`.

  * **Tipo:** cadena

  ### `project_id`

  El ID del proyecto.

  * **Tipo:** cadena

  ### `rage.selector`

  Similar a la propiedad de búsqueda `click.selector`, pero solo busca [rage clicks](/product/explore/session-replay/replay-page-and-filters/#:~:text=Rage%20Clicks%3A%20Five%20or%20more%20clicks%20on%20a%20dead%20element%20\(it%20exhibits%20no%20page%20activity%20after%207%20seconds.\)%20Rage%20clicks%20are%20a%20subset%20of%20dead%20clicks). Un elemento identificado mediante un subconjunto de la sintaxis de selectores CSS. Por ejemplo, `#section-1`, `span.active`, `span[role=button]` o `.active[role=button]` coincidirían con el elemento `<span id="section-1" class="active" role="button"/>`. Ten en cuenta que no se admiten combinadores de CSS, pseudoselectores ni selectores de atributos distintos de `=`.

  * **Tipo:** cadena

  ### `release`

  Una versión publicada es una versión de tu código implementada en un entorno. Puedes crear un token que coincida exactamente con la versión de una publicación, o usar `release:latest` para seleccionar la publicación más reciente. [Más información](/es/product/releases/usage/sorting-filtering/#latest-release).

  * **Tipo:** cadena

  ### `tipo_de_reproducción`

  El motivo por el que se inició una repetición. Por ejemplo, `session` cuando surte efecto `replaysSessionSampleRate`, o `buffer` cuando se muestrea con `replaysOnErrorSampleRate` en lugar de `session`. Más información en la [documentación de Session Replay](/es/platforms/javascript/session-replay/understanding-sessions/).

  * **Tipo:** cadena

  ### `pantalla`

  Una pantalla o vista específica que el usuario visitó durante la reproducción. También puedes buscar varias pantallas a la vez usando `screens` y pasando un array de strings.
  Alias: `url`

  ### `sdk.name`

  Nombre del SDK de Sentry que envió el evento.

  * **Tipo:** cadena

  ### `sdk.version`

  Versión del SDK de Sentry que envió el evento.

  * **Tipo:** cadena

  ### `seen_by_me`

  Si has visto esta reproducción.
  Alias: `viewed_by_me`

  * **Tipo:** booleano

  ### `trace`

  Una traza representa el registro de toda la operación que quieres medir o rastrear — como la carga de una página, que se busca usando el UUID generado por el SDK de Sentry.

  * **Tipo:** UUID

  ### `url`

  Una URL específica que el usuario visitó durante la reproducción. También puedes buscar varias URLs a la vez usando `urls` y pasando un array de cadenas.
  Alias: `screen`

  * **Tipo:** cadena

  ### `user.email`

  Una alternativa o complemento al nombre de usuario. Sentry reconoce las direcciones de correo electrónico y, por lo tanto, puede mostrar elementos como Gravatars y habilitar funciones de mensajería.

  * **Tipo:** cadena

  ### `usuario.geo.ciudad`

  Nombre completo de la ciudad según la geolocalización del usuario.

  * **Tipo:** cadena

  ### `user.geo.country_code`

  País según la geolocalización del usuario. El código de país se basa en [ISO 3166-1](https://www.iso.org/iso-3166-country-codes.html).

  * **Tipo:** cadena

  ### `usuario.geo.region`

  Nombre completo del país según la geolocalización del usuario.

  * **Tipo:** cadena

  ### `user.geo.subdivision`

  Nombre completo de la subdivisión de la geolocalización del usuario (por ejemplo, estado, provincia, etc.).

  * **Tipo:** cadena

  ### `user.id`

  Identificador interno de la aplicación específico para el usuario.

  * **Tipo:** cadena

  ### `user.ip`

  Dirección IP del usuario. Sentry utiliza la dirección IP como identificador único para los usuarios no autenticados.

  * **Tipo:** cadena

  ### `user.username`

  Nombre de usuario, que generalmente es una etiqueta más descriptiva que `user.id`.

  * **Tipo:** cadena
</DefinitionList>
