[User Feedback](/es/product/user-feedback/) permite que tus usuarios creen informes de errores para avisarte de inmediato sobre problemas difíciles de detectar. Cada informe incluirá automáticamente reproducciones relacionadas, etiquetas y errores, lo que hace que solucionar el problema sea muy sencillo.

Puedes buscar por propiedades de comentarios de usuarios en la página **User Feedback**.

<div id="searchable-properties">
  ## Propiedades de búsqueda
</div>

A continuación se muestra una lista de claves y tokens que se pueden usar en la búsqueda de comentarios de usuario.

<DefinitionList>
  ### `assigned`

  Devuelve los envíos de comentarios de usuario asignados al(los) usuario(s) o equipo(s) definidos. Los valores pueden ser un ID de usuario (tu dirección de correo electrónico), `me` para ti mismo, `none` para sin asignación, `my_teams` o `#team-name` para equipos a los que perteneces.

  * **Tipo:** usuario de equipo u organización

  ### `browser.name`

  Nombre del navegador.

  * **Tipo:** string

  ### `device.brand`

  Marca del dispositivo.

  * **Tipo:** string

  ### `device.family`

  Familia del dispositivo. Normalmente, la parte común del nombre de un modelo a lo largo de generaciones. Por ejemplo, iPhone, Samsung Galaxy.

  * **Tipo:** string

  ### `device.model_id`

  Revisión interna de hardware para identificar exactamente el dispositivo.

  * **Tipo:** n/a

  ### `device.name`

  Nombre del dispositivo.

  * **Tipo:** string

  ### `dist`

  Distingue variantes de compilación o despliegue de la misma versión de una aplicación. Por ejemplo, el dist puede ser el número de compilación de una build de Xcode o el código de versión de una build de Android.

  * **Tipo:** string

  ### `environment`

  El entorno en el que se vio el evento por primera vez.

  * **Tipo:** string

  ### `id`

  El ID del comentario.

  * **Tipo:** UUID

  ### `is`

  Propiedades de un envío de comentarios de usuario. Los valores pueden ser: `unresolved`, `resolved`, `assigned`, `unassigned`, `linked` o `unlinked`. Los valores `linked` y `unlinked` devuelven envíos de comentarios de usuario según si están vinculados a un rastreador de incidencias externo o no.

  * **Tipo:** status

  ### `level`

  La gravedad de los envíos de comentarios de usuario. Si el comentario llegó a través del modal de informe de fallos, el nivel está vinculado al error experimentado por el usuario final. Para comentarios del widget de comentarios de usuario, el nivel es `info`.

  * **Tipo:** string

  ### `os.name`

  El nombre del sistema operativo.

  * **Tipo:** string

  ### `sdk.name`

  Nombre del SDK de Sentry que envió el evento.

  * **Tipo:** string

  ### `sdk.version`

  Versión del SDK de Sentry que envió el evento.

  * **Tipo:** string

  ### `timestamp`

  La marca de tiempo de finalización de la transacción. Devuelve eventos con fecha y hora coincidentes.

  * **Tipo:** datetime

  ### `transaction`

  El identificador del nombre del error o la transacción.

  * **Tipo:** string

  ### `url`

  La URL de la página en la que se activa el comentario.

  * **Tipo:** string

  ### `user.email`

  La dirección de correo electrónico del usuario existe solo si Sentry la recibió mediante la opción del SDK `Sentry.setUser` como etiqueta. Esta propiedad no se aplica a los correos electrónicos introducidos en el widget de comentarios de usuario, que no se pueden buscar.

  * **Tipo:** string

  ### `user.id`

  Identificador interno específico de la aplicación para el usuario.

  * **Tipo:** string

  ### `user.ip`

  Dirección IP del usuario. Sentry usa la dirección IP como identificador único para usuarios no autenticados.

  * **Tipo:** string

  ### `user.username`

  Nombre de usuario, que suele ser una mejor etiqueta que `user.id`.

  * **Tipo:** string
</DefinitionList>
