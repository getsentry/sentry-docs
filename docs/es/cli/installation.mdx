Según tu plataforma, existen distintos métodos para instalar `sentry-cli`.

<div id="manual-download">
  ## Descarga manual
</div>

Puedes encontrar la lista de versiones en [la página de lanzamientos de GitHub](https://github.com/getsentry/sentry-cli/releases/). Ofrecemos ejecutables para Linux, macOS y Windows. Es un único archivo; una vez descargado, puedes renombrarlo a `sentry-cli` o `sentry-cli.exe` para usarlo.

<div id="automatic-installation">
  ## Instalación automática
</div>

Si usas macOS o Linux, puedes utilizar el descargador automático, que obtendrá la última versión publicada y la instalará:

```bash
curl -sL https://sentry.io/get-cli/ | sh
```

Sin embargo, te recomendamos anclar una versión específica de la CLI para que tus compilaciones sean siempre reproducibles.
Para hacerlo, puedes usar exactamente el mismo método, añadiendo un especificador de versión:

```bash
curl -sL https://sentry.io/get-cli/ | SENTRY_CLI_VERSION="{{@inject apps.version('sentry-cli') }}" sh
```

Esto descargará automáticamente la versión correcta de `sentry-cli` para tu sistema operativo y la instalará. Si es necesario, te pedirá la contraseña de administrador para `sudo`. Para instalar en otra ubicación o en sistemas sin `sudo` (como Windows), puedes ejecutar `export INSTALL_DIR=/custom/installation/path` antes de este comando.

Para comprobar que se instaló correctamente, puedes abrir la ayuda:

```bash
sentry-cli --help
```

<div id="installation-via-npm">
  ## Instalación mediante NPM
</div>

También puedes instalar `sentry-cli` mediante npm para casos de uso específicos; por ejemplo, en servidores de compilación. El paquete se llama `@sentry/cli` y, durante la postinstalación, descargará el binario de la versión correspondiente:

```bash {tabTitle:npm}
npm install @sentry/cli
```

```bash {tabTitle:yarn}
yarn add @sentry/cli
```

```bash {tabTitle:pnpm}
pnpm add @sentry/cli
```

Luego podrás encontrarlo en la carpeta `.bin`:

```bash
./node_modules/.bin/sentry-cli --help
```

Si quieres instalar esto globalmente con npm usando sudo, tendrás que pasar la opción `--unsafe-perm`:

```bash
sudo npm install -g @sentry/cli --unsafe-perm
```

<Alert>
  Sin embargo, no se recomienda esta instalación.
</Alert>

<div id="downloading-from-a-custom-source">
  ### Descargar desde un origen personalizado
</div>

De forma predeterminada, este paquete descargará sentry-cli desde el CDN gestionado por [Fastly](https://www.fastly.com/). Para usar un CDN personalizado, establece la propiedad de configuración de npm `sentrycli_cdnurl`. El descargador añadirá `"/<version>/sentry-cli-<dist>"`.

```bash
npm install @sentry/cli --sentrycli_cdnurl=https://mymirror.local/path
```

O agrega la propiedad en tu archivo `.npmrc` ([https://docs.npmjs.com/files/npmrc](https://docs.npmjs.com/files/npmrc))

```bash
sentrycli_cdnurl=https://mymirror.local/path
```

Otra opción es utilizar la variable de entorno `SENTRYCLI_CDNURL`.

```bash
SENTRYCLI_CDNURL=https://mymirror.local/path npm install @sentry/cli
```

<div id="available-installation-options">
  ### Opciones de instalación disponibles
</div>

Las opciones que se enumeran a continuación controlan cómo se comporta el script de instalación de `sentry-cli` cuando se instala a través de `npm`.

`SENTRYCLI_CDNURL`:

Si se establece, el script usará la URL indicada para obtener el binario. De forma predeterminada es `https://downloads.sentry-cdn.com/sentry-cli`.

`SENTRYCLI_USE_LOCAL`:

Si se establece en `1`, el binario de `sentry-cli` se tomará de tu `$PATH` y se copiará localmente en lugar de descargarse desde servidores externos. Aun así verificará el número de versión, que debe coincidir.

`SENTRYCLI_SKIP_DOWNLOAD`:

Si se establece en `1`, el script omitirá por completo la descarga del binario.

`SENTRYCLI_SKIP_CHECKSUM_VALIDATION`:

Si se establece en `1`, el script omitirá la fase de validación del checksum. Puedes verificar manualmente los checksums visitando la página [Checksums de compilación](/es/cli/installation/#build-checksums).

`SENTRYCLI_NO_PROGRESS_BAR`:

Si se establece en `1`, el script no mostrará las barras de progreso de la descarga. Este es el comportamiento predeterminado en entornos de CI.

`SENTRYCLI_LOG_STREAM`:

Si se establece, el script cambiará dónde escribe su salida. Los valores posibles son `stdout` y `stderr`. De forma predeterminada es `stdout`.

<div id="installation-via-homebrew">
  ## Instalación con Homebrew
</div>

Si usas macOS, puedes instalar `sentry-cli` con Homebrew:

```bash
brew install getsentry/tools/sentry-cli
```

<div id="installation-via-scoop">
  ## Instalación con Scoop
</div>

Si usas Windows, puedes instalar `sentry-cli` con [Scoop](https://scoop.sh):

```powershell
> scoop install sentry-cli
```

<div id="docker-image">
  ## Imagen de Docker
</div>

Para distribuciones y sistemas de CI no compatibles, ofrecemos una imagen de Docker con `sentry-cli` preinstalado. Se recomienda usar la etiqueta `latest`, pero también puedes fijar una versión específica. De forma predeterminada, el comando se ejecuta en el directorio `/work`. Monta allí las carpetas relevantes del proyecto y los artefactos de compilación para que `sentry-cli` pueda analizar los recursos:

```bash
docker pull getsentry/sentry-cli
docker run --rm -v $(pwd):/work getsentry/sentry-cli --help
```

<div id="updating-and-uninstalling">
  ## Actualización y desinstalación
</div>

Puedes usar `sentry-cli update` y `sentry-cli uninstall` para actualizar o desinstalar el binario de `sentry-cli`. Es posible que estos comandos no estén disponibles en ciertas situaciones, generalmente cuando `sentry-cli` se ha instalado con una herramienta como Homebrew o Yarn, ya sea directamente o como dependencia de otro paquete. En esos casos, será necesario usar la misma herramienta para actualizar y desinstalar. Si ves que `sentry-cli update` y `sentry-cli uninstall` no funcionan y no sabes cómo se instaló el paquete, ejecutar `which sentry-cli` a menudo dará una pista sobre qué herramienta usar.

<div id="build-checksums">
  ## Sumas de verificación de compilaciones
</div>

Al descargar un ejecutable desde un servidor remoto, es buena práctica comprobar que lo descargado sea exactamente lo que esperamos. Para asegurarlo, podemos validar su suma de verificación. Una suma de verificación es un valor calculado a partir del contenido de un archivo en forma de hash —en nuestro caso, SHA256— y actúa como una comprobación de integridad de datos, ya que siempre produce la misma salida para una entrada dada.

A continuación encontrarás la tabla de sumas de verificación SHA256 para todos los targets de compilación disponibles que admite nuestro CLI. Para calcular el hash de un archivo descargado, puedes usar la utilidad `sha256sum`, que viene preinstalada en macOS y en la mayoría de las distribuciones de Linux.
