Si estás desarrollando sobre la API de Sentry (es decir, usando [tokens de autenticación](/es/api/auth/)), necesitarás ciertos alcances para acceder a distintos endpoints de la API.

Para configurar los alcances de un [token de integración](/es/organization/integrations/integration-platform/#permissions), selecciónalos en el menú desplegable. Se pueden editar más tarde.

Para configurar los alcances de un token personal, selecciónalos en el menú desplegable al [crear un token personal](https://sentry.io/api/). Estos no se pueden editar después.

![](./img/user-auth-token-create.png)

Si buscas información sobre roles de membresía, visita la documentación de
[membership](/es/organization/membership/).

<div id="organizations">
  ### Organizaciones
</div>

|              |             |
| :----------: | :---------: |
|   **GET**    | `org:read`  |
| **PUT/POST** | `org:write` |
|  **DELETE**  | `org:admin` |

<div id="projects">
  ### Proyectos
</div>

|              |                 |
| :----------: | :-------------: |
|    **GET**   | `project:read`  |
| **PUT/POST** | `project:write` |
|  **DELETE**  | `project:admin` |

<Alert level="warning" title="Nota">
  El alcance `project:releases` te da acceso a los endpoints de lanzamientos tanto de **project**
  como de **organization**. Los endpoints disponibles están en la sección
  [Releases](/es/api/releases/) de la documentación de la API.
</Alert>

<div id="issues-events">
  ### Incidencias y eventos
</div>

|            |               |
| :--------: | :-----------: |
|  **GET**   | `event:read`  |
|  **PUT**   | `event:write` |
| **DELETE** | `event:admin` |

<Alert level="warning" title="Nota">
  Los métodos **PUT/DELETE** solo se aplican a actualizar o eliminar incidencias.
  Los eventos en Sentry son inmutables y solo pueden eliminarse al eliminar la incidencia completa.
</Alert>

<div id="releases">
  ### Versiones
</div>

|                         |                    |
| :---------------------: | :----------------: |
| **GET/PUT/POST/DELETE** | `project:releases` |

<Alert level="warning" title="Nota">
  Ten en cuenta que si usas `sentry-cli` para [gestionar tus versiones](/es/cli/releases/), necesitarás un token que también tenga el permiso `org:read`.
</Alert>
