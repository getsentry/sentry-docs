La paginación en la API se gestiona mediante el estándar del encabezado Link:

```bash
curl -i https://sentry.io/api/0/organizations/acme/projects/1/groups/
```

```http
HTTP/1.0 200 OK
Date: Sat, 14 Feb 2015 18:47:20 GMT
Content-Type: application/json
Content-Language: es
Allow: GET, HEAD, OPTIONS
Link: <https://sentry.io/api/0/organizations/acme/projects/1/groups/?&cursor=1420837590:0:1>;
  rel="previous"; results="false",
  <https://sentry.io/api/0/organizations/acme/projects/1/groups/?&cursor=1420837533:0:0>;
  rel="next"; results="true"
```

Cuando sea compatible, los cursores **siempre** se devolverán tanto para la página anterior como para la siguiente, incluso si no hay resultados en esas páginas. Esto te permite consultar la API en busca de resultados aún no descubiertos. Un ejemplo de uso sería cuando estás implementando un sondeo y quieres ver si hay datos nuevos. Devolvemos un indicador `results="[true|false]"` para determinar si realmente necesitas paginar.


<div id="pagination-example">
  ## Ejemplo de paginación
</div>

Aquí tienes un ejemplo de paginación usando este endpoint de la API:

https://docs.sentry.io/api/events/list-an-issues-events/

La solicitud HTTP en este ejemplo devuelve 100 eventos para la incidencia y contiene el siguiente encabezado Link en la respuesta:

```bash {tabTitle:Http}
<https://sentry.io/api/0/organizations/acme/issues/123456/events/?&cursor=0:0:1>; rel="prev"; results="false"; cursor="0:0:1", <https://sentry.io/api/0/organizations/acme/issues/123456/events/?&cursor=0:100:0>; rel="next"; results="true"; cursor="0:100:0"
```

Una de las URL en la respuesta del enlace tiene `rel=next`, lo que indica la siguiente página de resultados. También tiene `results=true`, lo que significa que hay más resultados.

A partir de esto, la siguiente solicitud es `GET <https://sentry.io/api/0/organizations/acme/issues/123456/events/?&cursor=0:100:0>`.

Esta solicitud devolverá los siguientes 100 eventos de ese issue, nuevamente con el siguiente encabezado de enlace:

```bash {tabTitle:Http}
<https://sentry.io/api/0/organizations/acme/issues/123456/events/?&cursor=0:0:1>; rel="previous"; results="true"; cursor="0:0:1", <https://sentry.io/api/0/organizations/acme/issues/123456/events/?&cursor=0:200:0>; rel="next"; results="true"; cursor="0:200:0"
```

El proceso se repite hasta que la URL con `rel=next` tenga el indicador `results=false`, lo que señala la última página.

Los tres valores del cursor son: identificador del cursor (entero, normalmente 0), desplazamiento de fila y is&#95;prev (1 o 0).
