---
title: Límites de velocidad
sidebar_order: 3
---

Sentry limita cada solicitud a la API para prevenir abusos y el uso excesivo de recursos. El límite se aplica a cada combinación única de emisor y endpoint.

Restringimos tanto la frecuencia con la que se hacen las solicitudes (límite de solicitudes por segundo) como la cantidad de solicitudes simultáneas que puede hacer el emisor (límite de concurrencia).

El límite de solicitudes por segundo sigue un enfoque de ventana fija. Las solicitudes se cuentan en intervalos de tiempo determinados por el tamaño de la ventana. Si la cantidad de solicitudes del emisor supera las permitidas dentro de una ventana, esa solicitud será rechazada. Cada endpoint tiene su propio máximo de solicitudes y tamaño de ventana.

Por su parte, el limitador de concurrencia rechazará solicitudes si el emisor tiene demasiadas solicitudes en curso al mismo tiempo.

Puedes monitorear el uso de tus límites revisando encabezados especiales en la respuesta.

<div id="headers">
  ## Encabezados
</div>

Cada respuesta a una solicitud de API incluye los siguientes encabezados:

* `X-Sentry-Rate-Limit-Limit`
  * El número máximo de solicitudes permitidas dentro de la ventana
* `X-Sentry-Rate-Limit-Remaining`
  * La cantidad de solicitudes que le quedan a este cliente en este endpoint dentro de la ventana actual
* `X-Sentry-Rate-Limit-Reset`
  * El momento en que comienza la siguiente ventana de limitación de tasa y se reinicia el conteo, medido en segundos UTC desde el inicio de la época (epoch)
* `X-Sentry-Rate-Limit-ConcurrentLimit`
  * El número máximo de solicitudes simultáneas permitidas
* `X-Sentry-Rate-Limit-ConcurrentRemaining`
  * La cantidad de solicitudes simultáneas que le quedan a este cliente

<div id="additional-information">
  ## Información adicional
</div>

El limitador de velocidad se basa en la identidad del solicitante, no en el token de portador ni en la cookie. Por lo tanto, no se puede evitar el límite de velocidad generando varios tokens.

<Alert>
  Hacer sondeos a la API para obtener actualizaciones probablemente activará rápidamente la limitación de velocidad. Recomendamos usar nuestros [webhooks](/es/organization/integrations/integration-platform/webhooks/), si es posible.
</Alert>
