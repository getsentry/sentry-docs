<div id="auth-tokens">
  ## Tokens de autenticación
</div>

Los tokens de autenticación se envían mediante un encabezado de autenticación y se usan para identificarte como una cuenta de usuario u organización en la API. En nuestra documentación, verás varios marcadores de posición que aparecen entre llaves o signos de mayor/menor, como `{API_KEY}` o `<auth_token>`, que tendrás que reemplazar por uno de tus tokens de autenticación para poder usar la llamada a la API de forma eficaz.

Por ejemplo, cuando la documentación dice:

```bash
curl -H 'Authorization: Bearer {TOKEN}' https://sentry.io/api/0/organizations/{organization_slug}/projects/
```

Si tu token de autenticación es `1a2b3c` y el slug de tu organización es `acme`, entonces el comando debería ser:

```bash
curl -H 'Authorization: Bearer 1a2b3c' https://sentry.io/api/0/organizations/acme/projects/
```

Puedes crear tokens de autenticación en Sentry [creando una integración interna](/es/organization/integrations/integration-platform/#internal-integrations). Esta opción también está disponible en Sentry autoalojado.

<div id="user-authentication-tokens">
  ### Tokens de autenticación de usuario
</div>

Algunos endpoints de la API requieren un token de autenticación asociado a tu cuenta de usuario, en lugar de un token de una integración interna. Estos tokens pueden crearse en Sentry en la página &quot;User settings&quot; (**User settings &gt; Personal Tokens**) y se les pueden asignar ámbitos específicos.

Los endpoints que requieren un token de autenticación de usuario son específicos de tu usuario, como [Retrieve an Organization](/es/api/organizations/retrieve-an-organization/).

<div id="dsn-authentication">
  ## Autenticación con DSN
</div>

Algunos endpoints de la API pueden permitir la autenticación basada en DSN. Por lo general, su uso es muy limitado y cada endpoint indicará si la admite. Funciona de forma similar a la autenticación con token Bearer, pero utiliza tu DSN (Client Key).

```bash
curl -H 'Authorization: DSN {DSN}' https://sentry.io/api/0/{organization_slug}/{project_slug}/user-reports/
```

<div id="api-keys">
  ## Claves de API
</div>

<Alert level="warning" title="Nota">
  Las claves de API son un método de autenticación legado. Seguirán siendo compatibles, pero están deshabilitadas para las cuentas nuevas. Deberías usar **tokens de autenticación** siempre que sea posible.
</Alert>

Las claves de API se envían mediante autenticación básica HTTP, donde el nombre de usuario es tu clave de API y la contraseña se deja en blanco.

Por ejemplo, para obtener información sobre el proyecto al que está asociada tu clave, podrías hacer una solicitud como esta:

```bash
curl -u {API_KEY}: https://sentry.io/api/0/organizations/{organization_slug}/projects/
```

<Alert>
  Debes **sí o sí** proporcionar un valor para la contraseña; por eso el `:` aparece en nuestro ejemplo.
</Alert>
