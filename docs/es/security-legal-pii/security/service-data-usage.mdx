<div id="delivering-a-better-user-experience">
  ## Ofrecer una mejor experiencia de usuario
</div>

Sentry procesa los datos de tu servicio —es decir, los datos que configuras para que se recopilen y se envíen a tu instancia de Sentry— para poder prestarte nuestro servicio. Sin embargo, a medida que el servicio de Sentry ha evolucionado, los enfoques anteriores basados en heurísticas ya no pueden ofrecer el valor de producto que esperamos. Para entrenar y validar modelos de agrupación, notificaciones y mejoras de flujo de trabajo, Sentry necesitará acceso a datos de servicio adicionales para brindar una mejor experiencia de usuario.

Puedes actualizar estos ajustes en la sección “Service Data Usage” de la [página Legal &amp; Compliance en Sentry](https://sentry.io/orgredirect/organizations/:orgslug/settings/legal/), que se encuentra dentro de la configuración “Usage &amp; Billing”.

<div id="use-of-non-identifying-data">
  ### Uso de datos no identificables
</div>

De acuerdo con nuestros Términos de servicio, Sentry puede usar elementos no identificables de los datos de tu servicio para mejorar el producto. Por ejemplo, podemos agregar datos de Web Vitals para mostrar el rendimiento de tu sitio frente a un benchmark desarrollado por Sentry. Los datos utilizados para el benchmark no pueden vincularse a ningún proyecto o cliente en particular, por lo que no son identificables.

<div id="use-of-aggregated-identifying-data">
  ### Uso de datos identificados agregados
</div>

Con tu autorización, Sentry puede usar ciertos tipos de datos del servicio para mejorar nuestro producto. Estos incluyen:

* Mensajes de error
* Trazas de pila
* Spans
* Interacciones del DOM

<div id="use-of-identifying-data-for-generative-ai-features">
  ## Uso de datos identificables para funciones de IA generativa
</div>

Para funciones de IA generativa como [Seer](/es/product/ai-in-sentry/seer/) o el resumen de incidencias, Sentry puede acceder a los datos de tu servicio para ofrecerte estas funcionalidades, incluido el contenido generado por IA. De forma predeterminada, tus datos no se utilizarán para entrenar ningún modelo de IA generativa sin tu permiso. El contenido generado por IA a partir de tus datos se muestra solo a ti, no a otros clientes. Para más detalles sobre Sentry, la IA y la privacidad, consulta nuestros [Principios de privacidad de IA](/es/product/ai-in-sentry/ai-privacy-and-security/).

<div id="data-access-summary">
  ## Resumen de acceso a los datos
</div>

| **Tipo de acceso**                              | **¿Los datos subyacentes son identificables?** | **¿Con quién se compartirán estos datos (o cualquier resultado)?** | **¿Se usarán estos datos para entrenar modelos de Sentry?** | **¿Se usarán estos datos para entrenar modelos de terceros?** |
| ----------------------------------------------- | ---------------------------------------------- | ------------------------------------------------------------------- | ------------------------------------------------------------ | -------------------------------------------------------------- |
| **Datos no identificables**                     | No                                             | Otros clientes de Sentry*                                          | Sí                                                           | No                                                             |
| **Datos identificables agregados**              | Sí                                             | Subencargados aprobados                                             | No (a menos que se autorice)                                 | No                                                             |
| **Datos identificables para funciones de IA generativa** | Sí                                             | Subencargados aprobados                                             | No (a menos que se autorice)                                 | No                                                             |

**En estos casos no compartimos los datos subyacentes, solo agregaciones o resultados generados a partir de los datos*

<div id="our-data-handling">
  ## Nuestro manejo de datos
</div>

Dónde estamos autorizados a usar datos de servicio para mejorar el producto:

1. Seguiremos animando a todos los clientes a usar nuestras [distintas herramientas de depuración de datos](/es/security-legal-pii/scrubbing/) para que los datos de servicio estén depurados antes de que los recibamos.

2. Aplicaremos las mismas reglas de eliminación y retención a nuestros datos de entrenamiento que aplicamos a los datos de servicio subyacentes. Esto significa que, si eliminas datos de servicio, también se eliminarán automáticamente de nuestros modelos de aprendizaje automático.

3. Depuraremos los datos para eliminar PII antes de que entren en cualquier conjunto de entrenamiento.

Confiamos en que, con estos controles en vigor, podremos usar los datos de servicio para mejorar y ofrecer nuestros productos, al mismo tiempo que protegemos esos datos.
