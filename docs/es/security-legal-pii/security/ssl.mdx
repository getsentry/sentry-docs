<Alert title="Nota" level="warning">

El contenido de esta página se aplica únicamente al producto SaaS de Sentry. **No**
se aplica a implementaciones autogestionadas ni a entornos de inquilino único.

</Alert>

La API, el panel y el envío de eventos de Sentry requieren como mínimo la versión TLS 1.2 para comunicarse de forma segura.

Sentry ofrece una serie de protocolos y cifrados que buscan brindar a nuestros usuarios un equilibrio entre seguridad y compatibilidad. Nuestros servidores negociarán el cifrado hasta la combinación más segura que el cliente pueda admitir, en el orden de preferencia especificado a continuación. El conjunto de cifrados depende del endpoint y de la versión de TLS negociada.

### sentry.io y \*.ingest.sentry.io

Las versiones TLS 1.2 y 1.3 están disponibles tanto para la API y el panel de Sentry, como para el envío de eventos heredado (a través de `sentry.io`) y para los dominios de ingesta de eventos (a través de `*.ingest.sentry.io`).

Cifrados admitidos:

<div id="tls-13">
  #### TLS 1.3
</div>

- TLS_AES_128_GCM_SHA256
- TLS_AES_256_GCM_SHA384
- TLS_CHACHA20_POLY1305_SHA256

<div id="tls-12">
  #### TLS 1.2
</div>

- TLS_ECDHE_RSA_WITH_AES_128_GCM_SHA256
- TLS_ECDHE_RSA_WITH_CHACHA20_POLY1305_SHA256
- TLS_ECDHE_RSA_WITH_AES_256_GCM_SHA384
- TLS_ECDHE_RSA_WITH_AES_128_CBC_SHA
- TLS_ECDHE_RSA_WITH_AES_256_CBC_SHA
- TLS_RSA_WITH_AES_128_GCM_SHA256
- TLS_RSA_WITH_AES_256_GCM_SHA384
- TLS_RSA_WITH_AES_128_CBC_SHA
- TLS_RSA_WITH_AES_256_CBC_SHA
- TLS_RSA_WITH_3DES_EDE_CBC_SHA

Esta suite de protocolos y cifrados representa la compatibilidad oficial de Sentry; no obstante, los clientes pueden negociar cifrados individuales que superen los niveles de seguridad de los mencionados anteriormente.
