<div id="whats-sentry">
  ## ¿Qué es Sentry?
</div>

Sentry es una herramienta de supervisión de software que ayuda a los desarrolladores a identificar y depurar errores y problemas de rendimiento. Desde el trazado distribuido de extremo a extremo hasta el monitoreo del rendimiento, Sentry ofrece observabilidad a nivel de código que facilita diagnosticar problemas y aprender continuamente sobre el estado del código de tu aplicación en distintos sistemas y servicios.

<div id="what-can-sentry-help-with">
  ## ¿Con qué puede ayudar Sentry?
</div>

Como solución de monitoreo de aplicaciones full‑stack, Sentry ofrece las siguientes funciones para ayudarte a mantener tu app funcionando sin inconvenientes y a tus usuarios satisfechos:

<div id="error-monitoring">
  ### Monitoreo de errores
</div>

Las funcionalidades de monitoreo de errores de Sentry te permiten llegar rápidamente a la raíz de un problema para que puedas entender y corregir lo que está fallando.

Sentry captura automáticamente las excepciones no controladas (errores). Luego agrupa errores similares en [issues](/es/product/issues/), lo que permite priorizarlos y gestionarlos de forma más eficiente.

* Si estás monitoreando un proyecto web, puedes [**reproducir**](/es/product/explore/session-replay/) lo que la persona usuaria experimentó en tu app y vincular ese error con la línea de código problemática mediante [**source maps**](/es/product/sentry-basics/integrate-frontend/upload-source-maps/).
* Si configuras Sentry tanto para el frontend como para el backend, podrás usar el [**trazado distribuido**](/es/product/sentry-basics/concepts/tracing/distributed-tracing/) para ver la secuencia de eventos (tanto de frontend como de backend) que llevó al evento de interés.
* Además, puedes configurar [**integraciones**](/es/organization/integrations/source-code-mgmt/) para vincular tu código fuente, de modo que Sentry pueda mostrar [**commits sospechosos**](/es/product/issues/suspect-commits/) y [**stack traces**](/es/organization/integrations/source-code-mgmt/github/#stack-trace-linking) para señalar la línea de código problemática, así como [**sugerir personas asignadas**](/es/organization/integrations/source-code-mgmt/github/#suspect-commits-and-suggested-assignees) a los issues según las autorías de los commits.

<div id="tracing">
  ### Trazas
</div>

Las trazas te ofrecen una visión completa de lo que sucede en tu aplicación, desde el frontend hasta el backend y entre sistemas, al observar las interacciones entre los distintos componentes de tu stack. Esto es especialmente útil si dependes de microservicios o intentas encontrar la causa raíz de una queja de usuario difícil. Con el [trazado distribuido](/es/concepts/key-terms/tracing/distributed-tracing/), puedes identificar rápidamente cuellos de botella de rendimiento.

También puedes obtener el contexto completo de lo que ocurría cuando se produjo un error o un problema de rendimiento consultando la página [Trace View](/es/concepts/key-terms/tracing/trace-view/). Esto te permitirá, entre otras cosas, rastrear páginas de carga lenta hasta llegar a llamadas a la API con bajo rendimiento. Por último, puedes usar la página [Trace Explorer](/es/product/explore/traces/) para Buscar datos de spans, incluidas condiciones relacionadas con la estructura de la propia traza.

<div id="profiling">
  ### Perfilado
</div>

Sentry [**Profiling**](/es/product/profiling/) ofrece información a nivel de código sobre el rendimiento de tu aplicación en producción para que puedas encontrar y corregir más rápido las líneas de código problemáticas. Puedes perfilar el código en cualquier entorno donde se ejecute tu aplicación para ver rápidamente qué funciones están provocando spans lentos.

<div id="performance-monitoring">
  ### Monitoreo de rendimiento
</div>

Nuestra función de monitoreo de rendimiento te ayuda a seguir el desempeño de tu aplicación, medir métricas como el caudal (throughput) y la latencia, y ver el impacto de los errores en múltiples servicios.

* Sentry ofrece información para ayudarte a supervisar el rendimiento de tu app. Esto incluye web vitals, mobile vitals, consultas, solicitudes HTTP y más. Puedes profundizar en muestras de eventos para investigar variaciones en el rendimiento.

<div id="release-health-monitoring">
  ### Monitoreo de la salud de las versiones
</div>

Dar a Sentry visibilidad de tus [**versiones**](/es/product/releases/) te permite detectar en qué momento una versión empieza a degradarse para que puedas actuar rápidamente. Obtienes visibilidad en tiempo real en todas tus versiones, lo que te permite ver métricas clave como sesiones sin fallos, adopción de versiones y tasa de fallos.

Las versiones están integradas con el resto de Sentry, por lo que puedes ver directamente cómo una versión afectó a un error o a un problema de rendimiento, además de poder resolver problemas específicos de la versión.

<div id="ai-agents-monitoring">
  ### Monitoreo de agentes de IA
</div>

Nuestra función de [**Monitoreo de agentes de IA**](/es/product/insights/ai/agents/) te da visibilidad de los flujos de trabajo de tus agentes de IA en el contexto general de tu aplicación. Cuando ejecutes `pip install sentry` en un proyecto que use agentes de IA, Sentry capturará automáticamente métricas útiles como invocaciones de agentes, ejecuciones de herramientas, traspasos entre agentes y uso de tokens, enviándolas a nuestro panel de AI Agents Insights.

<div id="uptime-monitoring">
  ### Monitoreo de Uptime
</div>

El [**Monitoreo de Uptime**](/es/product/uptime-monitoring/) de Sentry te ayuda a mantener la disponibilidad de tus servicios web al monitorear URL relevantes. Realiza un seguimiento continuo de las URL configuradas, enviando alertas e insights para identificar rápidamente caídas y solucionar problemas. Al aprovechar el [trazado distribuido](/es/product/uptime-monitoring/uptime-tracing/), Sentry te permite detectar cualquier error que ocurra durante una verificación de uptime, simplificando el triaje y acelerando el análisis de la causa raíz. El monitoreo de uptime incluye [spans de solicitud de uptime](/es/product/uptime-monitoring/#uptime-request-spans) de forma predeterminada. Estos actúan como la raíz del rastro de cualquier incidencia de uptime, brindándote mejor contexto para depurar más rápido.

<div id="recurring-job-monitoring">
  ### Monitoreo de trabajos recurrentes
</div>

[**Cron Monitors**](/es/product/crons/) te permite monitorear la disponibilidad y el rendimiento de cualquier trabajo programado y recurrente en Sentry. Una vez implementado, podrás recibir alertas y métricas que te ayudarán a resolver errores, detectar timeouts y prevenir interrupciones en tu servicio.

<div id="visibility-into-your-data-across-environments">
  ### Visibilidad de tus datos en todos los entornos
</div>

[**Discover**](/es/product/explore/discover-queries/) ofrece visibilidad de tus datos en todos los entornos al basarse en tus datos de errores y enriquecerlos. Puedes [**hacer consultas**](/es/product/explore/discover-queries/query-builder/) para obtener información sobre el estado de salud de todo tu sistema y respuestas a preguntas clave del negocio, todo en un solo lugar. También puedes crear tus propios [**tableros**](/es/product/dashboards/) para monitorear lo que más importa de tu app.

<div id="code-coverage">
  ### Cobertura de código
</div>

[Codecov](/es/product/codecov/) proporciona **cobertura de pruebas** para desarrolladores. Ayuda a mejorar las pruebas, la cobertura y la confiabilidad del código, y te permite obtener información sobre la cobertura de pruebas directamente en Sentry mientras supervisas tus errores.

<div id="seamless-developer-workflow">
  ### Flujo de trabajo fluido para desarrolladores
</div>

Sentry ofrece muchas [**integraciones**](/es/organization/integrations/) para facilitar un flujo de trabajo fluido para desarrolladores, entre ellas:

* [Gestión de código fuente](/es/organization/integrations/source-code-mgmt/), asignación de código, responsables de código y commits sospechosos con GitHub, GitLab, Azure DevOps y Bitbucket.
* [Seguimiento de incidencias](/es/organization/integrations/issue-tracking/) con Jira, GitHub y muchas otras herramientas.
* [Alertas y notificaciones](/es/organization/integrations/notification-incidents/) con Slack, Discord, PagerDuty y más.

<div id="sdk-support-for-over-100-platforms">
  ### Compatibilidad del SDK con más de 100 plataformas
</div>

Sentry es compatible con más de [100 plataformas y frameworks](/es/platforms/), incluidos más de 30 lenguajes de programación, para que puedas detectar errores y problemas de rendimiento sin importar sobre qué esté construida tu aplicación. Nuestros SDK te cubren las espaldas para que puedas empezar a usar Sentry con solo unas pocas líneas de código.
