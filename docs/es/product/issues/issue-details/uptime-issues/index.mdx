Un problema de disponibilidad es un conjunto de eventos de inactividad detectados para una URL específica. Un evento de inactividad se genera por
alertas activas de disponibilidad cuando las solicitudes HTTP no cumplen con nuestros
[criterios de comprobación de disponibilidad](/es/product/uptime-monitoring/#uptime-check-criteria).

![Detalles del problema de disponibilidad](./img/uptime-issue-details.png)

<div id="traced-errors">
  ## Errores con traza
</div>

Las comprobaciones de disponibilidad realizadas sobre servicios web configurados con uno de los SDK compatibles de Sentry incluyen una
[traza](/es/concepts/key-terms/tracing/) que puede usarse para seguir los errores detectados a raíz de comprobaciones de disponibilidad HTTP fallidas. El navegador de trazas te permite explorar las posibles causas raíz de las caídas y es una herramienta potente para identificar y resolver problemas rápidamente.

<div id="issue-lifecycle">
  ## Ciclo de vida de los issues
</div>

Los problemas de disponibilidad se agrupan por la URL monitorizada y se crean al detectarse la primera interrupción. Sentry resuelve automáticamente un problema de disponibilidad en curso cuando la URL monitorizada vuelve a un estado saludable y cumple nuestros [criterios de verificación de disponibilidad](/es/product/uptime-monitoring/#uptime-check-criteria). Si la URL vuelve a presentar interrupciones, el estado del issue cambiará a regresado.
