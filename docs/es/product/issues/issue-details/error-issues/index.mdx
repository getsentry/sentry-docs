Un *problema de error* es un conjunto de eventos de error. Lo que se considera un error varía según la plataforma, pero, en general, si algo se parece a una excepción, Sentry puede capturarlo como error. Sentry captura automáticamente errores, excepciones no detectadas y rechazos no controlados, además de otros tipos de errores, según la plataforma. Agrupamos eventos de error similares en problemas basándonos en una huella. En los problemas de error, la huella la define principalmente el stack trace del evento.

<div id="subscribe-to-issue-alerts">
  ## Suscribirse a alertas de incidencias
</div>

Para suscribirte a una incidencia de error y recibir alertas al respecto, haz clic en el icono de la campana y luego ajusta las [notificaciones de flujo de trabajo](/es/product/alerts/notifications/#workflow-notifications) relacionadas con la incidencia en **Configuración de usuario &gt; Notificaciones**.

<div id="error-levels">
  ## Niveles de error
</div>

La descripción del evento se muestra justo debajo del título del issue, junto con un ícono que representa el nivel de error del evento:

![Ícono de nivel de error en los detalles del issue](./img/error-level-icon-2.png)

El nivel puede ser:

* Error - naranja
* Información - azul
* Advertencia - amarillo
* Fatal - rojo
* Depuración - gris
* Muestra - morado

En la barra lateral derecha, [sentry.io](https://sentry.io) muestra un resumen que incluye información como la frecuencia con la que el <SandboxLink scenario="oneIssue" projectSlug="react">issue</SandboxLink> de error ha ocurrido en las últimas 24 horas y en los últimos 30 días, así como la última y la primera vez que se detectó el issue. Si el issue está vinculado a algún issue de GitHub o Jira, también se muestra aquí. Además, puedes usar esta sección para vincular issues existentes de GitHub o Jira. Obtén más información sobre cómo vincular issues en [Integrations](/es/organization/integrations/). Por último, hay un mapa de facetas que muestra la distribución de valores de las etiquetas para todos los eventos incluidos en el issue de error. Todos estos valores se basan en el entorno que hayas seleccionado en el menú desplegable de entornos.

<div id="event-grouping-information">
  ## Información de agrupación de eventos
</div>

![Event Grouping](./img/event-grouping.png)

La sección &quot;Información de agrupación de eventos&quot; ofrece detalles sobre cómo Sentry generó la huella del evento para agruparlo. Puedes ver qué partes del stack trace contribuyeron a la huella, así como todos los valores disponibles. También puedes usar el menú desplegable &quot;Grouping Config&quot; para probar otros algoritmos de agrupación.
