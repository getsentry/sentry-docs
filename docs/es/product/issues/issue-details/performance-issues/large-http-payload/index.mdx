Los problemas de carga útil HTTP grande se generan cuando un span de HTTP tiene un tamaño de cuerpo codificado que supera un umbral.

<div id="detection-criteria">
  ## Criterios de detección
</div>

El detector de este problema de rendimiento busca un único span HTTP que cumpla los siguientes criterios:

- La URL HTTP no debe comenzar con \_next/static/ o \_next/data/
- La URL HTTP no debe terminar con una extensión (por ejemplo, .js, .css, .png, .jpg, .jpeg, .mp3), a menos que la extensión sea .json
- El span HTTP debe tener un `http.response_content_length` (agregado por el SDK `@sentry/browser`, versión `7.53.0`) que supere `300kb`
- La duración del span HTTP debe superar los 100 ms

Si Sentry no está detectando un problema de Large HTTP Payload donde esperabas uno, probablemente sea porque la transacción no cumplió alguno de los criterios anteriores.

Puedes configurar los umbrales del detector para problemas de Large HTTP Payload en **Project Settings > Performance**:

![Configuración del umbral del detector de Large HTTP Payload](./img/large-http-payload-detector-settings.png)

<div id="span-evidence">
  ## Evidencia de span
</div>

Puedes encontrar información adicional sobre tu problema de Large HTTP Payload a partir de cuatro aspectos principales en la sección "Span Evidence":

- **Transaction name**
- **Large HTTP Payload Span:** El span HTTP grande, que generalmente contiene la URL del recurso.
- **Payload Size:** El tamaño del payload en bytes.

![Evidencia de span de Large HTTP payload](./img/span-evidence.png)

Consúltalo yendo a la página de **Issues** en Sentry, seleccionando tu proyecto, haciendo clic en el issue de Large HTTP Payload que quieres revisar y desplazándote hacia abajo hasta la sección "Span Evidence" en la pestaña "Details".

<div id="fixing-large-http-payload-issues">
  ## Solución de problemas con payloads HTTP grandes
</div>

Las razones para recibir problemas consecutivos de payloads HTTP grandes varían, pero aquí tienes algunas sugerencias para ayudarte a resolverlos:

- Si tienes control sobre el backend, verifica si estás enviando más datos de los necesarios o si los datos pueden procesarse más antes de enviarlos al cliente.
- Si toda la información del endpoint es necesaria, considera métodos como la paginación para dividir los datos entre vistas.
- Revisa si se hicieron cambios en tu serializador que hayan provocado un tamaño del cuerpo mayor de lo necesario.
