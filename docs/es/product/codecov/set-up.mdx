Si usas tanto Codecov como Sentry, puedes habilitar la integración de Codecov con un clic en la página **Organization Settings** de la app de Sentry. Podrás ver si tienes cobertura de pruebas parcial o completa para la línea de código que está causando un error, directamente en el stack trace en Sentry.

<div id="prerequisites">
  ## Requisitos previos
</div>

Hay algunos requisitos previos que deben cumplirse para que Codecov funcione en Sentry. Debes tener:

* Permisos de nivel propietario o administrador en Sentry.io.

* La [integración de GitHub](/es/organization/integrations/source-code-mgmt/github/) instalada en Sentry con los [mapeos de código](/es/organization/integrations/source-code-mgmt/github/#stack-trace-linking) habilitados.

  <Alert>
    Si tienes la integración de GitHub configurada correctamente, tus trazas de pila
    mostrarán un aviso de “Abrir esta línea en GitHub” que te llevará al código de tu aplicación.
  </Alert>

* Una cuenta de [Codecov](https://about.codecov.io/) con la [cobertura de código fuente](https://about.codecov.io/product/feature/source-code-coverage/) habilitada.

<div id="enable-codecov-insights">
  ## Habilitar Codecov Insights
</div>

Una vez cumplidos los requisitos previos, puedes habilitar Codecov yendo a la página de **Organization Settings** en Sentry y activando “Enable Code Coverage Insights”.

![Codecov in Org Settings](./img/codecov-setup-in-new-ui.png)

<Alert>
  Sentry comprueba automáticamente cada 24 horas si tu organización de GitHub está en Codecov. Si Codecov no está habilitado y vemos que cumples todos los requisitos previos mencionados arriba, lo activaremos por ti.
</Alert>
