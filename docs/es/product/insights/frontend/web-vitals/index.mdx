Los Web Vitals son un conjunto de métricas clave que miden cómo los usuarios realmente experimentan tu sitio; incluyen la velocidad de carga, la capacidad de respuesta y la estabilidad visual. [Obtén más información sobre estas métricas en Web Vitals Concepts](/es/product/insights/web-vitals/web-vitals-concepts/).

Para tus aplicaciones web instrumentadas, Sentry recopila Web Vitals del tráfico real de usuarios en [navegadores compatibles](web-vitals-concepts/#browser-support). Estas métricas se usan para calcular un [puntaje de rendimiento](#performance-score) de tu aplicación web. Con estas métricas, destacamos las páginas que tienen mayor [oportunidad](#opportunity) de mejorar el rendimiento general de tu app.

La demo interactiva a continuación muestra cómo obtener información sobre tus Web Vitals.

<div id="web-vitals-page">
  ## Página de Web Vitals
</div>

La página de **Web Vitals**, ubicada en la pestaña **Insights**, te ofrece una visión general del rendimiento de carga e interacción de tu aplicación para los proyectos seleccionados. Es un punto de partida para investigar los Web Vitals deficientes que afectan tu app y profundizar para entender mejor qué páginas impactan más tu rendimiento web.

En la parte superior izquierda, el anillo de [Performance Score](#performance-score) muestra la calificación general de rendimiento de tu aplicación. Cada componente del anillo representa un Web Vital individual y su peso e impacto relativos en el Performance Score. El gráfico de áreas a la derecha muestra un desglose —por Web Vitals— de tu Performance Score a lo largo del tiempo, lo que facilita detectar qué métricas están mejorando o empeorando.

Debajo de esto, puedes ver los valores p75 (percentil 75) de tu app y las puntuaciones individuales para cada Web Vital. Estos destacan cuáles requieren más atención. Al hacer clic en cualquier Web Vital, se abre un resumen más detallado de esa métrica y se muestran las páginas con mayor [opportunity](#opportunity) de mejora.

Al final de la página, una tabla ordenable enumera todas las páginas de tu app, junto con sus valores p75 asociados para cada Web Vital y sus Performance Scores individuales. La columna Opportunity muestra la mejora potencial de la puntuación de una página en el Performance Score general de tu aplicación si se maximiza a 100. La barra de búsqueda sobre esta tabla te permite filtrar páginas específicas por nombre de ruta.

<Alert>
  La página de **Web Vitals** solo muestra datos de las transacciones e interacciones de carga inicial de tu aplicación. Las navegaciones **no** están incluidas.
</Alert>

<div id="page-summary">
  ## Resumen de la página
</div>

Desde la tabla en **Web Vitals** o desde un panel de resumen de Web Vitals, haz clic en una página para abrir su **Resumen de la página**. Aquí puedes ver un resumen de los Web Vitals de esa página. En **Resumen de la página**, puedes profundizar en una carga de página específica: [Trace](/es/concepts/key-terms/tracing/#whats-a-trace), [Elemento LCP](/es/product/insights/frontend/web-vitals/web-vitals-concepts/#largest-contentful-paint-lcp), [Replay](/es/product/explore/session-replay/) o [Profile](/es/product/explore/profiling/). La barra lateral derecha de la página muestra un gráfico circular que destaca la puntuación de rendimiento de la página desglosada por métrica, así como un gráfico de “Cargas de página”.

<div id="performance-score">
  ## Puntuación de rendimiento
</div>

La **Puntuación de rendimiento** es una calificación de 0 a 100 que resume el rendimiento percibido de tu app web, basada en los **Web Vitals**. Cada **Web Vital** recibe una puntuación de 0 a 100 mediante una [distribución lognormal](https://www.desmos.com/calculator/o98tbeyt1t), que luego se pondera y se agrega por página y de forma global para toda tu aplicación web.

Las **cargas de página** pueden incluir hasta 4 componentes individuales de **Web Vitals** en cada evento, mientras que las interacciones contribuyen a **INP** (Interaction to Next Paint). Los pesos predeterminados y los umbrales de puntuación de cada **Web Vital** se muestran en las tablas siguientes.

<div id="opportunity">
  ## Oportunidad
</div>

Las puntuaciones de **Oportunidad** son valores numéricos asociados a cada página y tienen como objetivo darte una idea de qué páginas conviene mejorar. La puntuación de Oportunidad es el incremento máximo posible en la Puntuación de Rendimiento global de tu aplicación si elevas la puntuación de una página a 100. Si una página de tu aplicación ya tiene una puntuación de 100, la puntuación de Oportunidad sería 0, ya que no hay forma de optimizarla más.

El cálculo de la puntuación de Oportunidad también se pondera según el tráfico que recibe una página. Las páginas con mucho tráfico tienden a tener puntuaciones de Oportunidad más altas. Por ejemplo, una página con una puntuación de 70 que registra 1,000 cargas diarias tendrá una puntuación de Oportunidad más alta que una con una puntuación de 50 que registra solo 10 cargas diarias.

<div id="prerequisites-and-limitations">
  ### Requisitos previos y limitaciones
</div>

**Web Vitals** y **Performance Scores** se capturan automáticamente con los SDK de JavaScript de Sentry mediante la integración [BrowserTracing](/es/platforms/javascript/configuration/integrations/browsertracing/).

Puedes ver cómo configurar Sentry en tu aplicación web en la [documentación de instalación](/es/platforms/javascript/). Si usas un framework de JavaScript, puede que necesites seguir instrucciones adicionales específicas para tu framework:

* [React SDK](/es/platforms/javascript/guides/react/)
* [Vue SDK](/es/platforms/javascript/guides/vue/)
* [Angular SDK](/es/platforms/javascript/guides/angular/)
* [Svelte SDK](/es/platforms/javascript/guides/svelte/)

Aunque Sentry ofrece instrumentación automática para muchos frameworks de frontend, hay entornos y casos que pueden no estar totalmente compatibles. Si no encuentras tus páginas en la sección de **Web Vitals**, verifica que las cargas de página cumplan estas condiciones:

* **Performance Scores** solo son compatibles con los navegadores de escritorio y móviles **Chrome**, **Firefox**, **Safari**, **Opera** y **Edge**.
* Algunas cargas de página pueden no capturar todas las Web Vitals admitidas por Sentry. Esto puede depender del navegador del cliente o del comportamiento específico de tu página.
  * Consulta la lista completa de Web Vitals en la tabla de [compatibilidad del navegador](/es/product/insights/web-vitals/web-vitals-concepts/#browser-support).
