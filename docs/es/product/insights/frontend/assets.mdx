Si tienes habilitado el monitoreo de rendimiento para tu frontend, puedes ver cómo están funcionando tus recursos del navegador en Sentry.

Comienza en la página de [**Recursos**](#assets-page) para obtener una visión general de alto nivel de cómo van tus recursos. Desde ahí, puedes profundizar en la página de [**Resumen del recurso**](#asset-summary-page) de un recurso específico y luego investigar eventos de muestra en la [Lista de muestras](#sample-list) para entender mejor el contexto de su rendimiento en una página concreta.

<div id="prerequisites-and-limitations">
  ## Requisitos previos y limitaciones
</div>

Las páginas de recursos solo están disponibles para proyectos de JavaScript de frontend con el monitoreo de rendimiento habilitado. Actualmente, solo se admiten JavaScript, CSS, imágenes y ciertos recursos iniciados por CSS (como las fuentes).

Para obtener la mejor experiencia, recomendamos habilitar la instrumentación automática mediante la integración `BrowserTracing` en tu proyecto de frontend para ver los datos de rendimiento de los recursos. Esto es compatible con las siguientes plataformas de JavaScript:

* [JavaScript puro](/es/platforms/javascript/performance/)
* [Angular](/es/platforms/javascript/guides/angular/performance/)
* [Astro](/es/platforms/javascript/guides/astro/performance/)
* [Ember](/es/platforms/javascript/guides/ember/performance/)
* [Gatsby](/es/platforms/javascript/guides/gatsby/performance/)
* [Next.js](/es/platforms/javascript/guides/nextjs/performance/)
* [React](/es/platforms/javascript/guides/react/performance/)
* [Remix](/es/platforms/javascript/guides/remix/performance/)
* [Svelte](/es/platforms/javascript/guides/svelte/performance/)
* [SvelteKit](/es/platforms/javascript/guides/sveltekit/performance/)
* [Vue](/es/platforms/javascript/guides/vue/performance/)

<div id="span-eligibility">
  ### Elegibilidad de spans
</div>

Sentry intenta extraer métricas analizando spans relacionados con recursos (assets).

El SDK de JavaScript genera automáticamente spans de tipo `asset` en transacciones de `pageload` y `navigation` usando la [API Resource Timing](https://developer.mozilla.org/en-US/docs/Web/API/Performance_API/Resource_timing) del navegador.

Si usas la <PlatformLink to="/tracing/instrumentation/automatic-instrumentation">instrumentación automática</PlatformLink>, el monitoreo de recursos funcionará sin ninguna configuración.

Si has instrumentado Sentry manualmente, asegúrate de que tus spans cumplan nuestros estándares para obtener la mejor experiencia:

* El campo `op` del span tiene el prefijo `resource` (por ejemplo, `resource.script` o `resource.css`).
* La `description` del span contiene la URL del recurso cargado, que debería corresponder al [campo name de PerformanceResourceTiming](https://developer.mozilla.org/en-US/docs/Web/API/PerformanceEntry/name).
* El valor de datos del span `http.response_transfer_size` se establece en el [tamaño total de la transferencia](https://developer.mozilla.org/en-US/docs/Web/API/PerformanceResourceTiming/transferSize) del recurso.
* El valor de datos del span `http.response_content_length` se establece en el [tamaño codificado del cuerpo](https://developer.mozilla.org/en-US/docs/Web/API/PerformanceResourceTiming/encodedBodySize) del recurso.
* El valor de datos del span `http.decoded_response_content_length` se establece en el [tamaño decodificado del cuerpo](https://developer.mozilla.org/en-US/docs/Web/API/PerformanceResourceTiming/decodedBodySize) del recurso.
* El valor de datos del span `resource.render_blocking_status` se establece en el [estado de bloqueo de renderizado](https://developer.mozilla.org/en-US/docs/Web/API/PerformanceResourceTiming/renderBlockingStatus) del recurso.

Aquí tienes un ejemplo de cómo crear manualmente un span de recurso con el SDK.

```javascript
// https://developer.mozilla.org/en-US/docs/Web/API/PerformanceResourceTiming
const resources = performance.getEntriesByType("resource");
resources.forEach((entry) => {
  const startTime = msToSec(entry.startTime);
  const duration = msToSec(entry.duration);

  const span = Sentry.startInactiveSpan({
    startTimestamp: startTime,
    op: entry.initiatorType
      ? `resource.${entry.initiatorType}`
      : "resource.other",
    description: entry.name,
    data: {
      "http.response_transfer_size": entry.transferSize,
      "http.response_content_length": entry.encodedBodySize,
      "http.decoded_response_content_length": entry.decodedBodySize,
      "resource.render_blocking_status": entry.renderBlockingStatus,
    },
  });

  // sobrescribir la marca de tiempo de finalización para que coincida con la entrada de rendimiento
  span.finish(startTime + duration);
});
```

<div id="assets-page">
  ## Página de Assets
</div>

La página de **Assets** te ofrece una visión rápida del rendimiento de los recursos de tu aplicación para los proyectos seleccionados. Puedes usar esta página como punto de partida para investigar posibles recursos problemáticos y profundizar para entender mejor cómo distintos recursos están afectando el rendimiento de tu app.

Abre la página de **Assets** haciendo clic en &quot;Assets&quot; en la barra lateral, bajo &quot;Insights&quot;. Como alternativa, la página de **Performance** también muestra los recursos de mayor impacto en el widget &quot;Most Time Consuming Assets&quot;. Desde ahí, puedes hacer clic en &quot;View All&quot; para abrir la página de **Assets**.

En la parte superior de la página, los gráficos de resumen de solicitudes por minuto (throughput) y duración promedio ofrecen una visión general del rendimiento de tus recursos. Si ves una anomalía o quieres investigar más un intervalo de tiempo, puedes hacer clic y arrastrar para seleccionar un rango directamente en un gráfico y filtrar los datos para ese periodo.

La tabla de recursos a continuación muestra una lista de recursos agrupados, junto con su tipo, su volumen (solicitudes por min), duración promedio, el tiempo total que tu app pasó cargando ese recurso (time spent) y el [tamaño codificado promedio](https://developer.mozilla.org/en-US/docs/Web/API/PerformanceResourceTiming/encodedBodySize) del recurso.

De forma predeterminada, la tabla de recursos se ordena por el mayor time spent, lo cual sirve como un indicador del impacto relativo en el rendimiento de un recurso determinado. El time spent de un recurso es la suma de todas sus duraciones en un periodo de tiempo dado o, dicho de otro modo, el producto de su duración promedio y de las solicitudes por minuto. Esto significa que los recursos en la parte superior suelen cargarse muy lentamente, muy frecuentemente o ambas cosas.

Puedes hacer clic en el encabezado de una columna para cambiar cómo se ordena la tabla. Ordena por solicitudes por minuto para ver los recursos que se cargan con mayor frecuencia o por duración promedio para ver los recursos que se cargan más lentamente.

También puedes filtrar los recursos por dominio, tipo, página en la que se encuentra y si bloquea el renderizado.

Para ver más detalles, haz clic en un recurso de la tabla para abrir su página de **Asset Summary**.

<div id="what-does-a-render-blocking-mean">
  ### ¿Qué significa el bloqueo de renderizado?
</div>

Un recurso de bloqueo de renderizado es aquel que impide que el navegador renderice cualquier contenido en la pantalla hasta que el recurso se haya descargado y procesado por completo.

Un ejemplo de esto es un `<script/>` dentro del `<head>` de un documento HTML. Al cargarse, el navegador intentará cargar este script por completo antes de renderizar el contenido, ya que asume que podría necesitar algo de ese script para renderizar. Si no es el caso, puedes agregar el atributo `defer` o `async` al script para reducir o eliminar el bloqueo de renderizado.

Sentry captura el estado de bloqueo de renderizado de un recurso mediante la propiedad `resource.render_blocking_status` en la [API PerformanceResourceTiming](https://developer.mozilla.org/en-US/docs/Web/API/PerformanceResourceTiming/renderBlockingStatus).

<div id="asset-parameterization">
  ### Parametrización de recursos
</div>

Para permitir que Sentry agrupe recursos similares, Sentry parametriza las URL de los recursos y elimina elementos potencialmente dinámicos. Esto ayuda a seguir el rendimiento de un recurso en particular a lo largo de diferentes lanzamientos, incluso cuando tienen segmentos dinámicos (usados para invalidar cachés o CDNs).

Si deseas mejorar aún más tus agrupaciones, considera las siguientes reglas que usamos al parametrizar URL. Estas reglas pueden ayudarte a entender cómo nombrar las URL para mejorar la agrupación.

Los siguientes tokens se reemplazarán por * dentro de una URL de recurso:

1. Una cadena de versión (`myfile.v3.0.js` se reemplaza por `myfile.*.js`)
2. Cadenas hexadecimales con más de 5 dígitos (`myfile.7A9B3E.js` se reemplaza por `myfile.*.js`)
3. UUID
4. IDs enteros con más de un dígito (`1234.7A9B3E.js` se reemplaza por `*.*.js`)
5. Cadenas de más de 25 caracteres
6. Toda la ruta, excepto segmentos comunes como static, chunks, media, etc.

<Alert>
  La parametrización de recursos aún está en desarrollo. A medida que se realicen estas mejoras, verás temporalmente instancias de las agrupaciones nuevas y antiguas en tu módulo de recursos.

  Envíanos tus comentarios a través de GitHub Issues.
</Alert>

Para ver un ejemplo de una URL de recurso de un grupo, pasa el cursor sobre una URL en la tabla de recursos.

<div id="asset-summary-page">
  ## Página de resumen del recurso
</div>

Para abrir la página de **Resumen del recurso** de un recurso, haz clic en el recurso en la tabla de la página **Recursos**.

En la parte superior de la página se muestran el tamaño codificado promedio, el tamaño decodificado promedio, el tamaño de transferencia promedio, la duración promedio y las solicitudes por minuto para el rango de tiempo seleccionado. Justo debajo, puedes ver gráficos de resumen de las solicitudes por minuto, la duración promedio y el tamaño promedio del recurso a lo largo del tiempo.

En la parte inferior encontrarás una lista de las páginas en las que aparece el recurso, ordenadas por las solicitudes por minuto del recurso en esa página.

Si quieres investigar una página específica, haz clic en ella para abrir una barra lateral con algunos eventos de muestra.

<div id="image-samples">
  ### Muestras de imágenes
</div>

Al ver un recurso de imagen, la página de resumen del recurso también mostrará las 5 muestras de imagen más grandes. Sentry determina que un tramo es una imagen si tiene una de las siguientes extensiones: &#39;jpg&#39;, &#39;jpeg&#39;, &#39;png&#39;, &#39;gif&#39;, &#39;svg&#39;, &#39;webp&#39;, &#39;apng&#39;, &#39;avif&#39;.

Si no hay información de tamaño disponible, las muestras no se ordenarán de ninguna manera en particular. Si las imágenes son de acceso público y la configuración del proyecto &#39;Enable Images&#39; está habilitada, entonces las imágenes reales se mostrarán en la interfaz.

<div id="sample-list">
  ## Lista de muestras
</div>

Haz clic en una página para abrir su lista de muestras. Este panel lateral muestra métricas resumidas (solicitudes por minuto, duración promedio y tiempo dedicado) del recurso en esa página.

La tabla muestra una lista de eventos de muestra, su duración de span y la diferencia de duración respecto al promedio. Esta tabla está ordenada por la mayor duración de span.

Sentry encuentra automáticamente una variedad de muestras para ayudarte a investigar problemas de rendimiento. Los eventos seleccionados cubrirán todo el intervalo de tiempo elegido, así como un rango de duraciones (más rápidas, más lentas y cercanas a la duración promedio). Puedes hacerte una idea de las características de rendimiento comparando distintos eventos de muestra.

Estos mismos eventos también se representan como triángulos en el gráfico de duración promedio de arriba.

Puedes generar una nueva lista aleatoria de eventos de muestra haciendo clic en el botón &quot;Try Different Samples&quot; al final.

Desde la lista de muestras, puedes profundizar en ejemplos buenos, promedio o malos de un recurso determinado dentro de una página específica. Haz clic en un ID de evento para ver en detalle el span del recurso dentro de la cascada de spans de la página [Trace View](/es/concepts/key-terms/tracing/trace-view/#product-walkthrough-trace-details-page).
