Con User Feedback, puedes recopilar comentarios de tus usuarios mediante tres métodos:

1. Un widget de comentarios que puede mostrarse en cualquier parte de tu aplicación (varios SDK y script de carga)
2. Una ventana modal de informe de fallos que aparece automáticamente cuando tus usuarios experimentan un fallo (script HTML)
3. Una API que puedes integrar en tu propia interfaz (varios SDK y API web)

Para obtener instrucciones sobre cómo configurarlo y empezar a recopilar comentarios, haz clic en uno de los SDK compatibles a continuación.

<div id="supported-sdks-for-user-feedback-widget">
  ## SDKs compatibles con el widget de comentarios de usuarios
</div>

El widget de comentarios de usuarios puede usarse en todas las aplicaciones basadas en el navegador. Esto incluye sitios web estáticos, aplicaciones de una sola página y también aplicaciones con renderizado del lado del servidor, incluidos frameworks como:
[Django](/es/platforms/python/integrations/django/),
[Spring](/es/platforms/java/guides/spring-boot/),
[ASP.NET](/es/platforms/dotnet/guides/aspnetcore/),
[Laravel](/es/platforms/php/guides/laravel/),
[Express](/es/platforms/javascript/guides/express/) y
[Rails](/es/platforms/ruby/guides/rails/).
Si no usas `npm` o `yarn`, puedes usar [nuestra etiqueta `script` del Loader](/es/platforms/javascript/install/loader/) en tu plantilla HTML principal.

El SDK de Sentry que inyecta el widget de comentarios se ejecuta en el navegador del cliente y viene integrado en `@sentry/browser` y en nuestros SDK de frameworks para navegador:

* [JavaScript puro](/es/platforms/javascript/user-feedback/)
* [Angular](/es/platforms/javascript/guides/angular/user-feedback/)
* [Astro](/es/platforms/javascript/guides/astro/user-feedback/)
* [Capacitor](/es/platforms/javascript/guides/capacitor/user-feedback/)
* [Electron](/es/platforms/javascript/guides/electron/user-feedback/)
* [Ember](/es/platforms/javascript/guides/ember/user-feedback/)
* [Gatsby](/es/platforms/javascript/guides/gatsby/user-feedback/)
* [Next.js](/es/platforms/javascript/guides/nextjs/user-feedback/)
* [React](/es/platforms/javascript/guides/react/user-feedback/)
* [Remix](/es/platforms/javascript/guides/remix/user-feedback/)
* [Svelte](/es/platforms/javascript/guides/svelte/user-feedback/)
* [SvelteKit](/es/platforms/javascript/guides/sveltekit/user-feedback/)
* [Vue](/es/platforms/javascript/guides/vue/user-feedback/)

<div id="crash-report-modal">
  ## Modal de informe de fallos
</div>

El modal de informe de fallos permite a los usuarios enviar comentarios después de encontrarse con un error, mediante una ventana emergente automática que aparece tras producirse el error. Esto se implementa añadiendo una etiqueta script a tu plantilla HTML.

* [JavaScript](/es/platforms/javascript/user-feedback/#crash-report-modal)
* [PHP](/es/platforms/php/user-feedback/#crash-report-modal)

<div id="supported-sdks-for-user-feedback-api">
  ## SDK compatibles con la API de comentarios de usuario
</div>

También ofrecemos una API para enviar comentarios de usuario de forma programática. Esto te permite conectar tu propia interfaz de usuario a Sentry. Si tu SDK no es compatible, puedes enviar datos a la [API web](/es/api/projects/submit-user-feedback/).

Los siguientes SDK son compatibles:

* [Android](/es/platforms/android/user-feedback/#user-feedback-api)
* [Apple](/es/platforms/apple/user-feedback/#user-feedback-api)
* [Dart](/es/platforms/dart/user-feedback/#user-feedback-api)
* [.NET](/es/platforms/dotnet/user-feedback/#use-the-net-sdk)
* [Flutter](/es/platforms/dart/guides/flutter/user-feedback/#user-feedback-api)
* [Java](/es/platforms/java/user-feedback/#user-feedback-api)
* [Kotlin](/es/platforms/kotlin/guides/kotlin-multiplatform/user-feedback/#user-feedback-api)
* [Node.js](/es/platforms/javascript/guides/node/user-feedback/#user-feedback-api)
* [React Native](/es/platforms/react-native/user-feedback/#user-feedback-api)
* [Unity](/es/platforms/unity/user-feedback/#use-the-net-sdk)
* [Unreal](/es/platforms/unreal/user-feedback/#user-feedback-api)
