Sentry detecta automáticamente los errores que arroja tu aplicación, como problemas de rendimiento y de experiencia de usuario, por ejemplo, rage clicks. Pero hay otras frustraciones que tus usuarios pueden encontrar (flujos de permisos interrumpidos, enlaces rotos, errores tipográficos, UX confusa, fallos en la lógica de negocio, etc.).

Ahí es donde entran las funciones User Feedback Widget y Crash-Report Modal de Sentry. Configurar una o ambas te permite recopilar comentarios de tus usuarios finales cuando se topan con errores en tu aplicación. Sentry luego vincula esos comentarios con un contexto de depuración enriquecido disponible, incluidos errores, reproducciones y etiquetas como la URL, lo que te permite llegar a la raíz del problema más rápido.

Por ejemplo, usar [Session Replay](/es/product/explore/session-replay/) junto con el User Feedback Widget puede ahorrarte tiempo, ayudándote a entender rápidamente el contexto del problema del usuario al ver exactamente lo que experimentó antes de enviar sus comentarios.

<div id="user-feedback-widget">
  ## Widget de comentarios de usuarios
</div>

El widget de comentarios de usuarios es un componente persistente y listo para usar que puedes agregar en cualquier parte de tu aplicación web. Permite a los usuarios enviar comentarios de forma rápida y sencilla cada vez que encuentren algo que no funciona como se espera. Obtén más información sobre cómo instalarlo y personalizarlo [aquí](/es/product/user-feedback/setup/) y consulta las prácticas recomendadas [aquí](/es/product/sentry-basics/user-feedback-basics/).

![Widget de comentarios de usuarios.](./img/user-feedback-widget.png)

Los comentarios recopilados mediante el widget de comentarios pueden incluir parte o toda la siguiente información (según lo que marques como obligatorio y lo que esté disponible en el contexto del error):

* Descripción del usuario
* Captura de pantalla del usuario
* Correo electrónico del usuario
* Evento de Replay asociado (con hasta 60 segundos de grabación de Replay antes de que el usuario enviara el comentario)
  * Obtén más información sobre cómo configurar Session Replay [aquí](/es/product/explore/session-replay/getting-started/).
* URL de la página en la que estaba el usuario cuando envió el comentario
* Etiquetas

<div id="crash-report-modal">
  ## Modal de informe de fallos
</div>

Otra forma de recopilar comentarios es instalando el modal de informe de fallos. Esto permite a los usuarios enviar comentarios después de encontrar un error mediante un modal automático que aparece cuando se produce el error. Obtén más información sobre cómo instalarlo y personalizarlo [aquí](/es/platforms/javascript/user-feedback/#crash-report-modal).

![Modal de informe de fallos.](./img/crash-report-user-feedback.png)

Los comentarios recopilados mediante el modal de informe de fallos incluirán la siguiente información:

* Descripción del usuario
* Vista previa de la incidencia de Sentry y enlace a la página de detalles de la incidencia
* URL de la página en la que estaba el usuario cuando envió el comentario
* Etiquetas

<div id="user-feedback-api">
  ## API de comentarios de usuarios
</div>

También ofrecemos una API para enviar comentarios de usuarios de forma programática. Esto te permite conectar tu propia interfaz de usuario con Sentry. Para más detalles, consulta la [guía de configuración](/es/product/user-feedback/setup/#supported-sdks-for-user-feedback-api).

<div id="where-to-view-user-feedback">
  ## Dónde ver los comentarios de los usuarios
</div>

Ya sea que recopiles comentarios mediante el Feedback Widget, el Crash-Report Modal o ambos, podrás verlos en la página **User Feedback**. La página está organizada cronológicamente, con los comentarios más recientes y sin resolver en la parte superior. Puedes hacer clic en cada comentario para ver más detalles en una sola vista, similar a tu bandeja de entrada de correo. En el lado derecho se muestra la vista **User Feedback Details**, que incluye la descripción del usuario y cualquier contexto de depuración disponible.

Puedes acotar los resultados de la lista de comentarios usando los filtros de proyecto, entorno y rango de fechas.

![Lista de User Feedback.](./img/user-feedback-list.png)

<div id="ai-powered-user-feedback-summaries">
  ## Resúmenes de comentarios de usuarios impulsados por IA
</div>

Sentry ahora ofrece **resúmenes de comentarios de usuarios impulsados por IA** que analizan automáticamente tus comentarios para brindarte información accionable de un vistazo. Este resumen te permite comprender grandes volúmenes de comentarios sin tener que clasificar cada uno manualmente, convirtiendo una cantidad abrumadora en información útil y ayudándote a priorizar mejoras en la experiencia del usuario.

<div id="how-it-works">
  ### Cómo funciona
</div>

En la parte superior del índice de User Feedback, verás un resumen que destaca los sentimientos más comunes de tus usuarios en los proyectos y el rango de fechas que hayas seleccionado. El resumen ofrece una visión general concisa de lo que los usuarios opinan sobre tu producto. También incluye categorías generadas por IA en las que puedes hacer clic para filtrar los comentarios. La lista filtrada mostrará comentarios relevantes según el contexto; por ejemplo, sobre el “rendimiento” de tu app.

<div id="triaging-user-feedback-submissions">
  ## Clasificación de envíos de comentarios de usuarios
</div>

Puedes resolver manualmente los envíos de comentarios de usuarios en la vista **User Feedback Details** haciendo clic en el Button “Resolve” en la esquina superior derecha, o resolver varios a la vez seleccionando múltiples comentarios:

![Clasificación de la lista de comentarios de usuarios.](./img/user-feedback-triaging.png)

También puedes asignar a un miembro del equipo a un envío específico de comentarios de usuarios en la vista **User Feedback Details**. Si encuentras comentarios que consideras spam, selecciona los comentarios y haz clic en “Mark as Spam”. Estos comentarios se filtrarán y se moverán a la sección “Spam”.

<div id="creating-or-linking-external-issues-from-user-feedback">
  ## Crear o vincular incidencias externas a partir de comentarios de usuarios
</div>

Si tienes GitHub, Jira u otra [integración de seguimiento de incidencias](https://sentry.io/integrations/) configurada con Sentry, puedes crear una incidencia externa o vincular una existente desde la página de Comentarios de usuarios. En la vista de **Detalles de comentarios de usuarios**, deberías ver todas tus integraciones en el Header, debajo del nombre del proyecto.

![Encabezado de comentarios de usuarios.](./img/user-feedback-header.png)

Al hacer clic en el nombre de una integración se abrirá un modal que te permitirá crear una nueva incidencia externa o vincular una ya existente.

![Modal de incidencia vinculada de comentarios de usuarios.](./img/user-feedback-linked-issue-modal.png)

El modal se rellenará con un título predeterminado (&quot;User Feedback&quot;) y una descripción con los detalles básicos del comentario, incluido un enlace al comentario en Sentry y el mensaje del usuario. Puedes editar el título y la descripción, así como configurar el repositorio de la incidencia, las etiquetas y la persona asignada.

Para crear la incidencia, simplemente haz clic en el botón &quot;Create Issue&quot; en la parte inferior del modal. Para vincular una incidencia existente, haz clic en la pestaña &quot;Link&quot; del modal y luego en &quot;Link Issue&quot;.

<div id="searching-for-user-feedback">
  ## Búsqueda de comentarios de usuarios
</div>

Puedes buscar comentarios con el cuadro de búsqueda. Por ejemplo, puedes buscar comentarios enviados en una página específica.

Búsqueda de ejemplo: `url:*/payments/*`

Para ver la lista de campos de búsqueda, consulta la [documentación de búsqueda](/es/concepts/search/searchable-properties/user-feedback/).

<div id="spam-detection-for-user-feedback">
  ## Detección de spam en los comentarios de usuarios
</div>

Si tu organización habilitó las funciones de IA generativa, Sentry detectará automáticamente los envíos de comentarios de usuarios que probablemente sean spam y los colocará en la carpeta “Spam” en la página de **User Feedback**. Esto mejora la calidad y la utilidad de los comentarios que aparecen en la lista principal. Para mejorar la calidad de tus alertas, los comentarios marcados como spam omitirán cualquier regla de alerta de User Feedback que hayas configurado y no recibirás alertas por spam.

Para posibilitar la detección automática de spam, usamos un modelo de lenguaje de gran tamaño que clasifica los mensajes. Este modelo es interno de Google Cloud Platform y no se almacena ni conserva ningún dato. Esto forma parte de nuestro [acuerdo con subencargados del tratamiento](https://sentry.io/legal/dpa/#subprocessors).

Si deseas excluirte de la detección de spam, desactiva “Enable Spam Detection” en [Settings](https://sentry.io/orgredirect/organizations/:orgslug/settings/) &gt; Projects &gt; [Project Name] &gt; User Feedback (en “Project”).

<div id="getting-user-feedback-alerts">
  ## Recibir alertas de comentarios de usuarios
</div>

Si tienes activada la alerta de incidencia predeterminada de Sentry (&quot;Alertarme en cada incidencia nueva&quot;) para los proyectos con comentarios de usuarios configurados, recibirás automáticamente una alerta cada vez que llegue nuevo feedback de usuario mediante el User Feedback Widget.

Si no tienes activada la alerta de incidencia predeterminada de Sentry, sigue estos pasos para configurar alertas para cada nuevo feedback:

1. Crea una [nueva regla de alerta](https://sentry.io/alerts/new/issue/) en Sentry.
2. Desplázate a la sección &quot;Set conditions&quot; y establece el filtro &quot;IF&quot; en `The issue's category is equal to… "Feedback"`.
3. Elige qué acciones realizar en el filtro &quot;THEN&quot;.
4. Añade un nombre y un responsable a la alerta.

![Configuración de alerta de comentarios de usuarios.](./img/user-feedback-alert-configuration.png)

Para recibir notificaciones cuando llegue feedback de informes de bloqueo (crash reports), asegúrate de activar &quot;Enable Crash Report Notifications&quot; en Settings &gt; Projects &gt; [Project Name] &gt; User Feedback.

<div id="automated-issue-creation-for-user-feedback">
  ## Creación automática de incidencias a partir de comentarios de usuarios
</div>

Si quieres crear automáticamente una incidencia externa (como una incidencia en GitHub o Jira) cada vez que recibas comentarios de usuarios, puedes hacerlo desde la página Alerts en Sentry, de forma similar a cómo crearías una alerta de incidencia para comentarios de usuarios (como se muestra en la sección anterior).

Sigue estos pasos para configurar la creación automática de incidencias en GitHub o Jira para cada nuevo comentario:

1. Crea una [New Alert Rule](https://sentry.io/alerts/new/issue/) en Sentry.
2. Desplázate a la sección &quot;Set conditions&quot; y establece el filtro &quot;IF&quot; en `The issue's category is equal to… "Feedback"`.
3. En las acciones del filtro &quot;THEN&quot;, desplázate a la sección &quot;Create new...&quot; y selecciona la integración que quieras usar.
4. La configuración predeterminada del enlace de la incidencia, como el repositorio, las etiquetas y las personas asignadas, se puede ajustar haciendo clic en el Button &quot;Issue Link Settings&quot;.
5. Añade un nombre y un responsable para la alerta.

Una vez configurada esta alerta, verás que se crea automáticamente una incidencia en GitHub o Jira cada vez que se envían comentarios de usuarios.

![Configuración de la alerta de creación automática de incidencias a partir de comentarios de usuarios.](./img/user-feedback-auto-issue-creation-configuration.png)
