Para capturar errores y otros eventos, necesitas tener un proyecto configurado en Sentry. En este recorrido crearemos un proyecto de frontend en React y otro de backend por separado en Express.

Como puedes crear un proyecto de Sentry para *cada* lenguaje o framework utilizado en tu aplicación (por ejemplo, podrías tener proyectos separados para tu servidor de API y tu cliente de frontend), una sola aplicación podría tener varios proyectos asociados. Una vez que hayas configurado en Sentry todos los proyectos que correspondan a las distintas partes de tu aplicación, podrás acotar los eventos a una aplicación específica de tu organización y asignar la responsabilidad a usuarios y equipos concretos. Para más información, consulta las [mejores prácticas para crear proyectos](/es/organization/getting-started/#4-create-projects).

<div id="create-a-frontend-project">
  ## Crear un proyecto de frontend
</div>

Sigue los pasos a continuación para crear un nuevo proyecto de Sentry para una aplicación de ejemplo de React:

1. Inicia sesión en tu [organización de Sentry](https://sentry.io).

2. Selecciona **Projects** en el menú de navegación de la izquierda.

3. Haz clic en &quot;Create Project&quot; y configúralo según corresponda para tu aplicación:

   * **Elige tu plataforma**: Selecciona el lenguaje o framework de tu proyecto según el código que deseas monitorear. En este caso, elige **React**.

   * **Configura la frecuencia de alertas**: Para los fines de este tutorial, selecciona **Alert me on high priority issues**.

     > Si te preocupa la fatiga por alertas, obtén más información sobre cómo configurar tus alertas en [Mejores prácticas de alertas](/es/product/alerts/best-practices/).

   * **Nombra tu proyecto y asígnalo a un equipo**: Asigna un nombre a tu proyecto en el campo **Project name** y asigna un equipo seleccionando uno en el menú desplegable **Team** (o haz clic en **+** para crear un equipo nuevo).

   * Haz clic en **Create Project**. Esto te llevará a la guía rápida para configurar el SDK de React, que ofrece una descripción general de cómo configurarlo. Este tutorial cubre la configuración del SDK con más detalle.

   * Notarás que el código de configuración del SDK de Sentry para React ya viene precompletado con el DSN (Data Source Name) único del proyecto. El DSN le indica al SDK adónde enviar los eventos, para que se asocien con el proyecto correcto. Más adelante en el tutorial necesitarás pegar la clave DSN en el código de configuración del SDK. Puedes tomar nota ahora o, en cualquier momento, encontrar el DSN de un proyecto en **[Project] &gt; Settings &gt; SDK Setup &gt; Client Keys (DSN)**

4. Haz clic en **Take me to Issues** para ir a la página **Issues** de tu nuevo proyecto.

<div id="create-a-backend-project">
  ## Crear un proyecto de backend
</div>

A continuación, sigue estos pasos para crear un nuevo proyecto de Sentry para una aplicación de backend de ejemplo:

1. Inicia sesión en tu [organización de Sentry](https://sentry.io).

2. Selecciona **Projects** en el menú de navegación izquierdo para ver la lista de todos tus proyectos.

3. Haz clic en &quot;Create Project&quot; y configúralo según corresponda para tu aplicación:

   * **Choose your platform**: En este caso, elige **Express**.

   * **Set your alert frequency**: Una vez más, selecciona **Alert me on high priority issues**.

   * **Name your project and assign it a team**: Asegúrate de darle un nombre de proyecto diferente al de tu aplicación frontend.

   * Haz clic en **Create Project**. Esto te llevará a la guía rápida para configurar el SDK de Express, que ofrece una descripción general de cómo configurarlo.

4. Copia la clave DSN y tenla a mano.

   > También puedes encontrar el DSN de un proyecto en cualquier momento en **[Project] &gt; Settings &gt; SDK Setup &gt; Client Keys (DSN)**.

5. Haz clic en **Take me to Issues** para ir a la página **Issues** de tu nuevo proyecto.
