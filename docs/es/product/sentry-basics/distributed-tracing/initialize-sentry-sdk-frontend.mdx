Esta sección te guía para importar la aplicación de ejemplo en tu entorno de desarrollo local, agregar el SDK de Sentry e inicializarlo.

Si estás usando tu propio código, puedes omitir esta sección. En su lugar:

* Selecciona tu [plataforma](/es/platforms/) y sigue la guía de **Introducción** para agregar el SDK de Sentry a tu código.
* Luego, pasa al [siguiente paso](/es/product/sentry-basics/distributed-tracing/initialize-sentry-sdk-backend/).

<div id="1-clone-the-sample-application">
  ## 1. Clona la aplicación de ejemplo
</div>

La aplicación de ejemplo es una aplicación básica de solo frontend que utiliza React y webpack.

1. Haz un fork del [repositorio de la aplicación de ejemplo](https://github.com/getsentry/tracing-tutorial-frontend) en GitHub.

2. Clona el repositorio forkeado en tu entorno local:

   ```bash
   git clone git@github.com:<your_username>/tracing-tutorial-frontend.git
   ```

3. Abre el proyecto `tracing-tutorial-frontend` en tu editor de código preferido.

<div id="2-add-the-sentry-react-sdk">
  ## 2. Agrega el SDK de Sentry para React
</div>

Sentry captura datos usando un SDK específico de la plataforma que agregas al entorno de ejecución de tu aplicación. Para usar el SDK, impórtalo y configúralo en tu código fuente. Este proyecto de demostración usa el [SDK de React de Sentry](https://github.com/getsentry/sentry-javascript/tree/develop/packages/react).

1. Instala el SDK de Sentry para React usando NPM.

   Asegúrate de estar en la carpeta del proyecto `tracing-tutorial-frontend`.

   ```bash {tabTitle:npm}
   npm install @sentry/react --save
   ```

   ```bash {tabTitle:yarn}
   yarn add @sentry/react
   ```

   ```bash {tabTitle:pnpm}
   pnpm add @sentry/react
   ```

2. Importa e inicializa el SDK.

   Abre `src/index.js` y agrega las siguientes líneas de código debajo de la última instrucción de importación:

   ```javascript {filename:src/index.js} {12}
   import * as Sentry from "@sentry/react";

   Sentry.init({
     dsn: "your_DSN_key",
     integrations: [
       Sentry.browserTracingIntegration(),
       Sentry.replayIntegration(),
     ],
     // Tracing
     tracesSampleRate: 1.0, //  Captura el 100% de las transacciones
     // Configura 'tracePropagationTargets' para controlar en qué URLs se habilita el tracing distribuido
     tracePropagationTargets: ["localhost", /^https:\/\/yourserver\.io\/api/],
     // Session Replay
     replaysSessionSampleRate: 0.1, // Establece la tasa de muestreo en 10%. Durante el desarrollo puedes subirla al 100% y bajarla en producción.
     replaysOnErrorSampleRate: 1.0, // Si no estás muestreando toda la sesión, usa 100% al muestrear sesiones donde ocurren errores.
   });
   ```

   Es importante importar e inicializar el SDK lo antes posible en el ciclo de vida de tu aplicación para que Sentry pueda capturar errores durante todo ese ciclo.

3. Agrega tu clave DSN a la configuración del SDK de Sentry.

   Pega el valor de la clave DSN que copiaste del proyecto creado en la [sección anterior](/es/product/sentry-basics/distributed-tracing/create-new-project/).

   ```javascript {filename:src/index.js}
   Sentry.init({
     dsn: "<your_DSN_key>",
     // ...
   });
   ```

4. Guarda el archivo.

Las opciones definidas en `Sentry.init()` constituyen la configuración del SDK. La única opción de configuración obligatoria es el DSN. Sin embargo, el SDK admite muchas otras opciones de configuración. Obtén más información en nuestra documentación de [Configuration](/es/platforms/javascript/guides/react/configuration/).

La configuración anterior habilita la supervisión de errores de Sentry, así como las funciones de [**Tracing**](/es/product/tracing/) y [**Session Replay**](/es/platforms/javascript/guides/react/session-replay). Ten en cuenta la opción `tracePropagationTargets`; esto es necesario para habilitar el tracing en cualquier URL donde se estén ejecutando tus proyectos. Dado que ambos proyectos de este tutorial se ejecutan en `localhost`, estamos listos.

<div id="3-build-and-run-the-sample-application">
  ## 3. Compila y ejecuta la aplicación de ejemplo
</div>

En la carpeta del proyecto `tracing-tutorial-frontend`:

1. Instala las dependencias del proyecto.

   ```bash
   npm install
   ```

2. Inicia la aplicación en modo desarrollo.

   ```bash
   npm start
   ```

   Una vez que la aplicación se inicie, verás un mensaje de confirmación similar a este en tu terminal:

   ```bash
   Compiled successfully!

   You can now view tracing-demo in the browser.

      Local:            http://localhost:3000
      On Your Network:  http://10.0.0.39:3000
   ```

   > Consejo para la solución de problemas: si la aplicación no se inicia debido a errores de sintaxis o por dependencias/módulos faltantes, asegúrate de usar Node 18+ e instala las dependencias de nuevo. Si usas [nvm](https://github.com/nvm-sh/nvm), verifica que estás en la versión correcta de Node con `nvm use 18` y luego `npm install`.

3. Abre la aplicación de ejemplo en tu navegador.

   La app de ejemplo debería estar ejecutándose en [http://localhost:3000/](http://localhost:3000/) o en la URL que apareció en tu terminal en el paso anterior. Deberías ver una página de comercio electrónico de ejemplo; los botones de esta página no funcionarán correctamente y generarán un error de tiempo de ejecución no capturado hasta que pongas tu backend en marcha.
