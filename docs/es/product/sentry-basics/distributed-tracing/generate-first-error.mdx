Ahora que las aplicaciones de ejemplo están en funcionamiento en tu entorno local e integradas con los SDK de Sentry, estás listo para generar el primer error. Asegúrate de tener AMBAS aplicaciones en ejecución (en http://localhost:3000 y http://localhost:3001).

Deberías ver el sitio “One-Stop Shop” ejecutándose en http://localhost:3000. Al hacer clic en cualquiera de los botones de producto se realizará una llamada al backend para obtener información del producto. Mientras tu servidor de Express esté en ejecución, deberías ver títulos, imágenes y descripciones de cada producto.

<div id="1-trigger-an-error">
  ## 1. Generar un error
</div>

Para empezar a usar la función de monitoreo de errores de Sentry, primero necesitas algunos errores. Agreguemos un error evidente que sea fácil de ver en Sentry.

Si estás usando tu propio código, omite este paso. En su lugar, selecciona tu [plataforma](/es/platforms/) y sigue el paso **Verify** dentro de la guía **Getting Started** para provocar un error.

1. En el repositorio `tracing-tutorial-frontend`, abre `src/App.js` y actualiza el controlador `onClick`, reemplazando la cadena que se pasa a la función `getProduct()` de `nonfat-water` por `debug-sentry`.

```jsx diff {filename:App.js}
    <div className="btn-parent">
        <button className="btn" onClick={() => getProduct("clown-shoes")}>
            Zapatos de payaso
        </button>
    </div>
    <div className="btn-parent">
-      <button className="btn" onClick={() => getProduct("nonfat-water")}>
+      <button className="btn" onClick={() => getProduct("debug-sentry")}>
        Agua sin grasa
       </button>
    </div>
```

Hay una función de middleware que gestiona la ruta `/products/debug-sentry` en el archivo `server.js` del repositorio `tracing-tutorial-backend` que arroja un error de ejemplo:

```javascript {filename:server.js}
app.get("/products/debug-sentry", (req, res) => {
  console.log("¡Error de Sentry lanzado!");
  throw new Error("¡Mi primer error de Sentry!");
});
```

1. Guarda el archivo.

2. Vuelve a la ventana del navegador e intenta hacer clic de nuevo en el botón **Nonfat Water**; ya no verás la información del producto esperada. En su lugar, verás un mensaje de error `500`.

3. Abre la ventana de la terminal que está ejecutando el servidor `tracing-tutorial-backend`. Aquí verás un registro que te indica que se ha producido un error:

```bash
¡Se produjo un error de Sentry!
```

Esto confirma que el error que acabamos de provocar en el frontend ha generado un error correspondiente en el backend.

<div id="2-view-the-error-in-sentry">
  ## 2. Ver el error en Sentry
</div>

Ahora que has provocado un error, veamos cómo se muestra en Sentry.

1. Abre la página [**Traces**](https://sentry.io/orgredirect/organizations/:orgslug/traces/) en Sentry.io. Asegúrate de que uno (o ambos) de los proyectos que estás usando para este tutorial esté seleccionado en el menú desplegable de proyectos.

2. Deberías ver un trace que muestra los íconos de React y Express, lo que indica que este trace recorre ambos proyectos. Haz clic en el ID de trace azul a la izquierda para ver la Trace View.

3. En la página [Trace View](/es/concepts/key-terms/tracing/trace-view/), puedes ver cada span que ocurrió dentro de este trace. Puedes ver el error en la app de React que desencadenó directamente el error en la app de Express.
