Ahora que la aplicación de ejemplo está en funcionamiento en tu entorno local e integrada con el SDK de Sentry, ya puedes generar tu primer error.

<div id="1-trigger-an-error">
  ## 1. Provocar un error
</div>

Para empezar a usar la función de monitoreo de errores de Sentry, primero necesitas algunos errores. Agreguemos un error obvio que sea fácil de ver en Sentry.

Si estás usando tu propio código, omite este paso. En su lugar, selecciona tu [plataforma](/es/platforms/) y sigue el paso **Verify** dentro de la guía **Getting Started** para provocar un error.

1. Abre `src/components/App.js` y descomenta la línea 97.

   ```javascript {filename:src/components/App.js}
   this.myCodeIsPerfect();
   ```

   La función `myCodeIsPerfect()` no está definida, así que esto generará un error.

2. Guarda el archivo.

3. Actualiza el navegador y abre la consola de desarrollador.

   La app se está ejecutando en modo de desarrollo, así que el cambio debería aplicarse tras actualizar. Aún no deberías ver errores en la consola.

4. Haz clic en cualquiera de los botones &quot;Buy!&quot; para agregar productos al carrito.

5. Haz clic en el botón &quot;Checkout&quot; en el panel derecho para generar un error.

   <Arcade src="https://demo.arcade.software/fEC5K0E8owyg5R45ePEo?embed" />

   Observa que:

   * En la app se muestra el mensaje de error &quot;Something went wrong&quot;.

   * El error aparece en la consola del navegador.

     ![Import and Configure SDK](./img/generate-first-error-01.png)

   * Se envía una alerta a la dirección de correo electrónico que tienes configurada en Sentry para avisarte de un error que ocurrió en tu app.

     ![Alert Email](./img/new_issue_email.png)

<div id="2-view-the-error-in-sentry">
  ## 2. Ver el error en Sentry
</div>

Ahora que has provocado un error, veamos cómo se muestra en Sentry.

1. Abre la página de **Issues** en [Sentry.io](https://sentry.io/). Asegúrate de que tu proyecto esté seleccionado en el menú desplegable de proyectos.

2. Selecciona el error de la lista de issues para ver todos los detalles y el contexto de este error.

   Ten en cuenta que Sentry agrupa errores similares (eventos) en un solo [Issue](/es/product/issues/). Si provocas este error varias veces, no verás aparecer más issues. En su lugar, verás aumentar el número de eventos de ese issue.

3. En la página de [Issue Details](/es/product/issues/issue-details/), desplázate hacia abajo hasta el stack trace.

   Observa que el stack trace está minificado. Normalmente, JavaScript se minifica para reducir el tamaño del código fuente. Esto significa que no puedes ver las líneas reales de código de tu app en el stack trace.

   Con un poco de configuración, Sentry puede desminificar tu código para volverlo legible y mostrar tu código fuente en cada frame del stack. Configurarás esto en la siguiente sección.
