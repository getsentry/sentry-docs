Al crear aplicaciones web, la mayoría de los flujos de trabajo de desarrollo implican transformar tu código JavaScript mediante transpilación y minificación para que se ejecute de forma más eficiente en el navegador. Los archivos de source map sirven como guía para herramientas como Sentry y permiten convertir la versión transformada de tu JavaScript de nuevo al código que escribiste. Tu herramienta de compilación de JavaScript puede generar estos archivos.

El proyecto de ejemplo usa webpack como herramienta de compilación. Lo configuraremos para generar source maps y verificaremos que se generen.

Si tu proyecto tiene una configuración diferente, usa el [Sentry Wizard](/es/platforms/javascript/sourcemaps/#uploading-source-maps) para configurar tus source maps y salta al paso 3.

<div id="1-generate-source-map-files-using-your-build-tool">
  ## 1. Genera archivos de mapas de código fuente con tu herramienta de compilación
</div>

1. Para configurar webpack para generar mapas de código fuente, abre `webpack.config.js` y agrega la siguiente línea dentro de `module.exports`:

   ```javascript {filename:webpack.config.js}
   ...
   module.exports = {
      entry: "./src/index.js",
      output: {
         filename: "main.js",
         path: path.resolve(__dirname, "build"),
      },
      devtool: "source-map",
      ...
   }
   ...
   ```

   Si estás usando el proyecto de ejemplo, puedes descomentar la línea 11.

2. Guarda el archivo.

3. Webpack solo genera mapas de código fuente cuando creas una compilación de producción. Para hacerlo, detén tu app que se está ejecutando en modo desarrollo con `Ctrl + C` y comienza una compilación de producción ejecutando:

   ```bash
   npm run build
   ```

   Deberías ver una nueva carpeta `build` en tu proyecto. Dentro, debería haber un archivo `main.js` y un archivo `main.js.map`. `main.js` es tu JavaScript minificado. `main.js.map` es el archivo del mapa de código fuente.

<div id="2-upload-your-source-maps-to-sentry">
  ## 2. Sube tus source maps a Sentry
</div>

1. Instala el complemento de webpack de Sentry

   ```bash
   npm install @sentry/webpack-plugin --save-dev
   ```

2. Para generar un token de autenticación de Sentry y guardarlo para su uso, pega lo siguiente en tu terminal:

   <OrgAuthTokenNote />

   ```bash {filename:.env}
   export SENTRY_AUTH_TOKEN=___ORG_AUTH_TOKEN___
   ```

3. Abre `webpack.config.js` y añade las siguientes líneas para agregar la configuración del complemento a webpack.

   ```javascript {filename:webpack.config.js}
   const { sentryWebpackPlugin } = require("@sentry/webpack-plugin");

   ...

   module.exports = {
      ...
     plugins: [
         sentryWebpackPlugin({
            org: "___ORG_SLUG___",
            project: "___PROJECT_SLUG___",
            authToken: process.env.SENTRY_AUTH_TOKEN,
         }),
      ],
      ...
   }
   ```

4. Puedes encontrar tu `org` como parte de tu URL de Sentry (por ejemplo, https://**&lt;organization&#95;slug&gt;**.sentry.io/), o en la página de **Organization Settings**.

<div id="3-trigger-another-error">
  ## 3. Generar otro error
</div>

1. Crea una compilación de producción nueva con tus cambios del paso anterior:

   ```bash
   npm run build
   ```

2. Confirma que tus source maps se subieron correctamente a Sentry y se asociaron con la última versión de tu proyecto. La salida de tu terminal debería incluir un `Source Map Upload Report`, similar a lo siguiente:

   ```bash
   > Organization: <your_org>
   > Project: <your_project>
   > Release: 0d40018e21151113e224f208fb934a0d29f10508
   > Dist: None
   > Upload type: artifact bundle
   Source Map Upload Report
      Minified Scripts
         ~/f320b889-aa78-4850-8625-802a2ee9aca3-0.js (sourcemap at main.js.map, debug id f320b889-aa78-4850-8625-802a2ee9aca3)
      Source Maps
         ~/f320b889-aa78-4850-8625-802a2ee9aca3-0.js.map (debug id f320b889-aa78-4850-8625-802a2ee9aca3)
   [sentry-webpack-plugin] Info: Successfully uploaded source maps to Sentry
   ```

3. Ejecuta tu compilación de producción:

   ```bash
   npx serve build
   ```

4. Abre en tu navegador la compilación de producción de la aplicación de ejemplo.

   La aplicación de ejemplo debería estar ejecutándose en [http://localhost:3000/](http://localhost:3000/) o en la URL que apareció en tu terminal en el paso anterior.

5. En tu navegador, asegúrate de que la consola de desarrollo esté abierta y realiza un “Empty Cache and Hard Reload” para confirmar que se sirva el código actualizado.

   ![Versión creada](./img/upload-source-maps-08.png)

6. Genera el error nuevamente añadiendo productos a tu carrito y haciendo clic en “Checkout”.

7. Ve a la página de **Issues** en Sentry. Como agregaste una versión nueva, debería crearse un issue nuevo aunque sea el mismo error que antes.

8. Haz clic en el issue nuevo para abrir su página de **Issue Details**.

9. Observa que ahora hay una etiqueta de versión para el issue y que el stack trace del error ya no está minificado. El stack trace ahora incluye el nombre del archivo, el nombre del método, los números de línea y columna, y el contexto del código fuente en cada frame de la pila.
