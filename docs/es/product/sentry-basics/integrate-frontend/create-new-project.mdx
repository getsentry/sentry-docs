Esta sección te guía para crear un proyecto nuevo en tu cuenta de Sentry. Necesitas tener un proyecto configurado en Sentry para capturar errores y otros eventos.

*Los proyectos* te permiten acotar los eventos a una aplicación específica de tu organización y asignar responsabilidades a usuarios y equipos concretos. Puedes crear un proyecto para un lenguaje o framework específico usado en tu aplicación. Por ejemplo, podrías tener proyectos separados para tu servidor de API y tu cliente frontend. Para obtener más información, consulta las [mejores prácticas para crear proyectos](/es/organization/getting-started/#4-create-projects).

<div id="create-a-project">
  ## Crear un proyecto
</div>

Sigue los pasos a continuación para crear un nuevo proyecto de Sentry para una aplicación de ejemplo en React:

1. Inicia sesión en tu [organización de Sentry](https://sentry.io).

2. Selecciona **Projects** en el menú de navegación lateral para ver la lista de todos tus proyectos.

3. Haz clic en &quot;Create Project&quot; y configúralo según corresponda para tu aplicación:

   * **Choose your platform**: Selecciona el lenguaje o framework de tu proyecto según el código que deseas monitorear. En este caso, elige **React**.

   * **Set your alert frequency**: Para los fines de este tutorial, selecciona **Alert me on high priority issues**.

     > Si te preocupa la fatiga por alertas, obtén más información sobre cómo configurar tus alertas en [Alerts Best Practices](/es/product/alerts/best-practices/).

   * **Name your project and assign it a team**: Asigna un nombre a tu proyecto en el campo **Project name** y asigna un equipo seleccionando uno en el menú desplegable **Team** (o haz clic en **+** para crear uno nuevo).

   * Haz clic en **Create Project**. Esto te llevará a la guía rápida Configure React SDK, que ofrece una descripción general de cómo configurar el SDK. Este tutorial cubre la configuración del SDK con más detalle.

4. Copia la clave DSN y tenla a mano. Cada proyecto tiene un DSN (Data Source Name) único. El DSN le indica al SDK adónde enviar los eventos, para que se asocien con el proyecto correcto. Más adelante deberás pegar la clave DSN en tu código fuente para que los errores generados en este tutorial vayan a tu nuevo proyecto.

   > También puedes encontrar el DSN de un proyecto en cualquier momento en **[Project] &gt; Settings &gt; SDK Setup &gt; Client Keys (DSN)**.

5. Haz clic en **Take me to Issues** para ir a la página **Issues** de tu nuevo proyecto.
