Sentry ofrece varias opciones de configuración para mejorar la funcionalidad del SDK. Estas opciones pueden proporcionar datos adicionales para depurar problemas más rápido o ayudarte a controlar, mediante filtros, qué se envía a Sentry. Obtén más información en [Configuración](/es/platforms/python/configuration/).

<div id="releases">
  ## Lanzamientos
</div>

Un *lanzamiento* es una versión de tu código que se despliega en un entorno. Configurar el lanzamiento te ayuda a detectar regresiones en tu código, garantizar la responsabilidad, resolver problemas en Sentry y mantenerte al día con tus despliegues. Los lanzamientos deben configurarse en tu SDK y luego gestionarse con [sentry-cli](/es/cli/).

Sentry admite integraciones con GitHub, Bitbucket, Azure DevOps, GitLab y otros. Para ver la lista completa, consulta nuestra [documentación de integraciones](/es/organization/integrations/).

Para configurar el lanzamiento en este proyecto:

1. Abre el archivo `settings.py`. Observa que añadimos la opción de configuración `release` al inicializar el SDK:

   ```python {filename: myproject/settings.py}
   sentry_sdk.init(
       ...
       release=os.environ.get("VERSION"),
       ...
   )
   ```

2. Abre el `Makefile` que ejecutaste en el tutorial anterior.

   ![Makefile](./img/makefile.png)

3. Observa que estamos definiendo el nombre de la versión del lanzamiento como una variable de entorno que luego se usa en tiempo de ejecución de la aplicación. Dejamos que la CLI proponga un nombre de versión, pero probablemente quieras aplicar tus propias convenciones de nombres:

   ```Makefile {filename: Makefile}
   VERSION=`sentry-cli releases propose-version`
   ```

4. Luego creamos un nuevo lanzamiento para nuestro proyecto con el nombre propuesto o elegido.

   ```Makefile {filename: Makefile}
   create_release:
       sentry-cli releases -o $(SENTRY_ORG) new -p $(SENTRY_PROJECT) $(VERSION)
   ```

5. En el tutorial anterior, configuramos la integración con GitHub y añadimos el repositorio de código para el seguimiento de commits. Ahora podemos asociar los commits de ese repositorio al nuevo lanzamiento ejecutando el siguiente comando:

   ```Makefile {filename: Makefile}
   associate_commits:
       sentry-cli releases -o $(SENTRY_ORG) -p $(SENTRY_PROJECT) \
           set-commits $(VERSION) --auto
   ```

<div id="breadcrumbs">
  ## Rastro de eventos
</div>

*Los breadcrumbs* son la secuencia de eventos que condujeron a un error. Son muy útiles para reproducir un problema. Según la plataforma, el SDK rastreará distintos tipos de breadcrumbs de forma predeterminada (en SDKs de backend, por ejemplo, consultas a la base de datos, eventos de red, logs y otros). También puedes agregar tus propios breadcrumbs personalizados.

Para agregar breadcrumbs a nuestra app:

1. Abre el archivo `myapp > view.py`.

2. Observa que importamos la librería `sentry_sdk`, que contiene el método `add_breadcrumb`:

   ```python {filename: myapp/views.py}
   import sentry_sdk
   ```

3. Luego creamos un breadcrumb personalizado para cada manejador de método en las clases de vista. Este breadcrumb se agrega al rastro de breadcrumbs asociado con cualquier error desencadenado a través de estos flujos de llamadas de método. Por ejemplo, en `HandledErrorView.get`:

   ```python {filename: myapp/views.py}
   sentry_sdk.add_breadcrumb(
       category='URL Endpoints',
       message='In the handled function',
       level='info',
   )
   ```

<div id="environment">
  ## Entorno
</div>

*Entorno* es una potente opción de configuración que permite a los desarrolladores que usan Sentry realizar distintos flujos de trabajo (como filtrar incidencias y activar alertas) en el contexto del entorno de implementación en el que se produjeron los errores.

1. Abre el archivo `settings.py`.

2. Observa que inicializamos el SDK con la opción de configuración `environment`. Cualquier evento que el SDK capture posteriormente se etiquetará con el valor de entorno configurado.

   ```python {filename: myproject/settings.py}
   sentry_sdk.init(
       ...
       environment="Production"
       ...
   )
   ```

   > Los valores de entorno son cadenas de texto libres. Ni el SDK de Sentry ni [sentry.io](https://sentry.io) te limitarán a un valor o formato específico. En este ejemplo, fijamos el valor en el código. En una aplicación real, probablemente se determinaría dinámicamente mediante un archivo de propiedades, un sistema o una variable de entorno.
