En este tutorial, importarás el código fuente de la app de backend a tu entorno local de desarrollo, añadirás el SDK de Sentry y lo inicializarás.

<Alert>
  Si usas tu propio código fuente, puedes omitir este tutorial y:

  * Seguir las instrucciones de la documentación de [Primeros pasos](/es/platforms/python/).
  * Ir directamente al [siguiente paso](/es/guides/integrate-backend/configuration-options/).
</Alert>

<div id="prerequisites">
  ## Requisitos previos
</div>

1. El código fuente de la aplicación de demostración requiere un entorno de desarrollo de Python para compilar, instalar y ejecutar la aplicación. Asegúrate de contar con lo siguiente:

   * Un editor de código (como [VS Code](https://code.visualstudio.com))
   * [Python](https://www.python.org/)
   * [Sentry CLI](/es/cli/)

2. Para comenzar a monitorizar errores en tu aplicación, deberás crear un proyecto nuevo en tu cuenta de Sentry.

<div id="step-1-create-a-project">
  ## Paso 1: Crea un proyecto
</div>

Sigue los pasos a continuación para crear un nuevo proyecto de Sentry para una aplicación de ejemplo de Django:

1. Inicia sesión en tu [organización de Sentry](https://sentry.io).

2. Selecciona **Projects** en el menú de navegación lateral para ver la lista de todos tus proyectos.

3. Haz clic en &quot;Create Project&quot; y configúralo según corresponda para tu aplicación:

   * **Elige tu plataforma**: Selecciona el lenguaje o framework de tu proyecto según el código que quieras monitorizar. En este caso, elige **Django**.

   * **Configura la frecuencia de alertas**: Para este tutorial, selecciona **I&#39;ll create my own alerts later**.

     > Si te preocupa la fatiga por alertas, obtén más información sobre cómo configurar tus alertas en [Mejores prácticas de alertas](/es/product/alerts/best-practices/).

   * **Ponle un nombre a tu proyecto y asígnalo a un equipo**: Escribe el nombre en el campo **Project name** y asigna un equipo seleccionándolo en el desplegable **Team** (o haz clic en **+** para crear uno nuevo).

   * Haz clic en **Create Project**. Esto te llevará a la guía rápida Configure Django SDK, que ofrece una descripción general de cómo configurar el SDK. Este tutorial cubre la configuración del SDK con más detalle.

4. Copia la clave DSN y tenla a mano. Cada proyecto tiene un DSN (Data Source Name) único. El DSN le indica al SDK adónde enviar los eventos, para que se asocien al proyecto correcto. Más adelante tendrás que pegar la clave DSN en tu código fuente para que los errores generados en este tutorial vayan a tu nuevo proyecto.

   > También puedes encontrar el DSN de un proyecto en cualquier momento en **[Project] &gt; Settings &gt; SDK Setup &gt; Client Keys (DSN)**.

<div id="step-2-get-the-code">
  ## Paso 2: Obtén el código
</div>

1. Abre en GitHub el [repositorio de código](https://github.com/sentry-tutorials/backend-monitoring) de ejemplo.

2. Haz clic en &quot;Fork&quot; y selecciona la cuenta de GitHub de destino en la que quieres crear el fork de este repositorio.

3. Una vez completado el fork, haz clic en &quot;Clone or download&quot; y copia la URL del repositorio. Puedes hacerlo mediante HTTPS, SSH o con GitHub CLI.

   ![Clonar repositorio](./img/fork_n_clone.png)

4. Clona el repositorio bifurcado en tu entorno local:

   ```bash
   > git clone <repository HTTPS or SSH url>
   ```

5. Ahora que el código de ejemplo está disponible en local, abre el proyecto `backend-monitoring` en tu editor de código preferido.

<div id="step-3-enable-commit-tracking-for-your-repository">
  ## Paso 3: Habilita el seguimiento de commits para tu repositorio
</div>

Sentry puede ayudarte a resolver errores más rápido sugiriendo un [commit sospechoso](/es/product/sentry-basics/integrate-frontend/configure-scms/) que podría haber introducido el error en tu código. Esto se activa al configurar el *seguimiento de commits*. Para habilitarlo, debes integrar tu solución de gestión de código fuente y agregar tus repositorios. Obtén más información en nuestra [documentación sobre releases](/es/product/releases/?platform=node/suspect-commits/).

1. Abre tu cuenta de Sentry y ve a **Settings &gt; Integrations** para habilitar la integración con GitHub y agregar tu repositorio `backend-monitoring`. Para más detalles, sigue los pasos descritos en [nuestra documentación de GitHub](/es/organization/integrations/source-code-mgmt/github/).

<div id="step-4-install-the-sdk">
  ## Paso 4: Instalar el SDK
</div>

Sentry captura datos mediante un SDK específico de la plataforma dentro del entorno de ejecución de tu aplicación. Para usar el SDK, impórtalo, inicialízalo y configúralo en tu código fuente.

1. Instala `sentry-sdk` definiendo la dependencia en el archivo `requirements.txt`. La documentación del SDK y la información sobre versiones están disponibles en el repositorio de GitHub [Sentry SDK](https://github.com/getsentry/sentry-python).

2. Abre el archivo `settings.py` (ubicado en `./backend-monitoring/myproject/settings.py`). Aquí es donde importamos, inicializamos y configuramos el SDK de Sentry en nuestra aplicación.

   ```python {filename: myproject/settings.py}
   import sentry_sdk

   sentry_sdk.init(
       dsn="YOUR_DSN",  # replace with your DSN
       ...
   )
   ```

   > En `settings.py`, reemplaza `YOUR_DSN` por el valor de la clave `DSN` que copiaste del proyecto creado en el [tutorial para crear un proyecto](/es/guides/integrate-frontend/create-new-project/).

<div id="step-5-install-dependencies-and-run-the-demo-app">
  ## Paso 5: Instalar dependencias y ejecutar la aplicación de demostración
</div>

Para compilar y ejecutar la aplicación de demostración en tu localhost:

1. Abre una terminal y cambia al directorio raíz del proyecto `backend-monitoring`.

2. Si aún no tienes Python instalado, hazlo ejecutando lo siguiente:

   ```bash
   brew install python
   ```

3. Crea un entorno virtual de Python en la raíz del proyecto.

   ```bash
   python -m venv .venv
   ```

   > Puedes nombrar el entorno virtual como quieras. Nosotros lo llamamos &quot;.venv&quot;.

4. Para activar el entorno virtual, ejecuta:

   ```bash
   source .venv/bin/activate
   ```

5. Instala la herramienta de línea de comandos de Sentry para usar el seguimiento de releases y la integración con GitHub para los datos de commits:

   ```bash
   python -m pip install "sentry-cli"
   ```

6. Abre el `Makefile` incluido en la carpeta raíz del proyecto. Aquí se usa para imitar un flujo de CI/CD.

7. Sigue el flujo de ejecución del objetivo `deploy`.

Los comandos mencionados dentro del `Makefile` se explicarán en detalle en la siguiente parte del tutorial, [Opciones de configuración](/es/product/sentry-basics/integrate-backend/configuration-options/).

<Alert>
  Además de instalar los requisitos de Python y ejecutar el servidor, también usamos `sentry-cli` para crear una nueva release de Sentry y asociar commits a esa release. Sentry revisará estos commits al señalar un commit sospechoso.
</Alert>

8. Para ejecutar los comandos de `sentry-cli`, sigue estas [instrucciones](/es/product/sentry-basics/integrate-frontend/upload-source-maps/#step-1-prepare-the-build-environment) para obtener los valores de tus variables de entorno `SENTRY_AUTH_TOKEN`, `SENTRY_ORG` y `SENTRY_PROJECT`.

   ![Makefile Config](./img/makefile_config.png)

   > `sentry-cli` se puede configurar proporcionando estos valores ya sea a través de variables de entorno o mediante un archivo de configuración dedicado. Más información en [Sentry CLI: Configuration and Authentication](/es/cli/configuration/).

9. Ejecuta el siguiente comando para instalar las bibliotecas de Python necesarias, configurar la release de Sentry y ejecutar el servidor de Django:

   ```bash
   make deploy
   ```

   ![Deploy & Serve](./img/run_django_server.png)

   > Observa que se crea una nueva release en la terminal y se le asocian commits. Una vez que el deploy finalice correctamente, verás la confirmación en tu terminal.
