Sentry es una plataforma de seguimiento de errores y monitoreo del rendimiento pensada para desarrolladores. Si aún no lo has hecho:

**→ [Crea tu cuenta en Sentry aquí](https://sentry.io/signup/)**

<div id="how-to-get-the-most-out-of-sentry">
  ## Cómo sacarle el máximo partido a Sentry
</div>

Hay tres cosas que te recomendamos hacer para sacarle el máximo partido a Sentry:

* **Conecta tu código**, y asegúrate de configurar también las [integraciones de gestión de código fuente](/es/organization/integrations/source-code-mgmt/) y los [source maps](/es/product/sentry-basics/integrate-frontend/upload-source-maps/) (si corresponde). Esto permitirá que Sentry vincule los errores y los problemas de rendimiento directamente con la línea de código problemática mediante trazas legibles y commits sospechosos.
* **Configura las [alertas](/es/product/alerts/)** para enterarte cuando algo importante falle. Puedes elegir recibir alertas a través de una de nuestras integraciones de chat como [Slack](/es/organization/integrations/notification-incidents/slack/) o [Discord](/es/organization/integrations/notification-incidents/discord/). Consulta nuestras [mejores prácticas de alertas](/es/product/alerts/best-practices/) para aprender a recibir solo las notificaciones relevantes.
* **Personaliza [Discover](/es/product/explore/discover-queries/), los [Dashboards](/es/product/dashboards/) y las [consultas](/es/product/explore/discover-queries/query-builder/)** para controlar mejor el tipo de información que obtienes cuando algo va mal en tu app. Por último, crea [etiquetas personalizadas](/es/concepts/search/searchable-properties/#custom-tags) adaptadas a tu caso de uso para búsquedas y consultas más granulares.

¿Quieres un resumen rápido para empezar? Descarga nuestra [&quot;Guía de referencia rápida para desarrolladores&quot;](/es/pdfs/developer-quick-reference-guide.pdf)
