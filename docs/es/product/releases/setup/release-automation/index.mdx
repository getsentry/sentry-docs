Puedes automatizar la gestión de versiones de Sentry con cualquier plataforma de integración y entrega continuas (CI/CD) o servidor de automatización.

<div id="install-a-repository-integration">
  ## Instalar una integración de repositorio
</div>

La forma más sencilla de notificar a Sentry sobre una versión y proporcionar metadatos de los commits es usando una de nuestras integraciones de repositorio. También admitimos un [proceso manual](/es/product/releases/associate-commits/) para enviarnos tus datos de commits si no quieres que Sentry se conecte a tu repositorio o si usas un proveedor de repositorios o VCS no compatible.

<Alert level="warning" title="Permisos necesarios">
  Para configurar o gestionar una integración, debes ser Owner, Manager o Admin de tu organización de Sentry. Obtén más información sobre los [roles en Sentry](/es/organization/membership/).

  Si vas a vincular un repositorio de GitHub, debes tener permisos de Admin u Owner en el repositorio, y Sentry debe ser una app de GitHub autorizada en la [configuración de tu cuenta de GitHub](https://github.com/settings/applications).
</Alert>

1. Instala la integración correspondiente. Las integraciones disponibles aparecen en **Organization Settings &gt; Integrations**.

2. Haz clic en el botón &quot;Configure&quot; junto a tu instancia.

   ![](./img/releases-repo-integrations.png)

3. En el panel **Repositories**, haz clic en &quot;Add Repository&quot; para añadir los repositorios de los que quieras rastrear commits. La integración enviará a Sentry metadatos (como autores y archivos modificados) sobre cada commit enviado a esos repositorios.

![](./img/releases-repo-add.png)

<Alert>
  Si tienes problemas para añadir la integración, [desconecta](https://sentry.io/account/settings/identities/) y luego [vuelve a conectar](https://sentry.io/account/settings/social/associate/github/) tu identidad de GitHub.
</Alert>
