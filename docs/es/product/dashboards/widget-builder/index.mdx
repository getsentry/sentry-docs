Al agregar un widget o editar uno existente, se abre un panel lateral donde puedes ver el creador de widgets del panel:

![Vista general del creador de widgets.](./img/widget-builder-overview.png)

El creador de widgets incluye varias opciones de configuración que puedes usar para transformar datos y agregar información a tu panel. Estas opciones se presentan como pasos en el creador de widgets, pero no es necesario seguirlos en el orden en que aparecen.

Algunas de estas opciones son estándar y se muestran para todos los widgets, mientras que otras solo se muestran si haces selecciones específicas. A continuación encontrarás una lista de todas las opciones que pueden mostrarse al configurar tu widget.

<div id="dataset">
  ## Conjunto de datos
</div>

En el paso de selección del conjunto de datos, elige qué tipo de datos quieres usar en tu widget. Estos datos se clasifican en cuatro conjuntos de datos: [errors](#error), [transactions](#transactions), [issues](#issues) y [releases](#releases).

<div id="errors">
  ### Errores
</div>

Seleccionar &quot;Errores&quot; te permite consultar y agregar eventos de error del mismo modo que lo harías con una [consulta de Discover](/es/product/explore/discover-queries/). Estos datos se componen de errores que ocurren en tu aplicación, que Sentry usará para agrupar en incidencias, por ejemplo:

* Usuarios más afectados por errores

<div id="transactions">
  ### Transacciones
</div>

Seleccionar &quot;Transacciones&quot; te permite consultar y agregar eventos de transacción del mismo modo que lo harías con una [consulta de Discover](/es/product/explore/discover-queries/). Estos datos se componen de eventos que registran el rendimiento de las operaciones en tu aplicación. Algunos ejemplos de widgets incluyen:

* Seguimiento del rendimiento de un endpoint
* Throughput por país

<div id="issues">
  ### Incidencias
</div>

Elegir &quot;Incidencias (Estados, Asignación, Tiempo, etc.)&quot; te permite consultar propiedades de incidencias, como `is:unresolved`, o propiedades de eventos como `error.handled:0`, y ordenar por campos de incidencias como `First Seen`. Al buscar por propiedades de eventos, la búsqueda devolverá cualquier incidencia que tenga uno o más eventos que coincidan con los filtros de eventos proporcionados.
Elige este conjunto de datos si quieres personalizar una lista de incidencias en tu panel. Algunos ejemplos de widgets incluyen:

* Incidencias asignadas a tu equipo
* Incidencias que ocurren con mayor frecuencia
* El conjunto de datos &quot;Incidencias&quot; solo está disponible en widgets con visualización de tabla y se deshabilita si tienes seleccionadas otras visualizaciones.

<div id="releases">
  ### Lanzamientos
</div>

Seleccionar &quot;Lanzamientos (sesiones, tasas de fallos)&quot; te permite consultar propiedades específicas de una versión de tu aplicación, principalmente datos de sesiones. Elige este conjunto de datos si quieres mostrar sesiones correctas, con errores y con fallos, así como tasas de fallos para tus lanzamientos o proyectos en tu panel. Algunos widgets de ejemplo que usan este conjunto de datos incluyen:

* Tasas de fallos de tu último lanzamiento
* Recuento de sesiones entre lanzamientos

<div id="visualization-type">
  ## Tipo de visualización
</div>

En este paso, puedes elegir entre cuatro tipos de visualización:

* [Gráfico de área / Gráfico de barras / Gráfico de líneas](#area-chart--bar-chart--line-chart)
* [Tabla](#table)
* [Número destacado](#big-number)

<div id="area-chart-bar-chart-line-chart">
  ### Gráfico de área / Gráfico de barras / Gráfico de líneas
</div>

Las visualizaciones de gráficos de área, barras y líneas te permiten representar una o varias funciones de agregación como una serie temporal.

![Visualizaciones de series temporales.](./img/widget-builder-time-series-visualizations.png)

Cada tipo de gráfico te ayuda a resolver problemas distintos:

* Los gráficos de área son ideales para mostrar resultados acumulativos o cuando quieres desglosar información. Un ejemplo sería “recuento de errores por cada navegador principal”.
* Los gráficos de barras agrupan los resultados por día, lo que los hace adecuados para resúmenes diarios o para ver el “panorama general”. Un ejemplo sería “recuento de errores por día”.
* Los gráficos de líneas son una visualización de uso general. Asegúrate de que las unidades del eje Y coincidan si agregas varias series de datos. Mezclar recuento con duración no dará resultados útiles.

<div id="table">
  ### Tabla
</div>

Las tablas muestran los resultados de una [consulta de Discover](/es/product/explore/discover-queries/). Esta visualización es ideal para presentar campos clave y agregados relacionados. Por ejemplo, “percentiles de duración de los nombres de transacción más visitados”. También puedes mostrar datos de [issues](/es/product/issues/) en una tabla usando el [selector de conjuntos de datos](/es/product/dashboards/widget-builder/#choose-your-dataset).

<div id="big-number">
  ### Número grande
</div>

Una visualización de número grande muestra el valor actual de una única función o [ecuación](/es/product/explore/discover-queries/query-builder/query-equations/). Esta visualización es ideal para agregaciones de alto nivel. Un ejemplo sería “p95 de todas las transacciones”.

<div id="columns">
  ## Columnas
</div>

La opción para configurar columnas solo está disponible para [visualizaciones de tabla](/es/product/dashboards/widget-builder/#table).

De forma predeterminada, la primera columna ya está configurada, pero puedes modificarla. Para agregar más columnas, que apilan eventos, agrega [funciones](/es/product/explore/discover-queries/query-builder/#stacking-functions) (`f(x)`) que acepten parámetros. [Las columnas de campos y etiquetas](/es/concepts/search/searchable-properties/#event-properties) te ayudarán a ver más detalles sobre los eventos (por ejemplo, `title` para ver la descripción de un issue o `users` para ver cuántos usuarios se ven afectados).

También puedes agregar ecuaciones a tu consulta usando las columnas de la tabla como variables de la ecuación; los resultados se mostrarán en la tabla de resultados de la consulta. Obtén más información en [Agregar ecuaciones de consulta](/es/product/explore/discover-queries/query-builder/query-equations/).

Junto a cada columna y ecuación, tienes la opción de establecer un alias que reemplazará el nombre de la columna en el encabezado de la tabla.

Si agregas dos o más columnas, puedes cambiar el orden en que se muestran en la tabla arrastrándolas y soltándolas:

![Arrastra y suelta la columna.](./img/widget-builder-drag-and-drop-column.png)

<div id="visualize">
  ## Visualizar
</div>

La opción para definir agregaciones para la visualización está disponible en todas las visualizaciones excepto en [table](/es/product/dashboards/widget-builder/#table). En el caso de las [visualizaciones de series temporales](/es/product/dashboards/widget-builder/#area-chart--bar-chart--line-chart), podrás trazar hasta tres valores que pueden ser funciones o ecuaciones. Si las unidades de tus selecciones entran en conflicto, el eje y del gráfico se etiquetará usando las unidades de la primera selección.

También puedes crear una ecuación haciendo clic en “Add Equation”. Al hacerlo, se mostrará un cuadro de texto donde podrás introducir tu ecuación. Las funciones que hayas agregado previamente en este paso aparecerán para autocompletar, pero también puedes añadir otras funciones.

<div id="filter">
  ## Filtro
</div>

Usa este campo para filtrar tus resultados por las propiedades de búsqueda de tu [selección de dataset](/es/product/dashboards/widget-builder/#choose-your-dataset). Para obtener más información sobre qué propiedades se aplican a cada dataset, consulta nuestra [documentación completa sobre propiedades de búsqueda](/es/concepts/search/searchable-properties/#properties-table).

En las [visualizaciones de series temporales](/es/product/dashboards/widget-builder/#area-chart--bar-chart--line-chart) puedes añadir hasta tres consultas para filtrar tus resultados, cada una con sus propias condiciones de búsqueda, para comparar resultados. Cada consulta generará una nueva serie en tu gráfico para cada eje y. Para que sea más fácil diferenciarlas en la leyenda y el tooltip, puedes añadir un alias de leyenda que se agregará como prefijo a las etiquetas, como en el ejemplo a continuación:

![Filtra tus resultados.](./img/widget-builder-filter-your-results-example.png)

<div id="group-by">
  ## Agrupar por
</div>

La opción de agrupar resultados solo está disponible para [visualizaciones de series temporales](/es/product/dashboards/widget-builder/#area-chart--bar-chart--line-chart). Puedes agrupar tus series temporales por diferentes campos o etiquetas, como proyecto, entorno o transacción. Puedes seleccionar hasta 20 campos para agrupar tus datos.

Si se agregan dos o más grupos, se mostrará un elemento que puedes arrastrar para ordenar los campos, pero ese orden no afecta la forma en que se muestran los datos en la visualización:

![Arrastrar y soltar para agrupar por.](./img/widget-builder-drag-and-drop-group-by.png)

<div id="sort-by">
  ## Ordenar por
</div>

La opción de ordenar por una columna solo está disponible para [visualizaciones de tabla](/es/product/dashboards/widget-builder/#table). Elige una de las columnas de tu tabla para ordenar y define si quieres ordenar de mayor a menor o de menor a mayor.

La opción de ordenar en gráficos de series temporales solo está disponible si aplicaste un agrupamiento a tus datos (en [“Agrupar por”](/es/product/dashboards/widget-builder/#group-by)). El orden se aplica a los grupos subyacentes en tus resultados y, por lo tanto, afecta las series que aparecen en tu gráfico. Puedes seleccionar los campos que usaste para agrupar, elegir ecuaciones definidas en los ejes y, construir cualquier función de forma similar a como elegirías una función para graficar, o crear una ecuación personalizada.

Si seleccionas “Ecuación personalizada”, aparece un campo adicional donde puedes ingresar una ecuación de tu elección que se aplicará a la consulta:

![Campo de ecuación personalizada.](./img/widget-builder-custom-equation-field.png)

<div id="limit">
  ## Límite
</div>

La opción para limitar tus resultados solo está disponible para [visualizaciones de series temporales](/es/product/dashboards/widget-builder/#area-chart--bar-chart--line-chart) en las que hayas aplicado un agrupamiento. Según tu configuración (por ejemplo, múltiples ejes Y o múltiples consultas), los resultados del gráfico podrían volverse difíciles de manejar, por lo que se impone un límite. Puedes cambiar este límite seleccionándolo en el menú desplegable. Nuestro objetivo es mostrar como máximo 10 series (es decir, líneas en un gráfico de líneas o barras en un gráfico de barras) cuando se seleccionan múltiples ejes Y o consultas.

<div id="set-thresholds">
  ## Establecer umbrales
</div>

La opción para establecer umbrales solo está disponible para las [visualizaciones de número grande](/es/product/dashboards/widget-builder/#big-number). Los umbrales ayudan a identificar widgets problemáticos mediante indicadores de color. Al pasar el cursor sobre los indicadores de color en el panel se muestran los umbrales asociados a un widget. Se pueden agregar unidades a los umbrales para métricas de tipo duración o de tasa. Un ejemplo de métrica con unidades sería “p95 de todas las transacciones”.

![Paso para establecer umbrales](./img/widget-builder-thresholds.png)
![Indicador de color del widget al pasar el cursor](./img/widget-builder-thresholds-hover.png)

<div id="widget-title">
  ## Título del widget
</div>

De forma predeterminada, todos los widgets nuevos se llaman “Custom Widget”, pero puedes editar el título haciendo clic en el texto. Te recomendamos actualizar el título.

<div id="extracted-metrics">
  ## Métricas extraídas
</div>

<Include name="feature-available-for-user-group-early-adopter.mdx" />

Si estás creando un widget con datos de transacciones y quieres usar filtros similares a eventos (p. ej., `transaction.duration:>100ms`) o etiquetas personalizadas que añadas a tus transacciones, recopilaremos métricas de forma periódica para mostrarte recuentos y tasas precisos.

![Información sobre métricas extraídas](/product/dashboards/widget-builder/widget-builder-extracted-metrics-tooltip.png)

No todos los widgets son compatibles con la extracción de métricas. En los siguientes casos, tu widget seguirá usando datos basados en eventos:

* Ecuaciones — Al usar ecuaciones se mostrarán datos de eventos en lugar de métricas.
* Widgets que muestran errores y transacciones simultáneamente — La extracción solo aplica a datos de transacciones. Debes usar el filtro `event.type:transaction` junto con tus etiquetas personalizadas para obtener métricas extraídas.
