La biblioteca de widgets contiene una colección de widgets prediseñados que puedes añadir a tus [paneles personalizados](/es/product/dashboards/customize-dashboards/). Puedes acceder a la biblioteca de widgets haciendo clic en el botón &quot;Add Widget&quot; y seleccionando &quot;From Widget Library&quot; en el panel.

La biblioteca incluye los siguientes widgets:

* **Distribución de duración**: Un gráfico de múltiples series de datos que muestra la distribución de la duración de las transacciones en varios percentiles (p50, p75 y p95)
* **Transacciones de alto rendimiento**: Las cinco transacciones principales con el mayor número de eventos, representadas en un gráfico de áreas
* **LCP por país**: Tabla que muestra el p75 de los tiempos de carga de página por país
* **Usuarios insatisfechos**: El número total de usuarios únicos que han experimentado transacciones lentas (duración de la transacción mayor a 1200 ms)
* **Transacciones lentas vs. rápidas**: Gráfico de barras que compara el porcentaje de transacciones por encima de 300 ms (lentas) y por debajo de 300 ms (rápidas)
* **Issues para revisión**: Una tabla de issues sin resolver para revisión, ordenadas por las vistas más recientes
* **Tipos de errores no controlados más frecuentes**: Los errores no controlados encontrados con mayor frecuencia
* **Usuarios afectados por errores**: Una comparación entre el número total de errores y la cantidad de usuarios únicos afectados por los errores
* **Tasas de bloqueo en versiones recientes**: Porcentaje de sesiones con bloqueo en tus versiones recientes
* **Salud de sesiones**: El número total de sesiones anómalas, con bloqueo, con error y saludables

Puedes cambiar el título, las consultas, los campos, los tipos de visualización, el orden de clasificación y otros campos de estos widgets prediseñados para adaptarlos a tu caso de uso haciendo clic en el menú contextual del widget y seleccionando &quot;Edit Widget&quot;.

<div id="example-use-cases">
  ## Ejemplos de casos de uso
</div>

<div id="performance-across-release-versions">
  ### Rendimiento entre versiones
</div>

Para monitorear cómo se desempeña tu app en una versión específica, puedes modificar el widget &quot;Duration Distribution&quot; agregando la versión a la condición de búsqueda:

* Condición de búsqueda 1: `release.version:{version}`

Para comparar el rendimiento con otra versión, puedes agregar otra consulta haciendo clic en el Button &quot;Add Query&quot; y buscar una versión diferente:

* Condición de búsqueda 2: `release.version:{another_version}`

También puedes comparar el rendimiento antes y después de una versión específica con las siguientes condiciones:

* Condición de búsqueda 1: `release.version:<{version}`
* Condición de búsqueda 2: `release.version:>={version}`

<div id="response-thresholds">
  ### Umbrales de respuesta
</div>

El widget &quot;Duration Distribution&quot; muestra la distribución de las duraciones de las transacciones. Esta visualización se puede modificar para indicar si las duraciones están dentro de ciertos umbrales objetivo y clasificarlas como transacciones &quot;satisfactorias&quot;, &quot;tolerables&quot; o &quot;frustrantes&quot;. Para crear este tipo de visualización, cambia &quot;Visualization Display&quot; a &quot;Area Chart&quot;. Los gráficos de área apilan resultados y son más adecuados para resultados acumulativos:

* Visualization Display: `Area Chart`

Configura uno de los valores del eje Y como `count()` y elimina los otros dos ejes:

* Y-Axis: `count()`

En la primera consulta, agrega la condición de búsqueda para la duración satisfactoria de la transacción (en este ejemplo se usan 300 ms como umbral de respuesta satisfactorio):

* Search condition 1: `transaction.duration:<300`

Agrega otra consulta para la condición tolerable haciendo clic en el botón &quot;Add Query&quot;. En este ejemplo, usamos la definición de [Apdex](/es/product/insights/overview/metrics/#apdex), donde los tiempos de respuesta tolerables están entre el umbral satisfactorio y cuatro veces el umbral satisfactorio:

* Search condition 2: `transaction.duration:<1200 AND transaction.duration:>=300`

Por último, agrega una tercera consulta para la duración de transacción frustrante:

* Search condition 3: `transaction.duration:>=1200`

El gráfico ahora muestra recuentos acumulativos en distintos umbrales de tiempo de respuesta.

<div id="my-top-issues">
  ### Mis principales incidencias
</div>

Puedes hacer un seguimiento del impacto de las incidencias asignadas a ti o a tu equipo contando los eventos de error asociados a cada incidencia. Modifica el widget &quot;Issues For Review&quot; actualizando la condición de búsqueda para filtrar por asignación:

* Condición de búsqueda: `is:unresolved is:for_review assigned_or_suggested:me`

Actualiza &quot;Columns&quot; para agregar `project` y así poder ver el proyecto asociado a cada incidencia, y cambia &quot;Sort by&quot; a `Events` para que el widget muestre las incidencias con más eventos de error en orden descendente:

* Columnas: `issue, assignee, events, title, project`
* Ordenar por: `Events`

<div id="priority-issues">
  ### Problemas prioritarios
</div>

El [conjunto de datos de issues](/es/product/dashboards/widget-builder/#issues) es único porque puedes agregar un [token](/es/concepts/search/#syntax) libre a tu condición de búsqueda que filtrará los títulos de los issues que contengan ese token. Por ejemplo, puedes buscar `integrity` para filtrar todos los títulos de issues que contengan `integrity`. Actualiza la condición de búsqueda con un token de tu elección:

* Condición de búsqueda: `is:unresolved is:for_review {token}`

Actualiza &quot;Columns&quot; para agregar `links` y así poder ver cualquier enlace externo relacionado con el issue, como issues de GitHub o Jira. Actualiza &quot;Sort by&quot; a `Priority` para que el widget muestre issues que han estado en tendencia al alza recientemente:

* Columns: `issue, assignee, events, title, links`
* Sort by: `Priority`

<div id="release-health">
  ### Salud de la versión
</div>

Para supervisar la salud de tus versiones a lo largo del tiempo, puedes modificar el widget &quot;Session Health&quot; de la siguiente manera:

* Tipo de visualización: `Area Chart`

Añade filtros de búsqueda para acotar los datos a una versión específica:

* Condición de búsqueda 1: `release:{version}`
