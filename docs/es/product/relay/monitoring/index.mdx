<div id="logging">
  ## Registros
</div>

Relay genera registros en el flujo de error estándar (`stderr`), con nivel `INFO` de forma predeterminada.
Por ejemplo, después de iniciar Relay, podrías ver una salida como esta:

```
INFO  relay::setup > iniciando relay desde la carpeta de configuración .relay
INFO  relay::setup >   modo de relay: administrado
INFO  relay::setup >   id del relay: cde0d72e-0c4e-4550-a934-c1867d8a177c
INFO  relay::setup >   nivel de log: INFO
```

Este ejemplo muestra los mensajes con el nivel de registro predeterminado (`INFO`), que puedes ajustar para mostrar más o menos información. Para obtener detalles sobre cómo configurar el registro, consulta la sección [Logging](/es/product/relay/options/#logging) en la página de opciones.

<div id="error-reporting">
  ## Notificación de errores
</div>

De forma predeterminada, Relay registra los errores en el registrador configurado. Puedes habilitar la notificación de errores para tu proyecto en Sentry en el archivo de configuración de Relay:

```yaml
sentry:
  enabled: true
  dsn: <tu_dsn>
```

Puedes encontrar más información sobre las opciones disponibles y su significado [en la página de opciones](/es/product/relay/options/#internal-error-reporting).

<div id="health-checks">
  ## Comprobaciones de estado
</div>

Relay proporciona dos URL para comprobar el estado del sistema:

* `GET /api/relay/healthcheck/live/`: Verifica si Relay se está ejecutando y está a la escucha de solicitudes HTTP.
* `GET /api/relay/healthcheck/ready/`: Verifica si Relay está autenticado con el upstream y funcionando con normalidad.

Ambos endpoints devuelven una respuesta *200 OK* si todo está correcto:

```json
{
  "is_healthy": true
}
```

<div id="metrics">
  ## Métricas
</div>

Puedes enviar estadísticas a un servidor StatsD configurando la clave `metrics.statsd` con una tupla `ip:puerto`.

<div id="example-configuration">
  ### Ejemplo de configuración
</div>

```yaml
metrics:
  # Endpoint de tu servidor StatsD
  statsd: 127.0.0.1:8126
  # Antepone esta cadena a todos los nombres de métricas
  prefix: mycompany.relay
```

Las opciones para configurar el envío de métricas están documentadas en la [página de opciones](/es/product/relay/options/#statsd-metrics).
