title: Modos de Relay
description: "Relay puede funcionar en distintos modos según las necesidades de tu organización."

---

Relay puede operar en uno de varios modos principales, y es fundamental entenderlos de antemano si vas a configurar el servidor Relay en lugar de usar la configuración predeterminada.

El modo se define en el archivo de configuración, que contiene el campo `relay.mode`. Este campo especifica el modo en el que se ejecutará Relay: `managed` o `proxy`. El modo de Relay determina si Relay obtiene o no la configuración del proyecto para los eventos.

En Sentry, el procesamiento de eventos se configura según los ajustes tanto del proyecto como de la organización. Algunos ajustes, como los controles de privacidad, se establecen a nivel de organización y luego se heredan en todos los proyectos de esa organización; otros ajustes se especifican por proyecto. En el caso de Relay, los eventos se procesan de acuerdo con la configuración heredada del proyecto al que se envía el evento.

La configuración de Relay se actualiza a intervalos regulares mediante sondeos a Sentry. Sentry no necesita tener visibilidad directa de tus Relays. Consulta [Opciones de configuración](/es/product/relay/options/) para conocer la configuración de intervalos, tiempos de espera y reintentos.

<div id="managed-mode">
  ## Modo gestionado
</div>

Gestionado es el modo predeterminado de Relay. Como la configuración se obtiene de Sentry, en este modo se requiere autenticación. Si la autenticación falla, Relay no aceptará eventos.

A medida que Relay reciba eventos de tus aplicaciones, solicitará a Sentry la configuración del proyecto para procesarlos. Si Sentry no puede proporcionar la configuración para un proyecto en particular, se descartarán todos los datos de ese proyecto.

Para activar el modo gestionado, establece esta configuración:

```yaml
relay:
  mode: managed
```

<div id="proxy-mode">
  ## Modo proxy
</div>

El modo proxy reenvía todos los eventos con un procesamiento mínimo y no recibe ninguna configuración del proyecto de Sentry.

<Alert title="Limitación de velocidad">
  La limitación de velocidad sigue aplicándose en el modo `proxy` para todos los proyectos.
</Alert>

Para activar el modo proxy, establece esta configuración:

```yaml
relay:
  mode: proxy
```

<div id="static-mode">
  ## Modo estático
</div>

El modo estático quedó obsoleto a partir de la versión 25.9.0 de Relay. Usa en su lugar el modo `managed` o `proxy`. Al cambiar al modo `managed`, no olvides migrar las reglas de depuración de datos de tu directorio de configuración configurándolas en la interfaz de Sentry (/security-legal-pii/scrubbing/).
