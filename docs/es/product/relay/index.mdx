Sentry Relay ofrece seguridad de datos de nivel empresarial al proporcionar un servicio independiente que actúa como una capa intermedia entre tu aplicación y sentry.io.

Relay está diseñado específicamente para:

* Depurar información de identificación personal (PII) de forma centralizada antes de enviarla a Sentry
* Mejorar los tiempos de respuesta de eventos en regiones con poco ancho de banda o conectividad limitada
* Actuar como un proxy opaco para organizaciones que restringen toda la comunicación HTTP a un dominio personalizado

<div id="use-cases-for-relay">
  ## Casos de uso de Relay
</div>

Relay está diseñado para ayudar a organizaciones con requisitos específicos de seguridad empresarial en el depurado de datos de información de identificación personal (PII), los tiempos de respuesta y la gestión de dominios a nivel empresarial.

<div id="pii-data-scrubbing">
  ### Depuración de datos PII
</div>

Sentry ya depura PII en dos lugares:

1. En el SDK antes de enviar el evento
2. Al llegar a la infraestructura de Sentry

Relay añade una tercera opción que depura los datos en un punto central antes de enviarlos a Sentry.

Para elegir el lugar adecuado para la depuración de datos, ten en cuenta:

* Si prefieres configurar la depuración de datos en un punto central, puedes dejar que Sentry se encargue. Cuando llega un evento, Sentry aplicará inmediatamente la [depuración del lado del servidor](/es/security-legal-pii/scrubbing/server-side-scrubbing/).

* Si no puedes enviar PII fuera de tu infraestructura, pero aun así prefieres configurar la depuración de datos de forma centralizada, configura tu SDK para enviar eventos a Relay, que utiliza los ajustes de privacidad definidos en Sentry y depura la PII antes de reenviar cualquier dato.

* Si debes cumplir requisitos estrictos de privacidad de datos, puedes configurar los SDK para depurar PII usando los hooks [`beforeSend` y `beforeSendTransaction`](/es/platform-redirect/?next=/configuration/options/%23hooks), lo que evitará que se recopilen datos en el dispositivo. Esto puede requerir replicar la misma lógica en todas tus aplicaciones y puede afectar al rendimiento.

<div id="response-time">
  ### Tiempo de respuesta
</div>

Relay responde muy rápido a las solicitudes. Instalar Relay cerca de tu infraestructura mejorará aún más el tiempo de respuesta al enviar eventos. Este enfoque de instalación puede reducir en especial el tiempo de ida y vuelta en ubicaciones remotas.

<div id="enterprise-domain-management">
  ### Gestión de dominios empresariales
</div>

De forma predeterminada, los SDK deben configurarse con un Data Source Name (DSN) que apunte a `sentry.io`. Si necesitas restringir toda la comunicación HTTP a un nombre de dominio personalizado, Relay puede actuar como un proxy opaco que reenvía los eventos de forma fiable a Sentry.
