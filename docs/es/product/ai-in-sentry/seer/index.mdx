Seer es el agente de depuración con IA de Sentry. Utiliza el contexto detallado de Sentry (detalles de incidencias, datos de trazas, registros y perfiles) para ayudarte a diagnosticar y solucionar errores y problemas de rendimiento más rápidamente.

<div id="seer-capabilities">
  ## Capacidades de Seer
</div>

Seer ofrece tres capacidades principales para mejorar tu depuración:

* [**Solución de issues**](#issue-fix): Un agente colaborativo experto en análisis de causa raíz y resolución de issues
* [**Escaneo de issues**](#issue-scan): Escanea automáticamente los issues a medida que llegan, aportando contexto adicional a tus alertas y automatizando el triage
* [**Automatización**](#automation): Combina la solución y el escaneo de issues en un flujo de trabajo potente: identifica issues accionables en tiempo real y genera correcciones de alta calidad de forma automática

<div id="issue-fix">
  ### Corrección de incidencias
</div>

Seer combina todo el contexto relevante de tu código con los datos de telemetría de Sentry para ofrecer una experiencia de depuración de incidencias de primer nivel. El flujo de trabajo consta de tres pasos:

1. Determina la causa raíz de la incidencia usando el contexto de Sentry y tu base de código
2. Identifica una solución a partir del análisis
3. Genera un parche de código que implemente la solución y abre un PR

Para ver una guía completa del producto, consulta [corrección de incidencias](./issue-fix/).

<div id="issue-scan">
  ### Análisis de incidencias
</div>

Seer puede analizar automáticamente las incidencias a medida que Sentry las ingiere. Esta revisión inicial se usa para:

* Generar una puntuación de &quot;actionability&quot; (capacidad de acción), que indica la probabilidad de que una incidencia concreta pueda resolverse con un cambio de código
* Enriquecer las alertas de incidencias con un análisis útil sobre el problema
* Añadir una estimación inicial de cuál podría ser el problema potencial en la página de detalles de la incidencia

La puntuación de actionability se representa en el feed de incidencias con un ícono de Seer, que señala las incidencias que probablemente sean arreglos sencillos.

<div id="automation">
  ### Automatización
</div>

Puedes usar la automatización para activar análisis y correcciones de incidencias. Hay tres opciones de automatización que puedes configurar:

1. **Automatizar análisis de incidencias**: Habilita la función de análisis de incidencias de Seer.
2. **Automatizar correcciones de incidencias**: Te permite ajustar el nivel de automatización que emplea Seer. Por ejemplo, al establecerlo en «Altamente accionables y superiores», se activarán automáticamente las correcciones para todas tus incidencias altamente accionables.
3. **Punto de detención para correcciones automáticas**: Seer te permite definir hasta dónde llevar la automatización. De forma predeterminada, se detiene en el paso de solución, pero puedes hacer que llegue hasta cambios de código o incluso generar PR automáticamente.

<div id="what-seer-uses">
  ## Qué utiliza Seer
</div>

Seer es un potente agente de depuración, con acceso a una amplia variedad de fuentes de datos y herramientas. Mientras depura problemas, puede examinar:

* **Contexto del issue**: Mensajes de error, trazas de pila y metadatos de eventos de tus [Issues](/es/product/issues/)
* **Datos de trazas**: [Traces](/es/concepts/key-terms/tracing/#whats-a-trace) distribuidas e información de spans
* **Registros**: [Logs](/es/product/explore/logs/) estructurados de tu aplicación (beta)
* **Tu base de código**: Código relevante de repositorios de GitHub vinculados, con soporte para múltiples repositorios en servicios distribuidos
* **Datos de rendimiento**: Perfiles y métricas de rendimiento
* **Comentarios interactivos**: Tu aporte y guía durante el proceso

<div id="getting-started-with-seer">
  ## Introducción a Seer
</div>

Para empezar a usar las funciones de Seer en tus proyectos con Sentry habilitado, sigue estos pasos:

1. **Habilita las funciones de IA generativa**: Asegúrate de que “Show Generative AI Features” esté activado en la [configuración de tu organización](https://sentry.io/orgredirect/organizations/:orgslug/settings/#hideAiFeatures)
2. **Configura la integración con GitHub**: Para poder corregir issues, [conecta tus repositorios de GitHub](/es/organization/integrations/source-code-mgmt/github/).
   * *Opcional:* Para crear Pull Requests con Seer, [configura la aplicación de GitHub de Seer](/es/organization/integrations/source-code-mgmt/github/#installing-the-seer-github-app).
3. **Configura los ajustes**: Personaliza el comportamiento de Seer en la configuración de tu proyecto, en `Seer Automation`. Seer está habilitado de forma predeterminada con estos ajustes:

   * **Análisis automáticos de issues**: Activado
   * **Correcciones automáticas de issues**: Activado para issues con alta capacidad de acción y superiores

   > **Nota**: Puedes monitorear tu uso de Seer en la [pestaña Stats &amp; Usage](https://sentry.io/settings/stats/) y obtener más información sobre cómo gestionar tu presupuesto de Seer [aquí](/es/pricing/quotas/manage-seer-budget/).
4. **Empieza a usarlo**: Busca opciones de Seer en las páginas de detalles de tus issues

Para empezar a usar Seer para revisiones de código con IA, [configura la aplicación de GitHub de Seer](/es/organization/integrations/source-code-mgmt/github/#installing-the-seer-github-app).

<div id="privacy-and-security">
  ## Privacidad y seguridad
</div>

Sentry ofrece sólidas garantías para la privacidad y la seguridad de tus datos. En resumen:

* Sentry no entrena modelos de IA generativa con tus datos de forma predeterminada ni sin tu permiso.
* El contenido generado por IA se muestra solo a ti y a otros usuarios autorizados de tu cuenta.

Puedes obtener más información sobre nuestras prácticas de privacidad de datos [en la documentación de seguridad y asuntos legales](/es/security-legal-pii/security/ai-ml-policy/#use-of-identifying-data-for-generative-ai-features).
