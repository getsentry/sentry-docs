Sentry ofrece varias opciones de configuración para crear una alerta de disponibilidad según las necesidades de tu organización, como se detalla a continuación.

<div id="1-environment">
  ## 1. Entorno
</div>

Primero, especifica a qué <PlatformLink to="/configuration/environments/">entorno</PlatformLink> pertenece esta regla de alerta. Cualquier [incidencia de tiempo de actividad](/es/product/issues/issue-details/uptime-issues/) que se cree a partir de esta regla de alerta se asignará al entorno que indiques.

El menú desplegable “Entorno” muestra los mismos entornos disponibles en tu proyecto, excluyendo los ocultos.

<div id="2-project">
  ## 2. Proyecto
</div>

Especifica el proyecto asociado a tu regla de alerta. Cualquier [incidencia de disponibilidad](/es/product/issues/issue-details/uptime-issues/) que se cree aparecerá en este proyecto.

<div id="3-request-configuration">
  ## 3. Configuración de la solicitud
</div>

![Configuración de la solicitud de alerta de Uptime](./img/uptime-alert-request-config.png)

Configura cómo Sentry realiza las comprobaciones de disponibilidad HTTP estableciendo las siguientes opciones:

* **Intervalo**: El tiempo entre cada comprobación de disponibilidad. Opciones: `1 minuto`, `5 minutos`, `10 minutos`, `20 minutos`, `30 minutos` y `1 hora`.
* **Tiempo de espera**: El tiempo máximo que Sentry espera una respuesta antes de considerar la solicitud fallida (hasta 30 segundos).
* **URL**: La URL de destino para la comprobación de disponibilidad.
* **Método**: El método HTTP utilizado (`GET`, `POST`, `HEAD`, `PUT`, `DELETE`, `PATCH` u `OPTIONS`).
* **Encabezados**: Encabezados personalizados incluidos en la solicitud.
* **Cuerpo**: La carga útil de la solicitud, disponible para los métodos `POST`, `PUT` y `PATCH`.
* **Permitir muestreo**: Activa el muestreo de spans para solicitudes mediante el Sentry SDK. Consulta la [trazabilidad distribuida con uptime](/es/product/uptime-monitoring/uptime-tracing/) para más detalles.

<Alert level="warning">
  Al agregar encabezados HTTP, ten cuidado de no incluir datos sensibles, como tokens de API o información personal, para evitar exposiciones o almacenamiento no deseados.
</Alert>

![Solicitud esperada de alerta de Uptime](./img/uptime-alert-expected-check-request.png)

Debajo de la configuración de la solicitud encontrarás un ejemplo de la solicitud esperada que Sentry enviará a la URL especificada, incluidos el método, los encabezados y el cuerpo. Sentry agrega automáticamente los encabezados `User-Agent` y `Sentry-Trace`.

Notas adicionales:

* Incluye un encabezado `Content-Type` si lo requiere la URL de destino. Por ejemplo, una carga JSON debe tener `Content-Type: application/json`.
* El intervalo seleccionado afecta la velocidad de detección de caídas. Sentry crea una incidencia de disponibilidad tras tres fallos consecutivos. Por ejemplo, con un intervalo de 5 minutos, la caída se detecta como mínimo 15 minutos después del primer fallo. Más información sobre los [criterios de comprobación de disponibilidad](/es/product/uptime-monitoring/#uptime-check-criteria).
* Si la URL especificada está detrás de un firewall, asegúrate de que el Uptime Bot de Sentry pueda realizar solicitudes hacia ella. Más información sobre la [configuración del firewall con el monitoreo de disponibilidad](/es/product/uptime-monitoring/troubleshooting/#verify-firewall-configuration).
* Sentry Uptime Tracing agrega automáticamente un encabezado `sentry-trace` a las solicitudes salientes para la trazabilidad distribuida. [Más información](/es/product/uptime-monitoring/uptime-tracing/).

<div id="4-alert-name">
  ## 4. Nombre de la alerta
</div>

Asigna a tu alerta un nombre descriptivo, por ejemplo, «Landing Page» o «Contact Page».

<div id="5-ownership">
  ## 5. Responsables
</div>

Asigna un equipo o un miembro del equipo para administrar la alerta. Si no se asigna ningún equipo, cualquier usuario puede modificarla. Los [Uptime issues](/es/product/issues/issue-details/uptime-issues/) creados a partir de esta regla de alerta se asignarán al equipo o miembro del equipo especificado.
