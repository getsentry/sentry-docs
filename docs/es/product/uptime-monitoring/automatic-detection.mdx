---
title: Detección automática
sidebar_order: 51
description: "Aprende cómo funciona la detección automática del monitoreo de disponibilidad."
---

Para detectar automáticamente alertas de disponibilidad, analizamos todas las URL presentes en los datos de errores capturados de tu proyecto para encontrar el nombre de host que aparece con mayor frecuencia. Esto ayuda a garantizar que tu nombre de host más crítico esté monitorizado de forma continua, mejorando la confiabilidad y disponibilidad de tus servicios web. Luego creamos una alerta de disponibilidad si cumple nuestros [criterios de verificación de disponibilidad](/es/product/uptime-monitoring/#uptime-check-criteria). Las verificaciones automáticas de disponibilidad están configuradas para ejecutarse una vez por minuto como solicitudes GET.

La creación automática de monitores de disponibilidad solo ocurre si no hay otros monitores de disponibilidad ya configurados.

Para evitar alertas inestables, el nombre de host pasa por un “período de incorporación” de tres días. Durante este período, enviamos solicitudes HTTP al nombre de host cada hora. Si la solicitud falla tres veces o más, se descartará el nombre de host y se reevaluará después de siete días.

<Alert>
  Sentry ejecutará verificaciones de disponibilidad contra la ruta raíz del nombre de host de las URL más frecuentes. Por ejemplo, si la URL más frecuente en tus eventos es `GET https://www.example.com/docs/introduction`, la verificación será `GET https://www.example.com/`.
</Alert>

Una vez creada una alerta de disponibilidad detectada automáticamente, podrás [personalizar su configuración](/es/product/alerts/create-alerts/uptime-alert-config/), incluidos el método de la solicitud HTTP, los encabezados y el cuerpo de la solicitud.

<div id="disabling-automatic-detection">
  ## Desactivar la detección automática
</div>

Si prefieres no usar alertas de disponibilidad detectadas automáticamente, puedes eliminarlas directamente en tu [página de alertas](https://sentry.io/orgredirect/organizations/:orgslug/alerts/rules).

**Una vez eliminadas, estas alertas de disponibilidad detectadas automáticamente no se volverán a crear en el futuro.**
