<div id="verify-firewall-configuration">
  ## Verifica la configuración del firewall
</div>

Algunas plataformas de alojamiento pueden bloquear las solicitudes entrantes del Uptime Bot de Sentry y activar alertas de disponibilidad por error. Te recomendamos verificar la configuración de tu firewall para asegurarte de que se permitan las solicitudes entrantes de Sentry.

Si necesitas configurar la lista de permitidos de tu firewall para incluir el Uptime Bot de Sentry, te recomendamos validar mediante nuestro `User-Agent`, ya que nuestras direcciones IP pueden cambiar sin previo aviso.

<div id="user-agent">
  ### Agente de usuario
</div>

Nuestras comprobaciones de disponibilidad usan el siguiente `User-Agent`:

```
SentryUptimeBot/1.0 (+http://docs.sentry.io/product/uptime-monitoring/)
```

<div id="ip-addresses">
  ### Direcciones IP
</div>

Consulta [rangos de IP](/es/security-legal-pii/security/ip-ranges/#uptime-monitoring) para ver la lista completa de direcciones IP utilizadas en las comprobaciones de disponibilidad.

<div id="verify-that-issue-alerts-match-downtime-issues">
  ## Verifica que las alertas de incidencias coincidan con los periodos de inactividad
</div>

Las alertas de disponibilidad generan incidencias de inactividad. Si no recibes notificaciones cuando se detectan periodos de inactividad, asegúrate de haber [configurado correctamente una alerta de incidencias](/es/product/alerts/create-alerts/issue-alert-config/) con la categoría de incidencia «uptime».
