Al finalizar esta guía, podrás crear y configurar proyectos en Sentry para obtener información detallada sobre los problemas, incluidas las líneas exactas de código afectadas, el entorno en el que se produjeron y las versiones de las que provienen. Además, te guiaremos por las integraciones esenciales de Sentry y la configuración de SSO/SCIM. Si te quedas atascado en algún punto, [pregunta a Sentry AI](https://sentry.zendesk.com/hc/en-us?askAI=true) o únete a nuestra comunidad en [Discord](https://discord.gg/sentry).

<div id="step-1-initial-project-creation-and-configuration">
  ## Paso 1: Creación y configuración inicial del proyecto
</div>

Al finalizar este paso, habrás habilitado algunas funciones, enviado datos a Sentry y agregado metadatos importantes como entornos y releases para ayudarte a filtrar.

1. El flujo de creación de proyectos dentro de Sentry te permite habilitar Tracing y Session Replay. Revisa este [tutorial paso a paso](https://sentry.io/orgredirect/projects/new/) sobre cómo crear y configurar un proyecto de Sentry en tu entorno. Como mínimo, esto te permitirá aprovechar la cuota gratuita asignada de Tracing y Session Replay.

2. El flujo de creación de proyectos de Sentry incluye un “Source Map Wizard” para ayudarte a agilizar la carga de tus artefactos (source maps, archivos de información de depuración, símbolos).

   * Los source maps (JS) y los debug symbols son esenciales para traducir código minimizado u ofuscado a un formato legible, lo que te permite identificar las líneas exactas de código que causan los problemas. Aquí puedes encontrar instrucciones para [JavaScript](/es/platforms/javascript/sourcemaps/uploading/), así como para [Android](/es/platforms/android/enhance-errors/source-context/) e [iOS](/es/platforms/apple/guides/ios/sourcecontext/).

   <Arcade src="https://demo.arcade.software/rxKoIgKxfVE5rXhizlEL?embed" />

3. Agrega [Environments](/es/concepts/key-terms/environments/#creating-environments) y [Releases](/es/platform-redirect/?next=/configuration/releases/) a tu Sentry init.

   * Configura entornos (producción, staging, etc.) y releases en Sentry para optimizar el seguimiento de errores. Los entornos filtran los issues por contexto, mientras que las releases vinculan los errores a versiones de código para facilitar la depuración.

   <Arcade src="https://demo.arcade.software/MKxHyDhuA1edeTqEhESL?embed" />

<div id="step-2-get-your-team-on-sentry-by-implementing-sso-or-scim-for-new-organizations">
  ## Paso 2: Incorpora a tu equipo a Sentry implementando SSO o SCIM - Para organizaciones nuevas
</div>

Para gestionar la autenticación y el aprovisionamiento de usuarios, puedes configurar inicio de sesión único ([SSO](/es/organization/authentication/sso/)) y el sistema de gestión de identidades entre dominios ([SCIM](/es/organization/authentication/sso/okta-sso/okta-scim/)).

<div id="step-3-integrations">
  ## Paso 3: integraciones
</div>

[Gestión de código fuente](): Integrar tu SCM con Sentry es fundamental para resolver incidencias de forma eficiente. Vincula los cambios de código con los errores, ayudándote a identificar y resolver problemas rápidamente. Con el seguimiento de commits y los Suspect Commits, los usuarios pueden identificar cambios problemáticos y rastrearlos hasta la traza de pila relevante.

<Alert>
  Aspectos destacados de la integración de SCM (GitHub):

  * [Vinculación de trazas de pila](/es/organization/integrations/source-code-mgmt/github/#stack-trace-linking)
  * [Resaltar commits sospechosos en el detalle de una incidencia](/es/organization/integrations/source-code-mgmt/github/#suspect-commits-and-suggested-assignees)
  * [Vincular automáticamente code owners](/es/organization/integrations/source-code-mgmt/github/#code-owners)

  <Arcade src="https://demo.arcade.software/qFzrbkftPVGTGLveRedZ?embed" />
</Alert>

[Mensajería/Notificaciones](/es/organization/integrations/notification-incidents/): Una integración de mensajería te permitirá configurar alertas según la gravedad de la incidencia, resolver o ignorar problemas directamente desde Slack y volver a Sentry para una investigación más profunda. Esto mantiene a los equipos proactivos al alertarlos al instante sobre incidencias críticas.

[Seguimiento de incidencias](/es/organization/integrations/#issue-tracking): Ayuda a los equipos a rastrear y gestionar incidencias de Sentry dentro de su sistema de tickets. Podrás activar la creación automática de incidencias a partir de eventos de Sentry, sincronizar actualizaciones de estado entre herramientas y vincular incidencias de Sentry a tickets existentes para agilizar la resolución.

* Vincular Sentry con JIRA mantiene sincronizados los estados de las incidencias, marcando automáticamente como resueltas las incidencias de Sentry cuando corresponde. Esto garantiza que los problemas que reaparecen se marquen como regresiones.

<Arcade src="https://demo.arcade.software/atttrvOjh9g1x9V7c8Z9?embed" />

* Para gestionar incidencias obsoletas en Sentry, habilita la autorresolución para aquellas que no se hayan visto dentro de un período determinado. **Recomendamos configurar la autorresolución en 2 semanas** para un seguimiento óptimo de incidencias y regresiones. Configúralo en **Project Settings &gt; General Settings &gt; Auto Resolve**.

![Captura de pantalla de la configuración de Auto Resolve](./img/auto-resolve.png)

Al llegar hasta el final, ya tienes una gran ventaja para sacar el máximo partido de Sentry. Para una mayor personalización, explora la documentación oficial de Sentry o [pregunta a Sentry AI](https://sentry.zendesk.com/hc/en-us?askAI=tru).
