La sección de Gestión de datos explica cómo los clientes pueden gestionar los datos enviados a Sentry usando el SDK. Ofrecemos una sección paralela en el producto, centrada en la configuración en [sentry.io](http://sentry.io) que un usuario puede usar para controlar los datos. Un framework nuevo puede adoptar el contenido común, que se encuentra en `/docs/platforms/<SDK>/common/data-management`. Siéntete libre de revisar los archivos de ese directorio para resolver cualquier duda.

<Alert >

No cambies el contenido común. Ese cambio se propagará a todas las guías que dependen de él. En su lugar, abre un issue en GitHub con preguntas o sugerencias.

</Alert >

La mayoría de estas páginas son bastante intuitivas, así que solo se proporciona información básica. Aun así, si surge alguna duda, abre un issue en GitHub.

<div id="start-here">
  ## Comienza aquí
</div>

Determina si la página corresponde:

- Si el contenido _no corresponde_, agrega la guía a la lista `notSupported` en el frontmatter del archivo. Esto desactivará la visualización del contenido de esta guía.
- Si el contenido _sí corresponde_, agrega el archivo `include` al directorio correcto indicado para cada página. Si no se proporciona el ejemplo de código, la página mostrará un gran recuadro gris que pedirá a los usuarios que abran un issue por el ejemplo faltante.

<div id="sensitive-data">
  ### Datos confidenciales
</div>

Este archivo es `/sensitive-data/index.mdx`. Explica cómo depurar o filtrar datos dentro del SDK para que no se envíen con un evento.

Para el depurado de datos que la guía NO admite, agrega el nombre de la guía a la instrucción `PlatformSection notSupported`, lo que garantiza que el contenido no se muestre. Por ejemplo:

`<PlatformSection notSupported={["javascript.react"]}>`

Agrega el ejemplo de código a estos directorios:

- `/src/platform-includes/configuration/before-send`
- `/src/platform-includes/sensitive-data/set-tag`
- `/src/platform-includes/sensitive-data/set-user`

<div id="debug-files">
  ### Archivos de depuración
</div>

Este archivo es `/sensitive-data/debug-files.mdx`. Explica cómo funcionan los archivos de información de depuración en Sentry.

Para los controles específicos que admite la guía, agrega el nombre de la guía a la instrucción `PlatformSection supported`, lo que garantiza que se muestre el contenido.
