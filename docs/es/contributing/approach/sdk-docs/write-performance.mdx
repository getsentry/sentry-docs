La sección de rendimiento del contenido trata sobre la instrumentación de rendimiento para nuestros SDK. Un nuevo framework puede adoptar el contenido común, que se encuentra en `/docs/platforms/<SDK>/common/performance`. Puedes revisar los archivos de ese directorio para ayudarte a responder cualquier pregunta.

<Alert >

No modifiques el contenido común. Ese cambio se propagará a todas las guías que dependen de él. En su lugar, abre un issue en GitHub con tus preguntas o sugerencias.

</Alert >

<div id="start-here">
  ## Comienza aquí
</div>

Determina si la página corresponde:

- Si el contenido **no corresponde**, agrega la guía a la lista `notSupported` en el frontmatter del archivo. Esto desactivará la visualización del contenido para esta guía.
- Si el contenido **sí corresponde**, agrega el archivo `include` al directorio correcto, según se indica para cada página. Si no se proporciona el ejemplo de código, la página mostrará un gran recuadro gris solicitando que los usuarios abran un issue por la muestra faltante.

<div id="set-up">
  ### Configuración
</div>

La página de configuración es un posible punto de inicio para los lectores, aunque la nota inicial redirige al contenido de instrumentación, si corresponde. El recorrido del lector depende de si el SDK ya se instrumentó para el monitoreo de rendimiento en la página actualizada de Introducción. Para quienes adopten esta función más adelante, aquí se explica cómo configurarla.

Esperamos que este contenido aplique a todos los SDK, según corresponda.

Este archivo es `index.mdx`. Explica cómo instalar o configurar (si corresponde) y cómo habilitar el tracing.

Para desarrollar el contenido de este archivo:

1. Si la guía aún está en desarrollo y se publica como una función experimental o de acceso anticipado, agrega la plataforma al `<PlatformSection>` que envuelve la alerta correspondiente ubicada al comienzo del contenido de la página.

1. Si se espera que la guía se use en un entorno de alto tráfico, agrega la plataforma al siguiente `<PlatformSection>`, que recomienda realizar pruebas antes de desplegar en producción.

1. If la instrumentación automática aún está en vista previa/beta para la guía, agrega tu plataforma al siguiente `<PlatformSection>` que envuelve el texto que lo describe.

1. Si la instrumentación automática es compatible, agrega tu plataforma al `<PlatformSection>` inmediatamente después de **Verify**, que aconseja a los usuarios usar instrumentación automática o personalizada (o manual) para verificar la función. Omite la plataforma de la lista `supported` o agrégala a la lista `notSupported` si la instrumentación automática no está disponible.

1. Agrega un ejemplo de código en `/src/platform-includes/performance/configure-sample-rate/` que cubra cómo establecer las tasas de muestreo. Asegúrate de actualizar `/docs/platforms/<SDK>/common/configuration/sampling.mdx` y **Tracing Options** en `/docs/platforms/<SDK>/common/configuration/options.mdx`.

1. Realiza una de las siguientes acciones según si la instrumentación automática está disponible para la guía o no:
   a. Si la instrumentación automática es compatible, agrega tu plataforma al `<PlatformSection>` inmediatamente después de **Verify**, que aconseja a los usuarios usar instrumentación automática o personalizada (también llamada manual) para verificar la función. Omite la plataforma de `supported` o agrégala a `notSupported`.

   b. Si la instrumentación automática no es compatible, agrega tu plataforma al segundo `<PlatformSection>` después de **Verify**, que aconseja a los usuarios probar el tracing usando instrumentación personalizada (manual). Omite la plataforma de la lista `supported` o agrégala a la lista `notSupported` si la instrumentación automática está disponible.

1. Determina si el párrafo sobre carga de página y navegación aplica a la guía. Si es así, agrega la guía a la lista `supported` en `<PlatformSection>`. Si no, omite la guía de la lista `supported` o agrégala a la lista `notSupported`, lo que impedirá que el contenido se muestre para esta guía.

<div id="custom-instrumentation">
  ### Instrumentación personalizada
</div>

Esperamos que este contenido se aplique a todos los SDK.

Este archivo es `performance/instrumentation/custom-instrumentation.mdx`. Explica cómo instrumentar transacciones manualmente.

Si la instrumentación personalizada para el framework se ofrece como una función experimental o de acceso anticipado, añade la guía a la `<PlatformSection>` que envuelve la alerta correspondiente situada al inicio del contenido de la página.

Para desarrollar el contenido de este archivo, añade ejemplos de código en estos directorios:

- `/src/platform-includes/performance/enable-manual-instrumentation/`
- `/src/platform-includes/performance/add-spans-example/`
- `/src/platform-includes/performance/retrieve-transaction/`

Si alguno de los archivos anteriores no aplica a tu framework, ubica la `<PlatformSection>` que envuelve el `<PlatformContent>` que apunta al archivo no compatible y añade tu guía a su lista `notSupported`. Si esa `<PlatformSection>` todavía no existe, crea una y añade tu guía a su lista `notSupported`.

Luego, decide si debes incluir un ejemplo de código que muestre cómo conectar errores y spans. Si es así, añade el SDK a la lista `Supported` en la `<PlatformSection>` correspondiente y luego añade un ejemplo de código en `/src/platform-includes/performance/connect-errors-spans/`.

<div id="automatic-instrumentation">
  ### Instrumentación automática
</div>

Llena el directorio `sdk_name/common/performance/instrumentation/` con el archivo `automatic-instrumentation.mdx`.

<div id="troubleshooting">
  ### Solución de problemas
</div>

Este archivo es `troubleshooting-performance.mdx`. Explica casos extremos y cómo resolver problemas conocidos.

1. Determina si el framework necesita un ejemplo de cómo agrupar transacciones. Si es así, agrega el framework a la lista `Supported` en `<PlatformSection>` y luego agrega un ejemplo de código en `performance/group-transaction-example/`. Si no, agrega el framework a la lista `notSupported`, lo que evitará que se muestre el encabezado **Group Transactions** para ese framework.

2. Agrega un ejemplo en `src/platform-includes/performance/control-data-truncation/`
