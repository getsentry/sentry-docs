La sección de Enriquecimiento de eventos explica cómo los clientes pueden añadir contexto a los datos enviados a Sentry. Un framework nuevo puede adoptar el contenido común, que se encuentra en `/docs/platforms/<SDK>/common/enriching-events`. Si quieres, puedes revisar los archivos de ese directorio para resolver cualquier duda.

<Alert >

No cambies el contenido común. Ese cambio se propagará a todas las guías que dependen de él. En su lugar, abre un issue en GitHub con preguntas o sugerencias.

</Alert >

La mayoría de estas páginas se explican por sí solas, por lo que solo se proporciona información básica. Aun así, no dudes en abrir un issue en GitHub para hacer preguntas.

<div id="start-here">
  ## Comienza aquí
</div>

Determina si la página corresponde:

- Si el contenido _no corresponde_, agrega la guía a la lista `notSupported` en el frontmatter del archivo. Esto desactivará la visualización del contenido de esta guía.
- Si el contenido _sí corresponde_, agrega el archivo `include` al directorio correcto según se indique para cada página. Si no se proporciona el ejemplo de código, la página mostrará un gran cuadro gris solicitando a los usuarios que abran un issue por la muestra faltante.
- La familia de contenidos de JS necesita importar el SDK para que los ejemplos de código tengan sentido en cada página. Agrega la instrucción de importación en `/src/platform-includes/enriching-events/enriching-events/import`.

<div id="add-context">
  ### Añadir contexto
</div>

Este archivo es `context.mdx`. Explica cómo habilitar contextos personalizados. Añade el ejemplo de código a este directorio:

- `/src/platform-includes/enriching-events/set-context/`

Si la guía puede pasar contexto directamente, añádelo a la lista de guías compatibles justo encima de "Passing Context Directly".

<div id="identify-users">
  ### Identificar usuarios
</div>

Este archivo es `identify-user.mdx`. Explica cómo identificar al usuario. Añade los ejemplos de código en estos directorios:

- `/src/platform-includes/enriching-events/set-user/`
- `/src/platform-includes/enriching-events/unset-user/`

<div id="set-transaction-name">
  ### Configurar el nombre de la transacción
</div>

Este archivo es `transaction-name.mdx`. Explica cómo sobrescribir el nombre de la transacción. Agrega el ejemplo de código a este directorio:

- `/src/platform-includes/enriching-events/set-transaction-name/`

Si la guía puede controlar el inicio y la detención de transacciones, agrégala a la lista de guías compatibles que remite a los clientes a nuestra documentación de trazas.

<div id="customize-tags">
  ### Personalizar etiquetas
</div>

Este archivo es `tags.mdx`. Explica cómo personalizar las etiquetas de un evento. Añade el ejemplo de código a este directorio:

- `/src/platform-includes/enriching-events/set-tag/`

Si la guía no asigna etiquetas al alcance actual, añádela a la lista de guías que no admiten esta opción.

<div id="attachments">
  ### Archivos adjuntos
</div>

Este archivo es `/attachments/index.mdx`. Explica cómo adjuntar archivos junto con el evento. Agrega el ejemplo de código a este directorio:

- `/src/platform-includes/enriching-events/add-attachment/`

Si la guía es parte de la familia nativa, agrégala a la lista de guías que admiten contenido nativo en relación con:

- Archivos de información de depuración y compatibilidad integrada para bloqueos nativos
- Informes de bloqueos de gran tamaño
- La sección "Informes de bloqueos y privacidad"
- El párrafo sobre limitar los informes de bloqueo por incidencia

<div id="breadcrumbs">
  ### Migas de pan
</div>

Este archivo es `breadcrumbs.mdx`. Explica las migas de pan manuales. Agrega el ejemplo de código para registrar manualmente una miga de pan en este directorio:

- `/src/platform-includes/enriching-events/breadcrumbs/breadcrumbs-example/`
- `/src/platform-includes/enriching-events/breadcrumbs/before-breadcrumb/`

Un matiz:

- Actualiza, según corresponda, la lista de guías compatibles en `PlatformSection supported`, que precede a "The available breadcrumb keys..."

<div id="user-feedback">
  ### Comentarios de usuarios
</div>

Este archivo es `user-feedback.mdx`. Explica cómo usar el widget de JS insertable o la API.

- Actualiza, según corresponda, la lista de SDK compatibles en `PlatformSection supported` que precede a "User Feedback API".
  Luego, agrega el ejemplo de código al directorio `src/platform-includes/user-feedback/sdk-api-example`.
- Para la familia .NET, si la API de comentarios de usuario tiene una integración específica, agrégala a `PlatformSection supported`, que precede a "Use the .NET SDK", y añade el enlace correspondiente.
- Actualiza, según corresponda, los SDK que no admiten el widget de JS en `PlatformSection notSupported`, que precede a "Embeddable JavaScript Widget".
- Para los SDK que usan el widget de JS, agrega el ejemplo de código a `/src/platform-includes/user-feedback/user-feedback-example-widget/`.
