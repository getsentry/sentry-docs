<Alert type="info">
  Esta página describe el formato heredado de «Primeros pasos». Los nuevos SDK y
  las guías que se hayan actualizado deben usar el [formato de guía de inicio
  rápido](/es/contributing/approach/sdk-docs/write-quick-start/).
</Alert>

Una nueva plataforma o SDK debería basar su página de Primeros pasos en la de un SDK existente similar.

A continuación, se presenta una descripción general de los componentes de una página de Primeros pasos.

<div id="configyml">
  ## config.yml
</div>

Este archivo se encuentra en el directorio `/docs/platform/<SDK_name>/`. Un ejemplo:

```yaml
title: JavaScript
caseStyle: camelCase
supportLevel: production
sdk: "sentry.javascript.browser"
categories:
  - browser
```

<div id="primer-content">
  ## Contenido introductorio
</div>

Explica el SDK de forma general en estas dos partes:

1. Una frase en cursiva que indique qué permite el SDK (errores, o errores y transacciones)
2. Un breve párrafo con información técnica y las versiones compatibles

   <Alert>
     Si el contenido introductorio no está desarrollado, se mostrará este contenido de reserva: &quot;*Los SDK de Sentry permiten el reporte automático de errores y excepciones*.&quot;
   </Alert>

Agrega el contenido introductorio a este directorio:

* [`/platform-includes/getting-started-primer/`](https://github.com/getsentry/sentry-docs/tree/master/platform-includes/getting-started-primer)

<div id="but-thats-not-all">
  ## Pero eso no es todo
</div>

El contenido de Introducción también lo utiliza nuestro asistente de incorporación en la aplicación para que los clientes pongan en marcha Sentry. Mantener todo el contenido actualizado es fundamental.

**Si actualizas los ejemplos de código de un SDK, actualiza también el asistente.**

<div id="update-the-wizard">
  ### Actualiza el asistente
</div>

El asistente dentro de la aplicación es donde los clientes implementan por primera vez nuestros SDK. No es un simple copiar y pegar de Getting Started por dos razones:

1. Los clientes nuevos carecen del contexto que brindan nuestros Docs (piénsalo así: no hay barra lateral izquierda para consultar).
2. Es probable que los clientes con experiencia tengan las alertas y demás desactivadas, así que no esperes un flujo de trabajo claro. Además, no revises el flujo. Indica a los clientes que vayan al producto para hacer un tour.

Como resultado, un simple copiar y pegar de Getting Started no basta. Consulta el asistente de JavaScript como ejemplo.
