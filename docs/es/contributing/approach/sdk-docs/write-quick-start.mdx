Los nuevos SDK deben usar el formato de la guía de inicio rápido descrito en esta página. Es posible que todavía encuentres guías de introducción para SDK existentes.

<Alert type="info" title="Nota">
  Si quieres actualizar una guía de introducción existente sin refactorizarla
  a una guía de inicio rápido, consulta [Cómo escribir: Introducción](/es/contributing/approach/sdk-docs/write-getting-started/). Si no estás
  seguro de qué tipo de guía estás viendo, envía tu pregunta [abriendo un
  issue en GitHub](https://github.com/getsentry/sentry-docs/issues/new/choose).
</Alert>

<Expandable type="info" title="¿Por qué el cambio?">
  Este cambio surge de nuestro compromiso de seguir las mejores prácticas y hacer que nuestra documentación sea lo más accesible posible.

  Con las nuevas guías de inicio rápido, queremos ofrecer un enfoque más estructurado y focalizado, ayudando a quienes escriben a crear contenido valioso con confianza y, en última instancia, ayudando a las personas usuarias a alcanzar sus objetivos con Sentry.

  **Inicio rápido vs. Introducción:**
  | | Inicio rápido | Introducción |
  | -------- | ----------------------------------------------------------------------------------- | --------------------------------------------------------------------- |
  | **Enfoque** | Orientado a objetivos, acción inmediata, pasos mínimos | Orientado al aprendizaje, conceptos más amplios, información contextual |
  | **Contenido** | Destaca los pasos críticos con menos énfasis en el cómo y el por qué | Más explicativa y con información contextual |
  | **Audiencia** | Desarrolladores que quieren comenzar rápidamente | Desarrolladores que buscan una comprensión más profunda |
  | **Formato** | Página única; concisa | A menudo abarca varias páginas; exhaustiva |
</Expandable>

Nuestras guías de inicio rápido buscan:

* Permitir que las personas desarrolladoras integren Sentry rápidamente con funcionalidades como Error Monitoring, Tracing y Session Replay
* Mantener las instrucciones concisas, evitando explicaciones o información contextual innecesarias
* Centrarse en la acción, incluyendo ejemplos de código y comandos
* Lograr una configuración funcional de Sentry con resultados visibles en el proyecto de Sentry de la persona usuaria
* Servir como base para una personalización posterior

<div id="structuring-the-configyml-file">
  ## Estructuración del archivo config.yml
</div>

El archivo config.yml proporciona detalles a nivel de SDK, como su título, nivel de soporte y categorías de dispositivos. Este archivo se encuentra en el directorio `/docs/platform/<SDK_name>/`. Un ejemplo:

```yaml
title: JavaScript
caseStyle: camelCase
supportLevel: production
sdk: "sentry.javascript.browser"
categories:
  - browser
```

<div id="structure-templates">
  ## Plantillas de estructura
</div>

Hemos creado dos plantillas de guía de inicio rápido: una para la configuración mediante asistente y otra para la configuración manual. Úsalas como base y añade nuevas secciones (pasos) según sea necesario.

Si un SDK tiene ambas guías de inicio rápido, asegúrate de que ambas conduzcan al mismo resultado; es decir, que las modificaciones en el proyecto del desarrollador sean coherentes (dentro de lo razonable).

<div id="wizard-setup">
  ### Configuración con el asistente
</div>

El asistente es la forma más rápida de configurar Sentry; tu guía debería reflejarlo apoyándose en el asistente para la configuración principal y aportando información adicional cuando tenga sentido y ayude al desarrollador. Puedes incluir estas ampliaciones en componentes `Expandable` para ocultarlas de forma predeterminada y despejar visualmente la página.

Recomendamos basar tu guía en una página de SDK existente similar, como:

* [Guía de inicio rápido de Next.js: configuración con asistente](/es/platforms/javascript/guides/nextjs/)

**Estructura mínima:**

* **Título y resumen**
  * El título se toma automáticamente del archivo config.yml
  * Describe qué logrará el desarrollador al seguir esta guía
* **Requisitos previos**
* **Paso 1: Instalación**
  * Asegúrate de que &quot;Instalación&quot; sea siempre el primer paso para que los desarrolladores pasen a la acción rápidamente
  * Proporciona el o los métodos de instalación principales para este SDK (documentamos métodos alternativos en una página aparte, como en este [ejemplo](/es/platforms/javascript/install/))
  * Presenta brevemente las funciones que el asistente puede habilitar; usa un componente `Expandable` para ocultar el contenido de forma predeterminada
  * Enumera los cambios que el asistente aplica en la aplicación del desarrollador, como actualizar archivos de configuración y crear código de ejemplo para probar la configuración más adelante; usa un componente `Expandable` para ocultar el contenido de forma predeterminada
* **Paso 2: Verifica tu configuración**
  * Explica cómo probar la configuración. Por ejemplo, indica al desarrollador que use los ejemplos creados por el asistente
  * **Subsección: Ver datos capturados en Sentry**
    * Indica a los desarrolladores que revisen los errores capturados en su proyecto de Sentry
    * Opcional: Proporciona instrucciones detalladas usando un `<Expandable>`
* **Próximos pasos**
  * Cierra y resume lo que el/la desarrollador(a) logró
  * Proporciona una lista de enlaces a los próximos temas recomendados

<div id="manual-setup">
  ### Configuración manual
</div>

Usa esta plantilla para tus guías de configuración manual. Mantén tus instrucciones concisas y pregúntate: «¿El desarrollador necesita saber esto para completar la guía con éxito o es un tema que puede consultar más adelante en la documentación?»

Recomendamos basar tu guía en una página de SDK existente similar, como:

* [Configuración manual de la guía de inicio rápido de Next.js](/es/platforms/javascript/guides/nextjs/manual-setup/)

**Estructura mínima:**

* **Título y resumen**
  * El título se toma automáticamente del archivo config.yml
  * Describe qué logrará el desarrollador al seguir esta guía
* **Requisitos previos**
* **Paso 1: Install**
  * Asegúrate de que «Install» sea siempre el primer paso para que los desarrolladores se pongan manos a la obra rápidamente
  * Incluye opciones de onboarding (configuración de funcionalidades) aquí antes de las instrucciones de instalación (usa el componente [`OnboardingOption`](/es/contributing/pages/components/#onboarding-options))
    * **Importante**: Usa el componente `OnboardingOption` para alternar la visibilidad de las instrucciones específicas de cada funcionalidad en la guía. Por ejemplo, si un desarrollador quiere configurar Tracing pero no Session Replay, no debería recibir instrucciones sobre cómo configurar Session Replay.
  * Explica brevemente estas funcionalidades; usa un componente `Expandable` para ocultar el contenido de forma predeterminada
  * Proporciona el/los método(s) principal(es) de instalación para este SDK (documentamos métodos alternativos en una página aparte, como en este [ejemplo](/es/platforms/javascript/install/))
* **Paso 2: Configure**
  * Proporciona ejemplos de configuración y añade comentarios en el ejemplo de código para reducir el contenido del texto circundante
  * Muestra cómo configurar Sentry y las funcionalidades seleccionadas
  * Usa subsecciones para organizar lógicamente y dividir el contenido extenso
* **Paso 3 - Paso X: Instrucciones específicas del SDK**
  * Crea pasos para source maps, tunneling u otra instrumentación necesaria
  * Cada tema principal debe ser un paso independiente
* **Paso Y: Verify Your Setup**
  * Proporciona ejemplos de código de verificación para este SDK
  * Adapta esta sección a las funcionalidades de la opción de onboarding seleccionada. Por ejemplo, si Tracing está habilitado, debes describir cómo verificar que está configurado correctamente
  * **Subsección: Ver datos capturados en Sentry**
    * Indica a los desarrolladores que revisen los errores capturados en su proyecto de Sentry
    * Opcional: Proporciona instrucciones detalladas usando un `<Expandable>`
* **Siguientes pasos**
  * Cierra y resume lo que logró el desarrollador
  * Proporciona una lista de enlaces a los próximos temas recomendados

<div id="content-placement-and-reusability">
  ## Ubicación y reutilización del contenido
</div>

Si el contenido que quieres incluir aún no existe, crea un archivo nuevo para tu SDK en la carpeta correspondiente en [`/src/platform-includes/`](https://github.com/getsentry/sentry-docs/tree/master/platform-includes).

**Ejemplos:**

* **Instalación** [`/platform-includes/getting-started-install/`](https://github.com/getsentry/sentry-docs/tree/master/platform-includes/getting-started-install)
* **Configuración** [`/platform-includes/getting-started-config/`](https://github.com/getsentry/sentry-docs/tree/master/platform-includes/getting-started-config)
* **Verificación** [`/src/platform-includes/getting-started-verify/`](https://github.com/getsentry/sentry-docs/tree/master/platform-includes/getting-started-verify)

Integra este contenido en tu guía usando el componente `PlatformSection`.

<Alert type="info">
  Si el SDK admite el monitoreo de rendimiento, agrégalo a la lista que enlaza de vuelta
  al contenido del SDK desde Product, almacenada en
  [`/docs/product/insights/getting-started.mdx`](https://github.com/getsentry/sentry-docs/blob/master/docs/product/insights/getting-started.mdx).
</Alert>

<div id="additional-requirements-for-our-wizard">
  ## Requisitos adicionales para nuestro asistente
</div>

El contenido de Inicio rápido también es utilizado por nuestro asistente de incorporación en la aplicación para ayudar a los usuarios a comenzar a usar Sentry. Mantener todo el contenido actualizado es fundamental.

Importante:

* Si actualizas los ejemplos de código de un SDK, actualiza también el asistente
* Si agregas algo nuevo al asistente, como una configuración adicional, actualiza también la guía de configuración manual

<div id="update-the-wizard">
  ### Actualiza el asistente
</div>

El asistente dentro de la aplicación es donde los desarrolladores integran por primera vez nuestros SDK. No es un simple copiar y pegar del Quick Start por dos razones:

1. Los usuarios nuevos no tienen el contexto que ofrecen nuestros Docs (piénsalo así: no hay una barra lateral izquierda que revisar).
2. Es probable que los usuarios con experiencia tengan las alertas y demás desactivadas, así que no esperes un flujo de trabajo único y claro. Además, no revises el flujo de trabajo. Indica a los usuarios que vayan al producto para hacer un recorrido.

Como resultado, un simple copiar y pegar del Quick Start no bastará. Consulta el [asistente de Next.js](https://github.com/getsentry/sentry-wizard/tree/master/src/nextjs) como ejemplo.
