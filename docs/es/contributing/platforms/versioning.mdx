title: Versionado

El versionado permite mantener varias versiones de la misma documentación, lo cual es crucial para dar soporte a diferentes versiones del SDK. Esta guía explica cómo crear documentación versionada en la plataforma de documentación.

<div id="creating-a-versioned-page">
  ## Creación de una página con versión
</div>

Para crear una página con versión:

1. Asegúrate de que la página original ya exista.
2. Crea un archivo nuevo con el mismo nombre que el original, pero añade `__v{VERSION}` (dos guiones bajos) al nombre del archivo antes de la extensión.

**La versión debe ajustarse a uno de estos dos formatos:**

* Solo versión mayor (para rangos de versión generales): `__v{MAJOR}.x`
* Versión semántica completa (para versiones específicas): `__v{MAJOR}.{MINOR}.{PATCH}`

Por ejemplo:

```
index.mdx (original)
index__v7.x.mdx (versión 7.x.x)
index__v7.1.0.mdx (versión específica 7.1.0)
```


<div id="versioning-platform-includes">
  ## Versionado de los includes de la plataforma
</div>

Las páginas de la carpeta `platform-includes` se pueden versionar usando el mismo método. Sin embargo, la página raíz que hace referencia a estos includes también debe versionarse.

<div id="file-structure-example">
  ## Ejemplo de estructura de archivos
</div>

```
docs/
├── primeros-pasos/
│   ├── index.mdx
│   └── index__v7.x.mdx
└── inclusiones-de-plataforma/
    └── configuración/
        ├── ejemplo.mdx
        └── ejemplo__v7.x.mdx
```


<div id="version-selection-and-user-experience">
  ## Selección de versión y experiencia de usuario
</div>

Cuando existen varias versiones de una página:

1. Aparece un menú desplegable debajo del selector de plataforma que muestra todas las versiones disponibles para el archivo actual.

2. Cuando una persona usuaria selecciona una versión:

- Se le redirige a la versión elegida de la página.
- Se guarda una preferencia en localStorage, específica de la guía o plataforma actual.

3. Se muestra una alerta al ver páginas que no son la más reciente para informar que no están en la versión actual.

4. Si una persona usuaria tiene una preferencia de versión guardada:

- La app realizará una redirección del lado del cliente a esa versión siempre que esté disponible para otras páginas.

Este enfoque garantiza que las personas usuarias puedan navegar fácilmente entre versiones y mantener su preferencia de versión en toda la documentación.

<div id="best-practices">
  ## Mejores prácticas
</div>

1. Solo añade versionado a las páginas cuando sea necesario para evitar duplicar contenido y reducir los tiempos de compilación.
2. Asegúrate de que todo el contenido relacionado (incluidos los `platform-includes`) esté versionado de forma coherente.

<div id="limitations-and-considerations">
  ## Limitaciones y consideraciones
</div>

- El versionado solo está disponible para las páginas existentes.
- El sistema de versionado depende de la convención de nombres de los archivos, por lo que es importante seguir exactamente el formato `__v{VERSION}`.

Siguiendo estas pautas, es posible gestionar eficazmente múltiples versiones de la documentación y garantizar que los usuarios accedan a la información adecuada para la versión específica de su software o API.
