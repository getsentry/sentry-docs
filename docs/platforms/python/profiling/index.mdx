## Enable Profiling

<Note>

Python profiling is stable starting in SDK version `1.18.0`.

</Note>

<SignInNote />

```python
import sentry_sdk

def profiles_sampler(sampling_context):
    # ...
    # return a number between 0 and 1 or a boolean

sentry_sdk.init(
    dsn="___PUBLIC_DSN___",
    traces_sample_rate=1.0,

    # To set a uniform sample rate
    # Set profiles_sample_rate to 1.0 to profile 100%
    # of sampled transactions.
    # We recommend adjusting this value in production,
    profiles_sample_rate=1.0,

    # Alternatively, to control sampling dynamically
    profiles_sampler=profiles_sampler
)
```

<Note>

The <PlatformIdentifier name="profiles_sample_rate" /> setting is _relative_ to the <PlatformIdentifier name="traces_sample_rate" /> setting.

Profiling requires Sentryâ€™s performance monitoring to be enabled via `traces_sample_rate` in the example below. Read our <PlatformLink to="/performance/">performance setup documentation</PlatformLink> to learn how to configure sampling. Setting the sample rate to 1.0 means all transactions will be captured.

</Note>
