## Create Transaction Without Binding to The Current Scope

To avoid passing `ITransaction` objects to method calls when creating child spans, by default started transaction is bound to the current scope.

When automatic binding is not desired, transactions can be started without binding to the current scope with an overloaded `startTransaction` method. In such case, `Sentry#getSpan` returns `null` and transaction has to be passed to methods manually to create child span:

```java {tabTitle:Java}
import io.sentry.ISpan;
import io.sentry.ITransaction;
import io.sentry.Sentry;

void executeTask() {
  ITransaction transaction = Sentry.startTransaction("transaction-name", "task", false);

  executeSubTask(transaction);

  transaction.finish();
}

private void executeSubTask(ISpan span) {
  ISpan childSpan;
  if (span == null) {
    childSpan = Sentry.startTransaction("transaction-name", "sub-task");
  } else {
    childSpan = span.startChild("operation");
  }
  // ...
  childSpan.finish();
}
```

```kotlin {tabTitle:Kotlin}
import io.sentry.ISpan
import io.sentry.ITransaction
import io.sentry.Sentry

fun executeTask() {
  val transaction = Sentry.startTransaction("transaction-name", "task")

  executeSubTask(transaction)

  transaction.finish()
}

private fun executeSubTask(span: ISpan?) {
  val childSpan = if (span == null) {
    Sentry.startTransaction("transaction-name", "sub-task")
  } else {
    span.startChild("operation")
  }
  // ...
  childSpan.finish()
}
```
