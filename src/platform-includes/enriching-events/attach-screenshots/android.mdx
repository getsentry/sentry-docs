### Customize Screenshot Capturing

<Note>

Requires SDK version `6.24.0` or higher.

</Note>

On Android capturing screenshots can be an resource-intense operation, thus screenshot capturing is limited to one screenshot every 2 seconds using a debouncing mechanism. This behavior can be overruled by supplying a `BeforeCaptureCallback` for screenshots in the `SentryAndroidOptions`.

The `BeforeCaptureCallback` also allows you to customize the behavior based on event data, giving you full control over when a screenshot should be captured and when not.

For example, screenshot capturing can be limited to only crashed events:

```java
import io.sentry.android.core.SentryAndroid;

SentryAndroid.init(this, options -> {
    options.setBeforeScreenshotCaptureCallback((event, hint, debounce) -> {
        if (event.isCrashed()) {
            return true;
        }
        return false;
    });
});
```

```kotlin
import io.sentry.android.core.SentryAndroid

SentryAndroid.init(this) { options ->
    options.setBeforeScreenshotCaptureCallback { event, hint, debounce ->
        if (event.isCrashed) {
            return@setBeforeScreenshotCaptureCallback true
        }
        return@setBeforeScreenshotCaptureCallback false
    }
}
```
