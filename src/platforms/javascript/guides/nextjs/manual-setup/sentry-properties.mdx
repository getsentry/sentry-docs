## Properties File

You can create a `sentry.properties` file in the base directory of your project, as in the [setup step](/platforms/javascript/guides/nextjs/#setup):

```properties {filename:sentry.properties}
defaults.url=https://sentry.io/
defaults.org=___ORG_SLUG___
defaults.project=___PROJECT_SLUG___
auth.token=___TOKEN___
# cli.executable=../path/to/bin/sentry-cli
```

## Environment Variables

Using environment variables allows you more exhaustive control over the Sentry parameters the [Webpack Plugin](https://github.com/getsentry/sentry-webpack-plugin) needs, and removes the need for a properties file.
If you choose to combine environment variables with a properties file, the environment variables will take precedence over parameters set in the properties file.

| Property name      | Environment variable |
| ------------------ | -------------------- |
| `defaults.url`     | `SENTRY_URL`         |
| `defaults.org`     | `SENTRY_ORG`         |
| `defaults.project` | `SENTRY_PROJECT`     |
| `auth.token`       | `SENTRY_AUTH_TOKEN`  |

You can also set the Sentry DSN using the environment variable `SENTRY_DSN` or `NEXT_PUBLIC_SENTRY_DSN` instead of having it in the [Initialization Config Files](/platforms/javascript/guides/nextjs/manual-setup/manual-config/#set-initialization-config-files).
