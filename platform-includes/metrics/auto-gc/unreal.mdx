</Alert>

The Sentry SDK can automatically track garbage collection pause durations. GC pauses are a common source of hitches in Unreal Engine games — a single GC pass can freeze the game thread for 10–50ms or more, causing visible stutters. When enabled, the SDK measures the blocking GC pause and emits it as a metric after each collection cycle.

| Metric | Type | Description |
|--------|------|-------------|
| `game.perf.gc_time` | Distribution (ms) | Duration of the blocking GC pause |

The metric captures the interval between the GC lock being acquired and released — the actual game-thread-blocking pause that impacts gameplay. By default, the engine runs GC approximately every 60 seconds (controlled by the `gc.TimeBetweenPurgingPendingKillObjects` console variable), so this metric has negligible impact on metric volume.

### Enabling

Enable GC pause metrics in **Project Settings > Plugins > Sentry > Metrics > Experimental**:

1. Make sure **Enable Metrics** is checked
2. Check **Collect GC pause metrics**

Or configure programmatically:

```cpp
SentrySubsystem->InitializeWithSettings(FConfigureSettingsNativeDelegate::CreateLambda([=](USentrySettings* Settings)
{
    Settings->EnableMetrics = true;
    Settings->EnableGCMetrics = true;
}));
```

### Metric Attributes

The same attributes as frame time metrics are attached to GC pause metrics for consistent filtering:

| Attribute | Description |
|-----------|-------------|
| `gpu.name` | GPU model name |
| `cpu.cores` | Number of physical CPU cores |
| `ram.gb` | Total physical RAM in GB |
| `res.x`, `res.y` | Screen resolution |
| `map` | Current game map/level name |
