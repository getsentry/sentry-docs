Once the SDK is initialized, you can send metrics using the `Sentry.metrics` APIs.

The `metrics` namespace exposes methods for capturing different types of metric information: `count`, `gauge`, and `distribution`.

### Counter

Use `count` to track an incrementing value, such as the number of times a button was clicked or a function was called.

```dart
import 'package:sentry/sentry.dart';

Sentry.metrics.count('button_click', 1);
```

### Gauge

Use `gauge` to track a value that can go up and down, such as the current memory usage or the number of items in a queue.

```dart
import 'package:sentry/sentry.dart';

Sentry.metrics.gauge('queue_depth', 42);
```

### Distribution

Use `distribution` to track the distribution of a value, such as the response time of a request.

```dart
import 'package:sentry/sentry.dart';

Sentry.metrics.distribution('response_time', 187.5);
```

### Adding Attributes

You can pass additional attributes to any of the metric methods. Attributes allow you to filter and group metrics.

```dart
import 'package:sentry/sentry.dart';

Sentry.metrics.distribution(
  'page_load',
  1.0,
  unit: 'millisecond',
  attributes: {
    'browser': SentryAttribute.string('Firefox'),
    'region': SentryAttribute.string('us-west'),
  },
);
```

### Specifying Units

For `gauge` and `distribution` metrics, you can specify a unit using the `unit` parameter. This helps Sentry display the metric value in a human-readable format.

```dart
import 'package:sentry/sentry.dart';

Sentry.metrics.distribution(
  'response_time',
  187.5,
  unit: 'millisecond',
);

Sentry.metrics.gauge(
  'memory_usage',
  1024,
  unit: 'byte',
);
```
