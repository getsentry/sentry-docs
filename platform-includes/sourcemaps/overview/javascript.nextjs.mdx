### Manual Configuration

If you installed the SDK manually or the wizard didn't complete, configure source map uploads:

<OrgAuthTokenNote />

**Step 1:** Add your auth token to your environment (and to your CI):

```bash {filename:.env.local}
SENTRY_AUTH_TOKEN=___ORG_AUTH_TOKEN___
```

**Step 2:** Configure `withSentryConfig` in your Next.js config:

```javascript {filename:next.config.js}
const { withSentryConfig } = require("@sentry/nextjs");

module.exports = withSentryConfig(
  {
    // your existing Next.js config
  },
  {
    org: "___ORG_SLUG___",
    project: "___PROJECT_SLUG___",
    authToken: process.env.SENTRY_AUTH_TOKEN,
  }
);
```

### Common Options

| Option | Default | Use When |
|--------|---------|----------|
| `sourcemaps.deleteSourcemapsAfterUpload` | `true` | Set `false` to keep source maps in your build output |
| `sourcemaps.ignore` | `[]` | Exclude files from upload (e.g., `["**/node_modules/**"]`) |
| `widenClientFileUpload` | `false` | Some client frames show `[native code]` â€” enables uploading dependency source maps |

See <PlatformLink to="/configuration/build/#source-maps-options">Build Options</PlatformLink> for all available options.

### Webpack vs Turbopack

| Bundler | Upload Timing | Requirements |
|---------|--------------|--------------|
| **Webpack** | During build (default) | None |
| **Turbopack** | After build | `@sentry/nextjs@10.13.0+`, `next@15.4.1+` |

Turbopack uses the `runAfterProductionCompile` hook from Next.js to upload after the build completes.

### Troubleshooting

| Problem | Solution |
|---------|----------|
| Stack traces show minified code | Verify `SENTRY_AUTH_TOKEN` is set in CI |
| Some frames show `[native code]` | Enable `widenClientFileUpload: true` |
| Uploads failing in CI | Check auth token permissions and network access |

See <PlatformLink to="/sourcemaps/troubleshooting_js/">Troubleshooting Source Maps</PlatformLink> for detailed debugging steps.
