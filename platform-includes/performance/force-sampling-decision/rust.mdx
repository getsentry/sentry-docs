An explicit decision set with `ctx.set_sampled(true)` is honored by the fixed-rate strategy from `.traces_sample_rate(...)` (including `0.0`). It is not applied automatically when using `.traces_sampler(...)`; that callback must read `ctx.sampled()` itself. With the default disabled strategy, the transaction is not sent even if you force a positive decision, so configure a non-disabled strategy first:

```rust
let _guard = sentry::init(sentry::ClientOptions::new().traces_sample_rate(1.0));

let mut ctx = sentry::TransactionContext::new("transaction", "op");
ctx.set_sampled(true);

let transaction = sentry::start_transaction(ctx);
```
